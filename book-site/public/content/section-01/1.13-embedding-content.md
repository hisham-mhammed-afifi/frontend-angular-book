# 1.13 Embedding Content

> HTML provides elements for embedding external documents, media, and interactive content within a page. Understanding `<iframe>`, `<embed>`, and `<object>` along with their security, performance, and accessibility implications is essential for production web development.

---

## The `<iframe>` Element

The `<iframe>` element embeds an entire HTML document inside the current page. Each iframe creates a separate browsing context with its own DOM, JavaScript execution environment, and network requests.

```html
<iframe src="https://example.com/widget"
        title="Interactive widget"
        width="600"
        height="400"
        loading="lazy">
</iframe>
```

### Core Attributes

| Attribute | Description |
|-----------|-------------|
| `src` | URL of the document to embed |
| `srcdoc` | Inline HTML string for the frame's content. Overrides `src` if both are present. |
| `name` | Targetable name for the browsing context (usable in `<a target>`, `<form target>`) |
| `width` | Width in CSS pixels |
| `height` | Height in CSS pixels |
| `loading` | `"eager"` (default) or `"lazy"` for deferred loading |
| `sandbox` | Applies security restrictions (see Sandbox section) |
| `allow` | Permissions Policy for browser features (see Permissions section) |
| `referrerpolicy` | Controls what Referer header is sent when loading the resource |

### The `name` Attribute

The `name` attribute creates a targetable browsing context. Links and forms can load their responses into a named iframe.

```html
<iframe name="preview-frame" src="about:blank" width="800" height="600"></iframe>

<!-- Clicking this link loads the page into the iframe above -->
<a href="https://example.com/page" target="preview-frame">Open in Preview</a>

<!-- Submitting this form loads the response into the iframe -->
<form action="/api/search" target="preview-frame">
  <input type="text" name="q">
  <button type="submit">Search</button>
</form>
```

### The `referrerpolicy` Attribute

Controls what referrer information is sent when loading the iframe's content.

| Value | Behavior |
|-------|----------|
| `no-referrer` | No Referer header sent |
| `origin` | Only the origin (scheme + host + port) |
| `strict-origin` | Origin only for same-security level, nothing on downgrade |
| `strict-origin-when-cross-origin` | Full URL same-origin, origin cross-origin (default) |
| `unsafe-url` | Full URL always (privacy risk) |

## The `srcdoc` Attribute

`srcdoc` accepts an HTML string that defines the iframe's document inline, without fetching an external resource.

```html
<iframe srcdoc="<h1>Hello</h1><p>This content is inline.</p>"
        title="Inline content demo"
        width="400"
        height="200">
</iframe>
```

### Encoding Requirements

Since the value is inside an HTML attribute, double quotes must be entity-encoded and ampersands must be escaped:

```html
<iframe srcdoc="<p style=&quot;color: red&quot;>Styled text &amp; more</p>"
        title="Styled inline content">
</iframe>
```

### `srcdoc` Overrides `src`

When both attributes are present, `srcdoc` takes precedence. The `src` attribute serves as a fallback for browsers that do not support `srcdoc` (all modern browsers support it, so this is mainly for extremely old browsers).

```html
<iframe srcdoc="<p>Modern content</p>"
        src="fallback.html"
        title="Content with fallback">
</iframe>
```

### Security Warning

`srcdoc` is a significant XSS vector if user input is injected without sanitization. The browser parses the attribute value as full HTML, including script execution.

```html
<!-- DANGEROUS: user input directly in srcdoc -->
<iframe srcdoc="<p>User said: UNSANITIZED_INPUT_HERE</p>"></iframe>

<!-- SAFE: always sanitize input, and combine with sandbox -->
<iframe srcdoc="<p>User said: SANITIZED_VALUE</p>"
        sandbox="allow-scripts">
</iframe>
```

Always sanitize content with a library like DOMPurify before inserting it into `srcdoc`. Always combine `srcdoc` with the `sandbox` attribute.

## The `sandbox` Attribute

The `sandbox` attribute applies security restrictions to the iframe's content. When present with no value, it imposes maximum restrictions: no scripts, no form submission, no popups, no top navigation, and the content is treated as originating from a unique (opaque) origin.

```html
<!-- Maximum restrictions: everything blocked -->
<iframe src="https://untrusted.example.com" sandbox title="Sandboxed content"></iframe>
```

Each `allow-*` token removes a specific restriction. Tokens are space-separated.

### Token Reference

| Token | What It Unlocks |
|-------|-----------------|
| `allow-scripts` | JavaScript execution |
| `allow-same-origin` | Content treated as its actual origin (not a unique origin) |
| `allow-forms` | Form submission |
| `allow-popups` | `window.open()`, `target="_blank"` |
| `allow-popups-to-escape-sandbox` | Opened popups are not themselves sandboxed |
| `allow-modals` | `alert()`, `confirm()`, `prompt()`, `print()` |
| `allow-top-navigation` | Can navigate the top-level browsing context |
| `allow-top-navigation-by-user-activation` | Can navigate top-level, but only from a user gesture |
| `allow-downloads` | File downloads via `<a download>` |
| `allow-presentation` | Presentation API |
| `allow-orientation-lock` | Screen orientation lock |
| `allow-pointer-lock` | Pointer Lock API |
| `allow-storage-access-by-user-activation` | Storage Access API |

### The Dangerous Combination

Never combine `allow-scripts` and `allow-same-origin` unless you fully trust the embedded content. With both tokens, the iframe's JavaScript can access its real origin's cookies and storage AND execute scripts. It can programmatically remove the `sandbox` attribute from its own `<iframe>` element via the parent document, defeating the sandbox entirely.

```html
<!-- DANGEROUS: iframe can escape the sandbox -->
<iframe sandbox="allow-scripts allow-same-origin"
        src="https://untrusted.example.com">
</iframe>

<!-- SAFE: scripts run but in a unique origin (no cookie/storage access) -->
<iframe sandbox="allow-scripts"
        src="https://untrusted.example.com">
</iframe>

<!-- SAFE: real origin access but no script execution -->
<iframe sandbox="allow-same-origin"
        src="https://partner.example.com">
</iframe>
```

### Practical Sandbox Configurations

```html
<!-- Third-party ad: scripts allowed, forms allowed, no top navigation -->
<iframe sandbox="allow-scripts allow-forms allow-popups"
        src="https://ads.example.com/banner"
        title="Advertisement">
</iframe>

<!-- User-generated content: no scripts at all -->
<iframe sandbox="allow-same-origin"
        srcdoc="<p>User content here</p>"
        title="User comment preview">
</iframe>

<!-- Payment widget: needs scripts and forms, allow popups for 3DS -->
<iframe sandbox="allow-scripts allow-forms allow-popups allow-popups-to-escape-sandbox allow-same-origin"
        src="https://payments.trusted-provider.com/checkout"
        title="Payment form">
</iframe>
```

## The `allow` Attribute (Permissions Policy)

The `allow` attribute controls which browser features (camera, microphone, geolocation, etc.) the iframe can access. It replaced the older "Feature Policy."

```html
<!-- Allow camera and microphone for a video call iframe -->
<iframe src="https://meet.example.com/room"
        allow="camera; microphone; fullscreen"
        title="Video call">
</iframe>

<!-- Allow geolocation with specific origin -->
<iframe src="https://maps.example.com/embed"
        allow="geolocation https://maps.example.com"
        title="Interactive map">
</iframe>
```

### Common Feature Directives

| Feature | Controls |
|---------|----------|
| `camera` | `getUserMedia()` video |
| `microphone` | `getUserMedia()` audio |
| `geolocation` | Geolocation API |
| `fullscreen` | Fullscreen API |
| `autoplay` | Media autoplay |
| `payment` | Payment Request API |
| `display-capture` | `getDisplayMedia()` |
| `picture-in-picture` | Picture-in-Picture API |
| `encrypted-media` | Encrypted Media Extensions |
| `web-share` | Web Share API |

### Two-Layer Authorization

For a cross-origin iframe to use a feature, both layers must agree:
1. The parent page's `Permissions-Policy` HTTP header must allow the feature for the iframe's origin.
2. The iframe's `allow` attribute must grant the feature.

If either layer denies the feature, it is blocked.

## Clickjacking Protection

In a clickjacking attack, an attacker embeds your page in a transparent iframe positioned over decoy elements on their page. Users think they are clicking the decoy but actually interact with your hidden page, potentially transferring funds, changing settings, or granting permissions.

### Preventing Your Page from Being Framed

**CSP `frame-ancestors` (modern, recommended):**

```
Content-Security-Policy: frame-ancestors 'none'           /* cannot be framed at all */
Content-Security-Policy: frame-ancestors 'self'            /* only same-origin */
Content-Security-Policy: frame-ancestors 'self' https://trusted.example.com
```

`frame-ancestors` must be set as an HTTP header. It cannot be used in a `<meta>` tag. It does not fall back to `default-src`.

**`X-Frame-Options` (legacy, still useful for old browsers):**

```
X-Frame-Options: DENY
X-Frame-Options: SAMEORIGIN
```

When both headers are present, modern browsers prioritize `frame-ancestors` and ignore `X-Frame-Options`.

### Controlling What Your Page Can Embed

CSP `frame-src` controls which URLs your page can load into iframes:

```
Content-Security-Policy: frame-src 'self' https://youtube.com https://maps.google.com
```

Fallback hierarchy: `frame-src` -> `child-src` -> `default-src`.

### The Two Directions

| Directive | Direction | Set On |
|-----------|-----------|--------|
| `frame-src` | What your page can embed | The parent page |
| `frame-ancestors` | Who can embed your page | The embedded page |

## Cross-Origin Communication with `postMessage`

The Same-Origin Policy prevents JavaScript in one origin from accessing the DOM of another origin. `window.postMessage()` is the sanctioned mechanism for cross-origin iframe communication.

### Sending Messages

```javascript
// Parent sending to iframe
const iframe = document.querySelector('iframe');
iframe.contentWindow.postMessage(
  { type: 'update', payload: { theme: 'dark' } },
  'https://widget.example.com'  // target origin (NEVER use '*' for sensitive data)
);

// Iframe sending to parent
window.parent.postMessage(
  { type: 'resize', payload: { height: 500 } },
  'https://mysite.example.com'  // parent's expected origin
);
```

### Receiving Messages

```javascript
window.addEventListener('message', (event) => {
  // ALWAYS validate the sender's origin
  if (event.origin !== 'https://widget.example.com') return;

  // event.data    - the message payload
  // event.origin  - sender's origin
  // event.source  - reference to sender's Window (can reply to it)

  if (event.data.type === 'resize') {
    const iframe = document.querySelector('iframe');
    iframe.style.height = event.data.payload.height + 'px';
  }
});
```

### What Can Be Sent

`postMessage` uses the structured clone algorithm. It can serialize objects, arrays, strings, numbers, booleans, Maps, Sets, Dates, RegExps, Blobs, Files, ArrayBuffers, and more. It cannot serialize functions, DOM nodes, or Symbols (throws `DataCloneError`).

### Security Rules

1. **Always validate `event.origin`** before processing the message.
2. **Always specify a target origin** (not `"*"`) when sending sensitive data.
3. **Never pass `event.data` to `eval()` or `innerHTML`** without sanitization.
4. **Use a structured message format** (e.g., `{ type: string, payload: any }`) for maintainability.

### Bidirectional Communication Pattern

```javascript
// Parent: send request and expect response
iframe.contentWindow.postMessage(
  { type: 'getData', requestId: 'abc123' },
  'https://widget.example.com'
);

window.addEventListener('message', (event) => {
  if (event.origin !== 'https://widget.example.com') return;
  if (event.data.type === 'dataResponse' && event.data.requestId === 'abc123') {
    console.log('Got response:', event.data.payload);
  }
});

// Inside iframe: receive request and respond
window.addEventListener('message', (event) => {
  if (event.origin !== 'https://mysite.example.com') return;
  if (event.data.type === 'getData') {
    event.source.postMessage(
      { type: 'dataResponse', requestId: event.data.requestId, payload: { count: 42 } },
      event.origin
    );
  }
});
```

## The `<object>` Element

The `<object>` element embeds an external resource that can be treated as an image, a nested browsing context, or a plugin-handled resource. Its key advantage over `<iframe>` and `<embed>` is support for fallback content.

```html
<object data="document.pdf" type="application/pdf" width="800" height="600">
  <p>Your browser does not support embedded PDFs.
     <a href="document.pdf">Download the PDF</a>.</p>
</object>
```

### Attributes

| Attribute | Description |
|-----------|-------------|
| `data` | URL of the resource |
| `type` | MIME type |
| `name` | Browsing context name (targetable) |
| `width` | Display width |
| `height` | Display height |
| `form` | Associates with a form element |

### Nested Fallback

Content between `<object>` tags renders only if the resource fails to load. This supports cascading fallbacks:

```html
<object data="chart.svg" type="image/svg+xml" width="600" height="400">
  <object data="chart.png" type="image/png" width="600" height="400">
    <p>Chart could not be displayed.</p>
  </object>
</object>
```

### SVG Embedding with `<object>`

Embedding SVG via `<object>` preserves the SVG DOM, allowing CSS and JavaScript interaction with the SVG content (unlike `<img>` which treats SVG as a flat image):

```html
<object data="interactive-diagram.svg" type="image/svg+xml"
        width="800" height="600"
        aria-label="Interactive system architecture diagram">
</object>
```

## The `<embed>` Element

The `<embed>` element is a void element (no closing tag, no fallback content) that embeds external content. It was primarily designed for browser plugins (Flash, Java), most of which are now deprecated.

```html
<embed src="document.pdf" type="application/pdf" width="800" height="600">
```

### Attributes

| Attribute | Description |
|-----------|-------------|
| `src` | URL of the resource |
| `type` | MIME type |
| `width` | Width in CSS pixels (absolute only) |
| `height` | Height in CSS pixels (absolute only) |

### Limitations

- No fallback content (void element).
- No `sandbox` or `allow` attributes.
- No `loading="lazy"`.
- Poor accessibility (no mechanism for an accessible name via the element itself).
- Limited to content the browser can handle natively (mainly PDFs and SVGs in modern browsers).

---

## Comparison: `<iframe>` vs `<object>` vs `<embed>`

| Feature | `<iframe>` | `<object>` | `<embed>` |
|---------|-----------|-----------|----------|
| Embeds HTML pages | Yes | Yes | No |
| Fallback content | No | Yes | No (void element) |
| `sandbox` attribute | Yes | No | No |
| `allow` (Permissions Policy) | Yes | No | No |
| `loading="lazy"` | Yes | No | No |
| PDF embedding | Yes | Yes | Yes |
| SVG with DOM access | No (isolated document) | Yes | No |
| Accessible name | `title` attribute | `aria-label` or inner text | None |

### When to Use What

- **Embedding HTML pages or third-party widgets:** Use `<iframe>`.
- **Embedding PDFs with a download fallback:** Use `<object>` with a download link as fallback.
- **Embedding SVGs that need interaction:** Use `<object>` or inline SVG.
- **Video and audio:** Use `<video>` and `<audio>`, never `<embed>` or `<object>`.
- **Interactive third-party content (maps, social widgets):** Use `<iframe>`.

## Iframe Lazy Loading

The `loading` attribute defers iframe loading until the frame is near the viewport.

```html
<iframe src="https://youtube.com/embed/abc123"
        loading="lazy"
        width="560"
        height="315"
        title="Tutorial video">
</iframe>
```

### Performance Impact

- Lazy-loading a YouTube embed saves 500+ KiB on initial page load.
- Lazy-loading a Facebook Like button plugin saves 200+ KiB.
- Each iframe creates a separate browsing context with its own HTTP requests, JS execution, and memory allocation.

### Rules

- Only lazy-load below-the-fold iframes. Never lazy-load above-the-fold content (it delays LCP).
- Always set `width` and `height` to prevent layout shifts (CLS) when the iframe loads.
- Hidden iframes (`display: none`) are loaded eagerly regardless of the `loading` attribute.

## Responsive Iframes

Iframes do not resize to fit their parent container by default. The most reliable approach uses `aspect-ratio`:

```css
iframe {
  width: 100%;
  height: auto;
  aspect-ratio: 16 / 9;
  border: none;
}
```

For browsers without `aspect-ratio` support, the padding-bottom hack works:

```html
<div class="iframe-wrapper">
  <iframe src="https://example.com" title="Embedded content"></iframe>
</div>
```

```css
.iframe-wrapper {
  position: relative;
  width: 100%;
  padding-bottom: 56.25%; /* 16:9 ratio */
  height: 0;
}

.iframe-wrapper iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  border: none;
}
```

### Dynamic Resizing via postMessage

When the iframe content's height is dynamic, the iframe can communicate its height to the parent:

```javascript
// Inside the iframe
const observer = new ResizeObserver(() => {
  window.parent.postMessage(
    { type: 'resize', height: document.body.scrollHeight },
    'https://parent-site.com'
  );
});
observer.observe(document.body);

// In the parent
window.addEventListener('message', (event) => {
  if (event.origin !== 'https://iframe-source.com') return;
  if (event.data.type === 'resize') {
    document.querySelector('iframe').style.height = event.data.height + 'px';
  }
});
```

## Iframe Accessibility

### The `title` Attribute Is Required

Every iframe must have a `title` attribute with a concise, descriptive label. Without it, screen readers announce "frame" with no useful context.

```html
<!-- BAD: no title -->
<iframe src="https://maps.google.com/embed?..."></iframe>

<!-- GOOD: descriptive title -->
<iframe src="https://maps.google.com/embed?..."
        title="Google Maps showing 123 Main St, New York">
</iframe>
```

### Multiple Iframes Need Unique Titles

When a page has multiple iframes, each title must be unique so users can distinguish them in a frame list.

```html
<iframe src="map.html" title="Office location map"></iframe>
<iframe src="chat.html" title="Customer support chat"></iframe>
<iframe src="calendar.html" title="Event calendar"></iframe>
```

### Hidden Iframes

Tracking pixels and analytics iframes should be hidden from assistive technology:

```html
<iframe src="https://analytics.example.com/pixel"
        title="Analytics"
        aria-hidden="true"
        tabindex="-1"
        width="0" height="0"
        style="display: none">
</iframe>
```

## Common Mistakes

- **Missing `title` attribute on iframes.** Screen reader users cannot identify the iframe's purpose. Every iframe needs a descriptive `title`.

- **Combining `sandbox="allow-scripts allow-same-origin"` on untrusted content.** This lets the iframe escape the sandbox by removing its own `sandbox` attribute via the parent DOM.

- **Using `"*"` as `targetOrigin` in `postMessage` for sensitive data.** This broadcasts data to any origin. Always specify the expected receiver's origin.

- **Not validating `event.origin` in message handlers.** An open listener without origin checks is exploitable by any page that can embed or open your site.

- **Missing `width` and `height` on iframes.** Causes layout shifts (CLS) when the iframe content loads, hurting Core Web Vitals.

- **Not lazy-loading below-fold iframes.** Each iframe loads a full browsing context. Deferring off-screen iframes saves significant bandwidth and CPU on initial load.

- **Placing important content inside iframes for SEO.** Search engines do not treat iframe content as part of the embedding page. Critical content must be in the parent document.

- **Using `<embed>` or `<object>` for video/audio.** Always use `<video>` and `<audio>`. Plugin-based embedding is deprecated.

- **No `frame-ancestors` CSP header.** Your page can be embedded by any attacker for clickjacking without this protection.

---

## Best Practices

- **Always set `title` on every iframe.** Make it descriptive and unique.

- **Use `sandbox` on untrusted content.** Start with maximum restrictions and add only the tokens you need.

- **Use the `allow` attribute for feature gating.** Only grant camera, microphone, geolocation, etc. when needed.

- **Set CSP `frame-ancestors`** on your pages to prevent clickjacking. Use `X-Frame-Options` alongside it for legacy browser support.

- **Use `loading="lazy"` for below-fold iframes.** Always pair with explicit `width` and `height`.

- **Validate `event.origin` in all `postMessage` handlers.** Never process messages from unexpected origins.

- **Use a structured message format** for `postMessage` communication: `{ type: string, payload: any }`.

- **Prefer `<object>` over `<embed>` for PDFs** because `<object>` supports fallback content.

- **Use `aspect-ratio` for responsive iframes.** This is cleaner than the padding-bottom hack.

## Real-World Use Cases

### Embedded YouTube Video with Full Security

```html
<iframe src="https://www.youtube-nocookie.com/embed/dQw4w9WgXcQ"
        title="Rick Astley - Never Gonna Give You Up (Official Music Video)"
        width="560"
        height="315"
        loading="lazy"
        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
        sandbox="allow-scripts allow-same-origin allow-popups allow-presentation"
        referrerpolicy="strict-origin-when-cross-origin"
        style="border: none; width: 100%; aspect-ratio: 16/9;">
</iframe>
```

Key decisions:
- `youtube-nocookie.com` reduces tracking.
- `loading="lazy"` saves 500+ KiB if below the fold.
- `sandbox` grants only what YouTube needs.
- `allow` grants media-related features.
- `aspect-ratio` makes it responsive.

### PDF Viewer with Download Fallback

```html
<object data="/reports/q4-2025.pdf"
        type="application/pdf"
        width="100%"
        height="600"
        aria-label="Q4 2025 Financial Report">
  <p>
    Your browser does not support embedded PDFs.
    <a href="/reports/q4-2025.pdf" download>Download the report (PDF, 2.4 MB)</a>.
  </p>
</object>
```

### Third-Party Widget with postMessage Resize

```html
<iframe id="pricing-widget"
        src="https://widget.pricing-tool.com/embed/abc123"
        title="Pricing calculator"
        sandbox="allow-scripts allow-forms allow-popups"
        allow="payment"
        loading="lazy"
        width="100%"
        height="400"
        style="border: none;">
</iframe>

<script>
  window.addEventListener('message', (event) => {
    if (event.origin !== 'https://widget.pricing-tool.com') return;

    if (event.data.type === 'resize') {
      document.getElementById('pricing-widget').style.height =
        event.data.height + 'px';
    }

    if (event.data.type === 'purchase-complete') {
      // Handle purchase completion
      showConfirmation(event.data.orderId);
    }
  });
</script>
```

---

> **Key Takeaway:** Use `<iframe>` for embedding HTML pages and third-party content, always with `title` for accessibility, `sandbox` for security, `loading="lazy"` for performance, and CSP `frame-ancestors` to protect your own pages from being framed by attackers.
