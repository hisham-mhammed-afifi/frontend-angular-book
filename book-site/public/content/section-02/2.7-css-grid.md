# 2.7 CSS Grid

> CSS Grid is a two-dimensional layout system that handles rows and columns simultaneously. It lets you define a grid structure on the container and place items precisely within it, making it the right tool for page-level layouts, galleries, dashboards, and anything requiring alignment across both axes.

---

## Core Concepts

Setting `display: grid` (block-level) or `display: inline-grid` (inline-level) creates a grid container. Direct children become grid items.

### Terminology

| Term | Definition |
|---|---|
| Grid line | A horizontal or vertical dividing line. Lines are numbered starting at 1. |
| Grid track | The space between two adjacent lines (a row or a column). |
| Grid cell | The intersection of one row track and one column track (a single unit). |
| Grid area | A rectangular space bounded by four grid lines, spanning one or more cells. |
| Gap / gutter | The spacing between tracks. |

```
  1       2       3       4     <- column lines
  |       |       |       |
  +-------+-------+-------+  <- row line 1
  | cell  | cell  | cell  |
  +-------+-------+-------+  <- row line 2
  | cell  | cell  | cell  |
  +-------+-------+-------+  <- row line 3
```

## Defining Tracks

### `grid-template-columns` and `grid-template-rows`

```css
.grid {
  display: grid;
  grid-template-columns: 200px 1fr 2fr;
  grid-template-rows: 80px auto 60px;
}
```

### The `fr` Unit

The `fr` (fraction) unit distributes remaining space after fixed tracks and gaps are subtracted. This is fundamentally different from percentages.

```css
/* Percentages: 50% + 50% + gap = OVERFLOW */
.bad { grid-template-columns: 50% 50%; gap: 20px; }

/* fr: accounts for gaps automatically, never overflows */
.good { grid-template-columns: 1fr 1fr; gap: 20px; }
```

A bare `1fr` has an implicit minimum of `auto` (which resolves to `min-content`). This means `1fr 1fr 1fr` does NOT guarantee equal columns if content sizes differ. The track expands to fit its minimum content size before distributing remaining space.

To force truly equal columns regardless of content:

```css
/* Override the auto minimum with 0 */
grid-template-columns: repeat(3, minmax(0, 1fr));
```

### `minmax()`

Defines a size range (minimum and maximum) for a track:

```css
grid-template-columns: minmax(200px, 1fr) minmax(100px, 300px) 1fr;
```

Rules:
- If `max < min`, the max is ignored and the track uses the min value.
- `fr` is only valid as the **max** value. `minmax(1fr, 300px)` is invalid.
- `minmax(0, 1fr)` forces a track to shrink below content size when needed.

### `repeat()`

Repeats a track pattern:

```css
grid-template-columns: repeat(3, 1fr);             /* 3 equal columns */
grid-template-columns: repeat(2, 100px 1fr);       /* repeating pattern */
grid-template-columns: 200px repeat(3, 1fr) 200px; /* mixed */
```

### Sizing Keywords

| Keyword | Behavior |
|---|---|
| `min-content` | Smallest size without overflow (longest unbreakable word for text) |
| `max-content` | Size needed to fit all content on one line |
| `auto` | Behaves like `minmax(min-content, max-content)` in most cases |
| `fit-content(limit)` | Like `max-content` but capped at the given limit |

```css
/* Sidebar sized to its content, main fills the rest */
grid-template-columns: fit-content(300px) 1fr;

/* Sidebar never wider than 300px but can be narrower if content is small */
```

## `auto-fill` vs `auto-fit`

Both go inside `repeat()` to create as many tracks as fit in the container. The difference appears when there are fewer items than tracks.

```css
/* auto-fill: empty tracks preserved, items do NOT stretch */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: empty tracks collapse to 0, items stretch to fill */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

When there are enough items to fill all tracks, both behave identically.

**Use `auto-fill`** for consistent item sizing (image galleries, product grids).
**Use `auto-fit`** for items that should stretch edge-to-edge (navigation links, feature cards).

### The Responsive Grid Pattern (No Media Queries)

```css
.responsive-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 250px), 1fr));
  gap: 1.5rem;
}
```

The `min(100%, 250px)` prevents overflow on small screens where 250px would exceed the container width. This single line creates a fully responsive grid without media queries.

**Constraint:** `auto-fill`/`auto-fit` cannot use bare `fr` units. `repeat(auto-fill, 1fr)` is invalid because the browser needs a fixed minimum to calculate how many tracks fit.

## Explicit vs Implicit Grid

### Explicit Grid

Defined by `grid-template-columns` and `grid-template-rows`. You control exactly how many tracks exist.

### Implicit Grid

Created automatically when items are placed outside the explicit grid. If you define 2 rows but have 10 items, the extra items create implicit rows.

### Controlling Implicit Tracks

```css
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 200px;               /* explicit: 1 row */
  grid-auto-rows: minmax(150px, auto);      /* implicit rows: at least 150px */
}
```

Default implicit track size is `auto`. Always set `grid-auto-rows` when working with dynamic content to ensure consistent sizing.

### `grid-auto-flow`

Controls how auto-placed items fill the grid:

```css
grid-auto-flow: row;        /* default: fill row by row */
grid-auto-flow: column;     /* fill column by column */
grid-auto-flow: row dense;  /* backtrack to fill gaps (see Dense Packing below) */
```

## Item Placement

### Line-Based Placement

Grid lines are numbered starting at 1. Negative numbers count from the end of the explicit grid.

```css
.item {
  grid-column: 1 / 3;    /* start at line 1, end at line 3 (spans 2 columns) */
  grid-row: 2 / 4;       /* start at line 2, end at line 4 (spans 2 rows) */
}
```

### The `span` Keyword

```css
.item {
  grid-column: 1 / span 2;   /* start at line 1, span 2 columns */
  grid-row: span 3;          /* span 3 rows from auto-placed position */
}
```

### The `grid-area` Shorthand

```css
.item {
  grid-area: 2 / 1 / 4 / 3;
  /* row-start / col-start / row-end / col-end */
}
```

### Full-Width Items

```css
.full-width {
  grid-column: 1 / -1;  /* span the entire explicit grid */
}
```

**Gotcha:** `-1` references the last line of the **explicit** grid only. If items create implicit columns beyond the explicit grid, `-1` does not reach them.

### Overlapping Items

Grid items can occupy the same cells. Without `z-index`, later items in the DOM paint on top. Grid items accept `z-index` without needing `position: relative`:

```css
.background-image {
  grid-area: 1 / 1 / 3 / 3;
  z-index: 1;
}

.overlay-text {
  grid-area: 1 / 1 / 3 / 3;
  z-index: 2;
  /* Overlaps .background-image without positioning */
}
```

## Named Grid Lines

Lines can be named for readability:

```css
.grid {
  grid-template-columns:
    [sidebar-start] 250px
    [sidebar-end content-start] 1fr
    [content-end];
  grid-template-rows:
    [header-start] 80px
    [header-end main-start] 1fr
    [main-end];
}

.sidebar { grid-column: sidebar-start / sidebar-end; }
.main    { grid-column: content-start / content-end; }
```

Lines can have multiple names: `[sidebar-end content-start]`.

### Implicit Named Areas

If you name lines with `-start` and `-end` suffixes in both dimensions, the browser automatically creates a named area. Lines named `content-start` and `content-end` on both columns and rows create an area named `content`.

---

## `grid-template-areas`

The most readable way to define a grid layout. Each string represents a row, and each word is a cell.

```css
.layout {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar main    main"
    "footer  footer  footer";
  grid-template-columns: 250px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

### Rules

- Each row string must have the same number of cells.
- Areas must form rectangles. L-shapes and T-shapes are invalid.
- Use `.` for empty cells: `"header header ."`.
- Named areas automatically create named lines: `header` produces `header-start` and `header-end`.

```css
/* INVALID: L-shaped area */
grid-template-areas:
  "header header"
  "sidebar header";  /* header is not rectangular */

/* VALID */
grid-template-areas:
  "header  header"
  "sidebar main";
```

### Responsive Areas with Media Queries

```css
.layout {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "sidebar"
    "footer";
  grid-template-columns: 1fr;
}

@media (min-width: 768px) {
  .layout {
    grid-template-areas:
      "header  header"
      "sidebar main"
      "footer  footer";
    grid-template-columns: 250px 1fr;
  }
}
```

The grid-area assignments on items do not change. Only the template changes, and the layout restructures automatically.

## Alignment

Grid alignment operates on two axes with six properties:

### Aligning Items Within Their Cells

| Property | Axis | Scope |
|---|---|---|
| `justify-items` | Inline (row) | All items |
| `align-items` | Block (column) | All items |
| `place-items` | Both (shorthand) | All items |
| `justify-self` | Inline (row) | Single item |
| `align-self` | Block (column) | Single item |

```css
.grid {
  justify-items: start;   /* horizontal alignment of all items */
  align-items: center;    /* vertical alignment of all items */
  /* Or shorthand: */
  place-items: center start; /* align-items justify-items */
}

.special-item {
  justify-self: end;
  align-self: stretch;
}
```

Default is `stretch` for both. Items fill their cells unless they have an explicit size.

### Aligning the Grid Within Its Container

When the grid tracks do not fill the entire container:

| Property | Axis |
|---|---|
| `justify-content` | Inline (row) |
| `align-content` | Block (column) |
| `place-content` | Both (shorthand) |

Values: `start`, `end`, `center`, `stretch`, `space-between`, `space-around`, `space-evenly`.

```css
.grid {
  height: 100vh;
  display: grid;
  grid-template-columns: repeat(3, 200px);
  grid-template-rows: repeat(2, 150px);
  place-content: center; /* centers the grid tracks within the container */
}
```

## Dense Packing

```css
grid-auto-flow: row dense;
```

The default (sparse) algorithm only moves forward. If a large item leaves a gap, later small items skip it. Dense packing backtracks to fill holes left by larger items.

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
  grid-auto-rows: 150px;
  grid-auto-flow: dense;
  gap: 4px;
}

.gallery .wide { grid-column: span 2; }
.gallery .tall { grid-row: span 2; }
/* Small items fill gaps left by wide/tall items */
```

**Accessibility warning:** Dense packing reorders items visually but not in the DOM. Screen readers and keyboard navigation follow DOM order. Use `dense` only when visual order does not convey meaning (image galleries, not content lists).

## Subgrid

Subgrid lets a nested grid inherit track definitions from its parent grid.

### The Problem It Solves

Without subgrid, nested grids are independent. Cards in a row cannot align their headers, bodies, and footers across columns because each card defines its own row tracks.

### Syntax

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1.5rem;
}

.card {
  display: grid;
  grid-row: span 3;           /* span all 3 row tracks */
  grid-template-rows: subgrid; /* inherit parent row definitions */
  gap: 0;                     /* subgrid can have its own gap */
}

.card__header { /* aligns with headers in other cards */ }
.card__body   { /* aligns with bodies in other cards */ }
.card__footer { /* aligns with footers in other cards */ }
```

You can subgrid one axis and define the other independently:

```css
.child {
  grid-template-columns: subgrid;  /* inherit parent columns */
  grid-template-rows: auto 1fr;    /* define own rows */
}
```

### Primary Use Cases

- Card layouts with aligned internal sections.
- Form layouts with aligned labels and inputs.
- Pricing tables with aligned feature rows.
- Any nested component that should participate in a parent grid's track sizing.

Browser support: Chrome 117+, Firefox 71+, Safari 16+. ~97% global coverage.

## The `grid` Shorthand

The `grid` property is a shorthand for all explicit grid properties. It is powerful but hard to read:

```css
/* Explicit rows and columns */
.grid {
  grid: auto 1fr auto / 250px 1fr;
  /* Equivalent to:
     grid-template-rows: auto 1fr auto;
     grid-template-columns: 250px 1fr; */
}

/* With template-areas */
.grid {
  grid:
    "header header" auto
    "sidebar main" 1fr
    "footer footer" auto
    / 250px 1fr;
}
```

The shorthand also resets `grid-auto-rows`, `grid-auto-columns`, and `grid-auto-flow` to their defaults. Because of this resetting behavior and reduced readability, many developers prefer using the longhand properties.

## Common Mistakes

1. **Expecting `1fr 1fr 1fr` to produce equal columns.** The `fr` unit has an implicit minimum of `auto` (resolving to `min-content`). If one column has wider content, it expands. Use `repeat(3, minmax(0, 1fr))` for truly equal columns.

2. **Using percentages with gaps.** `50% + 50% + gap = overflow`. Use `fr` units, which account for gaps automatically.

3. **Forgetting `grid-auto-rows`.** Without it, implicit rows default to `auto`. Dynamic content with varying heights creates inconsistent row sizes.

4. **Negative line numbers with implicit tracks.** `grid-column: 1 / -1` only spans the explicit grid. Items in implicit tracks are not included.

5. **Non-rectangular template areas.** Areas in `grid-template-areas` must form rectangles. L-shapes and T-shapes cause the entire declaration to be invalid.

6. **Using `dense` with meaningful content order.** Dense packing reorders items visually but not in the DOM. Keyboard and screen reader order follows the DOM, creating navigation confusion.

7. **`auto-fill`/`auto-fit` with bare `fr`.** `repeat(auto-fill, 1fr)` is invalid. The browser needs a fixed minimum. Use `minmax(200px, 1fr)`.

8. **Forgetting `min()` in the responsive grid pattern.** `repeat(auto-fit, minmax(250px, 1fr))` breaks on screens narrower than 250px. Wrap the minimum: `minmax(min(100%, 250px), 1fr)`.

9. **Confusing `justify-items`/`align-items` with `justify-content`/`align-content`.** The `-items` properties align items within their cells. The `-content` properties align the grid tracks within the container.

10. **Not using subgrid for aligned nested layouts.** Before subgrid, developers used workarounds (flat grids, JavaScript). Now that subgrid has ~97% support, it should be the default approach for aligned nested content.

---

## Best Practices

1. **Use `grid-template-areas` for page layouts.** The visual string format maps directly to the screen layout, making code self-documenting.

2. **Always set `grid-auto-rows` with dynamic content.** `grid-auto-rows: minmax(100px, auto)` ensures consistent minimum sizing for implicit rows.

3. **Use `minmax(0, 1fr)` when you need truly equal columns.** This overrides the default `auto` minimum that prevents columns from shrinking below content size.

4. **Combine `auto-fit` with `minmax()` for responsive grids.** The pattern `repeat(auto-fit, minmax(min(100%, 250px), 1fr))` handles all screen sizes without media queries.

5. **Use subgrid for cards, forms, and any nested aligned layouts.** It eliminates the need for flat grid structures or JavaScript-based height equalization.

6. **Use Grid for 2D layouts, Flexbox for 1D layouts.** They complement each other. Grid for page structure and galleries. Flexbox for navbars, toolbars, and component internals.

7. **Use named lines or template areas for complex layouts.** Line numbers are hard to maintain. Named references make the code readable and refactorable.

8. **Debug with browser DevTools.** Firefox and Chrome have excellent grid inspectors showing line numbers, track sizes, areas, and implicit vs explicit boundaries.

## Real-World Use Cases

### 1. Responsive Card Grid with Aligned Internals (Subgrid)

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 280px), 1fr));
  gap: 1.5rem;
}

.card {
  display: grid;
  grid-template-rows: auto 1fr auto; /* header, body, footer */
  /* If parent has explicit rows and card spans them, use subgrid: */
  /* grid-template-rows: subgrid; */
  border: 1px solid #e0e0e0;
  border-radius: 8px;
  overflow: clip;
}

.card__image {
  width: 100%;
  aspect-ratio: 16 / 9;
  object-fit: cover;
}

.card__body {
  padding: 1rem;
  /* In the middle row: grows to fill space, pushing footer down */
}

.card__footer {
  padding: 0.75rem 1rem;
  border-top: 1px solid #eee;
  /* Always at the bottom across all cards */
}
```

### 2. Full-Bleed Layout with Constrained Content

```css
.page {
  display: grid;
  grid-template-columns:
    [full-start] 1fr
    [content-start] min(65ch, calc(100% - 2rem))
    [content-end] 1fr
    [full-end];
}

/* Default: constrained to content width */
.page > * {
  grid-column: content;
}

/* Full-bleed: spans the entire viewport width */
.page > .full-bleed {
  grid-column: full;
}

/* Breakout: slightly wider than content */
.page > .breakout {
  grid-column: full;
  max-width: 90ch;
  margin-inline: auto;
  width: 100%;
}
```

```html
<main class="page">
  <h1>Article Title</h1>
  <p>Regular content paragraph, constrained to 65ch.</p>
  <img class="full-bleed" src="hero.jpg" alt="Full-width hero image">
  <p>More content after the full-bleed image.</p>
</main>
```

### 3. Dashboard Layout with Template Areas

```css
.dashboard {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "sidebar footer  footer";
  grid-template-columns: 220px 1fr 300px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 1px;
  background: #e0e0e0; /* gap color via background */
}

.dashboard > * { background: white; }

.dashboard__header  { grid-area: header; padding: 1rem 1.5rem; }
.dashboard__sidebar { grid-area: sidebar; padding: 1rem; }
.dashboard__main    { grid-area: main; padding: 1.5rem; overflow-y: auto; }
.dashboard__aside   { grid-area: aside; padding: 1rem; }
.dashboard__footer  { grid-area: footer; padding: 0.75rem 1.5rem; }

@media (max-width: 1024px) {
  .dashboard {
    grid-template-areas:
      "header  header"
      "sidebar main"
      "footer  footer";
    grid-template-columns: 200px 1fr;
  }
  .dashboard__aside { display: none; }
}

@media (max-width: 640px) {
  .dashboard {
    grid-template-areas:
      "header"
      "main"
      "footer";
    grid-template-columns: 1fr;
  }
  .dashboard__sidebar { display: none; }
}
```

### 4. Image Gallery with Dense Packing

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
  grid-auto-rows: 150px;
  grid-auto-flow: dense;
  gap: 4px;
}

.gallery img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  border-radius: 2px;
}

/* Feature images span 2x2 */
.gallery .featured {
  grid-column: span 2;
  grid-row: span 2;
}

/* Wide images span 2 columns */
.gallery .wide {
  grid-column: span 2;
}

/* Tall images span 2 rows */
.gallery .tall {
  grid-row: span 2;
}
```

---

> **Key Takeaway:** CSS Grid is the right tool for two-dimensional layouts where items need to align across both rows and columns; master `minmax()` with `auto-fit`/`auto-fill` for responsive grids without media queries, use `grid-template-areas` for readable page layouts, and use subgrid to align nested content across grid items.
