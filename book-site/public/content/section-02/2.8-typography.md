# 2.8 Typography

> Typography in CSS controls how text is rendered, sized, spaced, and loaded. Getting it right means readable content, fast page loads, and consistent rendering across platforms.

---

## Font Stacks

A font stack is the ordered list of fonts the browser tries, falling back through each until one is available.

### System Font Stack

The modern approach uses `system-ui`, which resolves to the platform's default UI font (San Francisco on macOS/iOS, Segoe UI on Windows, Roboto on Android):

```css
body {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI",
               Roboto, Oxygen, Ubuntu, Cantarell, "Fira Sans",
               "Droid Sans", "Helvetica Neue", Arial, sans-serif;
}
```

`-apple-system` and `BlinkMacSystemFont` are legacy prefixes for older Safari/Chrome. They are becoming unnecessary as `system-ui` has universal modern browser support.

### Font-Family Rules

- Always end with a generic family name (`sans-serif`, `serif`, `monospace`) as the final fallback.
- Quote multi-word font names: `"Helvetica Neue"`, not `Helvetica Neue`.
- Do NOT quote generic family names: `sans-serif`, not `"sans-serif"`. Quoting turns it into a named font lookup instead of a generic fallback.

```css
/* BAD: "sans-serif" is treated as a font name, not a generic fallback */
body { font-family: "Inter", "sans-serif"; }

/* GOOD: sans-serif (unquoted) is a generic fallback */
body { font-family: "Inter", sans-serif; }
```

### New Generic Families

CSS defines additional generic families beyond the classic five:

| Generic | Purpose | Support |
|---|---|---|
| `system-ui` | Platform default UI font | All modern browsers |
| `ui-serif` | System serif font | Safari only |
| `ui-sans-serif` | System sans-serif | Safari only |
| `ui-monospace` | System monospace | Safari only |
| `ui-rounded` | System rounded font | Safari only |

## `@font-face` and Web Font Loading

### Basic Syntax

```css
@font-face {
  font-family: "Inter";
  src: url("/fonts/inter.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
```

**WOFF2 is the only format you need in 2025.** It provides ~30% better compression than WOFF. TTF, OTF, and SVG font formats are obsolete for the web.

### Performance Optimization

**1. Self-host fonts.** Google Fonts CDN no longer provides a caching advantage because browsers partition their cache by origin. Self-hosting eliminates a DNS lookup, a TLS handshake, and privacy concerns.

**2. Preload critical fonts:**

```html
<link rel="preload" href="/fonts/inter.woff2" as="font"
      type="font/woff2" crossorigin>
```

The `crossorigin` attribute is required even for same-origin fonts. Omitting it causes the browser to download the font twice.

**3. Subset fonts.** Most fonts contain thousands of glyphs. Subsetting to Basic Latin can reduce a 139KB font to ~15KB. Tools: `fonttools` (Python), Glyphhanger, Font Squirrel generator.

**4. Use `unicode-range` for progressive loading:**

```css
@font-face {
  font-family: "Inter";
  src: url("/fonts/inter-latin.woff2") format("woff2");
  unicode-range: U+0000-00FF, U+0131, U+0152-0153;
}

@font-face {
  font-family: "Inter";
  src: url("/fonts/inter-cyrillic.woff2") format("woff2");
  unicode-range: U+0400-045F, U+0490-0491;
}
```

The browser only downloads a file if characters in its range appear on the page. Note: `unicode-range` controls when the file downloads but does not reduce file size. You must combine it with actual subsetting.

## `font-display`

Controls what happens while a web font downloads. Every font load goes through three periods: block (text invisible), swap (fallback shown), and failure (stays on fallback).

| Value | Block period | Swap period | Best for |
|---|---|---|---|
| `auto` | Browser decides | Browser decides | Rarely useful |
| `block` | ~3 seconds | Infinite | Icon fonts only |
| `swap` | ~100ms | Infinite | Body text (always visible) |
| `fallback` | ~100ms | ~3 seconds | Balance of swap and optional |
| `optional` | ~100ms | None | Minimal layout shift |

```css
@font-face {
  font-family: "Inter";
  src: url("/fonts/inter.woff2") format("woff2");
  font-display: swap;
}
```

- **`swap`**: Recommended default. Shows fallback immediately, swaps when font loads (FOUT). Always visible.
- **`optional`**: Best for Cumulative Layout Shift (CLS). The browser may skip the swap entirely on slow connections.
- **`block`**: Only for icon fonts where showing a fallback character would be meaningless.

## Variable Fonts

A single font file containing multiple axes of variation (weight, width, slant) as continuous ranges. One variable font file replaces multiple static files.

### Registered Axes

| Axis | CSS property | Tag | Range |
|---|---|---|---|
| Weight | `font-weight` | `wght` | 100-900 |
| Width | `font-stretch` | `wdth` | 75%-125% |
| Slant | `font-style: oblique Xdeg` | `slnt` | -12 to 0 |
| Italic | `font-style: italic` | `ital` | 0 or 1 |
| Optical size | `font-optical-sizing` | `opsz` | 8-144 |

### Using Variable Fonts

```css
@font-face {
  font-family: "Inter Variable";
  src: url("/fonts/InterVariable.woff2") format("woff2-variations");
  font-weight: 100 900;   /* declare the full range */
  font-display: swap;
}

body {
  font-family: "Inter Variable", sans-serif;
  font-weight: 400;
}

h1 {
  font-weight: 650; /* any value in the range, not just 400/700 */
}

.thin-text {
  font-weight: 250;
}
```

### High-Level Properties vs `font-variation-settings`

Prefer high-level CSS properties over the low-level `font-variation-settings`:

```css
/* GOOD: high-level properties */
h1 {
  font-weight: 650;
  font-stretch: 110%;
  font-style: oblique 8deg;
}

/* AVOID: low-level property clobbers all axes not listed */
h1 {
  font-variation-settings: "wght" 650, "wdth" 110;
}
```

`font-variation-settings` resets ALL axes to their defaults except those explicitly listed. If you set `wght` in one rule and `wdth` in another, one declaration clobbers the other. Use it only for custom axes not covered by CSS properties:

```css
h1 {
  font-weight: 650;
  font-variation-settings: "GRAD" 88, "CASL" 1; /* custom axes only */
}
```

### `font-optical-sizing`

When enabled (`auto`, the default), the browser adjusts glyph shapes based on font size. Small text gets thicker strokes for legibility; large text gets more delicate strokes with greater contrast. Only works with variable fonts that have an `opsz` axis.

```css
body { font-optical-sizing: auto; }  /* default, enabled */
```

## Line Height

### Always Use Unitless Values

```css
/* BAD: computed px value is inherited, not the multiplier */
body { font-size: 16px; line-height: 24px; }
h1   { font-size: 32px; }
/* h1 inherits line-height: 24px (too tight for 32px text) */

/* GOOD: unitless ratio is inherited, each element computes its own */
body { font-size: 16px; line-height: 1.5; }
h1   { font-size: 32px; }
/* h1 computes line-height: 48px (32 * 1.5) */
```

### Recommended Ratios

| Context | Recommended `line-height` |
|---|---|
| Body text | 1.5 to 1.7 |
| Headings | 1.1 to 1.3 |
| Buttons / UI elements | 1 to 1.2 |
| Code blocks | 1.4 to 1.6 |

WCAG SC 1.4.12 requires line spacing of at least 1.5 for body text.

### The `normal` Value

`line-height: normal` is browser-dependent but typically ~1.2. This is too tight for body text. Always set an explicit `line-height`.

### Half-Leading

CSS distributes extra space (the difference between `line-height` and the font's em-box) equally above and below inline content. This creates vertical spacing inconsistencies, especially at the top and bottom of text blocks. The emerging `text-box-trim` and `text-box-edge` properties address this but have limited browser support as of 2025.

## Fluid Typography with `clamp()`

`clamp()` creates responsive font sizes that scale smoothly between a minimum and maximum:

```css
h1 {
  font-size: clamp(1.5rem, 1rem + 2vw, 3rem);
  /*              min      preferred    max     */
}
```

### The Preferred Value Formula

For a font that should be 18px at 320px viewport and 24px at 1200px viewport:

```
slope = (24 - 18) / (1200 - 320) = 0.00682
yIntercept = 18 - (0.00682 * 320) = 15.818px = ~0.989rem
preferred = 0.989rem + 0.682vw

font-size: clamp(1.125rem, 0.989rem + 0.682vw, 1.5rem);
```

Tools like [Utopia](https://utopia.fyi) and [Fluid Type Scale Calculator](https://www.fluid-type-scale.com/) generate these values automatically.

### Accessibility: Always Include `rem` in the Preferred Value

```css
/* BAD: vw alone does not respond to browser zoom */
font-size: clamp(1rem, 4vw, 3rem);

/* GOOD: rem component responds to zoom */
font-size: clamp(1rem, 0.5rem + 2vw, 3rem);
```

Using `vw` alone means users who zoom their browser see no font size change, violating WCAG 1.4.4 (Resize Text).

### A Complete Type Scale

```css
:root {
  --step--1: clamp(0.8rem, 0.73rem + 0.33vw, 1rem);
  --step-0: clamp(1rem, 0.89rem + 0.49vw, 1.31rem);
  --step-1: clamp(1.25rem, 1.08rem + 0.73vw, 1.71rem);
  --step-2: clamp(1.56rem, 1.3rem + 1.09vw, 2.24rem);
  --step-3: clamp(1.95rem, 1.56rem + 1.64vw, 2.93rem);
  --step-4: clamp(2.44rem, 1.86rem + 2.44vw, 3.84rem);
}

small { font-size: var(--step--1); }
body  { font-size: var(--step-0); }
h3    { font-size: var(--step-1); }
h2    { font-size: var(--step-2); }
h1    { font-size: var(--step-3); }
.display { font-size: var(--step-4); }
```

## Text Wrapping: `text-wrap`

### `text-wrap: balance`

Makes line lengths as even as possible for headings:

```css
h1, h2, h3, h4, h5, h6 {
  text-wrap: balance;
}
```

The browser considers all possible breakpoints and minimizes the difference between the longest and shortest lines.

**Limitations:**
- Performance cap: works on blocks with 6 or fewer lines (Chromium) or 10 (Firefox). Longer blocks are silently ignored.
- Apply only to headings and short text, never to body paragraphs.

### `text-wrap: pretty`

Optimizes paragraph-level line breaking to eliminate orphans (a single word on the last line):

```css
p {
  text-wrap: pretty;
}
```

Browser support: `balance` in Chrome 114+, Firefox 121+, Safari 17.5+. `pretty` in Chrome 117+, Safari 26+, but not yet in Firefox.

## Text Truncation

### Single-Line

All three properties are required:

```css
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

Remove any one and truncation breaks.

### Multi-Line (Line Clamp)

```css
.clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

Despite the `-webkit-` prefix, this works in all modern browsers. The unprefixed `line-clamp` is still in Editor's Draft.

**Gotchas:**
- Must use `display: -webkit-box` and `-webkit-box-orient: vertical`. Regular `display: flex` or `display: block` will not work.
- `text-overflow: ellipsis` is not needed; the ellipsis is automatic.
- Adding `padding-bottom` can cause a partial extra line to peek through. Apply padding on a wrapper instead.

## Spacing: `letter-spacing`, `word-spacing`, `font-kerning`

### `letter-spacing`

Adds uniform spacing between all characters. Use `em` units so spacing scales with font size:

```css
.uppercase-label {
  text-transform: uppercase;
  letter-spacing: 0.05em; /* uppercase text needs increased spacing */
}

.tight-display {
  font-size: 4rem;
  letter-spacing: -0.02em; /* tighten large display text */
}
```

Uppercase text almost always benefits from increased letter-spacing. Lowercase body text rarely needs it.

### `font-kerning`

Kerning adjusts space between specific character pairs (e.g., "AV", "To", "Wa") using tables built into the font. It is NOT uniform like `letter-spacing`.

```css
body { font-kerning: auto; }    /* browser decides (usually on) */
.heading { font-kerning: normal; } /* force kerning on */
```

`letter-spacing` is applied after kerning. They stack independently.

### `word-spacing`

```css
.spaced { word-spacing: 0.1em; }
```

Less commonly adjusted than letter-spacing. Default is determined by the font's space character width.

## Text Breaking: `white-space`, `overflow-wrap`, `word-break`

### `white-space`

Controls whitespace collapsing and line wrapping:

| Value | Collapses spaces | Collapses newlines | Wraps |
|---|---|---|---|
| `normal` | Yes | Yes | Yes |
| `nowrap` | Yes | Yes | No |
| `pre` | No | No | No |
| `pre-wrap` | No | No | Yes |
| `pre-line` | Yes | No | Yes |
| `break-spaces` | No | No | Yes (spaces can break) |

### `overflow-wrap` vs `word-break`

These are commonly confused:

```css
/* overflow-wrap: break-word -- only breaks words that WOULD overflow */
.safe {
  overflow-wrap: break-word;
}

/* word-break: break-all -- breaks ALL words at the end of every line */
.aggressive {
  word-break: break-all;
}
```

`overflow-wrap: break-word` is almost always what you want. It preserves whole words when they fit and only breaks long strings (URLs, hashes) that would overflow. `word-break: break-all` is more aggressive and less readable.

### `hyphens`

```css
p {
  hyphens: auto;
  -webkit-hyphens: auto; /* Safari */
}
```

**Requires the `lang` attribute on the HTML element** (`<html lang="en">`). Without it, `hyphens: auto` silently does nothing.

## OpenType Features

### High-Level Properties (Preferred)

```css
/* Tabular numerals for data tables */
.data-table {
  font-variant-numeric: tabular-nums lining-nums;
}

/* Small caps for abbreviations */
abbr {
  font-variant-caps: all-small-caps;
}

/* Disable ligatures where they interfere (e.g., code) */
code {
  font-variant-ligatures: none;
}

/* Old-style numerals for body text */
.body-text {
  font-variant-numeric: oldstyle-nums proportional-nums;
}

/* Diagonal fractions */
.recipe {
  font-variant-numeric: diagonal-fractions;
}
```

### Low-Level `font-feature-settings` (Use Sparingly)

```css
/* Only for features without a high-level CSS property */
.stylistic {
  font-feature-settings: "ss01" 1, "swsh" 1;
}
```

Avoid `font-feature-settings` for features covered by `font-variant-*` properties. It has a "clobber" problem: setting any value resets all other features not explicitly listed.

---

## `text-rendering`

```css
h1, h2, h3 { text-rendering: optimizeLegibility; }
```

| Value | Kerning | Ligatures | Use case |
|---|---|---|---|
| `auto` | Browser decides | Browser decides | Default |
| `optimizeSpeed` | Off | Off | Large text blocks, mobile performance |
| `optimizeLegibility` | On | On | Headings and short display text |

**Never apply `optimizeLegibility` to body text globally.** It can cause 30+ second loading delays on older Android devices. Apply it only to headings.

## Text Decoration

Modern CSS gives fine control over underlines:

```css
a {
  text-decoration-line: underline;
  text-decoration-color: #3498db;
  text-decoration-thickness: 2px;
  text-underline-offset: 3px;
  text-decoration-skip-ink: auto; /* default: skips descenders */
}

a:hover {
  text-decoration-thickness: 3px;
  text-underline-offset: 2px;
}
```

`text-underline-offset` moves the underline away from the baseline. Positive values move it down. `text-decoration-skip-ink: auto` (the default in modern browsers) makes underlines skip descenders (g, p, y) for cleaner appearance.

---

## `font-size-adjust`

Normalizes the apparent size of fallback fonts by scaling based on the x-height ratio:

```css
body {
  font-family: "Inter", "Helvetica Neue", sans-serif;
  font-size-adjust: 0.52; /* Inter's x-height ratio */
}
```

When the fallback loads (before the web font), letters appear at a similar visual size, reducing layout shift. Supported in all modern browsers since mid-2024.

## Common Mistakes

1. **Using `px` for `line-height`.** The computed pixel value is inherited, not the multiplier. Child elements with larger font sizes inherit a too-tight line height. Use unitless values.

2. **No generic fallback in `font-family`.** Without `sans-serif` or `serif` at the end, the browser defaults to its own choice (often Times New Roman).

3. **Thin font weights for body text.** Weights below 400 reduce legibility, especially on low-DPI screens. Reserve thin/light weights for large display text.

4. **Line length too wide.** Body text should be 45-75 characters per line. Without `max-width` constraints, text spans the full viewport and becomes hard to read.

5. **`vw`-only fluid typography.** Without a `rem` component, font sizes do not respond to browser zoom, violating WCAG 1.4.4.

6. **Forgetting `crossorigin` on font preload.** Omitting it causes the browser to download the font twice (once without CORS, once with).

7. **Applying `text-wrap: balance` to paragraphs.** Silently ignored on long blocks but wastes CPU cycles. Apply only to headings and short text.

8. **Forgetting the `lang` attribute with `hyphens: auto`.** Without `<html lang="en">`, automatic hyphenation does nothing.

9. **Using `font-feature-settings` instead of `font-variant-*`.** One declaration clobbers all other OpenType settings not explicitly listed.

10. **Loading multiple static font files when one variable font would suffice.** A single variable font file replaces all weight/width variants, reducing HTTP requests and total file size.

11. **Using `word-break: break-all` when `overflow-wrap: break-word` would suffice.** The former aggressively breaks all words; the latter only breaks words that would overflow.

12. **Setting `text-rendering: optimizeLegibility` on `body`.** Causes performance problems on mobile devices. Apply it to headings only.

## Best Practices

1. **Self-host fonts.** Eliminates third-party DNS lookups and gives you control over subsetting, `font-display`, and caching.

2. **Use WOFF2 only.** No need for WOFF, TTF, or other formats in 2025.

3. **Preload critical fonts with `crossorigin`.** Reduces time to first render of web fonts.

4. **Subset fonts aggressively.** If your site is English-only, subset to Basic Latin + Latin Extended for 75-90% file size reduction.

5. **Use `font-display: swap` for body text, `optional` for decorative fonts.** `swap` ensures text is always visible. `optional` minimizes layout shift.

6. **Use unitless `line-height`.** Ratios like 1.5 inherit correctly across all font sizes.

7. **Use `clamp()` with `rem + vw` for fluid typography.** Responsive sizing that respects user zoom.

8. **Use `text-wrap: balance` on headings and `text-wrap: pretty` on paragraphs.** Native text balancing without JavaScript.

9. **Use `font-variant-numeric: tabular-nums` for data.** Aligns numbers in columns. Critical for tables and dashboards.

10. **Constrain line length with `max-width: 65ch`.** The `ch` unit is based on the width of the "0" character, providing a content-appropriate constraint.

## Real-World Use Cases

### 1. Complete Typography System

```css
@font-face {
  font-family: "Inter Variable";
  src: url("/fonts/InterVariable.woff2") format("woff2-variations");
  font-weight: 100 900;
  font-display: swap;
}

:root {
  --font-sans: "Inter Variable", system-ui, -apple-system, sans-serif;
  --font-mono: ui-monospace, "Cascadia Code", "Fira Code", monospace;

  --step-0: clamp(1rem, 0.89rem + 0.49vw, 1.25rem);
  --step-1: clamp(1.25rem, 1.08rem + 0.73vw, 1.71rem);
  --step-2: clamp(1.56rem, 1.3rem + 1.09vw, 2.24rem);
  --step-3: clamp(1.95rem, 1.56rem + 1.64vw, 2.93rem);
}

body {
  font-family: var(--font-sans);
  font-size: var(--step-0);
  line-height: 1.6;
  font-optical-sizing: auto;
}

h1 { font-size: var(--step-3); line-height: 1.1; font-weight: 750; text-wrap: balance; }
h2 { font-size: var(--step-2); line-height: 1.2; font-weight: 700; text-wrap: balance; }
h3 { font-size: var(--step-1); line-height: 1.3; font-weight: 650; text-wrap: balance; }

p { text-wrap: pretty; max-width: 65ch; }

code { font-family: var(--font-mono); font-variant-ligatures: none; }
```

### 2. Data Table with Aligned Numerals

```css
.data-table {
  font-variant-numeric: tabular-nums lining-nums;
}

.data-table th {
  text-transform: uppercase;
  letter-spacing: 0.05em;
  font-weight: 600;
  font-size: 0.75rem;
}

.data-table td {
  font-variant-numeric: tabular-nums lining-nums;
}

.data-table .currency::before {
  content: "$";
  opacity: 0.5;
}
```

### 3. Card with Truncated Text

```css
.card__title {
  font-size: var(--step-1);
  font-weight: 600;
  line-height: 1.2;

  /* Single-line truncation */
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.card__description {
  font-size: var(--step-0);
  line-height: 1.5;
  color: #555;

  /* Multi-line truncation (3 lines) */
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}

.card__body {
  /* Handle long URLs or user-generated content */
  overflow-wrap: break-word;
}
```

---

> **Key Takeaway:** Self-host WOFF2 variable fonts with `font-display: swap`, use unitless `line-height` values, build fluid type scales with `clamp(rem + vw)` for zoom-safe responsive sizing, and use `font-variant-*` properties (not `font-feature-settings`) for OpenType features like tabular numerals and small caps.
