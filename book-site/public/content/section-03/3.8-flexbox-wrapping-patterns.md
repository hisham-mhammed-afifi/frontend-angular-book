# 3.8 Flexbox Wrapping Patterns

> Flexbox wrapping turns a one-dimensional layout engine into a powerful responsive tool. This guide covers wrapping behaviors, intrinsic responsive patterns without media queries, and production layout techniques. General flexbox fundamentals are covered in 2.6; responsive grid patterns are covered in 3.7.

---

## How Wrapping Works

The `flex-wrap` property controls whether flex items stay on a single line or flow onto new lines. The default is `nowrap`, which forces all items onto one line (shrinking them if necessary).

```css
.container {
  display: flex;
  flex-wrap: wrap; /* items wrap to new lines when they overflow */
}
```

### The Three Values

| Value | Behavior |
|---|---|
| `nowrap` (default) | All items on one line. Items shrink to fit. Overflow if they can't. |
| `wrap` | Items wrap to the next line in the cross-axis direction (downward for `row`). |
| `wrap-reverse` | Items wrap in the reverse cross-axis direction (upward for `row`). |

### When Does Wrapping Actually Happen?

Items wrap when their combined `flex-basis` values (plus gaps and margins) exceed the container's main axis size. The critical point: wrapping is evaluated **before** shrinking. Once `flex-wrap: wrap` is set, the browser prefers wrapping over shrinking.

```css
/* Three items, each with flex-basis: 300px. Container is 800px wide. */
/* 300 + 300 + 300 = 900 > 800, so the third item wraps. */
.item {
  flex: 0 1 300px;
}
```

If `flex-wrap` were `nowrap`, the same items would shrink from 300px each to ~267px to fit. With `wrap`, the third item drops to a new line instead.

## flex-wrap: wrap-reverse

`wrap-reverse` reverses the cross-axis direction of wrapped lines. New lines appear **above** the first line (for `flex-direction: row`).

```html
<div class="container">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
  <div class="item">4</div>
  <div class="item">5</div>
</div>
```

```css
.container {
  display: flex;
  flex-wrap: wrap-reverse;
  width: 400px;
}

.item {
  flex: 0 0 150px;
  height: 80px;
}
```

**Visual result (items 1-5, 150px each, in a 400px container):**

```
/* wrap: */
Row 1 (top):    [1] [2]
Row 2 (bottom): [3] [4]
Row 3:          [5]

/* wrap-reverse: */
Row 3 (top):    [5]
Row 2:          [3] [4]
Row 1 (bottom): [1] [2]
```

### Practical Use Cases for wrap-reverse

**Bottom-anchored chat messages.** Messages stack upward from the bottom, just like chat UIs.

```css
.chat-messages {
  display: flex;
  flex-direction: column-reverse; /* newest at the bottom, scroll up for old */
}
```

**Form with actions pinned to bottom.** When a form wraps on narrow screens, `wrap-reverse` can keep the action bar visually above the form fields if the design calls for it.

**Tag clouds where overflow should appear above.** On confined widgets, new rows of tags appear above rather than pushing content down.

## flex-basis, min-width, and Wrapping

`flex-basis` is the starting size of an item before grow/shrink kicks in. It directly controls when wrapping happens.

### flex-basis vs width

Both set the initial main-axis size, but `flex-basis` wins when both are set:

```css
.item {
  width: 200px;
  flex-basis: 300px; /* 300px is used, width is ignored for flex calculations */
}
```

| Property | Respects min/max-width? | Used by flex algorithm? |
|---|---|---|
| `width` | Yes | Only when `flex-basis: auto` |
| `flex-basis` | Yes, bounded by min/max-width | Always (overrides width) |

### flex-basis: 0 vs auto

This distinction is critical for wrapping behavior:

```css
/* flex-basis: auto */
/* Item's starting size = its content size (or explicit width if set). */
/* Wrapping decision is based on content widths. */
.item-auto {
  flex: 1 1 auto;
}

/* flex-basis: 0 */
/* Item's starting size = 0. All space is treated as distributable. */
/* Items are less likely to wrap because their basis is 0. */
.item-zero {
  flex: 1 1 0;
}
```

**The gotcha:** `flex: 1` is shorthand for `flex: 1 1 0`. This means items start at 0 width and grow equally. With wrapping enabled, items with `flex-basis: 0` are unlikely to wrap because their combined basis is 0, which is always less than the container width. The items just grow to fill the line.

```css
/* Bad Practice: flex: 1 with flex-wrap: wrap. Items never wrap. */
.container {
  display: flex;
  flex-wrap: wrap;
}
.item {
  flex: 1; /* flex: 1 1 0 -- basis is 0, so nothing triggers wrapping */
}

/* Good: set a meaningful flex-basis to trigger wrapping */
.item {
  flex: 1 1 250px; /* wraps when items can't fit at 250px each */
}
```

### The min-width: auto Trap

Flex items have `min-width: auto` by default, which prevents them from shrinking below their content size. This interacts with wrapping in unexpected ways:

```css
/* Problem: a long word or URL prevents the item from shrinking */
.item {
  flex: 1 1 200px;
  /* min-width: auto (default) -- item won't shrink below its content */
}

/* Fix: allow shrinking below content size */
.item {
  flex: 1 1 200px;
  min-width: 0; /* now the item can shrink below its content width */
}
```

The same applies to `min-height: auto` when `flex-direction` is `column`.

## Responsive Patterns Without Media Queries

The real power of flex wrapping is building layouts that respond to container width without any `@media` rules.

### Pattern 1: The Fluid Card Row

Items maintain a minimum width and wrap automatically:

```html
<div class="card-row">
  <div class="card">Card 1</div>
  <div class="card">Card 2</div>
  <div class="card">Card 3</div>
  <div class="card">Card 4</div>
</div>
```

```css
.card-row {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card {
  flex: 1 1 280px; /* grow to fill, shrink allowed, minimum 280px before wrapping */
}
```

**Behavior at different widths:**

| Container Width | Cards per Row | Card Width |
|---|---|---|
| 1200px | 4 | ~290px each |
| 900px | 3 | ~287px each |
| 600px | 2 | ~292px each |
| 400px | 1 | 400px |

No breakpoints. The layout responds to whatever container it lives in.

### Pattern 2: The "Deconstructed Pancake" (flex: auto)

A row of items that wraps to full width on small screens:

```css
.container {
  display: flex;
  flex-wrap: wrap;
}

/* Each item takes its content width, grows to fill, and wraps to 100% when space is tight */
.item {
  flex: 1 1 auto;
  min-width: 150px; /* wrapping threshold */
}
```

### Pattern 3: Sidebar + Main Content

```css
.layout {
  display: flex;
  flex-wrap: wrap;
}

.sidebar {
  flex: 1 1 200px; /* sidebar minimum 200px, grows proportionally */
}

.main {
  flex: 3 1 400px; /* main gets 3x the growth, minimum 400px */
}
```

At wide widths, the sidebar and main sit side by side (sidebar ~200px, main ~600px in an 800px container). When the container drops below 600px, the main content can't fit at 400px beside a 200px sidebar, so they stack vertically.

### Pattern 4: The Holy Albatross

Heydon Pickering's technique switches between horizontal and vertical layout without media queries, using a `calc()` trick:

```css
.container {
  display: flex;
  flex-wrap: wrap;
  --breakpoint: 600px;
}

.item {
  /* When container > 600px: (600px - 100%) is negative, clamped to 0 by min-width
     When container < 600px: (600px - 100%) is positive, multiplied by 999 = huge,
     clamped to 100% by max-width */
  flex-grow: 1;
  flex-basis: calc((var(--breakpoint) - 100%) * 999);
  min-width: 0;
  /* Items need a max-width to prevent them from growing beyond 100% */
}
```

**How it works:**

1. `flex-basis: calc((600px - 100%) * 999)` evaluates the container width vs. the breakpoint.
2. When the container is wider than 600px: `(600px - 800px) * 999 = -199800px`. Negative basis is treated as 0. Items share the row.
3. When the container is narrower than 600px: `(600px - 400px) * 999 = 199800px`. Each item wants 199800px, which is clamped to 100% of the container. Items stack.

**Simplified modern version using clamp():**

```css
.item {
  flex: 1 1 clamp(0px, (600px - 100%) * 999, 100%);
}
```

This is the closest flexbox gets to container queries without using `@container`.

## The gap Property with Wrapping

The `gap` property (formerly `grid-gap`) works in flex containers and applies spacing only between items, not on outer edges.

```css
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;          /* equal row and column gap */
  gap: 1.5rem 1rem;   /* row-gap column-gap */
}
```

### gap vs margin: Why gap Wins

```css
/* Bad Practice: margin on items for spacing */
.item {
  margin: 0.5rem; /* adds margin to outer edges too */
}
/* requires negative margin hack on container: */
.container {
  margin: -0.5rem;
}

/* Good: gap handles spacing cleanly */
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
```

### The gap + flex-basis Calculation Problem

Gap is **not** subtracted from the available space before flex-basis is evaluated. This means items with percentage-based widths can wrap prematurely:

```css
/* Problem: 3 items at 33.33% + gaps = more than 100% */
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.item {
  flex: 0 0 33.33%; /* 3 * 33.33% = 99.99% PLUS 2 * 1rem gap = overflow, wraps! */
}
```

**Fix: account for gap in the flex-basis calculation:**

```css
.item {
  /* 3 items per row, 2 gaps of 1rem between them */
  /* Each item = (100% - total gap) / number of items */
  flex: 0 0 calc((100% - 2rem) / 3);
}
```

**General formula:**

```
flex-basis: calc((100% - (N - 1) * gap) / N)
```

Where N is the desired number of items per row.

```css
/* 4 items per row with 20px gap */
.item-4 {
  flex: 0 0 calc((100% - 3 * 20px) / 4);
}

/* 3 items per row with 1.5rem gap */
.item-3 {
  flex: 0 0 calc((100% - 2 * 1.5rem) / 3);
}
```

**Why CSS Grid does not have this problem:** Grid subtracts gaps from the available space before sizing tracks. `repeat(3, 1fr)` with `gap: 1rem` always gives you 3 equal columns. This is the single biggest advantage of Grid for grid-like layouts.

## flex-grow, flex-shrink, and Wrapping Interaction

### How Grow/Shrink Work Per-Line

Each wrapped flex line is its own flex context. Items on line 1 grow and shrink independently of items on line 2.

```css
.container {
  display: flex;
  flex-wrap: wrap;
  width: 600px;
}

/* 5 items, each 200px basis. Line 1: items 1-3 (600px). Line 2: items 4-5 (400px). */
.item {
  flex: 1 1 200px; /* flex-grow: 1 on each line */
}
```

**Result:**
- Line 1: 3 items, each exactly 200px (600px / 3, no leftover space).
- Line 2: 2 items. 200px leftover space. Each item grows by 100px to 300px.

This is why a flexbox "grid" looks ragged. The last row's items are wider than previous rows because there is more space to distribute among fewer items.

### flex-shrink Has No Effect When Wrapping

When `flex-wrap: wrap` is active, items wrap to a new line rather than shrinking. `flex-shrink` only applies within a single line that cannot wrap further (i.e., when each line has exactly one item that is larger than the container).

```css
.item {
  flex: 0 0 300px; /* flex-shrink: 0, won't shrink even if it could */
}
/* With flex-wrap: wrap, shrink value is mostly irrelevant.
   Items wrap instead of shrinking. */
```

### The flex Shorthand Cheat Sheet

| Shorthand | Expands To | Wrapping Behavior |
|---|---|---|
| `flex: 1` | `1 1 0` | Items grow equally but never wrap (basis is 0) |
| `flex: auto` | `1 1 auto` | Items grow from content size, wrap based on content |
| `flex: none` | `0 0 auto` | Items are rigid, wrap based on content size |
| `flex: 0 1 250px` | as written | Items wrap at 250px threshold, can shrink within a line |
| `flex: 1 1 250px` | as written | Items wrap at 250px threshold, then grow to fill |

## The Last Row Problem

The most common flexbox wrapping pain point: when the last row has fewer items, those items stretch or space out inconsistently.

### The Problem

```html
<div class="grid">
  <div class="card">1</div>
  <div class="card">2</div>
  <div class="card">3</div>
  <div class="card">4</div>
  <div class="card">5</div> <!-- orphan on last row -->
</div>
```

```css
/* Bad Practice: last row items stretch to fill */
.grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.card {
  flex: 1 1 250px;
}
/* With 5 cards in a row of 3: row 1 has 3 equal cards,
   row 2 has 2 cards that each stretch to 50% of the width. */
```

### Solution 1: Use flex-grow: 0 with a Fixed Basis

```css
.card {
  flex: 0 0 calc((100% - 2rem) / 3); /* exactly 3 per row, last row stays left-aligned */
}
```

Items don't grow, so orphans on the last row keep the same width as items on previous rows.

### Solution 2: Invisible Spacer Elements

Add hidden elements to fill the last row:

```html
<div class="grid">
  <div class="card">1</div>
  <div class="card">2</div>
  <div class="card">3</div>
  <div class="card">4</div>
  <div class="card">5</div>
  <div class="card spacer" aria-hidden="true"></div> <!-- fills the gap -->
</div>
```

```css
.spacer {
  height: 0;
  padding: 0;
  border: none;
  flex: 0 0 calc((100% - 2rem) / 3); /* same width as real cards */
}
```

### Solution 3: Pseudo-Element Filler

```css
.grid::after {
  content: "";
  flex: 0 0 calc((100% - 2rem) / 3); /* match item width */
}
```

This only works when the last row needs exactly one filler. For rows that could be short by 2+ items, you need multiple spacers.

### Solution 4: Use CSS Grid Instead

CSS Grid solves this natively. Items don't stretch because they follow the grid tracks:

```css
/* Grid: last row items stay in their columns */
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}
/* Item 5 occupies column 2, columns 2-3 are empty. No stretching. */
```

For auto-responsive wrapping:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 1rem;
}
```

### Comparison: Flexbox vs Grid for the Last Row

| Aspect | Flexbox | CSS Grid |
|---|---|---|
| Last row behavior | Items stretch/space out | Items stay in their column tracks |
| Fix complexity | Requires workarounds | Works correctly by default |
| Responsive wrapping | Built-in with `flex-wrap` | Built-in with `auto-fill`/`auto-fit` |
| Item alignment across rows | Independent per line | Aligned to grid tracks |

**Rule of thumb:** If items must be the same width across all rows (a "grid" of cards), use CSS Grid. If items can have varying widths and you want flexible distribution, use flexbox wrapping.

## align-content: Controlling Wrapped Lines

`align-content` controls the spacing between wrapped flex lines along the cross axis. It only has an effect when there are multiple lines (i.e., `flex-wrap: wrap` or `wrap-reverse` is set) **and** the container has extra cross-axis space (explicit height larger than content).

```css
.container {
  display: flex;
  flex-wrap: wrap;
  height: 500px; /* extra vertical space triggers align-content behavior */
}
```

### align-content Values

```css
.stretch       { align-content: stretch; }       /* default: lines stretch to fill */
.start         { align-content: flex-start; }     /* lines packed to start */
.end           { align-content: flex-end; }       /* lines packed to end */
.center        { align-content: center; }         /* lines centered */
.between       { align-content: space-between; }  /* first line at start, last at end */
.around        { align-content: space-around; }   /* equal space around each line */
.evenly        { align-content: space-evenly; }   /* equal space between and at edges */
```

### Visual comparison (3 lines in a 500px tall container, content height 300px):

```
stretch (default):
|[Line 1 -- 167px tall ]|
|[Line 2 -- 167px tall ]|
|[Line 3 -- 167px tall ]|

flex-start:
|[Line 1 -- 100px]|
|[Line 2 -- 100px]|
|[Line 3 -- 100px]|
|                  |  <-- 200px empty
|                  |

space-between:
|[Line 1 -- 100px]|
|                  |  <-- 100px gap
|[Line 2 -- 100px]|
|                  |  <-- 100px gap
|[Line 3 -- 100px]|

center:
|                  |  <-- 100px empty
|[Line 1 -- 100px]|
|[Line 2 -- 100px]|
|[Line 3 -- 100px]|
|                  |  <-- 100px empty
```

### Common Mistake: align-content on Single-Line Containers

```css
/* Bad Practice: align-content has no effect here */
.container {
  display: flex;
  /* flex-wrap: nowrap (default) */
  align-content: center; /* does nothing -- only one line */
}

/* For single-line containers, use align-items instead */
.container {
  display: flex;
  align-items: center; /* centers items along the cross axis */
}
```

### align-content vs align-items with Wrapping

| Property | Controls | Requires wrap? |
|---|---|---|
| `align-items` | Alignment of items within their line | No |
| `align-content` | Distribution of lines within the container | Yes |

When both are set on a wrapping container:

```css
.container {
  display: flex;
  flex-wrap: wrap;
  height: 600px;
  align-content: center;    /* centers the group of lines vertically */
  align-items: flex-start;  /* within each line, items align to the top of that line */
}
```

## The order Property and Wrapping

The `order` property controls visual rendering order without changing the DOM. Default is 0. Items are laid out in ascending order value, then by source order within the same value.

### order with Wrapped Layouts

```css
.container {
  display: flex;
  flex-wrap: wrap;
}

/* Source order: A B C D E */
/* Visual order after reordering: */
.item-c { order: -1; } /* C moves before A */
/* Rendered: C A B D E */
/* Wrapping happens on the reordered sequence */
```

### Responsive Reordering with Media Queries

```html
<div class="layout">
  <aside class="sidebar">Sidebar</aside>
  <main class="content">Main Content</main>
  <nav class="nav">Navigation</nav>
</div>
```

```css
.layout {
  display: flex;
  flex-wrap: wrap;
}

.sidebar { flex: 1 1 200px; order: 1; }
.content { flex: 3 1 400px; order: 2; }
.nav     { flex: 1 1 100%;  order: 3; }

/* On small screens, rearrange so nav appears first */
@media (max-width: 600px) {
  .nav     { order: -1; flex-basis: 100%; }
  .content { order: 0;  flex-basis: 100%; }
  .sidebar { order: 1;  flex-basis: 100%; }
}
```

**Wide screen layout:**
```
[Sidebar 200px] [Main Content ~600px]
[Navigation 100%                     ]
```

**Narrow screen layout:**
```
[Navigation 100%]
[Main Content 100%]
[Sidebar 100%]
```

### Accessibility Warning

`order` changes only the visual rendering. Tab order and screen reader order still follow the DOM. If the visual and DOM order differ significantly, keyboard users will experience a confusing tab sequence.

```css
/* Bad Practice: drastically reordering interactive elements */
.first-button  { order: 3; }
.second-button { order: 1; }
.third-button  { order: 2; }
/* Tab order: first -> second -> third (DOM order) */
/* Visual order: second -> third -> first */
/* Keyboard users will be confused */
```

Use `order` only for minor visual tweaks (moving a sidebar, adjusting decorative elements). For significant reordering of interactive content, change the DOM order instead.

## Production Pattern: Responsive Navigation Bar

A navigation bar that lays out horizontally and wraps gracefully:

```html
<nav class="main-nav">
  <a href="/" class="nav-logo">Logo</a>
  <a href="/products" class="nav-link">Products</a>
  <a href="/pricing" class="nav-link">Pricing</a>
  <a href="/docs" class="nav-link">Documentation</a>
  <a href="/blog" class="nav-link">Blog</a>
  <a href="/contact" class="nav-link">Contact</a>
</nav>
```

```css
.main-nav {
  display: flex;
  flex-wrap: wrap;
  align-items: center;
  gap: 0.25rem 1rem; /* small vertical gap, larger horizontal gap */
  padding: 0.75rem 1.5rem;
}

.nav-logo {
  flex: 0 0 auto;   /* logo never shrinks or grows */
  margin-right: auto; /* pushes all nav links to the right */
  font-weight: bold;
}

.nav-link {
  flex: 0 0 auto;   /* links sized by content */
  padding: 0.5rem 0;
  text-decoration: none;
}
```

**Behavior:**
- Wide screen: logo on the left, all links on the right in a single row.
- Medium screen: links that don't fit wrap to a second row below.
- Narrow screen: all links wrap below the logo, each on its own line if necessary.

### Enhanced: Navigation That Stacks Without Media Queries

```css
.main-nav {
  display: flex;
  flex-wrap: wrap;
  gap: 0.5rem 1.5rem;
  padding: 1rem;
}

.nav-logo {
  flex: 1 1 100%; /* logo takes full width, pushing links to next line */
}

.nav-link {
  flex: 1 1 auto;
  min-width: max-content; /* prevent link text from wrapping */
  text-align: center;
  padding: 0.5rem 1rem;
}

/* On wider containers, logo shares the row with links */
@media (min-width: 768px) {
  .nav-logo {
    flex: 0 0 auto; /* logo reverts to content width */
  }
}
```

## Production Pattern: Card Grid with Wrapping

```html
<div class="card-grid">
  <article class="card">
    <img src="image.jpg" alt="" class="card-image">
    <div class="card-body">
      <h3 class="card-title">Title</h3>
      <p class="card-text">Description text goes here.</p>
    </div>
    <footer class="card-footer">
      <button>Action</button>
    </footer>
  </article>
  <!-- more cards... -->
</div>
```

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1.5rem;
}

.card {
  /* Each card is at least 280px. Grows to fill available space. */
  flex: 1 1 280px;
  /* Cap card width to prevent single cards from being absurdly wide */
  max-width: 400px;

  /* Card itself is a flex column for internal layout */
  display: flex;
  flex-direction: column;
}

.card-image {
  width: 100%;
  aspect-ratio: 16 / 9;
  object-fit: cover;
}

.card-body {
  flex: 1; /* body grows to push footer down */
  padding: 1rem;
}

.card-footer {
  padding: 1rem;
  margin-top: auto; /* alternative: pushes footer to bottom */
}
```

### Fixing the Last Row

```css
/* Prevent last-row cards from stretching */
.card-grid::after {
  content: "";
  flex: 1 1 280px; /* same basis as cards */
  max-width: 400px;
  /* Invisible filler that takes up space like a card */
}
```

For layouts where 2 or more fillers might be needed:

```css
/* Add multiple spacers via extra empty elements */
.card-grid .spacer {
  flex: 1 1 280px;
  max-width: 400px;
  height: 0;
  margin: 0;
  padding: 0;
  visibility: hidden;
}
```

### Equal-Height Cards

Flexbox wrapping with `align-items: stretch` (the default) makes all items in the same row the same height:

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1.5rem;
  /* align-items: stretch is the default, making cards equal height per row */
}
```

This only works per row. Cards on row 1 can be a different height than cards on row 2. For all cards to be the same height across all rows, use CSS Grid with `grid-auto-rows: 1fr`.

## Flexbox vs CSS Grid for Responsive Wrapping

### Side-by-Side Comparison

The same responsive card layout in both systems:

```css
/* Flexbox approach */
.flex-cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.flex-cards .card {
  flex: 1 1 250px;
}

/* Grid approach */
.grid-cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 250px), 1fr));
  gap: 1rem;
}
```

### When to Use Which

| Scenario | Use Flexbox Wrapping | Use CSS Grid |
|---|---|---|
| Items should fill available space | Yes | Yes |
| Items must align across rows | No (each line is independent) | Yes (grid tracks align) |
| Variable-width items in a row | Yes (flexbox strength) | Possible but awkward |
| Last row should not stretch | Requires workarounds | Works by default |
| Gap interacts with item sizing | Must account for gap manually | Gap handled automatically |
| Need intrinsic responsive behavior | Yes, with flex-basis | Yes, with auto-fill + minmax |
| Items of unknown/varying count | Both work | Both work |
| Single row of items with distribution | Flexbox is simpler | Overkill |
| Two-dimensional alignment matters | Use Grid | Use Grid |

### The Hybrid Approach

Use Grid for the overall layout and flexbox for components within grid cells:

```css
/* Grid for the page-level card grid */
.product-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 280px), 1fr));
  gap: 1.5rem;
}

/* Flexbox for internal card layout */
.product-card {
  display: flex;
  flex-direction: column;
}

.product-card .tags {
  display: flex;
  flex-wrap: wrap;
  gap: 0.5rem;
}
```

## Common Mistakes and Gotchas

### 1. Forgetting flex-wrap

```css
/* Bad Practice: items overflow or squish */
.container {
  display: flex;
  /* flex-wrap defaults to nowrap */
}

/* Good: explicitly enable wrapping */
.container {
  display: flex;
  flex-wrap: wrap;
}
```

### 2. Using flex: 1 with flex-wrap

```css
/* Bad Practice: flex: 1 sets flex-basis to 0, so items never wrap */
.container {
  display: flex;
  flex-wrap: wrap;
}
.item {
  flex: 1; /* same as flex: 1 1 0 */
}

/* Good: set a meaningful basis */
.item {
  flex: 1 1 200px;
}
```

### 3. Ignoring min-width: auto

```css
/* Bad Practice: item content overflows because min-width: auto prevents shrinking */
.item {
  flex: 1 1 0;
  /* Long text or large images prevent this from shrinking */
}

/* Good: reset min-width */
.item {
  flex: 1 1 0;
  min-width: 0;       /* allow shrinking below content size */
  overflow: hidden;     /* prevent visual overflow */
}
```

### 4. Percentage flex-basis Not Accounting for gap

```css
/* Bad Practice: 50% + 50% + gap > 100%, items wrap unexpectedly */
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.item {
  flex: 0 0 50%;
}

/* Good: subtract gap from percentage */
.item {
  flex: 0 0 calc(50% - 0.5rem);
  /* 2 items: 1 gap of 1rem. Each item absorbs half the gap: 0.5rem */
}
```

### 5. Expecting Two-Dimensional Alignment

```css
/* Bad Practice: expecting wrapped flex items to line up like a grid */
.container {
  display: flex;
  flex-wrap: wrap;
}
.item {
  flex: 1 1 200px; /* items on the last row will be different widths */
}
/* Each flex line is independent. No cross-line alignment. */

/* Good: use CSS Grid when you need items to align in both dimensions */
.container {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}
```

### 6. Not Setting a max-width on Growing Items

```css
/* Bad Practice: single card on last row stretches to full container width */
.card {
  flex: 1 1 300px;
}

/* Good: cap the maximum width */
.card {
  flex: 1 1 300px;
  max-width: 450px; /* prevents a single orphan from stretching absurdly */
}
```

### 7. Confusing align-items and align-content

```css
/* align-items: aligns items WITHIN a flex line (works with single and multi-line) */
/* align-content: aligns the LINES themselves (only works with multi-line) */

.container {
  display: flex;
  flex-wrap: wrap;
  height: 400px;
  align-items: center;     /* centers items vertically within each wrapped line */
  align-content: flex-start; /* packs all lines to the top of the container */
}
```

### 8. Images Breaking Flex Layouts

```css
/* Bad Practice: images have intrinsic dimensions that break flex sizing */
.flex-container img {
  /* Image retains original width, may overflow or cause unexpected wrapping */
}

/* Good: constrain images within flex items */
.flex-container img {
  max-width: 100%;
  height: auto;
  display: block; /* removes inline spacing */
}
```

## Wrapping Behavior Reference Table

A complete reference of how different `flex` values interact with `flex-wrap: wrap`:

| `flex` Value | `flex-basis` | Will items wrap? | Growth on last row? |
|---|---|---|---|
| `flex: 0 0 200px` | 200px | Yes, at 200px | No (grow is 0) |
| `flex: 1 0 200px` | 200px | Yes, at 200px | Yes, items stretch to fill |
| `flex: 1 1 200px` | 200px | Yes, at 200px | Yes, items stretch to fill |
| `flex: 0 1 200px` | 200px | Yes, at 200px | No (grow is 0) |
| `flex: 1` (= `1 1 0`) | 0 | No (basis is 0) | Yes (all space is distributed) |
| `flex: auto` (= `1 1 auto`) | content | Yes, at content size | Yes |
| `flex: none` (= `0 0 auto`) | content | Yes, at content size | No |
| `flex: 0 0 50%` | 50% | Yes, when 2+ items | No |
| `flex: 0 0 calc(33% - 1rem)` | ~32% | Yes, when 3+ items | No |

## Advanced: Responsive Tag/Chip Layout

A pattern where tags of varying width wrap naturally:

```html
<div class="tag-list">
  <span class="tag">JavaScript</span>
  <span class="tag">TypeScript</span>
  <span class="tag">CSS</span>
  <span class="tag">Responsive Design</span>
  <span class="tag">Performance</span>
  <span class="tag">A11y</span>
</div>
```

```css
.tag-list {
  display: flex;
  flex-wrap: wrap;
  gap: 0.5rem;
}

.tag {
  flex: 0 0 auto; /* each tag is exactly its content width, no grow/shrink */
  padding: 0.25rem 0.75rem;
  border-radius: 9999px;
  background: #e2e8f0;
  font-size: 0.875rem;
  white-space: nowrap; /* prevent tag text from breaking */
}
```

Tags flow left to right and wrap naturally. No flex-basis calculation needed because `auto` sizes each tag to its content.

## Advanced: Responsive Form Layout

```html
<form class="form-row">
  <div class="field">
    <label for="first">First Name</label>
    <input type="text" id="first">
  </div>
  <div class="field">
    <label for="last">Last Name</label>
    <input type="text" id="last">
  </div>
  <div class="field field-wide">
    <label for="email">Email</label>
    <input type="email" id="email">
  </div>
</form>
```

```css
.form-row {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.field {
  flex: 1 1 200px; /* fields wrap when below 200px */
}

.field-wide {
  flex: 2 1 300px; /* email gets double the growth, wraps below 300px */
}

.field label {
  display: block;
  margin-bottom: 0.25rem;
}

.field input {
  width: 100%;
  padding: 0.5rem;
  box-sizing: border-box;
}
```

On wide screens, all three fields sit side by side (first name and last name narrower, email wider). On narrow screens, each field takes full width.

## Key Takeaways

1. **flex-basis is the wrapping trigger.** Set it to a meaningful minimum size. `flex-basis: 0` (from `flex: 1`) prevents wrapping entirely.

2. **Wrapping takes priority over shrinking.** Once `flex-wrap: wrap` is set, items wrap to new lines rather than shrinking.

3. **Each wrapped line is independent.** There is no cross-line alignment in flexbox. If you need grid-like alignment across rows, use CSS Grid.

4. **gap does not reduce available space for flex-basis calculations.** Account for gap manually with `calc()` when using percentage-based sizing.

5. **The last row problem is inherent to flexbox.** Use `flex-grow: 0` with fixed basis, spacer elements, or switch to CSS Grid.

6. **align-content only works with wrapping.** It controls spacing between lines, not items within a line.

7. **Use `min-width: 0` defensively.** The `auto` default prevents items from shrinking below content size, causing unexpected overflow.

8. **Flexbox wrapping is ideal for** navigation bars, tag lists, form fields, toolbar buttons, and any layout where items have natural sizes and should reflow. **CSS Grid is better for** uniform card grids, dashboards, and layouts needing two-dimensional alignment.
