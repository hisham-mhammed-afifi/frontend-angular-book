# 4.12 Built-in Modules (`sass:math`, `sass:color`, `sass:list`, `sass:map`, `sass:string`)

> Sass ships seven built-in modules that provide functions for math, color manipulation, string operations, list/map processing, selector introspection, and meta-programming. Since Dart Sass 1.23.0, every built-in function lives behind a `sass:` namespace loaded with `@use`. Global function calls are deprecated as of Dart Sass 1.80.0 and will be removed in Dart Sass 3.0.0.

---

## The Module System for Built-ins

### Loading Built-in Modules

Built-in modules use the `sass:` URL scheme. The `@use` rule loads the module and creates a namespace:

```scss
@use "sass:math";
@use "sass:color";
@use "sass:list";
@use "sass:map";
@use "sass:string";
@use "sass:meta";
@use "sass:selector";
```

Access members with dot notation:

```scss
@use "sass:math";

.circle {
  // Variable access: namespace.$variable
  $circumference: 2 * math.$pi * 50px;

  // Function access: namespace.function()
  width: math.round($circumference);
}
```

### Custom Namespaces

Rename the namespace with `as`:

```scss
@use "sass:math" as m;

.box {
  width: m.div(100%, 3);
  border-radius: m.round(4.7px);  // 5px
}
```

### No Namespace (`as *`)

Load all members into the global scope. Useful for short scripts but risks name collisions:

```scss
@use "sass:math" as *;

.box {
  width: div(100%, 3);      // No prefix needed
  height: round(33.7px);    // 34px
  padding: $pi * 1px;       // 3.1415926536px
}
```

Only use `as *` with built-in modules where you control the entire file. In shared libraries, explicit namespaces prevent collisions.

### Key Rules

| Rule | Detail |
|------|--------|
| Placement | `@use` must appear before any rules, except other `@use` or `@forward` |
| Single load | Each module is loaded once per file, regardless of how many times `@use` appears |
| Immutable variables | Built-in module variables (like `math.$pi`) cannot be reassigned |
| No `@import` support | Built-in modules cannot be loaded via `@import` |

## `sass:math` -- Complete Reference

The math module provides arithmetic operations, trigonometry, exponential functions, unit inspection, and mathematical constants. None of these functions were covered in earlier topics.

### Constants

```scss
@use "sass:math";

// Fundamental constants
$circle: 2 * math.$pi;         // 6.2831853072 (pi = 3.1415926536)
$growth: math.$e;               // 2.7182818285 (Euler's number)

// Floating-point boundaries
$tiny: math.$epsilon;           // 2.220446049250313e-16
$huge: math.$max-number;        // 1.7976931348623157e+308
$small: math.$min-number;       // 5e-324

// Safe integer boundaries
$max-int: math.$max-safe-integer;   // 9007199254740991
$min-int: math.$min-safe-integer;   // -9007199254740991
```

| Constant | Value | Description |
|----------|-------|-------------|
| `$pi` | 3.1415926536 | Ratio of circle circumference to diameter |
| `$e` | 2.7182818285 | Base of the natural logarithm |
| `$epsilon` | ~2.22e-16 | Smallest difference between 1 and the next representable float |
| `$max-number` | ~1.80e+308 | Largest finite 64-bit floating-point number |
| `$min-number` | ~5e-324 | Smallest positive 64-bit floating-point number |
| `$max-safe-integer` | 9007199254740991 | Largest integer where n and n+1 are both exact |
| `$min-safe-integer` | -9007199254740991 | Smallest integer where n and n-1 are both exact |

### Division: `math.div()`

The `/` operator is deprecated for division in Sass. Use `math.div()` instead:

```scss
@use "sass:math";

// math.div($number1, $number2) -> number
.column {
  // Basic division
  width: math.div(100%, 3);           // 33.3333333333%

  // Units cancel when shared
  $ratio: math.div(300px, 100px);     // 3 (unitless)

  // Numerator keeps units, denominator units go to denominator
  $speed: math.div(100px, 2s);        // 50px/s
}
```

```scss
// BAD: Slash division (deprecated, removed in Dart Sass 2.0.0)
.column {
  width: (100% / 3);  // Deprecation warning
}

// GOOD: math.div()
@use "sass:math";
.column {
  width: math.div(100%, 3);
}
```

### Rounding Functions

```scss
@use "sass:math";

// math.ceil($number) -> rounds UP to nearest integer
@debug math.ceil(4.2);      // 5
@debug math.ceil(4.0);      // 4
@debug math.ceil(-4.2);     // -4 (toward positive infinity)
@debug math.ceil(4.2px);    // 5px (preserves units)

// math.floor($number) -> rounds DOWN to nearest integer
@debug math.floor(4.9);     // 4
@debug math.floor(-4.2);    // -5 (toward negative infinity)
@debug math.floor(4.9px);   // 4px

// math.round($number) -> rounds to nearest integer (0.5 rounds up)
@debug math.round(4.5);     // 5
@debug math.round(4.4);     // 4
@debug math.round(-4.5);    // -5
@debug math.round(10.5px);  // 11px
```

Practical use -- pixel-perfect rounding:

```scss
@use "sass:math";

@function px-round($value) {
  @return math.round($value);
}

.sidebar {
  // Avoid subpixel rendering artifacts
  width: px-round(math.div(100vw - 2 * 24px, 3));
}
```

### Clamping and Extremes

```scss
@use "sass:math";

// math.clamp($min, $number, $max) -> constrains value to range
@debug math.clamp(0, -10, 100);       // 0 (below min)
@debug math.clamp(0, 50, 100);        // 50 (within range)
@debug math.clamp(0, 150, 100);       // 100 (above max)
@debug math.clamp(0px, 5px, 10px);    // 5px

// math.max($number...) -> returns highest value
@debug math.max(1px, 4px, 2px);       // 4px
@debug math.max(10%, 50%, 30%);       // 50%

// math.min($number...) -> returns lowest value
@debug math.min(1px, 4px, 2px);       // 1px
```

All arguments to `clamp`, `max`, and `min` must have compatible units. You cannot compare `px` with `%`.

```scss
@use "sass:math";

// BAD: Incompatible units
// @debug math.clamp(0px, 50%, 100px);  // Error: incompatible units

// GOOD: Same unit family
@debug math.clamp(0em, 1.5em, 3em);   // 1.5em
```

### Absolute Value

```scss
@use "sass:math";

// math.abs($number) -> non-negative version
@debug math.abs(-10);     // 10
@debug math.abs(10);      // 10
@debug math.abs(-5.5px);  // 5.5px
```

### Percentage Conversion

```scss
@use "sass:math";

// math.percentage($number) -> equivalent to $number * 100%
// Input MUST be unitless
@debug math.percentage(0.5);    // 50%
@debug math.percentage(0.333);  // 33.3%
@debug math.percentage(2);      // 200%
```

### Random Numbers

```scss
@use "sass:math";

// math.random($limit: null)
// No argument: random decimal in [0, 1)
@debug math.random();      // e.g. 0.7291038462

// With integer $limit >= 1: random integer in [1, $limit]
@debug math.random(10);    // e.g. 7
@debug math.random(100);   // e.g. 42
```

Practical use -- randomized decorations:

```scss
@use "sass:math";

@for $i from 1 through 5 {
  .particle-#{$i} {
    left: math.random(100) * 1%;
    animation-delay: math.random() * 3s;
    opacity: 0.3 + math.random() * 0.7;
  }
}
```

### Exponential Functions

All exponential functions require unitless arguments:

```scss
@use "sass:math";

// math.sqrt($number) -> square root
@debug math.sqrt(100);    // 10
@debug math.sqrt(2);      // 1.4142135624

// math.pow($base, $exponent) -> $base raised to $exponent
@debug math.pow(2, 10);   // 1024
@debug math.pow(10, -2);  // 0.01

// math.log($number, $base: null)
// Natural logarithm (base e) when $base is null
@debug math.log(1);                // 0
@debug math.log(math.$e);         // 1

// Logarithm with specific base
@debug math.log(100, 10);         // 2
@debug math.log(256, 2);          // 8

// math.hypot($number...) -> Euclidean distance
// sqrt(a^2 + b^2 + ...) -- all args need compatible units
@debug math.hypot(3, 4);          // 5
@debug math.hypot(3px, 4px);      // 5px
@debug math.hypot(1, 2, 3);       // 3.7416573868
```

Practical use -- modular type scale with `math.pow()`:

```scss
@use "sass:math";

$base-size: 1rem;
$scale-ratio: 1.25; // Major third

@function type-scale($step) {
  @return $base-size * math.pow($scale-ratio, $step);
}

h1 { font-size: type-scale(5); }  // 3.0517578125rem
h2 { font-size: type-scale(4); }  // 2.44140625rem
h3 { font-size: type-scale(3); }  // 1.953125rem
h4 { font-size: type-scale(2); }  // 1.5625rem
h5 { font-size: type-scale(1); }  // 1.25rem
h6 { font-size: type-scale(0); }  // 1rem
```

### Trigonometric Functions

Input can be an angle with `deg`, `rad`, `grad`, or `turn` units, or a unitless number (treated as radians):

```scss
@use "sass:math";

// math.sin($number) -> unitless result in [-1, 1]
@debug math.sin(0deg);        // 0
@debug math.sin(90deg);       // 1
@debug math.sin(math.$pi);    // 0 (unitless = radians)

// math.cos($number) -> unitless result in [-1, 1]
@debug math.cos(0deg);        // 1
@debug math.cos(180deg);      // -1

// math.tan($number) -> unitless result
@debug math.tan(0deg);        // 0
@debug math.tan(45deg);       // 1 (approximately)
```

Inverse trigonometric functions always return degrees:

```scss
@use "sass:math";

// math.asin($number) -> result in deg
@debug math.asin(0);    // 0deg
@debug math.asin(1);    // 90deg

// math.acos($number) -> result in deg
@debug math.acos(1);    // 0deg
@debug math.acos(0);    // 90deg

// math.atan($number) -> result in deg
@debug math.atan(0);    // 0deg
@debug math.atan(1);    // 45deg

// math.atan2($y, $x) -> result in deg, preserves quadrant
@debug math.atan2(1, 1);     // 45deg   (quadrant I)
@debug math.atan2(1, -1);    // 135deg  (quadrant II)
@debug math.atan2(-1, -1);   // -135deg (quadrant III)
@debug math.atan2(-1, 1);    // -45deg  (quadrant IV)
```

Why `atan2` instead of `atan(div(y, x))`? `atan2` preserves the quadrant. `atan(div(1, -1))` gives -45deg (quadrant IV), but `atan2(1, -1)` correctly gives 135deg (quadrant II).

Practical use -- circular positioning:

```scss
@use "sass:math";

@mixin circle-positions($count, $radius) {
  @for $i from 0 through $count - 1 {
    $angle: math.div(360deg * $i, $count);
    &:nth-child(#{$i + 1}) {
      $x: math.cos($angle) * $radius;
      $y: math.sin($angle) * $radius;
      transform: translate($x, $y);
    }
  }
}

.orbit-item {
  position: absolute;
  @include circle-positions(8, 120px);
}
```

### Unit Inspection Functions

```scss
@use "sass:math";

// math.is-unitless($number) -> boolean
@debug math.is-unitless(10);      // true
@debug math.is-unitless(10px);    // false
@debug math.is-unitless(10%);     // false

// math.compatible($number1, $number2) -> boolean
// Can these be added/subtracted/compared?
@debug math.compatible(1px, 2px);    // true
@debug math.compatible(1px, 2in);    // true  (both are lengths)
@debug math.compatible(1px, 2em);    // false (px is absolute, em is relative)
@debug math.compatible(1px, 2%);     // false

// math.unit($number) -> quoted string (debugging only)
@debug math.unit(10px);          // "px"
@debug math.unit(10px * 2px);    // "px*px"
@debug math.unit(math.div(10px, 2s));  // "px/s"
@debug math.unit(10);            // ""
```

Practical use -- unit validation in functions:

```scss
@use "sass:math";
@use "sass:meta";

@function to-rem($value, $base: 16px) {
  @if not math.compatible($value, 1px) {
    @error "to-rem() expects a length value, got #{meta.type-of($value)}: #{$value}";
  }
  @if math.is-unitless($value) {
    @warn "Unitless value #{$value} passed to to-rem(). Assuming px.";
    $value: $value * 1px;
  }
  @return math.div($value, $base) * 1rem;
}

.box {
  padding: to-rem(24px);   // 1.5rem
  margin: to-rem(32px);    // 2rem
}
```

## `sass:color` -- Complete Reference

The color module handles color creation, channel inspection, color space conversion, and color manipulation. With Dart Sass 1.79.0+, it fully supports CSS Color Level 4 color spaces.

### Color Spaces Overview

Sass supports all predefined color spaces from the CSS Color Level 4 specification:

| Category | Space | CSS Function | Channels | Gamut |
|----------|-------|-------------|----------|-------|
| Legacy | `rgb` | `rgb()` / hex | red [0,255], green [0,255], blue [0,255] | sRGB |
| Legacy | `hsl` | `hsl()` | hue [0,360], saturation [0%,100%], lightness [0%,100%] | sRGB |
| Legacy | `hwb` | `hwb()` | hue [0,360], whiteness [0%,100%], blackness [0%,100%] | sRGB |
| Modern | `srgb` | `color(srgb ...)` | red [0,1], green [0,1], blue [0,1] | sRGB |
| Modern | `srgb-linear` | `color(srgb-linear ...)` | red [0,1], green [0,1], blue [0,1] | sRGB |
| Modern | `display-p3` | `color(display-p3 ...)` | red [0,1], green [0,1], blue [0,1] | Display P3 |
| Modern | `a98-rgb` | `color(a98-rgb ...)` | red [0,1], green [0,1], blue [0,1] | Adobe RGB |
| Modern | `prophoto-rgb` | `color(prophoto-rgb ...)` | red [0,1], green [0,1], blue [0,1] | ProPhoto |
| Modern | `rec2020` | `color(rec2020 ...)` | red [0,1], green [0,1], blue [0,1] | Rec. 2020 |
| CIE | `lab` | `lab()` | lightness [0%,100%], a [-125,125], b [-125,125] | Unbounded |
| CIE | `lch` | `lch()` | lightness [0%,100%], chroma [0,150], hue [0deg,360deg] | Unbounded |
| Perceptual | `oklab` | `oklab()` | lightness [0%,100%], a [-0.4,0.4], b [-0.4,0.4] | Unbounded |
| Perceptual | `oklch` | `oklch()` | lightness [0%,100%], chroma [0,0.4], hue [0deg,360deg] | Unbounded |
| CIE XYZ | `xyz` / `xyz-d65` | `color(xyz ...)` | x [0,1], y [0,1], z [0,1] | Unbounded |
| CIE XYZ | `xyz-d50` | `color(xyz-d50 ...)` | x [0,1], y [0,1], z [0,1] | Unbounded |

Legacy spaces (`rgb`, `hsl`, `hwb`) auto-convert between each other. Modern spaces preserve their original space unless explicitly converted.

### Channel Inspection

```scss
@use "sass:color";

$brand: #c71585;

// color.channel($color, $channel, $space: null) -> number
// $channel is a QUOTED string, $space is UNQUOTED
@debug color.channel($brand, "red", $space: rgb);         // 199
@debug color.channel($brand, "green", $space: rgb);       // 21
@debug color.channel($brand, "blue", $space: rgb);        // 133
@debug color.channel($brand, "hue", $space: hsl);         // 322.2471910112deg
@debug color.channel($brand, "saturation", $space: hsl);  // 80.9090909091%
@debug color.channel($brand, "lightness", $space: hsl);   // 43.1372549020%
@debug color.channel($brand, "alpha");                     // 1

// Inspect channels in modern spaces
$p3: color(display-p3 0.7 0.1 0.5);
@debug color.channel($p3, "red", $space: display-p3);     // 0.7
@debug color.channel($p3, "lightness", $space: oklch);    // (converted value)
@debug color.channel($p3, "chroma", $space: oklch);       // (converted value)
```

### Space Inspection and Conversion

```scss
@use "sass:color";

$hex: #663399;
$p3: color(display-p3 0.4 0.2 0.6);

// color.space($color) -> unquoted string
@debug color.space($hex);    // rgb
@debug color.space($p3);     // display-p3

// color.is-legacy($color) -> boolean
@debug color.is-legacy($hex);    // true
@debug color.is-legacy($p3);     // false

// color.to-space($color, $space) -> color in new space
$as-oklch: color.to-space($hex, oklch);
@debug $as-oklch;                       // oklch(...)
@debug color.space($as-oklch);          // oklch

// Converting back preserves the round-trip
$back: color.to-space($as-oklch, rgb);
@debug $back;   // approximately rgb(102, 51, 153)
```

### Gamut Operations

Colors in wide-gamut spaces can fall outside the gamut of narrower spaces. Sass can represent out-of-gamut values but provides tools to check and correct them:

```scss
@use "sass:color";

$wide: oklch(70% 0.25 150deg);

// color.is-in-gamut($color, $space: null) -> boolean
@debug color.is-in-gamut($wide);              // true (oklch is unbounded)
@debug color.is-in-gamut($wide, $space: rgb); // false (too vivid for sRGB)

// color.to-gamut($color, $space, $method: null) -> color
// $method is REQUIRED: "local-minde" (recommended) or "clip"
$safe: color.to-gamut($wide, rgb, $method: local-minde);
@debug $safe;  // An sRGB-safe color visually close to the original

// "clip" simply clamps channels to the valid range (less accurate)
$clipped: color.to-gamut($wide, rgb, $method: clip);
```

The `local-minde` algorithm reduces chroma in oklch space to find the closest in-gamut color, preserving hue and lightness. `clip` just clamps channel values, which can shift perceived color significantly.

### Missing and Powerless Channels

```scss
@use "sass:color";

// Missing channels use the CSS "none" keyword
$transparent-blue: hsl(240deg 100% 50% / none);

// color.is-missing($color, $channel) -> boolean
@debug color.is-missing($transparent-blue, "alpha");  // true
@debug color.is-missing($transparent-blue, "hue");    // false

// Powerless channels: channels that have no visual effect
// Example: hue is powerless when saturation/chroma is 0
$gray: hsl(0deg 0% 50%);
// color.is-powerless($color, $channel, $space: null) -> boolean
@debug color.is-powerless($gray, "hue");               // true
@debug color.is-powerless($gray, "hue", $space: hsl);  // true
```

### Color Sameness

```scss
@use "sass:color";

// color.same($color1, $color2) -> boolean
// Compares colors by converting to xyz space, not by channel values
$a: hsl(0deg 100% 50%);
$b: rgb(255, 0, 0);
@debug color.same($a, $b);  // true (same visual color)

$c: oklch(62.8% 0.258 29.23deg);
@debug color.same($a, $c);  // true (approximately the same)
```

### `color.adjust()` -- Fixed Amount Changes

Increases or decreases one or more channels by fixed amounts:

```scss
@use "sass:color";

$base: #c71585;

// Adjust in the color's own space (legacy = rgb)
@debug color.adjust($base, $red: 50);              // #f91585
@debug color.adjust($base, $blue: -50);            // #c71553

// Adjust in a different space with $space
@debug color.adjust($base, $lightness: 20%, $space: hsl);
@debug color.adjust($base, $hue: 30deg, $space: hsl);

// Adjust alpha
@debug color.adjust($base, $alpha: -0.3);          // rgba(199, 21, 133, 0.7)

// Adjust in perceptually uniform space
@debug color.adjust($base, $lightness: 10%, $space: oklch);
@debug color.adjust($base, $chroma: -0.05, $space: oklch);
```

Channel mixing restrictions: you cannot mix RGB channels with HSL or HWB channels in the same call. Use the `$space` parameter to target the space you want.

### `color.scale()` -- Proportional Changes

Scales channels toward their minimum or maximum. Arguments are percentages from -100% to 100%:

```scss
@use "sass:color";

$base: #c71585;

// Scale lightness UP by 30% (toward maximum)
@debug color.scale($base, $lightness: 30%);

// Scale lightness DOWN by 30% (toward minimum)
@debug color.scale($base, $lightness: -30%);

// Scale alpha down for a semi-transparent version
@debug color.scale($base, $alpha: -50%);    // 50% of current alpha

// Scale in a specific space
@debug color.scale($base, $lightness: 20%, $space: oklch);
```

Why `color.scale()` over `color.adjust()`? `scale()` is relative to the current value. Adjusting lightness by 10% on an already-light color may clip. Scaling by 30% always moves 30% of the remaining distance toward max, never overshooting:

```scss
@use "sass:color";

$light: hsl(0deg 100% 90%);

// BAD: adjust can overshoot and clip
@debug color.adjust($light, $lightness: 15%, $space: hsl);
// lightness becomes 105%, clamped to 100% = pure white

// GOOD: scale approaches max proportionally
@debug color.scale($light, $lightness: 50%);
// lightness moves 50% of remaining distance (100% - 90% = 10%)
// Result: 95% lightness -- still tinted, not pure white
```

### `color.change()` -- Set Absolute Values

Sets one or more channels to exact values (not relative adjustments):

```scss
@use "sass:color";

$base: #c71585;

// Set specific channel values
@debug color.change($base, $red: 255);                     // #ff1585
@debug color.change($base, $alpha: 0.5);                   // rgba(199, 21, 133, 0.5)
@debug color.change($base, $hue: 120deg, $space: hsl);     // Change hue to green
@debug color.change($base, $lightness: 50%, $space: oklch); // Set oklch lightness
```

Unlike `adjust()` and `scale()`, `change()` never clamps values. The raw value is stored even if out of gamut.

### `color.mix()` -- Color Mixing

Mixes two colors using the CSS `color-mix()` algorithm:

```scss
@use "sass:color";

$red: #ff0000;
$blue: #0000ff;

// Basic mix (50/50 by default)
@debug color.mix($red, $blue);                 // #800080 (purple)

// Weighted mix
@debug color.mix($red, $blue, 25%);            // 25% red, 75% blue
@debug color.mix($red, $blue, 75%);            // 75% red, 25% blue

// Mix in a specific color space with $method
@debug color.mix($red, $blue, $method: oklch);
@debug color.mix($red, $blue, 50%, $method: oklch);

// Hue interpolation method for polar spaces
@debug color.mix($red, $blue, $method: "oklch shorter hue");
@debug color.mix($red, $blue, $method: "oklch longer hue");
@debug color.mix($red, $blue, $method: "oklch increasing hue");
@debug color.mix($red, $blue, $method: "oklch decreasing hue");
```

The `$method` parameter accepts a color space name optionally followed by a hue interpolation method (`shorter hue`, `longer hue`, `increasing hue`, `decreasing hue`). This only applies to polar spaces (hsl, hwb, lch, oklch).

### `color.complement()` -- Hue Rotation by 180 Degrees

```scss
@use "sass:color";

$brand: #c71585;

// color.complement($color, $space: null)
// Rotates hue 180 degrees in the given polar space
@debug color.complement($brand, $space: hsl);     // Opposite on the HSL wheel
@debug color.complement($brand, $space: oklch);    // Perceptually opposite

// Without $space, defaults to hsl for legacy colors
@debug color.complement($brand);
```

The `$space` must be a polar color space (one with a hue channel): `hsl`, `hwb`, `lch`, or `oklch`.

### `color.invert()` -- Color Negation

```scss
@use "sass:color";

$base: #336699;

// color.invert($color, $weight: 100%, $space: null)
@debug color.invert($base, $space: rgb);           // Full inversion
@debug color.invert($base, 50%, $space: rgb);      // 50% inversion (toward gray)
@debug color.invert($base, $space: oklch);          // Perceptual inversion
```

### `color.grayscale()` -- Remove Color

```scss

// For legacy colors: sets HSL saturation to 0%
// For modern colors: sets oklch chroma to 0

```

### `color.hwb()` -- Create HWB Colors


| `color.blue($color)` | `color.channel($color, "blue", $space: rgb)` |
| `color.hue($color)` | `color.channel($color, "hue", $space: hsl)` |
| `color.saturation($color)` | `color.channel($color, "saturation", $space: hsl)` |
| `color.lightness($color)` | `color.channel($color, "lightness", $space: hsl)` |
| `color.whiteness($color)` | `color.channel($color, "whiteness", $space: hwb)` |
| `color.blackness($color)` | `color.channel($color, "blackness", $space: hwb)` |
| `color.alpha($color)` | `color.channel($color, "alpha")` |
| `adjust-hue($color, $deg)` | `color.adjust($color, $hue: $deg, $space: hsl)` |
| `lighten($color, $amount)` | `color.adjust($color, $lightness: $amount, $space: hsl)` or `color.scale()` |
| `darken($color, $amount)` | `color.adjust($color, $lightness: -$amount, $space: hsl)` or `color.scale()` |
| `saturate($color, $amount)` | `color.adjust($color, $saturation: $amount, $space: hsl)` or `color.scale()` |
| `desaturate($color, $amount)` | `color.adjust($color, $saturation: -$amount, $space: hsl)` or `color.scale()` |
| `opacify($color, $amount)` / `fade-in()` | `color.adjust($color, $alpha: $amount)` or `color.scale()` |
| `transparentize($color, $amount)` / `fade-out()` | `color.adjust($color, $alpha: -$amount)` or `color.scale()` |

**Why these were deprecated:** CSS Color Level 4 introduced many color spaces. Channel names like "red" and "hue" are ambiguous without knowing the space. The old functions assumed all colors were sRGB and could be freely converted. The new `color.channel()` function with an explicit `$space` parameter removes this ambiguity.

Migration example:

```scss
// BEFORE (deprecated)
@use "sass:color";
$brand: #c71585;
$r: color.red($brand);                          // 199
$lighter: lighten($brand, 15%);                  // Fixed amount
$faded: transparentize($brand, 0.3);             // Fixed amount

// AFTER (modern)
@use "sass:color";
$brand: #c71585;
$r: color.channel($brand, "red", $space: rgb);   // 199
$lighter: color.scale($brand, $lightness: 30%);  // Proportional (better)
$faded: color.scale($brand, $alpha: -30%);        // Proportional (better)
```

### Practical Color Recipes

#### Palette Generator with `oklch`

```scss
@use "sass:color";
@use "sass:map";

@function generate-palette($base, $steps: 9) {
  $palette: ();
  @for $i from 1 through $steps {
    $lightness: 95% - (($i - 1) * 10%);
    $shade: color.change(
      color.to-space($base, oklch),
      $lightness: $lightness,
      $space: oklch
    );
    // Convert back to original space for CSS output
    $shade: color.to-space($shade, color.space($base));
    $palette: map.set($palette, $i * 100, $shade);
  }
  @return $palette;
}

$blue-palette: generate-palette(#3b82f6);
// Produces: 100 (very light), 200, ..., 900 (very dark)

:root {
  @each $weight, $shade in $blue-palette {
    --blue-#{$weight}: #{$shade};
  }
}
```

#### Accessible Contrast Checker

```scss
@use "sass:color";
@use "sass:math";

// Relative luminance using oklch lightness as approximation
@function light-or-dark($bg) {
  $l: color.channel($bg, "lightness", $space: oklch);
  @if $l > 60% {
    @return "light";
  }
  @return "dark";
}

@function contrast-text($bg) {
  @if light-or-dark($bg) == "light" {
    @return #1a1a1a;
  }
  @return #f5f5f5;
}

$colors: (
  "primary": #3b82f6,
  "success": #22c55e,
  "warning": #eab308,
  "danger":  #ef4444,
);

@each $name, $bg in $colors {
  .badge-#{$name} {
    background-color: $bg;
    color: contrast-text($bg);
  }
}
```

#### Gamut-Safe P3 Colors

```scss
@use "sass:color";

// Define in Display P3 for wider gamut on supported displays
$vivid-green: color(display-p3 0.2 0.9 0.3);

// Provide sRGB fallback
$safe-green: color.to-gamut($vivid-green, rgb, $method: local-minde);

.highlight {
  // Fallback for older browsers
  background-color: $safe-green;
  // Modern browsers get the P3 version
  background-color: $vivid-green;
}
```

## `sass:selector` -- Complete Reference

The selector module provides programmatic access to Sass's selector engine. These functions inspect and manipulate selectors at compile time.

Selectors are represented as nested lists:
- Outer list: comma-separated (selector list)
- Inner lists: space-separated (complex selectors)
- Values: unquoted strings (compound selectors)

### `selector.nest()` -- Combine Selectors as Nesting

```scss
@use "sass:selector";

// selector.nest($selectors...) -> selector
// Combines selectors as if nested in the stylesheet
@debug selector.nest("ul", "li");
// ul li

@debug selector.nest(".alert, .warning", "p");
// .alert p, .warning p

// Supports & parent selector (except in the first argument)
@debug selector.nest(".alert", "&:hover");
// .alert:hover

@debug selector.nest(".card", "&__header");
// .card__header

@debug selector.nest("nav", "ul", "li", "a");
// nav ul li a
```

### `selector.append()` -- Concatenate Without Descendant Combinator

```scss
@use "sass:selector";

// selector.append($selectors...) -> selector
// Combines without whitespace between them
@debug selector.append("a", ".disabled");
// a.disabled

@debug selector.append(".accordion", "__copy");
// .accordion__copy

@debug selector.append(".btn", "--primary");
// .btn--primary

// With selector lists, each combination is produced
@debug selector.append(".accordion", "__copy, __image");
// .accordion__copy, .accordion__image
```

Practical use -- BEM helper:

```scss
@use "sass:selector";

@function bem($block, $element: null, $modifier: null) {
  $sel: $block;
  @if $element {
    $sel: selector.append($sel, "__#{$element}");
  }
  @if $modifier {
    $sel: selector.append($sel, "--#{$modifier}");
  }
  @return $sel;
}

@debug bem(".card");                         // .card
@debug bem(".card", "header");               // .card__header
@debug bem(".card", "header", "sticky");     // .card__header--sticky
```

### `selector.extend()` -- Programmatic @extend

```scss
@use "sass:selector";

// selector.extend($selector, $extendee, $extender) -> selector
// Replaces $extendee in $selector with "$extendee, $extender"
@debug selector.extend("a.disabled", "a", ".link");
// a.disabled, .link.disabled

@debug selector.extend(".content a.highlight", "a", ".link");
// .content a.highlight, .content .link.highlight

// If $extendee is not found, returns $selector unchanged
@debug selector.extend("a.disabled", "h1", "h2");
// a.disabled
```

### `selector.replace()` -- Smart Replacement

```scss
@use "sass:selector";

// selector.replace($selector, $original, $replacement) -> selector
// Uses @extend intelligent unification for replacement
@debug selector.replace("a.disabled", "a", ".link");
// .link.disabled

@debug selector.replace(".guide .info", ".info", ".content nav.sidebar");
// .guide .content nav.sidebar, .content .guide nav.sidebar

// Not found: returns unchanged
@debug selector.replace("p.intro", "h1", "h2");
// p.intro
```

### `selector.unify()` -- Find Intersection

```scss
@use "sass:selector";

// selector.unify($selector1, $selector2) -> selector | null
// Returns a selector matching elements matched by BOTH inputs
@debug selector.unify("a", ".disabled");
// a.disabled

@debug selector.unify("a.disabled", "a.outgoing");
// a.disabled.outgoing

// Returns null when no overlap exists
@debug selector.unify("a", "h1");
// null

@debug selector.unify(".card", ".card--large");
// .card.card--large
```

### `selector.is-superselector()` -- Subset Check

```scss
@use "sass:selector";

// selector.is-superselector($super, $sub) -> boolean
// Does $super match everything $sub matches?
@debug selector.is-superselector("a", "a.disabled");     // true
@debug selector.is-superselector("a.disabled", "a");     // false
@debug selector.is-superselector("a", "sidebar a");      // true
@debug selector.is-superselector(".btn", ".btn.active");  // true
```

### `selector.parse()` -- Parse to List Structure

```scss
@use "sass:selector";

// selector.parse($selector) -> nested list
@debug selector.parse(".main aside:hover, .sidebar p");
// ((.main aside:hover), (.sidebar p))

// Useful for iterating over complex selector components
$parsed: selector.parse(".a .b, .c .d");
@each $complex in $parsed {
  @debug $complex;  // ".a .b" then ".c .d"
}
```

### `selector.simple-selectors()` -- Decompose Compound Selector

```scss
@use "sass:selector";

// selector.simple-selectors($selector) -> comma-separated list
// Input must be a single compound selector (no spaces, no commas)
@debug selector.simple-selectors("a.disabled");
// a, .disabled

@debug selector.simple-selectors("main.blog:after");
// main, .blog, :after

@debug selector.simple-selectors(".btn.active.primary");
// .btn, .active, .primary
```

## Global Function Deprecation

### The Problem

Before the module system, all Sass built-in functions were global. This created two issues:

1. **Name collisions with CSS:** As CSS adds new functions (like `min()`, `max()`, `round()`), global Sass functions with the same name conflict. Sass must decide whether `min(100px, 50%)` is a Sass function call or a CSS `min()` pass-through.

2. **No encapsulation:** All functions lived in the same flat namespace. Third-party libraries could shadow built-in functions accidentally.

### What Is Deprecated

As of Dart Sass 1.80.0, calling any built-in function without its module namespace emits a deprecation warning. These will be errors in Dart Sass 3.0.0 (expected no earlier than 2026).

The `sass-migrator` tool converts global function calls to module-namespaced calls:

```bash
# Install the migrator
npm install -g sass-migrator

# Full migration: converts @import to @use and updates all function calls
sass-migrator module --migrate-deps your-entrypoint.scss

# Built-in functions only: keeps @import but updates function calls
sass-migrator module --built-in-only --migrate-deps your-entrypoint.scss
```

The `--migrate-deps` flag processes all files imported/used by the entrypoint, not just the entrypoint itself.

### Managing Deprecation Warnings

```bash
# Silence specific deprecation warnings during transition

# Treat deprecation warnings as errors (for CI)
sass --fatal-deprecation=global-builtin src/main.scss

# Silence @import deprecation warnings

```

## Common Gotchas

### 1. CSS Functions vs. Sass Functions

Sass passes through CSS functions it does not recognize. But when a CSS function shares a name with a global Sass function (like `min()`, `max()`, `round()`), Sass intercepts the call:

```scss
// This calls Sass's global min(), NOT CSS min()
.box {
  width: min(100%, 600px);  // Sass error: incompatible units % and px
}

// FIX 1: Use math.min() explicitly and let CSS min() pass through
@use "sass:math";
.box {
  // CSS min() passes through when Sass does not see a global match
  width: min(100%, 600px);  // Works in Dart Sass 1.11.0+ with smart detection
}

// FIX 2: Use interpolation to force CSS output
.box {
  width: #{"min(100%, 600px)"};
}
```

In modern Dart Sass, `min()` and `max()` with incompatible units are detected as CSS functions and passed through. But edge cases exist, especially with older versions.

### 2. Unit Requirements in `sass:math`

```scss
@use "sass:math";

// BAD: Trig functions with incompatible inputs
// math.sin(10px);  // Error: expected an angle or unitless

// GOOD: Use angle units or unitless
@debug math.sin(90deg);     // 1
@debug math.sin(math.$pi);  // 0 (unitless = radians)

// BAD: pow/sqrt/log with units
// math.sqrt(16px);  // Error: expected unitless

// GOOD: Strip units first or use unitless values
@debug math.sqrt(16);       // 4
```

### 3. `color.adjust()` vs. `color.scale()` Decision

```scss
@use "sass:color";

$colors: (
  "light": hsl(200deg 80% 85%),
  "dark":  hsl(200deg 80% 20%),
);

// adjust(lightness: 15%) adds the same amount regardless of starting point
// Light: 85% + 15% = 100% (clipped to white)
// Dark:  20% + 15% = 35%

// scale(lightness: 30%) moves 30% of the REMAINING distance to max
// Light: 85% + 30% * (100% - 85%) = 85% + 4.5% = 89.5%
// Dark:  20% + 30% * (100% - 20%) = 20% + 24% = 44%

// RULE OF THUMB:
// Use adjust() when you want precise, absolute channel control
// Use scale() when you want proportional, visually consistent changes
```

### 4. Color Space Awareness

```scss
@use "sass:color";

$brand: #ff6600;

// BAD: Adjusting lightness in HSL can produce washed-out results
$lighter-hsl: color.adjust($brand, $lightness: 20%, $space: hsl);

// GOOD: Adjusting in oklch preserves perceptual uniformity
$lighter-oklch: color.adjust($brand, $lightness: 10%, $space: oklch);

// The oklch version looks more natural because oklch accounts
// for how humans perceive brightness across different hues
```

### 5. `@use` Placement

```scss
// BAD: @use after a rule
.box { color: red; }
@use "sass:math";  // Error: @use must come before any rules

// BAD: @use after a variable declaration
$size: 16px;
@use "sass:math";  // Error

// GOOD: @use at the very top
@use "sass:math";
@use "sass:color";
$size: 16px;
.box { width: math.div(100%, 3); }
```
