# 5.13 Destructuring and Spread

> Destructuring extracts values from arrays and objects into individual variables, while spread/rest syntax expands or collects elements. Together, they form the backbone of modern JavaScript data manipulation, function signatures, and immutable update patterns.

---

## Array Destructuring

Array destructuring uses the **iterable protocol** (`Symbol.iterator`). It works on arrays, strings, Maps, Sets, generators, and any custom iterable.

### Basics

```js
const [a, b, c] = [1, 2, 3];
console.log(a, b, c); // 1 2 3
```

### Skipping Elements

```js
const [first, , third] = [10, 20, 30];
console.log(first, third); // 10 30

const [, , last] = ["a", "b", "c"];
console.log(last); // "c"
```

### Default Values

Defaults trigger only when the value is strictly `undefined`.

```js
const [x = 10, y = 20] = [5];
console.log(x, y); // 5 20

const [a = 1] = [undefined]; // a = 1 (default triggered)
const [b = 1] = [null];      // b = null (null is NOT undefined)
const [c = 1] = [0];         // c = 0
const [d = 1] = [false];     // d = false
const [e = 1] = [""];        // e = ""
```

### Swapping Variables

No temporary variable needed.

```js
let a = 1;
let b = 2;

[a, b] = [b, a];
console.log(a, b); // 2 1

// Works with more than two
let x = 1, y = 2, z = 3;
[x, y, z] = [z, x, y];
console.log(x, y, z); // 3 1 2
```

### Rest Element

Collects remaining elements into a real Array.

```js
const [head, ...tail] = [1, 2, 3, 4, 5];
console.log(head); // 1
console.log(tail); // [2, 3, 4, 5]

// Empty rest
const [only, ...rest] = [1];
console.log(rest); // [] (empty array, not undefined)
```

The rest element must be last. Anything after it is a syntax error.

```js
// SyntaxError: Rest element must be last element
const [...first, last] = [1, 2, 3];
```

### Nested Array Destructuring

```js
const matrix = [[1, 2], [3, 4], [5, 6]];
const [[a, b], [c, d]] = matrix;
console.log(a, b, c, d); // 1 2 3 4

// Nested rest
const [first, ...[second, ...remaining]] = [1, 2, 3, 4];
console.log(first, second, remaining); // 1 2 [3, 4]
```

## Destructuring Iterables

Array destructuring works on any iterable, not just arrays.

### Strings

```js
const [first, second, ...rest] = "hello";
console.log(first);  // "h"
console.log(second); // "e"
console.log(rest);   // ["l", "l", "o"]
```

### Maps

Map iteration yields `[key, value]` pairs.

```js
const map = new Map([
  ["name", "Alice"],
  ["age", 30]
]);

for (const [key, value] of map) {
  console.log(`${key}: ${value}`);
}
// "name: Alice"
// "age: 30"

// Destructure first entry
const [[firstKey, firstValue]] = map;
console.log(firstKey, firstValue); // "name" "Alice"
```

### Sets

```js
const [a, b] = new Set([10, 20, 30]);
console.log(a, b); // 10 20
```

### Custom Iterables

```js
function* fibonacci() {
  let a = 0, b = 1;
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

const [f0, f1, f2, f3, f4, f5, f6] = fibonacci();
console.log(f0, f1, f2, f3, f4, f5, f6); // 0 1 1 2 3 5 8
```

### Non-Iterables Throw TypeError

```js
// TypeError: {} is not iterable
const [x] = {};

// TypeError: 42 is not iterable
const [y] = 42;
```

## Object Destructuring

Object destructuring uses property access (not the iterator protocol). It works on any value with properties, including values from the prototype chain.

### Basics

```js
const user = { name: "Alice", age: 30, role: "admin" };
const { name, age, role } = user;
console.log(name, age, role); // "Alice" 30 "admin"
```

### Renaming (Aliases)

The colon in destructuring means the opposite of object literals. In `{ source: target }`, `source` is the property name and `target` is the variable name.

```js
const response = { status: 200, data: { users: [] } };

const { status: httpStatus, data: responseData } = response;
console.log(httpStatus);    // 200
console.log(responseData);  // { users: [] }
// 'status' and 'data' are NOT defined as variables
```

### Renaming with Defaults

```js
const { name: userName = "Anonymous", role: userRole = "guest" } = {};
console.log(userName);  // "Anonymous"
console.log(userRole);  // "guest"
```

### Default Values

Same rule as arrays: defaults trigger only for `undefined`.

```js
const { x = 10 } = { x: null };      // x = null
const { y = 10 } = { y: 0 };         // y = 0
const { z = 10 } = { z: undefined };  // z = 10
const { w = 10 } = {};                // w = 10
```

### Lazy Evaluation of Defaults

Default expressions are only evaluated when needed.

```js
function getDefault() {
  console.log("Computing default");
  return 42;
}

const { value = getDefault() } = { value: 10 };
// "Computing default" is NOT logged because value is 10

const { other = getDefault() } = {};
// "Computing default" IS logged, other = 42
```

### Computed Property Keys

Dynamic property names require an alias.

```js
const key = "email";
const user = { email: "alice@example.com" };

const { [key]: emailAddress } = user;
console.log(emailAddress); // "alice@example.com"

// Without alias: SyntaxError
// const { [key] } = user;
```

### Rest Properties

Collects remaining own enumerable properties into a new object.

```js
const { id, ...rest } = { id: 1, name: "Alice", age: 30, role: "admin" };
console.log(id);   // 1
console.log(rest); // { name: "Alice", age: 30, role: "admin" }
```

Rest properties exclude prototype properties:

```js
const proto = { inherited: true };
const obj = Object.create(proto);
obj.own = true;

const { ...all } = obj;
console.log(all); // { own: true } (no 'inherited')
```

### Nested Object Destructuring

```js
const response = {
  data: {
    user: {
      name: "Alice",
      address: {
        city: "Portland",
        zip: "97201"
      }
    }
  }
};

const {
  data: {
    user: {
      name,
      address: { city, zip }
    }
  }
} = response;

console.log(name, city, zip); // "Alice" "Portland" "97201"
```

### The Nested `undefined` Trap

Destructuring a nested property from `undefined` or `null` throws TypeError.

```js
// TypeError: Cannot destructure property 'b' of undefined
const { a: { b } } = { a: undefined };

// Fix: provide a default at each level
const { a: { b } = {} } = { a: undefined };
console.log(b); // undefined (no error)

// Deep nesting requires defaults at every level
const { x: { y: { z } = {} } = {} } = {};
console.log(z); // undefined (no error)
```

## Object Destructuring Without Declaration

When destructuring without `let`/`const`/`var`, you must wrap the expression in parentheses. Without them, `{` is parsed as a block statement.

```js
let name, age;

// SyntaxError without parentheses
// { name, age } = { name: "Alice", age: 30 };

// Correct
({ name, age } = { name: "Alice", age: 30 });
console.log(name, age); // "Alice" 30
```

## Destructuring in Function Parameters

### The Options Object Pattern

```js
function createUser({ name, age = 25, role = "user" }) {
  return { name, age, role, createdAt: Date.now() };
}

createUser({ name: "Alice" });
// { name: "Alice", age: 25, role: "user", createdAt: ... }
```

### The Empty Call Gotcha

Without a parameter default, calling with no arguments throws TypeError.

```js
// Bad Practice: crashes on createUser()
function createUser({ name, age = 25 }) { /* ... */ }
createUser(); // TypeError: Cannot destructure property 'name' of undefined

// Good Practice: default to empty object
function createUser({ name, age = 25 } = {}) { /* ... */ }
createUser(); // Works: name=undefined, age=25
```

### Array Destructuring in Parameters

```js
function getFirst([first, ...rest]) {
  return { first, rest };
}

console.log(getFirst([1, 2, 3])); // { first: 1, rest: [2, 3] }

// With for...of
const pairs = [[1, "a"], [2, "b"], [3, "c"]];
for (const [num, letter] of pairs) {
  console.log(`${num}: ${letter}`);
}
```

### Mixed Destructuring

```js
function processEntry({ id, tags: [primaryTag, ...otherTags] = [] }) {
  return { id, primaryTag, otherTags };
}

processEntry({ id: 1, tags: ["js", "ts", "node"] });
// { id: 1, primaryTag: "js", otherTags: ["ts", "node"] }
```

## Spread Syntax

Spread expands an iterable (for arrays/function calls) or an object's own enumerable properties (for object literals) into individual elements.

### Array Spread

```js
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// Concatenation
const combined = [...arr1, ...arr2];
// [1, 2, 3, 4, 5, 6]

// Inserting in the middle
const withInsert = [0, ...arr1, 3.5, ...arr2, 7];
// [0, 1, 2, 3, 3.5, 4, 5, 6, 7]

// Shallow copy
const copy = [...arr1];
```

### Object Spread

```js
const defaults = { theme: "light", lang: "en", debug: false };
const userPrefs = { theme: "dark", lang: "fr" };

const config = { ...defaults, ...userPrefs };
// { theme: "dark", lang: "fr", debug: false }
```

### Property Order: Last Wins

When the same key appears multiple times, the last value wins.

```js
const obj = { a: 1, ...{ a: 2, b: 3 }, a: 4 };
console.log(obj); // { a: 4, b: 3 }
```

This enables two patterns:

```js
// Pattern 1: Defaults (spread defaults first, overrides after)
const config = { ...defaults, ...overrides };

// Pattern 2: Enforced values (override last, cannot be changed by user)
const record = { ...userInput, id: generateId(), createdAt: Date.now() };
// id and createdAt are always set by the system, regardless of userInput
```

### Spread Ignores `null` and `undefined` in Object Literals

```js
const data = null;
const obj = { a: 1, ...data }; // { a: 1 } (no error)

const undef = undefined;
const obj2 = { ...undef }; // {} (no error)
```

### Function Call Spread

```js
const numbers = [5, 2, 8, 1, 9];
console.log(Math.max(...numbers)); // 9

function log(a, b, c) {
  console.log(a, b, c);
}
const args = [1, 2, 3];
log(...args); // 1 2 3
```

## Spread vs `Object.assign`

| Feature | Spread (`{ ...obj }`) | `Object.assign(target, source)` |
|---|---|---|
| Mutates target | No (always new object) | Yes (mutates first argument) |
| Triggers setters | No (uses `[[DefineOwnProperty]]`) | Yes (uses `[[Set]]`) |
| `null`/`undefined` sources | Silently ignored | Silently ignored |
| `null`/`undefined` target | N/A (always new `{}`) | TypeError |
| Return value | New plain object | The target object |
| Preserves prototype | No (always `Object.prototype`) | Yes (target keeps its prototype) |

```js
// Object.assign triggers setters
const target = {};
Object.defineProperty(target, "x", {
  set(v) { console.log("setter:", v); }
});

Object.assign(target, { x: 1 }); // logs "setter: 1"
const spread = { ...target, x: 1 }; // no setter triggered
```

Prefer spread for creating new objects. Use `Object.assign` when you need to mutate an existing object or need setter invocation.

## Shallow Copy Limitations

Both spread and `Object.assign` create shallow copies. Nested objects are shared references.

```js
const original = {
  name: "Alice",
  scores: [90, 85, 92],
  address: { city: "Portland" }
};

const copy = { ...original };

copy.name = "Bob";           // independent (primitive)
copy.scores.push(100);       // MUTATES original.scores!
copy.address.city = "Seattle"; // MUTATES original.address!

console.log(original.scores);       // [90, 85, 92, 100]
console.log(original.address.city); // "Seattle"
```

### Shallow Merge Trap

Spread replaces entire nested objects, not individual properties within them.

```js
const defaults = {
  settings: { theme: "dark", lang: "en", notifications: true }
};

const userPrefs = {
  settings: { theme: "light" }
};

const config = { ...defaults, ...userPrefs };
// config.settings is { theme: "light" }
// lang and notifications are GONE!

// Fix: merge nested objects explicitly
const config = {
  ...defaults,
  settings: { ...defaults.settings, ...userPrefs.settings }
};
// config.settings is { theme: "light", lang: "en", notifications: true }
```

### Deep Copy with `structuredClone`

```js
const original = {
  name: "Alice",
  scores: [90, 85, 92],
  metadata: { created: new Date(), tags: new Set(["admin"]) }
};

const deep = structuredClone(original);
deep.scores.push(100);
deep.metadata.tags.add("user");

console.log(original.scores);        // [90, 85, 92] (unchanged)
console.log(original.metadata.tags); // Set(1) {"admin"} (unchanged)
```

### Cloning Method Comparison

| Method | Deep | Functions | Date | Map/Set | Circular | Prototype |
|---|---|---|---|---|---|---|
| Spread / `Object.assign` | No | Reference | Reference | Reference | N/A | Lost |
| `JSON.parse(JSON.stringify())` | Yes | Lost | Becomes string | Lost | Throws | Lost |
| `structuredClone()` | Yes | Throws | Preserved | Preserved | Handled | Lost |

## Rest vs Spread: The Distinction

The `...` operator has two roles determined by context:

| Context | Name | Action | Example |
|---|---|---|---|
| Destructuring pattern (left of `=`) | Rest | Collects remaining | `const [a, ...rest] = arr` |
| Function parameter | Rest | Collects remaining args | `function f(...args)` |
| Array literal (right of `=`) | Spread | Expands elements | `[...arr1, ...arr2]` |
| Object literal | Spread | Expands properties | `{ ...obj1, ...obj2 }` |
| Function call | Spread | Expands arguments | `fn(...args)` |

```js
// Rest: collects into a variable
const [first, ...rest] = [1, 2, 3]; // rest = [2, 3]

// Spread: expands into a new context
const copy = [...rest]; // copy = [2, 3]
```

## The Spread-in-Reduce Anti-Pattern

Using spread inside `.reduce()` creates a new object on every iteration, resulting in O(n^2) time complexity.

```js
// Bad Practice: O(n^2) -- new object per iteration
const merged = items.reduce((acc, item) => ({ ...acc, [item.id]: item }), {});

// Good Practice: O(n) -- mutate the accumulator
const merged = items.reduce((acc, item) => {
  acc[item.id] = item;
  return acc;
}, {});

// Also good: Object.fromEntries
const merged = Object.fromEntries(items.map((item) => [item.id, item]));
```

For 10 items the difference is negligible. For 10,000 items, the spread version can be 100x slower.

## Practical Patterns

### Regex Match Extraction

```js
const dateStr = "2025-01-15";
const [, year, month, day] = /(\d{4})-(\d{2})-(\d{2})/.exec(dateStr);
// Skips the full match at index 0
console.log(year, month, day); // "2025" "01" "15"
```

### Property Omission (Object Rest)

```js
function omit(obj, ...keys) {
  const keySet = new Set(keys);
  return Object.fromEntries(
    Object.entries(obj).filter(([key]) => !keySet.has(key))
  );
}

// Or the destructuring approach for known keys:
const { password, secret, ...safeUser } = user;
// safeUser has everything except password and secret
```

### Property Picking

```js
function pick(obj, ...keys) {
  return Object.fromEntries(
    keys.filter((key) => key in obj).map((key) => [key, obj[key]])
  );
}

const user = { id: 1, name: "Alice", email: "a@b.com", password: "secret" };
const publicUser = pick(user, "id", "name", "email");
// { id: 1, name: "Alice", email: "a@b.com" }
```

### Conditional Property Spread

```js
function buildQuery(filters) {
  return {
    status: "active",
    ...(filters.minAge && { minAge: filters.minAge }),
    ...(filters.maxAge && { maxAge: filters.maxAge }),
    ...(filters.role && { role: filters.role })
  };
}

buildQuery({ minAge: 18, role: "admin" });
// { status: "active", minAge: 18, role: "admin" }
// maxAge is NOT present (not even as undefined)
```

Note: This pattern relies on falsy values being falsy. If `0` or `""` are valid values you want to include, use `!= null` instead:

```js
...(filters.minAge != null && { minAge: filters.minAge }),
```

## Common Mistakes

- **Assuming defaults trigger for `null`**: Defaults only trigger for `undefined`. `null`, `0`, `false`, and `""` are all preserved as-is.

- **Nested destructuring from `null`/`undefined`**: This throws TypeError. Always provide defaults at every nesting level: `const { a: { b } = {} } = obj`.

- **Thinking spread creates a deep copy**: Spread is shallow. Nested objects and arrays are shared references.

- **Using spread in `.reduce()`**: Creates O(n^2) performance. Mutate the accumulator instead.

- **Forgetting `= {}` on destructured function parameters**: Calling the function with no arguments throws TypeError.

- **Object destructuring without parentheses in assignment**: `{ a, b } = obj` is a syntax error. Use `({ a, b } = obj)`.

- **Overzealous destructuring**: Destructuring 10+ properties at the top of a function hurts readability. Sometimes `obj.property` is clearer.

- **Expecting spread to merge nested objects**: `{ ...defaults, ...overrides }` replaces entire nested objects, it does not deep-merge them.

- **Destructuring in catch without guarding**: `catch ({ message })` throws if the caught value is not an object (e.g., if someone throws a string).

---

## Best Practices

- Use default values (`= {}`, `= []`, `= ""`) at every destructuring level where `undefined` is possible.
- Prefer spread over `Object.assign` for creating new objects. Use `Object.assign` only when mutating a target is intentional.
- Use rest properties (`const { known, ...rest } = obj`) to cleanly separate known from unknown properties.
- Use `structuredClone()` when you need a true deep copy. Reserve spread for shallow updates.
- Keep destructuring at a reasonable depth. If you need more than two levels, consider intermediate variables for clarity.
- Use the options object pattern for functions with more than two or three parameters.
- Always add `= {}` as the default for destructured function parameters to handle zero-argument calls.
- Avoid spread inside `.reduce()`. Use a mutable accumulator or `Object.fromEntries`.

## Real-World Use Cases

### API Response Transformation

```js
function transformUser(apiResponse) {
  const {
    id,
    attributes: {
      first_name: firstName,
      last_name: lastName,
      email,
      created_at: createdAt,
      ...extraAttributes
    } = {},
    relationships: {
      organization: { data: orgData } = {}
    } = {}
  } = apiResponse;

  return {
    id,
    firstName,
    lastName,
    email,
    createdAt: new Date(createdAt),
    organizationId: orgData?.id ?? null,
    metadata: extraAttributes
  };
}

const raw = {
  id: "123",
  attributes: {
    first_name: "Alice",
    last_name: "Smith",
    email: "alice@example.com",
    created_at: "2025-01-15T00:00:00Z",
    department: "Engineering",
    level: "Senior"
  },
  relationships: {
    organization: { data: { id: "org-1", type: "organization" } }
  }
};

console.log(transformUser(raw));
// {
//   id: "123", firstName: "Alice", lastName: "Smith",
//   email: "alice@example.com", createdAt: Date(...),
//   organizationId: "org-1",
//   metadata: { department: "Engineering", level: "Senior" }
// }
```

### Immutable State Updates

```js
function appReducer(state, action) {
  switch (action.type) {
    case "SET_USER":
      return { ...state, user: action.payload };

    case "UPDATE_SETTINGS":
      return {
        ...state,
        settings: {
          ...state.settings,
          ...action.payload
        }
      };

    case "ADD_ITEM":
      return {
        ...state,
        items: [...state.items, action.payload]
      };

    case "REMOVE_ITEM": {
      const { [action.payload]: removed, ...remainingItems } = state.itemsById;
      return {
        ...state,
        itemsById: remainingItems,
        itemIds: state.itemIds.filter((id) => id !== action.payload)
      };
    }

    case "UPDATE_NESTED": {
      const { path, value } = action.payload;
      const [section, key] = path;
      return {
        ...state,
        [section]: {
          ...state[section],
          [key]: value
        }
      };
    }

    default:
      return state;
  }
}
```

### Configuration Builder with Spread Composition

```js
function createFetchConfig({
  method = "GET",
  headers: customHeaders = {},
  body,
  timeout = 30000,
  retry = 0,
  ...extraOptions
} = {}) {
  const baseHeaders = {
    "Content-Type": "application/json",
    Accept: "application/json"

  return {
    method,
    headers: {
      ...baseHeaders,
      ...customHeaders,
      ...(token && { Authorization: `Bearer ${token}` })

    ...(body && { body: JSON.stringify(body) }),
    signal: AbortSignal.timeout(timeout),
    ...extraOptions,
    // Internal metadata (always present, cannot be overridden)

    _timestamp: Date.now()
  };
}

const config = createFetchConfig({

```

---

> **Key Takeaway:** Destructuring and spread are powerful tools for working with data, but they are shallow by nature. Always provide defaults at every nesting level, remember that `null` does not trigger defaults, avoid spread inside `.reduce()` for performance, and use `structuredClone()` when you need true deep copies.
