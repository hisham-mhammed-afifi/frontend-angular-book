# 5.18 Collections (Map, Set, WeakMap, WeakSet)

> ES6 introduced four built-in keyed collection types that solve real problems plain objects and arrays cannot: ordered key-value storage with any key type (Map), unique value tracking (Set), and garbage-collection-friendly variants for object metadata (WeakMap, WeakSet).

---

## Map

A `Map` is an ordered collection of key-value pairs where keys can be **any type**: objects, functions, primitives, even `NaN`.

### Construction

```js
// Empty Map
const map = new Map();

// From an iterable of [key, value] pairs
const map2 = new Map([
  ['name', 'Alice'],
  ['age', 30],
  [42, 'the answer'],
]);

// From another Map (creates a shallow copy)
const copy = new Map(map2);

// From an Object using Object.entries()
const obj = { foo: 'bar', baz: 42 };
const mapFromObj = new Map(Object.entries(obj));

// From a generator
function* pairs() {
  yield ['a', 1];
  yield ['b', 2];
}
const mapFromGen = new Map(pairs());
```

The constructor accepts any iterable that yields `[key, value]` pairs. This is why `Object.entries()` works directly.

### Core Methods: get, set, has, delete, clear

```js
const config = new Map();

// set() returns the Map, so you can chain
config
  .set('theme', 'dark')
  .set('lang', 'en')
  .set('debug', false);

config.get('theme');    // 'dark'
config.get('missing');  // undefined (not an error)

config.has('lang');     // true
config.has('missing');  // false

config.delete('debug'); // true (found and removed)
config.delete('nope');  // false (not found)

config.size;            // 2

config.clear();         // removes all entries
config.size;            // 0
```

### Any Type as Key

This is the killer feature that separates Map from Object.

```js
const map = new Map();

// Object keys
const userA = { id: 1 };
const userB = { id: 2 };
map.set(userA, 'Admin');
map.set(userB, 'Editor');

// Function keys
const handler = () => {};
map.set(handler, { bound: true });

// Boolean, number, null, undefined keys
map.set(true, 'yes');
map.set(42, 'the answer');
map.set(null, 'nothing');
map.set(undefined, 'absent');

// All of these are separate keys
map.size; // 6
```

### Iteration Order Guarantee

Map iterates in **insertion order**. This is guaranteed by the spec, unlike plain objects (which mostly preserve insertion order but have edge cases with numeric keys).

```js
const map = new Map();
map.set('c', 3);
map.set('a', 1);
map.set('b', 2);

// keys(), values(), entries() all follow insertion order
for (const [key, value] of map) {
  console.log(key, value);
}
// c 3
// a 1
// b 2

// keys() returns a MapIterator of keys
[...map.keys()];    // ['c', 'a', 'b']

// values() returns a MapIterator of values
[...map.values()];  // [3, 1, 2]

// entries() returns a MapIterator of [key, value] pairs
// (equivalent to the default iterator)
[...map.entries()]; // [['c', 3], ['a', 1], ['b', 2]]

// forEach works too, but note the callback signature:
// (value, key, map) -- value comes first, matching Array.forEach
map.forEach((value, key) => {
  console.log(`${key}: ${value}`);
});
```

### size Property

Unlike `Object.keys(obj).length`, `Map.size` is a direct O(1) property:

```js
const map = new Map([['a', 1], ['b', 2], ['c', 3]]);
map.size; // 3 -- no iteration needed
```

## Map vs Object

This is the most common comparison. Both store key-value data, but they serve different purposes.

### Feature Comparison

| Feature | Map | Object |
|---------|-----|--------|
| Key types | Any value (objects, functions, primitives) | Strings and Symbols only |
| Key order | Guaranteed insertion order | Mostly insertion order, but integer-like keys are sorted first |
| Size | `map.size` (O(1)) | `Object.keys(obj).length` (O(n)) |
| Iteration | Directly iterable (`for...of`, spread) | Requires `Object.keys()`, `Object.entries()`, etc. |
| Default keys | None | Inherits from `Object.prototype` (`toString`, `hasOwnProperty`, etc.) |
| Performance (insert/delete) | Optimized for frequent add/remove | `delete` operator is notoriously slow in V8 |
| JSON support | No direct `JSON.stringify` | Native JSON support |
| Destructuring | Not supported | Native destructuring |
| Prototype pollution | Immune | Vulnerable if used as a dictionary with user input |

### Integer Key Ordering Problem with Objects

```js
// Objects sort integer-like keys first
const obj = {};
obj['b'] = 2;
obj['1'] = 'one';
obj['a'] = 1;
obj['2'] = 'two';

Object.keys(obj); // ['1', '2', 'b', 'a'] -- integers sorted first!

// Map preserves insertion order exactly
const map = new Map();
map.set('b', 2);
map.set('1', 'one');
map.set('a', 1);
map.set('2', 'two');

[...map.keys()]; // ['b', '1', 'a', '2'] -- exact insertion order
```

### Prototype Pollution with Objects

```js
// BAD: Object as dictionary can clash with inherited properties
const dict = {};
dict['toString'] = 'oops';
dict.toString; // 'oops' -- the built-in method is gone

// Calling dict.toString() would now throw because it is a string, not a function

// SAFER: Object.create(null) removes the prototype
const safeDict = Object.create(null);
safeDict['toString'] = 'fine';
// No prototype methods to collide with

// BEST: Map is immune to this problem
const map = new Map();
map.set('toString', 'fine');
map.get('toString'); // 'fine'
// Map's own toString still works
map.toString(); // '[object Map]'
```

### Performance: When Map Wins

```js
// Map excels at frequent additions and deletions
const map = new Map();
const obj = {};

// Adding 100,000 entries
// Map: ~2x faster than Object for string keys
for (let i = 0; i < 100_000; i++) {
  map.set(`key_${i}`, i);
}

// Deleting entries
// Map.delete() is dramatically faster than `delete obj[key]`
// The `delete` operator on objects triggers deoptimization in V8
for (let i = 0; i < 100_000; i++) {
  map.delete(`key_${i}`); // fast
}

for (let i = 0; i < 100_000; i++) {
  delete obj[`key_${i}`]; // slow -- forces V8 to abandon hidden class optimizations
}

// Iteration: Map is ~4x faster than Object for large datasets
// Object.keys() creates an intermediate array; Map iterates directly
```

### When to Use Which

**Use Map when:**
- Keys are unknown at author time (dynamic keys from user input, API data)
- Keys are not strings (DOM nodes, objects, functions)
- You need guaranteed insertion order
- You need frequent additions and deletions
- You need the `size` property
- You want protection against prototype pollution

**Use Object when:**
- You have a fixed, known set of properties (config, options, records)
- You need JSON serialization
- You need destructuring
- You want property access shorthand (`obj.name` vs `map.get('name')`)
- You are modeling structured data (a user, a product, a response)

## Map Edge Cases

### NaN as a Key

JavaScript's equality semantics are complex. `NaN !== NaN` is `true`, which would make it useless as a key under strict equality. Map solves this using the **SameValueZero** algorithm.

```js
const map = new Map();

map.set(NaN, 'not a number');
map.get(NaN);            // 'not a number'
map.get(Number('foo'));   // 'not a number' (Number('foo') is NaN)
map.has(NaN);            // true

// NaN is treated as equal to itself inside Map
// This is different from === where NaN !== NaN
```

### The SameValueZero Algorithm

Map and Set both use **SameValueZero** for key/value comparison. Understanding it prevents subtle bugs.

| Comparison | `===` | `SameValueZero` | `Object.is()` (SameValue) |
|-----------|-------|-----------------|--------------------------|
| `NaN` vs `NaN` | `false` | `true` | `true` |
| `+0` vs `-0` | `true` | `true` | `false` |
| `1` vs `'1'` | `false` | `false` | `false` |
| `null` vs `undefined` | `false` | `false` | `false` |

```js
const map = new Map();

// -0 and +0 are the same key
map.set(-0, 'negative zero');
map.get(+0); // 'negative zero'
map.get(0);  // 'negative zero'

// There is only one entry, not two
map.size; // 1
```

### Object Reference Keys: The Biggest Gotcha

When using objects as keys, equality is based on **reference identity**, not structural equality.

```js
const map = new Map();

// This works: same reference
const key = { id: 1 };
map.set(key, 'found');
map.get(key); // 'found'

// This FAILS: different reference, same shape
map.get({ id: 1 }); // undefined -- this is a NEW object

// Common mistake: creating the key inline
map.set({ name: 'Alice' }, 100);
// You can never retrieve this value because the key object
// is only accessible through map iteration now

// You MUST hold a reference to the key object
const alice = { name: 'Alice' };
map.set(alice, 100);
map.get(alice); // 100
```

This has real implications for memory:

```js
// MEMORY LEAK: Map holds a strong reference to key objects
const cache = new Map();

function process(data) {
  // Every call creates a new object key that will never be GC'd
  // as long as `cache` exists
  cache.set(data, expensiveComputation(data));
}

let bigData = generateBigData();
process(bigData);
bigData = null; // You expect GC, but cache still holds it!
// Solution: use WeakMap instead
```

### Iterating and Modifying

```js
const map = new Map([['a', 1], ['b', 2], ['c', 3]]);

// Safe: you can delete during forEach
map.forEach((value, key) => {
  if (value === 2) {
    map.delete(key); // this is safe
  }
});

// Entries added during iteration that have not been reached
// will be visited. Entries deleted before being reached will not be visited.
```

## Set

A `Set` is a collection of **unique values**. No duplicates allowed.

### Construction

```js
// Empty Set
const set = new Set();

// From an iterable
const set2 = new Set([1, 2, 3, 2, 1]); // {1, 2, 3}

// From a string (each character)
const chars = new Set('hello'); // {'h', 'e', 'l', 'o'}

// Classic use: deduplicate an array
const arr = [1, 2, 2, 3, 3, 3, 4];
const unique = [...new Set(arr)]; // [1, 2, 3, 4]
```

### Core Methods: add, has, delete, clear

```js
const tags = new Set();

// add() returns the Set, so you can chain
tags.add('javascript').add('typescript').add('javascript');

tags.size; // 2 -- duplicate 'javascript' was ignored

tags.has('javascript');  // true
tags.has('python');      // false

tags.delete('typescript'); // true (found and removed)
tags.delete('rust');       // false (not found)

tags.clear();
tags.size; // 0
```

### Uniqueness and SameValueZero

Set uses the same **SameValueZero** algorithm as Map for determining uniqueness:

```js
const set = new Set();

// NaN is equal to NaN (unlike ===)
set.add(NaN);
set.add(NaN);
set.size; // 1

// -0 and +0 are the same
set.add(-0);
set.add(+0);
set.size; // 2 (NaN and 0)

// Objects are compared by reference, NOT by value
set.add({ id: 1 });
set.add({ id: 1 });
set.size; // 4 -- two different object references
```

### Iteration

Set iterates in **insertion order**.

```js
const set = new Set(['banana', 'apple', 'cherry']);

// for...of
for (const item of set) {
  console.log(item); // banana, apple, cherry
}

// forEach
set.forEach((value) => {
  console.log(value);
});

// Spread
const arr = [...set]; // ['banana', 'apple', 'cherry']

// keys() and values() return the same iterator (keys === values for Set)
[...set.keys()];   // ['banana', 'apple', 'cherry']
[...set.values()]; // ['banana', 'apple', 'cherry']

// entries() returns [value, value] pairs (for Map compatibility)
[...set.entries()]; // [['banana', 'banana'], ['apple', 'apple'], ['cherry', 'cherry']]
```

### Set Lacks Array Methods

Set does not have `map()`, `filter()`, or `reduce()`. You must convert to an array first:

```js
const numbers = new Set([1, 2, 3, 4, 5]);

// BAD: these do not exist
// numbers.map(n => n * 2);     // TypeError
// numbers.filter(n => n > 2);  // TypeError

// Convert to array, then use array methods
const doubled = [...numbers].map(n => n * 2); // [2, 4, 6, 8, 10]
const filtered = [...numbers].filter(n => n > 2); // [3, 4, 5]
```

## Set Operations (ES2025)

ES2025 adds seven native set operation methods. These reached Stage 4 and are available in all major browsers as of mid-2024 (Chrome 122+, Safari 17+, Firefox 127+).

### union()

Returns a new Set with elements from **either or both** sets.

```js
const frontend = new Set(['HTML', 'CSS', 'JavaScript']);
const backend = new Set(['Python', 'JavaScript', 'SQL']);

const all = frontend.union(backend);
// Set {'HTML', 'CSS', 'JavaScript', 'Python', 'SQL'}
```

### intersection()

Returns a new Set with elements present in **both** sets.

```js
const frontend = new Set(['HTML', 'CSS', 'JavaScript']);
const backend = new Set(['Python', 'JavaScript', 'SQL']);

const shared = frontend.intersection(backend);
// Set {'JavaScript'}
```

### difference()

Returns a new Set with elements in the first set but **not** in the second.

```js
const frontend = new Set(['HTML', 'CSS', 'JavaScript']);
const backend = new Set(['Python', 'JavaScript', 'SQL']);

const frontendOnly = frontend.difference(backend);
// Set {'HTML', 'CSS'}

const backendOnly = backend.difference(frontend);
// Set {'Python', 'SQL'}
```

### symmetricDifference()

Returns a new Set with elements in **either** set but **not both** (the XOR of sets).

```js
const frontend = new Set(['HTML', 'CSS', 'JavaScript']);
const backend = new Set(['Python', 'JavaScript', 'SQL']);

const exclusive = frontend.symmetricDifference(backend);
// Set {'HTML', 'CSS', 'Python', 'SQL'}
```

### isSubsetOf()

Returns `true` if every element in this set is also in the other.

```js
const web = new Set(['HTML', 'CSS', 'JavaScript']);
const all = new Set(['HTML', 'CSS', 'JavaScript', 'Python', 'SQL']);

web.isSubsetOf(all); // true
all.isSubsetOf(web); // false
```

### isSupersetOf()

Returns `true` if this set contains every element of the other.

```js
const all = new Set(['HTML', 'CSS', 'JavaScript', 'Python', 'SQL']);
const web = new Set(['HTML', 'CSS', 'JavaScript']);

all.isSupersetOf(web); // true
web.isSupersetOf(all); // false
```

### isDisjointFrom()

Returns `true` if the two sets share **no** elements.

```js
const frontend = new Set(['HTML', 'CSS']);
const backend = new Set(['Python', 'SQL']);
const fullstack = new Set(['JavaScript', 'HTML']);

frontend.isDisjointFrom(backend);  // true -- no overlap
frontend.isDisjointFrom(fullstack); // false -- both contain 'HTML'
```

### Set-like Argument Flexibility

The `other` argument does not have to be a `Set`. It only needs to be **Set-like**: an object with a `size` property, a `has()` method, and a `keys()` method. This means you can use these methods with custom collections:

```js
const mySet = new Set([1, 2, 3]);

// A Set-like object works as the argument
const setLike = {
  size: 2,
  has(value) { return value === 2 || value === 4; },
  *keys() { yield 2; yield 4; }
};

mySet.intersection(setLike); // Set {2}
mySet.union(setLike);        // Set {1, 2, 3, 4}
```

### Before ES2025: Manual Set Operations

If you need to support older environments without polyfills:

```js
// Union
function union(setA, setB) {
  return new Set([...setA, ...setB]);
}

// Intersection
function intersection(setA, setB) {
  return new Set([...setA].filter(x => setB.has(x)));
}

// Difference
function difference(setA, setB) {
  return new Set([...setA].filter(x => !setB.has(x)));
}

// Symmetric Difference
function symmetricDifference(setA, setB) {
  const diff = new Set(setA);
  for (const elem of setB) {
    if (diff.has(elem)) {
      diff.delete(elem);
    } else {
      diff.add(elem);
    }
  }
  return diff;
}
```

## WeakMap

A `WeakMap` holds key-value pairs where keys must be **objects** (or non-registered Symbols) and are held via **weak references**. If the only remaining reference to a key is the WeakMap entry, that entry is eligible for garbage collection.

### Key Restrictions

```js
const wm = new WeakMap();

// ALLOWED: objects as keys
wm.set({}, 'object');
wm.set([], 'array');
wm.set(function() {}, 'function');
wm.set(document.body, 'dom node');
wm.set(Symbol('unregistered'), 'unregistered symbol');

// NOT ALLOWED: primitives as keys
wm.set('string', 'value');    // TypeError
wm.set(42, 'value');          // TypeError
wm.set(true, 'value');        // TypeError
wm.set(null, 'value');        // TypeError

// NOT ALLOWED: registered symbols
wm.set(Symbol.for('global'), 'value'); // TypeError
// Symbol.for() creates globally registered symbols that are never GC'd,
// which defeats the purpose of WeakMap
```

### Available Methods (Limited API)

```js
const wm = new WeakMap();
const key = { id: 1 };

wm.set(key, 'data');
wm.get(key);    // 'data'
wm.has(key);    // true
wm.delete(key); // true

// These DO NOT exist on WeakMap:
// wm.size       -- undefined
// wm.keys()     -- TypeError
// wm.values()   -- TypeError
// wm.entries()  -- TypeError
// wm.forEach()  -- TypeError
// wm.clear()    -- removed in the final spec

// WeakMap is NOT iterable
// for (const [k, v] of wm) {} -- TypeError
```

The lack of iteration is intentional. If you could enumerate WeakMap entries, you would observe garbage collection behavior, which is nondeterministic and engine-specific.

### Garbage Collection in Action

```js
const wm = new WeakMap();

let user = { name: 'Alice' };
wm.set(user, { visits: 42 });

wm.get(user); // { visits: 42 }

// Remove the only strong reference
user = null;

// At some point after this, the GC will collect both:
// - the { name: 'Alice' } object (the key)
// - the { visits: 42 } object (the value)
// The WeakMap entry disappears automatically

// Compare with Map:
const map = new Map();
let user2 = { name: 'Bob' };
map.set(user2, { visits: 10 });
user2 = null;
// { name: 'Bob' } is still in memory! Map holds a strong reference.
// This is a memory leak if you forget to call map.delete()
```

### Use Case 1: Private Data

Before `#private` class fields existed, WeakMap was the standard pattern for truly private instance data:

```js
// Private data that cannot be accessed from outside
const _private = new WeakMap();

class Person {
  constructor(name, ssn) {
    // Store private data keyed by the instance
    _private.set(this, {
      ssn,                      // truly private
      createdAt: Date.now(),    // truly private
    });
    this.name = name;           // public
  }

  getLastFourSSN() {
    return _private.get(this).ssn.slice(-4);
  }
}

const alice = new Person('Alice', '123-45-6789');
alice.name;              // 'Alice'
alice.getLastFourSSN();  // '6789'

// No way to access SSN from outside
// _private is module-scoped, not exported
// Object.keys(alice)       -> ['name']
// Object.getOwnPropertyNames(alice) -> ['name']
// Reflect.ownKeys(alice)   -> ['name']

// When alice is GC'd, the private data is also GC'd
```

This pattern is still relevant when:
- You need cross-instance access to private data (one instance reading another's private state)
- You are adding private metadata to objects you do not own
- You are working in environments without `#private` field support

### Use Case 2: Caching with Automatic Cleanup

```js
const cache = new WeakMap();

function computeExpensive(obj) {
  if (cache.has(obj)) {
    return cache.get(obj); // return cached result
  }

  const result = heavyCalculation(obj);
  cache.set(obj, result);
  return result;
}

let data = fetchData();
computeExpensive(data); // computed and cached
computeExpensive(data); // returned from cache

data = null;
// Cache entry is automatically eligible for GC
// No manual cache invalidation needed
```

Compare this with a Map-based cache where you must manually evict entries or risk unbounded memory growth.

### Use Case 3: DOM Node Metadata

```js
const clickCounts = new WeakMap();

document.querySelectorAll('button').forEach(btn => {
  clickCounts.set(btn, 0);

  btn.addEventListener('click', () => {
    const count = clickCounts.get(btn) + 1;
    clickCounts.set(btn, count);
    console.log(`Button clicked ${count} times`);
  });
});

// If a button is removed from the DOM and all references are dropped,
// its click count data is automatically GC'd.
// With a regular Map, you would need to manually delete the entry
// when the button is removed, or you leak memory.
```

### Use Case 4: Memoizing Methods on Foreign Objects

```js
// Attach computed metadata to objects you do not control
const boundingBoxCache = new WeakMap();

function getCachedBoundingBox(element) {
  if (!boundingBoxCache.has(element)) {
    boundingBoxCache.set(element, element.getBoundingClientRect());
  }
  return boundingBoxCache.get(element);
}

// Framework removes the element -> WeakMap entry is GC'd
// No coordination with the framework needed
```

## WeakSet

A `WeakSet` stores **objects only** (or non-registered Symbols) via weak references. Like WeakMap, entries are automatically removed when no other references exist.

### API

```js
const ws = new WeakSet();
const obj = { id: 1 };

ws.add(obj);
ws.has(obj);    // true
ws.delete(obj); // true

// Primitives are not allowed
ws.add('string'); // TypeError
ws.add(42);       // TypeError

// Not iterable, no size
// ws.size      -> undefined
// ws.keys()    -> TypeError
// ws.forEach() -> TypeError
```

### Use Case 1: Tracking Processed Objects

```js
const processed = new WeakSet();

function processOnce(obj) {
  if (processed.has(obj)) {
    return; // already processed
  }

  doExpensiveWork(obj);
  processed.add(obj);
}

let data = { payload: 'heavy' };
processOnce(data); // processes
processOnce(data); // skips

data = null;
// Entry in WeakSet is eligible for GC
```

### Use Case 2: Circular Reference Detection

This is one of the most practical uses. When traversing object graphs (deep clone, serialization), you need to detect cycles:

```js
function deepClone(obj, seen = new WeakSet()) {
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // Circular reference detected
  if (seen.has(obj)) {
    throw new Error('Circular reference detected');
  }

  seen.add(obj);

  if (Array.isArray(obj)) {
    return obj.map(item => deepClone(item, seen));
  }

  const clone = {};
  for (const key of Object.keys(obj)) {
    clone[key] = deepClone(obj[key], seen);
  }
  return clone;
}

// Using a WeakSet instead of a Set means the `seen` collection
// does not prevent GC of the objects after cloning is complete
```

`JSON.stringify` itself uses a similar approach internally to detect circular references.

### Use Case 3: Object Branding / Validation

Verify that objects passed to your API were actually created by your factory:

```js
const validTokens = new WeakSet();

function createToken(userId) {
  const token = {
    userId,
    createdAt: Date.now(),
    expiresAt: Date.now() + 3600_000,
  };
  validTokens.add(token);
  return token;
}

function validateToken(token) {
  if (!validTokens.has(token)) {
    throw new Error('Invalid or forged token');
  }
  if (Date.now() > token.expiresAt) {
    validTokens.delete(token);
    throw new Error('Token expired');
  }
  return true;
}

const token = createToken('user_123');
validateToken(token); // true

// Forged token with identical properties fails
const fake = { userId: 'user_123', createdAt: token.createdAt, expiresAt: token.expiresAt };
validateToken(fake); // throws 'Invalid or forged token'
```

This is a security pattern. Since WeakSet uses reference identity, structurally identical objects fail the check. And since WeakSet is not iterable, external code cannot enumerate valid tokens.

### Use Case 4: Preventing Duplicate Event Handling

```js
const listenersAttached = new WeakSet();

function attachClickHandler(element) {
  if (listenersAttached.has(element)) {
    return; // already attached
  }

  element.addEventListener('click', handleClick);
  listenersAttached.add(element);
}

// When the element is removed from the DOM and GC'd,
// the WeakSet entry is cleaned up automatically
```

## WeakRef and FinalizationRegistry

These are related concepts that give you more explicit control over weak references and cleanup callbacks. They were introduced in ES2021.

### WeakRef

A `WeakRef` holds a weak reference to a target object. Unlike WeakMap/WeakSet, you can read the reference back using `deref()`.

```js
let bigObject = { data: new Array(1_000_000).fill('x') };
const ref = new WeakRef(bigObject);

// Access the object through the weak reference
ref.deref(); // { data: [...] }

bigObject = null;

// After GC runs (timing is nondeterministic):
ref.deref(); // undefined (or still the object if GC has not run yet)
```

### FinalizationRegistry

Lets you register a cleanup callback that runs when a target object is garbage collected:

```js
const registry = new FinalizationRegistry((heldValue) => {
  console.log(`Object with id "${heldValue}" was garbage collected`);
  // Clean up external resources associated with heldValue
});

let obj = { id: 'resource_42' };
registry.register(obj, 'resource_42');

obj = null;
// Eventually: "Object with id "resource_42" was garbage collected"
```

### Practical Example: Smart Cache with WeakRef

```js
class WeakCache {
  #cache = new Map();
  #registry = new FinalizationRegistry((key) => {
    // Only delete if the WeakRef is actually dead
    const ref = this.#cache.get(key);
    if (ref && ref.deref() === undefined) {
      this.#cache.delete(key);
    }
  });

  set(key, value) {
    const ref = new WeakRef(value);
    this.#cache.set(key, ref);
    this.#registry.register(value, key);
  }

  get(key) {
    const ref = this.#cache.get(key);
    if (!ref) return undefined;

    const value = ref.deref();
    if (value === undefined) {
      // Object was GC'd, clean up the entry
      this.#cache.delete(key);
    }
    return value;
  }
}

const cache = new WeakCache();
let heavyData = { payload: new Array(1_000_000) };
cache.set('report', heavyData);

cache.get('report'); // { payload: [...] }

heavyData = null;
// After GC: cache.get('report') returns undefined
// The FinalizationRegistry callback cleans up the Map entry
```

### Critical Caveats

```js
// DO NOT rely on FinalizationRegistry for correctness
// GC timing is nondeterministic and engine-specific

// BAD: using finalization for critical cleanup
const registry = new FinalizationRegistry((fd) => {
  closeFileDescriptor(fd); // May run too late, or never
});

// GOOD: use explicit cleanup (try/finally, using declarations, AbortController)
// FinalizationRegistry is a safety net, not a primary mechanism
```

The spec explicitly warns: "Correct code must not depend on cleanup callbacks being called in a timely manner, or at all."

## Performance Comparisons

### Map vs Object: Benchmarks

Based on V8 engine benchmarks with string keys:

| Operation | Map | Object | Winner |
|-----------|-----|--------|--------|
| Insert 100K entries | ~15ms | ~30ms | Map (~2x) |
| Insert 1M entries | ~200ms | ~260ms | Map (~1.3x) |
| Lookup (has/in) | ~12ms/100K | ~11ms/100K | Roughly equal |
| Delete 100K entries | ~18ms | ~150ms | Map (~8x) |
| Iterate 1M entries | ~22ms | ~87ms | Map (~4x) |
| Get size | O(1) | O(n) | Map |

Key insights:
- Map's advantage in deletion is dramatic because `delete` on objects triggers V8 to abandon hidden class optimizations and fall back to dictionary mode
- For read-heavy workloads with string keys where the object shape does not change, Object can be slightly faster due to V8's hidden class optimizations
- Map becomes unusably slow much later than Object (Objects degrade around 8.3M entries)

### Set vs Array: Lookup Performance

| Data size | `Array.includes()` | `Set.has()` | Winner |
|-----------|---------------------|-------------|--------|
| 10 items | ~0.002ms | ~0.003ms | Array (negligible) |
| 100 items | ~0.005ms | ~0.003ms | Set |
| 1,000 items | ~0.05ms | ~0.003ms | Set (~17x) |
| 10,000 items | ~0.5ms | ~0.003ms | Set (~167x) |
| 100,000 items | ~5ms | ~0.003ms | Set (~1,667x) |

The crossover point is around 50-100 elements. Below that, the overhead of Set's hash table initialization makes Array competitive. Above that, Set's O(1) lookup dominates Array's O(n) linear scan.

```js
// Rule of thumb: if you call .includes() or .indexOf() in a loop,
// convert to a Set first

// BAD: O(n * m) -- for each item, scan the whole blocklist
const blocklist = ['spam', 'phishing', 'malware', /* ...thousands */];
const messages = getMessages(); // thousands of messages

const filtered = messages.filter(msg =>
  !blocklist.includes(msg.category) // linear scan each time
);

// GOOD: O(n + m) -- build the Set once, then O(1) lookups
const blockSet = new Set(blocklist);
const filtered2 = messages.filter(msg =>
  !blockSet.has(msg.category)
);
```

## Converting Between Collections

### Map to/from Object

```js
// Map -> Object
const map = new Map([['name', 'Alice'], ['age', 30]]);
const obj = Object.fromEntries(map);
// { name: 'Alice', age: 30 }

// Object -> Map
const obj2 = { theme: 'dark', lang: 'en' };
const map2 = new Map(Object.entries(obj2));
// Map { 'theme' => 'dark', 'lang' => 'en' }

// GOTCHA: Object.fromEntries converts numeric keys to strings
const numMap = new Map([[1, 'one'], [2, 'two']]);
const numObj = Object.fromEntries(numMap);
// { '1': 'one', '2': 'two' } -- keys are now strings!
```

### Map to/from JSON

```js
// Map with string keys -> JSON (via Object)
function mapToJson(map) {
  return JSON.stringify(Object.fromEntries(map));
}

// JSON -> Map
function jsonToMap(json) {
  return new Map(Object.entries(JSON.parse(json)));
}

// Map with non-string keys -> JSON (via Array of pairs)
function mapToJsonFull(map) {
  return JSON.stringify([...map.entries()]);
}

// JSON -> Map (from Array of pairs)
function jsonToMapFull(json) {
  return new Map(JSON.parse(json));
}

// Example
const map = new Map([[1, 'one'], [{ x: 1 }, 'obj']]);
const json = mapToJsonFull(map);
// '[[1,"one"],[{"x":1},"obj"]]'
const restored = jsonToMapFull(json);
// Map { 1 => 'one', { x: 1 } => 'obj' }
// NOTE: the object key is a NEW object, not the same reference!
```

### Set to/from Array

```js
// Set -> Array (three equivalent ways)
const set = new Set([1, 2, 3]);

const arr1 = [...set];           // spread (most common)
const arr2 = Array.from(set);    // Array.from
const arr3 = Array.from(set, x => x * 2); // with mapping

// Array -> Set
const arr = [1, 2, 2, 3, 3, 3];
const unique = new Set(arr); // Set {1, 2, 3}

// Deduplicate preserving order
const deduped = [...new Set(arr)]; // [1, 2, 3]
```

### Map to/from Array

```js
// Map -> Array of pairs
const map = new Map([['a', 1], ['b', 2]]);
const pairs = [...map]; // [['a', 1], ['b', 2]]

// Map -> Array of keys
const keys = [...map.keys()]; // ['a', 'b']

// Map -> Array of values
const values = [...map.values()]; // [1, 2]

// Array of objects -> Map
const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
];
const userMap = new Map(users.map(u => [u.id, u]));
// Map { 1 => { id: 1, name: 'Alice' }, 2 => { id: 2, name: 'Bob' } }
```

## Iterability Summary

| Collection | Iterable | `for...of` | Spread | `size` | `keys()` | `values()` | `entries()` | `forEach()` |
|-----------|----------|------------|--------|--------|----------|------------|-------------|-------------|
| Map | Yes | Yes | Yes | Yes | Yes | Yes | Yes | Yes |
| Set | Yes | Yes | Yes | Yes | Yes | Yes | Yes | Yes |
| WeakMap | No | No | No | No | No | No | No | No |
| WeakSet | No | No | No | No | No | No | No | No |

WeakMap and WeakSet are intentionally non-iterable. If they were iterable, you could observe when garbage collection occurs, which would expose nondeterministic engine behavior and make programs behave differently across engines.

## Common Mistakes

### Mistake 1: Using Bracket Notation on Map

```js
const map = new Map();

// BAD: treats Map as a plain object, bypasses the Map data structure
map['key'] = 'value';
map['key'];           // 'value' -- but this is on the object, not in the Map
map.has('key');        // false!
map.get('key');        // undefined!
map.size;             // 0!

// GOOD: use Map methods
map.set('key', 'value');
map.get('key');        // 'value'
map.has('key');        // true
map.size;             // 1
```

### Mistake 2: Expecting Object-Like Key Comparison

```js
const map = new Map();

// You set a value with an object key
map.set({ x: 1 }, 'data');

// You try to retrieve it with a "matching" object
map.get({ x: 1 }); // undefined -- different reference!

// Same problem with arrays
map.set([1, 2], 'array data');
map.get([1, 2]); // undefined
```

### Mistake 3: Forgetting Set Stores by Reference for Objects

```js
const set = new Set();
set.add({ name: 'Alice' });
set.add({ name: 'Alice' });
set.size; // 2 -- these are different objects!

// If you want to deduplicate objects by a property, use Map:
const users = [
  { id: 1, name: 'Alice' },
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
];

// Deduplicate by id using Map
const uniqueUsers = [...new Map(users.map(u => [u.id, u])).values()];
// [{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }]
```

### Mistake 4: Using Map Where Object Is Better

```js
// BAD: Map for a fixed config
const config = new Map();
config.set('host', 'localhost');
config.set('port', 3000);
config.set('debug', true);
// Overly verbose, cannot destructure, no TypeScript type safety

// GOOD: Object for fixed structure
const config2 = {
  host: 'localhost',
  port: 3000,
  debug: true,
};
const { host, port } = config2; // destructuring works
```

### Mistake 5: JSON.stringify on Map or Set

```js
const map = new Map([['a', 1], ['b', 2]]);
JSON.stringify(map); // '{}' -- empty object!

const set = new Set([1, 2, 3]);
JSON.stringify(set); // '{}' -- empty object!

// You MUST convert first
JSON.stringify(Object.fromEntries(map));  // '{"a":1,"b":2}'
JSON.stringify([...set]);                 // '[1,2,3]'

// Or use a replacer function for nested structures
function replacer(key, value) {
  if (value instanceof Map) {
    return Object.fromEntries(value);
  }
  if (value instanceof Set) {
    return [...value];
  }
  return value;
}

const data = {
  users: new Map([['alice', { age: 30 }]]),
  tags: new Set(['js', 'ts']),
};

JSON.stringify(data, replacer);
// '{"users":{"alice":{"age":30}},"tags":["js","ts"]}'
```

### Mistake 6: Using Map/Set for WeakMap/WeakSet Use Cases

```js
// BAD: using Map to store metadata about DOM nodes
const elementData = new Map();

function attachData(el, data) {
  elementData.set(el, data);
}

// When the element is removed from the DOM, the Map still holds
// a strong reference. You must manually clean up:
function removeElement(el) {
  el.remove();
  elementData.delete(el); // easy to forget!
}

// GOOD: WeakMap handles cleanup automatically
const elementData2 = new WeakMap();

function attachData2(el, data) {
  elementData2.set(el, data);
}

function removeElement2(el) {
  el.remove();
  // No manual cleanup needed
}
```

### Mistake 7: Assuming WeakMap/WeakSet Have size or Iteration

```js
const wm = new WeakMap();
wm.set({}, 'val');

// These all fail or return unexpected results
wm.size;        // undefined (not a property)
[...wm];        // TypeError: wm is not iterable

// If you need to know the count, WeakMap is the wrong tool.
// Use a Map and handle cleanup yourself.
```

## Best Practices

### 1. Use Map for Dynamic Key-Value Data

```js
// Counting occurrences
function countWords(text) {
  const counts = new Map();
  for (const word of text.split(/\s+/)) {
    counts.set(word, (counts.get(word) || 0) + 1);
  }
  return counts;
}

const result = countWords('the cat sat on the mat');
// Map { 'the' => 2, 'cat' => 1, 'sat' => 1, 'on' => 1, 'mat' => 1 }
```

### 2. Use Set for Membership Testing

```js
// Feature flags
const enabledFeatures = new Set(['dark-mode', 'notifications', 'beta-ui']);

function isFeatureEnabled(feature) {
  return enabledFeatures.has(feature); // O(1)
}

// Permission checking
const userPermissions = new Set(['read', 'write', 'admin']);
const requiredPermissions = new Set(['read', 'write']);

const hasAll = requiredPermissions.isSubsetOf(userPermissions); // true
```

### 3. Use WeakMap for Object-to-Data Associations

```js
// Attaching metadata without modifying objects
const metadata = new WeakMap();

function enrich(obj, extra) {
  metadata.set(obj, { ...metadata.get(obj), ...extra });
}

function getMeta(obj) {
  return metadata.get(obj) || {};
}
```

### 4. Use WeakSet for Object-Level Flags

```js
// Marking objects as "seen" or "visited"
const visited = new WeakSet();

function traverse(node) {
  if (visited.has(node)) return;
  visited.add(node);

  // Process node
  for (const child of node.children) {
    traverse(child);
  }
}
```

### 5. Prefer Set Over Array for Uniqueness Constraints

```js
// BAD: manual dedup with array
const uniqueIds = [];
function addId(id) {
  if (!uniqueIds.includes(id)) { // O(n) scan
    uniqueIds.push(id);
  }
}

// GOOD: Set enforces uniqueness automatically
const uniqueIds2 = new Set();
function addId2(id) {
  uniqueIds2.add(id); // O(1), duplicates silently ignored
}
```

### 6. Convert to Set Before Repeated Lookups

```js
function findCommonElements(arr1, arr2) {
  const set = new Set(arr2); // O(m) one-time cost
  return arr1.filter(item => set.has(item)); // O(n) total
  // Total: O(n + m) instead of O(n * m)
}
```

### 7. Use Map for Bidirectional Lookups

```js
class BiMap {
  #forward = new Map();
  #reverse = new Map();

  set(key, value) {
    this.#forward.set(key, value);
    this.#reverse.set(value, key);
  }

  getByKey(key) {
    return this.#forward.get(key);
  }

  getByValue(value) {
    return this.#reverse.get(value);
  }
}

const statusCodes = new BiMap();
statusCodes.set(200, 'OK');
statusCodes.set(404, 'Not Found');

statusCodes.getByKey(200);         // 'OK'
statusCodes.getByValue('Not Found'); // 404
```

## Real-World Production Patterns

### Pattern 1: Event Bus with Weak Subscriber References

```js
class EventBus {
  #handlers = new Map(); // event -> Set<WeakRef<handler>>
  #registry = new FinalizationRegistry(({ event, ref }) => {
    const handlers = this.#handlers.get(event);
    if (handlers) {
      handlers.delete(ref);
      if (handlers.size === 0) {
        this.#handlers.delete(event);
      }
    }
  });

  on(event, handler) {
    if (!this.#handlers.has(event)) {
      this.#handlers.set(event, new Set());
    }
    const ref = new WeakRef(handler);
    this.#handlers.get(event).add(ref);
    this.#registry.register(handler, { event, ref });
  }

  emit(event, ...args) {
    const handlers = this.#handlers.get(event);
    if (!handlers) return;

    for (const ref of handlers) {
      const handler = ref.deref();
      if (handler) {
        handler(...args);
      } else {
        handlers.delete(ref);
      }
    }
  }
}
```

### Pattern 2: LRU Cache with Map (Insertion Order Exploitation)

Map's insertion order guarantee makes it naturally suited for LRU (Least Recently Used) caches:

```js
class LRUCache {
  #max;
  #cache = new Map();

  constructor(max = 100) {
    this.#max = max;
  }

  get(key) {
    if (!this.#cache.has(key)) return undefined;

    // Move to end (most recently used)
    const value = this.#cache.get(key);
    this.#cache.delete(key);
    this.#cache.set(key, value);
    return value;
  }

  set(key, value) {
    // If key exists, delete first to move it to the end
    if (this.#cache.has(key)) {
      this.#cache.delete(key);
    }

    this.#cache.set(key, value);

    // Evict the oldest entry (first in insertion order)
    if (this.#cache.size > this.#max) {
      const oldestKey = this.#cache.keys().next().value;
      this.#cache.delete(oldestKey);
    }
  }

  has(key) {
    return this.#cache.has(key);
  }

  get size() {
    return this.#cache.size;
  }
}
```

### Pattern 3: Object Pool with WeakSet

```js
const inUse = new WeakSet();

class ConnectionPool {
  #available = [];

  acquire() {
    let conn = this.#available.pop();
    if (!conn) {
      conn = createConnection();
    }
    inUse.add(conn);
    return conn;
  }

  release(conn) {
    if (!inUse.has(conn)) {
      throw new Error('Connection not from this pool');
    }
    inUse.delete(conn);
    this.#available.push(conn);
  }
}
```

### Pattern 4: Grouping with Map

```js
// Group an array by a key function
function groupBy(items, keyFn) {
  const groups = new Map();
  for (const item of items) {
    const key = keyFn(item);
    if (!groups.has(key)) {
      groups.set(key, []);
    }
    groups.get(key).push(item);
  }
  return groups;
}

const people = [
  { name: 'Alice', dept: 'Engineering' },
  { name: 'Bob', dept: 'Marketing' },
  { name: 'Charlie', dept: 'Engineering' },
  { name: 'Diana', dept: 'Marketing' },
];

const byDept = groupBy(people, p => p.dept);
// Map {
//   'Engineering' => [{ name: 'Alice', ... }, { name: 'Charlie', ... }],
//   'Marketing' => [{ name: 'Bob', ... }, { name: 'Diana', ... }]
// }

// NOTE: Object.groupBy() is now available as a native method (ES2024),
// but it returns a null-prototype object, not a Map.
// Use Map.groupBy() if you need a Map specifically.
```

### Pattern 5: Efficient Tag/Label System

```js
class TagSystem {
  #tagToItems = new Map();  // tag -> Set of items
  #itemToTags = new WeakMap(); // item -> Set of tags

  tag(item, ...tags) {
    if (!this.#itemToTags.has(item)) {
      this.#itemToTags.set(item, new Set());
    }
    const itemTags = this.#itemToTags.get(item);

    for (const t of tags) {
      itemTags.add(t);
      if (!this.#tagToItems.has(t)) {
        this.#tagToItems.set(t, new Set());
      }
      this.#tagToItems.get(t).add(item);
    }
  }

  getByTag(tag) {
    return this.#tagToItems.get(tag) || new Set();
  }

  getTagsFor(item) {
    return this.#itemToTags.get(item) || new Set();
  }

  // Find items that have ALL the given tags
  findWithAllTags(...tags) {
    if (tags.length === 0) return new Set();
    const sets = tags.map(t => this.getByTag(t));
    // Start with the smallest set for efficiency
    sets.sort((a, b) => a.size - b.size);
    let result = new Set(sets[0]);
    for (let i = 1; i < sets.length; i++) {
      result = result.intersection(sets[i]); // ES2025
    }
    return result;
  }
}
```

## Decision Guide: Which Collection to Use

```
Do you need key-value pairs?
├── Yes
│   ├── Are keys always strings? Is the shape fixed?
│   │   └── Yes -> Object
│   │   └── No
│   │       ├── Do keys need to be garbage-collected when unused?
│   │       │   └── Yes -> WeakMap
│   │       │   └── No -> Map
│   └──
└── No (just values)
    ├── Do you need uniqueness?
    │   ├── Yes
    │   │   ├── Are values objects that should be garbage-collected?
    │   │   │   └── Yes -> WeakSet
    │   │   │   └── No -> Set
    │   └── No -> Array
    └──
```

| Scenario | Use |
|----------|-----|
| Dynamic dictionary with non-string keys | Map |
| Config object, DTO, API response | Object |
| Counting, grouping, lookup tables | Map |
| Remove duplicates from array | Set |
| Membership testing (is X in the collection?) | Set |
| Track which objects have been processed | WeakSet |
| Store private data for class instances | WeakMap (or #private fields) |
| Cache expensive computations on objects | WeakMap |
| Attach metadata to DOM elements | WeakMap |
| Detect circular references in traversal | WeakSet |
| Validate object origin (branding) | WeakSet |
| LRU cache | Map (exploiting insertion order) |
