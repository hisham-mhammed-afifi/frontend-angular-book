# 5.23 Regular Expressions

> Regular expressions are patterns used to match character combinations in strings, enabling powerful search, validation, extraction, and transformation operations.

---

## Creating Regular Expressions

Two ways to create a regex:

```js
// Literal notation: compiled at script load time
const re1 = /\d+/g;

// Constructor: compiled at runtime (useful for dynamic patterns)
const re2 = new RegExp('\\d+', 'g');
const re3 = new RegExp(/\d+/, 'g'); // can pass a literal too
```

Use literals when the pattern is known at write time. Use the constructor when the pattern is dynamic (user input, variables).

### Double Escaping in the Constructor

Inside a string, backslashes are escape characters themselves. You must double-escape:

```js
// Literal: single backslash
const re1 = /\d+/;

// Constructor: MUST double-escape
const re2 = new RegExp('\\d+');  // Correct

// BAD: \d in a string literal is just 'd'
const re3 = new RegExp('\d+');   // Matches 'd+', not digits
```

### RegExp.escape() (ES2025)

Safely escape user input for use in a regex pattern:

```js
const userInput = 'price is $9.99 (USD)';
const escaped = RegExp.escape(userInput);
// 'price\x20is\x20\\$9\\.99\x20\\(USD\\)'

const re = new RegExp(escaped);
re.test('price is $9.99 (USD)'); // true

// Pre-ES2025 fallback:
function escapeRegExp(str) {
  return str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}
```

## Flags

| Flag | Name | Purpose |
|------|------|---------|
| `g` | Global | Match all occurrences, not just the first |
| `i` | Case-insensitive | `/abc/i` matches "ABC", "Abc", etc. |
| `m` | Multiline | `^` and `$` match start/end of each line |
| `s` | DotAll | `.` matches newline characters too |
| `u` | Unicode | Full Unicode support, surrogate pairs as single code points |
| `v` | UnicodeSets | Upgrade to `u` with set operations (ES2024). Mutually exclusive with `u` |
| `d` | HasIndices | Adds `indices` array with start/end positions per group |
| `y` | Sticky | Matches only at `lastIndex` position |

### The `d` Flag (HasIndices)

```js
const re = /(?<keyword>let|const|var)\s+(?<name>\w+)/d;
const match = re.exec('const foo = 42');

match.indices[0];          // [0, 9]  full match "const foo"
match.indices[1];          // [0, 5]  group 1 "const"
match.indices.groups.name; // [6, 9]  named group "foo"
```

### The `y` Flag (Sticky)

The sticky flag does NOT scan forward. It must match exactly at `lastIndex`:

```js
const gRegex = /\d+/g;
const yRegex = /\d+/y;
const str = 'abc 123';

gRegex.lastIndex = 0;
gRegex.exec(str);  // ["123"] at index 4 (scanned forward)

yRegex.lastIndex = 0;
yRegex.exec(str);  // null (no digits at index 0)

yRegex.lastIndex = 4;
yRegex.exec(str);  // ["123"] (digits start exactly at index 4)
```

The sticky flag is primarily useful for tokenizers/lexers where each match must immediately follow the previous one.

## Character Classes

| Class | Meaning | Equivalent |
|-------|---------|------------|
| `\d` | Digit | `[0-9]` |
| `\D` | Non-digit | `[^0-9]` |
| `\w` | Word character | `[a-zA-Z0-9_]` |
| `\W` | Non-word character | `[^a-zA-Z0-9_]` |
| `\s` | Whitespace | Spaces, tabs, newlines, and more |
| `\S` | Non-whitespace | Negation of `\s` |
| `.` | Any char except newline | With `s` flag: any char including newline |

### Character Sets and Negation

```js
/[aeiou]/     // Matches any vowel
/[a-zA-Z]/    // Matches any letter
/[0-9a-f]/i   // Matches hex digit
/[^aeiou]/    // Matches any NON-vowel

// Gotcha: hyphen position matters
/[a-z]/       // Range: a through z
/[-az]/       // Literal hyphen, a, or z (hyphen at start)
/[a\-z]/      // Literal a, hyphen, or z (escaped hyphen)
```

**Gotcha:** `.` inside character classes `[.]` is a literal dot, not a metacharacter.

**Gotcha:** `\b` inside character classes `[\b]` means backspace, not word boundary.

## Quantifiers

| Greedy | Lazy | Meaning |
|--------|------|---------|
| `*` | `*?` | 0 or more |
| `+` | `+?` | 1 or more |
| `?` | `??` | 0 or 1 |
| `{n}` | `{n}?` | Exactly n |
| `{n,}` | `{n,}?` | n or more |
| `{n,m}` | `{n,m}?` | Between n and m |

### Greedy vs Lazy

Greedy quantifiers match as much as possible, then backtrack. Lazy quantifiers match as little as possible, then expand.

```js
const str = '<em>hello</em> world <em>foo</em>';

// Greedy: grabs everything between first < and last >
str.match(/<.+>/);
// ["<em>hello</em> world <em>foo</em>"]

// Lazy: stops at the first >
str.match(/<.+?>/);
// ["<em>"]

// All matches with /g
str.match(/<.+?>/g);
// ["<em>", "</em>", "<em>", "</em>"]
```

**Better approach:** Use negated character classes instead of lazy quantifiers. They avoid backtracking entirely:

```js
// Instead of lazy /<.+?>/
/<[^>]+>/g
// Faster and more predictable
```

**Gotcha:** No spaces inside `{n,m}`:

```js
/\d{2, 4}/   // WRONG: treated as literal string "{2, 4}"
/\d{2,4}/    // Correct: matches 2 to 4 digits
```

## Anchors

| Anchor | Meaning |
|--------|---------|
| `^` | Start of string (or line with `m` flag) |
| `$` | End of string (or line with `m` flag) |
| `\b` | Word boundary |
| `\B` | Non-word boundary |

### Multiline Behavior

```js
const text = 'line1\nline2\nline3';

// Without m: ^ and $ match entire string boundaries
text.match(/^\w+$/);    // null (\n prevents full match)

// With m: ^ and $ match each line
text.match(/^\w+$/gm);  // ["line1", "line2", "line3"]
```

### Word Boundaries

```js
/\bcat\b/.test('the cat sat');  // true
/\bcat\b/.test('concatenate');  // false
/\bcat\b/.test('cat');          // true
```

**Gotcha:** `^` inside `[^...]` means negation, not anchor:

```js
/^abc/    // Anchor: matches "abc" at start of string
/[^abc]/  // Negation: matches any char that is NOT a, b, or c
```

## Groups and Capturing

### Capturing Groups

```js
/(foo)(bar)/.exec('foobar');
// ["foobar", "foo", "bar"]
// match[1] = "foo", match[2] = "bar"
```

### Non-Capturing Groups

Group without capturing (useful for alternation or quantifiers without polluting matches):

```js
/(?:foo|bar)baz/.exec('foobaz');
// ["foobaz"] -- no captured group for foo
```

### Named Capturing Groups

```js
const re = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/;
const match = '2024-06-15'.match(re);

match.groups.year;   // "2024"
match.groups.month;  // "06"
match.groups.day;    // "15"

// Destructuring
const { year, month, day } = match.groups;
```

### Backreferences

```js
// Numbered backreference: \1 refers to first captured group
/(["']).*?\1/.test('"hello"');   // true (opening " matches closing ")
/(["']).*?\1/.test('"hello\'');  // false (opening " doesn't match closing ')

// Named backreference: \k<name>
/(?<quote>["']).*?\k<quote>/.test("'hello'"); // true
```

### Duplicate Named Groups (ES2025)

Same name allowed in different alternatives:

```js
const dateRe = /(?<year>\d{4})-(?<month>\d{2})|(?<month>\d{2})\/(?<year>\d{4})/v;
// "2024-06" -> groups.year = "2024", groups.month = "06"
// "06/2024" -> groups.year = "2024", groups.month = "06"
```

## Lookahead and Lookbehind

Zero-length assertions that check for patterns without consuming characters.

### Lookahead

```js
// Positive lookahead (?=...): match X if followed by Y
/\d+(?= dollars)/.exec('100 dollars');  // ["100"]
/\d+(?= dollars)/.exec('100 euros');    // null

// Negative lookahead (?!...): match X if NOT followed by Y
/\d+(?! dollars)/.exec('100 euros');    // ["100"]
```

### Lookbehind

```js
// Positive lookbehind (?<=...): match X if preceded by Y
/(?<=\$)\d+/.exec('Price: $50');   // ["50"]

// Negative lookbehind (?<!...): match X if NOT preceded by Y
/(?<!\$)\d+/.exec('Price: 50');    // ["50"]
```

### Practical Examples

**Password strength with multiple lookaheads:**

```js
// At least 8 chars, 1 uppercase, 1 lowercase, 1 digit, 1 special char
const strongPwd = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[!@#$%^&*]).{8,}$/;
```

**Number formatting with commas:**

```js
'1234567890'.replace(/\B(?=(\d{3})+(?!\d))/g, ',');
// "1,234,567,890"
```

**Gotcha:** Lookahead placement matters:

```js
// WRONG: lookahead checks for "foo" at position, then tries to match "bar" at SAME position
/(?=foo)bar/.exec('foobar');  // null

// CORRECT: consume "foo" first, then check if "bar" follows
/foo(?=bar)/.exec('foobar');  // ["foo"]
```

## String Methods with Regex

### test() -- Returns boolean

```js
/\d+/.test('abc123');  // true
// Fastest for boolean checks
```

### exec() -- Detailed match info, stateful with /g

```js
const re = /(\d+)/g;
let m;
while ((m = re.exec('abc 123 def 456')) !== null) {
  console.log(m[0], 'at', m.index);
}
// "123" at 4
// "456" at 12
```

### match() -- Behavior changes with /g

```js
// Without /g: detailed match (like exec)
'abc 123 def 456'.match(/(\d+)/);
// ["123", "123", index: 4, ...]

// With /g: all matches, but NO capture groups
'abc 123 def 456'.match(/(\d+)/g);
// ["123", "456"]
```

### matchAll() -- Iterator with full details

```js
// REQUIRES /g flag
const matches = 'abc 123 def 456'.matchAll(/(?<num>\d+)/g);
for (const m of matches) {
  console.log(m[0], m.groups.num, m.index);
}
// "123" "123" 4
// "456" "456" 12
```

### replace() and replaceAll()

```js
// Simple replacement
'foo bar foo'.replace(/foo/, 'baz');   // "baz bar foo" (first only)
'foo bar foo'.replace(/foo/g, 'baz');  // "baz bar baz" (all)

// Capture group references
'John Smith'.replace(/(\w+) (\w+)/, '$2, $1');
// "Smith, John"

// Named group references
'2024-06-15'.replace(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
  '$<month>/$<day>/$<year>'
);
// "06/15/2024"

// Callback function
'hello world'.replace(/\b\w/g, char => char.toUpperCase());
// "Hello World"

// Callback with named groups
'2024-06-15'.replace(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
  (...args) => {
    const groups = args.at(-1);
    return `${groups.day}/${groups.month}/${groups.year}`;
  }
);
// "15/06/2024"
```

**Special replacement patterns:**

| Pattern | Inserts |
|---------|---------|
| `$$` | Literal `$` |
| `$&` | The matched substring |
| `` $` `` | Text before the match |
| `$'` | Text after the match |
| `$n` | nth capturing group |
| `$<name>` | Named capturing group |

### search() -- Returns index

```js
'hello world'.search(/world/);  // 6
'hello world'.search(/xyz/);    // -1
```

### split() with Regex

```js
'one,two;three four'.split(/[,;\s]+/);
// ["one", "two", "three", "four"]

// Capturing groups in split are included in results
'one,two;three'.split(/([,;])/);
// ["one", ",", "two", ";", "three"]
```

## The lastIndex Gotcha

Regex objects with `/g` or `/y` flags are **stateful**. The `lastIndex` property persists between calls to `test()` and `exec()`:

```js
const re = /foo/g;

re.test('foo bar'); // true  (lastIndex is now 3)
re.test('foo bar'); // false (starts at index 3, no more "foo")
re.test('foo bar'); // true  (lastIndex reset to 0 after failure)
```

This causes bugs when reusing the same regex across different strings:

```js
const re = /\d+/g;

re.test('abc 123');  // true, lastIndex = 7
re.test('xyz 456');  // false! Starts at index 7, but string is only 7 chars
```

### Solutions

```js
// 1. Don't use /g for boolean checks
const re = /\d+/;  // No g flag
re.test('abc 123'); // Always works correctly

// 2. Reset lastIndex manually
re.lastIndex = 0;

// 3. Use String methods (they reset lastIndex internally)
'abc 123'.match(/\d+/g);   // Always correct
'abc 123'.search(/\d+/);   // Always correct

// 4. Use matchAll() instead of exec() loops
[...'abc 123 def 456'.matchAll(/\d+/g)];
```

## Unicode Support

### The `u` Flag

Without the `u` flag, JavaScript regex treats strings as sequences of UTF-16 code units. Emoji and other characters outside the Basic Multilingual Plane are split:

```js
// Without u: emoji is two code units
/^.$/. test('😀');   // false
/^..$/. test('😀');  // true

// With u: emoji is one code point
/^.$/u.test('😀');   // true
```

### Unicode Property Escapes

Require the `u` or `v` flag:

```js
/\p{Letter}/u.test('a');   // true
/\p{Letter}/u.test('中');  // true
/\p{Letter}/u.test('1');   // false

// Script-based matching
/\p{Script=Greek}/u.test('α');     // true
/\p{Script=Cyrillic}/u.test('Д');  // true

// Common shorthand
/\p{L}/u    // Letter
/\p{N}/u    // Number
/\p{P}/u    // Punctuation
/\p{S}/u    // Symbol
/\p{Sc}/u   // Currency Symbol
/\p{Emoji}/u // Emoji

// Negation
/\P{Letter}/u.test('1'); // true (not a letter)
```

### The `v` Flag (ES2024 UnicodeSets)

An upgrade to `u` with set operations inside character classes:

```js
// Set intersection: A&&B
/[\p{ASCII}&&\p{Decimal_Number}]/v.test('4');  // true (ASCII digit)
/[\p{ASCII}&&\p{Decimal_Number}]/v.test('٣');  // false (Arabic digit)

// Set subtraction: A--B
/[\p{Decimal_Number}--[0-9]]/v.test('٣');  // true (non-ASCII digit)
/[\p{Decimal_Number}--[0-9]]/v.test('3');  // false (excluded)

// String literals in character classes
/[\q{abc|def}]/v.test('abc');  // true
/[\q{abc|def}]/v.test('ab');   // false

// Properties of strings (multi-code-point matching)
/^\p{RGI_Emoji_Flag_Sequence}$/v.test('🇺🇸');  // true
```

`u` and `v` are mutually exclusive. Prefer `v` for new code.

## Pattern Modifiers (ES2025)

Apply flags to a sub-section of the pattern:

```js
// Case-insensitive only for the first part
/(?i:hello) world/.test('HELLO world');  // true
/(?i:hello) world/.test('HELLO WORLD');  // false ("world" is case-sensitive)

// Disable case-insensitive for a section
/(?-i:exact)match/i.test('exactMATCH');  // true
/(?-i:exact)match/i.test('EXACTmatch');  // false ("exact" must be lowercase)
```

const isoDate = /^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12]\d|3[01])$/;

## Performance

### Catastrophic Backtracking (ReDoS)

Occurs when a regex explores exponentially many paths before failing:

```js
// DANGEROUS: nested quantifiers
const bad1 = /(a+)+$/;
// On "aaaaaaaaaaX": 2^n combinations tried

// DANGEROUS: overlapping alternatives with quantifier
const bad2 = /(\w+\s?)*$/;
// On "word word word !": exponential backtracking
```

A regex is vulnerable when:
1. A quantified group contains something that can match in multiple ways
2. The overall match can fail

### Solutions

```js
// 1. Use negated character classes instead of lazy .*?
// BAD:
/".*?"/
// GOOD:
/"[^"]*"/

// 2. Avoid nested quantifiers
// BAD:
/(a+)+/
// GOOD:
/a+/

// 3. Use bounded repetitions
// BAD:
/(\w+\s?)*/
// GOOD:
/(\w{1,100}\s?){1,50}/

// 4. Be specific with character classes
// BAD:
/.*@.*/
// GOOD:
/[^@]+@[^@]+/
```

### Other Performance Tips

```js
// Hoist regex outside loops
const phoneRe = /\d{3}-\d{4}/;
function validate(str) {
  return phoneRe.test(str); // Reuses compiled regex
}

// Use test() for boolean (faster than match)
/pattern/.test(str);

// Use anchors to fail faster
/^prefix/.test(str);

// Order alternations by frequency
/common|uncommon|rare/

// Use string methods for simple operations
str.includes('foo');     // Faster than /foo/.test(str)
str.startsWith('bar');   // Faster than /^bar/.test(str)
```

## Common Mistakes

### 1. Forgetting Double Escaping in Constructor

```js
// BAD: \d in string is just 'd'
new RegExp('\d+');

// GOOD
new RegExp('\\d+');
```

### 2. Using /g with test() in Loops

```js
const re = /foo/g;
// BUG: alternates true/false
[re.test('foo'), re.test('foo'), re.test('foo')];
// [true, false, true]

// FIX: remove /g flag for test()
```

### 3. Forgetting `.` Does Not Match Newlines

```js
/^.*$/.test('line1\nline2');    // false
/^.*$/s.test('line1\nline2');   // true (s flag)
/^[\s\S]*$/.test('line1\nline2'); // true (classic workaround)
```

### 4. Emoji Without the `u` Flag

```js
'😀'.match(/./g);    // ['\ud83d', '\ude00'] (broken)
'😀'.match(/./gu);   // ['😀'] (correct)
```

### 5. Misunderstanding `|` Precedence

```js
// BAD: matches ^foo OR bar$, not ^(foo|bar)$
/^foo|bar$/

// GOOD: group the alternation
/^(?:foo|bar)$/
```

### 6. `|` Inside Character Classes Is Literal

```js
/[a|b]/.test('|');  // true (matches a, |, or b)
/[ab]/.test('|');   // false (matches a or b only)
```

### 7. Forgetting Anchors for Validation

```js
/\d{3}/.test('abc1234');     // true! Matches "123" inside the string
/^\d{3}$/.test('abc1234');   // false (properly anchored)
```

### 8. Assuming match() with /g Returns Groups

```js
'a1 b2'.match(/([a-z])(\d)/g);
// ["a1", "b2"] -- groups are NOT included

// FIX: use matchAll()
[...'a1 b2'.matchAll(/([a-z])(\d)/g)];
// Full match objects with groups
```

## Best Practices

- **Use literal notation for static patterns.** Reserve the constructor for dynamic patterns.

- **Always use the `u` or `v` flag** for proper Unicode handling. Prefer `v` for new code.

- **Use `matchAll()` instead of `exec()` loops.** It returns a clean iterator and avoids `lastIndex` pitfalls.

- **Use named capture groups** for readability. `$<name>` in replacements is far clearer than `$1`.

- **Prefer negated character classes over lazy `.*?`**. `/"[^"]*"/` is faster and more predictable than `/".*?"/`.

- **Use `RegExp.escape()` (ES2025)** when building patterns from user input.

- **Use `test()` for boolean checks** and `match()`/`matchAll()` for extracting data.

- **Watch for catastrophic backtracking.** Avoid nested quantifiers and overlapping alternatives.

- **Anchor validation patterns** with `^...$` to prevent partial matches.

- **Use string methods for simple tasks.** `includes()`, `startsWith()`, `endsWith()` are faster and clearer than regex.

## Real-World Use Cases

### 1. Log Parser with Named Groups

```js
const logRe = /^(?<timestamp>\d{4}-\d{2}-\d{2}T[\d:.]+Z)\s+\[(?<level>\w+)]\s+(?<message>.+)$/gm;

const logs = `
2024-06-15T10:30:00.123Z [ERROR] Connection refused to database
2024-06-15T10:30:01.456Z [INFO] Retrying connection attempt 2
2024-06-15T10:30:02.789Z [ERROR] Connection timeout after 5000ms
`;

const errors = [...logs.matchAll(logRe)]
  .filter(m => m.groups.level === 'ERROR')
  .map(m => ({
    time: m.groups.timestamp,
    message: m.groups.message,
  }));

// [
//   { time: "2024-06-15T10:30:00.123Z", message: "Connection refused to database" },
//   { time: "2024-06-15T10:30:02.789Z", message: "Connection timeout after 5000ms" }
// ]
```

### 2. Template String Interpolation Engine

```js
function interpolate(template, data) {
  return template.replace(
    /\{\{(?<key>\w+(?:\.\w+)*)\}\}/g,
    (_, key) => {
      const value = key.split('.').reduce((obj, k) => obj?.[k], data);
      return value !== undefined ? String(value) : `{{${key}}}`;
    }
  );
}

const result = interpolate(
  'Hello {{user.name}}, you have {{count}} messages.',
  { user: { name: 'Alice' }, count: 5 }
);
// "Hello Alice, you have 5 messages."
```

### 3. Syntax Highlighter Tokenizer

```js
const TOKEN_RULES = [
  { type: 'KEYWORD',    re: /\b(?:const|let|var|function|return|if|else)\b/y },
  { type: 'NUMBER',     re: /\d+(?:\.\d+)?/y },
  { type: 'STRING',     re: /"[^"]*"|'[^']*'/y },
  { type: 'IDENTIFIER', re: /[a-zA-Z_$]\w*/y },
  { type: 'OPERATOR',   re: /[+\-*\/=<>!&|]+/y },
  { type: 'PUNCTUATION',re: /[{}()[\];,.]/y },
  { type: 'WHITESPACE', re: /\s+/y },
];

function tokenize(code) {
  const tokens = [];
  let pos = 0;

  while (pos < code.length) {
    let matched = false;
    for (const rule of TOKEN_RULES) {
      rule.re.lastIndex = pos;
      const match = rule.re.exec(code);
      if (match) {
        if (rule.type !== 'WHITESPACE') {
          tokens.push({ type: rule.type, value: match[0], start: pos });
        }
        pos = rule.re.lastIndex;
        matched = true;
        break;
      }
    }
    if (!matched) {
      throw new SyntaxError(`Unexpected character at position ${pos}: ${code[pos]}`);
    }
  }

  return tokens;
}

tokenize('const x = 42;');
// [
//   { type: "KEYWORD", value: "const", start: 0 },
//   { type: "IDENTIFIER", value: "x", start: 6 },
//   { type: "OPERATOR", value: "=", start: 8 },
//   { type: "NUMBER", value: "42", start: 10 },
//   { type: "PUNCTUATION", value: ";", start: 12 }
// ]
```

> **Key Takeaway:** Regular expressions are powerful for pattern matching but require care with stateful `/g` behavior, proper Unicode handling via the `u`/`v` flags, anchoring for validation, and avoiding catastrophic backtracking in production patterns.
