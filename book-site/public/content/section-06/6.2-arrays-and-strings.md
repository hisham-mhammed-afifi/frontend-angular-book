# 6.2 Arrays and Strings (Two Pointers, Sliding Window, Frequency Counters)

> Three foundational patterns that transform brute-force nested loops into clean, linear-time solutions for the majority of array and string problems you will encounter.

---

## Why These Three Patterns

Most array and string problems fall into one of a few categories:

- **Searching for pairs or subsequences** in sorted/unsorted data
- **Processing contiguous subarrays or substrings** of fixed or variable size
- **Comparing distributions** of elements between two collections

The naive approach to all of these involves nested loops (O(n^2) or worse). The three patterns in this guide reduce that to O(n) in nearly every case.

| Pattern | Core Idea | When to Use |
|---------|-----------|-------------|
| Two Pointers | Two indices traverse the data, moving toward each other or in the same direction | Sorted arrays, pair finding, partitioning, palindromes |
| Sliding Window | A window of elements expands and contracts as it moves through the data | Contiguous subarray/substring problems (max, min, count) |
| Frequency Counter | An object or Map tallies occurrences of each element | Comparing compositions, anagram checks, duplicate detection |

---

## Pattern 1: Two Pointers

### The Core Idea

Place two pointers (indices) at strategic positions in the data. Move them based on conditions until they converge or a result is found. This eliminates the need to check every possible pair.

### Variant A: Opposite-Direction Pointers

Start one pointer at the beginning and one at the end. Move them toward each other.

**Use when:** the input is sorted and you need to find pairs, or the problem involves symmetry (palindromes).

```js
// Find two numbers in a SORTED array that sum to a target
function twoSumSorted(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left < right) {
    const sum = arr[left] + arr[right];

    if (sum === target) return [left, right];
    if (sum < target) left++;   // Sum too small, move left pointer right
    else right--;               // Sum too large, move right pointer left
  }

  return null; // No pair found
}

// twoSumSorted([1, 3, 5, 7, 9, 11], 14)
// Step 1: 1 + 11 = 12 < 14 -> left++
// Step 2: 3 + 11 = 14 === 14 -> return [1, 5]

// Time: O(n), Space: O(1)
// Brute force would be O(n^2)
```

**Palindrome check:**

```js
function isPalindrome(str) {
  let left = 0;
  let right = str.length - 1;

  while (left < right) {
    if (str[left] !== str[right]) return false;
    left++;
    right--;
  }

  return true;
}

// isPalindrome("racecar") => true
// isPalindrome("hello")   => false

// Time: O(n/2) = O(n), Space: O(1)
```

**Three Sum (extending two pointers):**

```js
// Find all unique triplets that sum to zero
function threeSum(nums) {
  const result = [];
  nums.sort((a, b) => a - b); // Sort first: O(n log n)

  for (let i = 0; i < nums.length - 2; i++) {
    // Skip duplicate values for the first element
    if (i > 0 && nums[i] === nums[i - 1]) continue;

    let left = i + 1;
    let right = nums.length - 1;

    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];

      if (sum === 0) {
        result.push([nums[i], nums[left], nums[right]]);
        // Skip duplicates for the second and third elements
        while (left < right && nums[left] === nums[left + 1]) left++;
        while (left < right && nums[right] === nums[right - 1]) right--;
        left++;
        right--;
      } else if (sum < 0) {
        left++;
      } else {
        right--;
      }
    }
  }

  return result;
}

// threeSum([-1, 0, 1, 2, -1, -4])
// => [[-1, -1, 2], [-1, 0, 1]]

// Time: O(n^2) -- one loop * two pointers
// Without two pointers, brute force would be O(n^3)
```

### Variant B: Same-Direction Pointers (Fast/Slow)

Both pointers start at the same position and move in the same direction at different speeds or under different conditions.

**Use when:** removing duplicates in-place, partitioning, or detecting cycles.

```js
// Remove duplicates from a sorted array in-place
// Returns the new length of the unique portion
function removeDuplicates(arr) {
  if (arr.length === 0) return 0;

  let slow = 0; // Points to the last unique element

  for (let fast = 1; fast < arr.length; fast++) {
    if (arr[fast] !== arr[slow]) {
      slow++;
      arr[slow] = arr[fast]; // Overwrite the next slot with the unique value
    }
  }

  return slow + 1; // Length of unique portion
}

// arr = [1, 1, 2, 2, 3, 4, 4, 5]
// After: [1, 2, 3, 4, 5, ...] with return value 5
// Time: O(n), Space: O(1)
```

**Move zeroes to the end:**

```js
function moveZeroes(arr) {
  let slow = 0; // Position to place the next non-zero element

  // Fast pointer scans for non-zero elements
  for (let fast = 0; fast < arr.length; fast++) {
    if (arr[fast] !== 0) {
      // Swap non-zero element to the slow pointer position
      [arr[slow], arr[fast]] = [arr[fast], arr[slow]];
      slow++;
    }
  }

  return arr;
}

// moveZeroes([0, 1, 0, 3, 12])
// => [1, 3, 12, 0, 0]
// Time: O(n), Space: O(1)
```

**Squaring a sorted array:**

```js
// Given a sorted array (may contain negatives), return squares in sorted order
function sortedSquares(arr) {
  const result = new Array(arr.length);
  let left = 0;
  let right = arr.length - 1;
  let pos = arr.length - 1; // Fill result from the end

  while (left <= right) {
    const leftSq = arr[left] ** 2;
    const rightSq = arr[right] ** 2;

    if (leftSq > rightSq) {
      result[pos] = leftSq;
      left++;
    } else {
      result[pos] = rightSq;
      right--;
    }
    pos--;
  }

  return result;
}

// sortedSquares([-4, -1, 0, 3, 10])
// => [0, 1, 9, 16, 100]
// Time: O(n), Space: O(n)
// Without two pointers: sort after squaring = O(n log n)
```

### Variant C: Two Arrays, Two Pointers

One pointer per array. Useful for merging or comparing sorted data.

```js
// Merge two sorted arrays into one sorted array
function mergeSorted(arr1, arr2) {
  const result = [];
  let i = 0, j = 0;

  while (i < arr1.length && j < arr2.length) {
    if (arr1[i] <= arr2[j]) {
      result.push(arr1[i]);
      i++;
    } else {
      result.push(arr2[j]);
      j++;
    }
  }

  // Append remaining elements from whichever array is not exhausted
  while (i < arr1.length) result.push(arr1[i++]);
  while (j < arr2.length) result.push(arr2[j++]);

  return result;
}

// mergeSorted([1, 3, 5], [2, 4, 6])
// => [1, 2, 3, 4, 5, 6]
// Time: O(n + m), Space: O(n + m)
```

**Finding intersection of two sorted arrays:**

```js
function intersection(arr1, arr2) {
  const result = [];
  let i = 0, j = 0;

  while (i < arr1.length && j < arr2.length) {
    if (arr1[i] === arr2[j]) {
      // Avoid duplicates in result
      if (result.length === 0 || result[result.length - 1] !== arr1[i]) {
        result.push(arr1[i]);
      }
      i++;
      j++;
    } else if (arr1[i] < arr2[j]) {
      i++;
    } else {
      j++;
    }
  }

  return result;
}

// intersection([1, 2, 2, 3, 4], [2, 2, 3, 5])
// => [2, 3]
// Time: O(n + m), Space: O(min(n, m))
```

### When Two Pointers Works

- Input is sorted (or can be sorted as a preprocessing step)
- You need to find pairs or groups satisfying a condition
- You need to partition or rearrange elements in place
- You need to process two sorted sequences together

### When Two Pointers Does NOT Work

- The input is unsorted and sorting would lose important information (e.g., original indices)
- You need to find elements that are not positionally related (use a hash map instead)
- The relationship between elements is not monotonic (moving a pointer does not consistently move you toward or away from the solution)

## Pattern 2: Sliding Window

### The Core Idea

Maintain a "window" (a contiguous range of elements) defined by two boundaries. Instead of recalculating everything from scratch for each position, **update incrementally** by adding the new element entering the window and removing the element leaving it.

### Fixed-Size Window

The window size `k` is given. Slide it one position at a time.

```js
// Find the maximum sum of any contiguous subarray of size k
function maxSubarraySum(arr, k) {
  if (arr.length < k) return null;

  // Calculate the sum of the first window
  let windowSum = 0;
  for (let i = 0; i < k; i++) {
    windowSum += arr[i];
  }

  let maxSum = windowSum;

  // Slide the window: add the next element, remove the first element
  for (let i = k; i < arr.length; i++) {
    windowSum += arr[i];       // Add element entering the window
    windowSum -= arr[i - k];   // Remove element leaving the window
    maxSum = Math.max(maxSum, windowSum);
  }

  return maxSum;
}

// maxSubarraySum([2, 1, 5, 1, 3, 2], 3)
// Windows: [2,1,5]=8, [1,5,1]=7, [5,1,3]=9, [1,3,2]=6
// => 9

// Time: O(n), Space: O(1)
// Brute force (recalculating sum for each window): O(n * k)
```

**Moving average:**

```js
// Compute the moving average of the last k elements as you process a stream
function movingAverage(arr, k) {
  const averages = [];
  let windowSum = 0;

  for (let i = 0; i < arr.length; i++) {
    windowSum += arr[i];

    if (i >= k) {
      windowSum -= arr[i - k]; // Remove the element that slid out
    }

    if (i >= k - 1) {
      averages.push(windowSum / k);
    }
  }

  return averages;
}

// movingAverage([1, 3, 2, 6, -1, 4, 1, 8, 2], 5)
// => [2.2, 2.8, 2.4, 2.8, 2.8]
```

**Check if any permutation of `pattern` exists as a substring:**

```js
function checkInclusion(pattern, text) {
  if (pattern.length > text.length) return false;

  const pFreq = new Array(26).fill(0); // Frequency of pattern characters
  const wFreq = new Array(26).fill(0); // Frequency of window characters

  const charIndex = (c) => c.charCodeAt(0) - 'a'.charCodeAt(0);

  // Build frequency of pattern
  for (const c of pattern) pFreq[charIndex(c)]++;

  for (let i = 0; i < text.length; i++) {
    // Add character entering the window
    wFreq[charIndex(text[i])]++;

    // Remove character leaving the window (if window exceeds pattern length)
    if (i >= pattern.length) {
      wFreq[charIndex(text[i - pattern.length])]--;
    }

    // Compare frequencies
    if (i >= pattern.length - 1) {
      if (pFreq.every((val, idx) => val === wFreq[idx])) {
        return true;
      }
    }
  }

  return false;
}

// checkInclusion("ab", "eidbaooo") => true ("ba" is a permutation of "ab")
// Time: O(n * 26) = O(n), Space: O(1)
```

### Variable-Size Window

The window grows or shrinks based on a condition. This is the more powerful (and trickier) variant.

**Template for "Longest" problems:**

```js
function longestWindowTemplate(arr /* or string */) {
  let left = 0;
  let maxLength = 0;
  // ... state tracking (Map, counter, etc.)

  for (let right = 0; right < arr.length; right++) {
    // 1. Expand: add arr[right] to window state

    // 2. Shrink: while window is INVALID, remove arr[left] and move left++
    while (/* window is invalid */) {
      // Remove arr[left] from window state
      left++;
    }

    // 3. Update: window is now valid, check if it's the best
    maxLength = Math.max(maxLength, right - left + 1);
  }

  return maxLength;
}
```

**Template for "Shortest" problems:**

```js
function shortestWindowTemplate(arr /* or string */) {
  let left = 0;
  let minLength = Infinity;
  // ... state tracking

  for (let right = 0; right < arr.length; right++) {
    // 1. Expand: add arr[right] to window state

    // 2. Shrink: while window is VALID, update result and shrink
    while (/* window is valid */) {
      minLength = Math.min(minLength, right - left + 1);
      // Remove arr[left] from window state
      left++;
    }
  }

  return minLength === Infinity ? 0 : minLength;
}
```

Notice the key difference: for "longest" you shrink when **invalid** and update after shrinking. For "shortest" you shrink when **valid** and update before shrinking.

### Variable Window Examples

**Longest substring without repeating characters:**

```js
function lengthOfLongestSubstring(str) {
  const seen = new Map(); // character -> its most recent index
  let left = 0;
  let maxLength = 0;

  for (let right = 0; right < str.length; right++) {
    const char = str[right];

    // If we've seen this char and it's within the current window, jump left past it
    if (seen.has(char) && seen.get(char) >= left) {
      left = seen.get(char) + 1;
    }

    seen.set(char, right);
    maxLength = Math.max(maxLength, right - left + 1);
  }

  return maxLength;
}

// lengthOfLongestSubstring("abcabcbb")
// Window moves: "a" -> "ab" -> "abc" -> "bca" -> "cab" -> "abc" -> "cb" -> "b"
// Max length: 3 ("abc")

// Time: O(n), Space: O(min(n, alphabet_size))
```

**Smallest subarray with sum >= target:**

```js
function minSubArrayLen(target, arr) {
  let left = 0;
  let sum = 0;
  let minLength = Infinity;

  for (let right = 0; right < arr.length; right++) {
    sum += arr[right]; // Expand

    // Shrink while the window sum meets or exceeds the target
    while (sum >= target) {
      minLength = Math.min(minLength, right - left + 1);
      sum -= arr[left]; // Remove leftmost element
      left++;
    }
  }

  return minLength === Infinity ? 0 : minLength;
}

// minSubArrayLen(7, [2, 3, 1, 2, 4, 3])
// => 2 (subarray [4, 3])

// Time: O(n) -- each element is added and removed at most once
// Space: O(1)
```

**Longest substring with at most K distinct characters:**

```js
function longestWithKDistinct(str, k) {
  const freq = new Map();
  let left = 0;
  let maxLength = 0;

  for (let right = 0; right < str.length; right++) {
    const char = str[right];
    freq.set(char, (freq.get(char) || 0) + 1); // Add to window

    // Shrink until we have at most k distinct characters
    while (freq.size > k) {
      const leftChar = str[left];
      freq.set(leftChar, freq.get(leftChar) - 1);
      if (freq.get(leftChar) === 0) freq.delete(leftChar);
      left++;
    }

    maxLength = Math.max(maxLength, right - left + 1);
  }

  return maxLength;
}

// longestWithKDistinct("araaci", 2)
// => 4 ("araa")

// Time: O(n), Space: O(k)
```

### Why the Sliding Window is O(n) and Not O(n^2)

It looks like there is a nested loop (the `for` and the `while`), but the `left` pointer only moves forward. Across the entire execution, `left` moves from 0 to at most n. The `right` pointer also moves from 0 to n. So the total number of operations is at most 2n = O(n). Each element is processed (added to the window and removed from the window) at most once.

## Pattern 3: Frequency Counter

### The Core Idea

Build a frequency map (object, `Map`, or array) of elements. Use it to compare, count, or match distributions without nested iteration.

```js
// Build a frequency map from an array or string
function buildFrequency(input) {
  const freq = {};
  for (const item of input) {
    freq[item] = (freq[item] || 0) + 1;
  }
  return freq;
}

// buildFrequency("hello") => { h: 1, e: 1, l: 2, o: 1 }
// buildFrequency([1, 2, 1, 3, 2, 1]) => { 1: 3, 2: 2, 3: 1 }
```

### Anagram Detection

Two strings are anagrams if they have the same character frequency.

```js
// BAD: Sort and compare -- O(n log n)
function isAnagramSlow(str1, str2) {
  return [...str1].sort().join('') === [...str2].sort().join('');
}

// GOOD: Frequency counter -- O(n)
function isAnagram(str1, str2) {
  if (str1.length !== str2.length) return false;

  const freq = {};

  // Count characters in str1
  for (const char of str1) {
    freq[char] = (freq[char] || 0) + 1;
  }

  // Subtract characters in str2
  for (const char of str2) {
    if (!freq[char]) return false; // Character not found or count exhausted
    freq[char]--;
  }

  return true;
}

// isAnagram("listen", "silent") => true
// isAnagram("hello", "world") => false

// Time: O(n), Space: O(k) where k = unique characters
```

### Comparing Array Compositions

```js
// Check if arr2 contains the squares of arr1 elements with matching frequency
function sameSquared(arr1, arr2) {
  if (arr1.length !== arr2.length) return false;

  const freq1 = {};
  const freq2 = {};

  for (const val of arr1) freq1[val] = (freq1[val] || 0) + 1;
  for (const val of arr2) freq2[val] = (freq2[val] || 0) + 1;

  for (const key in freq1) {
    const squared = key ** 2;
    if (freq2[squared] !== freq1[key]) return false;
  }

  return true;
}

// sameSquared([1, 2, 3], [9, 1, 4]) => true
// sameSquared([1, 2, 3], [9, 1, 1]) => false (frequency mismatch)

// Time: O(n), Space: O(n)
// Brute force with nested loops: O(n^2)
```

### Finding Duplicates

```js
// Check if any value appears more than once
function hasDuplicate(arr) {
  const freq = {};
  for (const item of arr) {
    if (freq[item]) return true;
    freq[item] = 1;
  }
  return false;
}
// Time: O(n), Space: O(n)

// Alternatively, with a Set (slightly cleaner)
function hasDuplicateSet(arr) {
  return new Set(arr).size !== arr.length;
}
// Time: O(n), Space: O(n)
```

### Find All Anagrams in a String

This combines frequency counter with sliding window:

```js
function findAnagrams(text, pattern) {
  const result = [];
  if (pattern.length > text.length) return result;

  const pFreq = {};
  const wFreq = {};

  // Build pattern frequency
  for (const c of pattern) pFreq[c] = (pFreq[c] || 0) + 1;

  let matched = 0; // Number of characters with matching frequency
  const required = Object.keys(pFreq).length;

  for (let right = 0; right < text.length; right++) {
    // Add character to window
    const rChar = text[right];
    wFreq[rChar] = (wFreq[rChar] || 0) + 1;
    if (pFreq[rChar] && wFreq[rChar] === pFreq[rChar]) matched++;

    // Remove character leaving the window
    if (right >= pattern.length) {
      const lChar = text[right - pattern.length];
      if (pFreq[lChar] && wFreq[lChar] === pFreq[lChar]) matched--;
      wFreq[lChar]--;
    }

    // Check if all characters match
    if (matched === required) {
      result.push(right - pattern.length + 1);
    }
  }

  return result;
}

// findAnagrams("cbaebabacd", "abc")
// => [0, 6]
// Index 0: "cba" is an anagram of "abc"
// Index 6: "bac" is an anagram of "abc"

// Time: O(n), Space: O(k)
```

### Using Map vs Object for Frequency Counting

```js
// Object works for string keys and small integer keys
const freqObj = {};
for (const char of str) {
  freqObj[char] = (freqObj[char] || 0) + 1;
}

// Map works for ANY key type (objects, numbers, etc.)
const freqMap = new Map();
for (const item of arr) {
  freqMap.set(item, (freqMap.get(item) || 0) + 1);
}

// Use Map when:
// - Keys are not strings (objects, numbers with -0/+0 distinction)
// - You need .size for O(1) count of unique keys
// - You need guaranteed insertion order iteration
// - You want to avoid prototype pollution

// Use a plain object when:
// - Keys are strings or easily coerced strings
// - You want slightly simpler syntax
// - Performance difference is negligible for your use case
```

### Using Array as Frequency Counter (for Known Ranges)

When the input consists of lowercase English letters (or any fixed, small range), an array is faster than an object or Map.

```js
function charFrequencyArray(str) {
  const freq = new Array(26).fill(0);
  const base = 'a'.charCodeAt(0);

  for (const char of str) {
    freq[char.charCodeAt(0) - base]++;
  }

  return freq;
}

// charFrequencyArray("abca") => [2, 1, 1, 0, 0, ..., 0]
// Index 0 = 'a' count, Index 1 = 'b' count, etc.

// Comparison is trivial:
function isAnagramArray(s1, s2) {
  if (s1.length !== s2.length) return false;
  const f1 = charFrequencyArray(s1);
  const f2 = charFrequencyArray(s2);
  return f1.every((v, i) => v === f2[i]);
}
```

## Combining Patterns

Real problems often combine two or more patterns. Recognizing which patterns apply is the core skill.

### Two Pointers + Frequency Counter

```js
// Find the longest substring with at most 2 distinct characters
// Two pointers define the window, frequency counter tracks distinct chars
function longestTwoDistinct(str) {
  const freq = new Map();
  let left = 0;
  let maxLen = 0;

  for (let right = 0; right < str.length; right++) {
    freq.set(str[right], (freq.get(str[right]) || 0) + 1);

    while (freq.size > 2) {
      const lChar = str[left];
      freq.set(lChar, freq.get(lChar) - 1);
      if (freq.get(lChar) === 0) freq.delete(lChar);
      left++;
    }

    maxLen = Math.max(maxLen, right - left + 1);
  }

  return maxLen;
}

// longestTwoDistinct("eceba") => 3 ("ece")
```

### Sliding Window + Prefix Sum

```js
// Count subarrays with sum exactly equal to target
// Use a Map to store prefix sums (a frequency counter of prefix sums)
function subarraySum(arr, target) {
  const prefixCount = new Map();
  prefixCount.set(0, 1); // Empty prefix has sum 0

  let sum = 0;
  let count = 0;

  for (const num of arr) {
    sum += num;
    // If (sum - target) was seen before, those prefixes form valid subarrays
    if (prefixCount.has(sum - target)) {
      count += prefixCount.get(sum - target);
    }
    prefixCount.set(sum, (prefixCount.get(sum) || 0) + 1);
  }

  return count;
}

// subarraySum([1, 1, 1], 2) => 2
// Subarrays: [1,1] at index 0-1 and [1,1] at index 1-2

// Time: O(n), Space: O(n)
// Note: this is NOT a sliding window per se, but a frequency counter on prefix sums
// A sliding window won't work here because elements can be negative
```

## Choosing the Right Pattern

| Problem Characteristic | Pattern | Why |
|----------------------|---------|-----|
| Sorted array, find pair with sum | Two Pointers (opposite) | Sorted order lets you adjust by moving pointers |
| Check palindrome | Two Pointers (opposite) | Compare symmetric positions |
| Remove duplicates in-place | Two Pointers (fast/slow) | Slow pointer tracks write position |
| Merge sorted arrays | Two Pointers (one per array) | Process both in order |
| Max/min/count of subarray of size k | Sliding Window (fixed) | Slide and update incrementally |
| Longest/shortest subarray meeting condition | Sliding Window (variable) | Expand right, shrink left |
| Are these two strings anagrams? | Frequency Counter | Compare character distributions |
| Find all anagram occurrences | Sliding Window + Frequency Counter | Fixed window + frequency match |
| Count elements with specific frequency | Frequency Counter | Tally and query |
| Subarray with exact sum (positives only) | Sliding Window (variable) | Shrink when sum exceeds target |
| Subarray with exact sum (any integers) | Prefix Sum + Frequency Counter | Can't use window with negatives |

## Common Mistakes

- **Mistake 1: Using two pointers on unsorted data for pair-finding.** The opposite-direction pointer technique only works when the sort order guarantees that moving a pointer in one direction consistently increases or decreases a value. If you need pair-finding on unsorted data, use a hash map instead.

- **Mistake 2: Not handling the window shrink correctly.** In variable-size windows, forgetting to undo the state change (removing the element from a frequency map, subtracting from a running sum) when shrinking the left boundary causes incorrect results. Every "add" operation in the expand step needs a symmetric "remove" in the shrink step.

```js
// BAD: Forgetting to undo state when shrinking
while (freq.size > k) {
  left++; // Moved the pointer but did not update freq!
}

// GOOD: Undo state before moving
while (freq.size > k) {
  const lChar = str[left];
  freq.set(lChar, freq.get(lChar) - 1);
  if (freq.get(lChar) === 0) freq.delete(lChar);
  left++;
}
```

- **Mistake 3: Off-by-one errors in window size calculation.** The length of a window from index `left` to index `right` is `right - left + 1`, not `right - left`. This mistake silently returns results that are off by one.

- **Mistake 4: Using object keys for numeric frequency counting with mixed types.** Object keys are always strings. `obj[1]` and `obj["1"]` reference the same key. If you need to distinguish numeric types, use a `Map`.

```js
// BAD: Object coerces keys
const freq = {};
freq[1] = 5;
freq["1"] = 10;
console.log(freq[1]); // 10 -- overwritten!

// GOOD: Map preserves key types
const freq2 = new Map();
freq2.set(1, 5);
freq2.set("1", 10);
console.log(freq2.get(1));   // 5
console.log(freq2.get("1")); // 10
```

- **Mistake 5: Applying sliding window when elements can be negative.** The variable-size sliding window for "subarray with sum >= target" only works with positive numbers. With negatives, adding more elements can decrease the sum, breaking the monotonic assumption. Use prefix sums with a frequency counter instead.

- **Mistake 6: Creating a new frequency map for each window position.** The entire point of the sliding window is incremental updates. Rebuilding the frequency counter from scratch inside the loop turns O(n) back into O(n * k).

```js
// BAD: O(n * k) -- rebuilds frequency for each window
for (let i = 0; i <= str.length - k; i++) {
  const window = str.slice(i, i + k);
  const freq = buildFrequency(window); // O(k) each time
  // check freq...
}

// GOOD: O(n) -- incremental update
// Add new char, remove old char, check freq
```

## Best Practices

- **Start by identifying the pattern.** Before coding, ask: "Is the input sorted?" (two pointers), "Am I looking at contiguous subarrays?" (sliding window), "Am I comparing element distributions?" (frequency counter). This single decision eliminates 90% of the thinking.

- **Dry-run on paper first.** Walk through 5-6 iterations with a small input before coding. Draw the pointers, the window boundaries, and the frequency map at each step. Bugs in these patterns are almost always in the state transitions, and dry-running catches them.

- **Use the templates.** The "longest" and "shortest" sliding window templates are battle-tested. Start from the template and customize the condition and state tracking. Do not reinvent the structure each time.

- **Handle empty inputs and edge cases upfront.** Check `arr.length === 0`, `k > arr.length`, or `str.length < pattern.length` at the top of the function before entering the main logic.

- **Choose the right data structure for frequency tracking.** Use a plain object for string characters, a `Map` for arbitrary key types, and a fixed-size array for known small ranges (26 lowercase letters, ASCII range, digit range).

- **Know when NOT to use these patterns.** If the problem requires considering non-contiguous elements (subsequences, not substrings), sliding window does not apply. If the data has no meaningful order, two pointers on sorted data adds an unnecessary O(n log n) sort.

## Real-World Use Cases

### Use Case 1: Form Input Validation (Password Strength)

```js
// Check if a password has at least one uppercase, one lowercase,
// one digit, and one special character, with no more than 3 consecutive
// identical characters

function validatePassword(password) {
  if (password.length < 8) return { valid: false, reason: "Too short" };

  // Frequency counter for character categories
  const categories = { upper: 0, lower: 0, digit: 0, special: 0 };

  for (const char of password) {
    if (/[A-Z]/.test(char)) categories.upper++;
    else if (/[a-z]/.test(char)) categories.lower++;
    else if (/[0-9]/.test(char)) categories.digit++;
    else categories.special++;
  }

  // Check all categories are represented
  for (const [cat, count] of Object.entries(categories)) {
    if (count === 0) return { valid: false, reason: `Missing ${cat}` };
  }

  // Sliding window: check for 3+ consecutive identical characters
  for (let i = 2; i < password.length; i++) {
    if (password[i] === password[i - 1] && password[i] === password[i - 2]) {
      return { valid: false, reason: "3+ consecutive identical characters" };
    }
  }

  return { valid: true };
}

// validatePassword("Aa1!xxxx") => { valid: true }
// validatePassword("Aa1!aaa")  => { valid: false, reason: "3+ consecutive..." }
// validatePassword("password") => { valid: false, reason: "Missing upper" }
```

### Use Case 2: Log Analysis (Rate Limiting)

```js
// Count requests per IP in a sliding time window
// Determine if any IP exceeds the rate limit

function detectRateLimitViolations(logs, windowMs, maxRequests) {
  // Group logs by IP
  const byIp = new Map();
  for (const log of logs) {
    if (!byIp.has(log.ip)) byIp.set(log.ip, []);
    byIp.get(log.ip).push(log.timestamp);
  }

  const violations = [];

  for (const [ip, timestamps] of byIp) {
    timestamps.sort((a, b) => a - b); // Ensure sorted order

    // Sliding window: fixed time range, variable element count
    let left = 0;

    for (let right = 0; right < timestamps.length; right++) {
      // Shrink window: remove timestamps outside the time window
      while (timestamps[right] - timestamps[left] > windowMs) {
        left++;
      }

      // Count of requests in window = right - left + 1
      if (right - left + 1 > maxRequests) {
        violations.push({
          ip,
          count: right - left + 1,
          windowStart: timestamps[left],
          windowEnd: timestamps[right]
        });
        break; // One violation per IP is enough
      }
    }
  }

  return violations;
}

// Usage
const logs = [
  { ip: "192.168.1.1", timestamp: 1000 },
  { ip: "192.168.1.1", timestamp: 1200 },
  { ip: "192.168.1.1", timestamp: 1400 },
  { ip: "192.168.1.1", timestamp: 1500 },
  { ip: "192.168.1.1", timestamp: 1600 },
  { ip: "192.168.1.2", timestamp: 1000 },
];

detectRateLimitViolations(logs, 1000, 3);
// => [{ ip: "192.168.1.1", count: 4, ... }]
```

### Use Case 3: Search Result Deduplication

```js
// Given search results from multiple sources, remove near-duplicates
// based on title similarity using a frequency counter approach

function tokenize(text) {
  return text.toLowerCase().replace(/[^a-z0-9\s]/g, '').split(/\s+/).filter(Boolean);
}

function cosineSimilarity(tokens1, tokens2) {
  // Build frequency counters
  const freq1 = new Map();
  const freq2 = new Map();

  for (const t of tokens1) freq1.set(t, (freq1.get(t) || 0) + 1);
  for (const t of tokens2) freq2.set(t, (freq2.get(t) || 0) + 1);

  // Calculate dot product and magnitudes
  let dotProduct = 0;
  let mag1 = 0;
  let mag2 = 0;

  const allTokens = new Set([...freq1.keys(), ...freq2.keys()]);

  for (const token of allTokens) {
    const v1 = freq1.get(token) || 0;
    const v2 = freq2.get(token) || 0;
    dotProduct += v1 * v2;
    mag1 += v1 * v1;
    mag2 += v2 * v2;
  }

  if (mag1 === 0 || mag2 === 0) return 0;
  return dotProduct / (Math.sqrt(mag1) * Math.sqrt(mag2));
}

function deduplicateResults(results, threshold = 0.8) {
  const unique = [];

  for (const result of results) {
    const tokens = tokenize(result.title);
    let isDuplicate = false;

    for (const existing of unique) {
      const existingTokens = tokenize(existing.title);
      if (cosineSimilarity(tokens, existingTokens) >= threshold) {
        isDuplicate = true;
        break;
      }
    }

    if (!isDuplicate) unique.push(result);
  }

  return unique;
}

// Usage
const results = [
  { title: "JavaScript Array Methods Guide" },
  { title: "Guide to JavaScript Array Methods" },  // Near-duplicate
  { title: "CSS Grid Layout Tutorial" },
  { title: "Array Methods in JavaScript: A Guide" }, // Near-duplicate
];

deduplicateResults(results);
// => [{ title: "JavaScript Array Methods Guide" }, { title: "CSS Grid Layout Tutorial" }]
```
