# 6.6 Hash Maps and Hash Sets (Lookup Optimization)

> Hash-based data structures provide O(1) average-time lookups, insertions, and deletions. They are the single most important tool for turning O(n^2) brute-force solutions into O(n) algorithms.

---

## How Hash Tables Work

A hash table maps keys to values using a **hash function** that converts a key into an array index (called a bucket). When you look up a key, the hash function tells you exactly where to find the value, without scanning the entire collection.

```
Key "alice" --hash function--> index 3 --> bucket[3] stores the value

Instead of searching all N entries: O(n)
Jump directly to the bucket: O(1)
```

### Hash Function

A hash function takes input of any size and produces a fixed-size integer. For a hash table with `m` buckets, the index is `hash(key) % m`.

```js
// Simplified string hash function (for illustration only)
function simpleHash(str, tableSize) {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    hash = (hash * 31 + str.charCodeAt(i)) % tableSize;
  }
  return hash;
}

simpleHash("alice", 16); // Some index between 0 and 15
simpleHash("bob", 16);   // A different index (usually)
```

### Collisions

Two different keys can hash to the same index. This is called a **collision**. Every hash table must handle collisions.

**Chaining:** Each bucket stores a linked list (or array) of entries. On collision, the new entry is appended to the list.

```js
// Conceptual hash table with chaining
class HashTable {
  constructor(size = 16) {
    this.buckets = new Array(size).fill(null).map(() => []);
    this.size = 0;
  }

  _hash(key) {
    let hash = 0;
    const str = String(key);
    for (let i = 0; i < str.length; i++) {
      hash = (hash * 31 + str.charCodeAt(i)) % this.buckets.length;
    }
    return hash;
  }

  set(key, value) {
    const index = this._hash(key);
    const bucket = this.buckets[index];

    // Check if key already exists
    for (const entry of bucket) {
      if (entry[0] === key) {
        entry[1] = value; // Update
        return;
      }
    }

    bucket.push([key, value]); // Insert
    this.size++;
  }

  get(key) {
    const index = this._hash(key);
    const bucket = this.buckets[index];

    for (const entry of bucket) {
      if (entry[0] === key) return entry[1];
    }
    return undefined;
  }

  has(key) {
    return this.get(key) !== undefined;
  }

  delete(key) {
    const index = this._hash(key);
    const bucket = this.buckets[index];

    for (let i = 0; i < bucket.length; i++) {
      if (bucket[i][0] === key) {
        bucket.splice(i, 1);
        this.size--;
        return true;
      }
    }
    return false;
  }
}
```

**Open addressing (linear probing):** On collision, search the next available slot. V8 uses this approach for `Map` and `Set`.

### Load Factor and Rehashing

The **load factor** is `entries / buckets`. As it increases, collisions become more frequent and performance degrades. When the load factor exceeds a threshold (typically 0.75), the table is **rehashed**: a new, larger array is allocated, and all entries are re-inserted.

This rehashing is O(n) but happens rarely, making the **amortized** cost of each operation O(1).

### Time Complexity

| Operation | Average | Worst (all collisions) |
|-----------|---------|----------------------|
| Insert | O(1) | O(n) |
| Lookup | O(1) | O(n) |
| Delete | O(1) | O(n) |

The worst case is theoretical. With a good hash function and reasonable load factor, you will not hit O(n) in practice.

## JavaScript's Map

`Map` is JavaScript's built-in hash map. It stores key-value pairs with O(1) average operations and preserves insertion order.

### Core API

```js
const map = new Map();

// Set key-value pairs
map.set("name", "Alice");
map.set(42, "the answer");
map.set(true, "yes");

// Get values
map.get("name");    // "Alice"
map.get(42);        // "the answer"
map.get("missing"); // undefined

// Check existence
map.has("name");    // true
map.has("age");     // false

// Delete
map.delete(42);     // true (deleted)
map.delete(99);     // false (not found)

// Size (O(1), unlike Object)
map.size;           // 2

// Clear all entries
map.clear();
map.size;           // 0
```

### Any Value as Key

Unlike objects (string/symbol keys only), `Map` accepts any value as a key: objects, functions, numbers, `NaN`, `undefined`.

```js
const map = new Map();

// Object keys
const userA = { id: 1 };
const userB = { id: 2 };
map.set(userA, "Alice's data");
map.set(userB, "Bob's data");

map.get(userA); // "Alice's data"
map.get({ id: 1 }); // undefined -- different reference!

// NaN as key (works correctly, unlike Object)
map.set(NaN, "not a number");
map.get(NaN); // "not a number"

// Keys are compared by reference for objects, by value for primitives
map.set(1, "one");
map.set("1", "string one");
map.get(1);   // "one"
map.get("1"); // "string one" -- no type coercion
```

### Iteration

`Map` preserves insertion order and provides multiple iteration methods.

```js
const map = new Map([
  ["a", 1],
  ["b", 2],
  ["c", 3],
]);

// for...of (default: entries)
for (const [key, value] of map) {
  console.log(key, value);
}
// a 1, b 2, c 3

// Keys only
for (const key of map.keys()) {
  console.log(key);
}

// Values only
for (const value of map.values()) {
  console.log(value);
}

// Entries
for (const [key, value] of map.entries()) {
  console.log(key, value);
}

// forEach
map.forEach((value, key) => {
  console.log(key, value); // Note: value comes first in callback
});
```

### Initialize from Arrays or Other Iterables

```js
// From array of pairs
const map1 = new Map([
  ["x", 10],
  ["y", 20],
]);

// From Object
const obj = { a: 1, b: 2, c: 3 };
const map2 = new Map(Object.entries(obj));

// Back to Object
const backToObj = Object.fromEntries(map2);

// From another Map (copy)
const map3 = new Map(map2);
```

## JavaScript's Set

`Set` is a collection of **unique values** with O(1) average-time add, has, and delete. Think of it as a `Map` where you only care about keys, not values.

### Core API

```js
const set = new Set();

// Add values
set.add(1);
set.add(2);
set.add(3);
set.add(2); // Ignored -- already present

// Check existence
set.has(2); // true
set.has(5); // false

// Delete
set.delete(2); // true
set.has(2);    // false

// Size
set.size; // 2

// Clear
set.clear();
```

### Uniqueness

`Set` uses the SameValueZero algorithm for equality (same as `===` but treats `NaN` as equal to `NaN`).

```js
const set = new Set();

// Primitives: compared by value
set.add(1);
set.add("1");
set.size; // 2 -- number 1 and string "1" are different

// NaN: treated as equal
set.add(NaN);
set.add(NaN);
set.size; // 3 -- only one NaN

// Objects: compared by reference
set.add({ x: 1 });
set.add({ x: 1 });
set.size; // 5 -- two different objects, even though they look identical

// Same reference: treated as duplicate
const obj = { x: 1 };
set.add(obj);
set.add(obj);
set.size; // 6 -- obj added only once
```

### Set Operations (ES2025)

Recent JavaScript added built-in set operations. Check runtime support before using.

```js
const a = new Set([1, 2, 3, 4]);
const b = new Set([3, 4, 5, 6]);

// Union: elements in either set
a.union(b);                 // Set {1, 2, 3, 4, 5, 6}

// Intersection: elements in both sets
a.intersection(b);          // Set {3, 4}

// Difference: elements in a but not in b
a.difference(b);            // Set {1, 2}

// Symmetric difference: elements in one but not both
a.symmetricDifference(b);   // Set {1, 2, 5, 6}

// Subset check
new Set([3, 4]).isSubsetOf(a);   // true

// Superset check
a.isSupersetOf(new Set([1, 2])); // true

// Disjoint check: no common elements
a.isDisjointFrom(new Set([7, 8])); // true
```

If these are unavailable, implement them manually:

```js
function union(setA, setB) {
  return new Set([...setA, ...setB]);
}

function intersection(setA, setB) {
  return new Set([...setA].filter(x => setB.has(x)));
}

function difference(setA, setB) {
  return new Set([...setA].filter(x => !setB.has(x)));
}
```

### Common Pattern: Deduplicate an Array

```js
const arr = [1, 2, 3, 2, 1, 4, 3, 5];
const unique = [...new Set(arr)];
// [1, 2, 3, 4, 5]

// For objects, Set deduplicates by reference, not by content
// You need a Map or manual approach for object deduplication
```

## Map vs Object

| Feature | Map | Object |
|---------|-----|--------|
| Key types | Any value (objects, functions, primitives) | Strings and Symbols only |
| Key order | Insertion order guaranteed | Insertion order (mostly, with caveats for integer keys) |
| Size | `map.size` (O(1)) | `Object.keys(obj).length` (O(n)) |
| Iteration | Built-in iterables (for...of, forEach) | `Object.keys/values/entries` |
| Performance (frequent add/delete) | Optimized for this | Not optimized |
| Prototype chain | No inherited keys | Has prototype properties (toString, etc.) |
| Default keys | None | Has prototype properties |
| Serialization | Not JSON-serializable directly | `JSON.stringify` works directly |
| Destructuring | Not supported | Supported |

### When to Use Map

- Keys are not strings (objects, numbers, mixed types)
- You add and remove entries frequently
- You need `.size` without counting
- You iterate over entries often
- You want to avoid prototype pollution

### When to Use Object

- Keys are known, static strings
- You need JSON serialization
- You use destructuring
- It represents a "thing" with properties (a user, a config), not a collection

```js
// USE MAP: dynamic key-value collection
const cache = new Map();
cache.set(request.url, response);
cache.get(request.url);

// USE OBJECT: structured data
const user = { name: "Alice", age: 30, role: "admin" };
const { name, role } = user;
```

## Solving Problems with Hash Maps

The fundamental pattern: **trade O(n) space for O(1) lookups** to eliminate nested loops.

### Pattern 1: Two Sum

```js
// Find two numbers that sum to a target. Return their indices.

// BAD: O(n^2) brute force
function twoSumBrute(nums, target) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] + nums[j] === target) return [i, j];
    }
  }
  return null;
}

// GOOD: O(n) with hash map
function twoSum(nums, target) {
  const seen = new Map(); // value -> index

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (seen.has(complement)) {
      return [seen.get(complement), i];
    }
    seen.set(nums[i], i);
  }

  return null;
}

// twoSum([2, 7, 11, 15], 9) => [0, 1]
// Iteration 0: complement = 7, not in map, store 2->0
// Iteration 1: complement = 2, found at index 0, return [0, 1]
```

### Pattern 2: Frequency Counting

```js
// Count occurrences of each element
function frequency(arr) {
  const freq = new Map();
  for (const item of arr) {
    freq.set(item, (freq.get(item) || 0) + 1);
  }
  return freq;
}

// Find the most frequent element
function mostFrequent(arr) {
  const freq = frequency(arr);
  let maxCount = 0;
  let maxItem = null;

  for (const [item, count] of freq) {
    if (count > maxCount) {
      maxCount = count;
      maxItem = item;
    }
  }

  return maxItem;
}

// mostFrequent([1, 3, 2, 1, 4, 1, 3]) => 1
```

### Pattern 3: Grouping

```js
// Group array elements by a computed key
function groupBy(arr, keyFn) {
  const groups = new Map();

  for (const item of arr) {
    const key = keyFn(item);
    if (!groups.has(key)) groups.set(key, []);
    groups.get(key).push(item);
  }

  return groups;
}

const people = [
  { name: "Alice", dept: "Engineering" },
  { name: "Bob", dept: "Design" },
  { name: "Charlie", dept: "Engineering" },
  { name: "Diana", dept: "Design" },
];

const byDept = groupBy(people, p => p.dept);
// Map {
//   "Engineering" => [Alice, Charlie],
//   "Design" => [Bob, Diana]
// }

// Note: Object.groupBy() is available in modern runtimes (ES2024)
// but returns a null-prototype object, not a Map
```

### Pattern 4: Index Building

```js
// Build a lookup index for fast repeated queries
function buildIndex(records, keyField) {
  const index = new Map();
  for (const record of records) {
    index.set(record[keyField], record);
  }
  return index;
}

const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Charlie" },
];

const userById = buildIndex(users, "id");
userById.get(2); // { id: 2, name: "Bob" } -- O(1) lookup

// Without the index, finding user by id requires scanning: O(n) per lookup
// With the index, N lookups cost O(n) build + O(1) * N = O(n + N)
// Without: O(n * N)
```

## Solving Problems with Hash Sets

Sets answer one question efficiently: **is this element in the collection?**

### Pattern 1: Existence Checking

```js
// Check if an array has duplicates
function hasDuplicate(arr) {
  const seen = new Set();
  for (const item of arr) {
    if (seen.has(item)) return true;
    seen.add(item);
  }
  return false;
}

// Or simply:
function hasDuplicate2(arr) {
  return new Set(arr).size !== arr.length;
}
```

### Pattern 2: Array Intersection, Union, Difference

```js
function intersect(arr1, arr2) {
  const set = new Set(arr2);          // O(m) build
  return arr1.filter(x => set.has(x)); // O(n) scan with O(1) lookups
}
// Total: O(n + m)

// BAD: O(n * m) without Set
function intersectBrute(arr1, arr2) {
  return arr1.filter(x => arr2.includes(x)); // includes is O(m)
}
```

### Pattern 3: Missing/Extra Element

```js
// Find the first missing positive integer
function firstMissingPositive(nums) {
  const set = new Set(nums);

  for (let i = 1; ; i++) {
    if (!set.has(i)) return i;
  }
}

// firstMissingPositive([3, 4, -1, 1]) => 2
// firstMissingPositive([1, 2, 3]) => 4
```

### Pattern 4: Longest Consecutive Sequence

```js
// Find the length of the longest consecutive element sequence
// Input: [100, 4, 200, 1, 3, 2] => 4 (sequence: 1, 2, 3, 4)

function longestConsecutive(nums) {
  const set = new Set(nums);
  let maxLength = 0;

  for (const num of set) {
    // Only start counting from the beginning of a sequence
    if (!set.has(num - 1)) {
      let length = 1;
      let current = num;

      while (set.has(current + 1)) {
        current++;
        length++;
      }

      maxLength = Math.max(maxLength, length);
    }
  }

  return maxLength;
}

// The key insight: only start a chain from the smallest element
// (the one without a predecessor). This ensures O(n) total,
// not O(n^2), because each element is visited at most twice.
```

### Pattern 5: Visited Tracking

```js
// Track visited nodes during graph traversal
function bfs(graph, start) {
  const visited = new Set();
  const queue = [start];
  visited.add(start);

  while (queue.length > 0) {
    const node = queue.shift();
    console.log(node);

    for (const neighbor of graph[node]) {
      if (!visited.has(neighbor)) { // O(1) check
        visited.add(neighbor);       // O(1) insert
        queue.push(neighbor);
      }
    }
  }
}
```

## WeakMap and WeakSet

### WeakMap

`WeakMap` holds weak references to its keys. If no other reference to a key exists, it can be garbage collected along with its associated value. Keys must be objects or non-registered symbols.

```js
const cache = new WeakMap();

function processUser(user) {
  if (cache.has(user)) return cache.get(user);

  const result = expensiveComputation(user);
  cache.set(user, result);
  return result;
}

let user = { id: 1, name: "Alice" };
processUser(user); // Computed and cached

user = null; // No more references to the user object
// The cache entry is automatically eligible for garbage collection
// With a regular Map, the entry would persist forever (memory leak)
```

**Use cases:**

- **Caching with auto-cleanup:** Associate computed data with objects without preventing garbage collection.
- **Private data:** Store data that is only accessible if you have a reference to the key object.
- **DOM metadata:** Attach data to DOM elements that disappears when the element is removed.

```js
// Private data pattern
const privateData = new WeakMap();

class User {
  constructor(name, password) {
    this.name = name;
    privateData.set(this, { password }); // Not accessible from outside
  }

  checkPassword(attempt) {
    return privateData.get(this).password === attempt;
  }
}

const user = new User("Alice", "secret123");
user.name;                    // "Alice" -- public
user.password;                // undefined -- not a regular property
user.checkPassword("secret123"); // true
```

### WeakSet

`WeakSet` stores objects with weak references. If the object has no other references, it is garbage collected and removed from the set.

```js
// Track which DOM elements have been initialized
const initialized = new WeakSet();

function initElement(el) {
  if (initialized.has(el)) return; // Already done
  initialized.add(el);
  // ... set up event listeners, etc.
}

// When the DOM element is removed and no references remain,
// it is automatically removed from the WeakSet
```

### Limitations of Weak Collections

| Feature | Map/Set | WeakMap/WeakSet |
|---------|---------|-----------------|
| Key types | Any | Objects/symbols only |
| Iteration | Yes (for...of, forEach) | No |
| `.size` | Yes | No |
| `.clear()` | Yes | No (removed in spec) |
| Garbage collection | Prevents GC of keys | Allows GC of keys |

You cannot iterate over `WeakMap` or `WeakSet` because the entries are non-deterministic (they depend on garbage collection timing).

## Performance Patterns

### Replace O(n) Lookups with O(1)

The most impactful optimization pattern: anytime you use `Array.includes()`, `Array.indexOf()`, or `Array.find()` inside a loop, consider building a Set or Map first.

```js
// BAD: O(n * m) -- includes is O(m) per call
function findCommonUsers(teamA, teamB) {
  return teamA.filter(user => teamB.includes(user));
}

// GOOD: O(n + m) -- build Set once, O(1) lookups
function findCommonUsers(teamA, teamB) {
  const setB = new Set(teamB);
  return teamA.filter(user => setB.has(user));
}
```

### Avoid Repeated Object.keys() Counting

```js
// BAD: O(n) every time you check size
if (Object.keys(myObj).length > 0) { /* ... */ }

// GOOD: Use Map if you need frequent size checks
const myMap = new Map();
if (myMap.size > 0) { /* ... */ } // O(1)
```

### Choose the Right Key Strategy for Objects

When using objects as conceptual keys, you need a strategy since `Map` uses reference equality.

```js
// Problem: two identical objects are different keys
const map = new Map();
map.set({ x: 1, y: 2 }, "point A");
map.get({ x: 1, y: 2 }); // undefined -- different object

// Solution 1: Use a string key derived from the object
function pointKey(p) {
  return `${p.x},${p.y}`;
}

const pointMap = new Map();
pointMap.set(pointKey({ x: 1, y: 2 }), "point A");
pointMap.get(pointKey({ x: 1, y: 2 })); // "point A"

// Solution 2: Reuse the same object reference
const points = new Map();
const p1 = { x: 1, y: 2 };
points.set(p1, "point A");
points.get(p1); // "point A"
```

## Common Mistakes

- **Mistake 1: Using `includes()` or `indexOf()` inside a loop.** This is the most common source of accidental O(n^2). Build a Set or Map first, then query in O(1).

```js
// BAD: O(n * m)
const banned = ["spam", "scam", "fake"];
const filtered = messages.filter(m => !banned.includes(m.category));

// GOOD: O(n + m)
const bannedSet = new Set(["spam", "scam", "fake"]);
const filtered = messages.filter(m => !bannedSet.has(m.category));
```

- **Mistake 2: Expecting object equality by value in Map/Set.** `Map` and `Set` use reference equality for object keys. Two objects with identical properties are different keys.

```js
const set = new Set();
set.add({ id: 1 });
set.has({ id: 1 }); // false -- different object reference

// Use a string representation as the key if you need value equality
```

- **Mistake 3: Using plain objects as hash maps with untrusted keys.** Plain objects have a prototype chain. Keys like `"constructor"`, `"toString"`, or `"__proto__"` can clash with inherited properties. Use `Map` or `Object.create(null)` instead.

```js
// BAD: prototype pollution
const counts = {};
counts["constructor"]; // [Function: Object] -- not undefined!

// GOOD: Map has no prototype issues
const counts = new Map();
counts.get("constructor"); // undefined

// GOOD: null-prototype object
const counts = Object.create(null);
counts["constructor"]; // undefined
```

- **Mistake 4: Forgetting that Map's `forEach` callback has `(value, key)` order.** Unlike Array's `forEach(element, index)`, Map's callback puts value first: `forEach((value, key) => ...)`. This catches people who expect `(key, value)`.

- **Mistake 5: Using `Map` where a plain object suffices.** For simple string-keyed config or structured data, a plain object is simpler, supports destructuring, and is JSON-serializable. Do not reach for `Map` reflexively.

- **Mistake 6: Not considering memory when building large hash maps.** Each Map entry stores the key, value, and hash metadata. For millions of entries, memory usage can become significant. Profile before assuming a hash map is free.

---

## Best Practices

- **Build lookup structures before query loops.** If you will look up the same collection multiple times, build a Map or Set once. The O(n) build cost pays for itself after the second lookup.

- **Use Set for membership, Map for association.** If you only need "is X in the collection?", use Set. If you need "what is the value associated with X?", use Map. Do not use Map when you only care about keys.

- **Prefer Map over plain objects for dynamic collections.** When keys are added and removed at runtime, Map is faster, safer (no prototype issues), and provides `.size` in O(1).

- **Use WeakMap for metadata attached to objects.** If you are caching computed results for objects, or attaching auxiliary data to DOM elements, WeakMap prevents memory leaks by allowing garbage collection of unused keys.

- **Create composite keys with string serialization.** When you need to use multi-field values as keys (coordinates, compound IDs), serialize them to a string: `"${x},${y}"` or `JSON.stringify([a, b])`.

- **Profile before optimizing.** Replacing `includes()` with `Set.has()` in a loop over 10 items adds complexity for negligible gain. Focus on loops over large datasets.

## Real-World Use Cases

### Use Case 1: Request Deduplication

```js
// Prevent duplicate API requests for the same resource
class RequestDeduplicator {
  constructor() {
    this.inFlight = new Map(); // url -> Promise
  }

  async fetch(url) {
    // If a request for this URL is already in flight, return the same promise
    if (this.inFlight.has(url)) {
      return this.inFlight.get(url);
    }

    const promise = globalThis.fetch(url)
      .then(res => res.json())
      .finally(() => {
        this.inFlight.delete(url); // Clean up after completion
      });

    this.inFlight.set(url, promise);
    return promise;
  }
}

const dedup = new RequestDeduplicator();

// These three calls result in only ONE actual HTTP request
const p1 = dedup.fetch("/api/users/1");
const p2 = dedup.fetch("/api/users/1");
const p3 = dedup.fetch("/api/users/1");

const [r1, r2, r3] = await Promise.all([p1, p2, p3]);
// r1 === r2 === r3 -- same response from a single request
```

### Use Case 2: Multi-Field Indexing

```js
// Build indexes on a dataset for fast lookup by different fields
class IndexedCollection {
  constructor(records, indexFields) {
    this.records = records;
    this.indexes = new Map();

    for (const field of indexFields) {
      const index = new Map();
      for (const record of records) {
        const key = record[field];
        if (!index.has(key)) index.set(key, []);
        index.get(key).push(record);
      }
      this.indexes.set(field, index);
    }
  }

  findBy(field, value) {
    const index = this.indexes.get(field);
    if (!index) throw new Error(`No index on field: ${field}`);
    return index.get(value) || [];
  }
}

const products = [
  { id: 1, category: "electronics", brand: "Acme", price: 299 },
  { id: 2, category: "electronics", brand: "Beta", price: 199 },
  { id: 3, category: "clothing", brand: "Acme", price: 49 },
  { id: 4, category: "clothing", brand: "Beta", price: 79 },
];

const db = new IndexedCollection(products, ["category", "brand"]);
db.findBy("category", "electronics"); // [product 1, product 2] -- O(1)
db.findBy("brand", "Acme");           // [product 1, product 3] -- O(1)

// Without indexes: O(n) per query
// With indexes: O(1) per query after O(n * fields) build
```

### Use Case 3: Change Detection with Snapshots

```js
// Detect which fields changed between two versions of an object
function detectChanges(previous, current) {
  const prevMap = new Map(Object.entries(previous));
  const changes = { added: {}, removed: {}, modified: {} };

  // Check current against previous
  for (const [key, value] of Object.entries(current)) {
    if (!prevMap.has(key)) {
      changes.added[key] = value;
    } else if (prevMap.get(key) !== value) {
      changes.modified[key] = { from: prevMap.get(key), to: value };
    }
    prevMap.delete(key); // Mark as processed
  }

  // Remaining keys in prevMap were removed
  for (const [key, value] of prevMap) {
    changes.removed[key] = value;
  }

  return changes;
}

detectChanges(
  { name: "Alice", age: 30, role: "dev" },
  { name: "Alice", age: 31, title: "Sr. Dev" }
);
// => {
//   added: { title: "Sr. Dev" },
//   removed: { role: "dev" },
//   modified: { age: { from: 30, to: 31 } },
// }
```

---

> **Key Takeaway:** Hash maps (Map) and hash sets (Set) provide O(1) lookups that transform O(n^2) brute-force algorithms into O(n) solutions. The pattern is always the same: build a lookup structure once, then query it repeatedly. Use Map for key-value associations, Set for membership checks, and WeakMap/WeakSet when entries should be garbage-collected with their keys.
