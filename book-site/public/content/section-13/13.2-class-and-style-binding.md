# 13.2 Class and Style Binding

> Angular provides dedicated syntax for toggling CSS classes and setting inline styles dynamically. Knowing when to use `[class.name]`, `[class]`, `[style.prop]`, and `[style]` over the `NgClass`/`NgStyle` directives keeps templates clean and performant.

---

## Single Class Binding with `[class.name]`

The most common pattern. Toggle a single CSS class on or off based on a boolean expression.

```html
<div [class.active]="isActive()">Tab</div>
<div [class.disabled]="isDisabled()">Button</div>
<div [class.error]="hasError()">Field</div>
<tr [class.selected]="isSelected()">Row</tr>
<span [class.badge-premium]="user().isPremium">{{ user().plan }}</span>
```

When the expression is truthy, Angular adds the class. When falsy, Angular removes it. This happens through the DOM `classList` API.

### Multiple Single-Class Bindings

You can stack multiple `[class.name]` bindings on the same element:

```html
<div
  class="card"
  [class.card-highlighted]="isHighlighted()"
  [class.card-disabled]="isDisabled()"
  [class.card-selected]="isSelected()"
  [class.card-error]="hasError()"
>
  Card content
</div>
```

Each binding operates independently. They do not interfere with each other or with static `class` attributes already on the element. Angular merges them all together.

### Hyphenated Class Names

Class names with hyphens work directly in the binding:

```html
<div [class.is-loading]="loading()">Content</div>
<div [class.fade-in]="visible()">Animated</div>
<div [class.text-center]="centered()">Centered</div>
```

---

## Multi-Class Binding with `[class]`

Bind the entire `class` property to an expression that returns a string, array, or object.

### String Value

```html
<div [class]="classString()">Content</div>
```

```typescript
classString = computed(() => {
  const classes = ['base'];
  if (this.isActive()) classes.push('active');
  if (this.size()) classes.push(`size-${this.size()}`);
  return classes.join(' ');
});
```

The string replaces all dynamically bound classes. Static classes from the `class` attribute are preserved.

### Array Value

```html
<div [class]="classArray()">Content</div>
```

```typescript
classArray = computed(() => {
  const classes = ['card'];
  if (this.variant()) classes.push(`card-${this.variant()}`);
  if (this.elevated()) classes.push('elevated');
  return classes;
});
```

Each array element is a class name string. Angular adds all of them.

### Object Value

```html
<div [class]="classObject()">Content</div>
```

```typescript
classObject = computed(() => ({
  'card': true,
  'card-active': this.isActive(),
  'card-disabled': this.isDisabled(),
  'card-highlighted': this.isHighlighted(),
  [`card-${this.variant()}`]: !!this.variant()
}));
```

Keys are class names, values are boolean expressions. `true` adds the class, `false` removes it.

### Comparison Table

| Syntax | Input Type | Use Case |
|---|---|---|
| `[class]="'a b c'"` | String | Simple static-like class list from a variable |
| `[class]="['a', 'b']"` | Array | Dynamic list of class names |
| `[class]="{ a: true, b: false }"` | Object | Conditional toggle of multiple classes |
| `[class.name]="bool"` | Boolean | Toggle a single class |

---

## Static `class` and Dynamic `[class]` Together

Angular intelligently merges static `class` attributes with dynamic `[class]` or `[class.name]` bindings:

```html
<!-- Static "card" class is always present -->
<!-- Dynamic classes are added/removed on top of it -->
<div class="card base-layout" [class.active]="isActive()" [class.error]="hasError()">
  Content
</div>
```

The rendered element will always have `card` and `base-layout`. The `active` and `error` classes toggle independently.

When using `[class]` (without a dot), the static classes are still preserved:

```html
<div class="always-present" [class]="dynamicClasses()">
  <!-- "always-present" stays, dynamic classes are managed separately -->
</div>
```

## The `NgClass` Directive

`NgClass` is the older directive approach. It accepts the same three input types (string, array, object):

```html
<!-- Object syntax -->
<div [ngClass]="{ 'active': isActive(), 'disabled': isDisabled() }">Content</div>

<!-- Array syntax -->
<div [ngClass]="['base', isActive() ? 'active' : 'inactive']">Content</div>

<!-- String syntax -->
<div [ngClass]="'base ' + (isActive() ? 'active' : '')">Content</div>

<!-- Expression from component -->
<div [ngClass]="computedClasses()">Content</div>
```

### When to Use `NgClass` vs `[class]`

The Angular team recommends preferring the built-in `[class]` and `[class.name]` bindings over `NgClass`:

| Feature | `[class]` / `[class.name]` | `NgClass` |
|---|---|---|
| Performance | Directly manipulates DOM | Additional directive overhead |
| Readability | Cleaner, closer to HTML | Requires import |
| String input | Yes | Yes |
| Array input | Yes | Yes |
| Object input | Yes | Yes |
| Single class toggle | `[class.name]="bool"` | Overkill for one class |
| Space-separated keys | No | Yes |
| Mutating object reference | No (needs new reference) | Yes (watches internal changes) |

**Use `NgClass` only when you need:**
1. Space-separated class names in a single object key (e.g., `{ 'class-a class-b': condition }`).
2. Mutation-based updates where the object reference stays the same but internal properties change.

For everything else, prefer `[class]` and `[class.name]`.

```typescript
// If using NgClass, you must import it
import { NgClass } from '@angular/common';

@Component({
  standalone: true,
  imports: [NgClass],
  template: `<div [ngClass]="classes()">Content</div>`
})
```

## Single Style Binding with `[style.property]`

Set a single CSS property on an element:

```html
<div [style.color]="textColor()">Colored text</div>
<div [style.background-color]="bgColor()">Background</div>
<div [style.font-weight]="isBold() ? 'bold' : 'normal'">Text</div>
<div [style.opacity]="isVisible() ? 1 : 0">Fadeable</div>
<div [style.display]="isHidden() ? 'none' : 'block'">Toggleable</div>
```

The property name uses the CSS hyphenated form (`background-color`) or the camelCase DOM form (`backgroundColor`). Both work:

```html
<!-- Hyphenated (CSS convention) -->
<div [style.background-color]="color()">A</div>
<div [style.font-size]="size()">B</div>
<div [style.border-radius]="radius()">C</div>

<!-- CamelCase (DOM convention) -->
<div [style.backgroundColor]="color()">A</div>
<div [style.fontSize]="size()">B</div>
<div [style.borderRadius]="radius()">C</div>
```

### Unit Suffix

Angular supports a `.unit` suffix that automatically appends a CSS unit to a numeric value:

```html
<!-- Pixels -->
<div [style.width.px]="widthPx()">Sized</div>
<div [style.height.px]="heightPx()">Sized</div>
<div [style.margin-top.px]="marginTop()">Spaced</div>
<div [style.padding.px]="padding()">Padded</div>

<!-- Percentage -->
<div [style.width.%]="widthPercent()">Responsive</div>
<div [style.height.%]="heightPercent()">Responsive</div>

<!-- Rem -->
<div [style.font-size.rem]="fontSizeRem()">Scalable</div>
<div [style.gap.rem]="gapRem()">Spaced</div>

<!-- Em -->
<div [style.padding.em]="paddingEm()">Relative</div>

<!-- Viewport units -->
<div [style.height.vh]="heightVh()">Viewport</div>
<div [style.width.vw]="widthVw()">Viewport</div>
```

The expression must evaluate to a number when using unit suffixes. Angular concatenates the number with the unit: `[style.width.px]="200"` becomes `width: 200px`.

```typescript
@Component({
  selector: 'app-progress-bar',
  standalone: true,
  template: `
    <div class="track">
      <div
        class="fill"
        [style.width.%]="clampedProgress()"
        [style.background-color]="barColor()"
      ></div>
    </div>
    <span>{{ clampedProgress() }}%</span>
  `,
  styles: [`
    .track { height: 8px; background: #e0e0e0; border-radius: 4px; overflow: hidden; }
    .fill { height: 100%; transition: width 0.3s ease; border-radius: 4px; }
  `]
})
export class ProgressBarComponent {
  progress = input.required<number>();
  color = input<string>('#4caf50');

  clampedProgress = computed(() => Math.max(0, Math.min(100, this.progress())));
  barColor = computed(() => {
    const p = this.clampedProgress();
    if (p < 30) return '#f44336';
    if (p < 70) return '#ff9800';
    return this.color();
  });
}
```

### Null and Undefined Remove the Style

When the expression evaluates to `null` or `undefined`, Angular removes the inline style property:

```html
<!-- If customColor() returns null, no inline color is set -->
<!-- The element falls back to CSS stylesheet rules -->
<div [style.color]="customColor()">Text</div>
```

This is useful for conditional overrides where you want the default CSS to apply unless a specific value is provided.

## Multi-Style Binding with `[style]`

Bind the entire `style` property to a string or object:

### String Value

```html
<div [style]="styleString()">Content</div>
```

```typescript
styleString = computed(() =>
  `color: ${this.textColor()}; font-size: ${this.fontSize()}px; padding: ${this.padding()}px`
);
```

### Object Value

```html
<div [style]="styleObject()">Content</div>
```

```typescript
styleObject = computed(() => ({
  'color': this.textColor(),
  'font-size.px': this.fontSize(),
  'padding.px': this.padding(),
  'background-color': this.isHighlighted() ? '#fff3e0' : null,
  'border': this.hasBorder() ? '1px solid #ccc' : null
}));
```

Object keys support the `.unit` suffix notation. Null values remove the property.

## The `NgStyle` Directive

Like `NgClass`, `NgStyle` is the older directive approach:

```html
<div [ngStyle]="{
  'color': textColor(),
  'font-size.px': fontSize(),
  'background-color': isActive() ? '#e8f5e9' : null
}">
  Styled content
</div>
```

The Angular team states that `NgStyle` no longer provides significant value over the built-in style binding syntax and may eventually be removed. Prefer `[style]` and `[style.prop]` in new code.

```typescript
// If you must use NgStyle, import it
import { NgStyle } from '@angular/common';

@Component({
  standalone: true,
  imports: [NgStyle],
  template: `<div [ngStyle]="styles()">Content</div>`
})
```

## Binding Priority and Collision Resolution

When multiple bindings target the same class or style property, Angular follows a specificity order:

### Class Binding Priority

From highest to lowest priority:

1. `[class.specific-name]="expr"` (most specific, wins)
2. `[class]="expr"` (multi-class binding)
3. `class="static-classes"` (static attribute, always present)

```html
<!-- "active" from [class.active] wins over anything in [class] -->
<div
  class="base"
  [class]="dynamicClasses()"
  [class.active]="forceActive()"
>
  Content
</div>
```

If `dynamicClasses()` returns `{ active: false }` but `forceActive()` is `true`, the element will have the `active` class because `[class.active]` is more specific.

### Style Binding Priority

From highest to lowest priority:

1. `[style.specific-prop]="expr"` (most specific, wins)
2. `[style]="expr"` (multi-style binding)
3. `style="static-styles"` (static attribute)

```html
<div
  style="color: gray"
  [style]="baseStyles()"
  [style.color]="overrideColor()"
>
  <!-- overrideColor() wins for the color property -->
</div>
```

## Host Element Bindings

Use the `host` metadata property to bind classes and styles on the component's host element:

```typescript
@Component({
  selector: 'app-alert',
  standalone: true,
  template: `<ng-content />`,
  host: {
    'class': 'alert',
    '[class.alert-success]': 'type() === "success"',
    '[class.alert-error]': 'type() === "error"',
    '[class.alert-warning]': 'type() === "warning"',
    '[class.alert-dismissible]': 'dismissible()',
    '[style.--alert-accent]': 'accentColor()',
  },
  styles: [`
    :host {
      display: block;
      padding: 12px 16px;
      border-radius: 4px;
      border-left: 4px solid var(--alert-accent, #1976d2);
    }
    :host(.alert-success) { background: #e8f5e9; --alert-accent: #4caf50; }
    :host(.alert-error) { background: #ffebee; --alert-accent: #f44336; }
    :host(.alert-warning) { background: #fff3e0; --alert-accent: #ff9800; }
  `]
})
export class AlertComponent {
  type = input<'success' | 'error' | 'warning'>('success');
  dismissible = input(false);

  accentColor = computed(() => {
    const colors = { success: '#4caf50', error: '#f44336', warning: '#ff9800' };
    return colors[this.type()];
  });
}
```

The `host` property is the modern, preferred approach over `@HostBinding`:

```typescript
// Legacy approach (still works but not preferred)
@HostBinding('class.active') get isActive() { return this._active; }
@HostBinding('style.width.px') get hostWidth() { return this._width; }

// Modern approach (preferred)
host: {
  '[class.active]': 'isActive()',
  '[style.width.px]': 'hostWidth()',
}
```

## CSS Custom Properties (Variables) Binding

You can bind CSS custom properties, which is powerful for theming:

```html
<div
  [style.--card-bg]="cardBackground()"
  [style.--card-radius]="cardRadius() + 'px'"
  [style.--card-shadow]="hasShadow() ? '0 2px 8px rgba(0,0,0,0.1)' : 'none'"
  class="card"
>
  <h3>{{ title() }}</h3>
  <p>{{ content() }}</p>
</div>
```

```scss
.card {
  background: var(--card-bg, white);
  border-radius: var(--card-radius, 8px);
  box-shadow: var(--card-shadow, none);
  padding: 16px;
}
```

This pattern lets parent components style children through CSS custom properties without breaking encapsulation.

## Signal-Driven Dynamic Styling

Computed signals are the ideal way to derive class and style values:

```typescript
@Component({
  selector: 'app-status-indicator',
  standalone: true,
  template: `
    <div
      class="indicator"
      [class]="statusClasses()"
      [style.--pulse-color]="pulseColor()"
    >
      <span class="dot"></span>
      <span class="label">{{ statusLabel() }}</span>
    </div>
  `,
  styles: [`
    .indicator {
      display: inline-flex;
      align-items: center;
      gap: 8px;
      padding: 4px 12px;
      border-radius: 16px;
      font-size: 0.875rem;
    }
    .dot {
      width: 8px;
      height: 8px;
      border-radius: 50%;
      background: var(--pulse-color, #9e9e9e);
    }
    .status-online { background: #e8f5e9; color: #2e7d32; }
    .status-offline { background: #f5f5f5; color: #757575; }
    .status-busy { background: #fff3e0; color: #e65100; }
    .status-away { background: #fff8e1; color: #f9a825; }
    .status-online .dot { animation: pulse 2s infinite; }

    @keyframes pulse {
      0%, 100% { opacity: 1; }
      50% { opacity: 0.4; }
    }
  `]
})
export class StatusIndicatorComponent {
  status = input.required<'online' | 'offline' | 'busy' | 'away'>();

  statusClasses = computed(() => ({
    'status-online': this.status() === 'online',
    'status-offline': this.status() === 'offline',
    'status-busy': this.status() === 'busy',
    'status-away': this.status() === 'away',
  }));

  pulseColor = computed(() => {
    const colors: Record<string, string> = {
      online: '#4caf50',
      offline: '#9e9e9e',
      busy: '#ff5722',
      away: '#ffc107',
    };
    return colors[this.status()];
  });

  statusLabel = computed(() => {
    const labels: Record<string, string> = {
      online: 'Online',
      offline: 'Offline',
      busy: 'Do Not Disturb',
      away: 'Away',
    };
    return labels[this.status()];
  });
}
```

## Common Mistakes

- **Mistake 1: Using interpolation for class or style attributes.**
  Writing `class="card {{ dynamicClass() }}"` works but is fragile and hard to maintain. If `dynamicClass()` returns an empty string, you get a trailing space. Use `[class]` or `[class.name]` bindings instead, which Angular manages properly through the DOM API.

- **Mistake 2: Using `NgClass` or `NgStyle` for simple single-property toggles.**
  `[ngClass]="{ 'active': isActive() }"` is more verbose than `[class.active]="isActive()"` and requires importing `NgClass`. The built-in binding is simpler, faster, and needs no import.

- **Mistake 3: Forgetting to use `.unit` suffix with numeric values.**
  `[style.width]="200"` sets `width: 200`, which is invalid CSS (no unit). Use `[style.width.px]="200"` to get `width: 200px`. Alternatively, include the unit in the string: `[style.width]="'200px'"`.

- **Mistake 4: Expecting `[class]` to mutate an existing object.**
  If you bind `[class]="classObj"` and then mutate `classObj.active = true` without changing the object reference, Angular will not detect the change. Create a new object reference for the binding to update. This is not an issue with computed signals, which always return new values.

- **Mistake 5: Setting `[style.display]="false"` to hide elements.**
  `false` is not a valid CSS value for `display`. Use `[style.display]="isVisible() ? 'block' : 'none'"` or better yet, use `[hidden]="!isVisible()"` or `@if` to remove the element entirely.

- **Mistake 6: Mixing static and dynamic class attributes carelessly.**
  While Angular merges static `class` and dynamic `[class]` bindings, the behavior can be confusing when both set the same class. The dynamic binding has higher priority. Understand the priority order to avoid unexpected results.

---

## Best Practices

- **Use `[class.name]` for individual class toggles.** It is the simplest, most readable, and most performant syntax for conditional classes. Stack multiple bindings when needed.

- **Use computed signals for complex class/style logic.** Instead of inline expressions, move logic to a `computed` signal that returns an object. This keeps templates clean and makes the logic testable and memoized.

- **Prefer `[style.prop]` over `[style]` for individual properties.** Setting one property at a time is more readable and has higher specificity in case of conflicts.

- **Use CSS custom property binding for themeable components.** Bind `[style.--token-name]` to expose styling hooks that consumers can use without breaking encapsulation.

- **Prefer the `host` metadata property for host element styling.** Instead of `@HostBinding`, use the `host` object in the `@Component` decorator. It is more declarative and works naturally with signal expressions.

- **Avoid `NgClass` and `NgStyle` in new code.** The built-in `[class]` and `[style]` bindings are recommended by the Angular team, are faster, and require no directive imports.

## Real-World Use Cases

### Responsive Grid with Dynamic Column Sizing

A grid component where column widths and gap sizes are bound dynamically through style bindings.

```typescript
@Component({
  selector: 'app-responsive-grid',
  standalone: true,
  template: `
    <div
      class="grid"
      [style.grid-template-columns]="gridColumns()"
      [style.gap.px]="gap()"
      [style.padding.px]="padding()"
    >
      <ng-content />
    </div>
  `,
  styles: [`
    .grid { display: grid; }
  `]
})
export class ResponsiveGridComponent {
  columns = input(3);
  minColumnWidth = input(200);
  gap = input(16);
  padding = input(0);

  gridColumns = computed(() => {
    const min = this.minColumnWidth();
    return `repeat(auto-fill, minmax(${min}px, 1fr))`;
  });
}
```

```html
<app-responsive-grid [columns]="4" [gap]="24" [minColumnWidth]="250">
  @for (item of items(); track item.id) {
    <app-card [data]="item" />
  }
</app-responsive-grid>
```

### Data Table Row Styling with Multiple Conditions

A table where rows have different styles based on data state, selection, and hover state.

```typescript
@Component({
  selector: 'app-order-table',
  standalone: true,
  imports: [CurrencyPipe, DatePipe],
  template: `
    <table class="orders-table">
      <thead>
        <tr>
          <th>Order ID</th>
          <th>Customer</th>
          <th>Amount</th>
          <th>Date</th>
          <th>Status</th>
        </tr>
      </thead>
      <tbody>
        @for (order of orders(); track order.id) {
          <tr
            [class.row-selected]="selectedOrderId() === order.id"
            [class.row-overdue]="isOverdue(order)"
            [class.row-high-value]="order.amount > 1000"
            [class.row-cancelled]="order.status === 'cancelled'"
            [style.opacity]="order.status === 'cancelled' ? 0.6 : 1"
            (click)="selectedOrderId.set(order.id)"
          >
            <td>{{ order.id }}</td>
            <td>{{ order.customerName }}</td>
            <td>{{ order.amount | currency }}</td>
            <td>{{ order.date | date:'shortDate' }}</td>
            <td>
              <span
                class="badge"
                [class.badge-pending]="order.status === 'pending'"
                [class.badge-shipped]="order.status === 'shipped'"
                [class.badge-delivered]="order.status === 'delivered'"
                [class.badge-cancelled]="order.status === 'cancelled'"
              >
                {{ order.status }}
              </span>
            </td>
          </tr>
        }
      </tbody>
    </table>
  `,
  styles: [`
    .orders-table { width: 100%; border-collapse: collapse; }
    .orders-table th, .orders-table td { padding: 10px 12px; text-align: left; border-bottom: 1px solid #e0e0e0; }
    .row-selected { background: #e3f2fd; }
    .row-overdue { border-left: 3px solid #f44336; }
    .row-high-value { font-weight: 600; }
    .row-cancelled td { text-decoration: line-through; }
    .badge { padding: 2px 8px; border-radius: 12px; font-size: 0.75rem; text-transform: uppercase; }
    .badge-pending { background: #fff3e0; color: #e65100; }
    .badge-shipped { background: #e3f2fd; color: #1565c0; }
    .badge-delivered { background: #e8f5e9; color: #2e7d32; }

})
export class OrderTableComponent {
  orders = input.required<Order[]>();
  selectedOrderId = signal<string | null>(null);

  isOverdue(order: Order): boolean {
    return order.status === 'pending' && new Date(order.date) < new Date(Date.now() - 7 * 86400000);
  }
}
```

### Animated Dashboard Widget with Computed Styles

A widget component that uses style bindings for dynamic positioning, sizing, and animation based on signal-driven data.

```typescript
@Component({
  selector: 'app-gauge',
  standalone: true,
  template: `
    <div class="gauge" [style.--gauge-size.px]="size()">
      <svg [attr.viewBox]="'0 0 ' + size() + ' ' + size()">
        <!-- Background circle -->
        <circle
          [attr.cx]="center()"
          [attr.cy]="center()"
          [attr.r]="radius()"
          fill="none"
          stroke="#e0e0e0"
          [attr.stroke-width]="strokeWidth()"
        />
        <!-- Value arc -->
        <circle
          [attr.cx]="center()"
          [attr.cy]="center()"
          [attr.r]="radius()"
          fill="none"
          [attr.stroke]="gaugeColor()"
          [attr.stroke-width]="strokeWidth()"
          [attr.stroke-dasharray]="circumference()"
          [attr.stroke-dashoffset]="dashOffset()"
          [style.transition]="'stroke-dashoffset 0.6s ease'"
          stroke-linecap="round"
          transform="rotate(-90 {{ center() }} {{ center() }})"
        />
      </svg>
      <div class="gauge-label">
        <span
          class="gauge-value"
          [style.font-size.px]="size() * 0.2"
          [style.color]="gaugeColor()"
        >
          {{ displayValue() }}
        </span>
        <span class="gauge-unit" [style.font-size.px]="size() * 0.08">
          {{ unit() }}
        </span>
      </div>
    </div>
  `,
  styles: [`
    .gauge {
      position: relative;
      display: inline-flex;
      width: var(--gauge-size);
      height: var(--gauge-size);
    }
    .gauge-label {
      position: absolute;
      inset: 0;
      display: flex;
      flex-direction: column;
      align-items: center;
      justify-content: center;
    }
    .gauge-value { font-weight: 700; }
    .gauge-unit { color: #757575; }
  `]
})
export class GaugeComponent {
  value = input.required<number>();
  max = input(100);
  size = input(120);
  unit = input('%');
  strokeWidth = input(8);

  center = computed(() => this.size() / 2);
  radius = computed(() => (this.size() - this.strokeWidth()) / 2);
  circumference = computed(() => 2 * Math.PI * this.radius());

  clampedValue = computed(() => Math.max(0, Math.min(this.value(), this.max())));
  percentage = computed(() => this.clampedValue() / this.max());
  dashOffset = computed(() => this.circumference() * (1 - this.percentage()));
  displayValue = computed(() => Math.round(this.clampedValue()));

  gaugeColor = computed(() => {
    const p = this.percentage();
    if (p < 0.3) return '#f44336';
    if (p < 0.7) return '#ff9800';
    return '#4caf50';
  });
}
```

> **Key Takeaway:** Use `[class.name]` for single class toggles and `[style.prop]` with unit suffixes for individual style properties. Prefer the built-in binding syntax over `NgClass`/`NgStyle` for better performance and simplicity, and move complex conditional logic into computed signals to keep templates clean and memoized.
