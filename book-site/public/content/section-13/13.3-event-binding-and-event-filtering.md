# 13.3 Event Binding and Event Filtering

> Event binding connects DOM events to component methods, enabling user interaction. Angular's key filtering (pseudo-events) lets you respond to specific keystrokes without manual `if` checks, keeping handlers clean and change detection efficient.

---

## The `(event)` Syntax

Event binding uses parentheses around the event name, with a template statement on the right:

```html
<button (click)="onSave()">Save</button>
<input (input)="onInput($event)" />
<form (submit)="onSubmit($event)">...</form>
<div (mouseenter)="onHover()" (mouseleave)="onLeave()">Hoverable</div>
```

The template statement executes when the event fires. It runs in the context of the component instance.

```typescript
@Component({
  selector: 'app-counter',
  standalone: true,
  template: `
    <div class="counter">
      <button (click)="decrement()">-</button>
      <span>{{ count() }}</span>
      <button (click)="increment()">+</button>
    </div>
  `
})
export class CounterComponent {
  count = signal(0);

  increment() {
    this.count.update(c => c + 1);
  }

  decrement() {
    this.count.update(c => c - 1);
  }
}
```

## The `$event` Object

Angular passes the native DOM event object as `$event`. Its type depends on the event:

| Event | `$event` Type | Common Properties |
|---|---|---|
| `click` | `MouseEvent` | `target`, `clientX`, `clientY`, `button`, `ctrlKey`, `shiftKey` |
| `input` | `Event` (InputEvent) | `target.value`, `data`, `inputType` |
| `keydown`/`keyup` | `KeyboardEvent` | `key`, `code`, `ctrlKey`, `altKey`, `shiftKey`, `metaKey` |
| `submit` | `SubmitEvent` | `target`, `submitter` |
| `change` | `Event` | `target.value`, `target.checked` |
| `focus`/`blur` | `FocusEvent` | `target`, `relatedTarget` |
| `mouseenter`/`mouseleave` | `MouseEvent` | `target`, `clientX`, `clientY` |
| `dragstart`/`drop` | `DragEvent` | `dataTransfer`, `target` |
| `scroll` | `Event` | `target.scrollTop`, `target.scrollLeft` |

### Accessing `$event` in the Handler

```html
<input (input)="onInput($event)" />
```

```typescript
onInput(event: Event) {
  const input = event.target as HTMLInputElement;
  this.searchTerm.set(input.value);
}
```

You must cast `event.target` because TypeScript's `Event` type does not know the specific element type. The cast to `HTMLInputElement` gives you access to `.value`.

### Inline Expressions with `$event`

For simple operations, you can use `$event` directly in the template statement:

```html
<!-- Directly set a signal from the event -->
<input (input)="searchTerm.set($any($event.target).value)" />

<!-- Toggle on click -->
<div (click)="isOpen.set(!isOpen())">Toggle</div>

<!-- Pass specific event data to the handler -->
<div (click)="onClickAt($event.clientX, $event.clientY)">Click map</div>
```

The `$any()` cast disables type checking. Use it sparingly for inline event target access. For better type safety, use a handler method with proper typing.

### Type-Safe Pattern

```typescript
@Component({
  selector: 'app-search',
  standalone: true,
  template: `
    <input
      type="text"
      [value]="query()"
      (input)="onQueryChange($event)"
      placeholder="Search..."
    />
  `
})
export class SearchComponent {
  query = signal('');

  onQueryChange(event: Event) {
    const target = event.target as HTMLInputElement;
    this.query.set(target.value);
  }
}
```

## Template Statements vs Template Expressions

Template expressions (used in `[property]` and `{{ }}`) must be side-effect-free. Template statements (used in `(event)`) are meant to have side effects. They respond to user actions by updating state.

### What Is Allowed in Template Statements

Template statements support a subset of JavaScript, including:
- Method calls: `onSave()`
- Assignment: `count = 5` (in legacy code), `count.set(5)` (signals)
- Chaining with `;`: `doA(); doB()`
- Basic operators

```html
<!-- Multiple statements chained with semicolons -->
<button (click)="save(); close()">Save & Close</button>

<!-- Assignment (works in templates, but prefer signals) -->
<button (click)="submitted = true">Submit</button>

<!-- Signal-based state changes -->
<button (click)="isOpen.set(false); saved.set(true)">Confirm</button>
```

### What Is NOT Allowed

- `new` operator
- Increment/decrement (`++`, `--`)
- Compound assignment (`+=`, `-=`)
- Bitwise operators
- Pipe operator (`|`)

## Key Event Filtering (Pseudo-Events)

Angular's most powerful event binding feature. Instead of listening to all keyboard events and manually checking which key was pressed, you filter at the binding level.

### Basic Key Filtering

```html
<!-- Only fires when Enter key is pressed -->
<input (keyup.enter)="onSubmit()" />

<!-- Only fires when Escape key is pressed -->
<input (keydown.escape)="onCancel()" />

<!-- Only fires when Tab key is pressed -->
<input (keydown.tab)="onTabAway()" />

<!-- Only fires when Space key is pressed -->
<button (keydown.space)="onActivate()">Action</button>

<!-- Arrow keys -->
<div (keydown.arrowUp)="moveUp()" (keydown.arrowDown)="moveDown()">
  Navigable
</div>
<div (keydown.arrowLeft)="movePrev()" (keydown.arrowRight)="moveNext()">
  Slider
</div>
```

The key name after the dot corresponds to the `KeyboardEvent.key` value in lowercase.

### Why This Matters

Without key filtering, you write verbose handler code:

```typescript
// BAD PRACTICE: Manual key checking
onKeyDown(event: KeyboardEvent) {
  if (event.key === 'Enter') {
    this.onSubmit();
  } else if (event.key === 'Escape') {
    this.onCancel();
  } else if (event.key === 'ArrowUp') {
    this.moveUp();
  } else if (event.key === 'ArrowDown') {
    this.moveDown();
  }
}
```

```html
<!-- BAD PRACTICE: One generic handler for all keys -->
<input (keydown)="onKeyDown($event)" />
```

With key filtering:

```html
<!-- GOOD: Declarative, specific, no manual checking -->
<input
  (keydown.enter)="onSubmit()"
  (keydown.escape)="onCancel()"
/>
```

Each binding also means Angular only triggers change detection when that specific key is pressed, not on every keystroke.

### Supported Key Names

| Category | Key Names |
|---|---|
| Navigation | `arrowUp`, `arrowDown`, `arrowLeft`, `arrowRight` |
| Control | `enter`, `escape`, `tab`, `backspace`, `delete` |
| Function | `f1` through `f12` |
| Modifiers | `shift`, `control`, `alt`, `meta` |
| Letters | `a` through `z` |
| Numbers | `0` through `9` |
| Punctuation | `period`, `comma`, `slash`, `space`, `minus`, `equal` |
| Special | `home`, `end`, `pageUp`, `pageDown`, `insert` |

### Modifier Key Combinations

Combine modifier keys using dots:

```html
<!-- Ctrl+S -->
<div (keydown.control.s)="onSave($event)">
  Press Ctrl+S to save
</div>

<!-- Ctrl+Shift+Z (redo) -->
<div (keydown.control.shift.z)="onRedo()">Editor</div>

<!-- Alt+Enter -->
<textarea (keydown.alt.enter)="onSubmitComment()"></textarea>

<!-- Meta+K (Cmd+K on Mac) -->
<div (keydown.meta.k)="openCommandPalette()">App</div>

<!-- Shift+Tab (reverse tab) -->
<input (keydown.shift.tab)="onReverseTab($event)" />
```

The modifier names are: `control`, `shift`, `alt`, `meta`. The order does not matter: `control.shift.s` is the same as `shift.control.s`.

### Preventing Default Behavior

For keyboard shortcuts that conflict with browser defaults, call `preventDefault()`:

```html
<div (keydown.control.s)="onSave($event)">Editor content</div>
```

```typescript
onSave(event: KeyboardEvent) {
  event.preventDefault(); // Prevent browser's "Save Page" dialog
  this.saveDocument();
}
```

### The `code` Suffix

By default, Angular matches against `KeyboardEvent.key` (the character produced). To match against `KeyboardEvent.code` (the physical key position), use the `.code` suffix:

```html
<!-- Matches the physical key labeled "A" regardless of keyboard layout -->
<div (keydown.code.keyA)="onKeyA()">Physical key binding</div>

<!-- Matches the physical Digit1 key -->
<div (keydown.code.digit1)="onDigit1()">Physical key binding</div>
```

The `key` value changes based on keyboard layout (QWERTY vs AZERTY vs Dvorak). The `code` value represents the physical key position. Use `code` for game-like controls or shortcut systems that should work regardless of layout.

## Common DOM Events

### Mouse Events

```html
<div
  (click)="onClick($event)"
  (dblclick)="onDoubleClick()"
  (contextmenu)="onRightClick($event)"
  (mousedown)="onMouseDown($event)"
  (mouseup)="onMouseUp()"
  (mousemove)="onMouseMove($event)"
  (mouseenter)="onMouseEnter()"
  (mouseleave)="onMouseLeave()"
>
  Interactive area
</div>
```

`mouseenter`/`mouseleave` do not bubble (they fire only on the target element). `mouseover`/`mouseout` do bubble (they fire when moving between child elements too). Prefer `mouseenter`/`mouseleave` to avoid unintended triggers.

### Focus Events

```html
<input
  (focus)="onFocus()"
  (blur)="onBlur()"
  (focusin)="onFocusIn()"
  (focusout)="onFocusOut()"
/>
```

`focus`/`blur` do not bubble. `focusin`/`focusout` do bubble. Use `focusin`/`focusout` on a container to detect when any child receives or loses focus.

### Form Events

```html
<form (submit)="onSubmit($event)" (reset)="onReset()">
  <input (change)="onChange($event)" />
  <select (change)="onSelectChange($event)">
    <option value="a">A</option>
    <option value="b">B</option>
  </select>
  <input type="checkbox" (change)="onCheckboxChange($event)" />
</form>
```

```typescript
onSubmit(event: SubmitEvent) {
  event.preventDefault(); // Prevent page reload
  this.submitForm();
}

onSelectChange(event: Event) {
  const select = event.target as HTMLSelectElement;
  this.selectedValue.set(select.value);
}

onCheckboxChange(event: Event) {
  const checkbox = event.target as HTMLInputElement;
  this.isChecked.set(checkbox.checked);
}
```

### Drag and Drop Events

```html
<div
  class="drop-zone"
  (dragover)="onDragOver($event)"
  (drop)="onDrop($event)"
  (dragleave)="onDragLeave()"
>
  Drop files here
</div>

<div
  draggable="true"
  (dragstart)="onDragStart($event)"
  (dragend)="onDragEnd()"
>
  Draggable item
</div>
```

```typescript
onDragOver(event: DragEvent) {
  event.preventDefault(); // Required to allow drop
  this.isDragOver.set(true);
}

onDrop(event: DragEvent) {
  event.preventDefault();
  this.isDragOver.set(false);
  const files = event.dataTransfer?.files;
  if (files?.length) {
    this.handleFiles(Array.from(files));
  }
}
```

### Scroll Events

```html
<div class="scrollable" (scroll)="onScroll($event)">
  Long content here...
</div>
```

```typescript
onScroll(event: Event) {
  const el = event.target as HTMLElement;
  const scrollPercentage = el.scrollTop / (el.scrollHeight - el.clientHeight);
  if (scrollPercentage > 0.9) {
    this.loadMore();
  }
}
```

## Event Propagation: `stopPropagation` and `preventDefault`

### Event Bubbling

DOM events bubble up from the target element through ancestors. This can cause unexpected handler triggers:

```html
<!-- Both handlers fire when the button is clicked -->
<div (click)="onContainerClick()">
  <button (click)="onButtonClick()">Click me</button>
</div>
```

The button click bubbles up to the div, triggering `onContainerClick()` too.

### Stopping Propagation

```typescript
onButtonClick(event: MouseEvent) {
  event.stopPropagation(); // Prevents the event from reaching the container
  this.doButtonAction();
}
```

```html
<div (click)="onContainerClick()">
  <button (click)="onButtonClick($event)">Click me</button>
</div>
```

Or inline (less readable but concise):

```html
<button (click)="$event.stopPropagation(); onButtonClick()">Click me</button>
```

### Preventing Default

Some events have default browser behaviors:

| Event | Default Behavior |
|---|---|
| `submit` on `<form>` | Reloads the page |
| `click` on `<a>` | Navigates to the href |
| `keydown` Ctrl+S | Opens browser "Save Page" dialog |
| `contextmenu` | Opens the right-click context menu |
| `dragover` | Rejects the drop (prevents drop) |

```typescript
onSubmit(event: SubmitEvent) {
  event.preventDefault();
  // Handle form submission in Angular instead
}

onRightClick(event: MouseEvent) {
  event.preventDefault();
  // Show custom context menu
  this.showContextMenu(event.clientX, event.clientY);
}
```

## Host Event Listeners

Bind events on the component's host element using the `host` metadata:

```typescript
@Component({
  selector: 'app-dialog',
  standalone: true,
  template: `
    <div class="dialog-backdrop" (click)="onBackdropClick($event)">
      <div class="dialog-content" (click)="$event.stopPropagation()">
        <ng-content />
      </div>
    </div>
  `,
  host: {
    '(document:keydown.escape)': 'onEscape()',
    '(window:resize)': 'onWindowResize($event)',
  }
})
export class DialogComponent {
  closed = output<void>();

  onEscape() {
    this.closed.emit();
  }

  onWindowResize(event: Event) {
    // Respond to window resize
  }

  onBackdropClick(event: MouseEvent) {
    this.closed.emit();
  }
}
```

### Global Event Targets

The `host` binding supports special prefixes for global event targets:

| Prefix | Target |
|---|---|
| `document:` | The `document` object |
| `window:` | The `window` object |
| `body:` | The `document.body` element |

```typescript
host: {
  '(document:click)': 'onDocumentClick($event)',
  '(window:scroll)': 'onWindowScroll()',
  '(document:keydown.control.k)': 'onOpenSearch($event)',
}
```

**Warning:** Global event listeners run change detection on every event. Use them sparingly. For high-frequency events like `scroll` or `mousemove`, consider using `fromEvent()` from RxJS with throttling/debouncing outside Angular's zone, or use `afterRenderEffect` for frame-synced work.

## Custom Events from Child Components

When a child component uses `output()`, the parent binds to it the same way as DOM events:

```typescript
// Child component
@Component({
  selector: 'app-rating',
  standalone: true,
  template: `
    <div class="stars">
      @for (star of stars; track star) {
        <button
          [class.filled]="star <= currentRating()"
          (click)="rate(star)"
          (mouseenter)="hoverRating.set(star)"
          (mouseleave)="hoverRating.set(0)"
        >
          ★
        </button>
      }
    </div>
  `
})
export class RatingComponent {
  maxStars = input(5);
  value = input(0);

  rated = output<number>();

  stars = [1, 2, 3, 4, 5];
  hoverRating = signal(0);
  currentRating = computed(() => this.hoverRating() || this.value());

  rate(star: number) {
    this.rated.emit(star);
  }
}
```

```html
<!-- Parent template: binding to the custom "rated" event -->
<app-rating [value]="userRating()" (rated)="onRated($event)" />
```

```typescript
// Parent component
onRated(stars: number) {
  this.userRating.set(stars);
  this.saveRating(stars);
}
```

For custom events, `$event` carries the emitted value directly (not a DOM event). If the child emits a `number`, `$event` is a `number`.

## Performance Considerations

### Change Detection and Events

Every event binding triggers change detection in the component's subtree (with default change detection strategy). For high-frequency events, this can be expensive.

```html
<!-- BAD PRACTICE: mousemove triggers change detection on every pixel -->
<div (mousemove)="onMouseMove($event)">Canvas</div>
```

Solutions:

1. **Use `OnPush` change detection.** Only events on the component's own template trigger change detection.

2. **Debounce or throttle in the handler.** For `input` events on search fields, debounce before making API calls:

```typescript
@Component({
  selector: 'app-search',
  standalone: true,
  template: `
    <input (input)="onInput($event)" placeholder="Search..." />
  `
})
export class SearchComponent {
  private searchSubject = new Subject<string>();

  searchResults = signal<Result[]>([]);

  constructor() {
    this.searchSubject.pipe(
      debounceTime(300),
      distinctUntilChanged(),
      switchMap(term => this.searchService.search(term)),
      takeUntilDestroyed()
    ).subscribe(results => this.searchResults.set(results));
  }

  private searchService = inject(SearchService);

  onInput(event: Event) {
    const value = (event.target as HTMLInputElement).value;
    this.searchSubject.next(value);
  }
}
```

3. **Use key filtering for keyboard events.** `(keydown.enter)` triggers change detection only on Enter, not every keystroke.

### Event Delegation

Angular attaches event listeners to individual elements, not through event delegation on a parent. For lists with hundreds of items, this means hundreds of listeners:

```html
<!-- Each button gets its own click listener -->
@for (item of items(); track item.id) {
  <button (click)="onItemClick(item)">{{ item.name }}</button>
}
```

For most applications, this is fine. Modern browsers handle thousands of listeners efficiently. But for extreme cases (thousands of interactive rows), consider a single listener on the container:

```html
<div (click)="onContainerClick($event)">
  @for (item of items(); track item.id) {
    <button [attr.data-id]="item.id">{{ item.name }}</button>
  }
</div>
```

```typescript
onContainerClick(event: MouseEvent) {
  const target = event.target as HTMLElement;
  const button = target.closest('button[data-id]') as HTMLElement;
  if (button) {
    const id = button.dataset['id'];
    const item = this.items().find(i => i.id === id);
    if (item) this.handleItemAction(item);
  }
}
```

This pattern is an optimization for extreme cases. In most scenarios, individual bindings are clearer and preferred.

## Common Mistakes

- **Mistake 1: Forgetting `$event` when you need the event object.**
  Writing `(click)="onClick()"` without `$event` means the handler receives no argument. If you need the event (to call `preventDefault()`, `stopPropagation()`, or read event properties), you must pass it: `(click)="onClick($event)"`.

- **Mistake 2: Not calling `preventDefault()` on form submit.**
  Without `event.preventDefault()` in the submit handler, the browser performs a full page reload. Angular applications are SPAs; form submission should be handled in JavaScript.

- **Mistake 3: Using `(keydown)` when `(keydown.enter)` is sufficient.**
  Listening to all keydown events and filtering manually is verbose and triggers change detection on every keystroke. Use Angular's key filtering to only fire on the key you care about.

- **Mistake 4: Not casting `$event.target` for input events.**
  `$event.target` is typed as `EventTarget | null`, which has no `.value` property. You must cast to `HTMLInputElement` (or use `$any($event.target).value` for inline access). Forgetting the cast causes TypeScript errors with strict templates.

- **Mistake 5: Binding high-frequency events on `window` or `document` without throttling.**
  `(window:scroll)="onScroll()"` or `(window:mousemove)="onMove($event)"` triggers change detection on every event. For scroll-based animations or position tracking, use `fromEvent()` with `throttleTime()` outside Angular's zone, or use `afterRenderEffect`.

- **Mistake 6: Confusing event bubbling with event binding scope.**
  A `(click)` on a parent div fires for clicks on any child element that does not stop propagation. This can cause unintended handler calls. Use `$event.stopPropagation()` on child handlers to prevent bubbling when needed.

## Best Practices

- **Use key filtering pseudo-events for all keyboard interactions.** `(keydown.enter)`, `(keydown.escape)`, `(keydown.control.s)` are more readable and more performant than a generic `(keydown)` handler with manual key checks.

- **Always call `preventDefault()` on form submit events.** This prevents the browser from reloading the page. It should be the first line of every submit handler.

- **Use typed handler methods instead of inline `$event` casts.** While `$any($event.target).value` works inline, a handler method with `event: Event` and `event.target as HTMLInputElement` is type-safe and easier to test.

- **Debounce high-frequency input events.** For search fields, auto-save, or real-time validation, use `debounceTime()` from RxJS or a manual debounce to avoid excessive API calls or computations.

- **Prefer `(mouseenter)`/`(mouseleave)` over `(mouseover)`/`(mouseout)`.** The enter/leave variants do not bubble and fire only when entering/leaving the target element, not when moving between child elements.

- **Use the `host` property for component-level event listeners.** Global events like document keydown or window resize belong in the `host` metadata, not in template elements.

## Real-World Use Cases

### Keyboard-Navigable Dropdown

A dropdown menu that responds to arrow keys, Enter, and Escape for full keyboard accessibility.

```typescript
@Component({
  selector: 'app-dropdown',
  standalone: true,
  template: `
    <div
      class="dropdown"
      [class.open]="isOpen()"
    >
      <button
        class="dropdown-trigger"
        (click)="toggle()"
        (keydown.arrowDown)="openAndFocusFirst($event)"
        (keydown.escape)="close()"
        [attr.aria-expanded]="isOpen()"
        [attr.aria-haspopup]="'listbox'"
      >
        {{ selectedLabel() ?? placeholder() }}
      </button>

      @if (isOpen()) {
        <ul
          class="dropdown-menu"
          role="listbox"
          (keydown.arrowDown)="focusNext($event)"
          (keydown.arrowUp)="focusPrev($event)"
          (keydown.enter)="selectFocused($event)"
          (keydown.escape)="close()"
        >
          @for (option of options(); track option.value; let i = $index) {
            <li
              role="option"
              [class.focused]="focusedIndex() === i"
              [class.selected]="selectedValue() === option.value"
              [attr.aria-selected]="selectedValue() === option.value"
              (click)="select(option)"
              (mouseenter)="focusedIndex.set(i)"
            >
              {{ option.label }}
            </li>
          }
        </ul>
      }
    </div>
  `,
  host: {
    '(document:click)': 'onOutsideClick($event)',
  }
})
export class DropdownComponent {
  options = input.required<Option[]>();
  placeholder = input('Select...');
  selectedValue = signal<string | null>(null);
  isOpen = signal(false);
  focusedIndex = signal(0);

  selected = output<Option>();

  selectedLabel = computed(() => {
    const val = this.selectedValue();
    return this.options().find(o => o.value === val)?.label ?? null;
  });

  toggle() {
    this.isOpen.update(v => !v);
  }

  close() {
    this.isOpen.set(false);
  }

  openAndFocusFirst(event: KeyboardEvent) {
    event.preventDefault();
    this.isOpen.set(true);
    this.focusedIndex.set(0);
  }

  focusNext(event: KeyboardEvent) {
    event.preventDefault();
    this.focusedIndex.update(i => Math.min(i + 1, this.options().length - 1));
  }

  focusPrev(event: KeyboardEvent) {
    event.preventDefault();
    this.focusedIndex.update(i => Math.max(i - 1, 0));
  }

  selectFocused(event: KeyboardEvent) {
    event.preventDefault();
    const option = this.options()[this.focusedIndex()];
    if (option) this.select(option);
  }

  select(option: Option) {
    this.selectedValue.set(option.value);
    this.selected.emit(option);
    this.close();
  }

  onOutsideClick(event: MouseEvent) {
    // Close if click is outside this component
    const el = inject(ElementRef).nativeElement as HTMLElement;
    if (!el.contains(event.target as Node)) {
      this.close();
    }
  }

  private inject = inject;
  constructor() {
    // Fix: inject in constructor for the onOutsideClick closure
  }
}

interface Option {
  label: string;
  value: string;
}
```

    .upload-zone.dragging { border-color: #1976d2; background: #e3f2fd; }
    .upload-zone.has-file { border-style: solid; border-color: #4caf50; }
    .hint { color: #999; font-size: 0.875rem; }
  `]
})
export class FileUploadComponent {
  accept = input<string>('');
  file = signal<File | null>(null);

  fileSelected = output<File>();

  isDragging = signal(false);

  onDragOver(event: DragEvent) {
    event.preventDefault();
    event.stopPropagation();
    this.isDragging.set(true);
  }

  onDragLeave() {
    this.isDragging.set(false);
  }

  onDrop(event: DragEvent) {
    event.preventDefault();
    event.stopPropagation();
    this.isDragging.set(false);

    const files = event.dataTransfer?.files;
    if (files?.length) {
      this.setFile(files[0]);
    }
  }

  onFileSelect(event: Event) {
    const input = event.target as HTMLInputElement;
    if (input.files?.length) {
      this.setFile(input.files[0]);
    }
  }

  removeFile(event: MouseEvent) {
    event.stopPropagation(); // Prevent triggering the zone click
    this.file.set(null);
  }

  private setFile(file: File) {
    this.file.set(file);
    this.fileSelected.emit(file);
  }

  formatSize(bytes: number): string {
    if (bytes < 1024) return `${bytes} B`;
    if (bytes < 1048576) return `${(bytes / 1024).toFixed(1)} KB`;
    return `${(bytes / 1048576).toFixed(1)} MB`;
  }
}

### Keyboard Shortcut Manager

A component that registers global keyboard shortcuts declaratively using host event bindings.

```typescript
@Component({
  selector: 'app-editor',
  standalone: true,
  template: `
    <div class="editor-container">
      <div class="toolbar">
        <button (click)="onSave()">Save (Ctrl+S)</button>
        <button (click)="onUndo()">Undo (Ctrl+Z)</button>
        <button (click)="onRedo()">Redo (Ctrl+Shift+Z)</button>
        <button (click)="onFind()">Find (Ctrl+F)</button>
      </div>
      <textarea
        class="editor-area"
        [value]="content()"
        (input)="onContentChange($event)"
        (keydown.tab)="onTab($event)"
      ></textarea>
    </div>
  `,
  host: {
    '(document:keydown.control.s)': 'onSaveShortcut($event)',
    '(document:keydown.control.z)': 'onUndoShortcut($event)',
    '(document:keydown.control.shift.z)': 'onRedoShortcut($event)',
    '(document:keydown.control.f)': 'onFindShortcut($event)',
  }
})
export class EditorComponent {
  content = signal('');
  saved = output<string>();

  onSaveShortcut(event: KeyboardEvent) {
    event.preventDefault();
    this.onSave();
  }

  onUndoShortcut(event: KeyboardEvent) {
    event.preventDefault();
    this.onUndo();
  }

  onRedoShortcut(event: KeyboardEvent) {
    event.preventDefault();
    this.onRedo();
  }

  onFindShortcut(event: KeyboardEvent) {
    event.preventDefault();
    this.onFind();
  }

  onSave() {
    this.saved.emit(this.content());
  }

  onUndo() { /* undo logic */ }
  onRedo() { /* redo logic */ }
  onFind() { /* open find dialog */ }

  onTab(event: KeyboardEvent) {
    event.preventDefault();
    // Insert tab character instead of moving focus
    const textarea = event.target as HTMLTextAreaElement;
    const start = textarea.selectionStart;
    const end = textarea.selectionEnd;
    const value = this.content();
    this.content.set(value.substring(0, start) + '  ' + value.substring(end));
    // Restore cursor position after Angular updates the textarea
    setTimeout(() => {
      textarea.selectionStart = textarea.selectionEnd = start + 2;
    });
  }

  onContentChange(event: Event) {
    this.content.set((event.target as HTMLTextAreaElement).value);
  }
}
```

---

> **Key Takeaway:** Use Angular's key filtering pseudo-events (`(keydown.enter)`, `(keydown.control.s)`) instead of generic keyboard handlers to write cleaner, more declarative, and more performant event bindings. Always call `preventDefault()` on form submissions and browser shortcut conflicts, and cast `$event.target` for type safety.
