# 15.4 Query Parameters

> Query parameters are the key-value pairs after the `?` in a URL. They are global, optional, and persist across route changes when configured correctly. They are the primary mechanism for shareable, bookmarkable UI state like filters, pagination, sorting, and search terms.

---

## Query Parameters vs Path Parameters

| Aspect | Path Parameters | Query Parameters |
|--------|----------------|-----------------|
| Syntax | `/products/:id` | `/products?page=2&sort=name` |
| Required | Yes (part of route matching) | No (optional by nature) |
| Scope | Bound to a specific route segment | Global across the entire URL |
| Purpose | Identify a resource | Modify how a resource is displayed |
| Example | `/users/42` | `/users?role=admin&active=true` |

Use path parameters to identify **what** to show. Use query parameters to describe **how** to show it.

## Setting Query Parameters

### With `RouterLink`

```typescript
import { Component } from '@angular/core';
import { RouterLink } from '@angular/router';

@Component({
  imports: [RouterLink],
  template: `
    <!-- Static query params -->
    <a
      routerLink="/products"
      [queryParams]="{ page: 1, sort: 'price', order: 'asc' }"
    >
      Products sorted by price
    </a>

    <!-- Dynamic query params -->
    <a
      routerLink="/search"
      [queryParams]="{ q: searchTerm(), category: selectedCategory() }"
    >
      Search
    </a>

    <!-- Query params with fragment -->
    <a
      routerLink="/docs"
      [queryParams]="{ version: '21' }"
      fragment="installation"
    >
      Installation Docs v21
    </a>
    <!-- Produces: /docs?version=21#installation -->
  `,
})
export class NavComponent {
  searchTerm = signal('angular');
  selectedCategory = signal('frameworks');
}
```

### With `Router.navigate()`

```typescript
import { Component, inject } from '@angular/core';
import { Router } from '@angular/router';

@Component({ ... })
export class SearchComponent {
  private router = inject(Router);

  search(term: string) {
    this.router.navigate(['/search'], {
      queryParams: { q: term, page: 1 },
    });
    // Produces: /search?q=angular&page=1
  }

  goToPage(page: number) {
    this.router.navigate(['/search'], {
      queryParams: { page },
      queryParamsHandling: 'merge', // Keep existing params, update page
    });
    // If current URL is /search?q=angular&page=1
    // Produces: /search?q=angular&page=2
  }
}
```

### With `Router.navigateByUrl()`

```typescript
// Full URL string with query params
this.router.navigateByUrl('/products?category=electronics&sort=price');
```

## `queryParamsHandling`

This option controls how new query parameters interact with existing ones during navigation.

| Value | Behavior |
|-------|----------|
| `''` (default) | **Replace** all existing query params with the new ones |
| `'merge'` | **Merge** new params into existing ones (existing keys are overwritten, others preserved) |
| `'preserve'` | **Ignore** the new params entirely, keep existing ones |

### Replace (Default)

```typescript
// Current URL: /products?category=electronics&sort=price&page=3

this.router.navigate(['/products'], {
  queryParams: { page: 1 },
});
// Result: /products?page=1
// category and sort are GONE
```

### Merge

```typescript
// Current URL: /products?category=electronics&sort=price&page=3

this.router.navigate(['/products'], {
  queryParams: { page: 1 },
  queryParamsHandling: 'merge',
});
// Result: /products?category=electronics&sort=price&page=1
// Existing params preserved, page updated
```

### Preserve

```typescript
// Current URL: /products?category=electronics&sort=price&page=3

this.router.navigate(['/products'], {
  queryParams: { page: 1 },  // This is ignored
  queryParamsHandling: 'preserve',
});
// Result: /products?category=electronics&sort=price&page=3
// Nothing changed
```

### On `RouterLink`

```typescript
<a
  routerLink="/products"
  [queryParams]="{ page: nextPage() }"
  queryParamsHandling="merge"
>
  Next Page
</a>
```

## Reading Query Parameters

### Signal Inputs with `withComponentInputBinding()` (Modern)

With `withComponentInputBinding()` enabled, query parameters automatically bind to inputs that share the same name:

```typescript
// URL: /products?page=2&sort=price&order=desc

@Component({
  template: `
    <p>Page: {{ page() }}, Sort: {{ sort() }}, Order: {{ order() }}</p>
  `,
})
export class ProductListComponent {
  page = input<string>();    // '2'
  sort = input<string>();    // 'price'
  order = input<string>();   // 'desc'
}
```

### With Transform for Type Conversion

```typescript
import { numberAttribute } from '@angular/core';

@Component({ ... })
export class ProductListComponent {
  // Convert page from string '2' to number 2
  page = input(1, { transform: numberAttribute });

  // Default value if query param is missing
  sort = input('name');

  order = input<'asc' | 'desc'>('asc');
}
```

### Priority Warning

Remember the binding priority: **static route data > path params > query params**. If a route has `data: { sort: 'relevance' }` and the URL has `?sort=price`, the input receives `'relevance'` from static data, not `'price'` from the query param.

## `ActivatedRoute` Approach (Legacy)

### `queryParamMap` Observable

```typescript
import { Component, inject, signal } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({ ... })
export class ProductListComponent {
  private route = inject(ActivatedRoute);

  page = signal(1);
  sort = signal('name');

  constructor() {
    this.route.queryParamMap.subscribe(params => {
      this.page.set(Number(params.get('page')) || 1);
      this.sort.set(params.get('sort') ?? 'name');
    });
  }
}
```

### `queryParamMap` Methods

| Method | Returns | Purpose |
|--------|---------|---------|
| `get(key)` | `string \| null` | Get a single value |
| `getAll(key)` | `string[]` | Get all values for a key (multi-value) |
| `has(key)` | `boolean` | Check if a key exists |
| `keys` | `string[]` | List all parameter names |

### Multi-Value Query Parameters

A single key can have multiple values:

```
/products?color=red&color=blue&color=green
```

```typescript
this.route.queryParamMap.subscribe(params => {
  const colors = params.getAll('color');
  // ['red', 'blue', 'green']
});
```

### Snapshot vs Observable

```typescript
// Snapshot: one-time read
const page = this.route.snapshot.queryParamMap.get('page');

// Observable: reacts to changes
this.route.queryParamMap.subscribe(params => {
  const page = params.get('page');
});
```

**Use the observable** when query params change while the component stays on screen (pagination, filtering). **Use the snapshot** only if you are certain the params do not change during the component's lifetime (rare for query params).

## Converting Query Parameters to Signals

### With `toSignal`

```typescript
import { Component, inject, computed } from '@angular/core';
import { ActivatedRoute } from '@angular/router';
import { toSignal } from '@angular/core/rxjs-interop';

@Component({ ... })
export class ProductListComponent {
  private route = inject(ActivatedRoute);

  private queryParams = toSignal(this.route.queryParamMap);

  readonly page = computed(() => {
    const p = this.queryParams()?.get('page');
    return p ? Number(p) : 1;
  });

  readonly sort = computed(() =>
    this.queryParams()?.get('sort') ?? 'name'
  );

  readonly filters = computed(() => {
    const params = this.queryParams();
    return {
      category: params?.get('category') ?? 'all',
      minPrice: Number(params?.get('minPrice')) || 0,
      maxPrice: Number(params?.get('maxPrice')) || Infinity,
    };
  });
}
```

### Reusable `injectQueryParam` Utility

```typescript
import { assertInInjectionContext, inject, computed } from '@angular/core';
import { ActivatedRoute } from '@angular/router';
import { toSignal } from '@angular/core/rxjs-interop';

export function injectQueryParam(name: string): Signal<string | null> {
  assertInInjectionContext(injectQueryParam);
  const route = inject(ActivatedRoute);
  const queryParams = toSignal(route.queryParamMap);
  return computed(() => queryParams()?.get(name) ?? null);
}

export function injectQueryParams(): Signal<Record<string, string>> {
  assertInInjectionContext(injectQueryParams);
  const route = inject(ActivatedRoute);
  const params = toSignal(route.queryParams, { initialValue: {} as Record<string, string> });
  return params;
}

// Usage
@Component({ ... })
export class SearchComponent {
  searchTerm = injectQueryParam('q');
  page = injectQueryParam('page');
}
```

## Updating Query Parameters Programmatically

### Replacing a Single Parameter

```typescript
@Component({ ... })
export class ProductListComponent {
  private router = inject(Router);

  onSortChange(sort: string) {
    this.router.navigate([], {
      queryParams: { sort },
      queryParamsHandling: 'merge',
    });
    // navigate([]) with no path stays on the current route
  }
}
```

### Removing a Query Parameter

Set the parameter to `null` or `undefined`:

```typescript
clearFilter() {
  this.router.navigate([], {
    queryParams: { category: null },
    queryParamsHandling: 'merge',
  });
  // If URL was /products?category=electronics&page=1
  // Becomes: /products?page=1
}
```

### Replacing All Parameters

```typescript
resetFilters() {
  this.router.navigate([], {
    queryParams: { page: 1 },
    // Default handling: replaces all existing params
  });
}
```

### Batch Update

```typescript
applyFilters(filters: { category: string; minPrice: number; maxPrice: number }) {
  this.router.navigate([], {
    queryParams: {
      ...filters,
      page: 1, // Reset to page 1 when filters change
    },
    queryParamsHandling: 'merge',
  });
}
```

## Query Parameters Across Route Changes

By default, query parameters are lost when navigating to a different route:

```typescript
// Current URL: /products?category=electronics
this.router.navigate(['/checkout']);
// Result: /checkout (no query params)
```

To preserve params across route changes:

```typescript
// Preserve all params when navigating to a different route
this.router.navigate(['/checkout'], {
  queryParamsHandling: 'preserve',
});
// Result: /checkout?category=electronics
```

Or in a template:

```typescript
<a routerLink="/checkout" queryParamsHandling="preserve">
  Checkout
</a>
```

## Default Values Pattern

A clean pattern for query parameters with defaults:

```typescript
@Component({ ... })
export class ProductListComponent {
  private router = inject(Router);

  // Define defaults
  private readonly defaults = {
    page: 1,
    pageSize: 25,
    sort: 'name',
    order: 'asc' as 'asc' | 'desc',
    category: 'all',
  };

  // Signal inputs with defaults via transform
  page = input(this.defaults.page, {
    transform: (v: string | number) => Number(v) || this.defaults.page,
  });

  pageSize = input(this.defaults.pageSize, {
    transform: (v: string | number) => {
      const n = Number(v);
      return [10, 25, 50, 100].includes(n) ? n : this.defaults.pageSize;
    },
  });

  sort = input(this.defaults.sort);
  order = input<'asc' | 'desc'>(this.defaults.order);
  category = input(this.defaults.category);

  // Navigate with new params
  updateParams(changes: Partial<typeof this.defaults>) {
    this.router.navigate([], {
      queryParams: changes,
      queryParamsHandling: 'merge',
    });
  }
}
```

## Fragment (Hash)

The URL fragment (`#section`) is separate from query parameters but often used alongside them.

### Setting a Fragment

```typescript
// In template
<a routerLink="/docs" fragment="getting-started">Getting Started</a>
// Produces: /docs#getting-started

// Programmatically
this.router.navigate(['/docs'], { fragment: 'getting-started' });
```

### Reading the Fragment

```typescript
// With ActivatedRoute
this.route.fragment.subscribe(fragment => {
  console.log(fragment); // 'getting-started'
});

// With snapshot
const fragment = this.route.snapshot.fragment;

// With withComponentInputBinding() -- not directly supported for fragment
// Use ActivatedRoute for fragment access
```

### Scrolling to Fragment

Enable anchor scrolling in the router config:

```typescript
provideRouter(
  routes,
  withInMemoryScrolling({ anchorScrolling: 'enabled' }),
);
```

Now navigating to `/docs#installation` automatically scrolls to the element with `id="installation"`.

## Encoding and Special Characters

Query parameter values are automatically URL-encoded by Angular's router:

```typescript
this.router.navigate(['/search'], {
  queryParams: { q: 'angular & react' },
});
// URL: /search?q=angular%20%26%20react
```

When reading, Angular automatically decodes the values:

```typescript
// URL: /search?q=angular%20%26%20react
const q = this.route.snapshot.queryParamMap.get('q');
// q === 'angular & react'
```

### Arrays in Query Parameters

Angular does not have a built-in array format for query params. Common approaches:

```typescript
// Approach 1: Repeated keys
// URL: /products?color=red&color=blue
this.router.navigate(['/products'], {
  queryParams: { color: ['red', 'blue'] },
});
// Read with getAll:
const colors = params.getAll('color'); // ['red', 'blue']

// Approach 2: Comma-separated
this.router.navigate(['/products'], {
  queryParams: { colors: 'red,blue,green' },
});
// Read and split:
const colors = params.get('colors')?.split(',') ?? [];
```

## Two-Way Sync: URL as Single Source of Truth

A powerful pattern where the URL drives the component state, and user interactions update the URL:

```typescript
@Component({
  template: `
    <div class="filters">
      <input
        [value]="searchTerm()"
        (input)="onSearch($any($event.target).value)"
        placeholder="Search..."
      />

      <select [value]="sort()" (change)="onSort($any($event.target).value)">
        <option value="name">Name</option>
        <option value="price">Price</option>
        <option value="rating">Rating</option>
      </select>

      <select [value]="category()" (change)="onCategory($any($event.target).value)">
        <option value="all">All Categories</option>
        <option value="electronics">Electronics</option>
        <option value="clothing">Clothing</option>
      </select>
    </div>

    <app-product-grid [products]="filteredProducts()" />

    <app-pagination
      [currentPage]="page()"
      [totalPages]="totalPages()"
      (pageChange)="onPageChange($event)"
    />
  `,
})
export class ProductListComponent {
  private router = inject(Router);
  private productService = inject(ProductService);

  // All state comes from query parameters
  searchTerm = input('', { alias: 'q' });
  sort = input('name');
  category = input('all');
  page = input(1, { transform: numberAttribute });

  // Data loading driven by URL state
  private productResource = resource({
    request: () => ({
      q: this.searchTerm(),
      sort: this.sort(),
      category: this.category(),
      page: this.page(),
    }),
    loader: ({ request }) => this.productService.search(request),
  });

  filteredProducts = computed(() => this.productResource.value()?.items ?? []);
  totalPages = computed(() => this.productResource.value()?.totalPages ?? 1);

  // User actions update the URL, which updates inputs, which triggers resource reload
  onSearch(q: string) {
    this.navigate({ q, page: 1 });
  }

  onSort(sort: string) {
    this.navigate({ sort, page: 1 });
  }

  onCategory(category: string) {
    this.navigate({ category, page: 1 });
  }

  onPageChange(page: number) {
    this.navigate({ page });
  }

  private navigate(params: Record<string, string | number>) {
    this.router.navigate([], {
      queryParams: params,
      queryParamsHandling: 'merge',
    });
  }
}
```

Benefits:
- The URL is bookmarkable and shareable.
- Browser back/forward buttons undo/redo filter changes.
- Deep linking works out of the box.
- No separate state management needed for filter state.

## Common Mistakes

### Mistake 1: Losing Query Parameters on Navigation

```typescript
// BAD: All existing query params are lost
this.router.navigate(['/products'], {
  queryParams: { page: 2 },
});
// If URL was /products?category=electronics&sort=price&page=1
// Result: /products?page=2 (category and sort are gone!)

// GOOD: Merge to keep existing params
this.router.navigate(['/products'], {
  queryParams: { page: 2 },
  queryParamsHandling: 'merge',
});
// Result: /products?category=electronics&sort=price&page=2
```

### Mistake 2: Using Snapshot for Changing Query Parameters

```typescript
// BAD: Snapshot does not update when user changes filters
ngOnInit() {
  const page = this.route.snapshot.queryParamMap.get('page');
  this.loadProducts(Number(page) || 1); // Never updates
}

// GOOD: Use signal inputs or observable
page = input(1, { transform: numberAttribute });  // Reactive
```

### Mistake 3: Not Resetting Page on Filter Change

```typescript
// BAD: User is on page 5, changes category, stays on page 5 (likely empty)
onCategoryChange(category: string) {
  this.router.navigate([], {
    queryParams: { category },
    queryParamsHandling: 'merge',
  });
}

// GOOD: Reset page to 1 when filters change
onCategoryChange(category: string) {
  this.router.navigate([], {
    queryParams: { category, page: 1 },
    queryParamsHandling: 'merge',
  });
}
```

### Mistake 4: Forgetting That Query Params Are Always Strings

```typescript
// BAD: Comparing with strict equality to a number
if (this.route.snapshot.queryParamMap.get('page') === 1) { ... }
// Always false! get() returns '1' (string), not 1 (number)

// GOOD: Parse first
const page = Number(this.route.snapshot.queryParamMap.get('page'));
if (page === 1) { ... }

// BEST: Use transform on signal input
page = input(1, { transform: numberAttribute });
```

### Mistake 5: Confusing `queryParams` with `queryParamMap`

```typescript
// queryParams is a plain object: { key: value }
this.route.queryParams.subscribe(params => {
  const page = params['page']; // Direct property access
});

// queryParamMap is a ParamMap with methods
this.route.queryParamMap.subscribe(params => {
  const page = params.get('page');  // Method access
  const colors = params.getAll('color'); // Multi-value support
  const hasFilter = params.has('filter');
});
```

Prefer `queryParamMap` for its `getAll()` support and null-safe `get()`.

### Mistake 6: Data Priority Conflict

```typescript
// Route with static data that shadows a query param
{
  path: 'products',
  component: ProductListComponent,
  data: { page: 1 },  // Static data named 'page'
}
// URL: /products?page=5

// With withComponentInputBinding():
page = input<number>(); // Gets 1 (from data), NOT 5 (from query param)

// Fix: Rename the static data key to avoid shadowing
data: { defaultPage: 1 }
```

## Best Practices

### 1. Use the URL as the Source of Truth for Filter State

Do not store filter/sort/page state in component properties or services when query parameters can drive it. This gives you bookmarkability, shareability, and browser history integration for free.

### 2. Always Use `queryParamsHandling: 'merge'` for Incremental Updates

When changing a single filter, merge to keep the rest intact. Only use the default replace behavior when resetting all params.

### 3. Use `input()` with Transform for Type Safety

Always transform query parameters from strings to their actual types at the input level. This centralizes parsing and prevents scattered `Number()` and `parseInt()` calls.

### 4. Provide Default Values

Query parameters are optional by nature. Every input that reads a query param should have a sensible default:

```typescript
page = input(1, { transform: numberAttribute });
sort = input('name');
order = input<'asc' | 'desc'>('asc');
```

### 5. Reset Pagination on Filter Changes

When the user changes a filter or sort order, always reset the page to 1 to avoid showing an empty page.

### 6. Keep Query Parameter Names Short but Descriptive

URLs are visible to users. Use `q` for search, `page` for pagination, `sort` and `order` for sorting. Avoid verbose names like `searchQuery` or `currentPageNumber`.

## Real-World Use Cases

### Search Page with Debounced Input

```typescript
@Component({
  template: `
    <input
      #searchInput
      [value]="q()"
      (input)="onSearchInput(searchInput.value)"
      placeholder="Search products..."
    />
    @if (isLoading()) {
      <app-spinner />
    } @else {
      @for (product of results(); track product.id) {
        <app-product-card [product]="product" />
      }
    }
  `,
})
export class SearchPageComponent {
  private router = inject(Router);
  private productService = inject(ProductService);

  q = input('');
  page = input(1, { transform: numberAttribute });

  private searchResource = resource({
    request: () => ({ q: this.q(), page: this.page() }),
    loader: ({ request }) => this.productService.search(request.q, request.page),
  });

  results = computed(() => this.searchResource.value()?.items ?? []);
  isLoading = this.searchResource.isLoading;

  private debounceTimer: ReturnType<typeof setTimeout> | null = null;

  onSearchInput(value: string) {
    if (this.debounceTimer) clearTimeout(this.debounceTimer);
    this.debounceTimer = setTimeout(() => {
      this.router.navigate([], {
        queryParams: { q: value || null, page: 1 },
        queryParamsHandling: 'merge',
      });
    }, 300);
  }
}
```

### Data Table with Persistent URL State

```typescript
@Component({
  template: `
    <table>
      <thead>
        <tr>
          @for (col of columns; track col.key) {
            <th (click)="toggleSort(col.key)"
                [class.sorted]="sort() === col.key">
              {{ col.label }}
              @if (sort() === col.key) {
                <span>{{ order() === 'asc' ? '↑' : '↓' }}</span>
              }
            </th>
          }
        </tr>
      </thead>
      <tbody>
        @for (row of data(); track row.id) {
          <tr>
            @for (col of columns; track col.key) {
              <td>{{ row[col.key] }}</td>
            }
          </tr>
        }
      </tbody>
    </table>
    <div class="pagination">
      <button [disabled]="page() <= 1" (click)="goToPage(page() - 1)">Prev</button>
      <span>Page {{ page() }} of {{ totalPages() }}</span>
      <button [disabled]="page() >= totalPages()" (click)="goToPage(page() + 1)">Next</button>
    </div>
  `,
})
export class DataTableComponent {
  private router = inject(Router);

  columns = [
    { key: 'name', label: 'Name' },
    { key: 'email', label: 'Email' },
    { key: 'role', label: 'Role' },
    { key: 'createdAt', label: 'Created' },
  ];

  // All table state lives in the URL
  page = input(1, { transform: numberAttribute });
  pageSize = input(25, { transform: numberAttribute });
  sort = input('name');
  order = input<'asc' | 'desc'>('asc');

  private dataResource = resource({
    request: () => ({
      page: this.page(),
      pageSize: this.pageSize(),
      sort: this.sort(),
      order: this.order(),
    }),
    loader: ({ request }) => inject(UserService).getUsers(request),
  });

  data = computed(() => this.dataResource.value()?.items ?? []);
  totalPages = computed(() =>
    Math.ceil((this.dataResource.value()?.total ?? 0) / this.pageSize())
  );

  toggleSort(field: string) {
    const newOrder = this.sort() === field && this.order() === 'asc' ? 'desc' : 'asc';
    this.router.navigate([], {
      queryParams: { sort: field, order: newOrder, page: 1 },
      queryParamsHandling: 'merge',
    });
  }

  goToPage(page: number) {
    this.router.navigate([], {
      queryParams: { page },
      queryParamsHandling: 'merge',
    });
  }
}
```

Sharing a link like `/users?sort=createdAt&order=desc&page=3` reproduces the exact table state for any user.

---

> **Key Takeaway:** Treat query parameters as your URL-driven state management for filters, pagination, and sorting. Use `withComponentInputBinding()` with signal inputs and transforms to read them reactively, `queryParamsHandling: 'merge'` to update them incrementally, and let the URL be the single source of truth for shareable, bookmarkable UI state.
