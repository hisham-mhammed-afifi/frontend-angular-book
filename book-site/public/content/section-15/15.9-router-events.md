# 15.9 Router Events

> The Angular Router emits a rich stream of events at every stage of navigation, giving you hooks for loading indicators, analytics, error handling, scroll restoration, and debugging.

---

## The Router Event Stream

Every navigation the Angular Router performs emits a sequence of typed events on the `Router.events` observable. There are 17 distinct event types, and every navigation ends with exactly one terminal event: `NavigationEnd`, `NavigationCancel`, `NavigationError`, or `NavigationSkipped`.

```typescript
import { Router, Event } from '@angular/router';

// The Event type is a union of all 17 router event types
type Event =
  | NavigationStart
  | RouteConfigLoadStart
  | RouteConfigLoadEnd
  | RoutesRecognized
  | GuardsCheckStart
  | ChildActivationStart
  | ActivationStart
  | GuardsCheckEnd
  | ResolveStart
  | ResolveEnd
  | ChildActivationEnd
  | ActivationEnd
  | NavigationEnd
  | NavigationCancel
  | NavigationError
  | NavigationSkipped
  | Scroll;
```

---

## Event Sequence for a Successful Navigation

When a user navigates to a lazily-loaded route with guards and resolvers, the full event sequence is:

```
1.  NavigationStart           -- URL received, navigation begins
2.  RouteConfigLoadStart      -- lazy chunk download begins (only for lazy routes)
3.  RouteConfigLoadEnd        -- lazy chunk downloaded
4.  RoutesRecognized          -- URL matched to route config
5.  GuardsCheckStart          -- canDeactivate / canActivateChild / canActivate begin
6.  ChildActivationStart      -- child route activation begins
7.  ActivationStart           -- target route activation begins
8.  GuardsCheckEnd            -- all guards passed
9.  ResolveStart              -- resolver execution begins
10. ResolveEnd                -- all resolvers completed
11. ChildActivationEnd        -- child route activation completes
12. ActivationEnd             -- target route activation completes
13. NavigationEnd             -- navigation succeeded, URL updated
14. Scroll                    -- scroll position restored or reset
```

If the route is eagerly loaded, steps 2-3 (`RouteConfigLoadStart`/`RouteConfigLoadEnd`) are skipped.

If there are no guards, steps 5 and 8 still fire but execute immediately.

If there are no resolvers, steps 9 and 10 still fire but execute immediately.

## Subscribing to Router Events

### Basic Subscription

```typescript
import { Component, inject } from '@angular/core';
import { Router, NavigationStart, NavigationEnd } from '@angular/router';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Component({
  selector: 'app-root',
  template: `
    @if (loading) {
      <app-loading-bar />
    }
    <router-outlet />
  `
})
export class AppComponent {
  private router = inject(Router);
  loading = false;

  constructor() {
    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe(event => {
        if (event instanceof NavigationStart) {
          this.loading = true;
        }
        if (event instanceof NavigationEnd ||
            event instanceof NavigationCancel ||
            event instanceof NavigationError) {
          this.loading = false;
        }
      });
  }
}
```

### Using RxJS filter()

```typescript
import { filter } from 'rxjs';
import { NavigationEnd } from '@angular/router';

// Filter for a single event type
this.router.events
  .pipe(
    filter((event): event is NavigationEnd => event instanceof NavigationEnd),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    console.log('Navigated to:', event.urlAfterRedirects);
  });
```

### Signal-Based Approach with toSignal()

```typescript
import { Component, inject, computed } from '@angular/core';
import { toSignal } from '@angular/core/rxjs-interop';
import { Router, NavigationStart, NavigationEnd, NavigationCancel, NavigationError } from '@angular/router';
import { filter, map } from 'rxjs';

@Component({
  selector: 'app-root',
  template: `
    @if (isNavigating()) {
      <app-loading-bar />
    }
    <router-outlet />
  `
})
export class AppComponent {
  private router = inject(Router);

  private navigationEvent = toSignal(
    this.router.events.pipe(
      filter(e =>
        e instanceof NavigationStart ||
        e instanceof NavigationEnd ||
        e instanceof NavigationCancel ||
        e instanceof NavigationError
      )
    )
  );

  isNavigating = computed(() => {
    const event = this.navigationEvent();
    return event instanceof NavigationStart;
  });
}
```

## All 17 Event Types in Detail

### NavigationStart

Fired when the router begins processing a new navigation request.

```typescript
interface NavigationStart extends RouterEvent {
  id: number;                    // unique navigation ID
  url: string;                   // target URL
  navigationTrigger: 'imperative' | 'popstate' | 'hashchange';
  restoredState: { navigationId: number } | null;
}
```

The `navigationTrigger` tells you how the navigation was initiated:
- `'imperative'`: programmatic navigation via `router.navigate()` or `router.navigateByUrl()`
- `'popstate'`: browser back/forward button
- `'hashchange'`: hash-based URL change

The `restoredState` is non-null when the user presses back/forward. It contains the `navigationId` of the original navigation being restored.

```typescript
this.router.events
  .pipe(
    filter((e): e is NavigationStart => e instanceof NavigationStart),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    if (event.navigationTrigger === 'popstate') {
      console.log('User pressed back/forward, restoring nav:', event.restoredState?.navigationId);
    }
  });
```

### RouteConfigLoadStart / RouteConfigLoadEnd

Fired before and after a lazy-loaded route configuration is fetched. Only fires for routes using `loadComponent` or `loadChildren`.

```typescript
interface RouteConfigLoadStart {
  route: Route;   // the route config being loaded
}

interface RouteConfigLoadEnd {
  route: Route;   // the route config that was loaded
}
```

Use these to show chunk-loading progress:

```typescript
import { RouteConfigLoadStart, RouteConfigLoadEnd } from '@angular/router';

// Track which lazy chunks are loading
this.router.events
  .pipe(takeUntilDestroyed())
  .subscribe(event => {
    if (event instanceof RouteConfigLoadStart) {
      console.log('Loading lazy route:', event.route.path);
    }
    if (event instanceof RouteConfigLoadEnd) {
      console.log('Lazy route loaded:', event.route.path);
    }
  });
```

**Known issue:** If a navigation to a lazy-loaded route is canceled (by a guard), and then the user navigates to the same lazy route again, the `RouteConfigLoadStart` event may not fire the second time even though `RouteConfigLoadEnd` does fire. This is a documented Angular issue.

### RoutesRecognized

Fired when the router has matched the URL to one or more route configurations.

```typescript
interface RoutesRecognized extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;
  state: RouterStateSnapshot;    // the recognized router state
}
```

The `state` property gives you access to the full `RouterStateSnapshot`, including all activated route snapshots:

```typescript
import { RoutesRecognized } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is RoutesRecognized => e instanceof RoutesRecognized),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    // Access route data before guards run
    const root = event.state.root;
    const data = root.firstChild?.data;
    console.log('Recognized route data:', data);
  });
```

### GuardsCheckStart / GuardsCheckEnd

Bracket the guard evaluation phase.

```typescript
interface GuardsCheckStart extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;
  state: RouterStateSnapshot;
}

interface GuardsCheckEnd extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;
  state: RouterStateSnapshot;
  shouldActivate: boolean;       // true if all guards passed
}
```

`GuardsCheckEnd.shouldActivate` tells you whether guards allowed the navigation to proceed:

```typescript
import { GuardsCheckEnd } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is GuardsCheckEnd => e instanceof GuardsCheckEnd),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    if (!event.shouldActivate) {
      console.log('Navigation blocked by guard');
    }
  });
```

### ActivationStart / ActivationEnd

Fire when a specific route begins and finishes activating. The `snapshot` property gives the `ActivatedRouteSnapshot` of the route being activated.

```typescript
interface ActivationStart {
  snapshot: ActivatedRouteSnapshot;
}

interface ActivationEnd {
  snapshot: ActivatedRouteSnapshot;
}
```

```typescript
import { ActivationEnd } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is ActivationEnd => e instanceof ActivationEnd),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    const title = event.snapshot.data['title'];
    if (title) {
      document.title = title;
    }
  });
```

### ChildActivationStart / ChildActivationEnd

Fire when child routes begin and finish activating. Structurally identical to `ActivationStart`/`ActivationEnd` but scoped to child route trees.

```typescript
interface ChildActivationStart {
  snapshot: ActivatedRouteSnapshot;
}

interface ChildActivationEnd {
  snapshot: ActivatedRouteSnapshot;
}
```

### ResolveStart / ResolveEnd

Bracket the resolver execution phase.

```typescript
interface ResolveStart extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;
  state: RouterStateSnapshot;
}

interface ResolveEnd extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;
  state: RouterStateSnapshot;
}
```

```typescript
import { ResolveStart, ResolveEnd } from '@angular/router';

// Measure resolver execution time
this.router.events
  .pipe(takeUntilDestroyed())
  .subscribe(event => {
    if (event instanceof ResolveStart) {
      console.time('resolvers');
    }
    if (event instanceof ResolveEnd) {
      console.timeEnd('resolvers');
    }
  });
```

### NavigationEnd

Fired when navigation completes successfully. The URL has been updated and the new component is rendered.

```typescript
interface NavigationEnd extends RouterEvent {
  id: number;
  url: string;
  urlAfterRedirects: string;     // final URL after all redirects
}
```

`urlAfterRedirects` is the URL that was actually navigated to. This differs from `url` when redirect routes are involved.

### NavigationCancel

Fired when navigation is canceled. This happens when a guard returns `false`, a `UrlTree`, or a `RedirectCommand`.

```typescript
interface NavigationCancel extends RouterEvent {
  id: number;
  url: string;
  reason: string;                // human-readable reason (may change between versions)
  code?: NavigationCancellationCode;  // stable enum value
}

enum NavigationCancellationCode {
  Redirect = 0,                  // guard/resolver issued a redirect
  SupersededByNewNavigation = 1, // a newer navigation replaced this one
  NoDataFromResolver = 2,        // resolver returned EMPTY/completed without value
  GuardRejected = 3              // guard returned false
}
```

**Important:** Always use the `code` property, not `reason`, for programmatic logic. The `reason` string is unstable and intended for debugging only.

```typescript
import { NavigationCancel, NavigationCancellationCode } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is NavigationCancel => e instanceof NavigationCancel),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    switch (event.code) {
      case NavigationCancellationCode.GuardRejected:
        console.log('Guard blocked navigation to', event.url);
        break;
      case NavigationCancellationCode.Redirect:
        console.log('Redirected away from', event.url);
        break;
      case NavigationCancellationCode.SupersededByNewNavigation:
        console.log('Navigation superseded');
        break;
      case NavigationCancellationCode.NoDataFromResolver:
        console.log('Resolver returned no data');
        break;
    }
  });
```

### NavigationError

Fired when navigation fails due to an unhandled error.

```typescript
interface NavigationError extends RouterEvent {
  id: number;
  url: string;
  error: any;                    // the thrown error
}
```

```typescript
import { NavigationError } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is NavigationError => e instanceof NavigationError),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    console.error('Navigation failed:', event.error);
    // Example: chunk load failure for lazy routes
    if (event.error?.name === 'ChunkLoadError') {
      window.location.reload();
    }
  });
```

**Note:** With `withNavigationErrorHandler()`, you can handle these errors declaratively in `provideRouter()` instead of subscribing manually:

```typescript
provideRouter(
  routes,
  withNavigationErrorHandler((error) => {
    const router = inject(Router);
    if (error.error?.name === 'ChunkLoadError') {
      router.navigateByUrl('/error', { skipLocationChange: true });
    }
    return new RedirectCommand(router.parseUrl('/error'));
  })
)
```

### NavigationSkipped

Fired when the router skips a navigation entirely. This was added in Angular 16+.

```typescript
interface NavigationSkipped extends RouterEvent {
  id: number;
  url: string;
  reason: string;                // human-readable explanation
  code?: NavigationSkippedCode;  // stable enum value
}

enum NavigationSkippedCode {
  IgnoredSameUrlNavigation = 0,       // navigated to the same URL
  IgnoredByUrlHandlingStrategy = 1    // URL excluded by UrlHandlingStrategy
}
```

By default, navigating to the exact same URL is skipped. You can control this behavior with `onSameUrlNavigation` in `withRouterConfig()`:

```typescript
provideRouter(
  routes,
  withRouterConfig({
    onSameUrlNavigation: 'reload'    // 'ignore' (default) | 'reload'
  })
)
```

When set to `'ignore'`, a `NavigationSkipped` event fires with code `IgnoredSameUrlNavigation`. When set to `'reload'`, the full navigation cycle runs again.

### Scroll

Fired after `NavigationEnd` when scroll position is restored or reset. Requires `withInMemoryScrolling()`.

```typescript
interface Scroll {
  routerEvent: NavigationEnd;    // the navigation that triggered scrolling
  position: [number, number] | null;  // restored scroll position [x, y]
  anchor: string | null;              // fragment anchor (#section)
}
```

```typescript
import { Scroll } from '@angular/router';

this.router.events
  .pipe(
    filter((e): e is Scroll => e instanceof Scroll),
    takeUntilDestroyed()
  )
  .subscribe(event => {
    if (event.position) {
      // Back/forward navigation: restoring previous position
      console.log('Restoring scroll to', event.position);
    } else if (event.anchor) {
      // Fragment navigation: scrolling to anchor
      console.log('Scrolling to anchor', event.anchor);
    } else {
      // Forward navigation: scroll to top
      console.log('Scrolling to top');
    }
  });
```

## The RouterEvent Base Class

Most events extend `RouterEvent`, which provides:

```typescript
abstract class RouterEvent {
  id: number;    // unique navigation ID (shared across all events in one navigation)
  url: string;   // the target URL
}
```

The `id` is critical for correlating events. All events belonging to the same navigation share the same `id`. This lets you track the entire lifecycle of a single navigation:

```typescript
// Group events by navigation ID
const navigationTimings = new Map<number, { start: number; end?: number }>();

this.router.events
  .pipe(takeUntilDestroyed())
  .subscribe(event => {
    if (event instanceof NavigationStart) {
      navigationTimings.set(event.id, { start: performance.now() });
    }
    if (event instanceof NavigationEnd ||
        event instanceof NavigationCancel ||
        event instanceof NavigationError) {
      const timing = navigationTimings.get(event.id);
      if (timing) {
        timing.end = performance.now();
        console.log(`Navigation ${event.id} took ${timing.end - timing.start}ms`);
        navigationTimings.delete(event.id);
      }
    }
  });
```

**Note:** `ActivationStart`, `ActivationEnd`, `ChildActivationStart`, `ChildActivationEnd`, `RouteConfigLoadStart`, `RouteConfigLoadEnd`, and `Scroll` do NOT extend `RouterEvent`. They lack the `id` and `url` properties. To correlate them, you must track the current navigation `id` from the most recent `NavigationStart`.

## The currentNavigation() Signal

Angular provides `Router.currentNavigation` as a signal that returns the active `Navigation` object or `null`:

```typescript
@Component({
  selector: 'app-root',
  template: `
    @if (router.currentNavigation()) {
      <app-spinner />
    }
    <router-outlet />
  `
})
export class AppComponent {
  router = inject(Router);
}
```

This is simpler than subscribing to events for basic loading indicators. It is `null` when no navigation is in progress and an object during active navigations.

## Debugging with withDebugTracing()

During development, you can log every router event to the console:

```typescript
import { provideRouter, withDebugTracing } from '@angular/router';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(
      routes,
      withDebugTracing()   // logs ALL 17 event types to console
    )
  ]
};
```

This outputs events like:

```
NavigationStart(id: 1, url: '/products')
RouteConfigLoadStart(path: 'products')
RouteConfigLoadEnd(path: 'products')
RoutesRecognized(id: 1, url: '/products', urlAfterRedirects: '/products')
GuardsCheckStart(id: 1, url: '/products', ...)
GuardsCheckEnd(id: 1, url: '/products', shouldActivate: true)
ResolveStart(id: 1, url: '/products', ...)
ResolveEnd(id: 1, url: '/products', ...)
NavigationEnd(id: 1, url: '/products', urlAfterRedirects: '/products')
Scroll(anchor: null, position: null)
```

**Remove `withDebugTracing()` before deploying to production.** It generates significant console noise. Use `isDevMode()` if you want to conditionally enable it:

```typescript
import { isDevMode } from '@angular/core';

const routerFeatures = [
  withComponentInputBinding(),
  ...(isDevMode() ? [withDebugTracing()] : [])
];

provideRouter(routes, ...routerFeatures);
```

## Practical Patterns

### Global Loading Indicator

A centralized loading service that any component can consume:

```typescript
// loading.service.ts
import { Injectable, inject, signal } from '@angular/core';
import { Router, NavigationStart, NavigationEnd, NavigationCancel, NavigationError } from '@angular/router';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Injectable({ providedIn: 'root' })
export class LoadingService {
  private router = inject(Router);

  isLoading = signal(false);

  constructor() {
    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe(event => {
        if (event instanceof NavigationStart) {
          this.isLoading.set(true);
        }
        if (event instanceof NavigationEnd ||
            event instanceof NavigationCancel ||
            event instanceof NavigationError) {
          this.isLoading.set(false);
        }
      });
  }
}
```

```typescript
// app.component.ts
@Component({
  selector: 'app-root',
  template: `
    @if (loading.isLoading()) {
      <div class="progress-bar">
        <div class="progress-bar-fill"></div>
      </div>
    }
    <app-nav />
    <router-outlet />
  `
})
export class AppComponent {
  loading = inject(LoadingService);
}
```

### Analytics Tracking

Track page views on every successful navigation:

```typescript
// analytics.service.ts
import { Injectable, inject } from '@angular/core';
import { Router, NavigationEnd } from '@angular/router';
import { filter } from 'rxjs';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';

@Injectable({ providedIn: 'root' })
export class AnalyticsService {
  private router = inject(Router);

  constructor() {
    this.router.events
      .pipe(
        filter((e): e is NavigationEnd => e instanceof NavigationEnd),
        takeUntilDestroyed()
      )
      .subscribe(event => {
        this.trackPageView(event.urlAfterRedirects);
      });
  }

  private trackPageView(url: string): void {
    // Google Analytics 4
    gtag('event', 'page_view', {
      page_path: url,
      page_title: document.title
    });
  }
}

// Ensure the service is instantiated at app startup
// app.config.ts
export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    {
      provide: ENVIRONMENT_INITIALIZER,
      multi: true,
      useValue: () => inject(AnalyticsService)
    }
  ]
};
```

**Why `urlAfterRedirects`?** If you use `url`, you would log the pre-redirect URL. The `urlAfterRedirects` property reflects the actual page the user landed on.

### Unsaved Changes Warning

Detect when navigation is canceled due to an unsaved changes guard:

```typescript
// unsaved-changes-tracker.service.ts
@Injectable({ providedIn: 'root' })
export class UnsavedChangesTrackerService {
  private router = inject(Router);

  lastBlockedUrl = signal<string | null>(null);

  constructor() {
    this.router.events
      .pipe(
        filter((e): e is NavigationCancel => e instanceof NavigationCancel),
        filter(e => e.code === NavigationCancellationCode.GuardRejected),
        takeUntilDestroyed()
      )
      .subscribe(event => {
        this.lastBlockedUrl.set(event.url);
      });
  }
}
```

### Navigation Performance Monitoring

Track slow navigations in production:

```typescript
// nav-perf.service.ts
@Injectable({ providedIn: 'root' })
export class NavigationPerfService {
  private router = inject(Router);
  private pending = new Map<number, number>();

  private readonly SLOW_THRESHOLD_MS = 3000;

  constructor() {
    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe(event => {
        if (event instanceof NavigationStart) {
          this.pending.set(event.id, performance.now());
        }

        if (event instanceof NavigationEnd) {
          const start = this.pending.get(event.id);
          if (start) {
            const duration = performance.now() - start;
            this.pending.delete(event.id);

            if (duration > this.SLOW_THRESHOLD_MS) {
              this.reportSlowNavigation(event.urlAfterRedirects, duration);
            }
          }
        }

        // Clean up on cancel/error too
        if (event instanceof NavigationCancel || event instanceof NavigationError) {
          this.pending.delete(event.id);
        }
      });
  }

  private reportSlowNavigation(url: string, duration: number): void {
    console.warn(`Slow navigation to ${url}: ${duration.toFixed(0)}ms`);
    // Send to your monitoring service
  }
}
```

### Granular Phase Timing

Break down where time is spent during navigation:

```typescript
@Injectable({ providedIn: 'root' })
export class NavigationTimingService {
  private router = inject(Router);

  constructor() {
    let phases: Record<string, number> = {};

    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe(event => {
        const now = performance.now();

        if (event instanceof NavigationStart) {
          phases = { navigationStart: now };
        }
        if (event instanceof RouteConfigLoadStart) {
          phases['lazyLoadStart'] = now;
        }
        if (event instanceof RouteConfigLoadEnd) {
          phases['lazyLoadEnd'] = now;
        }
        if (event instanceof GuardsCheckStart) {
          phases['guardsStart'] = now;
        }
        if (event instanceof GuardsCheckEnd) {
          phases['guardsEnd'] = now;
        }
        if (event instanceof ResolveStart) {
          phases['resolveStart'] = now;
        }
        if (event instanceof ResolveEnd) {
          phases['resolveEnd'] = now;
        }
        if (event instanceof NavigationEnd) {
          const total = now - phases['navigationStart'];
          const lazyLoad = phases['lazyLoadEnd']
            ? phases['lazyLoadEnd'] - phases['lazyLoadStart']
            : 0;
          const guards = phases['guardsEnd'] - phases['guardsStart'];
          const resolvers = phases['resolveEnd'] - phases['resolveStart'];

          console.table({
            url: event.urlAfterRedirects,
            total: `${total.toFixed(0)}ms`,
            lazyLoad: `${lazyLoad.toFixed(0)}ms`,
            guards: `${guards.toFixed(0)}ms`,
            resolvers: `${resolvers.toFixed(0)}ms`
          });
        }
      });
  }
}
```

### Handling Chunk Load Failures

When a deployment happens while users are on the site, lazy-loaded chunks may 404. Handle this gracefully:

```typescript
@Injectable({ providedIn: 'root' })
export class ChunkErrorHandlerService {
  private router = inject(Router);

  constructor() {
    this.router.events
      .pipe(
        filter((e): e is NavigationError => e instanceof NavigationError),
        takeUntilDestroyed()
      )
      .subscribe(event => {
        // Check if the error is a chunk load failure
        if (this.isChunkLoadError(event.error)) {
          // Reload the page to fetch the new deployment's assets
          window.location.href = event.url;
        }
      });
  }

  private isChunkLoadError(error: any): boolean {
    return (
      error?.name === 'ChunkLoadError' ||
      error?.message?.includes('Loading chunk') ||
      error?.message?.includes('Failed to fetch dynamically imported module')
    );
  }
}
```

## Terminal Event Guarantees

Every navigation produces exactly one terminal event:

| Terminal Event | When it fires |
|---|---|
| `NavigationEnd` | Navigation completed successfully |
| `NavigationCancel` | Guard returned `false`, `UrlTree`, or `RedirectCommand`; resolver returned `EMPTY`; or a new navigation superseded this one |
| `NavigationError` | Unhandled error thrown during navigation (guard throw, resolver throw, chunk load failure) |
| `NavigationSkipped` | Same-URL navigation ignored, or URL excluded by `UrlHandlingStrategy` |

This guarantee means any "start on `NavigationStart`, stop on terminal" pattern is safe. You will never have a `NavigationStart` without a corresponding terminal event.

---

## Events That Do NOT Extend RouterEvent

Some events lack the `id` and `url` properties because they do not extend `RouterEvent`:

| Event | What it has instead |
|---|---|
| `ActivationStart` | `snapshot: ActivatedRouteSnapshot` |
| `ActivationEnd` | `snapshot: ActivatedRouteSnapshot` |
| `ChildActivationStart` | `snapshot: ActivatedRouteSnapshot` |
| `ChildActivationEnd` | `snapshot: ActivatedRouteSnapshot` |
| `RouteConfigLoadStart` | `route: Route` |
| `RouteConfigLoadEnd` | `route: Route` |
| `Scroll` | `routerEvent: NavigationEnd`, `position`, `anchor` |

To correlate these with a navigation ID, track the ID from the preceding `NavigationStart`.

## Common Mistakes

- **Using `reason` instead of `code` on `NavigationCancel`/`NavigationSkipped`.** The `reason` string is for debugging. It is not part of the public API contract and can change between Angular versions. Always use the `code` enum for programmatic checks.

- **Forgetting terminal events in loading indicators.** If you only listen for `NavigationStart` and `NavigationEnd`, your loading indicator will get stuck when a guard cancels or an error occurs. Always handle all four terminal events: `NavigationEnd`, `NavigationCancel`, `NavigationError`, and `NavigationSkipped`.

- **Not unsubscribing from router events.** `Router.events` is a long-lived observable that persists for the lifetime of the application. If you subscribe in a component without cleanup, the subscription leaks. Use `takeUntilDestroyed()` in the constructor or `takeUntil(this.destroy$)` in `ngOnInit`.

- **Leaving `withDebugTracing()` in production builds.** It floods the console with verbose output. Either remove it or gate it behind `isDevMode()`.

- **Subscribing to the wrong event for analytics.** Using `NavigationStart` for page tracking will log pages the user never actually reached (guard-blocked navigations). Use `NavigationEnd` with `urlAfterRedirects`.

- **Assuming all events have `id` and `url`.** Events like `ActivationStart`, `ActivationEnd`, and `Scroll` do not extend `RouterEvent`. Accessing `.id` on them will be `undefined`.

---

## Best Practices

- **Use `NavigationCancellationCode` and `NavigationSkippedCode` enums.** They provide stable values you can safely use in production logic, unlike the human-readable `reason` strings.

- **Centralize event handling in services.** Rather than subscribing to router events in multiple components, create dedicated services (`LoadingService`, `AnalyticsService`) and inject them where needed. Initialize them via `ENVIRONMENT_INITIALIZER` to ensure they start on app bootstrap.

- **Prefer `Router.currentNavigation()` for simple loading state.** If all you need is "is navigation in progress?", the `currentNavigation` signal is simpler than subscribing to events.

- **Correlate events by navigation `id`.** When you need to measure timing across multiple phases of a single navigation, use the shared `id` from `NavigationStart` to group related events.

- **Gate debug tracing with `isDevMode()`.** This lets you keep the configuration in your codebase without polluting production logs.

- **Handle chunk load errors explicitly.** After deployments, old lazy chunks may no longer exist. Detecting `ChunkLoadError` in `NavigationError` events and reloading the page prevents users from seeing blank screens.

## Real-World Use Cases

### 1. Route-Aware Breadcrumbs

Build breadcrumbs by listening to `ActivationEnd` events and traversing the route tree:

```typescript
@Injectable({ providedIn: 'root' })
export class BreadcrumbService {
  private router = inject(Router);

  breadcrumbs = signal<{ label: string; url: string }[]>([]);

  constructor() {
    this.router.events
      .pipe(
        filter((e): e is NavigationEnd => e instanceof NavigationEnd),
        takeUntilDestroyed()
      )
      .subscribe(() => {
        const crumbs: { label: string; url: string }[] = [];
        let route = this.router.routerState.root;
        let url = '';

        while (route.firstChild) {
          route = route.firstChild;
          const segment = route.snapshot.url.map(s => s.path).join('/');
          if (segment) {
            url += '/' + segment;
          }
          const label = route.snapshot.data['breadcrumb'];
          if (label) {
            crumbs.push({ label, url });
          }
        }

        this.breadcrumbs.set(crumbs);
      });
  }
}
```

### 2. Navigation Audit Log

Log all navigations with their outcomes for debugging or compliance:

```typescript
@Injectable({ providedIn: 'root' })
export class NavigationAuditService {
  private router = inject(Router);

  constructor() {
    this.router.events
      .pipe(takeUntilDestroyed())
      .subscribe(event => {
        if (event instanceof NavigationEnd) {
          this.log('success', event.urlAfterRedirects, event.id);
        }
        if (event instanceof NavigationCancel) {
          this.log('canceled', event.url, event.id, {
            code: NavigationCancellationCode[event.code!],
            reason: event.reason
          });
        }
        if (event instanceof NavigationError) {
          this.log('error', event.url, event.id, {
            error: event.error?.message ?? String(event.error)
          });
        }
      });
  }

  private log(outcome: string, url: string, navId: number, extra?: Record<string, any>): void {
    const entry = {
      timestamp: new Date().toISOString(),
      navId,
      outcome,
      url,
      ...extra
    };
    // Send to logging service or store locally
    console.log('[NavAudit]', entry);
  }
}
```

### 3. Scroll-to-Top with Exceptions

Custom scroll behavior that scrolls to top on forward navigation but preserves position for specific routes:

```typescript
@Injectable({ providedIn: 'root' })
export class ScrollService {
  private router = inject(Router);
  private viewportScroller = inject(ViewportScroller);

  // Routes that should preserve scroll position
  private readonly preserveScrollRoutes = ['/search', '/feed'];

  constructor() {
    this.router.events
      .pipe(
        filter((e): e is Scroll => e instanceof Scroll),
        takeUntilDestroyed()
      )
      .subscribe(event => {
        if (event.position) {
          // Back/forward: restore position
          setTimeout(() => this.viewportScroller.scrollToPosition(event.position!), 0);
        } else if (event.anchor) {
          // Fragment: scroll to element
          setTimeout(() => this.viewportScroller.scrollToAnchor(event.anchor!), 0);
        } else {
          // Forward navigation: check if we should preserve scroll
          const url = event.routerEvent.urlAfterRedirects;
          const shouldPreserve = this.preserveScrollRoutes.some(r => url.startsWith(r));
          if (!shouldPreserve) {
            this.viewportScroller.scrollToPosition([0, 0]);
          }
        }
      });
  }
}
```

> **Key Takeaway:** The router's 17 event types give you fine-grained hooks into every navigation phase; always handle all four terminal events (`NavigationEnd`, `NavigationCancel`, `NavigationError`, `NavigationSkipped`) and use the stable `code` enums rather than `reason` strings for programmatic logic.
