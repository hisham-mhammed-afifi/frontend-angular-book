# 16.5 Form State and Error Display

> Angular tracks rich state metadata on every form control (pristine, dirty, touched, valid, pending, submitted) and provides CSS classes, programmatic APIs, and observable streams to build polished, user-friendly error display systems.

---

## Control State Properties

Every `AbstractControl` (including `FormControl`, `FormGroup`, and `FormArray`) tracks these state properties:

| Property | Type | Meaning |
|---|---|---|
| `value` | `T` | Current value of the control |
| `status` | `string` | `'VALID'`, `'INVALID'`, `'PENDING'`, or `'DISABLED'` |
| `valid` | `boolean` | `true` if status is `VALID` |
| `invalid` | `boolean` | `true` if status is `INVALID` |
| `pending` | `boolean` | `true` if async validators are running |
| `disabled` | `boolean` | `true` if the control is disabled |
| `enabled` | `boolean` | `true` if the control is enabled |
| `pristine` | `boolean` | `true` if the user has NOT changed the value |
| `dirty` | `boolean` | `true` if the user HAS changed the value |
| `touched` | `boolean` | `true` if the user has focused and blurred the control |
| `untouched` | `boolean` | `true` if the user has never focused and blurred the control |
| `errors` | `ValidationErrors \| null` | Object containing active validation errors, or `null` |

### Pristine vs Dirty

- **Pristine:** The control's value has not been changed by the user through the UI. Programmatic changes via `setValue`/`patchValue` do NOT make a control dirty (unless you pass `{ emitEvent: true }` and subscribe).
- **Dirty:** The user has modified the control's value through the UI (typing, selecting, etc.).

Once a control becomes dirty, it stays dirty until explicitly reset.

### Touched vs Untouched

- **Untouched:** The control has never been focused and then blurred.
- **Touched:** The control has been focused and then blurred at least once.

Once touched, a control stays touched until reset. Touching does not require changing the value. Simply clicking into a field and clicking out sets `touched = true`.

### The Difference

```
User clicks into an empty field and clicks out:
  pristine = true    (value did not change)
  touched  = true    (field was visited)

User types "hello" in a field:
  pristine = false   (value changed)
  dirty    = true
  touched  = false   (may not have blurred yet)
```

## CSS State Classes

Angular automatically adds CSS classes to form elements based on their state:

| State | True | False |
|---|---|---|
| Validity | `.ng-valid` | `.ng-invalid` |
| Modification | `.ng-dirty` | `.ng-pristine` |
| Interaction | `.ng-touched` | `.ng-untouched` |
| Async validation | `.ng-pending` | (none) |
| Form submission | `.ng-submitted` | (none, on `<form>` only) |

These classes are applied to every element bound to a form control (`formControlName`, `formControl`, `ngModel`) and to the `<form>` element itself.

### Styling with CSS Classes

```css
/* Red border after user has interacted and value is invalid */
input.ng-invalid.ng-touched {
  border: 2px solid #dc3545;
}

/* Green border after user has changed value and it is valid */
input.ng-valid.ng-dirty {
  border: 2px solid #28a745;
}

/* Show all invalid fields after form submission */
form.ng-submitted input.ng-invalid {
  border: 2px solid #dc3545;
  background-color: #fff5f5;
}

/* Pending async validation */
input.ng-pending {
  border: 2px solid #ffc107;
}
```

### CSS-Only Error Display

You can show error messages purely with CSS, no `@if` needed:

```css
/* Hide error messages by default */
.field-error {
  display: none;
  color: #dc3545;
  font-size: 0.875rem;
}

/* Show error after the user has touched an invalid field */
input.ng-invalid.ng-touched + .field-error {
  display: block;
}

/* Show error after form submission */
form.ng-submitted input.ng-invalid + .field-error {
  display: block;
}
```

```html
<input formControlName="email" />
<span class="field-error">Please enter a valid email.</span>
```

This approach is limited to single error messages. For multiple context-specific errors, use template logic.

## Form-Level State: submitted

The `<form>` element (via `NgForm` or reactive `[formGroup]`) tracks a `submitted` property. After the user submits:

- `NgForm.submitted` becomes `true`.
- The `<form>` element gets the `.ng-submitted` CSS class.
- All child controls can use this to decide when to show errors.

```typescript
// Reactive form: access via the form group
@ViewChild('formRef') formRef!: NgForm;
// Or track it manually
submitted = false;

onSubmit(): void {
  this.submitted = true;
  if (this.form.invalid) {
    this.form.markAllAsTouched();
    return;
  }
  // process form...
}
```

## Programmatic State Manipulation

### markAsTouched / markAsUntouched

```typescript
// Mark a single control as touched
this.form.get('email')?.markAsTouched();

// Mark all controls in a group as touched (reveals all errors)
this.form.markAllAsTouched();

// Reset touched state
this.form.get('email')?.markAsUntouched();
```

### markAsDirty / markAsPristine

```typescript
// Mark as dirty (as if the user changed the value)
this.form.get('email')?.markAsDirty();

// Reset to pristine (as if the user never changed it)
this.form.markAsPristine();
```

### onlySelf Option

By default, marking a control propagates to its parent. Use `{ onlySelf: true }` to prevent this:

```typescript
// Only marks the email control, NOT the parent FormGroup
this.form.get('email')?.markAsTouched({ onlySelf: true });
```

### Common Pattern: Mark All on Submit

```typescript
onSubmit(): void {
  if (this.form.invalid) {
    // Show all validation errors at once
    this.form.markAllAsTouched();
    return;
  }
  this.saveData(this.form.getRawValue());
}
```

## Error Access Methods

### errors Property

Returns the full error object or `null`:

```typescript
const ctrl = this.form.get('username');

ctrl.errors;
// { required: true, minlength: { requiredLength: 3, actualLength: 1 } }
// or null if valid
```

### hasError()

Checks for a specific error key:

```typescript
ctrl.hasError('required');      // true/false
ctrl.hasError('minlength');     // true/false
ctrl.hasError('email');         // true/false
```

### getError()

Returns the value of a specific error key, or `null`:

```typescript
ctrl.getError('minlength');
// { requiredLength: 3, actualLength: 1 }
// or null if that error is not present

ctrl.getError('required');
// true
// or null
```

### hasError with Path

Access errors on nested controls from the parent:

```typescript
// Instead of: this.form.get('address.city')?.hasError('required')
this.form.hasError('required', ['address', 'city']);

// Or with dot notation:
this.form.hasError('required', 'address.city');
```

## Error Display Strategies

### Strategy 1: Inline Conditional Messages

The simplest approach. Show specific messages for each error type:

```html
<input formControlName="email" />

@if (form.get('email')?.invalid && form.get('email')?.touched) {
  @if (form.get('email')?.hasError('required')) {
    <p class="error">Email is required.</p>
  }
  @if (form.get('email')?.hasError('email')) {
    <p class="error">Enter a valid email address.</p>
  }
  @if (form.get('email')?.hasError('maxlength')) {
    <p class="error">
      Maximum {{ form.get('email')?.getError('maxlength').requiredLength }} characters.
    </p>
  }
}
```

This works for small forms but becomes verbose with many fields.

### Strategy 2: Helper Method

Move the visibility logic to the component:

```typescript
@Component({ /* ... */ })
export class FormComponent {
  form = this.fb.group({
    name: ['', [Validators.required, Validators.minLength(2)]],
    email: ['', [Validators.required, Validators.email]]
  });

  submitted = false;

  shouldShowError(controlName: string): boolean {
    const ctrl = this.form.get(controlName);
    return !!(ctrl?.invalid && (ctrl.touched || this.submitted));
  }

  getErrorMessage(controlName: string): string {
    const ctrl = this.form.get(controlName);
    if (!ctrl?.errors) return '';

    if (ctrl.hasError('required')) return 'This field is required.';
    if (ctrl.hasError('email')) return 'Enter a valid email.';
    if (ctrl.hasError('minlength')) {
      const err = ctrl.getError('minlength');
      return `Minimum ${err.requiredLength} characters (entered ${err.actualLength}).`;
    }
    if (ctrl.hasError('maxlength')) {
      const err = ctrl.getError('maxlength');
      return `Maximum ${err.requiredLength} characters.`;
    }
    if (ctrl.hasError('pattern')) return 'Invalid format.';

    return 'Invalid value.';
  }
}
```

```html
<input formControlName="name" />
@if (shouldShowError('name')) {
  <p class="error">{{ getErrorMessage('name') }}</p>
}
```

### Strategy 3: Reusable Error Component

Create a component that handles error display for any control:

```typescript
// form-error.component.ts
@Component({
  selector: 'app-form-error',
  standalone: true,
  template: `
    @if (control()?.invalid && (control()?.touched || submitted())) {
      <div class="error-container">
        @for (message of errorMessages(); track message) {
          <p class="error">{{ message }}</p>
        }
      </div>
    }
  `,
  styles: [`
    .error-container { margin-top: 4px; }
    .error { color: #dc3545; font-size: 0.8rem; margin: 2px 0; }
  `]
})
export class FormErrorComponent {
  control = input<AbstractControl | null>(null);
  submitted = input(false);
  messages = input<Record<string, string>>({});

  errorMessages = computed(() => {
    const ctrl = this.control();
    const customMessages = this.messages();
    if (!ctrl?.errors) return [];

    return Object.keys(ctrl.errors).map(key => {
      if (customMessages[key]) return customMessages[key];
      return this.defaultMessage(key, ctrl.errors![key]);
    });
  });

  private defaultMessage(errorKey: string, errorValue: any): string {
    const defaults: Record<string, (val: any) => string> = {
      required: () => 'This field is required.',
      email: () => 'Enter a valid email address.',
      minlength: (v) => `Minimum ${v.requiredLength} characters (you entered ${v.actualLength}).`,
      maxlength: (v) => `Maximum ${v.requiredLength} characters.`,
      min: (v) => `Minimum value is ${v.min}.`,
      max: (v) => `Maximum value is ${v.max}.`,
      pattern: () => 'Invalid format.',
    };

    const formatter = defaults[errorKey];
    return formatter ? formatter(errorValue) : 'Invalid value.';
  }
}
```

Usage:

```html
<input formControlName="email" />
<app-form-error
  [control]="form.get('email')"
  [submitted]="submitted"
  [messages]="{ email: 'Please use your company email address.' }"
/>
```

### Strategy 4: Error Directive

A structural directive that injects the control and handles display:

```typescript
@Directive({
  selector: '[appFieldError]',
  standalone: true
})
export class FieldErrorDirective implements OnInit, OnDestroy {
  private controlName = input.required<string>({ alias: 'appFieldError' });
  private controlContainer = inject(ControlContainer);
  private vcr = inject(ViewContainerRef);
  private templateRef = inject(TemplateRef<any>);
  private destroy$ = new Subject<void>();

  ngOnInit(): void {
    const control = this.controlContainer.control?.get(this.controlName());
    if (!control) return;

    control.statusChanges
      .pipe(takeUntil(this.destroy$))
      .subscribe(() => {
        this.vcr.clear();
        if (control.invalid && control.touched) {
          this.vcr.createEmbeddedView(this.templateRef, {
            $implicit: control.errors
          });
        }
      });

    // Also react to touch state changes
    control.valueChanges
      .pipe(takeUntil(this.destroy$))
      .subscribe(() => {
        if (control.invalid && control.touched) {
          this.vcr.clear();
          this.vcr.createEmbeddedView(this.templateRef, {
            $implicit: control.errors
          });
        }
      });
  }

  ngOnDestroy(): void {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
```

Usage:

```html
<form [formGroup]="form">
  <input formControlName="email" />
  <ng-template appFieldError="email" let-errors>
    @if (errors['required']) { <p class="error">Required.</p> }
    @if (errors['email']) { <p class="error">Invalid email.</p> }
  </ng-template>
</form>
```

### Strategy 5: Centralized Error Message Map

Define all error messages in a configuration service:

```typescript
// error-messages.ts
import { InjectionToken } from '@angular/core';

export type ErrorMessageMap = Record<string, (params: any) => string>;

export const ERROR_MESSAGES = new InjectionToken<ErrorMessageMap>('ErrorMessages', {
  providedIn: 'root',
  factory: () => ({
    required: () => 'This field is required.',
    email: () => 'Enter a valid email address.',
    minlength: (p: any) => `At least ${p.requiredLength} characters required.`,
    maxlength: (p: any) => `No more than ${p.requiredLength} characters.`,
    min: (p: any) => `Minimum value is ${p.min}.`,
    max: (p: any) => `Maximum value is ${p.max}.`,
    pattern: () => 'Invalid format.',
    passwordMismatch: () => 'Passwords do not match.',
    usernameTaken: () => 'This username is already taken.',
    emailTaken: () => 'This email is already registered.'
  })
});
```

```typescript
// error-message.pipe.ts
@Pipe({ name: 'errorMessage', standalone: true })
export class ErrorMessagePipe implements PipeTransform {
  private messages = inject(ERROR_MESSAGES);

  transform(errors: ValidationErrors | null): string[] {
    if (!errors) return [];

    return Object.entries(errors).map(([key, value]) => {
      const messageFn = this.messages[key];
      return messageFn ? messageFn(value) : `Validation error: ${key}`;
    });
  }
}
```

Usage:

```html
@if (form.get('email')?.invalid && form.get('email')?.touched) {
  @for (msg of form.get('email')?.errors | errorMessage; track msg) {
    <p class="error">{{ msg }}</p>
  }
}
```

## Observing State Changes

### statusChanges

```typescript
this.form.statusChanges
  .pipe(takeUntilDestroyed())
  .subscribe(status => {
    // 'VALID' | 'INVALID' | 'PENDING' | 'DISABLED'
    console.log('Form status:', status);
  });
```

### events (Angular 18+)

The `events` observable provides granular state change events:

```typescript
import {
  TouchedChangeEvent,
  PristineChangeEvent,
  StatusChangeEvent,
  ValueChangeEvent
} from '@angular/forms';

this.form.events
  .pipe(takeUntilDestroyed())
  .subscribe(event => {
    if (event instanceof TouchedChangeEvent) {
      console.log('Touched:', event.touched);
    }
    if (event instanceof PristineChangeEvent) {
      console.log('Pristine:', event.pristine);
    }
    if (event instanceof StatusChangeEvent) {
      console.log('Status:', event.status);
    }
  });
```

### Converting to Signals

```typescript
formValid = toSignal(
  this.form.statusChanges.pipe(map(s => s === 'VALID')),
  { initialValue: this.form.valid }
);
```

## Scroll to First Error

After marking all controls as touched, scroll the user to the first invalid field:

```typescript
onSubmit(): void {
  if (this.form.invalid) {
    this.form.markAllAsTouched();
    this.scrollToFirstError();
    return;
  }
  // process form
}

private scrollToFirstError(): void {
  const firstInvalid = document.querySelector('.ng-invalid[formControlName]');
  if (firstInvalid) {
    firstInvalid.scrollIntoView({ behavior: 'smooth', block: 'center' });
    (firstInvalid as HTMLElement).focus();
  }
}
```

### As a Reusable Service

```typescript
@Injectable({ providedIn: 'root' })
export class FormErrorScrollService {
  scrollToFirstError(formElement?: HTMLElement): void {
    const container = formElement ?? document;
    const firstInvalid = container.querySelector(
      '.ng-invalid[formControlName], .ng-invalid[formControl], .ng-invalid[ngModel]'
    ) as HTMLElement | null;

    if (firstInvalid) {
      firstInvalid.scrollIntoView({ behavior: 'smooth', block: 'center' });
      setTimeout(() => firstInvalid.focus(), 300);
    }
  }
}
```

## Pending State Display

Show feedback while async validators are running:

```html
<div class="field">
  <input formControlName="username" />

  @if (form.get('username')?.pending) {
    <span class="spinner">Checking availability...</span>
  }

  @if (form.get('username')?.invalid && form.get('username')?.touched &&
       !form.get('username')?.pending) {
    <app-form-error [control]="form.get('username')" />
  }
</div>
```

Disable the submit button while any control is pending:

```html
<button type="submit" [disabled]="form.invalid || form.pending">
  Submit
</button>
```

## Common Mistakes

- **Showing errors immediately on page load.** Displaying errors before the user interacts creates a hostile user experience. Always gate error display behind `touched`, `dirty`, or `submitted`.

- **Forgetting to handle the `PENDING` state.** While async validators are running, the control is neither valid nor invalid. If you only check `invalid`, you may hide errors during validation. If you only check `valid`, the submit button stays disabled during pending.

- **Using `markAsDirty()` when you mean `markAllAsTouched()`.** Dirty means "value changed by user," touched means "field was visited." For revealing validation errors on submit, use `markAllAsTouched()`. Marking as dirty misrepresents the actual interaction.

- **Not resetting state after form submission.** After successfully submitting and clearing the form, call `form.reset()` or `form.resetForm()` (for NgForm). Otherwise, the form retains the `submitted` and `dirty` states.

- **Hardcoding error messages in every template.** Repeating the same error text across forms leads to inconsistency. Centralize messages in a service, pipe, or component.

- **Checking `form.get('ctrl')` without null safety.** `get()` returns `AbstractControl | null`. Always use optional chaining (`?.`) or a non-null assertion when you are certain the control exists.

---

## Best Practices

- **Use `touched || submitted` as the error visibility condition.** This covers both progressive disclosure (showing errors as the user moves through fields) and batch disclosure (showing all errors on submit).

- **Create a reusable error component or directive.** This eliminates repetitive `@if` blocks and ensures consistent error styling across the application.

- **Centralize error messages with an `InjectionToken`.** Define all validation messages in one place. This makes localization straightforward and keeps messages consistent.

- **Use CSS classes for visual feedback, template logic for text messages.** The `.ng-invalid.ng-touched` CSS pattern handles borders and backgrounds. Use `@if` blocks or error components for the actual text.

- **Scroll to the first error on submit.** On long forms, the user may not see that a field at the top is invalid. Scrolling and focusing guides them directly to the problem.

- **Disable the submit button for `invalid || pending`.** This prevents submission while async validators are still running.

- **Use the `events` observable (Angular 18+) for fine-grained state tracking.** It provides typed events for touched, pristine, and status changes without combining multiple subscriptions.

## Real-World Use Cases

### 1. Complete Form with Progressive Error Disclosure

```typescript
@Component({
  selector: 'app-checkout',
  standalone: true,
  imports: [ReactiveFormsModule, FormErrorComponent],
  template: `
    <form [formGroup]="form" (ngSubmit)="submit()" #formRef>
      <div class="field">
        <label for="cardName">Name on Card</label>
        <input id="cardName" formControlName="cardName" />
        <app-form-error [control]="form.controls.cardName" [submitted]="submitted" />
      </div>

      <div class="field">
        <label for="cardNumber">Card Number</label>
        <input id="cardNumber" formControlName="cardNumber" />
        <app-form-error
          [control]="form.controls.cardNumber"
          [submitted]="submitted"
          [messages]="{ pattern: 'Enter a valid 16-digit card number.' }"
        />
      </div>

      <div class="row">
        <div class="field">
          <label for="expiry">Expiry</label>
          <input id="expiry" formControlName="expiry" placeholder="MM/YY" />
          <app-form-error [control]="form.controls.expiry" [submitted]="submitted" />
        </div>
        <div class="field">
          <label for="cvv">CVV</label>
          <input id="cvv" formControlName="cvv" />
          <app-form-error [control]="form.controls.cvv" [submitted]="submitted" />
        </div>
      </div>

      <button type="submit" [disabled]="form.pending">
        @if (form.pending) { Validating... } @else { Pay Now }
      </button>
    </form>
  `
})
export class CheckoutComponent {
  private fb = inject(NonNullableFormBuilder);
  private scrollService = inject(FormErrorScrollService);
  submitted = false;

  form = this.fb.group({
    cardName: ['', Validators.required],
    cardNumber: ['', [Validators.required, Validators.pattern(/^\d{16}$/)]],
    expiry: ['', [Validators.required, Validators.pattern(/^(0[1-9]|1[0-2])\/\d{2}$/)]],
    cvv: ['', [Validators.required, Validators.pattern(/^\d{3,4}$/)]]
  });

  submit(): void {
    this.submitted = true;
    if (this.form.invalid) {
      this.form.markAllAsTouched();
      this.scrollService.scrollToFirstError();
      return;
    }
    console.log('Processing payment:', this.form.getRawValue());
  }
}
```

### 2. Form with Summary Error Banner

```typescript
@Component({
  selector: 'app-registration',
  standalone: true,
  imports: [ReactiveFormsModule, ErrorMessagePipe],
  template: `
    @if (submitted && form.invalid) {
      <div class="error-banner" role="alert">
        <h3>Please fix the following errors:</h3>
        <ul>
          @for (field of invalidFields(); track field.name) {
            <li>
              <a (click)="focusField(field.name)">
                {{ field.label }}: {{ field.errors | errorMessage | join:', ' }}
              </a>
            </li>
          }
        </ul>
      </div>
    }

    <form [formGroup]="form" (ngSubmit)="submit()">
      <input formControlName="name" id="name" />
      <input formControlName="email" id="email" />
      <input formControlName="phone" id="phone" />
      <button type="submit">Register</button>
    </form>
  `
})
export class RegistrationComponent {
  private fb = inject(NonNullableFormBuilder);
  submitted = false;

  fieldLabels: Record<string, string> = {
    name: 'Full Name',
    email: 'Email Address',
    phone: 'Phone Number'
  };

  form = this.fb.group({
    name: ['', Validators.required],
    email: ['', [Validators.required, Validators.email]],
    phone: ['', [Validators.required, Validators.pattern(/^\+?\d{10,15}$/)]]
  });

  invalidFields = computed(() => {
    if (!this.submitted) return [];

    return Object.keys(this.form.controls)
      .filter(name => this.form.get(name)?.invalid)
      .map(name => ({
        name,
        label: this.fieldLabels[name] ?? name,
        errors: this.form.get(name)?.errors ?? null
      }));
  });

  focusField(name: string): void {
    const el = document.getElementById(name);
    el?.scrollIntoView({ behavior: 'smooth', block: 'center' });
    el?.focus();
  }

  submit(): void {
    this.submitted = true;
    if (this.form.invalid) {
      this.form.markAllAsTouched();
      return;
    }
    console.log('Registered:', this.form.getRawValue());
  }
}
```

### 3. Dirty State Tracking for Unsaved Changes Warning

```typescript
@Component({
  selector: 'app-settings',
  standalone: true,
  imports: [ReactiveFormsModule],
  template: `
    @if (form.dirty) {
      <div class="unsaved-banner">
        You have unsaved changes.
        <button (click)="save()">Save</button>
        <button (click)="discard()">Discard</button>
      </div>
    }

    <form [formGroup]="form">
      <input formControlName="displayName" />
      <input formControlName="timezone" />
      <select formControlName="language">
        <option value="en">English</option>
        <option value="es">Spanish</option>
      </select>
    </form>
  `
})
export class SettingsComponent {
  private fb = inject(NonNullableFormBuilder);
  private settingsService = inject(SettingsService);

  form = this.fb.group({
    displayName: [''],
    timezone: ['UTC'],
    language: ['en']
  });

  constructor() {
    // Load current settings
    this.settingsService.get().subscribe(settings => {
      this.form.patchValue(settings);
      this.form.markAsPristine();  // loaded data is not "user changes"
    });
  }

  save(): void {
    this.settingsService.update(this.form.getRawValue()).subscribe(() => {
      this.form.markAsPristine();  // saved, no more unsaved changes
    });
  }

  discard(): void {
    this.settingsService.get().subscribe(settings => {
      this.form.patchValue(settings);
      this.form.markAsPristine();
    });
  }

  // Use with canDeactivate guard
  hasUnsavedChanges(): boolean {
    return this.form.dirty;
  }
}
```

> **Key Takeaway:** Gate error visibility behind `touched || submitted` to avoid showing errors before user interaction, centralize error messages in a service or pipe for consistency, and build a reusable error component to eliminate repetitive template logic across forms.
