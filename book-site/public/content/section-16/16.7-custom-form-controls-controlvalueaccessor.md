# 16.7 Custom Form Controls (`ControlValueAccessor`)

> Build reusable form components that integrate seamlessly with Angular's forms API using the `ControlValueAccessor` interface, and learn how signal-based `FormValueControl` simplifies this in Angular 21.

---

## Why Custom Form Controls

Angular's built-in form directives handle standard HTML inputs. But real applications need custom components: star ratings, color pickers, tag inputs, rich text editors, date range selectors. These components need to work with `ngModel`, `formControl`, and `formControlName` just like native inputs.

The `ControlValueAccessor` (CVA) interface is the bridge that makes this possible. It defines the contract between your custom component and Angular's forms system.

```
FormControl <---> ControlValueAccessor <---> Your Component UI
     |                    |                        |
  setValue()        writeValue()              DOM update
  valueChanges      onChange()              User interaction
```

---

## The ControlValueAccessor Interface

The interface has four methods, three required and one optional:

```typescript
interface ControlValueAccessor {
  // Called by the forms API to write a value into the component
  writeValue(obj: any): void;

  // Registers a callback for when the component value changes
  registerOnChange(fn: (value: any) => void): void;

  // Registers a callback for when the component is touched/blurred
  registerOnTouched(fn: () => void): void;

  // Optional: called when the control's disabled state changes
  setDisabledState?(isDisabled: boolean): void;
}
```

### How the Methods Map to Data Flow

| Direction | Method | Trigger |
|-----------|--------|---------|
| Model to View | `writeValue()` | `FormControl.setValue()`, `patchValue()`, or initial value |
| View to Model | `onChange()` callback | User changes the value in the UI |
| Touch State | `onTouched()` callback | User interacts with the control (blur) |
| Disabled State | `setDisabledState()` | `FormControl.disable()` / `enable()` |

## Implementing a Basic Custom Control

Here is a complete example: a star rating component.

### Step 1: Create the Component

```typescript
import { Component, forwardRef } from '@angular/core';
import { ControlValueAccessor, NG_VALUE_ACCESSOR } from '@angular/forms';

@Component({
  selector: 'app-star-rating',
  standalone: true,
  template: `
    @for (star of stars; track star) {
      <button
        type="button"
        [class.filled]="star <= value"
        [disabled]="disabled"
        (click)="select(star)"
        (blur)="onTouched()">
        {{ star <= value ? '\u2605' : '\u2606' }}
      </button>
    }
  `,
  styles: [`
    button {
      background: none;
      border: none;
      font-size: 1.5rem;
      cursor: pointer;
      padding: 0 2px;
    }
    button.filled { color: gold; }
    button:disabled { cursor: not-allowed; opacity: 0.5; }
  `],
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => StarRatingComponent),
      multi: true,
    },
  ],
})
export class StarRatingComponent implements ControlValueAccessor {
  stars = [1, 2, 3, 4, 5];
  value = 0;
  disabled = false;

  // Default no-op callbacks (prevents crashes before registration)
  private onChange: (value: number) => void = () => {};
  private onTouched: () => void = () => {};

  // Called by Angular when the FormControl value changes programmatically
  writeValue(value: number): void {
    // Guard against null/undefined
    this.value = value ?? 0;
  }

  // Angular passes us the callback to invoke on value changes
  registerOnChange(fn: (value: number) => void): void {
    this.onChange = fn;
  }

  // Angular passes us the callback to invoke on blur
  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  // Angular calls this when FormControl.disable() or enable() is called
  setDisabledState(isDisabled: boolean): void {
    this.disabled = isDisabled;
  }

  // Internal method: user clicks a star
  select(star: number): void {
    if (this.disabled) return;
    this.value = star;
    this.onChange(this.value); // Notify Angular of the change
  }
}
```

### Step 2: Use It with Reactive Forms

```typescript
@Component({
  selector: 'app-review-form',
  standalone: true,
  imports: [ReactiveFormsModule, StarRatingComponent],
  template: `
    <form [formGroup]="form">
      <label>Rating</label>
      <app-star-rating formControlName="rating" />

      <label>Comment</label>
      <textarea formControlName="comment"></textarea>
    </form>
  `,
})
export class ReviewFormComponent {
  form = inject(FormBuilder).group({
    rating: [0, Validators.required],
    comment: [''],
  });
}
```

### Step 3: Use It with Template-Driven Forms

```html
<!-- Works with ngModel too -->
<app-star-rating [(ngModel)]="userRating" name="rating" />
```

No additional configuration needed. The `NG_VALUE_ACCESSOR` provider makes it compatible with both form systems.

## The NG_VALUE_ACCESSOR Provider

The `NG_VALUE_ACCESSOR` token is how Angular discovers your `ControlValueAccessor` implementation. It is a multi-provider, meaning multiple accessors can be registered, and Angular picks the correct one based on specificity.

### Registration Pattern

```typescript
providers: [
  {
    provide: NG_VALUE_ACCESSOR,
    useExisting: forwardRef(() => MyComponent),
    multi: true,
  },
]
```

Why `forwardRef`? Because the provider is declared in the component's decorator metadata, which is evaluated before the class itself is defined. `forwardRef` defers the reference resolution.

Why `multi: true`? Angular maintains a list of all value accessors. It selects the most specific one. Built-in accessors (for `<input>`, `<select>`, etc.) are registered the same way.

### Accessor Selection Priority

Angular follows this priority when selecting which accessor to use:

1. **Custom accessor** (your component) -- highest priority
2. **Built-in accessor** (e.g., `DefaultValueAccessor` for text inputs)
3. Falls back to `DefaultValueAccessor` if nothing else matches

This means your custom accessor always wins when registered.

## Handling Complex Value Types

Custom controls often work with objects, arrays, or other non-primitive types.

### Date Picker Example

```typescript
@Component({
  selector: 'app-date-picker',
  standalone: true,
  template: `
    <input
      type="date"
      [value]="dateString"
      [disabled]="disabled"
      (input)="onDateInput($event)"
      (blur)="onTouched()" />
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => DatePickerComponent),
      multi: true,
    },
  ],
})
export class DatePickerComponent implements ControlValueAccessor {
  dateString = '';
  disabled = false;

  private onChange: (value: Date | null) => void = () => {};
  onTouched: () => void = () => {};

  // Convert Date object to string for the native input
  writeValue(value: Date | null): void {
    if (value instanceof Date && !isNaN(value.getTime())) {
      this.dateString = value.toISOString().split('T')[0];
    } else {
      this.dateString = '';
    }
  }

  registerOnChange(fn: (value: Date | null) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled = isDisabled;
  }

  onDateInput(event: Event): void {
    const input = event.target as HTMLInputElement;
    const date = input.value ? new Date(input.value + 'T00:00:00') : null;
    this.onChange(date);
  }
}
```

Usage:

```typescript
form = inject(FormBuilder).group({
  birthDate: [new Date(1990, 0, 1)],
});
```

The form stores `Date` objects while the component displays formatted strings. The CVA handles the conversion in both directions.

## Adding Built-in Validation

Custom controls can also implement the `Validator` or `AsyncValidator` interface to provide validation that is intrinsic to the component itself.

### Combining CVA with Validator

```typescript
import {
  ControlValueAccessor,
  NG_VALUE_ACCESSOR,
  NG_VALIDATORS,
  Validator,
  AbstractControl,
  ValidationErrors,
} from '@angular/forms';

@Component({
  selector: 'app-phone-input',
  standalone: true,
  template: `
    <div class="phone-input">
      <select
        [value]="countryCode"
        [disabled]="disabled"
        (change)="onCountryChange($event)">
        <option value="+1">US +1</option>
        <option value="+44">UK +44</option>
        <option value="+49">DE +49</option>
      </select>
      <input
        type="tel"
        [value]="number"
        [disabled]="disabled"
        (input)="onNumberInput($event)"
        (blur)="onTouched()" />
    </div>
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => PhoneInputComponent),
      multi: true,
    },
    {
      provide: NG_VALIDATORS,
      useExisting: forwardRef(() => PhoneInputComponent),
      multi: true,
    },
  ],
})
export class PhoneInputComponent implements ControlValueAccessor, Validator {
  countryCode = '+1';
  number = '';
  disabled = false;

  private onChange: (value: string) => void = () => {};
  onTouched: () => void = () => {};

  writeValue(value: string | null): void {
    if (!value) {
      this.countryCode = '+1';
      this.number = '';
      return;
    }
    // Parse "+1-5551234567" format
    const match = value.match(/^(\+\d+)-(.+)$/);
    if (match) {
      this.countryCode = match[1];
      this.number = match[2];
    }
  }

  registerOnChange(fn: (value: string) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled = isDisabled;
  }

  // Intrinsic validation: phone number format
  validate(control: AbstractControl): ValidationErrors | null {
    if (!this.number) return null; // Let required validator handle empty
    const phonePattern = /^\d{7,15}$/;
    if (!phonePattern.test(this.number)) {
      return { phoneFormat: { actual: this.number } };
    }
    return null;
  }

  onCountryChange(event: Event): void {
    this.countryCode = (event.target as HTMLSelectElement).value;
    this.emitValue();
  }

  onNumberInput(event: Event): void {
    this.number = (event.target as HTMLInputElement).value;
    this.emitValue();
  }

  private emitValue(): void {
    const fullNumber = `${this.countryCode}-${this.number}`;
    this.onChange(fullNumber);
  }
}
```

The `Validator` interface allows the component to enforce its own format rules. External validators (like `Validators.required`) still work on the parent `FormControl` as usual.

## Using Signals Inside a CVA

Even with the traditional `ControlValueAccessor`, you can use signals internally for reactivity. This is a common modern pattern for Angular 17+ before full signal forms adoption.

```typescript
@Component({
  selector: 'app-tag-input',
  standalone: true,
  imports: [FormsModule],
  template: `
    <div class="tag-container" [class.disabled]="disabled()">
      @for (tag of tags(); track tag) {
        <span class="tag">
          {{ tag }}
          <button type="button" (click)="removeTag(tag)">&times;</button>
        </span>
      }
      <input
        type="text"
        [(ngModel)]="inputValue"
        [disabled]="disabled()"
        (keydown.enter)="addTag()"
        (blur)="onTouched()"
        placeholder="Add tag..." />
    </div>
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => TagInputComponent),
      multi: true,
    },
  ],
})
export class TagInputComponent implements ControlValueAccessor {
  // Internal state as signals
  tags = signal<string[]>([]);
  disabled = signal(false);
  inputValue = '';

  private onChange: (value: string[]) => void = () => {};
  onTouched: () => void = () => {};

  writeValue(value: string[] | null): void {
    this.tags.set(value ?? []);
  }

  registerOnChange(fn: (value: string[]) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled.set(isDisabled);
  }

  addTag(): void {
    const tag = this.inputValue.trim();
    if (tag && !this.tags().includes(tag)) {
      this.tags.update(tags => [...tags, tag]);
      this.onChange(this.tags());
    }
    this.inputValue = '';
  }

  removeTag(tag: string): void {
    this.tags.update(tags => tags.filter(t => t !== tag));
    this.onChange(this.tags());
  }
}
```

This pattern gives you signal reactivity inside the component while maintaining full compatibility with the existing forms API.

## Injecting the NgControl

Sometimes you need access to the parent `NgControl` (the `FormControlDirective` or `FormControlName` that wraps your component). This is useful for reading validation state, errors, or the control name.

### The Self-Injection Pattern

```typescript
@Component({
  selector: 'app-custom-input',
  standalone: true,
  template: `
    <input
      [value]="value"
      [disabled]="disabled"
      (input)="onInput($event)"
      (blur)="onTouched()" />
    @if (ngControl?.control?.touched && ngControl?.control?.invalid) {
      <div class="error">
        @if (ngControl.control.hasError('required')) {
          This field is required.
        }
      </div>
    }
  `,
})
export class CustomInputComponent implements ControlValueAccessor {
  value = '';
  disabled = false;

  // Inject NgControl without causing circular dependency
  ngControl = inject(NgControl, { optional: true, self: true });

  private onChange: (value: string) => void = () => {};
  onTouched: () => void = () => {};

  constructor() {
    // Set the value accessor directly instead of using providers
    if (this.ngControl) {
      this.ngControl.valueAccessor = this;
    }
  }

  writeValue(value: string | null): void {
    this.value = value ?? '';
  }

  registerOnChange(fn: (value: string) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled = isDisabled;
  }

  onInput(event: Event): void {
    this.value = (event.target as HTMLInputElement).value;
    this.onChange(this.value);
  }
}
```

When you inject `NgControl` and set `valueAccessor` directly, you do NOT add the `NG_VALUE_ACCESSOR` provider. This avoids the circular dependency that would occur if you tried to inject `NgControl` while also registering via `NG_VALUE_ACCESSOR`.

### Trade-offs

| Approach | Pros | Cons |
|----------|------|------|
| `NG_VALUE_ACCESSOR` provider | Standard pattern, no constructor setup | Cannot inject `NgControl` without circular dependency |
| `NgControl` self-injection | Access to control state, errors, name | Requires manual `valueAccessor` assignment |

Choose the `NgControl` injection approach only when you genuinely need access to the parent control's state inside the component.

## Wrapping Third-Party Libraries

A common use case for CVA is wrapping third-party UI libraries so they integrate with Angular forms.

### Wrapping a Rich Text Editor

```typescript
@Component({
  selector: 'app-rich-editor',
  standalone: true,
  template: `<div #editorContainer></div>`,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => RichEditorComponent),
      multi: true,
    },
  ],
})
export class RichEditorComponent
  implements ControlValueAccessor, AfterViewInit, OnDestroy
{
  private editorContainer = viewChild.required<ElementRef>('editorContainer');
  private editor: ThirdPartyEditor | null = null;
  private pendingValue: string | null = null;

  private onChange: (value: string) => void = () => {};
  private onTouched: () => void = () => {};

  ngAfterViewInit(): void {
    // Initialize the third-party editor
    this.editor = new ThirdPartyEditor(
      this.editorContainer().nativeElement,
      {
        onChange: (content: string) => {
          this.onChange(content);
        },
        onBlur: () => {
          this.onTouched();
        },
      }
    );

    // Apply any value that arrived before the editor was ready
    if (this.pendingValue !== null) {
      this.editor.setContent(this.pendingValue);
      this.pendingValue = null;
    }
  }

  writeValue(value: string | null): void {
    if (this.editor) {
      this.editor.setContent(value ?? '');
    } else {
      // Editor not ready yet, store for later
      this.pendingValue = value;
    }
  }

  registerOnChange(fn: (value: string) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.editor?.setReadOnly(isDisabled);
  }

  ngOnDestroy(): void {
    this.editor?.destroy();
  }
}
```

Key pattern: `writeValue` can be called before `ngAfterViewInit`. The `pendingValue` pattern handles this timing mismatch.

## Signal Forms: FormValueControl (Angular 21)

Angular 21 introduces signal-based forms with the `FormValueControl` interface, which replaces `ControlValueAccessor` for signal forms.

### The FormValueControl Interface

```typescript
// For controls that edit a single value (text, number, date, select, etc.)
interface FormValueControl<T> {
  value: ModelSignal<T>;

  // Optional state signals from FormUiControl
  touched?: ModelSignal<boolean>;
  disabled?: InputSignal<boolean>;
  required?: InputSignal<boolean>;
  errors?: InputSignal<readonly ValidationError[]>;
  // ... and more
}
```

### The FormCheckboxControl Interface

```typescript
// For boolean toggle controls (checkbox, switch)
interface FormCheckboxControl {
  checked: ModelSignal<boolean>;
  // Same optional state signals
}
```

### Implementing FormValueControl

```typescript
import { Component, model, input } from '@angular/core';
import { FormValueControl, ValidationError } from '@angular/forms/signals';

@Component({
  selector: 'app-star-rating',
  standalone: true,
  template: `
    @for (star of stars; track star) {
      <button
        type="button"
        [class.filled]="star <= value()"
        [disabled]="disabled()"
        (click)="select(star)">
        {{ star <= value() ? '\u2605' : '\u2606' }}
      </button>
    }
  `,
})
export class StarRatingComponent implements FormValueControl<number> {
  // The only required property: a model signal for the value
  value = model(0);

  // Optional: receive disabled state from the form
  disabled = input(false);

  stars = [1, 2, 3, 4, 5];

  select(star: number): void {
    if (!this.disabled()) {
      this.value.set(star);
    }
  }
}
```

That is it. No `writeValue`. No `registerOnChange`. No `registerOnTouched`. No `NG_VALUE_ACCESSOR`. No `forwardRef`. Angular's `[formField]` directive detects the `value` model signal and handles synchronization automatically.

### Using with Signal Forms

```typescript
import { FormField } from '@angular/forms/signals';

@Component({
  selector: 'app-review',
  standalone: true,
  imports: [StarRatingComponent, FormField],
  template: `
    <app-star-rating [formField]="ratingField" />
  `,
})
export class ReviewComponent {
  private form = inject(FormBuilder).signalGroup({
    rating: 0,
  });

  ratingField = this.form.controls.rating;
}
```

### Comparison: CVA vs FormValueControl

| Feature | ControlValueAccessor | FormValueControl |
|---------|---------------------|------------------|
| Methods to implement | 3-4 | 0 (just a signal property) |
| Provider registration | `NG_VALUE_ACCESSOR` + `forwardRef` | None |
| Value sync | Callback-based | Signal-based (automatic) |
| Disabled state | `setDisabledState()` method | `disabled` input signal |
| Touch tracking | `registerOnTouched()` callback | `touched` model signal |
| Validation errors | External only (or `NG_VALIDATORS`) | `errors` input signal |
| Works with `formControl` | Yes | No (signal forms only) |
| Works with `[formField]` | No | Yes |
| Boilerplate | High | Minimal |

### Value Transformation with linkedSignal

When the display format differs from the stored value:

```typescript
@Component({
  selector: 'app-currency-input',
  standalone: true,
  template: `
    <input
      type="text"
      [value]="displayValue()"
      [disabled]="disabled()"
      (input)="onInput($event)"
      (blur)="touched.set(true)" />
  `,
})
export class CurrencyInputComponent implements FormValueControl<number> {
  value = model(0);
  touched = model(false);
  disabled = input(false);

  // Transform number to display string, re-syncs when value changes
  displayValue = linkedSignal(() =>
    this.value().toLocaleString('en-US', {
      style: 'currency',
      currency: 'USD',
    })
  );

  onInput(event: Event): void {
    const raw = (event.target as HTMLInputElement).value;
    const parsed = parseFloat(raw.replace(/[^0-9.-]/g, ''));
    if (!isNaN(parsed)) {
      this.value.set(parsed);
    }
  }
}
```

```typescript
describe('StarRatingComponent', () => {
  let component: StarRatingComponent;

  it('should handle null in writeValue', () => {

    expect(component.value).toBe(0);

  it('should notify onChange when user selects a star', () => {

```

## Common Mistakes

### Mistake 1: Calling onChange inside writeValue

This is the single most common CVA bug. It creates a feedback loop where external value changes trigger internal change notifications, doubling `valueChanges` emissions.

```typescript
// BAD: triggers onChange when Angular sets the value
writeValue(value: number): void {
  this.value = value;
  this.onChange(value); // DO NOT DO THIS
}
```

```typescript
// GOOD: only update the internal state
writeValue(value: number): void {
  this.value = value ?? 0;
  // Do NOT call onChange here
}
```

The rule: `onChange` should only be called in response to user interactions, never in response to programmatic value changes from the form.

### Mistake 2: Not handling null/undefined in writeValue

Your component does not control what values the parent form sends. `null`, `undefined`, or unexpected types can arrive at any time via `setValue`, `reset()`, or initial values.

```typescript
// BAD: assumes value is always a valid string array
writeValue(value: string[]): void {
  this.tags = value;
}

// GOOD: defensive handling
writeValue(value: string[] | null | undefined): void {
  this.tags = Array.isArray(value) ? value : [];
}
```

### Mistake 3: Using CVA for multi-value composite components

`ControlValueAccessor` binds a single value to a single `FormControl`. If your component manages multiple independent values (like a date range with separate start/end), do not use CVA.

```typescript
// BAD: cramming two values into one CVA
writeValue(value: { start: Date; end: Date }): void { ... }
// This prevents independent validation of start and end

// GOOD: accept FormControls as inputs
@Component({
  selector: 'app-date-range',
  standalone: true,
  imports: [ReactiveFormsModule],
  template: `
    <input type="date" [formControl]="startControl()" />
    <input type="date" [formControl]="endControl()" />
  `,
})
export class DateRangeComponent {
  startControl = input.required<FormControl<Date>>();
  endControl = input.required<FormControl<Date>>();
}
```

### Mistake 4: Forgetting to initialize callback defaults

If `registerOnChange` or `registerOnTouched` has not been called yet (e.g., during initialization), calling `this.onChange()` or `this.onTouched()` will throw.

```typescript
// BAD: no default
private onChange!: (value: string) => void;
private onTouched!: () => void;

// GOOD: safe defaults
private onChange: (value: string) => void = () => {};
private onTouched: () => void = () => {};
```

### Mistake 5: Not implementing setDisabledState

While technically optional, omitting `setDisabledState` means `FormControl.disable()` and `FormControl.enable()` have no effect on your custom control's UI. Users will call `disable()` and expect the control to look and behave as disabled.

```typescript
// Always implement it
setDisabledState(isDisabled: boolean): void {
  this.disabled = isDisabled;
}
```

### Mistake 6: Calling onTouched at the wrong time

`onTouched` should be called when the user finishes interacting with the control (blur), not on every value change.

```typescript
// BAD: marking as touched on every change
select(star: number): void {
  this.value = star;
  this.onChange(this.value);
  this.onTouched(); // Wrong place
}

// GOOD: mark as touched on blur
// In template: (blur)="onTouched()"
```

## Best Practices

### 1. Keep the Value Type Simple

The value your CVA exposes should be the simplest representation that makes sense. Transform complex internal state into clean external values.

```typescript
// Internal: array of objects with metadata
// External: array of strings (just the IDs)
private items: { id: string; label: string; selected: boolean }[] = [];

writeValue(value: string[]): void {
  this.items.forEach(item => {
    item.selected = value?.includes(item.id) ?? false;
  });
}

// onChange emits string[], not the full objects
private emitChange(): void {
  const selected = this.items.filter(i => i.selected).map(i => i.id);
  this.onChange(selected);
}
```

### 2. Handle the writeValue Timing Issue

`writeValue` can be called before `ngOnInit` or `ngAfterViewInit`. If your component depends on view children or async initialization, buffer the value.

```typescript
private pendingValue: string | null = null;
private initialized = false;

ngAfterViewInit(): void {
  this.initialized = true;
  if (this.pendingValue !== null) {
    this.applyValue(this.pendingValue);
    this.pendingValue = null;
  }
}

writeValue(value: string | null): void {
  if (this.initialized) {
    this.applyValue(value);
  } else {
    this.pendingValue = value;
  }
}
```

### 3. Implement OnPush Change Detection

CVA components should work with `OnPush` change detection. When using signals internally, this is handled automatically. With plain properties, you may need to inject `ChangeDetectorRef`.

```typescript
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  // ...
})
export class MyControlComponent implements ControlValueAccessor {
  private cdr = inject(ChangeDetectorRef);

  writeValue(value: string): void {
    this.value = value ?? '';
    this.cdr.markForCheck(); // Needed for OnPush
  }
}
```

### 4. Provide Accessibility

Custom controls must be keyboard accessible and screen-reader friendly.

```typescript
template: `
  <div
    role="slider"
    [attr.aria-valuenow]="value"
    [attr.aria-valuemin]="min"
    [attr.aria-valuemax]="max"
    [attr.aria-disabled]="disabled"
    tabindex="0"
    (keydown.arrowRight)="increment()"
    (keydown.arrowLeft)="decrement()"
    (blur)="onTouched()">
    ...
  </div>
`
```

### 5. Plan for Migration to Signal Forms

If you are building new CVA components today, design them so migration to `FormValueControl` is straightforward:

- Use signals for internal state
- Keep the value transformation logic separate from the CVA plumbing
- Avoid relying on `NgControl` injection when possible

```typescript
// Today: CVA with signals (easy to migrate later)
export class MyControl implements ControlValueAccessor {
  value = signal('');
  disabled = signal(false);

  writeValue(v: string): void { this.value.set(v ?? ''); }
  registerOnChange(fn: any): void { this.onChange = fn; }
  registerOnTouched(fn: any): void { this.onTouched = fn; }
  setDisabledState(d: boolean): void { this.disabled.set(d); }
}

// Future: FormValueControl (just remove the CVA boilerplate)
export class MyControl implements FormValueControl<string> {
  value = model('');
  disabled = input(false);
  // Done. That's it.
}
```

## Real-World Use Cases

### 1. Multi-Select Chip Input

A searchable multi-select that displays selections as chips:

```typescript
@Component({
  selector: 'app-chip-select',
  standalone: true,
  imports: [FormsModule],
  template: `
    <div class="chip-select" [class.disabled]="disabled()">
      <div class="chips">
        @for (item of selected(); track item.id) {
          <span class="chip">
            {{ item.label }}
            <button type="button" (click)="remove(item)">&times;</button>
          </span>
        }
      </div>
      <input
        type="text"
        [(ngModel)]="searchTerm"
        [disabled]="disabled()"
        (focus)="showDropdown.set(true)"
        (blur)="onBlur()"
        placeholder="Search..." />
      @if (showDropdown()) {
        <ul class="dropdown">
          @for (option of filteredOptions(); track option.id) {
            <li (mousedown)="add(option)">{{ option.label }}</li>
          }
          @empty {
            <li class="no-results">No results</li>
          }
        </ul>
      }
    </div>
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => ChipSelectComponent),
      multi: true,
    },
  ],
})
export class ChipSelectComponent implements ControlValueAccessor {
  // Configuration
  options = input<{ id: string; label: string }[]>([]);

  // Internal state
  selected = signal<{ id: string; label: string }[]>([]);
  disabled = signal(false);
  showDropdown = signal(false);
  searchTerm = '';

  private onChange: (value: string[]) => void = () => {};
  private onTouched: () => void = () => {};

  filteredOptions = computed(() => {
    const term = this.searchTerm.toLowerCase();
    const selectedIds = new Set(this.selected().map(s => s.id));
    return this.options().filter(
      o => !selectedIds.has(o.id) && o.label.toLowerCase().includes(term)
    );
  });

  writeValue(value: string[] | null): void {
    if (!value) {
      this.selected.set([]);
      return;
    }
    // Map IDs back to full objects
    const optionMap = new Map(this.options().map(o => [o.id, o]));
    this.selected.set(
      value.map(id => optionMap.get(id)).filter(Boolean) as any[]
    );
  }

  registerOnChange(fn: (value: string[]) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled.set(isDisabled);
  }

  add(option: { id: string; label: string }): void {
    this.selected.update(s => [...s, option]);
    this.searchTerm = '';
    this.emitChange();
  }

  remove(option: { id: string; label: string }): void {
    this.selected.update(s => s.filter(i => i.id !== option.id));
    this.emitChange();
  }

  onBlur(): void {
    // Delay to allow mousedown on dropdown items to fire first
    setTimeout(() => {
      this.showDropdown.set(false);
      this.onTouched();
    }, 200);
  }

  private emitChange(): void {
    this.onChange(this.selected().map(s => s.id));
  }
}
```

### 2. File Upload Control

```typescript
@Component({
  selector: 'app-file-upload',
  standalone: true,
  template: `
    <div
      class="drop-zone"
      [class.disabled]="disabled"
      [class.dragover]="isDragOver"
      (dragover)="onDragOver($event)"
      (dragleave)="isDragOver = false"
      (drop)="onDrop($event)"
      (click)="fileInput.click()">

      @if (file) {
        <span class="file-name">{{ file.name }} ({{ formatSize(file.size) }})</span>
        <button type="button" (click)="clear($event)">Remove</button>
      } @else {
        <span>Drag a file here or click to browse</span>
      }

      <input
        #fileInput
        type="file"
        [accept]="accept()"
        [disabled]="disabled"
        (change)="onFileSelected($event)"
        hidden />
    </div>
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => FileUploadComponent),
      multi: true,
    },
  ],
})
export class FileUploadComponent implements ControlValueAccessor {
  accept = input('');

  file: File | null = null;
  disabled = false;
  isDragOver = false;

  private onChange: (value: File | null) => void = () => {};
  private onTouched: () => void = () => {};

  writeValue(value: File | null): void {
    this.file = value;
  }

  registerOnChange(fn: (value: File | null) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled = isDisabled;
  }

  onFileSelected(event: Event): void {
    const input = event.target as HTMLInputElement;
    const file = input.files?.[0] ?? null;
    this.setFile(file);
  }

  onDragOver(event: DragEvent): void {
    event.preventDefault();
    this.isDragOver = true;
  }

  onDrop(event: DragEvent): void {
    event.preventDefault();
    this.isDragOver = false;
    const file = event.dataTransfer?.files[0] ?? null;
    this.setFile(file);
  }

  clear(event: Event): void {
    event.stopPropagation();
    this.setFile(null);
  }

  formatSize(bytes: number): string {
    if (bytes < 1024) return bytes + ' B';
    if (bytes < 1048576) return (bytes / 1024).toFixed(1) + ' KB';
    return (bytes / 1048576).toFixed(1) + ' MB';
  }

  private setFile(file: File | null): void {
    this.file = file;
    this.onChange(file);
    this.onTouched();
  }
}
```

### 3. Toggle Button Group

```typescript
@Component({
  selector: 'app-toggle-group',
  standalone: true,
  template: `
    <div class="toggle-group" role="radiogroup">
      @for (option of options(); track option.value) {
        <button
          type="button"
          role="radio"
          [attr.aria-checked]="option.value === value()"
          [class.active]="option.value === value()"
          [disabled]="disabled()"
          (click)="select(option.value)"
          (blur)="onTouched()">
          {{ option.label }}
        </button>
      }
    </div>
  `,
  providers: [
    {
      provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => ToggleGroupComponent),
      multi: true,
    },
  ],
})
export class ToggleGroupComponent implements ControlValueAccessor {
  options = input<{ value: string; label: string }[]>([]);

  value = signal<string>('');
  disabled = signal(false);

  private onChange: (value: string) => void = () => {};
  onTouched: () => void = () => {};

  writeValue(value: string | null): void {
    this.value.set(value ?? '');
  }

  registerOnChange(fn: (value: string) => void): void {
    this.onChange = fn;
  }

  registerOnTouched(fn: () => void): void {
    this.onTouched = fn;
  }

  setDisabledState(isDisabled: boolean): void {
    this.disabled.set(isDisabled);
  }

  select(value: string): void {
    if (this.disabled()) return;
    this.value.set(value);
    this.onChange(value);
  }
}
```

---

> **Key Takeaway:** `ControlValueAccessor` is the standard way to make custom components work with Angular forms. Implement `writeValue` for model-to-view, use `onChange` only for user-initiated changes, always handle null values defensively, and plan for signal-based `FormValueControl` as the future replacement with dramatically less boilerplate.
