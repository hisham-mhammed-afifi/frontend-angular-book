# 18.6 Filtering Operators

> Filtering operators selectively pass or block values from a source Observable based on conditions, timing, position, or uniqueness. They never transform values; they only decide which ones reach downstream subscribers.

---

## Overview

Filtering operators control what gets through a reactive pipeline. Unlike transformation operators that reshape values, filtering operators emit the same value they received or nothing at all.

| Category | Operators |
|----------|----------|
| Condition-based | `filter`, `first`, `last`, `single`, `elementAt` |
| Uniqueness | `distinct`, `distinctUntilChanged`, `distinctUntilKeyChanged` |
| Count-based | `take`, `takeLast`, `skip` |
| Signal-based | `takeUntil`, `takeWhile`, `skipUntil`, `skipWhile` |
| Time-based | `debounceTime`, `debounce`, `throttleTime`, `throttle`, `auditTime`, `audit`, `sampleTime`, `sample` |
| Empty-stream handling | `defaultIfEmpty`, `throwIfEmpty`, `ignoreElements` |
| Angular-specific | `takeUntilDestroyed` (from `@angular/core/rxjs-interop`) |

---

## Condition-Based Filtering

### `filter(predicate)`

The most basic filtering operator. Emits only values that satisfy a predicate function. Conceptually identical to `Array.prototype.filter`, but operates on values over time.

```typescript
import { of, filter } from 'rxjs';

of(1, 2, 3, 4, 5, 6).pipe(
  filter(x => x % 2 === 0)
).subscribe(v => console.log(v));
// 2, 4, 6
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ4‚îÄ‚îÄ5‚îÄ‚îÄ6‚îÄ‚îÄ|
filter(x => x % 2 === 0)
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ2‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ4‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ6‚îÄ‚îÄ|
```

**Signature:**

```typescript
filter<T>(predicate: (value: T, index: number) => boolean): MonoTypeOperatorFunction<T>
```

The `index` parameter tracks how many values have passed through (starting at 0), not how many passed the filter.

Angular use cases:

```typescript
// Only react to non-null emissions from a signal-backed Observable
import { toObservable } from '@angular/core/rxjs-interop';

toObservable(this.selectedUser).pipe(
  filter((user): user is User => user !== null)
);

// Filter router events to a specific type
import { NavigationEnd, Router } from '@angular/router';

inject(Router).events.pipe(
  filter((event): event is NavigationEnd => event instanceof NavigationEnd)
).subscribe(event => console.log(event.urlAfterRedirects));

// Only process valid form submissions
this.form.valueChanges.pipe(
  filter(() => this.form.valid)
);
```

**Type narrowing with filter:**

When you use a TypeScript type predicate as the filter callback, the downstream type narrows automatically:

```typescript
// Without type predicate: Observable<User | null>
source$.pipe(filter(user => user !== null));

// With type predicate: Observable<User>
source$.pipe(filter((user): user is User => user !== null));
```

This is critical in Angular when filtering nullish values from signal-based Observables or route params.

### `first(predicate?, defaultValue?)`

Emits only the first value (or the first value matching a predicate), then completes.

```typescript
import { of, first } from 'rxjs';

// No predicate: emit the first value
of(1, 2, 3).pipe(
  first()
).subscribe(v => console.log(v));
// 1

// With predicate: emit the first matching value
of(1, 2, 3, 4).pipe(
  first(x => x > 2)
).subscribe(v => console.log(v));
// 3
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ4‚îÄ‚îÄ|
first(x => x > 2)
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ3|
                   ^ completes immediately after emitting
```

**Signature:**

```typescript
first<T, D>(
  predicate?: (value: T, index: number) => boolean,
  defaultValue?: D
): OperatorFunction<T, T | D>
```

**Error behavior:**

- If the source completes without emitting a matching value and no `defaultValue` is provided, `first()` throws an `EmptyError`.
- If `defaultValue` is provided, that value is emitted instead.

```typescript
import { EMPTY, first } from 'rxjs';

// Throws EmptyError
EMPTY.pipe(first()).subscribe({
  error: err => console.log(err.name) // "EmptyError"
});

// Emits default value
EMPTY.pipe(first(undefined, 'fallback')).subscribe(v => console.log(v));
// "fallback"
```

**`first()` vs `take(1)`:**

| Behavior | `first()` | `take(1)` |
|----------|-----------|-----------|
| Empty source | Throws `EmptyError` | Completes silently |
| Accepts predicate | Yes | No |
| Accepts default value | Yes | No |

Use `first()` when an empty stream is a bug. Use `take(1)` when an empty stream is acceptable.

### `last(predicate?, defaultValue?)`

Emits only the last value (or the last value matching a predicate) before the source completes.

```typescript
import { of, last } from 'rxjs';

of(1, 2, 3).pipe(
  last()
).subscribe(v => console.log(v));
// 3

of(1, 2, 3, 4, 5).pipe(
  last(x => x < 4)
).subscribe(v => console.log(v));
// 3
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ|
last()
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ3|
                    ^ waits for source to complete
```

**Signature:**

```typescript
last<T, D>(
  predicate?: (value: T, index: number) => boolean,
  defaultValue?: D
): OperatorFunction<T, T | D>
```

**Critical gotcha:** `last()` must wait for the source to complete. If the source never completes (e.g., a Subject that stays open, an interval that runs forever), `last()` will never emit.

```typescript
// Bad: interval never completes, last() hangs forever
import { interval, last } from 'rxjs';

interval(1000).pipe(last()); // never emits, never completes
```

**Error behavior:** Same as `first()`. Throws `EmptyError` on empty source unless `defaultValue` is provided.

### `single(predicate?)`

Emits the sole value from the source. Throws errors if zero or more than one value is emitted.

```typescript
import { of, single } from 'rxjs';

// Exactly one value: works
of(42).pipe(single()).subscribe(v => console.log(v));
// 42

// Multiple values: throws SequenceError
of(1, 2).pipe(single()).subscribe({
  error: err => console.log(err.name) // "SequenceError"
});

// Empty source: throws EmptyError
import { EMPTY } from 'rxjs';
EMPTY.pipe(single()).subscribe({
  error: err => console.log(err.name) // "EmptyError"
});
```

```
Source:  ‚îÄ‚îÄ42‚îÄ‚îÄ|
single()
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ42|
               ^ waits for completion to confirm only one value

Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ|
single()
Output:  ‚îÄ‚îÄ‚îÄ‚îÄX
             ^ error on second value
```

**Signature:**

```typescript
single<T>(predicate?: (value: T, index: number) => boolean): MonoTypeOperatorFunction<T>
```

**When to use:** Use `single()` as a runtime assertion that exactly one value comes through. It is stricter than `first()` or `take(1)` because it catches cases where you expected one value but got more.

Angular example:

```typescript
// Assert that a route resolver returns exactly one response
this.route.data.pipe(
  map(data => data['user']),
  single()
);
```

## Uniqueness Operators

### `distinct(keySelector?, flushes?)`

Emits values that have never been emitted before by the source. Uses a `Set` internally to track all previously seen values.

```typescript
import { of, distinct } from 'rxjs';

of(1, 2, 1, 3, 2, 4, 3).pipe(
  distinct()
).subscribe(v => console.log(v));
// 1, 2, 3, 4
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ1‚îÄ‚îÄ3‚îÄ‚îÄ2‚îÄ‚îÄ4‚îÄ‚îÄ3‚îÄ‚îÄ|
distinct()
Output:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ3‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ4‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ|
```

**Signature:**

```typescript
distinct<T, K>(
  keySelector?: (value: T) => K,
  flushes?: Observable<any>
): MonoTypeOperatorFunction<T>
```

**With keySelector:**

```typescript
interface User { id: number; name: string; }

of(
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },
  { id: 1, name: 'Alice Updated' },
  { id: 3, name: 'Charlie' }
).pipe(
  distinct(user => user.id)
).subscribe(v => console.log(v.name));
// "Alice", "Bob", "Charlie"
// "Alice Updated" is dropped because id=1 was already seen
```

**Memory leak warning:** `distinct()` stores every unique key it has ever seen. On long-lived streams this buffer grows without bound. Use the `flushes` parameter to periodically clear the internal Set:

```typescript
import { fromEvent, distinct, interval } from 'rxjs';

// Flush the internal set every 30 seconds
fromEvent<KeyboardEvent>(document, 'keydown').pipe(
  distinct(e => e.key, interval(30_000))
);
```

**When to use `distinct` vs `distinctUntilChanged`:**

| Operator | Compares against | Memory |
|----------|-----------------|--------|
| `distinct` | All previously seen values | Grows unbounded (use `flushes`) |
| `distinctUntilChanged` | Only the immediately previous value | Constant (stores 1 value) |

Use `distinct` when duplicates can appear at any position. Use `distinctUntilChanged` when only consecutive duplicates matter.

### `distinctUntilChanged(comparator?, keySelector?)`

Emits a value only when it differs from the immediately preceding value. Uses `===` strict equality by default.

```typescript
import { of, distinctUntilChanged } from 'rxjs';

of(1, 1, 2, 2, 3, 1, 1).pipe(
  distinctUntilChanged()
).subscribe(v => console.log(v));
// 1, 2, 3, 1
// Note: the final 1 IS emitted because it differs from the preceding 3
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ1‚îÄ‚îÄ1‚îÄ‚îÄ|
distinctUntilChanged()
Output:  ‚îÄ‚îÄ1‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ2‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ3‚îÄ‚îÄ1‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ|
```

**Signature:**

```typescript
distinctUntilChanged<T, K>(
  comparator?: (previous: K, current: K) => boolean,
  keySelector?: (value: T) => K
): MonoTypeOperatorFunction<T>
```

**The comparator returns `true` to SUPPRESS** the value (meaning "they are the same"). This is the most common source of bugs with this operator.

```typescript
// BAD: Inverted logic -- this SUPPRESSES when objects are DIFFERENT
source$.pipe(
  distinctUntilChanged((prev, curr) => prev.id !== curr.id) // WRONG
);

// GOOD: Return true when they are the same
source$.pipe(
  distinctUntilChanged((prev, curr) => prev.id === curr.id)
);
```

**Custom comparator for objects:**

Objects compared with `===` are compared by reference, not by value. Without a custom comparator, every new object reference passes through even if properties are identical:

```typescript
// BAD: Every emission passes through because each object is a new reference
this.store.select(selectUser).pipe(
  distinctUntilChanged() // useless for objects
);

// GOOD: Compare by value using JSON
this.store.select(selectUser).pipe(
  distinctUntilChanged((a, b) => JSON.stringify(a) === JSON.stringify(b))
);

// BETTER: Compare specific properties
this.store.select(selectUser).pipe(
  distinctUntilChanged((a, b) => a.id === b.id && a.name === b.name)
);
```

**With keySelector:**

The `keySelector` parameter extracts a comparison key before applying the comparator:

```typescript
interface User { id: number; name: string; lastLogin: Date; }

source$.pipe(
  // Only emit when the user's name changes, ignoring lastLogin updates
  distinctUntilChanged(undefined, user => user.name)
);
```

Angular search box pattern:

```typescript
@Component({
  template: `<input [formControl]="searchControl">`,
  // ...
})
export class SearchComponent {
  private destroyRef = inject(DestroyRef);
  searchControl = new FormControl('');

  constructor() {
    this.searchControl.valueChanges.pipe(
      debounceTime(300),
      distinctUntilChanged(), // skip if user typed same value again
      filter(term => term !== null && term.length >= 2),
      switchMap(term => this.searchService.search(term)),
      takeUntilDestroyed(this.destroyRef)
    ).subscribe(results => this.results.set(results));
  }
}
```

### `distinctUntilKeyChanged(key, comparator?)`

Shorthand for `distinctUntilChanged` when comparing a single object property.

```typescript
import { of, distinctUntilKeyChanged } from 'rxjs';

of(
  { name: 'Alice', status: 'online' },
  { name: 'Alice', status: 'away' },
  { name: 'Alice', status: 'away' },
  { name: 'Alice', status: 'offline' }
).pipe(
  distinctUntilKeyChanged('status')
).subscribe(v => console.log(v.status));
// "online", "away", "offline"
```

```
Source:  ‚îÄ‚îÄ{online}‚îÄ‚îÄ{away}‚îÄ‚îÄ{away}‚îÄ‚îÄ{offline}‚îÄ‚îÄ|
distinctUntilKeyChanged('status')
Output:  ‚îÄ‚îÄ{online}‚îÄ‚îÄ{away}‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ{offline}‚îÄ‚îÄ|
```

**Signature:**

```typescript
distinctUntilKeyChanged<T, K extends keyof T>(
  key: K,
  comparator?: (x: T[K], y: T[K]) => boolean
): MonoTypeOperatorFunction<T>
```

Equivalent to:

```typescript
distinctUntilChanged((prev, curr) => prev[key] === curr[key])
```

**With custom comparator:**

```typescript
// Case-insensitive comparison on the 'name' key
source$.pipe(
  distinctUntilKeyChanged('name', (a, b) => a.toLowerCase() === b.toLowerCase())
);
```

Angular use case with WebSocket data:

```typescript
// Only update UI when user status actually changes
this.websocketService.userUpdates$.pipe(
  distinctUntilKeyChanged('status'),
  takeUntilDestroyed()
).subscribe(user => this.userStatus.set(user.status));
```

## Count-Based Operators

### `take(count)`

Emits only the first `count` values, then completes.

```typescript
import { interval, take } from 'rxjs';

interval(1000).pipe(
  take(3)
).subscribe(v => console.log(v));
// 0 (at 1s), 1 (at 2s), 2 (at 3s), then completes
```

```
Source:  ‚îÄ‚îÄ0‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ4‚îÄ‚îÄ5‚îÄ‚îÄ...
take(3)
Output:  ‚îÄ‚îÄ0‚îÄ‚îÄ1‚îÄ‚îÄ2|
                  ^ completes after 3rd value
```

**Signature:**

```typescript
take<T>(count: number): MonoTypeOperatorFunction<T>
```

`take(0)` completes immediately without subscribing to the source. `take(Infinity)` passes everything through.

Angular use case:

```typescript
// One-shot HTTP-like behavior from a long-lived source
this.store.select(selectConfig).pipe(
  take(1) // grab current value, don't stay subscribed
);
```

---

### `takeLast(count)`

Waits for the source to complete, then emits the last `count` values.

```typescript
import { of, takeLast } from 'rxjs';

of(1, 2, 3, 4, 5).pipe(
  takeLast(2)
).subscribe(v => console.log(v));
// 4, 5
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ4‚îÄ‚îÄ5‚îÄ‚îÄ|
takeLast(2)
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ4‚îÄ5|
                          ^ all emitted at once when source completes
```

**Signature:**

```typescript
takeLast<T>(count: number): MonoTypeOperatorFunction<T>
```

**Gotcha:** `takeLast` buffers internally and requires the source to complete. On a never-completing Observable, it never emits and holds buffered values in memory.

## Signal-Based Operators

### `takeUntil(notifier)`

Emits values until a notifier Observable emits, then completes.

```typescript
import { interval, takeUntil, Subject } from 'rxjs';

const stop$ = new Subject<void>();

interval(500).pipe(
  takeUntil(stop$)
).subscribe(v => console.log(v));

// After 2.5 seconds:
setTimeout(() => stop$.next(), 2500);
// Logs: 0, 1, 2, 3 then completes
```

```
Source:   ‚îÄ‚îÄ0‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ4‚îÄ‚îÄ5‚îÄ‚îÄ...
Notifier: ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄX
takeUntil(notifier)
Output:   ‚îÄ‚îÄ0‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ|
                        ^ completes when notifier emits
```

**Signature:**

```typescript
takeUntil<T>(notifier: Observable<any>): MonoTypeOperatorFunction<T>
```

**Operator ordering gotcha:** `takeUntil` should typically be the LAST operator in a pipe chain. If placed before operators that add subscriptions (like `switchMap`), those inner subscriptions may not be cleaned up:

```typescript
// BAD: switchMap inner subscriptions outlive takeUntil
source$.pipe(
  takeUntil(destroy$),    // completes outer, but switchMap's inner may live
  switchMap(id => this.http.get(`/api/${id}`))
);

// GOOD: takeUntil at the end of the chain
source$.pipe(
  switchMap(id => this.http.get(`/api/${id}`)),
  takeUntil(destroy$)
);
```

The Angular-specific `takeUntilDestroyed` is now preferred over the manual `takeUntil` + `Subject` pattern. See the [Angular-Specific Section](#takeuntildestroyed) below.

---

### `takeWhile(predicate, inclusive?)`

Emits values as long as the predicate returns true. Completes the moment the predicate returns false.

```typescript
import { of, takeWhile } from 'rxjs';

of(1, 2, 3, 9, 2, 1).pipe(
  takeWhile(x => x < 5)
).subscribe(v => console.log(v));
// 1, 2, 3
// The 9 fails the predicate, so the stream completes. The trailing 2 and 1 are never seen.
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ9‚îÄ‚îÄ2‚îÄ‚îÄ1‚îÄ‚îÄ|
takeWhile(x => x < 5)
Output:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3|
                  ^ completes when 9 fails the predicate
```

**Inclusive mode:** When `inclusive` is `true`, the failing value is also emitted before completing:

```typescript
of(1, 2, 3, 9, 2, 1).pipe(
  takeWhile(x => x < 5, true) // inclusive
).subscribe(v => console.log(v));
// 1, 2, 3, 9
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ9‚îÄ‚îÄ2‚îÄ‚îÄ1‚îÄ‚îÄ|
takeWhile(x => x < 5, true)
Output:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ9|
                     ^ the failing value is included
```

**Signature:**

```typescript
takeWhile<T>(
  predicate: (value: T, index: number) => boolean,
  inclusive?: boolean
): MonoTypeOperatorFunction<T>
```

**`takeWhile` vs `filter`:**

| Behavior | `takeWhile` | `filter` |
|----------|-------------|----------|
| Failing value | Completes the stream | Silently skips the value |
| Later matching values | Never reached | Still emitted |

`takeWhile` is a one-way gate that permanently closes. `filter` is a checkpoint that evaluates every value independently.

```typescript
).subscribe(v => console.log(v));
// (skips 0, 1, 2) then logs: 3, 4, 5, ...
```

```typescript

```

### `skipWhile(predicate)`

Skips values as long as the predicate returns true. Once the predicate returns false, all subsequent values pass through regardless.

```typescript
import { of, skipWhile } from 'rxjs';

of(1, 2, 3, 9, 2, 1).pipe(
  skipWhile(x => x < 5)
).subscribe(v => console.log(v));
// 9, 2, 1
// Once x=9 fails the predicate, everything passes including 2 and 1
```

```
Source:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ3‚îÄ‚îÄ9‚îÄ‚îÄ2‚îÄ‚îÄ1‚îÄ‚îÄ|
skipWhile(x => x < 5)
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ9‚îÄ‚îÄ2‚îÄ‚îÄ1‚îÄ‚îÄ|
                     ^ gate opens, stays open
```

**Signature:**

```typescript
skipWhile<T>(predicate: (value: T, index: number) => boolean): MonoTypeOperatorFunction<T>
```

**Gotcha:** `skipWhile` does NOT re-evaluate after opening the gate. Once the predicate returns false, all subsequent values pass even if they would have satisfied the original predicate.

## Time-Based Filtering Operators

These four operator pairs control the rate at which values flow through a pipeline. They all work with time windows but differ in when they sample.

### Comparison Table

| Operator | When it emits | Resets timer on new value? | Emits on completion? |
|----------|--------------|--------------------------|---------------------|
| `debounceTime` | After silence period | Yes | Yes |
| `throttleTime` (default) | First value in window | No | N/A |

| `sampleTime` | At fixed intervals | No | No |

### `debounceTime(dueTime)`

Emits a value only after `dueTime` milliseconds of silence from the source. Each new emission resets the timer.

```typescript
import { fromEvent, debounceTime, map } from 'rxjs';

fromEvent(document.getElementById('search')!, 'input').pipe(
  map((e: Event) => (e.target as HTMLInputElement).value),
  debounceTime(300)
).subscribe(term => console.log('Search:', term));
// Only fires 300ms after the user stops typing
```

**Signature:**

```typescript
debounceTime<T>(dueTime: number, scheduler?: SchedulerLike): MonoTypeOperatorFunction<T>
```

**Key behavior:** If the source completes while a debounced value is pending, that value IS emitted before the completion notification.

Angular search pattern:

```typescript
@Component({
  template: `<input [formControl]="search">`,
  // ...
})
export class SearchComponent {
  private destroyRef = inject(DestroyRef);
  private searchService = inject(SearchService);
  search = new FormControl('');
  results = signal<SearchResult[]>([]);

  constructor() {
    this.search.valueChanges.pipe(
      debounceTime(300),
      distinctUntilChanged(),
      filter((term): term is string => term !== null && term.length >= 2),
      switchMap(term => this.searchService.search(term)),
      takeUntilDestroyed()
    ).subscribe(results => this.results.set(results));
  }
}
```

### `debounce(durationSelector)`

Like `debounceTime`, but the silence duration is determined by an Observable factory instead of a fixed time. This allows dynamic debounce durations.

```typescript
import { of, debounce, timer } from 'rxjs';

// Debounce longer for longer strings
source$.pipe(
  debounce(value =>
    timer(value.length > 5 ? 500 : 200)
  )
);
```

**Signature:**

```typescript
debounce<T>(durationSelector: (value: T) => ObservableInput<any>): MonoTypeOperatorFunction<T>
```

The timer resets whenever the source emits. The pending value is emitted when the duration Observable emits or completes.

### `throttleTime(duration, scheduler?, config?)`

Lets the first value through, then opens a time window and blocks subsequent values until the window closes.

```typescript
import { fromEvent, throttleTime } from 'rxjs';

// Limit button clicks to once per second
fromEvent(document.getElementById('btn')!, 'click').pipe(
  throttleTime(1000)
).subscribe(() => console.log('Clicked'));
```

**Signature:**

```typescript
throttleTime<T>(
  duration: number,
  scheduler?: SchedulerLike,
  config?: { leading: boolean; trailing: boolean }
): MonoTypeOperatorFunction<T>
```

**Leading and trailing configuration:**

The default config is `{ leading: true, trailing: false }`, meaning only the first value in each window is emitted.

```typescript
// Default: leading only
source$.pipe(throttleTime(1000));
// Emits: first value of each window

// Trailing only
source$.pipe(throttleTime(1000, asyncScheduler, { leading: false, trailing: true }));
// Emits: last value of each window (similar to auditTime)

// Both leading and trailing
source$.pipe(throttleTime(1000, asyncScheduler, { leading: true, trailing: true }));
// Emits: first AND last value of each window
```

### `auditTime(duration)`

When the source emits, starts a timer. When the timer expires, emits the most recent value from the source. Then waits for the next source emission to start the cycle again.

```typescript
import { fromEvent, auditTime } from 'rxjs';

fromEvent(window, 'mousemove').pipe(
  auditTime(200)
).subscribe(e => console.log(e.clientX, e.clientY));
// Emits latest position every 200ms while moving
```

```

auditTime(5)
Output:  ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄc‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄf‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ|
               ^ emits most recent when window closes
```

**Signature:**

```typescript
auditTime<T>(duration: number, scheduler?: SchedulerLike): MonoTypeOperatorFunction<T>
```

**auditTime vs trailing throttleTime:** They behave nearly identically, with one critical difference: if the source completes while a window is open, `throttleTime({ trailing: true })` still emits the pending value, but `auditTime` discards it.

Angular use case:

```typescript
// Track component resize at a reasonable rate
this.resizeObserver$.pipe(
  auditTime(150),
  takeUntilDestroyed()
).subscribe(entry => this.onResize(entry));
```

### `audit(durationSelector)`

Like `auditTime`, but the window duration is determined by an Observable factory.

```typescript
import { audit, timer } from 'rxjs';

source$.pipe(
  audit(() => timer(500))
);
```

**Signature:**

```typescript
audit<T>(durationSelector: (value: T) => ObservableInput<any>): MonoTypeOperatorFunction<T>
```

### `sampleTime(period)`

Emits the most recent value from the source at fixed, regular intervals. Unlike `auditTime`, the intervals are not triggered by source emissions; they tick independently.

```typescript
import { fromEvent, sampleTime } from 'rxjs';

fromEvent(window, 'mousemove').pipe(
  sampleTime(1000)
).subscribe(e => console.log(e.clientX));
// Logs latest mouse position every 1 second (only if mouse moved)
```

```
Source:  ‚îÄ‚îÄa‚îÄb‚îÄc‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄd‚îÄe‚îÄ‚îÄf‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ|
sampleTime(5)
         ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄc‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄe‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ|
              ^ fixed intervals, emit most recent if any
```

**Signature:**

```typescript
sampleTime<T>(period: number, scheduler?: SchedulerLike): MonoTypeOperatorFunction<T>
```

**Key difference from auditTime:** `sampleTime` ticks on a clock regardless of source activity. If the source has not emitted since the last tick, nothing is emitted at that tick. `auditTime` only starts ticking after a source emission.

### `sample(notifier)`

Emits the most recent source value whenever the notifier Observable emits.

```typescript
import { interval, sample, fromEvent } from 'rxjs';

// Emit current counter value whenever user clicks
interval(500).pipe(
  sample(fromEvent(document, 'click'))
).subscribe(v => console.log(v));
```

**Signature:**

```typescript
sample<T>(notifier: Observable<any>): MonoTypeOperatorFunction<T>
```

### Choosing the Right Time-Based Operator

| Scenario | Operator |
|----------|----------|
| Search input (wait for user to stop typing) | `debounceTime` |
| Button anti-spam (one click per interval) | `throttleTime` |
| Scroll event handler (latest position) | `auditTime` or `throttleTime({ trailing: true })` |
| Regular polling/sampling (fixed intervals) | `sampleTime` |
| Dynamic delay based on value | `debounce` / `throttle` / `audit` |

## Empty-Stream Handling

### `defaultIfEmpty(defaultValue)`

If the source completes without emitting any values, emit the provided default value.

```typescript
import { EMPTY, defaultIfEmpty } from 'rxjs';

EMPTY.pipe(
  defaultIfEmpty('no data')
).subscribe(v => console.log(v));
// "no data"

// If source emits, default is ignored
import { of } from 'rxjs';

of(1, 2, 3).pipe(
  defaultIfEmpty(0)
).subscribe(v => console.log(v));
// 1, 2, 3
```

```
defaultIfEmpty('fallback')
Output:  ‚îÄ‚îÄ1‚îÄ‚îÄ2‚îÄ‚îÄ|
```

**Signature:**

```typescript
defaultIfEmpty<T, R>(defaultValue: R): OperatorFunction<T, T | R>
```

Angular use case:

```typescript

this.items$.pipe(
  filter(item => item.category === selectedCategory),
  toArray(),
  defaultIfEmpty([])
);
```

### `throwIfEmpty(errorFactory?)`

If the source completes without emitting any values, throw an error.

```typescript
import { EMPTY, throwIfEmpty } from 'rxjs';

EMPTY.pipe(
  throwIfEmpty(() => new Error('Expected at least one value'))
).subscribe({
  error: err => console.log(err.message) // "Expected at least one value"
});
```

**Signature:**

```typescript
throwIfEmpty<T>(errorFactory?: () => any): MonoTypeOperatorFunction<T>
```

If no `errorFactory` is provided, throws an `EmptyError`.

Angular use case:

```typescript
// Ensure a resolver returns data
this.http.get<User[]>('/api/users').pipe(
  mergeMap(users => users), // flatten array to individual emissions
  filter(user => user.role === 'admin'),
  throwIfEmpty(() => new Error('No admin users found'))
);
```

### `ignoreElements()`

Ignores all `next` emissions from the source. Only forwards `complete` and `error` notifications.

```typescript
import { of, ignoreElements } from 'rxjs';

of(1, 2, 3).pipe(
  ignoreElements()
).subscribe({
  next: v => console.log('next:', v),       // never called
  complete: () => console.log('complete!')   // "complete!"
});
```

**Signature:**

```typescript

```

Angular use case:

```typescript
  endWith('done')
```

## Angular-Specific: `takeUntilDestroyed` {#takeuntildestroyed}

The `takeUntilDestroyed` operator from `@angular/core/rxjs-interop` automatically completes an Observable when the calling component, directive, or service is destroyed. It replaces the manual `takeUntil` + `Subject` + `ngOnDestroy` pattern entirely.

### Import

```typescript
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
```

### Basic Usage (In Injection Context)

When called inside a constructor or field initializer (injection context), no arguments are needed:

```typescript
@Component({ /* ... */ })
export class DashboardComponent {
  private dataService = inject(DataService);
  data = signal<DashboardData | null>(null);

  constructor() {
    this.dataService.liveData$.pipe(
      takeUntilDestroyed() // no argument needed in constructor
    ).subscribe(data => this.data.set(data));
  }
}
```

### Usage Outside Injection Context

Outside constructors (e.g., in `ngOnInit`, event handlers, or methods), you must explicitly pass a `DestroyRef`:

```typescript
@Component({ /* ... */ })
export class DashboardComponent implements OnInit {
  private destroyRef = inject(DestroyRef);
  private dataService = inject(DataService);

  ngOnInit() {
    this.dataService.liveData$.pipe(
      takeUntilDestroyed(this.destroyRef) // must pass DestroyRef
    ).subscribe(data => this.updateChart(data));
  }
}
```

**Common mistake:** Calling `takeUntilDestroyed()` without arguments outside an injection context throws a runtime error: `NG0203: takeUntilDestroyed() can only be used within an injection context`.

### Field Initializer Pattern

Field initializers run in injection context, so you can set up observables as class fields:

```typescript
@Component({ /* ... */ })
export class UserComponent {
  private userService = inject(UserService);

  // Field initializer IS an injection context
  private user$ = this.userService.currentUser$.pipe(
    takeUntilDestroyed() // works without arguments
  );
}
```

### Cross-Component Lifecycle

You can tie a subscription to a different component's lifecycle by passing its `DestroyRef`:

```typescript
@Component({ /* ... */ })
export class ParentComponent {
  private dataService = inject(DataService);

  setupChildTracking(childDestroyRef: DestroyRef) {
    // Subscription lives as long as the child component
    this.dataService.track$.pipe(
      takeUntilDestroyed(childDestroyRef)
    ).subscribe(data => this.handleChildData(data));
  }
}
```

### `takeUntilDestroyed` vs Legacy Patterns

```typescript
// LEGACY: Manual Subject + ngOnDestroy (verbose, error-prone)
@Component({ /* ... */ })
export class OldComponent implements OnDestroy {
  private destroy$ = new Subject<void>();

  ngOnInit() {
    this.source$.pipe(
      takeUntil(this.destroy$)
    ).subscribe();
  }

  ngOnDestroy() {
    this.destroy$.next();
    this.destroy$.complete();
  }
}

// MODERN: takeUntilDestroyed (concise, safe)
@Component({ /* ... */ })
export class ModernComponent {
  constructor() {
    this.source$.pipe(
      takeUntilDestroyed()
    ).subscribe();
  }
}
```

### When NOT to Use `takeUntilDestroyed`

- **HTTP requests** (`HttpClient`): They complete naturally after one response. No unsubscription needed unless you want cancellation.
- **Async pipe**: The `async` pipe manages its own subscription lifecycle. Do not combine with `takeUntilDestroyed`.
- **`toSignal()`**: Handles unsubscription internally. Do not add `takeUntilDestroyed` to the source.

## Common Mistakes and Gotchas

### 1. `distinctUntilChanged` with Object References

```typescript
// BUG: New object reference on every emission, so nothing is filtered
this.http.get<Config>('/api/config').pipe(
  distinctUntilChanged() // always passes because reference !== reference
);

// FIX: Custom comparator
this.http.get<Config>('/api/config').pipe(
  distinctUntilChanged((a, b) => a.version === b.version)
);
```

### 2. `takeUntil` Operator Ordering

```typescript
// BUG: Inner observable from switchMap can outlive the takeUntil
source$.pipe(
  takeUntil(destroy$),
  switchMap(val => longRunningRequest(val)) // may not be cancelled
);

// FIX: takeUntil last in the chain
source$.pipe(
  switchMap(val => longRunningRequest(val)),
  takeUntil(destroy$)
);
```

### 3. `takeWhile` vs `filter` Confusion

```typescript
// WRONG: Using takeWhile when you mean filter
// This completes the stream permanently on first invalid value
source$.pipe(
  takeWhile(item => item.isActive) // stream DIES when first inactive appears
);

// RIGHT: Use filter if inactive items can appear between active ones
source$.pipe(
  filter(item => item.isActive) // inactive items are skipped, active ones pass
);
```

### 4. Forgetting the Inclusive Flag on `takeWhile`

```typescript
// Counting down: 10, 9, 8, ..., 1
// MISS: The value that triggers completion (0) is lost
countdown$.pipe(
  takeWhile(val => val > 0) // never sees 0
);

// FIX: Use inclusive to capture the boundary value
countdown$.pipe(
  takeWhile(val => val > 0, true) // includes 0
);
```

### 5. `first()` on Empty Observables

```typescript
// BUG: Throws EmptyError if no items match
this.items$.pipe(
  first(item => item.id === targetId) // crashes if no match
);

// FIX: Provide a default value
this.items$.pipe(
  first(item => item.id === targetId, null) // returns null if no match
);
```

### 6. `debounceTime` vs `auditTime` on Completion

`debounceTime` DOES emit a pending value when the source completes. `auditTime` does NOT. This catches people off guard:

```typescript
// auditTime discards the pending value on source completion
of(1, 2, 3).pipe(
  auditTime(1000)
).subscribe(v => console.log(v));
// May log nothing if source completes synchronously before the window
```

### 7. `distinctUntilChanged` Comparator Return Value

```typescript
// BUG: Comparator returns true when values are DIFFERENT
// (suppresses when it should emit)
source$.pipe(
  distinctUntilChanged((prev, curr) => prev.id !== curr.id) // INVERTED
);

// FIX: Return true when values are THE SAME (to suppress)
source$.pipe(
  distinctUntilChanged((prev, curr) => prev.id === curr.id)
);
```

### 8. `filter(Boolean)` Loses Type Information

```typescript
// BUG: Type remains string | null | undefined
source$.pipe(
  filter(Boolean)
);

// FIX: Use a type predicate
source$.pipe(
  filter((v): v is string => Boolean(v))
);
```

### 9. Using `distinct` on Long-Lived Streams Without Flushes

```typescript
// BUG: Internal Set grows forever, potential memory leak
websocket$.pipe(
  distinct(msg => msg.id) // stores every ID seen, forever
);

// FIX: Flush periodically
websocket$.pipe(
  distinct(msg => msg.id, interval(60_000)) // clear every minute
);
```

### 10. `takeUntilDestroyed()` Without Arguments Outside Constructor

```typescript
// BUG: Runtime error NG0203
@Component({ /* ... */ })
export class MyComponent implements OnInit {
  ngOnInit() {
    source$.pipe(
      takeUntilDestroyed() // ERROR: not in injection context
    ).subscribe();
  }
}

// FIX: Inject DestroyRef and pass it
@Component({ /* ... */ })
export class MyComponent implements OnInit {
  private destroyRef = inject(DestroyRef);

  ngOnInit() {
    source$.pipe(
      takeUntilDestroyed(this.destroyRef) // works
    ).subscribe();
  }
}
```

## Combining Filtering Operators: Real-World Patterns

### Autocomplete Search

```typescript
@Component({ /* ... */ })
export class AutocompleteComponent {
  private searchService = inject(SearchService);
  searchControl = new FormControl('');
  results = signal<string[]>([]);

  constructor() {
    this.searchControl.valueChanges.pipe(
      debounceTime(300),                         // wait for typing pause
      map(v => v?.trim() ?? ''),                 // normalize
      distinctUntilChanged(),                    // skip duplicate terms
      filter(term => term.length >= 2),          // minimum length
      switchMap(term => this.searchService.search(term)),
      takeUntilDestroyed()
    ).subscribe(results => this.results.set(results));
  }
}
```

### Infinite Scroll

```typescript
@Component({ /* ... */ })
export class InfiniteScrollComponent {
  private destroyRef = inject(DestroyRef);
  private loading = signal(false);

  ngAfterViewInit() {
    fromEvent(this.scrollContainer.nativeElement, 'scroll').pipe(
      throttleTime(200, asyncScheduler, { leading: false, trailing: true }),
      filter(() => !this.loading()),             // skip if already loading
      filter(() => this.isNearBottom()),         // only when scrolled near bottom
      takeUntilDestroyed(this.destroyRef)
    ).subscribe(() => this.loadNextPage());
  }
}
```

### WebSocket with Deduplication

```typescript
@Component({ /* ... */ })
export class LivePriceComponent {
  private ws = inject(WebSocketService);

  constructor() {
    this.ws.messages$.pipe(
      filter(msg => msg.type === 'price_update'),
      distinctUntilKeyChanged('symbol'),          // skip duplicate ticks per symbol
      auditTime(100),                             // batch rapid updates
      takeUntilDestroyed()
    ).subscribe(update => this.priceMap.update(m => ({ ...m, [update.symbol]: update.price })));
  }
}
```

### One-Time Data Load with Timeout

```typescript
this.config$.pipe(
  filter(config => config !== null),
  first(),                                       // take first non-null config
  timeout(5000),                                 // fail if no config within 5s
).subscribe({
  next: config => this.initialize(config),
  error: err => this.showConfigError(err)
});
```

## Quick Reference

| Operator | What it does | Completes? |
|----------|-------------|-----------|
| `filter(fn)` | Pass values matching predicate | When source completes |
| `first(fn?)` | First (matching) value | After first emission |
| `last(fn?)` | Last (matching) value | When source completes |
| `single(fn?)` | Exactly one (matching) value | When source completes |
| `elementAt(n)` | Value at index n | After that emission |
| `distinct(fn?)` | Never-before-seen values | When source completes |
| `distinctUntilChanged(fn?)` | Different from previous | When source completes |
| `distinctUntilKeyChanged(k)` | Key changed from previous | When source completes |
| `take(n)` | First n values | After nth value |
| `takeLast(n)` | Last n values | When source completes |
| `takeUntil(obs$)` | Until notifier emits | When notifier emits |
| `takeWhile(fn)` | While predicate is true | When predicate fails |
| `skip(n)` | Skip first n values | When source completes |
| `skipUntil(obs$)` | Skip until notifier emits | When source completes |
| `skipWhile(fn)` | Skip while predicate is true | When source completes |
| `debounceTime(ms)` | After ms of silence | When source completes |
| `throttleTime(ms)` | Rate-limit to window | When source completes |
| `auditTime(ms)` | Latest in window | When source completes |
| `sampleTime(ms)` | Latest at fixed intervals | When source completes |
| `defaultIfEmpty(val)` | Fallback for empty source | When source completes |
| `throwIfEmpty(fn?)` | Error on empty source | When source completes |
| `ignoreElements()` | Block all values | When source completes |
| `takeUntilDestroyed()` | Until Angular destroys context | On destroy |
