# 1.14 Web Components Basics

> Web Components are a set of native browser APIs that let you create reusable, encapsulated custom HTML elements without a framework. They consist of Custom Elements, Shadow DOM, HTML Templates, and Slots.

---

## Custom Elements

Custom Elements let you define new HTML tags with their own behavior. You register them with `customElements.define()`.

```javascript
class MyGreeting extends HTMLElement {
  connectedCallback() {
    const name = this.getAttribute('name') || 'World';
    this.textContent = `Hello, ${name}!`;
  }
}

customElements.define('my-greeting', MyGreeting);
```

```html
<my-greeting name="Alice"></my-greeting>
<!-- Renders: Hello, Alice! -->
```

### Naming Rules

Custom element names must:
- Start with a lowercase ASCII letter (`a-z`)
- Contain at least one hyphen (`-`)
- Not contain uppercase ASCII letters
- Not use reserved names (`annotation-xml`, `color-profile`, `font-face`, `font-face-src`, `font-face-uri`, `font-face-format`, `font-face-name`, `missing-glyph`)

```javascript
// VALID
customElements.define('my-element', MyElement);
customElements.define('app-header', AppHeader);
customElements.define('x-1', X1);

// INVALID - throws DOMException
customElements.define('myElement', MyElement);   // no hyphen
customElements.define('My-Element', MyElement);  // uppercase
customElements.define('1-element', MyElement);   // starts with digit
```

The hyphen requirement prevents name collisions with current and future standard HTML elements, which never contain hyphens.

### Constructor Rules

The constructor must call `super()` as the first statement. It must not read or write attributes, children, or the DOM. These operations belong in `connectedCallback`.

```javascript
class MyElement extends HTMLElement {
  constructor() {
    super(); // MUST be first

    // ALLOWED: attach shadow root, set up internal state, add listeners to self
    this.attachShadow({ mode: 'open' });
    this._count = 0;

    // FORBIDDEN: these throw or produce undefined behavior
    // this.setAttribute('ready', '');
    // this.innerHTML = '<p>Hello</p>';
    // this.textContent = 'Hello';
  }
}
```

### Autonomous vs Customized Built-in Elements

| Feature | Autonomous | Customized Built-in |
|---------|-----------|-------------------|
| Extends | `HTMLElement` | Specific element class |
| Usage | `<my-element>` | `<button is="my-button">` |
| Safari support | Yes | No (will never be supported) |
| Inherits semantics | No | Yes |

Apple refused to implement customized built-in elements, arguing it would make evolving built-in elements unsafe. There is a polyfill, but it is imperfect. For cross-browser compatibility, stick with autonomous custom elements.

```javascript
// Autonomous (works everywhere)
class MyButton extends HTMLElement { /* ... */ }
customElements.define('my-button', MyButton);

// Customized built-in (NOT supported in Safari)
class FancyButton extends HTMLButtonElement { /* ... */ }
customElements.define('fancy-button', FancyButton, { extends: 'button' });
```

## Lifecycle Callbacks

| Callback | When It Fires |
|----------|---------------|
| `constructor()` | Element created or upgraded |
| `connectedCallback()` | Element added to the document |
| `disconnectedCallback()` | Element removed from the document |
| `attributeChangedCallback(name, oldVal, newVal)` | Observed attribute changes |
| `adoptedCallback()` | Element moved to a new document via `document.adoptNode()` |

### `connectedCallback`

This is where setup work belongs: rendering, adding event listeners, starting observers. It fires every time the element is added to the DOM, including when it is moved between containers.

```javascript
class MyElement extends HTMLElement {
  #initialized = false;

  connectedCallback() {
    // Guard one-time setup
    if (!this.#initialized) {
      this.#initialized = true;
      this._render();
    }

    // Per-connection setup (runs every time)
    this._startListening();
  }

  disconnectedCallback() {
    this._stopListening();
  }
}
```

### `disconnectedCallback`

Cleanup goes here: remove event listeners, disconnect observers, clear intervals. Without cleanup, moving or removing elements leaks memory.

```javascript
class MyElement extends HTMLElement {
  #abortController;

  connectedCallback() {
    this.#abortController = new AbortController();
    const { signal } = this.#abortController;

    // All listeners registered with this signal
    window.addEventListener('resize', this._onResize, { signal });
    document.addEventListener('scroll', this._onScroll, { signal });
  }

  disconnectedCallback() {
    // Single call removes ALL listeners
    this.#abortController.abort();
  }
}
```

Using `AbortController` with the `signal` option is the cleanest pattern for bulk listener cleanup.

### Timing Gotcha: `attributeChangedCallback` Before `connectedCallback`

When attributes are set in HTML markup, the parser calls `attributeChangedCallback` for each observed attribute before calling `connectedCallback`. Your code must handle this ordering.

```javascript
class MyElement extends HTMLElement {
  static get observedAttributes() { return ['color']; }

  attributeChangedCallback(name, oldVal, newVal) {
    // This fires BEFORE connectedCallback
    // Shadow root may not exist yet if you create it in connectedCallback
    if (this.shadowRoot) {
      this._applyColor(newVal);
    }
  }

  connectedCallback() {
    if (!this.shadowRoot) {
      this.attachShadow({ mode: 'open' });
      this._render();
    }
    // Apply any attributes that were set before connection
    this._applyColor(this.getAttribute('color'));
  }
}
```

## Attribute Observation

Only attributes listed in the static `observedAttributes` getter trigger `attributeChangedCallback`. Without this opt-in, every `class` or `style` change would trigger callbacks, causing severe performance issues.

```javascript
class StatusBadge extends HTMLElement {
  static get observedAttributes() {
    return ['status', 'label'];
  }

  attributeChangedCallback(name, oldValue, newValue) {
    if (name === 'status') {
      this._updateStatus(newValue);
    }
    if (name === 'label') {
      this._updateLabel(newValue);
    }
  }
}
```

### Attribute-Property Reflection

HTML attributes are always strings. JavaScript properties can be any type. For a good developer experience, implement reflection so both approaches stay in sync.

```javascript
class MyToggle extends HTMLElement {
  static get observedAttributes() { return ['disabled', 'checked']; }

  // Boolean attribute: present = true, absent = false
  get disabled() {
    return this.hasAttribute('disabled');
  }
  set disabled(val) {
    if (val) {
      this.setAttribute('disabled', '');
    } else {
      this.removeAttribute('disabled');
    }
  }

  // String attribute
  get label() {
    return this.getAttribute('label') || '';
  }
  set label(val) {
    this.setAttribute('label', val);
  }
}
```

### Boolean Attribute Gotcha

In HTML, boolean attributes are true when present, regardless of their value. `disabled="false"` is still truthy.

```html
<!-- ALL of these are truthy -->
<my-toggle disabled></my-toggle>
<my-toggle disabled=""></my-toggle>
<my-toggle disabled="false"></my-toggle>
<my-toggle disabled="0"></my-toggle>

<!-- The only way to set false: omit the attribute entirely -->
<my-toggle></my-toggle>
```

## Shadow DOM

Shadow DOM provides encapsulation: styles inside the shadow tree do not leak out, and external styles do not penetrate in. It creates a scoped sub-tree attached to the element.

```javascript
class MyCard extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        :host { display: block; border: 1px solid #ccc; border-radius: 8px; }
        .content { padding: 16px; }
        h2 { margin: 0 0 8px; color: navy; }
      </style>
      <div class="content">
        <h2><slot name="title">Default Title</slot></h2>
        <slot></slot>
      </div>
    `;
  }
}

customElements.define('my-card', MyCard);
```

```html
<my-card>
  <span slot="title">Custom Title</span>
  <p>Card body content goes here.</p>
</my-card>
```

### Open vs Closed Mode

| Aspect | `mode: 'open'` | `mode: 'closed'` |
|--------|----------------|-------------------|
| `element.shadowRoot` | Returns the `ShadowRoot` | Returns `null` |
| External JS access | Full access | Denied (in theory) |
| DevTools | Visible | Visible |

`closed` mode does not provide real security. Code can override `attachShadow` to capture references. Most libraries use `open`. Use `closed` only to signal "don't reach into my internals."

### Style Encapsulation

- External CSS does not apply inside the shadow tree.
- Shadow DOM styles do not leak to the outer document.
- **CSS custom properties inherit through the boundary.** This is the primary theming mechanism.
- Inherited properties (`color`, `font-family`, `font-size`, etc.) also inherit into shadow DOM.

```css
/* External: this does NOT affect shadow DOM */
h2 { color: red; }

/* But custom properties DO cross the boundary */
:root {
  --primary-color: #e91e63;
}
```

```css
/* Inside shadow DOM: use the custom property */
h2 {
  color: var(--primary-color, navy); /* falls back to navy if not set */
}
```

### `delegatesFocus`

```javascript
this.attachShadow({ mode: 'open', delegatesFocus: true });
```

When enabled, clicking a non-focusable area inside the shadow tree focuses the first focusable child. The host element gets `:focus` styling when any shadow child is focused.

## Slots and Content Projection

Slots are placeholders in the shadow DOM where light DOM content is projected.

### Named and Default Slots

```html
<!-- Shadow DOM template -->
<div class="dialog">
  <header><slot name="header">Dialog</slot></header>
  <main><slot></slot></main>
  <footer><slot name="footer"></slot></footer>
</div>
```

```html
<!-- Usage -->
<my-dialog>
  <h2 slot="header">Confirm Action</h2>
  <p>Are you sure you want to proceed?</p>
  <div slot="footer">
    <button>Cancel</button>
    <button>OK</button>
  </div>
</my-dialog>
```

### Key Rules

- Only direct children of the host can be slotted. The `slot` attribute on nested elements is ignored.
- Slotted content remains in the light DOM. It is visually projected but its `parentNode` is still the host.
- Content inside `<slot>` tags is fallback: it renders only when no content is projected to that slot.
- Only the first unnamed `<slot>` receives unassigned content.

### The `slotchange` Event

Fires when the set of nodes assigned to a slot changes (nodes added or removed).

```javascript
connectedCallback() {
  this.shadowRoot.querySelector('slot').addEventListener('slotchange', (e) => {
    const assigned = e.target.assignedElements();
    console.log('Slotted elements:', assigned);
  });
}
```

### Slot API

```javascript
// Get elements assigned to a slot
slot.assignedElements();
slot.assignedElements({ flatten: true }); // includes fallback content
slot.assignedNodes();                     // includes text nodes

// Get which slot an element is in
element.assignedSlot; // returns <slot> or null
```

## CSS for Web Components

### `:host`

Styles the host element itself from inside the shadow DOM.

```css
:host {
  display: block;
  border: 1px solid #ddd;
  padding: 16px;
}

/* Conditional: when host has a class or attribute */
:host(.active) {
  border-color: blue;
}

:host([disabled]) {
  opacity: 0.5;
  pointer-events: none;
}
```

`:host` styles have lower specificity than external styles. An external `my-element { display: inline; }` will override `:host { display: block; }`.

### `::slotted()`

Styles slotted elements. Only accepts compound selectors (no combinators) and only targets direct children of the host.

```css
::slotted(p) {
  margin: 0;
  color: navy;
}

::slotted(.highlight) {
  background: yellow;
}

/* DOES NOT WORK: no descendant selectors */
::slotted(p span) { }  /* INVALID */
```

### `::part()` and `exportparts`

Exposes specific internal elements for external styling.

```javascript
// Inside shadow DOM
this.shadowRoot.innerHTML = `
  <button part="button">
    <span part="label">Click</span>
  </button>
`;
```

```css
/* External stylesheet targets named parts */
my-element::part(button) {
  background: blue;
  color: white;
  border: none;
  padding: 8px 16px;
}
```

For nested components, use `exportparts` to forward inner parts:

```html
<!-- Inside outer-element shadow DOM -->
<inner-element exportparts="button: inner-btn"></inner-element>
```

```css
/* Consumer of outer-element */
outer-element::part(inner-btn) { border-radius: 8px; }
```

### Constructable Stylesheets

Share a single parsed stylesheet across all instances of a component.

```javascript
const styles = new CSSStyleSheet();
styles.replaceSync(`
  :host { display: block; }
  .container { padding: 16px; }
`);

class MyElement extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.adoptedStyleSheets = [styles]; // shared, not duplicated
  }
}
```

Multiple sheets can be composed: `shadow.adoptedStyleSheets = [reset, tokens, component];`

### Preventing FOUC (Flash of Undefined Custom Elements)

Custom elements are `display: inline` by default and show raw content until their class is defined. Use the `:defined` pseudo-class to hide them until ready:

```css
my-element:not(:defined) {
  visibility: hidden;
}
```

Using `visibility: hidden` instead of `display: none` reserves layout space and prevents layout shift when the element defines.

## Events in Shadow DOM

### Event Retargeting

When an event bubbles out of a shadow root, its `event.target` is retargeted to the host element. The original internal target is hidden from external listeners.

```javascript
// Inside shadow DOM
this.shadowRoot.querySelector('button').addEventListener('click', (e) => {
  console.log(e.target); // <button> (actual target)
});

// Outside
document.querySelector('my-element').addEventListener('click', (e) => {
  console.log(e.target); // <my-element> (retargeted to host)
});
```

Use `event.composedPath()` to see the full propagation path including shadow internals.

### The `composed` Property

Events with `composed: true` cross shadow boundaries. Events with `composed: false` stop at the shadow root.

Most native UI events (`click`, `focus`, `input`, `keydown`) are `composed: true`. Events like `slotchange`, `submit`, and `reset` are `composed: false`.

### Dispatching Custom Events

```javascript
// Public event: dispatch on the host, composed: true
this.dispatchEvent(new CustomEvent('value-changed', {
  bubbles: true,
  composed: true,
  detail: { value: this._value }
}));

// Internal event: dispatch on shadow child, composed: false
this.shadowRoot.querySelector('.input').dispatchEvent(
  new CustomEvent('internal-update', {
    bubbles: true,
    composed: false // stays inside shadow tree
  })
);
```

Dispatch public events on `this` (the host), not on shadow children. Use `composed: false` for internal events to avoid leaking implementation details.

## Form-Associated Custom Elements

Custom elements can participate in form submission and validation using `ElementInternals`.

```javascript
class MyInput extends HTMLElement {
  static formAssociated = true;

  #internals;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    this.attachShadow({ mode: 'open' });
    this.shadowRoot.innerHTML = `
      <label><slot></slot></label>
      <input type="text">
    `;
  }

  connectedCallback() {
    const input = this.shadowRoot.querySelector('input');
    input.addEventListener('input', () => {
      this.#internals.setFormValue(input.value);

      if (this.hasAttribute('required') && !input.value) {
        this.#internals.setValidity(
          { valueMissing: true },
          'This field is required.',
          input
        );
      } else {
        this.#internals.setValidity({});
      }
    });
  }

  // Form lifecycle callbacks
  formResetCallback() {
    this.shadowRoot.querySelector('input').value = '';
    this.#internals.setFormValue('');
  }

  formDisabledCallback(disabled) {
    this.shadowRoot.querySelector('input').disabled = disabled;
  }
}

customElements.define('my-input', MyInput);
```

```html
<form>
  <my-input name="username" required>Username</my-input>
  <button type="submit">Submit</button>
</form>
```

### Custom States

`ElementInternals` provides a `CustomStateSet` that can be matched with the CSS `:state()` pseudo-class.

```javascript
// Toggle custom state
this.#internals.states.add('checked');
this.#internals.states.delete('checked');
```

```css
my-toggle:state(checked) {
  background: green;
}
```

## Element Upgrading

When the parser encounters an unknown hyphenated tag before its class is defined, it creates a generic element. When `customElements.define()` is later called, all existing instances are upgraded: the constructor runs and `connectedCallback` fires.

```javascript
// Element exists before definition
const el = document.createElement('my-counter');
document.body.appendChild(el);
console.log(el instanceof MyCounter); // false

// Define the element
customElements.define('my-counter', MyCounter);
console.log(el instanceof MyCounter); // true (upgraded)
```

### `whenDefined`

Returns a Promise that resolves when an element is defined. Useful for waiting on dependencies.

```javascript
await customElements.whenDefined('child-widget');
const child = this.shadowRoot.querySelector('child-widget');
child.configure({ option: 'value' });
```

### `get`

Synchronous check for whether an element is already defined.

```javascript
if (customElements.get('my-element')) {
  // Already defined
}
```

## Declarative Shadow DOM

Creates shadow roots in static HTML without JavaScript.

```html
<my-alert type="warning">
  <template shadowrootmode="open">
    <style>
      :host { display: block; padding: 12px; border-radius: 4px; }
      :host([type="warning"]) { background: #fff3cd; border: 1px solid #ffc107; }
      :host([type="error"]) { background: #f8d7da; border: 1px solid #f5c6cb; }
    </style>
    <slot></slot>
  </template>
  Your session will expire in 5 minutes.
</my-alert>
```

The `<template>` is removed from the DOM after parsing and replaced by a `ShadowRoot`. This works without JavaScript and enables SSR of web components.

### Hydration Pattern

When JavaScript loads after the server sends Declarative Shadow DOM, check for an existing shadow root before calling `attachShadow`:

```javascript
class MyAlert extends HTMLElement {
  constructor() {
    super();
    // DSD may have already created the shadow root
    if (!this.shadowRoot) {
      this.attachShadow({ mode: 'open' });
      this._render();
    }
  }

  connectedCallback() {
    // Shadow root is guaranteed to exist here
    this.shadowRoot.addEventListener('click', this._onClick);
  }
}
```

## Accessibility

### Shadow DOM and Screen Readers

Screen readers see the flattened (rendered) DOM tree, including shadow DOM content. Slotted content retains its light DOM position in the accessibility tree.

### Cross-Shadow ARIA Problem

ARIA attributes that reference elements by ID (`aria-labelledby`, `aria-describedby`, `aria-controls`) cannot cross shadow boundaries. IDs are scoped to their root.

```html
<!-- BROKEN: label cannot reach input inside shadow DOM -->
<label for="inner-input">Name</label>
<my-input>
  #shadow-root
    <input id="inner-input">  <!-- different scope -->
</my-input>
```

Workarounds:
1. Keep both label and input in the same root.
2. Use `ElementInternals` ARIA: `this.#internals.role = 'textbox'; this.#internals.ariaLabel = 'Name';`
3. Wrap the label inside the component's shadow DOM.

### Keyboard Accessibility

Custom interactive elements must:
- Be focusable (`tabindex="0"` or `delegatesFocus: true`)
- Handle keyboard events (Enter/Space for activation, arrow keys for composite widgets)
- Have an appropriate ARIA role

```javascript
class MyButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open', delegatesFocus: true });
    if (!this.hasAttribute('role')) {
      this.setAttribute('role', 'button');
    }
    if (!this.hasAttribute('tabindex')) {
      this.setAttribute('tabindex', '0');
    }
  }

  connectedCallback() {
    this.addEventListener('keydown', (e) => {
      if (e.key === 'Enter' || e.key === ' ') {
        e.preventDefault();
        this.click();
      }
    });
  }
}
```

## Common Mistakes

- **Doing DOM work in the constructor.** Reading attributes, setting innerHTML, or accessing children in the constructor produces errors or undefined behavior. Use `connectedCallback`.

- **Not guarding `connectedCallback` for multiple calls.** It fires every time the element enters the DOM, including when moved. One-time setup must be guarded with a flag.

- **Forgetting cleanup in `disconnectedCallback`.** Event listeners, observers, and intervals leak when not removed. Use `AbortController` for bulk cleanup.

- **Setting `disabled="false"` expecting falsy behavior.** Boolean HTML attributes are true when present. `disabled="false"` is still truthy. Remove the attribute to make it false.

- **Assuming `::slotted()` works on nested children.** It only targets direct children of the host. Descendants are not matched.

- **Calling `attachShadow()` when Declarative Shadow DOM already created one.** Check `this.shadowRoot` first to avoid throwing an error.

- **Not setting `:host { display: block; }`.** Custom elements default to `display: inline`, which often breaks layout.

- **Using `composed: true` for all custom events.** This leaks internal implementation details. Use `composed: false` for events that should stay inside the component.

- **Reaching into another component's shadow DOM.** `el.shadowRoot.querySelector(...)` as a public API breaks encapsulation and creates fragile code.

---

## Best Practices

- **Create the shadow root in the constructor, render in `connectedCallback`.** The constructor can attach the shadow root, but rendering and DOM access should wait for connection.

- **Use `AbortController` for event listener cleanup.** Register listeners with a signal in `connectedCallback`, abort in `disconnectedCallback`.

- **Reflect attributes to properties and vice versa.** Users expect `el.disabled = true` and `el.setAttribute('disabled', '')` to behave the same way.

- **Use CSS custom properties for theming.** They are the only values that cross shadow boundaries. Define sensible defaults with `var(--prop, fallback)`.

- **Use `::part()` for controlled external styling.** Expose only the parts you want consumers to customize.

- **Use constructable stylesheets (`adoptedStyleSheets`).** A single stylesheet shared across all instances is more performant than inline `<style>` in each shadow root.

- **Dispatch public events on the host element.** Use `composed: true` only for events that genuinely need to cross shadow boundaries.

- **Use Declarative Shadow DOM for SSR.** Check `this.shadowRoot` before calling `attachShadow()` to support hydration.

## Real-World Use Cases

### Reusable Alert Banner

```javascript
const styles = new CSSStyleSheet();
styles.replaceSync(`
  :host { display: block; padding: 12px 16px; border-radius: 4px; margin-bottom: 12px; font-family: system-ui, sans-serif; }
  :host([type="info"]) { background: #d1ecf1; border: 1px solid #bee5eb; color: #0c5460; }
  :host([type="warning"]) { background: #fff3cd; border: 1px solid #ffc107; color: #856404; }
  :host([type="error"]) { background: #f8d7da; border: 1px solid #f5c6cb; color: #721c24; }
  :host([type="success"]) { background: #d4edda; border: 1px solid #c3e6cb; color: #155724; }
  button { float: right; background: none; border: none; font-size: 1.2em; cursor: pointer; color: inherit; }
`);

class AlertBanner extends HTMLElement {
  static get observedAttributes() { return ['type', 'dismissible']; }

  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.adoptedStyleSheets = [styles];
    shadow.innerHTML = `<button part="close" aria-label="Dismiss" hidden>&times;</button><slot></slot>`;
  }

  connectedCallback() {
    if (!this.hasAttribute('role')) this.setAttribute('role', 'alert');
    this._updateDismissible();
    this.shadowRoot.querySelector('button').addEventListener('click', () => {
      this.dispatchEvent(new CustomEvent('dismiss', { bubbles: true, composed: true }));
      this.remove();
    });
  }

  attributeChangedCallback(name) {
    if (name === 'dismissible') this._updateDismissible();
  }

  _updateDismissible() {
    const btn = this.shadowRoot?.querySelector('button');
    if (btn) btn.hidden = !this.hasAttribute('dismissible');
  }
}

customElements.define('alert-banner', AlertBanner);
```

```html
<alert-banner type="warning" dismissible>Your trial expires in 3 days.</alert-banner>
<alert-banner type="error">Payment failed. Please try again.</alert-banner>
```

### Tooltip Component with Slots and Parts

```javascript
class MyTooltip extends HTMLElement {
```

### Accordion with Custom States

```javascript
class MyAccordion extends HTMLElement {
  #internals;

  constructor() {
    super();
    this.#internals = this.attachInternals();
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.innerHTML = `
      <style>
        :host { display: block; border: 1px solid #ddd; border-radius: 4px; }
        .header { padding: 12px 16px; cursor: pointer; user-select: none;
                  display: flex; justify-content: space-between; align-items: center; }
        .content { padding: 0 16px; max-height: 0; overflow: hidden; transition: max-height 0.3s, padding 0.3s; }
        :host(:state(open)) .content { max-height: 500px; padding: 12px 16px; }
        :host(:state(open)) .arrow { transform: rotate(180deg); }
        .arrow { transition: transform 0.3s; }
      </style>
      <div class="header" part="header" role="button" tabindex="0" aria-expanded="false">
        <slot name="header">Accordion</slot>
        <span class="arrow" part="arrow">&#9660;</span>
      </div>
      <div class="content" part="content" role="region">
        <slot></slot>
      </div>
    `;
  }

  connectedCallback() {
    const header = this.shadowRoot.querySelector('.header');
    header.addEventListener('click', () => this._toggle());
    header.addEventListener('keydown', (e) => {
      if (e.key === 'Enter' || e.key === ' ') {
        e.preventDefault();
        this._toggle();
      }
    });
  }

  _toggle() {
    const isOpen = this.#internals.states.has('open');
    const header = this.shadowRoot.querySelector('.header');

    if (isOpen) {
      this.#internals.states.delete('open');
      header.setAttribute('aria-expanded', 'false');
    } else {
      this.#internals.states.add('open');
      header.setAttribute('aria-expanded', 'true');
    }
  }
}

customElements.define('my-accordion', MyAccordion);
```

> **Key Takeaway:** Web Components combine Custom Elements, Shadow DOM, Templates, and Slots to create reusable, encapsulated HTML elements with native browser APIs. Use the constructor only for shadow root attachment and internal state, `connectedCallback` for rendering and setup, CSS custom properties for theming across shadow boundaries, and `::part()` for controlled external styling.
