# 2.11 Backgrounds and Gradients

> CSS backgrounds go far beyond a single image or color. Multiple layers, three gradient types, color-space interpolation, and clipping modes let you build complex visual effects without a single image file.

---

## Background Sub-Properties

Before diving into shortcuts, understand what you are actually controlling:

| Property | Default | Purpose |
|----------|---------|---------|
| `background-color` | `transparent` | Solid color behind everything |
| `background-image` | `none` | Image or gradient |
| `background-position` | `0% 0%` | Where the image sits |
| `background-size` | `auto auto` | How the image is scaled |
| `background-repeat` | `repeat` | Tiling behavior |
| `background-attachment` | `scroll` | Scrolling behavior |
| `background-origin` | `padding-box` | Reference box for positioning |
| `background-clip` | `border-box` | Where background is visible |

---

## The Background Shorthand

The `background` shorthand sets all sub-properties in one declaration:

```css
background: [image] [position] / [size] [repeat] [attachment] [origin] [clip] [color];
```

### The Slash Rule

`background-size` MUST immediately follow `background-position`, separated by a `/`. You cannot specify size without position:

```css
/* Correct */
background: url(photo.jpg) center / cover no-repeat;

/* Correct: explicit position before slash */
background: url(photo.jpg) left top / 600px auto no-repeat;

/* WRONG: size without position */
background: url(photo.jpg) / cover no-repeat;

/* WRONG: slash not immediately after position */
background: url(photo.jpg) center no-repeat / cover;
```

### The Box Value Ambiguity

`background-origin` and `background-clip` accept the same keywords. In the shorthand:

- **One box keyword**: sets BOTH `origin` and `clip` to that value.
- **Two box keywords**: first sets `origin`, second sets `clip`.

```css
/* Both origin and clip are content-box */
background: url(img.png) content-box;

/* Origin is padding-box, clip is content-box */
background: url(img.png) padding-box content-box;
```

### The Reset Trap

The shorthand resets ALL unspecified sub-properties to their defaults. This is the most common source of background bugs:

```css
/* Bug: shorthand resets background-color to transparent */
background-color: red;
background: url(texture.jpg);  /* color is now transparent, NOT red */

/* Fix: include color in the shorthand */
background: red url(texture.jpg);

/* Or: use longhand for the image */
background-color: red;
background-image: url(texture.jpg);
```

When you only need to change one or two properties, use longhand to avoid accidental resets.

## Multiple Backgrounds

Multiple backgrounds are specified as a comma-separated list. The **first** listed is the **topmost** layer (closest to the viewer). The **last** listed is the **bottommost**:

```css
.hero {
  background:
    url(sparkles.png) center / contain no-repeat,          /* top layer */
    linear-gradient(rgba(0,0,0,0.4), transparent),          /* overlay */
    url(hero-photo.jpg) center / cover no-repeat;           /* bottom layer */
}
```

### Rules for Multiple Backgrounds

1. `background-color` can only appear in the **last** (bottommost) layer.
2. Each layer can have its own position, size, repeat, origin, and clip.
3. If you specify fewer values for a sub-property than there are layers, the list **cycles**. Three images with two positions means position 1, 2, 1.

### Longhand with Multiple Values

```css
.layered {
  background-image: url(overlay.png), url(photo.jpg);
  background-position: center, center;
  background-size: contain, cover;
  background-repeat: no-repeat, no-repeat;
}
```

## background-origin vs. background-clip

These two properties control different aspects of the background box:

| Property | Controls | Default |
|----------|----------|---------|
| `background-origin` | Where the background **starts positioning** | `padding-box` |
| `background-clip` | Where the background **is visible** | `border-box` |

### Visual Difference

```css
.origin-demo {
  border: 10px dashed rgba(0,0,0,0.3);
  padding: 20px;
  background: url(pattern.png) no-repeat;
  background-origin: content-box;  /* image positioned from content edge */
  /* Image starts inside the padding */
}

.clip-demo {
  border: 10px dashed rgba(0,0,0,0.3);
  padding: 20px;
  background: skyblue;
  background-clip: content-box;    /* background only visible in content area */
  /* Padding area shows no background */
}
```

`background-origin` moves where the image starts. `background-clip` cuts where the background shows. They share the same keywords (`border-box`, `padding-box`, `content-box`) but do fundamentally different things.

## background-clip: text

Clips the background to the text glyphs, making text a window into the background:

```css
.gradient-text {
  background: linear-gradient(90deg, #7953cd, #00affa, #0190cd, #764ada);
  -webkit-background-clip: text;        /* Required for Chrome/Safari */
  background-clip: text;
  -webkit-text-fill-color: transparent;
  color: transparent;                    /* Fallback for non-webkit */
}
```

### Why Both Prefixed and Unprefixed

As of 2025, Chrome and Safari still require `-webkit-background-clip: text`. Firefox supports the unprefixed version natively since version 49. Always include both for cross-browser support.

### Animated Gradient Text

```css
.shimmer-text {
  --bg-size: 400%;
  background: linear-gradient(
    90deg,
    #7953cd 20%, #00affa 30%, #0190cd 70%, #764ada 80%
  );
  background-size: var(--bg-size) 100%;
  -webkit-background-clip: text;
  background-clip: text;
  -webkit-text-fill-color: transparent;
  color: transparent;
  animation: shimmer 5s ease-in-out infinite alternate;
}

@keyframes shimmer {
  0%   { background-position: 0% 50%; }
  100% { background-position: 100% 50%; }
}
```

This animates `background-position` (cheap, GPU-friendly) rather than the gradient colors themselves (expensive, triggers repaint).

## background-size: cover vs. contain

| Behavior | `cover` | `contain` |
|----------|---------|-----------|
| Scaling | Fills entire container | Fits within container |
| Cropping | Image may be cropped | Image is never cropped |
| Empty space | None | May have letterbox gaps |
| Use case | Hero images, card thumbnails | Logos, diagrams, product photos |

```css
.hero {
  background: url(hero.jpg) center / cover no-repeat;
}

.logo-area {
  background: url(logo.svg) center / contain no-repeat;
}
```

### Gotcha: contain Without no-repeat

If you use `contain` without `no-repeat`, the image tiles to fill the remaining space:

```css
/* Bug: image tiles */
background: url(logo.svg) center / contain;

/* Fix: always pair contain with no-repeat */
background: url(logo.svg) center / contain no-repeat;
```

### background-size vs. object-fit

| Feature | `background-size` | `object-fit` |
|---------|--------------------|--------------|
| Applies to | Background images | `<img>`, `<video>`, `<iframe>` |
| Accessibility | No `alt` text, not crawled | `alt` text, crawled by search engines |
| Use for | Decorative images, patterns | Content images conveying information |

Use `<img>` with `object-fit` for meaningful images. Use `background-image` with `background-size` for decorative visuals.

## background-attachment

| Value | Behavior |
|-------|----------|
| `scroll` | Background scrolls with the element (default) |
| `fixed` | Background is fixed relative to the viewport |
| `local` | Background scrolls with the element's own scrollable content |

### The Parallax Trap

```css
.parallax {
  background: url(mountains.jpg) center / cover no-repeat fixed;
  min-height: 500px;
}
```

This creates a basic parallax effect. But it comes with serious problems:

1. **iOS Safari completely ignores `background-attachment: fixed`.** The background scrolls normally.
2. **Performance hit.** The browser must recomposite the fixed background against scrolling content on every frame.
3. **`will-change: transform` conflicts.** In Chrome, combining `will-change: transform` with `background-attachment: fixed` clips the image.

### Mobile-Safe Alternative

```css
/* Use a fixed pseudo-element instead */
.parallax {
  position: relative;
  min-height: 500px;
  overflow: hidden;
}

.parallax::before {
  content: '';
  position: fixed;
  inset: 0;
  z-index: -1;
  background: url(mountains.jpg) center / cover no-repeat;
}

/* Or simply disable on mobile */
@media (max-width: 768px) {
  .parallax {
    background-attachment: scroll;
  }
}
```

### background-attachment: local

`local` makes the background scroll with the element's own content, not with the page. This is useful for scrollable containers where you want the background to move with the content:

```css
.scrollable-panel {
  overflow-y: auto;
  height: 300px;
  background: url(watermark.png) center no-repeat;
  background-attachment: local;  /* Moves with panel content */
}
```

## Linear Gradients

### Basic Syntax

```css
background: linear-gradient(direction, color-stop1, color-stop2, ...);
```

Direction can be an angle (`45deg`, `135deg`) or a keyword (`to right`, `to bottom left`):

```css
background: linear-gradient(to right, red, blue);
background: linear-gradient(135deg, red, blue);
```

### Color Stop Positions

By default, stops are evenly distributed. You can set explicit positions:

```css
/* Blue dominates, red only at the very end */
background: linear-gradient(to right, blue 0%, blue 70%, red 100%);
```

### Hard Stops (No Transition)

Set two adjacent stops at the same position for a sharp edge:

```css
/* Sharp division at 50% */
background: linear-gradient(to right, red 50%, blue 50%);
```

### Transition Hints (Midpoint)

A lone percentage between two colors shifts the midpoint of the transition:

```css
/* Transition midpoint shifted to 30% */
background: linear-gradient(to right, red, 30%, blue);
/* Red fades to blue faster -- at 30%, the mix is 50/50 */
```

### Repeating Linear Gradient

`repeating-linear-gradient()` tiles the gradient pattern based on the distance between the first and last stop:

```css
/* Diagonal candy stripes */
background: repeating-linear-gradient(
  45deg,
  #ff6b6b 0px,
  #ff6b6b 20px,
  #4ecdc4 20px,
  #4ecdc4 40px
);
```

The pattern repeats every 40px (the last stop position).

## Radial Gradients

### Basic Syntax

```css
background: radial-gradient([shape] [size] [at position], color-stops);
```

### Shape and Size

Two shapes: `circle` and `ellipse` (default). Four sizing keywords:

| Keyword | Behavior |
|---------|----------|
| `closest-side` | Gradient ends at the nearest side |
| `farthest-side` | Gradient ends at the farthest side |
| `closest-corner` | Gradient ends at the nearest corner |
| `farthest-corner` | (Default) Gradient ends at the farthest corner |

```css
/* Compact: reaches nearest side only */
background: radial-gradient(ellipse closest-side at 30% 40%, blue, green, black);

/* Expansive: reaches farthest corner */
background: radial-gradient(circle farthest-corner at 30% 40%, blue, green, black);

/* Explicit pixel dimensions */
background: radial-gradient(150px 100px at center, #3b82f6, transparent);
```

### Off-Center Positioning

When the center is not at `50% 50%`, the sizing keywords produce dramatically different results because distances to each edge differ:

```css
/* Circle at top-left corner: closest-side barely shows */
background: radial-gradient(circle closest-side at 10% 10%, red, blue);

/* Same position, farthest-corner: fills most of the element */
background: radial-gradient(circle farthest-corner at 10% 10%, red, blue);
```

### Repeating Radial Gradient

```css
/* Concentric rings */
background: repeating-radial-gradient(
  circle at center,
  #3b82f6 0px,
  #3b82f6 10px,
  transparent 10px,
  transparent 20px
);
```

## Conic Gradients

Conic gradients sweep color stops around a center point, like the hands of a clock:

```css
background: conic-gradient([from angle] [at position,] color-stops);
```

The default start angle is 0deg (12 o'clock / top). Stops are specified in degrees or turns.

### Basic Color Wheel

```css
.wheel {
  width: 200px;
  height: 200px;
  border-radius: 50%;
  background: conic-gradient(red, yellow, lime, aqua, blue, magenta, red);
}
```

### Pie Chart

Use hard stops (same angle for adjacent colors) for sharp divisions:

```css
.pie {
  width: 200px;
  height: 200px;
  border-radius: 50%;
  background: conic-gradient(
    #e74c3c 0deg 120deg,      /* Red: 33.3% */
    #3498db 120deg 210deg,    /* Blue: 25% */
    #2ecc71 210deg 360deg     /* Green: 41.7% */
  );
}
```

### Donut Chart

Add a centered pseudo-element to create the hole:

```css
.donut {
  width: 200px;
  height: 200px;
  border-radius: 50%;
  position: relative;
  background: conic-gradient(
    #e74c3c 0deg 120deg,
    #3498db 120deg 210deg,
    #2ecc71 210deg 360deg
  );
}

.donut::after {
  content: '';
  position: absolute;
  inset: 25%;
  border-radius: 50%;
  background: white;  /* Must match parent background */
}
```

For transparent parent backgrounds, use `mask` instead:

```css
.donut {
  /* ... same gradient ... */
  mask: radial-gradient(circle, transparent 40px, black 41px);
}
```

### Rotating the Start Angle

```css
/* Start from the bottom (180deg) instead of the top */
background: conic-gradient(from 180deg, red, blue);
```

### Repeating Conic Gradient

```css
/* Starburst pattern */
background: repeating-conic-gradient(
  #000 0deg 10deg,
  #fff 10deg 20deg
);
```

## Gradient Color Interpolation

By default, gradients interpolate in sRGB, which produces a desaturated "gray dead zone" between complementary colors.

### The `in` Keyword

Add `in <color-space>` to control interpolation:

```css
/* Default sRGB: muddy midpoint between blue and yellow */
background: linear-gradient(blue, yellow);

/* OKLCH: vibrant, perceptually uniform midpoint */
background: linear-gradient(in oklch, blue, yellow);

/* OKLab: smooth without hue shifts */
background: linear-gradient(in oklab, red, blue);
```

### Hue Interpolation Direction

For polar color spaces (OKLCH, HSL), you can control which arc around the color wheel to follow:

```css
/* Shorter arc (default): fewer intermediate hues */
background: linear-gradient(in oklch shorter hue, red, blue);

/* Longer arc: passes through more hues */
background: linear-gradient(in oklch longer hue, red, blue);

/* Always clockwise */
background: linear-gradient(in oklch increasing hue, red, blue);

/* Always counter-clockwise */
background: linear-gradient(in oklch decreasing hue, red, blue);
```

### Color Space Comparison for Gradients

| Color Space | Muddy Midpoints | Hue Control | Best For |
|-------------|----------------|-------------|----------|
| `srgb` (default) | Yes | No | Legacy compatibility |
| `oklab` | No | No | Smooth neutral blends |
| `oklch` | No | Yes | Vibrant gradients with hue arcs |
| `hsl` | Sometimes | Yes | Hue-based transitions |

Default to `in oklch` for gradients where vibrancy matters.

## CSS-Only Patterns with Gradients

Gradients can create complex patterns without any image files. All patterns are resolution-independent with zero HTTP requests.

### Horizontal Stripes

```css
.h-stripes {
  background: repeating-linear-gradient(
    to bottom,
    #fff 0px,
    #fff 10px,
    #eee 10px,
    #eee 20px
  );
}
```

### Diagonal Stripes

```css
.diagonal {
  background: repeating-linear-gradient(
    45deg,
    #3b82f6 0px,
    #3b82f6 10px,
    transparent 10px,
    transparent 20px
  );
}
```

### Checkerboard (Conic Gradient Method)

The simplest checkerboard uses a single conic-gradient with `background-size`:

```css
.checkerboard {
  background: conic-gradient(
    #ccc 0.25turn,
    #fff 0.25turn 0.5turn,
    #ccc 0.5turn 0.75turn,
    #fff 0.75turn
  );
  background-size: 40px 40px;
}
```

### Polka Dots

```css
.dots {
  background-color: #fff;
  background-image:
    radial-gradient(circle, #333 10%, transparent 11%),
    radial-gradient(circle, #333 10%, transparent 11%);
  background-size: 30px 30px;
  background-position: 0 0, 15px 15px;
}
```

### Grid Lines

```css
.grid {
  background-image:
    linear-gradient(to right, #ddd 1px, transparent 1px),
    linear-gradient(to bottom, #ddd 1px, transparent 1px);
  background-size: 20px 20px;
}
```

### Pattern Sizing Rule

The `background-size` must precisely match the stop distances in your gradient for seamless tiling. If the size is off by even 1px, the pattern develops visible seams.

## Gradient Banding

Visible color bands instead of smooth transitions. This happens when:

- The gradient area is large with subtle color differences
- The display has limited color depth
- Colors are far apart in sRGB

### Fixing Banding

```css
/* Add intermediate color stops for smoother transition */
background: linear-gradient(white, #e0e8ff 50%, blue);

/* Use OKLCH for perceptually smooth interpolation */
background: linear-gradient(in oklch, white, blue);
```

For dark gradients on large areas, a noise texture overlay helps mask banding:

```css
.dark-gradient {
  background:
    url("noise.svg"),                                    /* noise texture on top */
    linear-gradient(in oklch, #1a1a2e, #16213e);         /* gradient below */
}
```

## Animating Gradients

You cannot directly transition or animate gradient values. The browser treats gradients as `<image>` values with no defined interpolation.

### The Wrong Way

```css
/* Does NOT animate -- snaps between states */
.box {
  background: linear-gradient(red, blue);
  transition: background 0.3s;
}
.box:hover {
  background: linear-gradient(green, yellow);
}
```

### Animate Background Position Instead

Move a wider-than-visible gradient by animating `background-position`:

```css
.animated-bg {
  background: linear-gradient(90deg, #ff0000, #00ff00, #0000ff, #ff0000);
  background-size: 300% 100%;
  animation: slide 5s linear infinite;
}

@keyframes slide {
  0%   { background-position: 0% 50%; }
  100% { background-position: 100% 50%; }
}
```

### Animate with @property

Register the individual gradient values as typed custom properties (covered in 2.10):

```css
@property --stop-1 {
  syntax: "<color>";
  inherits: false;
  initial-value: #3b82f6;
}

@property --stop-2 {
  syntax: "<color>";
  inherits: false;
  initial-value: #8b5cf6;
}

.hero {
  background: linear-gradient(135deg, var(--stop-1), var(--stop-2));
  transition: --stop-1 0.6s, --stop-2 0.6s;
}

.hero:hover {
  --stop-1: #ef4444;
  --stop-2: #f97316;
}
```

## Common Mistakes

1. **Background shorthand resetting color.** Writing `background: url(...)` after `background-color: red` resets the color to transparent. Use longhand properties when you only need to set one value.

2. **Forgetting position before size.** `background: url(img.jpg) / cover` is invalid. The slash requires a position before it: `background: url(img.jpg) center / cover`.

3. **Multiple background stacking order confusion.** The first listed background is on top, not the bottom. This is the reverse of what many developers expect from z-index mental models.

4. **`contain` without `no-repeat`.** The image tiles to fill remaining space. Always pair `contain` with `no-repeat`.

5. **Using `background-attachment: fixed` for mobile parallax.** iOS Safari ignores it entirely. Android behavior varies. Use CSS `perspective` transforms or disable on mobile.

6. **Trying to animate gradient colors directly.** Gradients are images and cannot be interpolated. Animate `background-position` or use `@property` to animate individual color stops.

7. **Diagonal stripe alignment.** Non-45-degree angles produce aliasing artifacts. Stick to multiples of 45deg for clean pattern edges.

8. **Gradient banding on large areas.** Add intermediate stops, use OKLCH interpolation, or overlay a noise texture.

---

## Best Practices

1. **Use longhand properties when changing one value.** This avoids accidental resets from the shorthand.

2. **Use CSS gradients instead of image files** for simple visual effects. They are resolution-independent, zero-HTTP, and easily adjustable.

3. **Interpolate gradients in OKLCH** for vibrant, banding-free results: `linear-gradient(in oklch, ...)`.

4. **Use `<img>` with `object-fit` for content images.** Reserve `background-image` for decorative visuals that carry no meaningful information.

5. **Test `background-attachment: fixed` on mobile.** If parallax matters, use CSS transform-based techniques or disable gracefully.

6. **Keep gradient stop counts low.** 2-4 stops covers most use cases. More stops increase rendering cost and make the code harder to maintain.

7. **Provide solid color fallbacks.** If a gradient or image fails to load, the fallback color prevents invisible content:
   ```css
   background: #1a1a2e linear-gradient(in oklch, #1a1a2e, #16213e);
   ```

## Real-World Use Cases

### 1. Hero Section with Image Overlay

```css
.hero {
  min-height: 80vh;
  display: flex;
  align-items: center;
  justify-content: center;
  color: white;

  background:
    /* Dark overlay for text readability */
    linear-gradient(
      to bottom,
      rgba(0, 0, 0, 0.6),
      rgba(0, 0, 0, 0.3) 60%,
      rgba(0, 0, 0, 0.7)
    ),
    /* Photo on the bottom layer */
    url(hero.jpg) center / cover no-repeat;

  /* Fallback if image fails */
  background-color: #1a1a2e;
}
```

### 2. Decorative Section Divider

```css
.divider {
  height: 4px;
  background: linear-gradient(
    in oklch,
    oklch(60% 0.25 250),    /* blue */
    oklch(65% 0.25 310),    /* purple */
    oklch(60% 0.22 20)      /* red */
  );
}
```

### 3. Data Visualization Donut Chart

```css
.donut-chart {
  --segment-1: 35%;  /* Revenue */
  --segment-2: 25%;  /* Expenses */
  --segment-3: 40%;  /* Profit */

  width: 200px;
  height: 200px;
  border-radius: 50%;
  background: conic-gradient(
    #3b82f6 0% var(--segment-1),
    #8b5cf6 var(--segment-1) calc(var(--segment-1) + var(--segment-2)),
    #10b981 calc(var(--segment-1) + var(--segment-2)) 100%
  );
  mask: radial-gradient(circle, transparent 55px, black 56px);
}
```

The `mask` approach creates a transparent hole without relying on a matching background color, making it work on any parent background.

---

> **Key Takeaway:** Master the background shorthand's slash rule and reset behavior, use multiple backgrounds to avoid wrapper elements, default to OKLCH interpolation for vibrant gradients, and remember that gradient animation requires either `background-position` shifts or `@property`-registered custom properties.
