# 2.6 Flexbox

> Flexbox is a one-dimensional layout model for distributing space and aligning items along a single axis. It excels at component-level layout: navigation bars, card content, centering, and distributing items of unequal sizes.

---

## Core Model

Setting `display: flex` (or `display: inline-flex`) on an element makes it a flex container. Its direct children become flex items. Only direct children participate in flex layout.

### The Two Axes

Everything in flexbox revolves around two axes:

- **Main axis**: defined by `flex-direction`. Items are laid out along this axis.
- **Cross axis**: perpendicular to the main axis.

| `flex-direction` | Main axis | Cross axis |
|---|---|---|
| `row` (default) | Horizontal (left to right in LTR) | Vertical |
| `row-reverse` | Horizontal (right to left in LTR) | Vertical |
| `column` | Vertical (top to bottom) | Horizontal |
| `column-reverse` | Vertical (bottom to top) | Horizontal |

### Default Behavior

When you set `display: flex` with no other properties, the defaults are:

```css
.container {
  display: flex;
  /* These are the implicit defaults: */
  flex-direction: row;
  flex-wrap: nowrap;
  justify-content: flex-start;
  align-items: stretch;
  align-content: stretch;
}

.item {
  /* These are the implicit defaults: */
  flex-grow: 0;    /* items do NOT grow */
  flex-shrink: 1;  /* items CAN shrink */
  flex-basis: auto; /* size from width/height or content */
  order: 0;
  align-self: auto;
}
```

Items display in a single row, packed to the start, stretched to fill the cross axis, sized by their content, and allowed to shrink (but not grow).

---

## Container Properties

### `flex-direction`

Controls the main axis direction:

```css
.row { flex-direction: row; }           /* default, horizontal */
.row-rev { flex-direction: row-reverse; }
.col { flex-direction: column; }        /* vertical */
.col-rev { flex-direction: column-reverse; }
```

### `flex-wrap`

Controls whether items wrap to new lines:

```css
.no-wrap { flex-wrap: nowrap; }    /* default, single line */
.wrap { flex-wrap: wrap; }         /* items wrap to new lines */
.wrap-rev { flex-wrap: wrap-reverse; } /* new lines added in reverse direction */
```

### `flex-flow` Shorthand

Combines `flex-direction` and `flex-wrap`:

```css
.container { flex-flow: row wrap; }
/* Equivalent to: flex-direction: row; flex-wrap: wrap; */
```

### `gap`

Defines space between items without affecting edges:

```css
.container {
  display: flex;
  gap: 16px;          /* equal row and column gap */
  gap: 16px 24px;     /* row-gap column-gap */
}
```

`gap` is superior to margins for flex item spacing:
- No "double margin" between items.
- No `:first-child`/`:last-child` overrides needed.
- No negative margin hack on the container.
- Works correctly with `flex-wrap: wrap`.

## Alignment Properties

### `justify-content` (Main Axis)

Controls how items are distributed along the main axis:

```css
.container { display: flex; }
```

| Value | Behavior |
|---|---|
| `flex-start` | Pack at start (default) |
| `flex-end` | Pack at end |
| `center` | Center items |
| `space-between` | First item at start, last at end, equal space between |
| `space-around` | Equal space around each item (half-space at edges) |
| `space-evenly` | Equal space everywhere, including edges |

```css
/* Centered navigation */
.nav { display: flex; justify-content: center; gap: 2rem; }

/* Logo left, links right */
.header { display: flex; justify-content: space-between; align-items: center; }
```

### `align-items` (Cross Axis, Per Line)

Controls how items are aligned within their flex line on the cross axis:

| Value | Behavior |
|---|---|
| `stretch` | Fill cross axis (default). Only works if no cross-axis size is set. |
| `flex-start` | Align to cross-axis start |
| `flex-end` | Align to cross-axis end |
| `center` | Center on cross axis |
| `baseline` | Align by text baseline |

```css
/* Vertically centered items */
.toolbar { display: flex; align-items: center; }
```

### `align-self` (Individual Item Override)

Overrides `align-items` for a single item:

```css
.container { display: flex; align-items: flex-start; }
.special-item { align-self: center; }
```

### `align-content` (Wrapped Lines)

Controls how wrapped flex lines are distributed on the cross axis. Only works when `flex-wrap: wrap` produces multiple lines. Has no effect on single-line flex containers.

| Value | Behavior |
|---|---|
| `stretch` | Lines stretch to fill the container (default) |
| `flex-start` | Lines packed at cross-axis start |
| `flex-end` | Lines packed at cross-axis end |
| `center` | Lines centered |
| `space-between` | First line at start, last at end, equal space between |
| `space-around` | Equal space around each line |
| `space-evenly` | Equal space everywhere |

**Key distinction:** `align-items` aligns items within each line. `align-content` aligns the lines themselves within the container. Confusing these is one of the most common flexbox mistakes.

### `place-content` Shorthand

Sets `align-content` and `justify-content` together:

```css
/* Perfect centering */
.container {
  display: flex;
  place-content: center;
  /* Equivalent to: align-content: center; justify-content: center; */
}
```

## The `flex` Shorthand

The `flex` property is shorthand for `flex-grow`, `flex-shrink`, and `flex-basis`. Understanding all three components is essential.

### `flex-basis`

Sets the initial main size of a flex item before free space is distributed.

```css
.item { flex-basis: 200px; }
/* In row direction: initial width is 200px */
/* In column direction: initial height is 200px */
```

- In `row` direction, `flex-basis` acts like `width`. In `column`, it acts like `height`.
- When set to `auto` (default), it looks at the item's `width`/`height`, then falls back to content size.
- **Takes priority over `width`** when both are set on a flex item in `row` direction.
- Can accept lengths, percentages, `auto`, `content`, `min-content`, `max-content`, `fit-content`.

### `flex-grow`

Controls how positive free space (leftover space) is distributed among items:

```css
.container { display: flex; width: 600px; }
.a { flex-grow: 2; flex-basis: 100px; }
.b { flex-grow: 1; flex-basis: 100px; }
.c { flex-grow: 1; flex-basis: 100px; }
/* Free space: 600 - 300 = 300px
   .a gets 2/4 * 300 = 150px -> total 250px
   .b gets 1/4 * 300 = 75px  -> total 175px
   .c gets 1/4 * 300 = 75px  -> total 175px */
```

The value is a proportion of the free space, not the total size. Default is `0` (items do not grow).

### `flex-shrink`

Controls how negative free space (overflow) is absorbed when items are too large:

```css
.container { display: flex; width: 300px; }
.a { flex-shrink: 2; flex-basis: 200px; }
.b { flex-shrink: 1; flex-basis: 200px; }
/* Overflow: (200 + 200) - 300 = 100px
   Shrink is weighted by basis:
   .a shrinks: (2*200) / (2*200 + 1*200) * 100 = 66.7px -> 133.3px
   .b shrinks: (1*200) / (2*200 + 1*200) * 100 = 33.3px -> 166.7px */
```

The shrink factor is weighted by the item's `flex-basis`. This prevents small items from disappearing to zero before large items noticeably shrink. Default is `1`.

### Named Shorthand Values

| Shorthand | Expands to | Use case |
|---|---|---|
| `flex: initial` | `0 1 auto` | Default. Cannot grow, can shrink, content-sized. |
| `flex: auto` | `1 1 auto` | Grow and shrink from content size. Content-aware distribution. |
| `flex: none` | `0 0 auto` | Fully inflexible. Fixed-size items. |
| `flex: 1` | `1 1 0%` | Grow and shrink from zero basis. Equal-width items. |
| `flex: <number>` | `<number> 1 0%` | Single number sets basis to `0%`, NOT `auto`. |

### The Critical `flex: 1` vs `flex: auto` Distinction

This is the most common source of flexbox sizing confusion:

```css
.container { display: flex; width: 600px; }

/* flex: 1 (basis: 0%) -- items ignore content, share space equally */
.item-a { flex: 1; }  /* 200px each */
.item-b { flex: 1; }  /* regardless of content length */
.item-c { flex: 1; }

/* flex: auto (basis: auto) -- items size to content FIRST, then share leftover */
.item-a { flex: auto; /* short text */ }   /* smaller */
.item-b { flex: auto; /* longer text */ }  /* larger */
.item-c { flex: auto; /* medium text */ }  /* medium */
```

`flex: 1` distributes total space equally. `flex: auto` distributes leftover space equally (after content sizing). Choose based on whether you want content to influence sizing.

## The Flex Sizing Algorithm

Understanding the algorithm helps debug unexpected sizes:

1. **Determine each item's hypothetical main size** from `flex-basis` (or `width`/`height` if basis is `auto`, or content size if neither is set).
2. **Clamp** each hypothetical size to `min-width`/`max-width` (or height equivalents in column).
3. **Calculate free space**: container main size minus the sum of all hypothetical sizes, margins, borders, padding, and gaps.
4. **If positive free space**: distribute according to `flex-grow` ratios.
5. **If negative free space**: absorb according to `flex-shrink` factors (weighted by basis).
6. **Re-check min/max constraints**: if any item exceeds its max or goes below its min, freeze it at the constraint and redistribute remaining space. Repeat until resolved.

## `flex-basis` vs `width`

| Aspect | `flex-basis` | `width` |
|---|---|---|
| Scope | Only flex items | Any element |
| Direction-aware | Yes (width in row, height in column) | No (always horizontal) |
| Priority in flex context | Wins over `width` when both are set | Used as fallback when `flex-basis: auto` |
| In `flex` shorthand | Set explicitly | May be silently overridden |

```css
/* BAD: flex shorthand overrides width */
.item {
  width: 200px;
  flex: 1; /* expands to flex: 1 1 0% -- basis 0% overrides width */
}

/* GOOD: explicit basis */
.item {
  flex: 1 1 200px;
}
```

Best practice: use `flex-basis` for the main-axis dimension. Use `width`/`height` for the cross-axis dimension.

## The `min-width: auto` Problem

By spec, flex items have `min-width: auto` and `min-height: auto` by default. This means a flex item will never shrink smaller than its content's intrinsic minimum size.

### The Problem

```css
.container { display: flex; width: 300px; }
.sidebar { flex: 0 0 200px; }
.content { flex: 1; }
/* If .content has a long URL or wide image, it overflows
   because min-width: auto prevents it from shrinking below content size */
```

### Three Solutions

```css
/* Solution 1: Override min-width */
.content {
  flex: 1;
  min-width: 0; /* allows shrinking below content size */
}

/* Solution 2: Overflow (implicitly resets min-width) */
.content {
  flex: 1;
  overflow: hidden; /* or auto, scroll, clip */
}

/* Solution 3: Make content breakable */
.content {
  flex: 1;
  overflow-wrap: anywhere;
}
```

This also applies to `min-height: auto` in `flex-direction: column`.

### Common Symptom: Truncated Text Fails

```css
/* BAD: text-overflow does not work */
.item {
  flex: 1;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
/* The item itself still overflows the flex container! */

/* GOOD: add min-width: 0 */
.item {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## Auto Margins in Flex

In flex containers, `margin: auto` absorbs all available free space in that direction. This is more powerful than in block layout.

```css
/* Push last item to the right */
.nav { display: flex; gap: 1rem; }
.nav .push-right { margin-left: auto; }
```

```html
<nav class="nav">
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="/contact" class="push-right">Contact</a>
</nav>
<!-- Home and About are left-aligned. Contact is right-aligned. -->
```

### Perfect Centering with Auto Margins

```css
.container { display: flex; height: 400px; }
.centered {
  margin: auto;
  /* Centers both horizontally and vertically */
}
```

**Important:** Auto margins consume free space before `justify-content` and `align-items` run. If any item has auto margins, the alignment properties are effectively overridden on that axis.

## Wrapping Behavior

### `flex-wrap: wrap`

Items break to new lines when they exceed the container width:

```css
.grid-like {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card {
  flex: 1 1 300px;
  /* Starts at 300px, grows to fill space, wraps when < 300px would be needed */
}
```

### Critical Wrapping Concepts

**1. Each flex line is independent.** Items in one line cannot align with items in another line. This is the fundamental difference from CSS Grid.

**2. `flex-grow` operates per line.** An item with `flex-grow: 1` on a line with 2 items gets more space than the same item on a line with 5 items.

**3. The last-row problem.** If you have 7 items wrapping into rows of 3, the last row has 1 item that will grow to fill the entire line if `flex-grow` is set:

```css
/* Problem: last row's single item stretches to full width */
.container { display: flex; flex-wrap: wrap; }
.item { flex: 1 1 calc(33.333% - 1rem); }
/* Last item on a row of 1 fills 100% */
```

Workarounds:
- Add invisible spacer elements (hacky).
- Set `max-width` on items.
- Use CSS Grid instead (recommended for grid-like layouts).

## The `order` Property

Changes visual display order without changing DOM order:

```css
.first-visually { order: -1; }  /* appears first */
.default { order: 0; }          /* normal position (default) */
.last-visually { order: 1; }    /* appears last */
```

Items are sorted by `order` value (ascending). Items with the same value maintain their source order.

### Accessibility Warning

`order` does NOT change:
- Screen reader reading order (follows DOM).
- Keyboard tab order (follows DOM).
- Content meaning or sequence.

Using `order` to significantly rearrange interactive elements violates WCAG 1.3.2 (Meaningful Sequence). A keyboard user will tab through items in DOM order, jumping around the visual layout unpredictably.

**Rule:** Fix the DOM order to match the desired reading/tab order. Use `order` only for minor decorative adjustments that do not affect interactive content.

## Images in Flexbox

Images have special behavior in flex containers:

```css
/* Problem: image stretches due to align-items: stretch */
.container { display: flex; }
img { /* stretches vertically to fill cross axis */ }

/* Fix 1: prevent stretching */
.container { display: flex; align-items: flex-start; }

/* Fix 2: per-item override */
img { align-self: flex-start; }

/* Problem: image overflows container */
img { /* intrinsic size exceeds flex item space */ }

/* Fix: constrain image */
img {
  max-width: 100%;
  height: auto;
  object-fit: cover; /* or contain */
}
```

## Flexbox vs Grid

| Criterion | Flexbox | Grid |
|---|---|---|
| Dimensionality | 1D (single axis) | 2D (rows and columns) |
| Layout approach | Content-out (content drives size) | Layout-in (grid defines size) |
| Cross-line alignment | Items in different lines are independent | Items align across rows and columns |
| Best for | Navbars, toolbars, card internals, centering | Page layouts, galleries, dashboards |
| Dynamic item counts | Natural wrapping and flexing | `auto-fill`/`auto-fit` for responsive grids |
| Overlap | Not possible | Items can overlap via `grid-area` |

**Rule of thumb:** If items need to align in both rows and columns, use Grid. If you are distributing items along a single axis, use Flexbox. They complement each other: Grid for page structure, Flexbox for component internals.

## Common Mistakes

1. **`flex: 1` when you mean `flex: auto`.** `flex: 1` sets basis to `0%`, making all items equal-width regardless of content. `flex: auto` respects content size first. Choose intentionally.

2. **Ignoring `min-width: auto`.** Items overflow instead of shrinking because the default minimum prevents them from going below content size. Set `min-width: 0` or use `overflow: hidden`.

3. **Confusing `align-items` with `align-content`.** `align-items` aligns items within each line. `align-content` aligns the lines themselves. `align-content` only works with `flex-wrap: wrap`.

4. **Using `order` for meaningful reordering.** Screen readers and keyboard navigation follow DOM order, not visual order. Fix the DOM instead.

5. **Setting `width` and then using `flex: 1`.** The flex shorthand `flex: 1` expands to `1 1 0%`, which overrides `width` with a basis of `0%`.

6. **Expecting grid-like alignment from wrapped flex.** Each flex line is independent. Items in different lines do not align vertically. Use CSS Grid for true grid alignment.

7. **Forgetting `align-items: start` for sticky sidebars.** The default `stretch` makes items fill the cross axis, breaking `position: sticky` behavior.

8. **Last-row stretching with `flex-grow`.** The last wrapped row may have fewer items, causing them to grow disproportionately. Use `max-width` or Grid instead.

9. **Auto margins overriding alignment.** Auto margins consume free space before `justify-content` runs. If an item has `margin-left: auto`, `justify-content` is effectively ignored.

10. **Images distorting in flex containers.** The default `align-items: stretch` stretches images along the cross axis. Set `align-self: flex-start` on images or `object-fit: cover`.

---

## Best Practices

1. **Use the `flex` shorthand, not individual properties.** `flex: 1 0 200px` is clearer than setting `flex-grow`, `flex-shrink`, and `flex-basis` separately. The shorthand also sets sensible defaults for omitted values.

2. **Always set `min-width: 0` (or `overflow: hidden`) on items that need to shrink below content size.** This prevents the most common flexbox overflow bug.

3. **Use `gap` instead of margins for spacing.** Gap only creates space between items, not at edges, and works correctly with wrapping.

4. **Use `flex: auto` for content-aware flexible items, `flex: 1` for equal-width items.** Understand the difference and choose deliberately.

5. **Prefer Flexbox for 1D layout, Grid for 2D layout.** Do not force flexbox into grid-like patterns with wrapping when Grid is the right tool.

6. **Use auto margins for alignment tricks.** `margin-left: auto` on a flex item pushes it and subsequent items to the end. Simpler than wrapper elements.

7. **Debug with browser DevTools.** Chrome, Firefox, and Edge all have dedicated Flexbox inspectors showing container boundaries, item sizing, and free space distribution.

## Real-World Use Cases

### 1. Navigation Bar with Logo and Actions

```css
.navbar {
  display: flex;
  align-items: center;
  padding: 0.75rem 1.5rem;
  gap: 1rem;
}

.navbar .logo {
  flex: none; /* fixed size, no grow/shrink */
}

.navbar .nav-links {
  display: flex;
  gap: 1.5rem;
  list-style: none;
  margin: 0;
  padding: 0;
}

.navbar .actions {
  margin-left: auto; /* pushes to the right */
  display: flex;
  gap: 0.5rem;
}
```

### 2. Card with Footer Pushed to Bottom

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1.5rem;
}

.card {
  flex: 1 1 300px;
  max-width: 400px;
  display: flex;
  flex-direction: column;
  border: 1px solid #e0e0e0;
  border-radius: 8px;
  overflow: clip;
}

.card__image {
  width: 100%;
  aspect-ratio: 16 / 9;
  object-fit: cover;
}

.card__body {
  flex: 1; /* grows to fill space, pushing footer down */
  padding: 1rem;
}

.card__footer {
  padding: 1rem;
  border-top: 1px solid #e0e0e0;
  /* Always at the bottom, regardless of body content length */
}
```

### 3. Sticky Footer Page Layout

```css
html, body {
  height: 100%;
  margin: 0;
}

.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.header {
  flex: none; /* auto-sized, no grow */
}

.main {
  flex: 1; /* fills remaining vertical space */
}

.footer {
  flex: none; /* auto-sized, no grow */
}
```

```html
<div class="page">
  <header class="header">Header</header>
  <main class="main">Content (even if short, footer stays at bottom)</main>
  <footer class="footer">Footer</footer>
</div>
```

### 4. Centered Content with Truncation

```css
.list-item {
  display: flex;
  align-items: center;
  gap: 0.75rem;
  padding: 0.5rem 1rem;
}

.list-item__icon {
  flex: none;
  width: 24px;
  height: 24px;
}

.list-item__text {
  flex: 1;
  min-width: 0; /* critical: allows shrinking below content size */
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.list-item__badge {
  flex: none;
  padding: 2px 8px;
  border-radius: 12px;
  background: #e0e0e0;
  font-size: 0.75rem;
}
```

---

> **Key Takeaway:** Flexbox distributes space along a single axis using the `flex-grow`, `flex-shrink`, and `flex-basis` algorithm; the most critical things to master are the difference between `flex: 1` (equal sizing from zero basis) and `flex: auto` (content-aware sizing), the `min-width: auto` overflow trap (fix with `min-width: 0`), and the fact that each wrapped line is an independent flex context (use Grid when you need cross-line alignment).
