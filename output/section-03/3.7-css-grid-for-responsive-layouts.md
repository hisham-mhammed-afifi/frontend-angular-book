# 3.7 CSS Grid for Responsive Layouts

> CSS Grid's intrinsic sizing capabilities can build responsive layouts that adapt to any container width without a single media query. This guide focuses on responsive Grid patterns, not Grid fundamentals (covered in 2.7).

---

## The Core Responsive Pattern: repeat() + auto-fit + minmax()

This single declaration creates a responsive column layout that wraps automatically:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}
```

Columns are at least 250px wide, expand with `1fr` to fill available space, and wrap to fewer columns as the container narrows. Zero media queries.

**Behavior at different container widths (250px minimum, 1.5rem gap):**

| Container Width | Columns | Approximate Column Width |
|----------------|---------|-------------------------|
| 1200px | 4 | ~285px |
| 800px | 3 | ~250px |
| 530px | 2 | ~253px |
| 300px | 1 | 300px |

---

## auto-fit vs auto-fill

Both create as many tracks as can fit. The difference appears when there are fewer items than available tracks.

```css
/* auto-fill: keeps empty tracks, items stay at minimum size */
.grid-fill {
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}

/* auto-fit: collapses empty tracks, items stretch to fill */
.grid-fit {
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
}
```

With 3 items in a 1200px container where 5 tracks of 200px could fit:
- **auto-fill:** 5 tracks. Items occupy the first 3 at ~230px each. 2 empty tracks take up space.
- **auto-fit:** 5 tracks created, 2 empty ones collapse to 0. Items stretch to ~390px each.

| Use auto-fill when | Use auto-fit when |
|-------------------|-------------------|
| Consistent card widths matter | Items should stretch to fill space |
| Many items, consistent grid | Few items, need to fill the row |
| Photo galleries, icon grids | Feature cards, dashboard widgets |

When items fill all tracks, both produce identical results.

**Common mistake:** Using `auto-fit` for a product grid. If only 1-2 products match a filter on a wide screen, cards stretch to absurd widths. Use `auto-fill` to keep consistent sizing.

## Preventing Overflow with min()

The basic pattern has a critical flaw: when the container is narrower than the minimum (250px), the column overflows.

```css
/* Bad Practice: overflows on containers < 250px */
grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));

/* Good: prevents overflow at any width */
grid-template-columns: repeat(auto-fit, minmax(min(100%, 250px), 1fr));
```

`min(100%, 250px)` returns whichever is smaller:
- Container = 400px: `min(400px, 250px)` = 250px. Normal behavior.
- Container = 200px: `min(200px, 250px)` = 200px. Single column fits.

### Reusable Version with Custom Properties

```css
.grid {
  --col-min: 250px;
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, var(--col-min)), 1fr));
  gap: 1.5rem;
}
```

Override `--col-min` per context:

```css
.card-grid   { --col-min: 280px; }
.icon-grid   { --col-min: 120px; }
.feature-grid { --col-min: 350px; }
```

## fr vs Percentage

`fr` distributes remaining space *after* gaps and fixed tracks are subtracted. Percentages are calculated against the total container width, ignoring gaps.

```css
/* Bad Practice: percentages do not account for gaps */
.grid {
  grid-template-columns: 33.33% 33.33% 33.33%;
  gap: 20px;
  /* Total: 99.99% + 40px of gaps = overflow */
}

/* Good: fr accounts for gaps automatically */
.grid {
  grid-template-columns: 1fr 1fr 1fr;
  gap: 20px;
  /* Leftover: container - 40px gaps = distributed equally */
}
```

### The Hidden auto Minimum in 1fr

`1fr` is shorthand for `minmax(auto, 1fr)`. The `auto` minimum prevents columns from shrinking below their `min-content` size. This means columns with long words or images can blow out the grid.

```css
/* Columns may not be equal if content varies */
grid-template-columns: 1fr 1fr 1fr;

/* Truly equal columns that can shrink to zero */
grid-template-columns: repeat(3, minmax(0, 1fr));
```

When using `minmax(0, 1fr)`, also add overflow protection to grid items:

```css
.grid > * {
  min-width: 0;
  overflow-wrap: break-word;
}

.grid img {
  max-width: 100%;
  height: auto;
}
```

## Grid Template Areas for Layout Restructuring

When the layout fundamentally changes across breakpoints (not just column count), `grid-template-areas` provides a visual, readable way to restructure.

```html
<div class="layout">
  <header class="header">Header</header>
  <nav class="nav">Navigation</nav>
  <main class="content">Main Content</main>
  <aside class="sidebar">Sidebar</aside>
  <footer class="footer">Footer</footer>
</div>
```

```css
/* Assign area names once */
.header  { grid-area: header; }
.nav     { grid-area: nav; }
.content { grid-area: content; }
.sidebar { grid-area: sidebar; }
.footer  { grid-area: footer; }

/* Mobile: single column, stacked */
.layout {
  display: grid;
  grid-template-areas:
    "header"
    "nav"
    "content"
    "sidebar"
    "footer";
  grid-template-columns: 1fr;
  gap: 0;
}

/* Tablet: content + sidebar */
@media (min-width: 768px) {
  .layout {
    grid-template-areas:
      "header  header"
      "nav     nav"
      "content sidebar"
      "footer  footer";
    grid-template-columns: 1fr 300px;
  }
}

/* Desktop: three-column holy grail */
@media (min-width: 1024px) {
  .layout {
    grid-template-areas:
      "header header  header"
      "nav    content sidebar"
      "footer footer  footer";
    grid-template-columns: 200px 1fr 300px;
    grid-template-rows: auto 1fr auto;
    min-height: 100vh;
  }
}
```

The layout restructures completely at each breakpoint by changing the area string. No DOM reordering, no absolute positioning.

**Accessibility note:** `grid-template-areas` allows visual reordering, but DOM order determines keyboard tab order and screen reader reading order. Ensure visual and DOM order convey the same meaning.

## Responsive Sidebar Patterns

### Pattern 1: Sidebar with Media Query

```css
.sidebar-layout {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1.5rem;
}

@media (min-width: 768px) {
  .sidebar-layout {
    grid-template-columns: minmax(200px, 25%) 1fr;
  }
}
```

### Pattern 2: Intrinsic Sidebar (No Media Query)

```css
.sidebar-layout {
  display: grid;
  grid-template-columns: fit-content(300px) minmax(50%, 1fr);
  gap: 1.5rem;
}
```

`fit-content(300px)` makes the sidebar shrink to its content but never exceed 300px. `minmax(50%, 1fr)` ensures the main content always gets at least half the width.

### Pattern 3: Full-Height Sticky Sidebar

```css
.sidebar-layout {
  display: grid;
  grid-template-columns: 280px 1fr;
  gap: 2rem;
  align-items: start;
}

.sidebar {
  position: sticky;
  top: 1rem;
}

@media (max-width: 767px) {
  .sidebar-layout {
    grid-template-columns: 1fr;
  }

  .sidebar {
    position: static;
  }
}
```

### Pancake Stack (Header / Main / Footer)

The simplest full-page layout. The middle row takes all remaining space.

```css
.page {
  display: grid;
  grid-template-rows: auto 1fr auto;
  min-height: 100svh;
}
```

## Subgrid for Aligned Responsive Components

Without subgrid, cards in a grid have independent internal layouts. A card with a long title pushes its body and footer down, misaligning them with adjacent cards.

```css
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 280px), 1fr));
  grid-auto-rows: auto 1fr auto; /* title, body, footer */
  gap: 1.5rem;
}

.card {
  display: grid;
  grid-template-rows: subgrid;
  grid-row: span 3;
  border: 1px solid #ddd;
  border-radius: 8px;
  padding: 1rem;
  gap: 0.75rem;
}
```

Now every card's title row aligns, body rows fill to the tallest, and footers sit at the same baseline across sibling cards.

### Subgrid on One Axis

You can borrow parent tracks on one axis and define your own on the other:

```css
.card {
  display: grid;
  grid-template-rows: subgrid;           /* inherit parent row tracks */
  grid-template-columns: 80px 1fr;       /* define own columns */
  grid-row: span 3;
}
```

**Browser support:** Chrome 117+, Firefox 71+, Safari 16+. ~97% global coverage.

## Grid + Container Queries

Media queries respond to the viewport. Container queries respond to the parent element's size. This makes Grid-based components reusable in any context.

```css
/* Declare the container */
.card-wrapper {
  container-type: inline-size;
}

/* Default: vertical card */
.card {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

/* When container is wide enough: horizontal card */
@container (min-width: 500px) {
  .card {
    grid-template-columns: 200px 1fr;
  }
}

@container (min-width: 700px) {
  .card {
    grid-template-columns: 250px 1fr 150px;
  }
}
```

The same card component adapts based on where it is placed: full-width main area, sidebar, modal.

### Grid Layout with Container-Queried Children

```css
/* Page layout with Grid */
.page {
  display: grid;
  grid-template-columns: 300px 1fr;
  gap: 2rem;
}

/* Both areas are containers */
.sidebar     { container-type: inline-size; }
.main-content { container-type: inline-size; }

/* A product card placed in either area adapts */
@container (min-width: 400px) {
  .product-card {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 1rem;
  }
}
```

## Dense Auto-Placement

`grid-auto-flow: dense` backtracks to fill gaps left by spanning items.

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
  grid-auto-rows: 150px;
  grid-auto-flow: dense;
  gap: 4px;
}

.gallery .wide     { grid-column: span 2; }
.gallery .tall     { grid-row: span 2; }
.gallery .featured { grid-column: span 2; grid-row: span 2; }
```

**This is NOT masonry.** Items still occupy strict row/column tracks. A short item in a tall row has whitespace within its cell. True masonry (items packing tightly with no row alignment) requires native CSS masonry, which is experimental.

**Accessibility warning:** Dense packing reorders items visually. Screen readers and keyboard navigation follow DOM order. Only use `dense` for content where order is unimportant (image galleries, tag clouds).

### Future: Native Masonry

```css
/* Experimental. Behind flags in all browsers as of Feb 2026. */
.masonry {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 250px), 1fr));
  grid-template-rows: masonry;
}

/* Progressive enhancement */
@supports (grid-template-rows: masonry) {
  .masonry {
    grid-template-rows: masonry;
  }
}
```

Not production-ready. Expected stable support: mid-2026.

## Fluid Gap Spacing

Use `clamp()` for gaps that scale smoothly between viewports.

```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 250px), 1fr));
  gap: clamp(0.75rem, 1.5vw + 0.25rem, 2rem);
}
```

- 320px viewport: gap = 0.75rem (floor).
- 768px viewport: gap ~1.4rem (fluid).
- 1440px viewport: gap = 2rem (ceiling).

### Separate Row and Column Gaps

```css
.grid {
  row-gap: clamp(1rem, 2vw + 0.5rem, 2.5rem);
  column-gap: clamp(0.75rem, 1.5vw, 2rem);
}
```

### Fluid Gap System with Custom Properties

```css
:root {
  --gap-sm: clamp(0.5rem, 1vw + 0.25rem, 1rem);
  --gap-md: clamp(0.75rem, 1.5vw + 0.375rem, 1.5rem);
  --gap-lg: clamp(1rem, 2vw + 0.5rem, 2.5rem);
}

.card-grid { gap: var(--gap-md); }
.section-grid { gap: var(--gap-lg); }
.icon-grid { gap: var(--gap-sm); }
```

## Handling Implicit Rows

When dynamic content generates more items than explicit rows define, implicit rows default to `auto`. Control them with `grid-auto-rows`:

```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 250px), 1fr));
  grid-auto-rows: minmax(200px, auto); /* implicit rows are at least 200px */
  gap: 1rem;
}
```

For equal-height card images regardless of content:

```css
.grid {
  grid-auto-rows: 1fr; /* all implicit rows are equal height */
}
```

## Common Mistakes

- **Omitting `min()` inside `minmax()` with `auto-fit`/`auto-fill`.** The grid overflows on containers narrower than the minimum. Always use `minmax(min(100%, Xpx), 1fr)`.

- **Using percentages for column widths with gaps.** `50% + 50% + gap = overflow`. Use `fr` instead. It automatically subtracts gaps before distributing space.

- **Content blowing out grid tracks.** Long URLs, unbreakable strings, or images without `max-width: 100%` force tracks wider than defined. Set `min-width: 0` on grid items and `overflow-wrap: break-word`.

- **Not styling implicit rows.** Dynamic grids generate implicit rows that default to `auto` height. Use `grid-auto-rows` to set a consistent minimum.

- **Using media queries when `auto-fit`/`auto-fill` suffices.** If the change is simply "fewer columns on smaller screens," the intrinsic pattern handles it without breakpoints. Reserve media queries for fundamental layout restructuring.

- **Using `grid-auto-flow: dense` on ordered content.** Dense packing reorders items visually, breaking the relationship between DOM order and visual order. Screen readers and keyboards follow DOM order.

- **Assuming `auto-fit` and `auto-fill` are the same.** They produce different results when items do not fill all available tracks. Test with few items on wide viewports.

- **Using `1fr` and expecting true zero-minimum columns.** `1fr` is `minmax(auto, 1fr)`. Use `minmax(0, 1fr)` for columns that can truly shrink to zero.

---

## Best Practices

- **Start with `repeat(auto-fit, minmax(min(100%, Xpx), 1fr))`** as your default responsive grid. It handles most cases without media queries.

- **Use `grid-template-areas` only when the layout fundamentally restructures** (sidebar moves from side to below content, nav jumps positions). Do not use it for simple column-count changes.

- **Pair Grid layout with container queries** for truly reusable components. The Grid provides the page structure; container queries let components adapt to their local context.

- **Use subgrid for card grids** where internal elements (title, body, footer) must align across sibling cards.

- **Use `clamp()` for gaps and padding** to create smooth scaling without media queries.

- **Set `min-width: 0` as a default** on grid items in contexts where content overflow is possible.

- **Use `auto-fill` for product/photo grids** where consistent sizing matters. Use `auto-fit` for feature sections where stretching is acceptable.

## Real-World Use Cases

### 1. E-Commerce Product Grid with Subgrid

```css
.product-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 260px), 1fr));
  grid-auto-rows: auto 1fr auto auto; /* image, title, price, button */
  gap: clamp(1rem, 2vw, 2rem);
}

.product-card {
  display: grid;
  grid-template-rows: subgrid;
  grid-row: span 4;
  border: 1px solid #e5e5e5;
  border-radius: 8px;
  overflow: clip;
}

.product-card img {
  width: 100%;
  aspect-ratio: 4 / 3;
  object-fit: cover;
}

.product-card__title {
  padding: 0.75rem 1rem 0;
  font-size: 1rem;
  font-weight: 600;
}

.product-card__price {
  padding: 0 1rem;
  font-size: 1.25rem;
  color: var(--text-primary);
}

.product-card__cta {
  margin: 0 1rem 1rem;
  padding: 0.5rem;
  background: var(--brand);
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}
```

### 2. Dashboard with Responsive Areas

```css
.dashboard {
  display: grid;
  grid-template-areas:
    "stats"
    "chart"
    "table";
  grid-template-columns: 1fr;
  gap: clamp(1rem, 2vw, 2rem);
  padding: clamp(1rem, 3vw, 2rem);
}

.stats { grid-area: stats; }
.chart { grid-area: chart; }
.table { grid-area: table; }

/* Stats row is itself a responsive grid */
.stats {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 200px), 1fr));
  gap: 1rem;
}

@media (min-width: 1024px) {
  .dashboard {
    grid-template-areas:
      "stats stats"
      "chart table";
    grid-template-columns: 1fr 400px;
  }
}

@media (min-width: 1400px) {
  .dashboard {
    grid-template-areas:
      "stats stats stats"
      "chart chart table";
    grid-template-columns: 1fr 1fr 400px;
  }
}
```

### 3. Magazine Layout with Dense Packing

```css
.magazine-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(min(100%, 200px), 1fr));
  grid-auto-rows: 200px;
  grid-auto-flow: dense;
  gap: 4px;
}

.article--featured {
  grid-column: span 2;
  grid-row: span 2;
}

.article--wide {
  grid-column: span 2;
}

.article--tall {
  grid-row: span 2;
}

.article img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}

/* On narrow screens, prevent spans from overflowing */
@media (max-width: 500px) {
  .article--featured,
  .article--wide {
    grid-column: span 1;
  }
}
```

## Pattern Quick Reference

| Pattern | Code | Needs Media Queries? |
|---------|------|---------------------|
| Auto-wrapping columns | `repeat(auto-fit, minmax(min(100%, 250px), 1fr))` | No |
| Layout restructuring | `grid-template-areas` at breakpoints | Yes |
| Intrinsic sidebar | `fit-content(300px) minmax(50%, 1fr)` | No |
| Fluid gaps | `gap: clamp(0.75rem, 1.5vw, 2rem)` | No |
| Card alignment | `grid-template-rows: subgrid` + `grid-row: span N` | No |
| Component responsive | `@container` queries on grid children | No (container) |
| Dense gallery | `grid-auto-flow: dense` with spanning items | No |
| Full-page stack | `grid-template-rows: auto 1fr auto; min-height: 100svh` | No |
| 12-column system | `repeat(12, 1fr)` with span utilities | Optional |

---

> **Key Takeaway:** `repeat(auto-fit, minmax(min(100%, 250px), 1fr))` is the foundation of responsive Grid layouts. It eliminates media queries for column wrapping, prevents overflow on narrow containers, and adapts to any width. Reserve media queries for fundamental layout restructuring with `grid-template-areas`, and use subgrid to align card internals across siblings.
