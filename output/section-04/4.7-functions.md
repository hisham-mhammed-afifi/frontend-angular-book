# 4.7 Functions (`@function`, `@return`, Built-in Modules)

> Functions compute and return values without emitting CSS. Combined with Sass's seven built-in modules, they enable unit conversions, color manipulation, collection operations, and compile-time calculations that keep stylesheets DRY and maintainable.

---

## Custom Functions

### Declaration and Return

```scss
@function double($value) {
  @return $value * 2;
}

.box {
  padding: double(8px);  // 16px
}
```

Key rules:
- Every code path must reach `@return`. A function without `@return` is a compile error.
- `@return` immediately ends the function (enables early return for guard clauses).
- A function can only return one value, but that value can be a list, map, or any Sass type.
- Functions are invoked inline in expressions, not with `@include`.

### Parameters

Functions support the same parameter patterns as mixins:

```scss
@use 'sass:math';

// Required parameters
@function spacing($multiplier) {
  @return 8px * $multiplier;
}

// Default values (can reference earlier parameters)
@function fluid-padding($base, $scale: $base * 0.5) {
  @return $base + $scale;
}

// Keyword arguments
.card {
  padding: fluid-padding($base: 16px, $scale: 8px);
}

// Rest arguments
@function sum($numbers...) {
  $total: 0;
  @each $n in $numbers {
    $total: $total + $n;
  }
  @return $total;
}

.box {
  width: sum(10px, 20px, 30px);  // 60px
}
```

### Pure Functions (No Side Effects)

Functions should only compute and return values. Sass strongly discourages side effects like setting global variables:

```scss
// BAD: Side effect in a function
$call-count: 0;
@function impure($value) {
  $call-count: $call-count + 1 !global;  // Side effect
  @return $value * 2;
}

// GOOD: Pure function
@function pure-double($value) {
  @return $value * 2;
}
```

Use mixins when you need side effects. Use functions only for computation.

### Early Return Pattern

```scss
@use 'sass:math';

@function safe-divide($a, $b) {
  @if $b == 0 {
    @warn "Division by zero in safe-divide(#{$a}, #{$b})";
    @return null;
  }
  @return math.div($a, $b);
}

.column {
  width: safe-divide(100%, 3);  // 33.3333%
}
```

### Input Validation with `@error`

```scss
@use 'sass:meta';
@use 'sass:math';

@function px-to-rem($px, $base: 16px) {
  @if meta.type-of($px) != 'number' {
    @error "px-to-rem() expects a number, got #{meta.type-of($px)}: #{meta.inspect($px)}";
  }
  @if math.is-unitless($px) {
    @error "px-to-rem() expects a value with px units, got unitless #{$px}";
  }
  @return math.div($px, $base) * 1rem;
}
```

`@error` stops compilation with a message. `@warn` prints a warning but continues.

## Functions vs Mixins

| Criteria | `@function` | `@mixin` |
|---|---|---|
| Purpose | Compute and return a value | Emit CSS declarations and rules |
| Invocation | Inline: `property: fn()` | With `@include mixin()` |
| Can output CSS | No | Yes |
| Can use `@content` | No | Yes |
| Side effects | Strongly discouraged | Acceptable |
| Returns | A value via `@return` | CSS output |

```scss
// FUNCTION: computes a value
@function spacing($n) {
  @return 8px * $n;
}

// MIXIN: emits CSS
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.card {
  @include flex-center;
  padding: spacing(3);  // 24px
}
```

Rule of thumb: if you need a **value**, write a function. If you need **CSS output**, write a mixin.

## Functions in the Module System

With `@use`, functions are accessed through namespaces:

```scss
// _utils.scss
@function spacing($n) {
  @return 8px * $n;
}

@function -internal-calc($x) {  // Private (- prefix)
  @return $x * 1.5;
}
```

```scss
// component.scss
@use 'utils';

.card {
  padding: utils.spacing(2);           // 16px
  // margin: utils.-internal-calc(10); // ERROR: private function
}
```

Custom namespace:

```scss
@use 'utils' as u;
.card { padding: u.spacing(2); }
```

## Built-in Module: `sass:math`

```scss
@use 'sass:math';
```

### Division

The `/` operator for division is deprecated. Use `math.div()`:

```scss
// Deprecated: $width / 3
$third: math.div(100%, 3);        // 33.3333%
$half: math.div(960px, 2);        // 480px
$ratio: math.div(100px, 50px);    // 2 (units cancel)
$speed: math.div(100px, 2s);      // 50px/s (complex units)
```

### Rounding and Clamping

```scss
math.ceil(4.2)                     // 5
math.floor(4.9)                    // 4
math.round(4.5)                    // 5
math.abs(-10px)                    // 10px

math.max(1rem, 2rem, 0.5rem)      // 2rem
math.min(100px, 50px)             // 50px
math.clamp(10px, 5px, 100px)      // 10px (clamped to min)
math.clamp(10px, 50px, 100px)     // 50px (within range)
```

`math.clamp()` requires all three values to have compatible units or all be unitless.

### Unit Inspection

```scss
math.is-unitless(42)               // true
math.is-unitless(42px)             // false
math.unit(42px)                    // "px"
math.unit(5px * 10px)             // "px*px"
math.compatible(10cm, 3mm)         // true
math.compatible(100px, 3em)        // false
```

### Other Functions

```scss
math.percentage(0.5)               // 50%
math.random()                      // Random decimal 0-1
math.random(100)                   // Random integer 1-100

// Exponential (require unitless inputs)
math.sqrt(100)                     // 10
math.pow(2, 8)                     // 256
math.log(100, 10)                  // 2

// Trigonometric
math.cos(0deg)                     // 1
math.sin(90deg)                    // 1
math.atan2(1, 1)                   // 45deg

// Constants
math.$pi                           // 3.1415926536
math.$e                            // 2.7182818285
```

## Built-in Module: `sass:color`

```scss
@use 'sass:color';
```

### The Three Manipulation Functions

These are the core of color work in Sass. Understanding their differences is critical:

**`color.adjust()`** changes channels by fixed amounts:

```scss
$brand: #3498db;

color.adjust($brand, $lightness: 20%)    // Adds 20% to lightness
color.adjust($brand, $lightness: -15%)   // Subtracts 15% from lightness
color.adjust($brand, $red: 40)           // Adds 40 to red channel
color.adjust($brand, $alpha: -0.3)       // Reduces alpha by 0.3
```

**`color.scale()`** moves channels proportionally toward their max or min:

```scss
$brand: #3498db;

color.scale($brand, $lightness: 50%)     // Move 50% toward white
color.scale($brand, $lightness: -50%)    // Move 50% toward black
color.scale($brand, $saturation: -30%)   // Reduce saturation by 30% of remaining
```

`scale()` is generally better for relative adjustments because it respects the current position. Scaling lightness by 50% on a light color produces a smaller absolute change than on a dark color.

**`color.change()`** sets channels to absolute values:

```scss
$brand: #3498db;

color.change($brand, $lightness: 90%)    // Set lightness to exactly 90%
color.change($brand, $alpha: 0.5)        // Set alpha to exactly 0.5
color.change($brand, $hue: 120deg)       // Set hue to green
```

### Color Mixing

```scss
color.mix(#3498db, white, 70%)           // 70% blue, 30% white
color.mix(#3498db, black, 80%)           // 80% blue, 20% black
color.mix(red, blue, 50%)               // Purple
```

### Other Color Functions

```scss
color.complement(#3498db)                // Rotate hue 180 degrees
color.invert(#3498db)                    // Negative
color.grayscale(#3498db)                 // Remove saturation

// Channel access
color.channel(#3498db, 'red')            // 52
color.channel(#3498db, 'lightness', $space: hsl)  // 53.3333%

// Color space inspection
color.space(#3498db)                     // rgb
color.is-legacy(#3498db)                 // true
color.is-in-gamut(oklch(50% 0.5 180))   // may be false
```

### Deprecated Global Functions

These still work but emit warnings and will be removed in Dart Sass 3.0.0:

| Deprecated | Replacement |
|---|---|
| `darken($color, $amount)` | `color.adjust($color, $lightness: -$amount)` |
| `lighten($color, $amount)` | `color.adjust($color, $lightness: $amount)` |
| `saturate($color, $amount)` | `color.adjust($color, $saturation: $amount)` |
| `desaturate($color, $amount)` | `color.adjust($color, $saturation: -$amount)` |
| `adjust-hue($color, $deg)` | `color.adjust($color, $hue: $deg)` |
| `opacify($color, $amount)` | `color.adjust($color, $alpha: $amount)` |
| `transparentize($color, $amount)` | `color.adjust($color, $alpha: -$amount)` |

## Built-in Module: `sass:map`

```scss
@use 'sass:map';
```

All map functions return new maps. Maps are immutable.

### Core Operations

```scss
$colors: (
  primary: #3498db,
  secondary: #2ecc71,
  danger: #e74c3c,
);

map.get($colors, primary)                // #3498db
map.get($colors, missing)                // null (no error)
map.has-key($colors, danger)             // true
map.keys($colors)                        // primary, secondary, danger
map.values($colors)                      // #3498db, #2ecc71, #e74c3c
map.remove($colors, danger)              // (primary: #3498db, secondary: #2ecc71)
map.merge($colors, (info: #1abc9c))      // Adds info key
map.set($colors, warning, #f39c12)       // Adds warning key
```

### Deep (Nested) Map Operations

```scss
$themes: (
  light: (bg: #fff, text: #333, border: #ddd),
  dark: (bg: #1a1a1a, text: #f0f0f0, border: #444),
);

// Deep get (multiple keys)
map.get($themes, dark, bg)               // #1a1a1a
map.get($themes, dark, missing)          // null

// Deep set
map.set($themes, dark, bg, #222)         // Returns updated map

// Deep merge (recursive: nested maps merged, not replaced)
$overrides: (dark: (bg: #222));
map.deep-merge($themes, $overrides)      // dark.text and dark.border preserved
```

## Built-in Module: `sass:list`

```scss
@use 'sass:list';
```

Lists are 1-indexed. All functions return new lists.

```scss
$sizes: 12px 14px 16px 18px 24px;

list.nth($sizes, 1)                      // 12px
list.nth($sizes, -1)                     // 24px (negative = from end)
list.length($sizes)                      // 5
list.index($sizes, 16px)                 // 3
list.index($sizes, 99px)                 // null

list.append($sizes, 32px)               // 12px 14px 16px 18px 24px 32px
list.set-nth($sizes, 2, 15px)           // 12px 15px 16px 18px 24px
list.join(10px 20px, 30px 40px)         // 10px 20px 30px 40px
list.separator($sizes)                   // space

// Zip creates pairs
list.zip(10px 20px, short long)         // 10px short, 20px long
```

Note: `list.nth()` throws an error on out-of-range indices (unlike `map.get()` which returns `null`).

## Built-in Module: `sass:string`

```scss
@use 'sass:string';
```

Strings are 1-indexed. Negative indices count from the end.

```scss
$name: 'Helvetica Neue';

string.length($name)                     // 14
string.index($name, 'Neue')             // 11
string.index($name, 'missing')          // null
string.slice($name, 1, 9)              // 'Helvetica'
string.slice($name, 11)                 // 'Neue'
string.to-upper-case($name)             // 'HELVETICA NEUE'
string.to-lower-case($name)             // 'helvetica neue'

string.insert($name, ' Bold', 10)       // 'Helvetica Bold Neue'
string.quote(sans-serif)                 // "sans-serif"
string.unquote("bold")                   // bold
string.unique-id()                       // e.g., "uabtrnzug"

// Split (Dart Sass 1.57.0+)
string.split('Segoe UI Emoji', ' ')     // ["Segoe", "UI", "Emoji"]
```

`to-upper-case()` and `to-lower-case()` only affect ASCII letters. Unicode characters pass through unchanged.

## Built-in Module: `sass:meta`

```scss
@use 'sass:meta';
```

### Type Inspection

```scss
meta.type-of(42px)                       // "number"
meta.type-of(red)                        // "color"
meta.type-of("hello")                    // "string"
meta.type-of(true)                       // "bool"
meta.type-of(null)                       // "null"
meta.type-of((a: 1))                     // "map"
meta.type-of(1px 2px)                    // "list"

meta.inspect(null)                       // "null" (as a string)
meta.inspect((a: 1, b: 2))             // "(a: 1, b: 2)"
```

### Existence Checks

```scss
meta.variable-exists('primary')          // true/false (name without $)
meta.function-exists('spacing')          // true/false
meta.mixin-exists('respond-to')          // true/false
meta.global-variable-exists('primary')   // true/false
meta.content-exists()                    // true if inside mixin with @content
```

### First-Class Functions

Functions can be stored as values and invoked dynamically:

```scss
@use 'sass:meta';
@use 'sass:list';

@function double($n) { @return $n * 2; }
@function triple($n) { @return $n * 3; }

@function apply-to-list($fn, $items) {
  $result: ();
  @each $item in $items {
    $result: list.append($result, meta.call($fn, $item));
  }
  @return $result;
}

$doubled: apply-to-list(meta.get-function('double'), 4px 8px 16px);
// Result: 8px 16px 32px

$tripled: apply-to-list(meta.get-function('triple'), 4px 8px 16px);
// Result: 12px 24px 48px
```

`meta.get-function($name, $css: false, $module: null)` retrieves a function as a value. Setting `$css: true` returns a plain CSS function reference.

### Module Introspection

```scss
@use 'sass:meta';
@use 'tokens';

// Get all variables from a module as a map
$all-vars: meta.module-variables('tokens');
// Returns: (primary: #3498db, secondary: #2ecc71, ...)

// Get all functions from a module
$all-fns: meta.module-functions('tokens');
```

### Dynamic CSS Loading

```scss
@use 'sass:meta';

.dark-theme {
  @include meta.load-css('themes/dark');
}
```

`meta.load-css()` includes a module's CSS output at the point of the `@include`. It does not make variables, functions, or mixins available.

## Built-in Module: `sass:selector`

```scss
@use 'sass:selector';

selector.nest('.parent', '.child')       // .parent .child
selector.nest('.alert', '&:hover')       // .alert:hover
selector.append('.card', '__header')     // .card__header
selector.unify('.card', '.featured')     // .card.featured

selector.simple-selectors('a.disabled')       // a, .disabled
selector.parse('.a .b, .c')                   // ((.a .b,), (.c,))
```

These functions are rarely needed in day-to-day work but are powerful for building mixin libraries that manipulate selectors programmatically.

## Plain CSS Functions in Sass

Sass must coexist with CSS functions like `calc()`, `min()`, `max()`, `clamp()`, `var()`, and `env()`. Here is how Sass handles them:

### `calc()` and `clamp()`

Sass variables work directly inside `calc()` without interpolation (Dart Sass 1.40.0+):

```scss
$sidebar-width: 300px;
$gap: 1rem;

.main {
  // Sass resolves $sidebar-width and $gap at compile time
  width: calc(100% - $sidebar-width - $gap);
}
```

Sass simplifies compatible units at compile time and passes incompatible units through to CSS:

```scss
// Sass simplifies this:
width: calc(100px + 200px);           // Output: width: 300px;

// Sass passes this through (incompatible units):
width: calc(100% - 300px);            // Output: width: calc(100% - 300px);
```

### `min()` and `max()`

Sass has its own `min()` and `max()` functions. To use the CSS versions, ensure arguments do not contain Sass-specific features:

```scss
// CSS min/max (passed through to browser)
width: min(100%, 600px);              // Contains no Sass features

// Sass min/max (evaluated at compile time)
width: min($a, $b);                   // Contains Sass variables

// Force CSS interpretation with interpolation
width: #{"min(100%, 600px)"};
```

## Common Mistakes

### Mistake 1: Using a function when a mixin is needed

```scss
// BAD: Function trying to emit CSS
@function card-styles() {
  // Cannot emit CSS from a function
  @return null;
}

// GOOD: Use a mixin for CSS output
@mixin card-styles { border: 1px solid #ddd; border-radius: 4px; }
```

### Mistake 2: Using interpolation to attach units

```scss
// BAD: Creates a string, not a number
@function to-px($n) {
  @return #{$n}px;  // Returns a string "16px", not the number 16px
}

// GOOD: Multiply to attach units
@function to-px($n) {
  @return $n * 1px;
}
```

### Mistake 3: Not validating `map.get()` results

```scss
// BAD: Silent failure if key is missing
@function get-color($name) {
  @return map.get($colors, $name);  // Returns null silently
}

// GOOD: Validate and error
@function get-color($name) {
  $value: map.get($colors, $name);
  @if $value == null {
    @error "Unknown color '#{$name}'. Available: #{map.keys($colors)}";
  }
  @return $value;
}
```

### Mistake 4: Using deprecated global functions

```scss
// BAD: Deprecated
$light: lighten(#3498db, 20%);
$half: percentage(0.5);
$third: 100% / 3;

// GOOD: Module functions
@use 'sass:color';
@use 'sass:math';

$light: color.adjust(#3498db, $lightness: 20%);
$half: math.percentage(0.5);
$third: math.div(100%, 3);
```

### Mistake 5: Expecting functions to work inside CSS custom properties

```scss
// BAD: Outputs literal function call text
:root {
  --spacing: spacing(2);  // Outputs: --spacing: spacing(2);
}

// GOOD: Use interpolation
:root {
  --spacing: #{spacing(2)};  // Outputs: --spacing: 16px;
}
```

### Mistake 6: Side effects in functions

```scss
// BAD: Global variable mutation in a function
$counter: 0;
@function next-id() {
  $counter: $counter + 1 !global;
  @return $counter;
}
```

Use a mixin for side effects. Functions should be pure.

## Best Practices

### Practice 1: Validate inputs for public functions

```scss
@function spacing($n) {
  @if meta.type-of($n) != 'number' or not math.is-unitless($n) {
    @error "spacing() expects a unitless number, got #{meta.inspect($n)}";
  }
  @return $n * 8px;
}
```

### Practice 2: Use `null` returns for "not found" scenarios

```scss
@function get-breakpoint($name) {
  @return map.get($breakpoints, $name);  // null if missing
}

// Caller decides how to handle null
$bp: get-breakpoint('md');
@if $bp {
  @media (min-width: $bp) { @content; }
}
```

### Practice 3: Prefer `color.scale()` over `color.adjust()` for relative changes

`scale()` produces more intuitive results because it respects the current value:

```scss
// Already-light color: adjust adds a fixed amount (can overflow)
color.adjust(#eee, $lightness: 20%)     // Might hit 100% ceiling

// scale moves proportionally toward the max
color.scale(#eee, $lightness: 50%)      // Moves halfway to white from current position
```

### Practice 4: Keep functions focused and composable

```scss
// Small, focused functions
@function strip-unit($n) {
  @return math.div($n, $n * 0 + 1);
}

@function px-to-rem($px, $base: 16px) {
  @return math.div($px, $base) * 1rem;
}

@function rem-to-px($rem, $base: 16px) {
  @return math.div($rem, 1rem) * $base;
}
```

## Real-World Use Cases

### Use Case 1: Type Scale System

```scss
@use 'sass:math';
@use 'sass:map';

$type-scale-ratio: 1.25 !default;  // Major Third
$type-base-size: 1rem !default;

@function type-scale($step) {
  @return $type-base-size * math.pow($type-scale-ratio, $step);
}

$type-sizes: (
  xs: type-scale(-2),    // 0.64rem
  sm: type-scale(-1),    // 0.8rem
  base: type-scale(0),   // 1rem
  md: type-scale(1),     // 1.25rem
  lg: type-scale(2),     // 1.5625rem
  xl: type-scale(3),     // 1.9531rem
  2xl: type-scale(4),    // 2.4414rem
  3xl: type-scale(5),    // 3.0518rem
);

@function font-size($name) {
  $size: map.get($type-sizes, $name);
  @if $size == null {
    @error "Unknown type size '#{$name}'. Available: #{map.keys($type-sizes)}";
  }
  @return $size;
}

h1 { font-size: font-size(3xl); }
h2 { font-size: font-size(2xl); }
h3 { font-size: font-size(xl); }
body { font-size: font-size(base); }
small { font-size: font-size(sm); }
```

### Use Case 2: Color Palette Generator

```scss
@use 'sass:color';
@use 'sass:map';

@function generate-shades($base-color) {
  @return (
    50:  color.scale($base-color, $lightness: 90%),
    100: color.scale($base-color, $lightness: 80%),
    200: color.scale($base-color, $lightness: 60%),
    300: color.scale($base-color, $lightness: 40%),
    400: color.scale($base-color, $lightness: 20%),
    500: $base-color,
    600: color.scale($base-color, $lightness: -20%),
    700: color.scale($base-color, $lightness: -40%),
    800: color.scale($base-color, $lightness: -60%),
    900: color.scale($base-color, $lightness: -80%),
  );
}

$blue-shades: generate-shades(#3498db);
$red-shades: generate-shades(#e74c3c);

// Generate CSS custom properties
@each $shade, $color in $blue-shades {
  :root {
    --blue-#{$shade}: #{$color};
  }
}
```

### Use Case 3: Fluid Typography with `clamp()`

```scss
@use 'sass:math';

$_fluid-min-bp: 320px !default;
$_fluid-max-bp: 1200px !default;

@function fluid($min-size, $max-size, $min-bp: $_fluid-min-bp, $max-bp: $_fluid-max-bp) {
  $slope: math.div($max-size - $min-size, $max-bp - $min-bp);
  $y-intercept: $min-size - $slope * $min-bp;

  $slope-vw: $slope * 100;
  $intercept-rem: math.div($y-intercept, 16px) * 1rem;
  $min-rem: math.div($min-size, 16px) * 1rem;
  $max-rem: math.div($max-size, 16px) * 1rem;

  @return clamp(#{$min-rem}, #{$slope-vw}vw + #{$intercept-rem}, #{$max-rem});
}

h1 { font-size: #{fluid(24px, 48px)}; }
h2 { font-size: #{fluid(20px, 36px)}; }
p  { font-size: #{fluid(16px, 20px)}; }

// Output example:
// h1 { font-size: clamp(1.5rem, 2.727vw + 0.955rem, 3rem); }
```

This calculates the exact linear interpolation between a minimum and maximum font size across a viewport range, using a single `clamp()` expression.

---

> **Key Takeaway:** Functions compute values, mixins emit CSS. Use the built-in modules (`sass:math`, `sass:color`, `sass:map`, `sass:list`, `sass:string`, `sass:meta`) instead of deprecated global functions, validate inputs for public-facing functions, and keep functions pure with no side effects.
