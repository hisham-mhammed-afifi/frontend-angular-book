# 5.19 Proxy and Reflect

> Proxy lets you intercept and redefine fundamental operations on objects, while Reflect provides the default behavior for those same operations, together forming JavaScript's metaprogramming foundation.

---

## Proxy Fundamentals

A `Proxy` wraps another object (the **target**) and intercepts operations like reading properties, writing properties, function calls, and more.

```js
const proxy = new Proxy(target, handler);
```

- **target**: The original object to wrap. Can be any object, including arrays, functions, or another proxy.
- **handler**: An object containing **traps** (methods that intercept operations). If a trap is not defined, the operation forwards directly to the target.

```js
// Empty handler: fully transparent proxy
const target = { name: 'Alice' };
const proxy = new Proxy(target, {});

console.log(proxy.name); // 'Alice' (forwarded to target)
proxy.age = 30;
console.log(target.age); // 30 (mutations go through to target)
```

Once an object is proxied, the proxy should completely replace the target everywhere. No code should reference the target directly, or it bypasses the proxy's interception.

---

## All 13 Handler Traps

Each trap intercepts a fundamental internal method defined in the ECMAScript specification:

| Trap | Triggered By | Signature | Returns |
|------|-------------|-----------|---------|
| `get` | Property read (`obj.prop`) | `(target, prop, receiver)` | any |
| `set` | Property write (`obj.prop = val`) | `(target, prop, value, receiver)` | boolean |
| `has` | `in` operator (`prop in obj`) | `(target, prop)` | boolean |
| `deleteProperty` | `delete obj.prop` | `(target, prop)` | boolean |
| `apply` | Function call (`fn()`) | `(target, thisArg, args)` | any |
| `construct` | `new` operator (`new Fn()`) | `(target, args, newTarget)` | object |
| `getPrototypeOf` | `Object.getPrototypeOf()` | `(target)` | object/null |
| `setPrototypeOf` | `Object.setPrototypeOf()` | `(target, proto)` | boolean |
| `isExtensible` | `Object.isExtensible()` | `(target)` | boolean |
| `preventExtensions` | `Object.preventExtensions()` | `(target)` | boolean |
| `getOwnPropertyDescriptor` | `Object.getOwnPropertyDescriptor()` | `(target, prop)` | descriptor/undefined |
| `defineProperty` | `Object.defineProperty()` | `(target, prop, descriptor)` | boolean |
| `ownKeys` | `Object.keys()`, `Object.getOwnPropertyNames()`, `for...in` | `(target)` | array |

### get Trap

The most commonly used trap. Intercepts all property reads:

```js
const handler = {
  get(target, prop, receiver) {
    console.log(`Reading property: ${String(prop)}`);
    return Reflect.get(target, prop, receiver);
  }
};

const user = new Proxy({ name: 'Alice', age: 30 }, handler);
console.log(user.name);
// Reading property: name
// Alice
```

The `receiver` parameter is the object that the property access was originally performed on (usually the proxy itself, but can differ with prototype chains).

### set Trap

Intercepts property writes. Must return `true` to indicate success (in strict mode, returning `false` throws a TypeError):

```js
const validator = {
  set(target, prop, value, receiver) {
    if (prop === 'age' && typeof value !== 'number') {
      throw new TypeError('Age must be a number');
    }
    if (prop === 'age' && (value < 0 || value > 150)) {
      throw new RangeError('Age must be between 0 and 150');
    }
    return Reflect.set(target, prop, value, receiver);
  }
};

const person = new Proxy({}, validator);
person.name = 'Alice'; // OK
person.age = 30;       // OK
person.age = -5;       // RangeError: Age must be between 0 and 150
person.age = 'thirty'; // TypeError: Age must be a number
```

### has Trap

Intercepts the `in` operator:

```js
const range = new Proxy({ start: 1, end: 10 }, {
  has(target, prop) {
    if (typeof prop === 'string') {
      const num = Number(prop);
      if (!isNaN(num)) {
        return num >= target.start && num <= target.end;
      }
    }
    return Reflect.has(target, prop);
  }
});

console.log(5 in range);  // true
console.log(15 in range); // false
console.log('start' in range); // true
```

### deleteProperty Trap

Intercepts the `delete` operator:

```js
const protected_ = new Proxy({ id: 1, name: 'Alice', _internal: 'secret' }, {
  deleteProperty(target, prop) {
    if (prop.startsWith('_')) {
      throw new Error(`Cannot delete private property: ${prop}`);
    }
    return Reflect.deleteProperty(target, prop);
  }
});

delete protected_.name;      // OK
delete protected_._internal; // Error: Cannot delete private property: _internal
```

### apply Trap

Intercepts function calls. The target must be a function:

```js
function sum(a, b) {
  return a + b;
}

const logged = new Proxy(sum, {
  apply(target, thisArg, args) {
    console.log(`Calling ${target.name}(${args.join(', ')})`);
    const result = Reflect.apply(target, thisArg, args);
    console.log(`Result: ${result}`);
    return result;
  }
});

logged(2, 3);
// Calling sum(2, 3)
// Result: 5
```

### construct Trap

Intercepts the `new` operator. Must return an object:

```js
class User {
  constructor(name) {
    this.name = name;
  }
}

const TrackedUser = new Proxy(User, {
  construct(target, args, newTarget) {
    console.log(`Creating instance with args: ${args}`);
    const instance = Reflect.construct(target, args, newTarget);
    instance.createdAt = Date.now();
    return instance;
  }
});

const user = new TrackedUser('Alice');
// Creating instance with args: Alice
console.log(user.name);      // Alice
console.log(user.createdAt); // 1709049600000
```

### ownKeys Trap

Intercepts operations that list property keys:

```js
const obj = new Proxy({ name: 'Alice', _secret: '123', age: 30 }, {
  ownKeys(target) {
    // Hide properties starting with underscore
    return Reflect.ownKeys(target).filter(key =>
      typeof key !== 'string' || !key.startsWith('_')
    );
  }
});

console.log(Object.keys(obj));                  // ['name', 'age']
console.log(Object.getOwnPropertyNames(obj));   // ['name', 'age']
console.log(JSON.stringify(obj));               // '{"name":"Alice","age":30}'
```

## The Reflect API

`Reflect` is a built-in object that provides static methods matching every Proxy trap. It serves three purposes:

### 1. Default Behavior for Traps

Inside a trap, use `Reflect` to perform the original operation:

```js
const handler = {
  get(target, prop, receiver) {
    // Custom logic here
    console.log(`Accessing: ${String(prop)}`);
    // Then delegate to the default behavior
    return Reflect.get(target, prop, receiver);
  }
};
```

### 2. Proper Receiver Forwarding

`Reflect.get` and `Reflect.set` accept a `receiver` parameter that correctly handles `this` binding in prototype chains and getter/setter inheritance:

```js
const parent = {
  get greeting() {
    return `Hello, I am ${this.name}`;
  }
};

const handler = {
  get(target, prop, receiver) {
    // BAD: target[prop] -- 'this' would be target, not the actual receiver
    // GOOD: Reflect.get forwards 'receiver' so 'this' resolves correctly
    return Reflect.get(target, prop, receiver);
  }
};

const child = Object.create(new Proxy(parent, handler));
child.name = 'Alice';
console.log(child.greeting); // "Hello, I am Alice"
// Without receiver forwarding, this.name would be undefined
```

### 3. Functional Versions of Operators

`Reflect` provides function-call equivalents of operations that are otherwise operators or statements:

```js
// Operator                  // Reflect equivalent
prop in obj                  Reflect.has(obj, prop)
delete obj[prop]             Reflect.deleteProperty(obj, prop)
new Target(...args)          Reflect.construct(Target, args)
fn.apply(thisArg, args)      Reflect.apply(fn, thisArg, args)
```

### All 13 Reflect Methods

| Method | Equivalent Operation |
|--------|---------------------|
| `Reflect.get(target, prop, receiver)` | `target[prop]` |
| `Reflect.set(target, prop, value, receiver)` | `target[prop] = value` |
| `Reflect.has(target, prop)` | `prop in target` |
| `Reflect.deleteProperty(target, prop)` | `delete target[prop]` |
| `Reflect.apply(fn, thisArg, args)` | `fn.apply(thisArg, args)` |
| `Reflect.construct(Target, args, newTarget)` | `new Target(...args)` |
| `Reflect.getPrototypeOf(target)` | `Object.getPrototypeOf(target)` |
| `Reflect.setPrototypeOf(target, proto)` | `Object.setPrototypeOf(target, proto)` |
| `Reflect.isExtensible(target)` | `Object.isExtensible(target)` |
| `Reflect.preventExtensions(target)` | `Object.preventExtensions(target)` |
| `Reflect.getOwnPropertyDescriptor(target, prop)` | `Object.getOwnPropertyDescriptor(target, prop)` |
| `Reflect.defineProperty(target, prop, desc)` | `Object.defineProperty(target, prop, desc)` |
| `Reflect.ownKeys(target)` | `Object.getOwnPropertyNames(target).concat(Object.getOwnPropertySymbols(target))` |

A key difference: `Reflect.defineProperty` returns a boolean, while `Object.defineProperty` throws on failure. This makes `Reflect` methods better suited for conditional logic.

## Proxy Invariants

The spec enforces **invariants** that proxies cannot violate. These prevent proxies from lying about the target's actual state when the target is non-configurable or non-extensible.

### Key Invariant Rules

1. **Non-configurable, non-writable properties**: The `get` trap must return the property's actual value. You cannot report a different value.

```js
const target = {};
Object.defineProperty(target, 'locked', {
  value: 42,
  writable: false,
  configurable: false
});

const proxy = new Proxy(target, {
  get(target, prop) {
    if (prop === 'locked') return 100; // Lie about the value
    return Reflect.get(target, prop);
  }
});

proxy.locked; // TypeError: 'get' on proxy: property 'locked' is a read-only
              // and non-configurable data property on the proxy target but
              // the proxy did not return its actual value
```

2. **Non-configurable properties cannot be reported as non-existent** by the `has` trap.

3. **Non-extensible targets**: The `ownKeys` trap must return exactly the target's own keys (no additions, no omissions).

4. **The `set` trap** must return `false` for non-writable, non-configurable properties if the new value differs from the current value.

5. **The `deleteProperty` trap** cannot report successful deletion of a non-configurable property.

These invariants ensure that proxies cannot break the guarantees that `Object.freeze()`, `Object.seal()`, and non-configurable property descriptors provide.

## Revocable Proxies

`Proxy.revocable()` creates a proxy that can be permanently disabled:

```js
const { proxy, revoke } = Proxy.revocable({ name: 'Alice' }, {
  get(target, prop, receiver) {
    return Reflect.get(target, prop, receiver);
  }
});

console.log(proxy.name); // 'Alice'

revoke(); // Permanently disable the proxy

proxy.name; // TypeError: Cannot perform 'get' on a proxy that has been revoked
```

After `revoke()` is called, ALL operations on the proxy throw a TypeError. The proxy cannot be un-revoked.

### Use Cases

**Temporary access grants:**

```js
function grantTemporaryAccess(sensitiveData, ttlMs) {
  const { proxy, revoke } = Proxy.revocable(sensitiveData, {});

  setTimeout(revoke, ttlMs);

  return proxy;
}

const data = grantTemporaryAccess({ secret: 'xyz' }, 5000);
console.log(data.secret); // 'xyz' (within 5 seconds)
// After 5 seconds: TypeError: Cannot perform 'get' on a proxy that has been revoked
```

**Third-party library sandboxing:**

```js
function createSandboxedApi(api) {
  const { proxy, revoke } = Proxy.revocable(api, {
    get(target, prop, receiver) {
      if (prop === 'dangerousMethod') {
        throw new Error('Access denied');
      }
      return Reflect.get(target, prop, receiver);
    }
  });

  return {
    api: proxy,
    destroy: revoke
  };
}
```

## Proxy Patterns

### Default Values (No More undefined)

```js
function withDefaults(target, defaults) {
  return new Proxy(target, {
    get(target, prop, receiver) {
      const value = Reflect.get(target, prop, receiver);
      if (value !== undefined) return value;
      return prop in defaults ? defaults[prop] : undefined;
    }
  });
}

const config = withDefaults(
  { host: 'localhost' },
  { host: '0.0.0.0', port: 3000, debug: false }
);

console.log(config.host);  // 'localhost' (from target)
console.log(config.port);  // 3000 (from defaults)
console.log(config.debug); // false (from defaults)
```

### Negative Array Indices

```js
function negativeArray(arr) {
  return new Proxy(arr, {
    get(target, prop, receiver) {
      const index = Number(prop);
      if (Number.isInteger(index) && index < 0) {
        return target[target.length + index];
      }
      return Reflect.get(target, prop, receiver);
    }
  });
}

const arr = negativeArray([10, 20, 30, 40, 50]);
console.log(arr[-1]); // 50
console.log(arr[-2]); // 40
console.log(arr[0]);  // 10
```

### Property Access Logging / Tracing

```js
function traceable(target, label = 'Proxy') {
  return new Proxy(target, {
    get(target, prop, receiver) {
      console.log(`[${label}] GET ${String(prop)}`);
      const value = Reflect.get(target, prop, receiver);
      // Recursively wrap nested objects
      if (value !== null && typeof value === 'object') {
        return traceable(value, `${label}.${String(prop)}`);
      }
      return value;
    },
    set(target, prop, value, receiver) {
      console.log(`[${label}] SET ${String(prop)} = ${JSON.stringify(value)}`);
      return Reflect.set(target, prop, value, receiver);
    }
  });
}

const state = traceable({ user: { name: 'Alice' } });
state.user.name = 'Bob';
// [Proxy] GET user
// [Proxy.user] SET name = "Bob"
```

### Observable Object (Change Detection)

```js
function observable(target) {
  const listeners = new Map();

  const proxy = new Proxy(target, {
    set(target, prop, value, receiver) {
      const oldValue = target[prop];
      const result = Reflect.set(target, prop, value, receiver);

      if (oldValue !== value && listeners.has(prop)) {
        for (const fn of listeners.get(prop)) {
          fn(value, oldValue, prop);
        }
      }
      return result;
    }
  });

  proxy.on = function (prop, callback) {
    if (!listeners.has(prop)) listeners.set(prop, new Set());
    listeners.get(prop).add(callback);
    return () => listeners.get(prop).delete(callback); // unsubscribe
  };

  return proxy;
}

const state = observable({ count: 0 });
const unsubscribe = state.on('count', (newVal, oldVal) => {
  console.log(`count changed: ${oldVal} -> ${newVal}`);
});

state.count = 1; // count changed: 0 -> 1
state.count = 2; // count changed: 1 -> 2
unsubscribe();
state.count = 3; // (no log)
```

### Schema Validation

```js
function createValidated(schema) {
  return new Proxy({}, {
    set(target, prop, value) {
      if (!(prop in schema)) {
        throw new Error(`Unknown property: ${prop}`);
      }

      const validator = schema[prop];
      if (!validator(value)) {
        throw new TypeError(`Invalid value for ${prop}: ${value}`);
      }

      return Reflect.set(target, prop, value);
    }
  });
}

const user = createValidated({
  name: (v) => typeof v === 'string' && v.length > 0,
  age: (v) => typeof v === 'number' && v >= 0 && v <= 150,
  email: (v) => typeof v === 'string' && v.includes('@'),
});

user.name = 'Alice';      // OK
user.age = 30;             // OK
user.email = 'not-email';  // TypeError: Invalid value for email: not-email
user.role = 'admin';       // Error: Unknown property: role
```

### Immutable View (Read-Only Proxy)

```js
function readonly(target) {
  return new Proxy(target, {
    set() { throw new TypeError('Cannot modify a read-only object'); },
    deleteProperty() { throw new TypeError('Cannot delete from a read-only object'); },
    defineProperty() { throw new TypeError('Cannot define properties on a read-only object'); },
    setPrototypeOf() { throw new TypeError('Cannot change prototype of a read-only object'); },

    get(target, prop, receiver) {
      const value = Reflect.get(target, prop, receiver);
      // Deep readonly: wrap nested objects too
      if (value !== null && typeof value === 'object') {
        return readonly(value);
      }
      return value;
    }
  });
}

const config = readonly({ db: { host: 'localhost', port: 5432 } });
console.log(config.db.host); // 'localhost'
config.db.host = 'other';    // TypeError: Cannot modify a read-only object
```

## The Private Fields Gotcha

Proxies break private field (`#field`) access. Private fields use an internal slot (brand check) tied to the original target object, not the proxy:

```js
class User {
  #name;
  constructor(name) { this.#name = name; }
  getName() { return this.#name; }
}

const user = new User('Alice');
const proxied = new Proxy(user, {});

user.getName();    // 'Alice'
proxied.getName(); // TypeError: Cannot read private member #name
                   // from an object whose class did not declare it
```

When `getName()` runs through the proxy, `this` is the proxy, not the original `user` object. The engine checks if `this` has the `#name` internal slot and it does not (the proxy does not have it).

### Workaround: Bind Methods to Target

```js
function proxyWithPrivateFields(target, handler) {
  return new Proxy(target, {
    get(target, prop, receiver) {
      const value = Reflect.get(target, prop, target); // Use target as receiver
      if (typeof value === 'function') {
        return value.bind(target); // Bind methods to the original target
      }
      if (handler.get) {
        return handler.get(target, prop, receiver);
      }
      return value;
    },
    ...handler
  });
}
```

This workaround sacrifices the ability to intercept method calls through the proxy, which is a significant limitation. Consider using WeakMap-based private data or Symbol-based "soft private" properties when you need both proxying and encapsulation.

## Proxy and Prototype Chains

Proxies interact with prototype chains. A proxy on a prototype affects all objects that inherit from it:

```js
const handler = {
  get(target, prop, receiver) {
    if (prop === 'fullName') {
      // 'receiver' is the actual object (child), not the prototype (target)
      return `${receiver.firstName} ${receiver.lastName}`;
    }
    return Reflect.get(target, prop, receiver);
  }
};

const proto = new Proxy({}, handler);
const child = Object.create(proto);
child.firstName = 'Alice';
child.lastName = 'Smith';

console.log(child.fullName); // 'Alice Smith'
```

The `receiver` parameter is critical here. It tells the trap which object the property access was originally performed on, allowing correct `this` behavior through the chain.

## Performance

Proxies add overhead to every intercepted operation. The cost varies by engine and trap, but here are the key points:

### Benchmarks

- Property access through a `get` trap is roughly 5-50x slower than direct property access, depending on the engine (V8, SpiderMonkey, JavaScriptCore) and whether the proxy handler is monomorphic.
- An empty handler (no traps) still adds measurable overhead because the engine must check for the trap's existence.
- V8 has improved proxy performance significantly over the years, but they remain slower than direct access.

### When NOT to Use Proxies

- **Hot loops** where property access happens millions of times per second
- **Performance-critical data structures** (use plain objects or Maps instead)
- **Serialization paths** (JSON.stringify, structured clone)

### When Proxies Are Worth the Overhead

- **Development-time validation** (type checking, schema enforcement) that gets stripped in production
- **Reactive systems** (Vue 3's reactivity) where the overhead is amortized across UI updates
- **API boundaries** where the cost is negligible compared to network I/O
- **Metaprogramming** that eliminates large amounts of boilerplate

### Minimizing Proxy Overhead

```js
// BAD: proxy on a frequently accessed inner object
const proxied = new Proxy(hugeDataStructure, handler);
// Every single property access has overhead

// GOOD: proxy only at the boundary
function createApiClient(baseUrl) {
  return new Proxy({}, {
    get(_, method) {
      return (path, options) => fetch(`${baseUrl}/${method}${path}`, options);
    }
  });
}
// Only the initial method dispatch goes through the proxy
```

## Common Mistakes

### 1. Forgetting to Forward with Reflect

```js
// BAD: silently breaks everything by not forwarding unhandled operations
const proxy = new Proxy(target, {
  get(target, prop) {
    if (prop === 'special') return 'intercepted';
    return target[prop]; // Breaks receiver forwarding for getters
  }
});

// GOOD: use Reflect for default forwarding
const proxy = new Proxy(target, {
  get(target, prop, receiver) {
    if (prop === 'special') return 'intercepted';
    return Reflect.get(target, prop, receiver); // Correct receiver
  }
});
```

### 2. Not Returning true from set Trap

In strict mode (including ES modules), a `set` trap that does not return `true` throws a TypeError:

```js
// BAD: no return value (returns undefined, which is falsy)
const proxy = new Proxy({}, {
  set(target, prop, value) {
    target[prop] = value;
    // Missing return true!
  }
});

proxy.x = 1; // TypeError in strict mode

// GOOD
const proxy = new Proxy({}, {
  set(target, prop, value, receiver) {
    return Reflect.set(target, prop, value, receiver); // Returns true
  }
});
```

### 3. Infinite Recursion with this

```js
// BAD: accessing proxy inside its own trap
const proxy = new Proxy({}, {
  get(target, prop) {
    return proxy[prop]; // Infinite loop! Triggers get trap again
  }
});

// GOOD: access the target
const proxy = new Proxy({}, {
  get(target, prop, receiver) {
    return Reflect.get(target, prop, receiver);
  }
});
```

### 4. Expecting Proxy Identity to Match Target

```js
const target = {};
const proxy = new Proxy(target, {});

console.log(proxy === target); // false
// The proxy is a different object

// This breaks Map/Set lookups and WeakMap/WeakSet keys
const map = new Map();
map.set(target, 'value');
console.log(map.get(proxy)); // undefined (different reference)
```

### 5. Proxying Objects with Internal Slots

Built-in objects like `Date`, `Map`, `Set`, `RegExp`, and `Promise` have internal slots that proxies cannot intercept. Method calls through a proxy fail because `this` is the proxy, not the original object:

```js
const date = new Date();
const proxied = new Proxy(date, {});

proxied.getTime(); // TypeError: this is not a Date object
```

**Workaround:** Bind methods to the original target.

### 6. Assuming Proxy Traps Catch Everything

Proxies cannot intercept:
- Identity comparisons (`===`, `!==`)
- `typeof`
- Primitive coercion (unless you define `Symbol.toPrimitive`)

---

## Best Practices

- **Always use `Reflect` inside traps** for default forwarding. It correctly handles `receiver`, returns booleans, and mirrors the trap signature exactly.

- **Always return the correct type from traps.** `set` and `deleteProperty` must return booleans. `construct` must return an object. `ownKeys` must return an array.

- **Use revocable proxies** when granting temporary or scoped access. Call `revoke()` as soon as access should end.

- **Keep proxies at API boundaries**, not deep inside hot paths. The overhead is acceptable for coarse-grained interception but costly for fine-grained per-property access in tight loops.

- **Test proxy behavior thoroughly.** Proxy invariants and edge cases with private fields, built-in objects, and identity comparisons can cause subtle bugs.

- **Prefer simpler alternatives** when they exist. If you only need property validation, a setter or `Object.defineProperty` may be simpler than a Proxy. Reserve Proxy for cases that genuinely need dynamic interception.

## Real-World Use Cases

### 1. Vue 3 Reactive System (Simplified)

Vue 3 uses Proxy to track which properties are read during rendering and trigger re-renders when they change:

```js
const targetMap = new WeakMap();
let activeEffect = null;

function reactive(target) {
  return new Proxy(target, {
    get(target, prop, receiver) {
      // Track: record that this property was accessed during the current effect
      if (activeEffect) {
        let depsMap = targetMap.get(target);
        if (!depsMap) {
          depsMap = new Map();
          targetMap.set(target, depsMap);
        }
        let deps = depsMap.get(prop);
        if (!deps) {
          deps = new Set();
          depsMap.set(prop, deps);
        }
        deps.add(activeEffect);
      }

      const value = Reflect.get(target, prop, receiver);
      if (value !== null && typeof value === 'object') {
        return reactive(value); // Deep reactivity
      }
      return value;
    },

    set(target, prop, value, receiver) {
      const oldValue = target[prop];
      const result = Reflect.set(target, prop, value, receiver);

      // Trigger: notify all effects that depend on this property
      if (oldValue !== value) {
        const depsMap = targetMap.get(target);
        if (depsMap) {
          const deps = depsMap.get(prop);
          if (deps) {
            deps.forEach(effect => effect());
          }
        }
      }
      return result;
    }
  });
}

function effect(fn) {
  activeEffect = fn;
  fn(); // Run once to collect dependencies
  activeEffect = null;
}

// Usage
const state = reactive({ count: 0 });
effect(() => {
  console.log('Count is:', state.count); // Automatically re-runs on change
});

state.count = 1; // "Count is: 1"
state.count = 2; // "Count is: 2"
```

### 2. API Client with Dynamic Method Dispatch

```js
function createApiClient(baseUrl) {
  return new Proxy({}, {
    get(_, resource) {
      return new Proxy(() => {}, {
        get(_, method) {
          return async (idOrBody, options = {}) => {
            const methods = {
              getAll: () => fetch(`${baseUrl}/${resource}`, options),
              getById: () => fetch(`${baseUrl}/${resource}/${idOrBody}`, options),
              create: () => fetch(`${baseUrl}/${resource}`, {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify(idOrBody),
                ...options,
              }),
              delete: () => fetch(`${baseUrl}/${resource}/${idOrBody}`, {
                method: 'DELETE',
                ...options,
              }),
            };

            if (!(method in methods)) {
              throw new Error(`Unknown method: ${method}`);
            }

            const response = await methods[method]();
            return response.json();
          };
        }
      });
    }
  });
}

const api = createApiClient('https://api.example.com');

// Dynamic method dispatch: no boilerplate endpoint definitions
await api.users.getAll();
await api.users.getById(42);
await api.posts.create({ title: 'Hello', body: 'World' });
```

### 3. Strict Schema Enforcement for Configuration Objects

```js
function strictConfig(defaults) {
  const allowedKeys = new Set(Object.keys(defaults));
  const config = { ...defaults };

  return new Proxy(config, {
    set(target, prop, value) {
      if (!allowedKeys.has(prop)) {
        throw new Error(
          `Unknown config key: "${prop}". Allowed: ${[...allowedKeys].join(', ')}`
        );
      }
      if (typeof value !== typeof defaults[prop]) {
        throw new TypeError(
          `"${prop}" must be ${typeof defaults[prop]}, got ${typeof value}`
        );
      }
      return Reflect.set(target, prop, value);
    },

    get(target, prop, receiver) {
      if (typeof prop === 'string' && !allowedKeys.has(prop) && prop !== 'toJSON') {
        throw new Error(`Unknown config key: "${prop}"`);
      }
      return Reflect.get(target, prop, receiver);
    },

    deleteProperty() {
      throw new Error('Cannot delete config properties');
    }
  });
}

const config = strictConfig({
  port: 3000,
  host: 'localhost',
  debug: false,
});

config.port = 8080;     // OK
config.host = 'prod';   // OK
config.port = '8080';   // TypeError: "port" must be number, got string
config.database = 'pg'; // Error: Unknown config key: "database"
```

---

> **Key Takeaway:** Proxy and Reflect together enable intercepting any fundamental object operation, powering patterns like reactive state tracking, validation, access control, and dynamic API generation, but use them judiciously since they add overhead and interact poorly with private fields and built-in internal slots.
