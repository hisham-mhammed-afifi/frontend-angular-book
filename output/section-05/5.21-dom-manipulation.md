# 5.21 DOM Manipulation

> The Document Object Model is a tree-structured programming interface that lets JavaScript read, modify, and react to the content and structure of an HTML document. Mastering DOM manipulation means understanding node types, selection strategies, traversal, mutation, event handling, and the performance consequences of every operation.

---

## The DOM Tree Structure

The browser parses HTML into a tree of nodes. Every piece of the document becomes a node, but not every node is an element.

### Node Types

Each node has a `nodeType` property returning a numeric constant:

| Constant                     | Value | Description                                      |
|------------------------------|-------|--------------------------------------------------|
| `Node.ELEMENT_NODE`         | 1     | An HTML element (`<div>`, `<p>`, `<span>`)       |
| `Node.TEXT_NODE`            | 3     | Text content inside an element                   |
| `Node.COMMENT_NODE`        | 8     | An HTML comment (`<!-- ... -->`)                 |
| `Node.DOCUMENT_NODE`       | 9     | The `document` object itself                     |
| `Node.DOCUMENT_TYPE_NODE`  | 10    | The `<!DOCTYPE html>` declaration               |
| `Node.DOCUMENT_FRAGMENT_NODE` | 11  | A lightweight container (DocumentFragment)       |

Deprecated types (5, 6, 12) still exist as constants but are never produced by modern parsers.

```js
const div = document.createElement('div');
div.textContent = 'Hello';

console.log(div.nodeType);                  // 1 (ELEMENT_NODE)
console.log(div.firstChild.nodeType);       // 3 (TEXT_NODE)
console.log(document.nodeType);             // 9 (DOCUMENT_NODE)
```

### Node Properties: nodeName, nodeValue, nodeType

```js
const comment = document.createComment('TODO: refactor');
console.log(comment.nodeName);   // "#comment"
console.log(comment.nodeValue);  // "TODO: refactor"
console.log(comment.nodeType);   // 8

const text = document.createTextNode('content');
console.log(text.nodeName);      // "#text"
console.log(text.nodeValue);     // "content"

const el = document.createElement('section');
console.log(el.nodeName);        // "SECTION" (uppercased in HTML mode)
console.log(el.nodeValue);       // null (elements have no nodeValue)
```

### The Document Object

`document` is the entry point to the DOM tree. It represents the entire page and provides all selection and creation methods. `document.documentElement` is the `<html>` element, `document.head` is `<head>`, and `document.body` is `<body>`.

```js
// Gotcha: document.body can be null if your script runs in <head>
// without defer/async, because <body> hasn't been parsed yet.
console.log(document.body); // null if script is in <head> and runs synchronously
```

### Whitespace Text Nodes

The DOM includes text nodes for whitespace between elements. This is a constant source of confusion:

```html
<ul>
  <li>One</li>
  <li>Two</li>
</ul>
```

```js
const ul = document.querySelector('ul');

// childNodes includes whitespace text nodes between <li> elements
console.log(ul.childNodes.length); // 5 (text, li, text, li, text)

// children only includes element nodes
console.log(ul.children.length);   // 2 (li, li)
```

## Selecting Elements

### getElementById

Returns a single element by its `id` attribute. The fastest selection method because browsers maintain an internal hash map of IDs.

```js
const header = document.getElementById('main-header');
// Returns null if no match. Does NOT throw.
// Does NOT require a # prefix (unlike CSS selectors).
```

**Gotcha**: If multiple elements share the same `id` (invalid HTML), `getElementById` returns the first one. The behavior is consistent across browsers, but the HTML is technically invalid.

**Gotcha**: `getElementById` exists only on `document`, not on individual elements. You cannot call `someDiv.getElementById('child')`.

### getElementsByClassName

Returns a **live HTMLCollection** of all elements with the given class name(s).

```js
const items = document.getElementsByClassName('menu-item');

// Multiple classes: space-separated (AND logic, not OR)
const active = document.getElementsByClassName('menu-item active');
// Returns elements that have BOTH "menu-item" AND "active"
```

### getElementsByTagName

Returns a **live HTMLCollection** of all elements with the given tag name.

```js
const paragraphs = document.getElementsByTagName('p');
const everything = document.getElementsByTagName('*'); // all elements
```

### querySelector

Returns the **first** element matching a CSS selector. Returns `null` if no match.

```js
const el = document.querySelector('.sidebar > ul:first-child');
const nested = document.querySelector('[data-role="admin"]');
```

Can be called on any element to scope the search:

```js
const container = document.querySelector('.container');
const btn = container.querySelector('.btn'); // searches within container only
```

**Gotcha**: When called on an element, the selector is still evaluated in the context of the entire document and then filtered to descendants. This leads to surprising behavior:

```js
const div = document.querySelector('#parent');

// This WILL match even though the selector starts from the document root
// and then checks if the result is a descendant of div
const result = div.querySelector('div > span');
// Use :scope to anchor the selector to the calling element
const scoped = div.querySelector(':scope > span');
```

### querySelectorAll

Returns a **static NodeList** of all matching elements. The snapshot does not update when the DOM changes.

```js
const cards = document.querySelectorAll('.card');

// Iteration support
cards.forEach((card, index) => {
  card.style.order = index;
});

// Spread into array for full array methods
const cardArray = [...cards];
const visible = cardArray.filter(c => !c.hidden);
```

**Gotcha**: `querySelectorAll` returns a NodeList, not an Array. It has `forEach` but lacks `map`, `filter`, `reduce`. Spread or `Array.from` to convert.

### Escaping Special Characters in Selectors

If an ID or class contains characters that are special in CSS (periods, colons, brackets), you must escape them:

```js
// Element: <div id="user.name"></div>

// This FAILS -- the parser thinks .name is a class selector
document.querySelector('#user.name');

// Option 1: Backslash escape
document.querySelector('#user\\.name');

// Option 2: CSS.escape() (recommended)
const id = 'user.name';
document.querySelector(`#${CSS.escape(id)}`);
```

### closest

Traverses the element itself and its ancestors (toward the document root) until it finds a node that matches the selector. Returns `null` if no match.

```js
document.addEventListener('click', (e) => {
  // Find the closest ancestor with the class .card
  const card = e.target.closest('.card');
  if (card) {
    // Clicked inside a card
  }
});
```

**Key detail**: `closest` starts from the element itself. If the element matches, it returns the element.

```js
const div = document.querySelector('.active');
div.closest('.active') === div; // true -- it checks itself first
```

### matches

Returns a boolean indicating whether the element matches the given CSS selector.

```js
const el = document.querySelector('.btn');

if (el.matches('.btn.primary')) {
  // Element has both .btn and .primary classes
}

// Useful in event delegation
document.addEventListener('click', (e) => {
  if (e.target.matches('button.delete')) {
    handleDelete(e.target);
  }
});
```

### Performance Comparison

| Method                        | Returns          | Live/Static | Speed   |
|-------------------------------|------------------|-------------|---------|
| `getElementById`             | Element or null  | N/A         | Fastest |
| `getElementsByClassName`     | HTMLCollection   | Live        | Fast    |
| `getElementsByTagName`       | HTMLCollection   | Live        | Fast    |
| `querySelector`              | Element or null  | N/A         | Moderate|
| `querySelectorAll`           | NodeList         | Static      | Moderate|

`getElementById` is roughly 2x faster than `querySelector` because it uses a direct hash lookup. However, the performance difference is negligible in most applications. Prefer whichever makes your code clearest.

## Live vs Static Collections

This distinction is a major source of bugs, especially in loops that modify the DOM.

### HTMLCollection (Live)

Returned by `getElementsByClassName`, `getElementsByTagName`, `document.forms`, `element.children`.

A live collection automatically reflects changes to the DOM:

```js
const items = document.getElementsByClassName('item');
console.log(items.length); // 3

// Add a new element with class "item"
const newItem = document.createElement('div');
newItem.className = 'item';
document.body.appendChild(newItem);

console.log(items.length); // 4 -- the collection updated automatically
```

### The Infinite Loop Trap

```js
// BAD: Infinite loop with live collection
const items = document.getElementsByClassName('old');
for (let i = 0; i < items.length; i++) {
  items[i].classList.remove('old');
  items[i].classList.add('new');
  // Removing 'old' shrinks the live collection.
  // i increments, items.length decrements. Elements get skipped.
}

// FIX 1: Iterate in reverse
for (let i = items.length - 1; i >= 0; i--) {
  items[i].classList.remove('old');
  items[i].classList.add('new');
}

// FIX 2: Convert to static array first
const itemsArray = [...items];
itemsArray.forEach(item => {
  item.classList.remove('old');
  item.classList.add('new');
});

// FIX 3: Use querySelectorAll (static NodeList)
const staticItems = document.querySelectorAll('.old');
staticItems.forEach(item => {
  item.classList.remove('old');
  item.classList.add('new');
});
```

### NodeList (Static or Live)

`querySelectorAll` returns a **static** NodeList. `childNodes` returns a **live** NodeList.

```js
const paragraphs = document.querySelectorAll('p'); // static
const children = document.body.childNodes;          // live

// Static NodeList: supports forEach natively
paragraphs.forEach(p => console.log(p.textContent));

// HTMLCollection: does NOT support forEach
const divs = document.getElementsByTagName('div');
// divs.forEach(...) // TypeError -- HTMLCollection has no forEach

// Convert HTMLCollection to array for iteration
[...divs].forEach(div => console.log(div.id));
// or
Array.from(divs).forEach(div => console.log(div.id));
```

### Quick Reference

| Source                          | Type             | Live/Static |
|---------------------------------|------------------|-------------|
| `getElementsByClassName()`     | HTMLCollection   | Live        |
| `getElementsByTagName()`      | HTMLCollection   | Live        |
| `element.children`            | HTMLCollection   | Live        |
| `document.forms`              | HTMLCollection   | Live        |
| `querySelectorAll()`          | NodeList         | Static      |
| `element.childNodes`          | NodeList         | Live        |
| `document.querySelectorAll()` | NodeList         | Static      |

## Traversing the DOM

### Parent Navigation

```js
const child = document.querySelector('.child');

// parentNode: returns any node type (Element, Document, DocumentFragment)
child.parentNode;

// parentElement: returns only Element nodes, null for Document/Fragment parents
child.parentElement;

// In most cases they are identical. The difference shows at the top:
document.documentElement.parentNode;    // #document (the Document node)
document.documentElement.parentElement; // null (Document is not an Element)
```

### Child Navigation

Two parallel sets of properties exist: one that includes ALL node types (text, comments, whitespace), and one that includes only element nodes.

```html
<div id="parent">
  <!-- comment -->
  <span>First</span>
  Text node here
  <span>Last</span>
</div>
```

```js
const parent = document.getElementById('parent');

// ALL nodes (includes text and comment nodes)
parent.childNodes;       // NodeList(7): [text, comment, text, span, text, span, text]
parent.firstChild;       // #text (whitespace)
parent.lastChild;        // #text (whitespace)

// ELEMENT nodes only (skips text and comments)
parent.children;         // HTMLCollection(2): [span, span]
parent.firstElementChild; // <span>First</span>
parent.lastElementChild;  // <span>Last</span>
parent.childElementCount; // 2
```

### Sibling Navigation

```js
const middle = document.querySelector('.middle');

// ALL node types
middle.nextSibling;       // could be a text node (whitespace)
middle.previousSibling;   // could be a text node (whitespace)

// ELEMENT nodes only
middle.nextElementSibling;     // next element or null
middle.previousElementSibling; // previous element or null
```

### Walking Up the Tree with closest()

```js
// Find the nearest ancestor <form>
const input = document.querySelector('input');
const form = input.closest('form');

// Walk up until you find an element with a specific data attribute
const section = input.closest('[data-section]');

// Returns null if no ancestor matches
const nonexistent = input.closest('.nonexistent'); // null
```

### Practical: Collect All Ancestors

```js
function getAncestors(element) {
  const ancestors = [];
  let current = element.parentElement;
  while (current) {
    ancestors.push(current);
    current = current.parentElement;
  }
  return ancestors;
}

// Returns [parentDiv, section, main, body, html]
```

### Practical: Find All Siblings

```js
function getSiblings(element) {
  return [...element.parentElement.children].filter(child => child !== element);
}
```

## Creating and Inserting Elements

### createElement and createTextNode

```js
const div = document.createElement('div');
div.className = 'card';
div.id = 'card-1';
div.textContent = 'Card content';

// Creating a text node explicitly (rarely needed since textContent exists)
const text = document.createTextNode('Hello, World');
div.appendChild(text);
```

### Modern Insertion Methods: append, prepend, before, after, replaceWith

These modern methods (available since ~2016 in all browsers) accept multiple arguments and accept both Nodes and strings:

```js
const container = document.querySelector('.container');
const newDiv = document.createElement('div');

// append: inserts at the end of the element's children
container.append(newDiv);
container.append('Plain text', document.createElement('span')); // multiple args

// prepend: inserts at the start of the element's children
container.prepend(newDiv);

// before: inserts before the element itself (as a sibling)
const ref = document.querySelector('.reference');
ref.before(newDiv);
ref.before('Text before', newDiv); // multiple args

// after: inserts after the element itself (as a sibling)
ref.after(newDiv);

// replaceWith: replaces the element entirely
ref.replaceWith(newDiv);
ref.replaceWith('Replaced with text'); // string argument creates a text node
```

**Key difference from appendChild**: `append` accepts strings (auto-converted to text nodes), `appendChild` does not. `append` accepts multiple arguments, `appendChild` takes exactly one. `append` returns `undefined`, `appendChild` returns the appended node.

```js
// appendChild vs append
container.appendChild('text'); // TypeError: not a Node
container.append('text');      // Works -- creates a text node

container.appendChild(el);    // Returns el
container.append(el);         // Returns undefined
```

### insertAdjacentHTML

Parses an HTML string and inserts the resulting nodes at the specified position. Unlike `innerHTML`, it does NOT reparse existing content.

```js
const container = document.querySelector('.container');

// Four positions:
container.insertAdjacentHTML('beforebegin', '<p>Before container</p>');
// Inserts as previous sibling

container.insertAdjacentHTML('afterbegin', '<p>First child</p>');
// Inserts as first child

container.insertAdjacentHTML('beforeend', '<p>Last child</p>');
// Inserts as last child

container.insertAdjacentHTML('afterend', '<p>After container</p>');
// Inserts as next sibling
```

```
<!-- beforebegin -->
<div class="container">
  <!-- afterbegin -->
  <p>existing content</p>
  <!-- beforeend -->
</div>
<!-- afterend -->
```

**Performance advantage**: `insertAdjacentHTML` does not destroy and recreate existing DOM nodes. When you set `innerHTML`, the browser destroys all existing children, reparses the entire string, and builds new nodes. `insertAdjacentHTML` only parses the new HTML and inserts the result.

### insertAdjacentElement and insertAdjacentText

```js
const ref = document.querySelector('.reference');

// Insert an existing element at a position
const el = document.createElement('span');
ref.insertAdjacentElement('afterend', el);

// Insert plain text at a position (creates a text node)
ref.insertAdjacentText('beforebegin', 'Some text');
```

### DocumentFragment

A lightweight container that holds nodes in memory without being part of the DOM tree. When appended, the fragment itself disappears and only its children are inserted, triggering a single reflow.

```js
const fragment = document.createDocumentFragment();

for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}

// Single DOM insertion -- one reflow instead of 1000
document.querySelector('ul').appendChild(fragment);

// After insertion, fragment is empty
console.log(fragment.childNodes.length); // 0
```

**When to use**: Whenever you need to add many elements at once. The performance difference is significant at scale (hundreds or thousands of elements).

**Modern alternative**: In many cases, `append` with multiple arguments achieves a similar result:

```js
const ul = document.querySelector('ul');
const items = Array.from({ length: 100 }, (_, i) => {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  return li;
});
ul.append(...items); // single insertion
```

### cloneNode

Creates a copy of a node. Pass `true` for a deep clone (includes all descendants), `false` for a shallow clone (element only, no children).

```js
const original = document.querySelector('.template-card');
const clone = original.cloneNode(true); // deep clone

// IMPORTANT GOTCHAS:

// 1. Event listeners added via addEventListener are NOT cloned
//    Only inline event handlers (onclick="...") in the HTML are cloned.
original.addEventListener('click', handleClick);
// clone does NOT have the click listener

// 2. Duplicate IDs -- you must change the ID
clone.id = 'card-copy';

// 3. Form element state is NOT preserved
//    Checkbox checked states, input values set by the user, etc. are lost.

// 4. Canvas content is NOT cloned
//    A cloned canvas element will be blank.

// 5. The clone is detached -- you must insert it
document.body.appendChild(clone);
```

## Removing Elements

### element.remove()

The modern, direct way to remove an element. No need to reference the parent.

```js
const el = document.querySelector('.obsolete');
el.remove(); // gone from the DOM

// The node still exists in memory if you hold a reference
console.log(el.textContent); // still accessible
// It will be garbage collected when all references are dropped
```

### parentNode.removeChild()

The legacy approach. Returns the removed node.

```js
const parent = document.querySelector('.list');
const child = parent.querySelector('.item');
const removed = parent.removeChild(child);

// 'removed' is the same node, still in memory
// Can be reinserted elsewhere
document.querySelector('.archive').appendChild(removed);
```

### Removing All Children

```js
const container = document.querySelector('.container');

// Method 1: innerHTML (fast but destroys event listeners and references)
container.innerHTML = '';

// Method 2: replaceChildren with no arguments (modern, clean)
container.replaceChildren();

// Method 3: Loop removal (preserves references if needed)
while (container.firstChild) {
  container.removeChild(container.firstChild);
}

// Method 4: textContent (fast, similar to innerHTML = '')
container.textContent = '';
```

### Memory Leak Warning

Detached DOM nodes are a common source of memory leaks. If you remove an element from the DOM but keep a reference to it (in a variable, array, Map, closure, or event listener), the entire subtree remains in memory.

```js
// BAD: Memory leak -- references to removed nodes persist
const cache = [];

function addCard(data) {
  const card = document.createElement('div');
  card.textContent = data.title;
  cache.push(card); // reference held forever
  document.body.appendChild(card);
}

function removeCard(card) {
  card.remove();
  // card is still in the cache array -- leaked!
}

// FIX: Use WeakRef or WeakMap, or remove from cache explicitly
function removeCardFixed(card) {
  card.remove();
  const index = cache.indexOf(card);
  if (index > -1) cache.splice(index, 1);
}
```

## Modifying Elements

### textContent vs innerText vs innerHTML

These three properties behave very differently. Choosing the wrong one causes security vulnerabilities, performance problems, or incorrect rendering.

```html
<div id="example">
  <span>Visible</span>
  <span style="display: none">Hidden</span>
  <script>let x = 1;</script>
</div>
```

```js
const el = document.getElementById('example');

// textContent: returns ALL text, including hidden elements and <script> content
// Does NOT trigger a reflow. Safe against XSS.
console.log(el.textContent);
// "  Visible\n  Hidden\n  let x = 1;\n"

// innerText: returns only VISIBLE text (CSS-aware)
// Triggers a reflow to calculate visibility. Slower.
console.log(el.innerText);
// "Visible"

// innerHTML: returns the raw HTML string
console.log(el.innerHTML);
// "<span>Visible</span>\n<span style=\"display: none\">Hidden</span>..."
```

| Property       | Returns            | CSS-Aware | Triggers Reflow | XSS Risk | Performance |
|----------------|--------------------|-----------|-----------------|----------|-------------|
| `textContent`  | All text (raw)     | No        | No              | None     | Fast        |
| `innerText`    | Visible text only  | Yes       | Yes             | None     | Slow        |
| `innerHTML`    | HTML string        | No        | No              | **Yes**  | Moderate    |

**Setting values**:

```js
// textContent: escapes HTML, safe for user input
el.textContent = '<script>alert("xss")</script>';
// Renders as literal text: <script>alert("xss")</script>

// innerText: also escapes HTML, but normalizes whitespace
el.innerText = 'Line 1\nLine 2';
// Inserts a <br> between the lines

// innerHTML: parses HTML, DANGEROUS with user input
el.innerHTML = '<b>Bold</b>'; // renders bold text
el.innerHTML = userInput; // NEVER DO THIS without sanitization
```

### innerHTML Security: XSS Attacks

```js
// BAD: Direct user input in innerHTML
const userInput = '<img src=x onerror="document.cookie">';
el.innerHTML = userInput; // Executes the onerror handler

// Note: <script> tags inserted via innerHTML do NOT execute.
// But event handlers on other elements DO execute.

// SAFE: Use textContent for plain text
el.textContent = userInput; // rendered as literal text

// SAFE: Use DOMPurify for HTML that must be rendered
// import DOMPurify from 'dompurify';
// el.innerHTML = DOMPurify.sanitize(userInput);
```

### outerHTML

Gets or sets the HTML including the element itself:

```js
const el = document.querySelector('.target');
console.log(el.outerHTML); // "<div class='target'>Content</div>"

// Setting outerHTML replaces the element entirely
el.outerHTML = '<section>New content</section>';
// 'el' still references the old (now detached) node
// The new <section> is in the DOM, but you have no reference to it
```

### Attributes: getAttribute, setAttribute, removeAttribute, hasAttribute

```js
const link = document.querySelector('a');

// Read
link.getAttribute('href');        // "/page" (the raw attribute value)
link.href;                         // "https://example.com/page" (the resolved URL)
// getAttribute returns the literal attribute value.
// The property returns the resolved/computed value.

// Write
link.setAttribute('target', '_blank');
link.setAttribute('data-id', '42');

// Check existence
link.hasAttribute('rel'); // true or false

// Remove
link.removeAttribute('target');
```

**Gotcha**: Attribute names are case-insensitive in HTML but case-sensitive in `getAttribute`/`setAttribute` for XML/SVG documents.

**Gotcha**: Some attributes and properties are not 1:1 mappings:

```js
const input = document.querySelector('input');

// 'value' attribute = initial value set in HTML
input.getAttribute('value'); // "initial"

// 'value' property = current value (what the user typed)
input.value; // "whatever the user typed"

// Setting the attribute does NOT update the property after user interaction
input.setAttribute('value', 'reset');
// input.value is still "whatever the user typed"

// Setting the property updates the displayed value
input.value = 'new value';
```

### dataset (Custom Data Attributes)

The `dataset` property provides a `DOMStringMap` to read and write `data-*` attributes using camelCase:

```html
<div id="user" data-user-id="42" data-role="admin" data-is-active="true"></div>
```

```js
const el = document.getElementById('user');

// Reading: data-user-id -> dataset.userId (camelCase conversion)
el.dataset.userId;    // "42" (always a string)
el.dataset.role;      // "admin"
el.dataset.isActive;  // "true" (string, not boolean)

// Writing
el.dataset.score = 100; // creates data-score="100" (converted to string)

// Deleting
delete el.dataset.role; // removes the data-role attribute

// Gotcha: values are always strings
el.dataset.count = 42;
typeof el.dataset.count; // "string" -- it's "42", not 42
```

**Naming conversion rules**: `data-abc-def` becomes `dataset.abcDef`. Every dash followed by a lowercase letter is removed and the letter is uppercased. Conversely, camelCase property names are converted to dash-case attribute names.

### classList

The `classList` property returns a live `DOMTokenList` for manipulating CSS classes. Always prefer `classList` over manually editing the `className` string.

```js
const el = document.querySelector('.card');

// Add one or more classes
el.classList.add('active', 'highlighted');

// Remove one or more classes
el.classList.remove('hidden', 'stale');

// Toggle: removes if present, adds if absent
el.classList.toggle('expanded');
// Returns true if class was added, false if removed

// Conditional toggle (second argument forces add/remove)
el.classList.toggle('dark', prefersDark); // add if true, remove if false

// Check
el.classList.contains('active'); // true or false

// Replace
el.classList.replace('old-class', 'new-class');
// Returns true if old-class existed and was replaced, false otherwise

// Iterate
for (const cls of el.classList) {
  console.log(cls);
}

// Length and indexed access
el.classList.length; // number of classes
el.classList[0];     // first class name
el.classList.item(0); // same as [0]
```

**Gotcha**: `classList.add` does not throw if the class already exists, it simply does nothing. `classList.remove` does not throw if the class is absent.

## Styles and Computed Styles

### element.style

The `style` property is a `CSSStyleDeclaration` representing the element's **inline** styles only. It does not reflect styles from stylesheets.

```js
const el = document.querySelector('.box');

// Setting individual properties (camelCase)
el.style.backgroundColor = 'red';
el.style.fontSize = '16px';
el.style.marginTop = '10px';

// CSS custom properties
el.style.setProperty('--card-bg', '#fff');

// Reading: only returns inline styles, not computed styles
el.style.color; // "" if color is set via stylesheet, not inline

// Removing a specific inline style
el.style.backgroundColor = ''; // removes the inline style
el.style.removeProperty('background-color'); // same, but uses CSS property name

// Setting multiple styles at once with cssText
el.style.cssText = 'color: red; font-size: 14px; margin: 0;';
// WARNING: cssText replaces ALL inline styles, not just the ones you set
```

**Best practice**: Prefer manipulating CSS classes over setting inline styles. It keeps style logic in CSS and behavior in JavaScript.

```js
// BAD: Mixing concerns, hard to maintain
el.style.color = isError ? 'red' : 'black';
el.style.fontWeight = isError ? 'bold' : 'normal';

// GOOD: Toggle a class
el.classList.toggle('error', isError);
```

### getComputedStyle

Returns a **read-only** `CSSStyleDeclaration` containing the final computed values of all CSS properties after applying all stylesheets, inheritance, and specificity.

```js
const el = document.querySelector('.box');
const computed = getComputedStyle(el);

// Returns resolved values (e.g., 'rgb(255, 0, 0)' not 'red')
computed.color;           // "rgb(255, 0, 0)"
computed.fontSize;        // "16px" (resolved from 1em, etc.)
computed.display;         // "flex"

// Shorthand properties may not work -- use individual properties
computed.margin;          // may return "" in some browsers
computed.marginTop;       // "10px" (reliable)

// Pseudo-elements
const beforeStyles = getComputedStyle(el, '::before');
beforeStyles.content;     // "'*'"
```

**Performance warning**: `getComputedStyle` forces a style recalculation (and potentially a reflow) if there are pending style changes. Do not call it in a loop that also modifies styles.

```js
// BAD: Layout thrashing
elements.forEach(el => {
  const height = getComputedStyle(el).height; // forces reflow
  el.style.width = height; // invalidates layout
});

// GOOD: Batch reads, then batch writes
const heights = elements.map(el => getComputedStyle(el).height);
elements.forEach((el, i) => {
  el.style.width = heights[i];
});
```

## Event Handling

### addEventListener

The primary way to attach event handlers. Supports multiple listeners per event, options, and proper `this` binding.

```js
const button = document.querySelector('.btn');

function handleClick(event) {
  console.log('Clicked:', event.target);
  console.log('this:', this); // the element (button) when using regular function
}

button.addEventListener('click', handleClick);
```

### The Event Object

Every event handler receives an `Event` object (or a subclass like `MouseEvent`, `KeyboardEvent`, `InputEvent`):

```js
element.addEventListener('click', (e) => {
  e.target;          // the element that dispatched the event (the actual click target)
  e.currentTarget;   // the element the listener is attached to
  e.type;            // "click"
  e.timeStamp;       // milliseconds since page load
  e.isTrusted;       // true if from user action, false if dispatched programmatically
  e.clientX;         // mouse X relative to viewport
  e.clientY;         // mouse Y relative to viewport
  e.pageX;           // mouse X relative to document
  e.pageY;           // mouse Y relative to document
});
```

**Gotcha**: `e.target` and `e.currentTarget` are different during bubbling:

```html
<div id="parent">
  <button id="child">Click me</button>
</div>
```

```js
document.getElementById('parent').addEventListener('click', (e) => {
  console.log(e.target);        // <button> (what was clicked)
  console.log(e.currentTarget); // <div#parent> (where listener is attached)
});
```

### Event Propagation: Capture, Target, Bubble

Events travel through the DOM in three phases:

```
Phase 1: CAPTURE  (document -> html -> body -> ... -> parent)
Phase 2: TARGET   (the element that was clicked/interacted with)
Phase 3: BUBBLE   (parent -> ... -> body -> html -> document)
```

```js
const parent = document.querySelector('.parent');
const child = document.querySelector('.child');

// Default: listener fires during BUBBLE phase
parent.addEventListener('click', () => console.log('parent bubble'));

// Capture: listener fires during CAPTURE phase
parent.addEventListener('click', () => console.log('parent capture'), true);
// or
parent.addEventListener('click', () => console.log('parent capture'), { capture: true });

child.addEventListener('click', () => console.log('child'));

// Click on child outputs:
// "parent capture"    (phase 1: capture, top-down)
// "child"             (phase 2: target)
// "parent bubble"     (phase 3: bubble, bottom-up)
```

### stopPropagation vs stopImmediatePropagation

```js
// stopPropagation: prevents the event from reaching parent elements
// but OTHER listeners on the SAME element still fire
child.addEventListener('click', (e) => {
  e.stopPropagation();
  console.log('handler 1'); // fires
});
child.addEventListener('click', () => {
  console.log('handler 2'); // also fires (same element)
});
// Parent listener: does NOT fire

// stopImmediatePropagation: stops everything -- no more listeners on this
// element OR any parent elements
child.addEventListener('click', (e) => {
  e.stopImmediatePropagation();
  console.log('handler 1'); // fires
});
child.addEventListener('click', () => {
  console.log('handler 2'); // does NOT fire
});
// Parent listener: does NOT fire
```

**Gotcha**: Avoid `stopPropagation` when possible. It breaks event delegation and makes debugging difficult. Other code (analytics, third-party libraries) may depend on events bubbling to the document level.

### preventDefault

Cancels the default browser action (form submission, link navigation, text selection) without stopping propagation.

```js
document.querySelector('a').addEventListener('click', (e) => {
  e.preventDefault(); // prevents navigation
  // Event still bubbles to parent elements
});

document.querySelector('form').addEventListener('submit', (e) => {
  e.preventDefault(); // prevents form submission
  // Handle form data with JavaScript
});

// Check if prevention is possible
element.addEventListener('click', (e) => {
  if (e.cancelable) {
    e.preventDefault();
  }
});
```

### addEventListener Options

The third argument can be a boolean (capture) or an options object:

```js
element.addEventListener('click', handler, {
  capture: false,   // fire during capture phase (default: false)
  once: true,       // auto-remove after first invocation
  passive: true,    // promise not to call preventDefault()
  signal: controller.signal // remove via AbortController
});
```

#### once

```js
// The listener fires exactly once, then is automatically removed
button.addEventListener('click', () => {
  console.log('This only runs once');
}, { once: true });
```

#### passive

Declares that the listener will never call `preventDefault()`. This allows the browser to start scrolling immediately without waiting for the handler to complete.

```js
// Passive listeners improve scroll performance
document.addEventListener('touchstart', handleTouch, { passive: true });

// Gotcha: calling preventDefault() in a passive listener is silently ignored
// (Chrome logs a warning to the console)
document.addEventListener('wheel', (e) => {
  e.preventDefault(); // Does nothing if passive: true, logs a warning
}, { passive: true });

// Modern browsers default to passive: true for touchstart, touchmove,
// and wheel events on document-level targets (window, document, body)
```

#### signal (AbortController)

The modern way to remove event listeners. No need to keep a reference to the handler function.

```js
const controller = new AbortController();

// Add multiple listeners tied to the same controller
element.addEventListener('click', handleClick, { signal: controller.signal });
element.addEventListener('mouseover', handleHover, { signal: controller.signal });
window.addEventListener('resize', handleResize, { signal: controller.signal });

// Remove ALL of them at once
controller.abort();

// Extremely useful for cleanup in component lifecycles
function initComponent(element) {
  const controller = new AbortController();
  const { signal } = controller;

  element.addEventListener('click', onClick, { signal });
  element.addEventListener('keydown', onKeyDown, { signal });
  window.addEventListener('scroll', onScroll, { signal });
  document.addEventListener('visibilitychange', onVisibility, { signal });

  return {
    destroy() {
      controller.abort(); // clean removal of all listeners
    }
  };
}
```

**Advantage over removeEventListener**: You do not need to keep a reference to the exact same function. Anonymous functions and arrow functions work fine.

```js
// BAD: Cannot remove -- no reference to the anonymous function
element.addEventListener('click', () => console.log('click'));
element.removeEventListener('click', () => console.log('click')); // does nothing

// GOOD: AbortController handles it
const controller = new AbortController();
element.addEventListener('click', () => console.log('click'), {
  signal: controller.signal
});
controller.abort(); // removed
```

### removeEventListener

Requires the exact same function reference, capture flag, and event type:

```js
function handler(e) { console.log(e.type); }

element.addEventListener('click', handler);
element.removeEventListener('click', handler); // works

// Gotcha: capture flag must match
element.addEventListener('click', handler, true); // capture
element.removeEventListener('click', handler, true); // must also pass true
element.removeEventListener('click', handler, false); // does NOT remove the capture listener

// Gotcha: bound functions create new references
class Component {
  constructor(el) {
    // BAD: bind creates a new function each time
    el.addEventListener('click', this.handleClick.bind(this));
    el.removeEventListener('click', this.handleClick.bind(this)); // not the same reference

    // GOOD: store the bound reference
    this.boundClick = this.handleClick.bind(this);
    el.addEventListener('click', this.boundClick);
    el.removeEventListener('click', this.boundClick); // works
  }
}
```

### Event Delegation

Attach a single listener to a parent element and use `event.target` to determine which child triggered the event. Handles dynamically added elements automatically.

```js
// BAD: One listener per item. Fails for dynamically added items.
document.querySelectorAll('.item').forEach(item => {
  item.addEventListener('click', handleItemClick);
});

// GOOD: One listener on the parent, handles all children
document.querySelector('.item-list').addEventListener('click', (e) => {
  // closest() walks up from the target to find the matching ancestor
  const item = e.target.closest('.item');
  if (!item) return; // click was not inside an .item

  // Verify the found item is actually inside our list (not a different .item)
  if (!e.currentTarget.contains(item)) return;

  handleItemClick(item);
});
```

**Advanced delegation with multiple actions**:

```js
document.querySelector('.toolbar').addEventListener('click', (e) => {
  const button = e.target.closest('[data-action]');
  if (!button) return;

  const action = button.dataset.action;
  switch (action) {
    case 'save':    save(); break;
    case 'delete':  remove(); break;
    case 'export':  exportData(); break;
  }
});
```

**Limitation**: Some events do not bubble and therefore cannot be delegated. These include `focus`, `blur`, `mouseenter`, `mouseleave`, `load`, `unload`, `scroll` (on non-document elements), and `resize`.

Workaround for focus/blur: use `focusin`/`focusout` instead, which do bubble.

```js
// BAD: focus does not bubble
parentForm.addEventListener('focus', handler); // never fires for child inputs

// GOOD: focusin bubbles
parentForm.addEventListener('focusin', handler); // fires when any child input gets focus
```

## MutationObserver

Watches for changes to the DOM tree and fires a callback with a batch of mutations. Replaces the deprecated `MutationEvent` API.

### Basic Usage

```js
const observer = new MutationObserver((mutations) => {
  for (const mutation of mutations) {
    console.log(mutation.type);       // "childList", "attributes", or "characterData"
    console.log(mutation.target);     // the node that changed
    console.log(mutation.addedNodes); // NodeList of added nodes
    console.log(mutation.removedNodes);
  }
});

const target = document.querySelector('.dynamic-content');
observer.observe(target, {
  childList: true,    // watch for added/removed children
  attributes: true,   // watch for attribute changes
  characterData: true, // watch for text content changes
  subtree: true,       // observe the entire subtree, not just direct children
  attributeOldValue: true,    // include the old attribute value in the mutation record
  characterDataOldValue: true, // include the old text content
  attributeFilter: ['class', 'data-state'] // only watch specific attributes
});
```

### Configuration Options

| Option                   | Type      | Description                                              |
|--------------------------|-----------|----------------------------------------------------------|
| `childList`              | boolean   | Watch for child node additions/removals                  |
| `attributes`             | boolean   | Watch for attribute changes                              |
| `characterData`          | boolean   | Watch for text node changes                              |
| `subtree`                | boolean   | Extend observation to the entire subtree                 |
| `attributeOldValue`      | boolean   | Record the previous attribute value                      |
| `characterDataOldValue`  | boolean   | Record the previous text content                         |
| `attributeFilter`        | string[]  | Only observe listed attributes                           |

At least one of `childList`, `attributes`, or `characterData` must be `true`. Otherwise, a `TypeError` is thrown.

### Practical: Wait for an Element to Appear

```js
function waitForElement(selector) {
  return new Promise((resolve) => {
    // Check if it already exists
    const existing = document.querySelector(selector);
    if (existing) {
      resolve(existing);
      return;
    }

    const observer = new MutationObserver((mutations, obs) => {
      const el = document.querySelector(selector);
      if (el) {
        obs.disconnect(); // stop observing once found
        resolve(el);
      }
    });

    observer.observe(document.body, {
      childList: true,
      subtree: true
    });
  });
}

// Usage
const el = await waitForElement('.lazy-loaded-widget');
```

### Performance Considerations

MutationObserver callbacks fire on the **microtask queue**, making them much faster than the old `MutationEvent` which fired synchronously for every single change. MutationObserver batches mutations and delivers them all at once.

However:

```js
// BAD: Observing the entire document for everything
observer.observe(document.body, {
  childList: true,
  attributes: true,
  characterData: true,
  subtree: true
});
// This captures every single DOM change on the page. Extremely expensive
// for complex pages with frequent updates.

// GOOD: Scope to the specific element and specific changes you care about
observer.observe(specificContainer, {
  childList: true,
  subtree: false,
  attributeFilter: ['data-state']
});
```

### Cleanup

Always disconnect observers when they are no longer needed:

```js
// Stop observing
observer.disconnect();

// Get pending mutations before disconnecting
const pendingMutations = observer.takeRecords();
observer.disconnect();
```

## DOM Performance

### Reflow and Repaint

**Repaint**: The browser redraws pixels on screen. Triggered by changes to visual properties that do not affect layout (color, background-color, visibility, box-shadow, outline).

**Reflow** (also called "layout"): The browser recalculates the geometry (position, size) of elements. More expensive than repaint. Triggered by changes to layout-affecting properties (width, height, margin, padding, display, position, font-size) and by reading layout properties.

### What Triggers Reflow

Any of these **reads** force the browser to flush pending style changes and compute the layout:

```js
// Reading any of these properties forces a synchronous reflow:
element.offsetTop;
element.offsetLeft;
element.offsetWidth;
element.offsetHeight;
element.scrollTop;
element.scrollWidth;
element.clientWidth;
element.clientHeight;
element.getBoundingClientRect();
getComputedStyle(element).anything;
element.focus(); // causes layout to calculate scroll position
```

### Layout Thrashing

Reading a layout property after writing a style change forces an immediate reflow. Doing this in a loop is called "layout thrashing" and causes severe performance degradation.

```js
// BAD: Layout thrashing -- N reflows in a loop
const elements = document.querySelectorAll('.box');
elements.forEach(el => {
  const width = el.offsetWidth;      // READ -- forces reflow
  el.style.width = (width + 10) + 'px'; // WRITE -- invalidates layout
  // Next iteration: offsetWidth forces ANOTHER reflow
});

// GOOD: Batch reads, then batch writes
const widths = [];
elements.forEach(el => {
  widths.push(el.offsetWidth); // all reads first (single reflow)
});
elements.forEach((el, i) => {
  el.style.width = (widths[i] + 10) + 'px'; // all writes (one reflow at end)
});
```

### requestAnimationFrame for DOM Writes

Schedule style mutations just before the browser's next paint:

```js
// GOOD: Write in rAF, read outside
const currentHeight = element.offsetHeight; // read

requestAnimationFrame(() => {
  element.style.height = (currentHeight * 2) + 'px'; // write in next frame
});

// PATTERN: Separate read and write frames
function animateElement(el) {
  const startHeight = el.offsetHeight; // read

  requestAnimationFrame(() => {
    el.style.height = (startHeight + 100) + 'px'; // write
  });
}
```

### Batch DOM Operations with DocumentFragment

```js
// BAD: 1000 individual DOM insertions, 1000 reflows
const ul = document.querySelector('ul');
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  ul.appendChild(li); // reflow on each append
}

// GOOD: Build in a DocumentFragment, insert once
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li); // no reflow (fragment is not in the DOM)
}
ul.appendChild(fragment); // single reflow
```

### Minimize DOM Access

```js
// BAD: Accessing the DOM in every loop iteration
for (let i = 0; i < 100; i++) {
  document.getElementById('count').textContent = i;
}

// GOOD: Cache the reference
const countEl = document.getElementById('count');
for (let i = 0; i < 100; i++) {
  countEl.textContent = i;
}
// Better yet, just set the final value:
countEl.textContent = 99;
```

### Offscreen Operations

For complex DOM restructuring, temporarily remove the element from the DOM, make changes, and reinsert:

```js
const container = document.querySelector('.complex-widget');
const parent = container.parentNode;
const next = container.nextSibling;

// Remove from DOM (no reflows during manipulation)
parent.removeChild(container);

// Perform many mutations
for (let i = 0; i < 500; i++) {
  const child = document.createElement('div');
  child.textContent = `Widget ${i}`;
  container.appendChild(child);
}

// Reinsert (single reflow)
parent.insertBefore(container, next);
```

### CSS Containment for Isolation

```css
/* Tell the browser this element's layout is independent */
.widget {
  contain: layout style paint;
  /* Reflows inside .widget do not affect the rest of the page */
}

/* content-visibility: auto skips rendering of off-screen elements */
.card {
  content-visibility: auto;
  contain-intrinsic-size: 0 200px; /* estimated size for scroll calculations */
}
```

### Animations and Transforms

Properties that trigger only compositing (no layout, no paint) are the cheapest to animate:

```js
// BAD: Animating layout-triggering properties
element.style.left = newX + 'px';   // triggers reflow
element.style.top = newY + 'px';    // triggers reflow

// GOOD: Animating compositor-only properties
element.style.transform = `translate(${newX}px, ${newY}px)`; // no reflow
element.style.opacity = 0.5;  // no reflow, no repaint (compositor only)
```

### will-change

Hint to the browser to optimize for upcoming changes:

```js
// Before animation starts
element.style.willChange = 'transform, opacity';

// After animation ends (clean up to free memory)
element.addEventListener('transitionend', () => {
  element.style.willChange = 'auto';
});
```

**Gotcha**: Do not set `will-change` on too many elements. Each promoted element consumes GPU memory. Use it sparingly and remove it after the animation completes.

## Template Element and Template Literals for DOM Creation

### The `<template>` Element

Content inside `<template>` is parsed but not rendered or executed. It exists as a `DocumentFragment` accessible via the `content` property.

```html
<template id="card-template">
  <div class="card">
    <h3 class="card-title"></h3>
    <p class="card-body"></p>
  </div>
</template>
```

```js
const template = document.getElementById('card-template');

function createCard(title, body) {
  // Clone the template content (deep clone to get all children)
  const clone = template.content.cloneNode(true);

  // Fill in the content
  clone.querySelector('.card-title').textContent = title;
  clone.querySelector('.card-body').textContent = body;

  return clone; // returns a DocumentFragment
}

const container = document.querySelector('.cards');
container.appendChild(createCard('Title 1', 'Body content'));
container.appendChild(createCard('Title 2', 'More content'));
```

**Advantages**:
- HTML is not duplicated in JavaScript strings. Designers can see and edit the template in the HTML file.
- Scripts inside `<template>` do not execute. Images do not load. CSS does not apply. No side effects until cloned and inserted.
- The browser parses the HTML once. `cloneNode` is faster than parsing HTML strings repeatedly.

### Template Literals with insertAdjacentHTML

For simpler cases where you want to write HTML inline in JavaScript:

```js
function renderCard(title, body) {
  // Escape user input to prevent XSS
  const safeTitle = escapeHTML(title);
  const safeBody = escapeHTML(body);

  return `
    <div class="card">
      <h3 class="card-title">${safeTitle}</h3>
      <p class="card-body">${safeBody}</p>
    </div>
  `;
}

// Helper to escape HTML entities
function escapeHTML(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.innerHTML;
}

const container = document.querySelector('.cards');
container.insertAdjacentHTML('beforeend', renderCard('Title', 'Body'));
```

**Gotcha**: Template literals with `innerHTML` or `insertAdjacentHTML` parse HTML from a string. Any user-provided data MUST be escaped or sanitized. Use `textContent` for the escaping trick above, or a library like DOMPurify.

### Tagged Template Literals for Safe HTML

```js
function html(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const value = i < values.length ? escapeHTML(String(values[i])) : '';
    return result + str + value;
  }, '');
}

// Usage: values are automatically escaped
const card = html`
  <div class="card">
    <h3>${userProvidedTitle}</h3>
    <p>${userProvidedBody}</p>
  </div>
`;
```

## Modern DOM APIs

### Element.checkVisibility()

Checks whether an element is visible, accounting for CSS `display`, `visibility`, `opacity`, and `content-visibility`.

```js
const el = document.querySelector('.widget');

// Basic check: is the element rendered?
el.checkVisibility();
// Returns false if display:none, content-visibility:hidden, or no associated box

// With options:
el.checkVisibility({
  checkOpacity: true,       // treat opacity: 0 as invisible
  checkVisibilityCSS: true  // treat visibility: hidden as invisible
});
```

**Use case**: Determining if an element is actually visible to the user before running animations or intersection logic.

### Popover API

Native popover behavior without JavaScript positioning logic. Baseline widely available since April 2025.

```html
<!-- Declarative: no JavaScript needed -->
<button popovertarget="my-popover">Toggle</button>
<div id="my-popover" popover>
  <p>Popover content</p>
</div>
```

```js
const popover = document.getElementById('my-popover');

// Programmatic control
popover.showPopover();
popover.hidePopover();
popover.togglePopover();

// Event: fires when popover state changes
popover.addEventListener('toggle', (e) => {
  console.log(e.newState); // "open" or "closed"
  console.log(e.oldState);
});
```

**Popover types**:
- `popover="auto"`: Light-dismiss (clicking outside or pressing Escape closes it). Only one auto popover can be open at a time.
- `popover="manual"`: Must be explicitly closed. Multiple can be open simultaneously.

### Dialog Element

```html
<dialog id="confirm-dialog">
  <h2>Confirm Action</h2>
  <p>Are you sure?</p>
  <form method="dialog">
    <button value="cancel">Cancel</button>
    <button value="confirm">Confirm</button>
  </form>
</dialog>
```

```js
const dialog = document.getElementById('confirm-dialog');

// show(): opens as a non-modal dialog (no backdrop, page is interactive)
dialog.show();

// showModal(): opens as a modal (backdrop, page is inert, Escape closes)
dialog.showModal();

// close(): closes the dialog
dialog.close();
dialog.close('confirm'); // pass a return value

// The close event fires after the dialog closes
dialog.addEventListener('close', () => {
  console.log(dialog.returnValue); // "cancel" or "confirm"
});

// The cancel event fires when the user presses Escape
dialog.addEventListener('cancel', (e) => {
  // e.preventDefault() to keep the dialog open on Escape
});
```

**Gotcha**: `show()` does not add a backdrop or make the rest of the page inert. Use `showModal()` for true modal behavior.

### replaceChildren

Replaces all children of an element with the provided nodes. Cleaner than `innerHTML = ''` followed by `append`.

```js
const container = document.querySelector('.list');
const newItems = ['A', 'B', 'C'].map(text => {
  const li = document.createElement('li');
  li.textContent = text;
  return li;
});

// Removes all existing children and inserts new ones in a single operation
container.replaceChildren(...newItems);

// Clear all children
container.replaceChildren(); // no arguments = remove everything
```

## Common Mistakes with DOM Manipulation

### 1. Manipulating the DOM Before It Is Ready

```js
// BAD: Script in <head> without defer
// <script src="app.js"></script>
const el = document.getElementById('main'); // null -- <body> hasn't been parsed

// FIX 1: Use defer attribute
// <script src="app.js" defer></script>

// FIX 2: Listen for DOMContentLoaded
document.addEventListener('DOMContentLoaded', () => {
  const el = document.getElementById('main'); // works
});

// FIX 3: Place script at the bottom of <body>
```

### 2. Using innerHTML for User Content (XSS)

```js
// BAD: XSS vulnerability
commentBox.innerHTML = userComment;

// GOOD: Escaped text
commentBox.textContent = userComment;
```

### 3. Modifying Live Collections While Iterating

```js
// BAD: Skipping elements due to live collection shrinking
const items = document.getElementsByClassName('removable');
for (let i = 0; i < items.length; i++) {
  items[i].remove(); // collection shrinks, i increments -- elements skipped
}

// GOOD: Convert to array or iterate in reverse
[...items].forEach(item => item.remove());
```

### 4. Forgetting That removeEventListener Needs the Same Reference

```js
// BAD: Different function references
el.addEventListener('click', () => doThing());
el.removeEventListener('click', () => doThing()); // no-op

// GOOD: Use AbortController
const ac = new AbortController();
el.addEventListener('click', () => doThing(), { signal: ac.signal });
ac.abort(); // properly removes
```

### 5. Layout Thrashing in Loops

```js
// BAD: Interleaving reads and writes
elements.forEach(el => {
  el.style.width = el.offsetWidth + 10 + 'px'; // read+write per element
});

// GOOD: Separate reads from writes
const widths = elements.map(el => el.offsetWidth);
elements.forEach((el, i) => {
  el.style.width = widths[i] + 10 + 'px';
});
```

### 6. Not Cleaning Up Event Listeners and Observers

```js
// BAD: Listener added in a function called repeatedly -- accumulates listeners
function init(el) {
  el.addEventListener('click', handleClick);
}

// GOOD: Clean up before re-adding, or use AbortController
function init(el) {
  if (el._controller) el._controller.abort();
  el._controller = new AbortController();
  el.addEventListener('click', handleClick, { signal: el._controller.signal });
}
```

### 7. Using innerHTML to Append (Destroys Existing Children)

```js
// BAD: Destroys existing children, event listeners, and form state
container.innerHTML += '<div>New item</div>';
// This is equivalent to:
// container.innerHTML = container.innerHTML + '<div>New item</div>';
// It serializes, concatenates, reparses, and rebuilds the entire subtree.

// GOOD: Use insertAdjacentHTML
container.insertAdjacentHTML('beforeend', '<div>New item</div>');

// GOOD: Use createElement + append
const div = document.createElement('div');
div.textContent = 'New item';
container.append(div);
```

### 8. Storing Data in the DOM When JavaScript Variables Would Suffice

```js
// BAD: Using the DOM as a data store
el.setAttribute('data-count', parseInt(el.getAttribute('data-count')) + 1);

// GOOD: Keep data in JavaScript, update the DOM for display only
let count = 0;
function increment() {
  count++;
  el.textContent = count;
}
```

## Best Practices for DOM Manipulation

### 1. Minimize DOM Access

Cache references to frequently accessed elements. Every DOM read crosses the JavaScript-to-native bridge, which is measurably slower than accessing regular JavaScript objects.

```js
// Cache at the top of your module or function
const container = document.querySelector('.container');
const output = document.querySelector('#output');
```

### 2. Batch DOM Mutations

Group all your DOM writes together. Use `DocumentFragment` for bulk insertions. Use `requestAnimationFrame` for style changes.

### 3. Use Event Delegation

Attach one listener to a stable parent element instead of many listeners to individual children. This reduces memory usage and automatically handles dynamically added elements.

### 4. Use AbortController for Listener Cleanup

Tie multiple event listeners to a single `AbortController` for clean, reliable removal.

### 5. Prefer classList Over className String Manipulation

`classList` methods are atomic operations. Directly manipulating the `className` string risks accidentally removing other classes.

### 6. Prefer textContent Over innerHTML for Text

`textContent` is faster, safer (no XSS), and does not trigger HTML parsing.

### 7. Prefer CSS Classes Over Inline Styles

Keep presentation in CSS. Toggle classes from JavaScript. This keeps code maintainable and leverages the browser's stylesheet optimizations.

### 8. Disconnect Observers When Done

Call `observer.disconnect()` on `MutationObserver`, `IntersectionObserver`, and `ResizeObserver` instances when they are no longer needed.

### 9. Sanitize HTML Before Insertion

If you must insert HTML from external sources, use DOMPurify or the Sanitizer API (where available). Never trust user input with `innerHTML` or `insertAdjacentHTML`.

### 10. Profile Before Optimizing

Use Chrome DevTools Performance tab to identify actual layout thrashing and expensive reflows before applying optimizations. The Performance Monitor panel shows real-time layout shifts, reflows, and DOM node counts.

### 11. Use Modern APIs

Prefer `append`/`prepend`/`before`/`after`/`replaceWith`/`replaceChildren` over legacy methods like `appendChild`, `insertBefore`, and `removeChild`. Modern methods accept strings, multiple arguments, and have cleaner semantics.

### 12. Consider content-visibility for Large Lists

For pages with hundreds of off-screen elements, use `content-visibility: auto` in CSS to skip rendering of invisible content. Combine with `contain-intrinsic-size` for correct scroll behavior.
