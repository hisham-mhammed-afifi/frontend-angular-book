# 5.22 Web APIs

> The browser exposes dozens of JavaScript APIs beyond the DOM. These are the workhorses of production web applications: fetching data, storing state, observing elements, measuring performance, communicating across tabs, reading files, and formatting text for global audiences. This guide covers every API you will reach for regularly, with the edge cases, gotchas, and patterns that separate production code from tutorial code.

---

## Fetch API

The Fetch API replaced `XMLHttpRequest` as the standard for making network requests. It is Promise-based, works with the Streams API, and pairs with `AbortController` for cancellation.

### Basic GET Request

```js
const response = await fetch('https://api.example.com/users');
const data = await response.json();
```

### The Biggest Gotcha: fetch Does NOT Reject on HTTP Errors

This is the single most common mistake. A `404` or `500` response is still a successful network transaction. `fetch` only rejects on *network failures* (DNS resolution failed, server unreachable, CORS blocked).

```js
// BAD: This catch block will NOT fire on 404 or 500
try {
  const response = await fetch('/api/users/999');
  const data = await response.json(); // might throw if body isn't JSON
  console.log(data);
} catch (err) {
  console.log('This only catches network errors or JSON parse errors');
}

// GOOD: Check response.ok explicitly
async function fetchJSON(url, options = {}) {
  const response = await fetch(url, options);

  if (!response.ok) {
    // response.ok is true for status 200-299
    const errorBody = await response.text();
    throw new Error(`HTTP ${response.status}: ${errorBody}`);
  }

  return response.json();
}
```

### The Request Object

You can construct a `Request` object separately from calling `fetch`. This is useful when you need to clone or inspect a request before sending it.

```js
const request = new Request('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer token123'
  },
  body: JSON.stringify({ name: 'Alice' }),
  // 'cors' is the default; 'no-cors' returns an opaque response
  mode: 'cors',
  // 'same-origin' sends cookies only to same origin
  credentials: 'same-origin',
  // Signal for aborting (covered below)
  signal: undefined
});

const response = await fetch(request);
```

### The Response Object

```js
const response = await fetch('/api/data');

// Status information
console.log(response.status);     // 200
console.log(response.statusText); // "OK"
console.log(response.ok);         // true (status 200-299)
console.log(response.url);        // final URL after redirects

// Headers
console.log(response.headers.get('Content-Type'));
// Iterate all headers
for (const [key, value] of response.headers) {
  console.log(`${key}: ${value}`);
}

// Body consumption methods (each can only be called ONCE)
const text = await response.text();
// OR
const json = await response.json();
// OR
const blob = await response.blob();
// OR
const buffer = await response.arrayBuffer();
// OR
const formData = await response.formData();
```

**Gotcha: The body can only be consumed once.** Calling `.json()` after `.text()` throws. If you need to read the body multiple times, clone the response first.

```js
const response = await fetch('/api/data');
const clone = response.clone();

const text = await response.text();   // works
const json = await clone.json();      // works on the clone
```

### The Headers Object

```js
const headers = new Headers();
headers.set('Content-Type', 'application/json');
headers.append('Accept', 'application/json');
headers.has('Content-Type');  // true
headers.get('Content-Type');  // "application/json"
headers.delete('Accept');

// Pass to fetch
fetch('/api', { headers });

// Or use a plain object (most common)
fetch('/api', {
  headers: {
    'Content-Type': 'application/json',
    'X-Custom-Header': 'value'
  }
});
```

**Gotcha: Forbidden headers.** The browser silently strips certain headers like `Cookie`, `Host`, and `Origin`. You cannot set them programmatically.

### AbortController: Cancelling Requests

```js
const controller = new AbortController();

// Cancel after 5 seconds
const timeoutId = setTimeout(() => controller.abort(), 5000);

try {
  const response = await fetch('/api/slow-endpoint', {
    signal: controller.signal
  });
  clearTimeout(timeoutId);
  const data = await response.json();
} catch (err) {
  if (err.name === 'AbortError') {
    console.log('Request was cancelled');
  } else {
    throw err; // re-throw non-abort errors
  }
}
```

**Modern shortcut with `AbortSignal.timeout()`:**

```js
// Built-in timeout signal (no manual AbortController needed)
try {
  const response = await fetch('/api/data', {
    signal: AbortSignal.timeout(5000) // auto-aborts after 5s
  });
  const data = await response.json();
} catch (err) {
  if (err.name === 'TimeoutError') {
    console.log('Request timed out');
  } else if (err.name === 'AbortError') {
    console.log('Request aborted');
  }
}
```

**Combining multiple abort signals with `AbortSignal.any()`:**

```js
const userController = new AbortController();

// Abort on either user action OR timeout
const signal = AbortSignal.any([
  userController.signal,
  AbortSignal.timeout(10000)
]);

const response = await fetch('/api/data', { signal });
```

### Streaming Responses

The `response.body` is a `ReadableStream`. You can process data as it arrives instead of waiting for the entire response.

```js
const response = await fetch('/api/large-file');
const reader = response.body.getReader();
const decoder = new TextDecoder();

let result = '';
while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  // value is a Uint8Array chunk
  result += decoder.decode(value, { stream: true });
  console.log('Received chunk:', value.length, 'bytes');
}

console.log('Full response:', result);
```

**Streaming with `for await...of` (modern):**

```js
const response = await fetch('/api/stream');

for await (const chunk of response.body) {
  // chunk is a Uint8Array
  console.log('Chunk size:', chunk.byteLength);
}
```

### Production Fetch Wrapper

```js
async function apiFetch(endpoint, options = {}) {
  const { timeout = 10000, ...fetchOptions } = options;

  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);

  try {
    const response = await fetch(endpoint, {
      ...fetchOptions,
      signal: controller.signal,
      headers: {
        'Content-Type': 'application/json',
        ...fetchOptions.headers
      }
    });

    if (!response.ok) {
      let errorMessage;
      try {
        // Some APIs return error details as JSON
        const errorBody = await response.json();
        errorMessage = errorBody.message || JSON.stringify(errorBody);
      } catch {
        errorMessage = await response.text();
      }
      throw new Error(`HTTP ${response.status}: ${errorMessage}`);
    }

    return response.json();
  } catch (err) {
    if (err.name === 'AbortError') {
      throw new Error(`Request to ${endpoint} timed out after ${timeout}ms`);
    }
    throw err;
  } finally {
    clearTimeout(timeoutId);
  }
}

// Usage
const user = await apiFetch('/api/users/1');
const created = await apiFetch('/api/users', {
  method: 'POST',
  body: JSON.stringify({ name: 'Alice' }),
  timeout: 5000
});
```

## URL and URLSearchParams

### URL Construction and Parsing

The `URL` constructor parses and validates URLs. It throws `TypeError` for invalid URLs.

```js
const url = new URL('https://example.com:8080/path?q=test#section');

console.log(url.protocol);  // "https:"
console.log(url.hostname);  // "example.com"
console.log(url.port);      // "8080"
console.log(url.host);      // "example.com:8080"
console.log(url.pathname);  // "/path"
console.log(url.search);    // "?q=test"
console.log(url.hash);      // "#section"
console.log(url.origin);    // "https://example.com:8080"
console.log(url.href);      // the full URL string
```

**Relative URLs with a base:**

```js
const url = new URL('/api/users', 'https://example.com');
console.log(url.href); // "https://example.com/api/users"

// Gotcha: relative path resolution follows standard rules
const url2 = new URL('../other', 'https://example.com/api/v1/');
console.log(url2.href); // "https://example.com/api/other"
```

**URL validation:**

```js
function isValidURL(string) {
  try {
    new URL(string);
    return true;
  } catch {
    return false;
  }
}

// URL.canParse() is the modern alternative (no try/catch needed)
console.log(URL.canParse('https://example.com')); // true
console.log(URL.canParse('not-a-url'));            // false
```

### URLSearchParams

```js
// Create from string
const params = new URLSearchParams('?page=1&sort=name&sort=date');

// Create from object
const params2 = new URLSearchParams({ page: '1', limit: '20' });

// Create from array of pairs
const params3 = new URLSearchParams([['page', '1'], ['limit', '20']]);

// Reading
params.get('page');        // "1"
params.getAll('sort');     // ["name", "date"]  (multiple values)
params.has('page');        // true
params.toString();         // "page=1&sort=name&sort=date"

// Modifying
params.set('page', '2');   // replaces ALL values for 'page'
params.append('tag', 'js');// adds another entry (doesn't replace)
params.delete('sort');     // removes ALL entries with key 'sort'

// Iteration
for (const [key, value] of params) {
  console.log(`${key} = ${value}`);
}

// Sorting (alphabetical by key)
params.sort();
```

**Integration with URL:**

```js
const url = new URL('https://api.example.com/search');
url.searchParams.set('q', 'hello world');
url.searchParams.set('page', '1');
console.log(url.href); // "https://api.example.com/search?q=hello+world&page=1"
```

**Gotcha: URLSearchParams encodes spaces as `+`, not `%20`.** This is correct for `application/x-www-form-urlencoded` encoding but may surprise you.

```js
const params = new URLSearchParams({ q: 'hello world' });
console.log(params.toString()); // "q=hello+world" (not "q=hello%20world")

// If you need %20 encoding, use encodeURIComponent
console.log(encodeURIComponent('hello world')); // "hello%20world"
```

**Gotcha: all values are strings.**

```js
const params = new URLSearchParams({ count: 5, active: true });
console.log(params.get('count'));  // "5" (string, not number)
console.log(params.get('active')); // "true" (string, not boolean)
```

## Storage APIs

### localStorage and sessionStorage

Both implement the same `Storage` interface but differ in persistence.

| Feature | localStorage | sessionStorage |
|---------|-------------|----------------|
| Persistence | Until explicitly cleared | Until tab/window closes |
| Scope | Same origin (all tabs) | Same origin, same tab only |
| Capacity | ~5-10 MB (browser-dependent) | ~5-10 MB (browser-dependent) |
| Cross-tab | Yes | No |
| Survives page reload | Yes | Yes |
| Storage event | Yes (fires in other tabs) | Yes (fires in other tabs) |

```js
// Store data (values must be strings)
localStorage.setItem('user', JSON.stringify({ name: 'Alice', age: 30 }));

// Retrieve data
const user = JSON.parse(localStorage.getItem('user'));

// Remove single item
localStorage.removeItem('user');

// Clear everything for this origin
localStorage.clear();

// Get number of stored items
console.log(localStorage.length);

// Get key by index
console.log(localStorage.key(0));
```

### Storage Gotchas

**1. Everything is a string.** Forgetting to serialize/deserialize is a classic bug.

```js
// BAD: Storing an object directly
localStorage.setItem('config', { debug: true });
console.log(localStorage.getItem('config')); // "[object Object]"

// GOOD: Serialize with JSON
localStorage.setItem('config', JSON.stringify({ debug: true }));
const config = JSON.parse(localStorage.getItem('config'));
```

**2. `JSON.parse(null)` returns `null`, not an error.** But `JSON.parse(undefined)` throws.

```js
// Safe pattern for reading
function getStoredJSON(key) {
  const raw = localStorage.getItem(key);
  if (raw === null) return null;
  try {
    return JSON.parse(raw);
  } catch {
    return null; // corrupted data
  }
}
```

**3. Synchronous and blocking.** Storage operations run on the main thread. Large reads/writes block rendering.

**4. Quota limits throw.** When storage is full, `setItem()` throws `QuotaExceededError`.

```js
try {
  localStorage.setItem('key', largeData);
} catch (err) {
  if (err.name === 'QuotaExceededError') {
    console.error('Storage quota exceeded');
    // Clean up old data or notify user
  }
}
```

**5. Private/incognito mode.** Safari mobile used to throw immediately on any storage access in private mode. Modern Safari allows storage but clears it when the session ends. Always wrap in try/catch.

```js
function isStorageAvailable() {
  try {
    const testKey = '__storage_test__';
    localStorage.setItem(testKey, 'test');
    localStorage.removeItem(testKey);
    return true;
  } catch {
    return false;
  }
}
```

**6. Key conflicts with built-in methods.** If you use object-style access, user-defined keys can shadow `Storage` methods.

```js
// BAD: Object-style access can collide with methods
localStorage.length = 'not a number'; // shadows the built-in .length property

// GOOD: Always use getItem/setItem
localStorage.setItem('length', '42');
localStorage.getItem('length'); // "42"
console.log(localStorage.length); // still the correct count
```

### The storage Event (Cross-Tab Synchronization)

The `storage` event fires in *other* tabs/windows of the same origin when localStorage changes. It does NOT fire in the tab that made the change.

```js
window.addEventListener('storage', (event) => {
  console.log('Key changed:', event.key);
  console.log('Old value:', event.oldValue);
  console.log('New value:', event.newValue);
  console.log('URL that made change:', event.url);
  console.log('Storage area:', event.storageArea); // localStorage or sessionStorage

  if (event.key === 'auth_token' && event.newValue === null) {
    // Another tab logged out; redirect to login
    window.location.href = '/login';
  }
});
```

**Gotcha: Setting the same value does not fire the event.** The browser optimizes away no-op writes.

```js
localStorage.setItem('theme', 'dark');
localStorage.setItem('theme', 'dark'); // no event fired in other tabs
```

### IndexedDB Overview

IndexedDB is an asynchronous, transactional, object-oriented database in the browser. Use it when localStorage's ~5 MB limit or string-only storage is insufficient.

| Feature | localStorage | IndexedDB |
|---------|-------------|-----------|
| Capacity | ~5-10 MB | Hundreds of MB to GB+ |
| Data types | Strings only | Structured clone (objects, arrays, blobs, dates) |
| Queries | Key lookup only | Indexes, key ranges, cursors |
| API style | Synchronous | Asynchronous (event-based) |
| Transactions | No | Yes (ACID) |
| Web Worker access | No | Yes |

```js
// Open (or create) a database
const request = indexedDB.open('MyApp', 1);

request.onupgradeneeded = (event) => {
  const db = event.target.result;

  // Create an object store (like a table)
  const store = db.createObjectStore('users', { keyPath: 'id' });
  store.createIndex('email', 'email', { unique: true });
  store.createIndex('age', 'age', { unique: false });
};

request.onsuccess = (event) => {
  const db = event.target.result;

  // Write data
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  store.put({ id: 1, name: 'Alice', email: 'alice@example.com', age: 30 });
  store.put({ id: 2, name: 'Bob', email: 'bob@example.com', age: 25 });

  tx.oncomplete = () => console.log('Data saved');

  // Read data
  const readTx = db.transaction('users', 'readonly');
  const readStore = readTx.objectStore('users');
  const getRequest = readStore.get(1);
  getRequest.onsuccess = () => console.log(getRequest.result);
};

request.onerror = (event) => {
  console.error('Database error:', event.target.error);
};
```

**IndexedDB limitations:**
- No built-in change events across tabs (unlike the `storage` event for localStorage).
- Transaction performance is the bottleneck, not data throughput.
- The event-based API is verbose. Libraries like `idb` provide Promise-based wrappers.
- No full-text search without manual implementation.

## Clipboard API

The async Clipboard API replaces the deprecated `document.execCommand('copy')`.

### Writing to Clipboard

```js
// Write plain text (most common)
await navigator.clipboard.writeText('Hello, clipboard!');

// Write rich content (HTML, images, etc.)
const blob = new Blob(['<b>Bold text</b>'], { type: 'text/html' });
const item = new ClipboardItem({ 'text/html': blob });
await navigator.clipboard.write([item]);
```

### Reading from Clipboard

```js
// Read plain text
const text = await navigator.clipboard.readText();
console.log('Clipboard text:', text);

// Read all formats
const items = await navigator.clipboard.read();
for (const item of items) {
  for (const type of item.types) {
    const blob = await item.getType(type);
    console.log(`Type: ${type}, Size: ${blob.size}`);
  }
}
```

### Permissions and Security

| Operation | Permission needed | User gesture required |
|-----------|------------------|----------------------|
| `writeText()` / `write()` | `clipboard-write` (auto-granted for active tab) | No (in active tab) |
| `readText()` / `read()` | `clipboard-read` (user must grant) | Yes (user interaction) |

```js
// Check permission before reading
const result = await navigator.permissions.query({ name: 'clipboard-read' });
if (result.state === 'granted' || result.state === 'prompt') {
  const text = await navigator.clipboard.readText();
}
```

**Gotcha: HTTPS only.** The Clipboard API is unavailable on `http://` pages (except `localhost`).

**Gotcha: Focus required.** The page must be focused. If the user switches tabs during an async operation, clipboard access may fail.

```js
// Production pattern with fallback
async function copyToClipboard(text) {
  try {
    await navigator.clipboard.writeText(text);
    return true;
  } catch {
    // Fallback for older browsers or non-secure contexts
    const textarea = document.createElement('textarea');
    textarea.value = text;
    textarea.style.position = 'fixed';
    textarea.style.opacity = '0';
    document.body.appendChild(textarea);
    textarea.select();
    const success = document.execCommand('copy');
    document.body.removeChild(textarea);
    return success;
  }
}
```

## Geolocation API

The Geolocation API provides the device's geographic position. It requires user permission and HTTPS.

### getCurrentPosition

```js
navigator.geolocation.getCurrentPosition(
  // Success callback
  (position) => {
    console.log('Latitude:', position.coords.latitude);
    console.log('Longitude:', position.coords.longitude);
    console.log('Accuracy (meters):', position.coords.accuracy);
    console.log('Altitude:', position.coords.altitude);         // may be null
    console.log('Heading:', position.coords.heading);           // may be null
    console.log('Speed:', position.coords.speed);               // may be null
    console.log('Timestamp:', position.timestamp);
  },
  // Error callback
  (error) => {
    switch (error.code) {
      case GeolocationPositionError.PERMISSION_DENIED:     // 1
        console.error('User denied geolocation permission');
        break;
      case GeolocationPositionError.POSITION_UNAVAILABLE:  // 2
        console.error('Position unavailable (device issue)');
        break;
      case GeolocationPositionError.TIMEOUT:               // 3
        console.error('Geolocation request timed out');
        break;
    }
  },
  // Options
  {
    enableHighAccuracy: true,  // GPS; uses more battery
    timeout: 10000,            // max wait time in ms
    maximumAge: 0              // 0 = no cached position; always fetch fresh
  }
);
```

### watchPosition (Continuous Tracking)

```js
const watchId = navigator.geolocation.watchPosition(
  (position) => {
    updateMapMarker(position.coords.latitude, position.coords.longitude);
  },
  (error) => {
    console.error('Watch error:', error.message);
  },
  {
    enableHighAccuracy: true,
    timeout: 15000,
    maximumAge: 0
  }
);

// Stop watching when done
navigator.geolocation.clearWatch(watchId);
```

**Gotcha: No Promise-based API.** Wrap it yourself:

```js
function getPosition(options = {}) {
  return new Promise((resolve, reject) => {
    navigator.geolocation.getCurrentPosition(resolve, reject, options);
  });
}

const position = await getPosition({ enableHighAccuracy: true });
```

**Gotcha: `maximumAge` is often misunderstood.** Setting `maximumAge: 60000` means the browser can return a cached position up to 60 seconds old. Setting it to `Infinity` will always return a cached position if one exists.

## Intersection Observer

Intersection Observer asynchronously reports when elements enter or exit the viewport (or another ancestor element). It replaced scroll event listeners for visibility detection.

### Basic Usage

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    console.log('Target:', entry.target);
    console.log('Is visible:', entry.isIntersecting);
    console.log('Intersection ratio:', entry.intersectionRatio); // 0.0 to 1.0
    console.log('Bounding rect:', entry.boundingClientRect);
  });
});

observer.observe(document.querySelector('#target'));

// Stop observing
observer.unobserve(document.querySelector('#target'));

// Stop observing everything
observer.disconnect();
```

### Options

```js
const observer = new IntersectionObserver(callback, {
  // Root element (null = viewport)
  root: null,

  // Margin around root (accepts CSS-like values)
  rootMargin: '0px 0px -100px 0px', // trigger 100px before bottom

  // Fire at these visibility thresholds
  threshold: [0, 0.25, 0.5, 0.75, 1.0]
});
```

### Lazy Loading Images

```js
function lazyLoadImages() {
  const images = document.querySelectorAll('img[data-src]');

  const observer = new IntersectionObserver((entries, obs) => {
    entries.forEach((entry) => {
      if (!entry.isIntersecting) return;

      const img = entry.target;
      img.src = img.dataset.src;

      // Optional: also handle srcset
      if (img.dataset.srcset) {
        img.srcset = img.dataset.srcset;
      }

      img.removeAttribute('data-src');
      obs.unobserve(img); // stop observing once loaded
    });
  }, {
    rootMargin: '200px 0px' // start loading 200px before visible
  });

  images.forEach((img) => observer.observe(img));
}
```

**HTML for lazy loading:**

```html
<!-- Set data-src instead of src; browser won't load the image -->
<img data-src="/images/photo.jpg" alt="Lazy loaded photo"
     width="800" height="600">
```

**Note:** Modern browsers support native lazy loading with `loading="lazy"`, which may be simpler for basic image deferral. Intersection Observer gives you fine-grained control.

### Infinite Scroll

```js
function setupInfiniteScroll(sentinel, loadMore) {
  const observer = new IntersectionObserver(async (entries) => {
    const entry = entries[0];
    if (!entry.isIntersecting) return;

    observer.unobserve(sentinel); // prevent duplicate triggers
    await loadMore();
    observer.observe(sentinel);   // re-observe after content loads
  }, {
    rootMargin: '300px' // trigger before the sentinel is visible
  });

  observer.observe(sentinel);
  return observer;
}

// Usage
const sentinel = document.querySelector('#scroll-sentinel');
setupInfiniteScroll(sentinel, async () => {
  const items = await fetchNextPage();
  appendItems(items);
});
```

### Visibility Tracking (Analytics)

```js
// Track which sections users actually see
function trackVisibility(elements, onVisible) {
  const seen = new Set();

  const observer = new IntersectionObserver((entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting && !seen.has(entry.target)) {
        seen.add(entry.target);
        onVisible(entry.target);
        observer.unobserve(entry.target);
      }
    });
  }, {
    threshold: 0.5 // at least 50% visible
  });

  elements.forEach((el) => observer.observe(el));
  return observer;
}
```

**Gotcha: Callbacks fire on the main thread.** Do not run heavy computation in the observer callback. Dispatch the work to a microtask or requestAnimationFrame.

**Gotcha: Set explicit dimensions on observed elements.** Without width/height, all images may appear "intersecting" initially because they have zero height and are stacked at the top.

## Resize Observer

Resize Observer monitors element size changes. Unlike `window.resize`, it observes individual elements and fires on content/layout changes, not just viewport changes.

### Basic Usage

```js
const observer = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const { width, height } = entry.contentRect;
    console.log(`${entry.target.id}: ${width}x${height}`);

    // Modern: use borderBoxSize for more precision
    if (entry.borderBoxSize?.length) {
      const { inlineSize, blockSize } = entry.borderBoxSize[0];
      console.log(`Border box: ${inlineSize}x${blockSize}`);
    }
  }
});

observer.observe(document.querySelector('#my-element'));
```

### Size Properties

| Property | Description |
|----------|-------------|
| `contentRect` | A `DOMRectReadOnly` with content box dimensions |
| `contentBoxSize` | Array of `{inlineSize, blockSize}` for the content box |
| `borderBoxSize` | Array of `{inlineSize, blockSize}` including padding and border |
| `devicePixelContentBoxSize` | Content size in physical device pixels (for high-DPI) |

### Responsive Component Pattern

```js
// Switch layout between row and column based on element width
const container = document.querySelector('.card-container');

const observer = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const width = entry.contentRect.width;
    const el = entry.target;

    if (width < 400) {
      el.classList.add('layout-column');
      el.classList.remove('layout-row');
    } else {
      el.classList.add('layout-row');
      el.classList.remove('layout-column');
    }
  }
});

observer.observe(container);
```

### Chart/Canvas Resizing

```js
const canvas = document.querySelector('#chart');
const ctx = canvas.getContext('2d');

const observer = new ResizeObserver((entries) => {
  const entry = entries[0];

  // Use devicePixelContentBoxSize for crisp rendering on high-DPI screens
  if (entry.devicePixelContentBoxSize?.length) {
    canvas.width = entry.devicePixelContentBoxSize[0].inlineSize;
    canvas.height = entry.devicePixelContentBoxSize[0].blockSize;
  } else {
    // Fallback
    const dpr = window.devicePixelRatio || 1;
    canvas.width = entry.contentRect.width * dpr;
    canvas.height = entry.contentRect.height * dpr;
  }

  redrawChart(); // re-render at new size
});

observer.observe(canvas);
```

**Gotcha: The callback fires immediately when you first observe an element.** This is by design, so you get the initial size, but be aware of it if your callback has side effects.

**Gotcha: Infinite loops.** If your callback resizes the observed element, you can trigger an infinite observation loop. The browser will throw an error and break the loop, but your layout will be broken.

## Performance API

The Performance API lets you measure precise timing for navigation, resource loading, and custom operations.

### performance.now()

Returns a high-resolution timestamp in milliseconds (with sub-millisecond precision) relative to `performance.timeOrigin`.

```js
const start = performance.now();
// ... expensive operation ...
const end = performance.now();
console.log(`Operation took ${end - start}ms`);
```

**Gotcha: In secure contexts, `performance.now()` has microsecond precision. In non-secure contexts, browsers reduce resolution to ~100 microseconds (or more) to mitigate Spectre-class timing attacks.**

### performance.mark() and performance.measure()

Create named timestamps and measure durations between them.

```js
// Create marks
performance.mark('fetch-start');
const data = await fetch('/api/users').then(r => r.json());
performance.mark('fetch-end');

performance.mark('render-start');
renderUsers(data);
performance.mark('render-end');

// Measure between marks
performance.measure('fetch-duration', 'fetch-start', 'fetch-end');
performance.measure('render-duration', 'render-start', 'render-end');

// Retrieve measurements
const measures = performance.getEntriesByType('measure');
measures.forEach((m) => {
  console.log(`${m.name}: ${m.duration.toFixed(2)}ms`);
});

// Clean up
performance.clearMarks();
performance.clearMeasures();
```

**Measure with detail metadata:**

```js
performance.mark('api-call', {
  detail: { endpoint: '/api/users', method: 'GET' }
});
```

### PerformanceObserver

Observe performance entries as they happen, instead of polling.

```js
// Observe custom marks and measures
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(`[${entry.entryType}] ${entry.name}: ${entry.duration || 0}ms`);
  }
});

observer.observe({ entryTypes: ['mark', 'measure'] });
```

**Observe resource loading:**

```js
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(`Resource: ${entry.name}`);
    console.log(`  Type: ${entry.initiatorType}`);      // "script", "img", "fetch", etc.
    console.log(`  Duration: ${entry.duration}ms`);
    console.log(`  Transfer size: ${entry.transferSize} bytes`);
    console.log(`  Cached: ${entry.transferSize === 0}`);
  }
});

observer.observe({ entryTypes: ['resource'] });
```

**Observe navigation timing:**

```js
const observer = new PerformanceObserver((list) => {
  const nav = list.getEntries()[0];
  console.log(`DOM Content Loaded: ${nav.domContentLoadedEventEnd}ms`);
  console.log(`Load Event: ${nav.loadEventEnd}ms`);
  console.log(`DOM Interactive: ${nav.domInteractive}ms`);
  console.log(`Response Start (TTFB): ${nav.responseStart}ms`);
});

observer.observe({ entryTypes: ['navigation'] });
```

**Observe Largest Contentful Paint (LCP), a Core Web Vital:**

```js
const observer = new PerformanceObserver((list) => {
  const entries = list.getEntries();
  const lastEntry = entries[entries.length - 1];
  console.log('LCP:', lastEntry.startTime, 'ms');
  console.log('Element:', lastEntry.element);
});

observer.observe({ type: 'largest-contentful-paint', buffered: true });
```

### buffered Option

Setting `buffered: true` retrieves entries that were recorded before the observer was created. Essential for entries like navigation timing that fire before your JavaScript runs.

```js
const observer = new PerformanceObserver((list) => {
  // Will include entries from before this observer existed
  list.getEntries().forEach((entry) => console.log(entry));
});

observer.observe({ type: 'resource', buffered: true });
```

## Broadcast Channel API

Enables communication between tabs, windows, iframes, and workers of the same origin. No server required.

### Basic Usage

```js
// Tab 1: Create channel and listen
const channel = new BroadcastChannel('app-events');

channel.onmessage = (event) => {
  console.log('Received:', event.data);
};

// Tab 2: Send message on the same channel name
const channel = new BroadcastChannel('app-events');
channel.postMessage({ type: 'THEME_CHANGE', theme: 'dark' });
```

**Gotcha: A tab does NOT receive its own messages.** `postMessage` only delivers to *other* browsing contexts on the same channel.

### Production Patterns

```js
// Synchronized logout across all tabs
const authChannel = new BroadcastChannel('auth');

function logout() {
  clearAuthTokens();
  authChannel.postMessage({ type: 'LOGOUT' });
  window.location.href = '/login';
}

authChannel.onmessage = (event) => {
  if (event.data.type === 'LOGOUT') {
    clearAuthTokens();
    window.location.href = '/login';
  }
};

// Always close when done
window.addEventListener('beforeunload', () => {
  authChannel.close();
});
```

```js
// Shared state synchronization
const stateChannel = new BroadcastChannel('shared-state');

stateChannel.onmessage = (event) => {
  switch (event.data.type) {
    case 'CART_UPDATED':
      updateCartBadge(event.data.itemCount);
      break;
    case 'USER_PREFS':
      applyPreferences(event.data.prefs);
      break;
  }
};

// Notify other tabs when cart changes
function addToCart(item) {
  cart.push(item);
  stateChannel.postMessage({
    type: 'CART_UPDATED',
    itemCount: cart.length
  });
}
```

### Broadcast Channel vs storage Event

| Feature | BroadcastChannel | storage event |
|---------|-----------------|---------------|
| Data types | Any structured-cloneable value | Strings only |
| Self-notification | No | No |
| Requires storage write | No | Yes (must write to localStorage) |
| Worker support | Yes | No |
| Cleanup required | `.close()` | No |

## structuredClone

`structuredClone()` creates a deep copy of a value using the structured clone algorithm. It replaced the `JSON.parse(JSON.stringify())` hack.

### Basic Usage

```js
const original = {
  name: 'Alice',
  scores: [95, 88, 72],
  metadata: {
    created: new Date('2024-01-01'),
    tags: new Set(['admin', 'active'])
  }
};

const clone = structuredClone(original);

// Truly deep: modifying nested values doesn't affect original
clone.scores.push(100);
clone.metadata.tags.add('editor');

console.log(original.scores);         // [95, 88, 72]
console.log(original.metadata.tags);  // Set(2) {"admin", "active"}
```

### What structuredClone Handles

| Type | Supported |
|------|-----------|
| Plain objects, arrays | Yes |
| `Date` | Yes |
| `Map`, `Set` | Yes |
| `ArrayBuffer`, typed arrays | Yes |
| `RegExp` | Yes |
| `Blob`, `File` | Yes |
| `ImageData` | Yes |
| Circular references | Yes |
| `Error` objects | Yes (but `stack` is not cloned) |

### What structuredClone Cannot Clone

| Type | Result |
|------|--------|
| Functions | Throws `DataCloneError` |
| DOM nodes | Throws `DataCloneError` |
| `WeakMap`, `WeakSet` | Throws `DataCloneError` |
| `Promise` | Throws `DataCloneError` |
| `Proxy` | Throws `DataCloneError` |
| Symbols (as property keys) | Silently skipped |
| Prototype chain | Stripped (class instances become plain objects) |
| Property descriptors (getters/setters) | Lost |
| Non-enumerable properties | Skipped |

```js
// Gotcha: class instances lose their prototype
class User {
  constructor(name) { this.name = name; }
  greet() { return `Hi, ${this.name}`; }
}

const user = new User('Alice');
const clone = structuredClone(user);

console.log(clone.name);           // "Alice"
console.log(clone instanceof User); // false (prototype stripped)
console.log(clone.greet);          // undefined (methods lost)
```

```js
// Gotcha: functions throw
const obj = { fn: () => 42 };
structuredClone(obj); // DataCloneError: () => 42 could not be cloned
```

### structuredClone vs JSON.parse(JSON.stringify())

| Feature | structuredClone | JSON round-trip |
|---------|----------------|-----------------|
| `Date` | Preserved as Date | Becomes string |
| `Map`, `Set` | Preserved | Lost (becomes `{}`) |
| `undefined` values | Preserved | Removed |
| `NaN`, `Infinity` | Preserved | Becomes `null` |
| Circular references | Handled | Throws |
| `RegExp` | Preserved | Becomes `{}` |
| Performance | Faster for complex objects | Faster for simple objects |

### Transfer (Move, Not Copy)

`structuredClone` supports transferring objects like `ArrayBuffer`, which moves ownership instead of copying.

```js
const buffer = new ArrayBuffer(1024);
const clone = structuredClone(buffer, { transfer: [buffer] });

console.log(buffer.byteLength);  // 0 (original is now empty/detached)
console.log(clone.byteLength);   // 1024
```

## Web Notifications API

### Requesting Permission

```js
// Only request in response to a user gesture (button click)
async function requestNotificationPermission() {
  if (!('Notification' in window)) {
    console.log('Notifications not supported');
    return 'denied';
  }

  // Check current permission
  if (Notification.permission === 'granted') return 'granted';
  if (Notification.permission === 'denied') return 'denied';

```

```js
    console.log('Notification dismissed');
  };

}
```

### Service Worker Notifications (Push)

Service worker notifications persist even after the user closes the tab.

```js
// In service worker (sw.js)
self.addEventListener('push', (event) => {
  const data = event.data?.json() ?? {};

  event.waitUntil(
```

## History API

### pushState and replaceState

```js
// pushState adds a new history entry
history.pushState(
  { page: 'products', id: 42 },  // state object (must be serializable)
  '',                              // title (ignored by most browsers)
  '/products/42'                   // URL (must be same-origin)
);

// replaceState modifies the current entry (no new entry created)
history.replaceState(
  { page: 'products', id: 42, scrollY: window.scrollY },
  '',
  '/products/42'
);
```

### The popstate Event

Fires when the user navigates with back/forward buttons.

```js
window.addEventListener('popstate', (event) => {
  console.log('State:', event.state);

  if (event.state?.page === 'products') {
    renderProductPage(event.state.id);
  } else {
    renderHomePage();
  }
});
```

**Gotcha: `pushState` and `replaceState` do NOT fire `popstate`.** Only browser back/forward triggers it.

**Gotcha: The state object is cloned with structured clone when stored, but `history.state` only reflects the current entry.** As you navigate the history, previous entries' state objects "disappear" from direct access.

### Reading Current State

```js
console.log(history.state); // the state object of the current entry
console.log(history.length); // total number of entries in the session
```

### history.scrollRestoration

```js
// Disable browser's automatic scroll restoration (useful for SPAs)
history.scrollRestoration = 'manual';

// Re-enable
history.scrollRestoration = 'auto';
```

### Navigation API (Modern Replacement)

The Navigation API is a newer standard that addresses limitations of the History API. It provides a way to intercept navigations and manage state more cleanly.

```js
// Check support
if ('navigation' in window) {
  // Intercept navigations
  navigation.addEventListener('navigate', (event) => {
    const url = new URL(event.destination.url);

    if (url.pathname.startsWith('/products/')) {
      event.intercept({
        handler: async () => {
          const id = url.pathname.split('/').pop();
          const product = await fetchProduct(id);
          renderProduct(product);
        }
      });
    }
  });

  // Navigate programmatically
  navigation.navigate('/products/42', {
    state: { fromSearch: true },
    info: { scrollToTop: true } // transient info, not persisted
  });

  // Access all entries
  const entries = navigation.entries();
  const current = navigation.currentEntry;
  console.log(current.url, current.getState());
}
```

**Key advantages of Navigation API over History API:**
- Intercept navigations before they happen (not just after).
- Access all history entries, not just the current one.
- Better state management with `getState()` on each entry.
- Transition tracking with `navigation.transition`.

## Encoding API (TextEncoder / TextDecoder)

### TextEncoder

Always encodes to UTF-8.

```js
const encoder = new TextEncoder();

// encode() returns a Uint8Array
const encoded = encoder.encode('Hello, World!');
console.log(encoded); // Uint8Array [72, 101, 108, 108, 111, 44, 32, 87, 111, 114, 108, 100, 33]
console.log(encoded.byteLength); // 13

// Multi-byte characters
const euroEncoded = encoder.encode('\u20AC'); // Euro sign
console.log(euroEncoded); // Uint8Array [226, 130, 172]  (3 bytes in UTF-8)

// encodeInto() writes directly into an existing buffer (zero-copy)
const buffer = new Uint8Array(100);
const { read, written } = encoder.encodeInto('Hello', buffer);
console.log(read);    // 5 (characters read from source)
console.log(written); // 5 (bytes written to destination)
```

### TextDecoder

Supports many encodings (UTF-8, UTF-16, ISO-8859-1, etc.).

```js
const decoder = new TextDecoder(); // defaults to UTF-8

const bytes = new Uint8Array([72, 101, 108, 108, 111]);
console.log(decoder.decode(bytes)); // "Hello"

// Decode with a different encoding
const latin1Decoder = new TextDecoder('iso-8859-1');
const latin1Bytes = new Uint8Array([0xE9]); // e with acute accent
console.log(latin1Decoder.decode(latin1Bytes)); // "e" (with accent)

// Streaming: handle multi-byte characters split across chunks
const streamDecoder = new TextDecoder('utf-8', { stream: true });

// Euro sign (U+20AC) in UTF-8: [0xE2, 0x82, 0xAC]
// Split across two chunks:
const chunk1 = new Uint8Array([0xE2, 0x82]); // incomplete character
const chunk2 = new Uint8Array([0xAC]);        // remaining byte

console.log(streamDecoder.decode(chunk1, { stream: true })); // "" (waiting)
console.log(streamDecoder.decode(chunk2)); // "\u20AC" (Euro sign)
```

### Practical Use Cases

```js
// Convert string to base64 (via Uint8Array)
function stringToBase64(str) {
  const encoder = new TextEncoder();
  const bytes = encoder.encode(str);
  const binString = Array.from(bytes, (b) => String.fromCodePoint(b)).join('');
  return btoa(binString);
}

// Convert base64 back to string
function base64ToString(base64) {
  const binString = atob(base64);
  const bytes = Uint8Array.from(binString, (c) => c.codePointAt(0));
  return new TextDecoder().decode(bytes);
}

// Hashing a string (using SubtleCrypto)
async function sha256(message) {
  const encoder = new TextEncoder();
  const data = encoder.encode(message);
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  return hashArray.map((b) => b.toString(16).padStart(2, '0')).join('');
}
```

### Uploading Files with Fetch

```js
// Simple upload
async function uploadFile(file) {
  const formData = new FormData();
  formData.append('file', file);
  formData.append('description', 'Profile photo');

  // Do NOT set Content-Type header; the browser sets it
  // with the correct multipart boundary automatically
  const response = await fetch('/api/upload', {
    method: 'POST',
    body: formData
  });

  return response.json();
}

// Upload with progress (requires XMLHttpRequest)
function uploadWithProgress(file, onProgress) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    const formData = new FormData();
    formData.append('file', file);

    xhr.upload.addEventListener('progress', (event) => {
      if (event.lengthComputable) {
        onProgress(event.loaded / event.total);
      }
    });

    xhr.addEventListener('load', () => {
      if (xhr.status >= 200 && xhr.status < 300) {
        resolve(JSON.parse(xhr.responseText));
      } else {
        reject(new Error(`Upload failed: ${xhr.status}`));
      }
    });

    xhr.addEventListener('error', () => reject(new Error('Network error')));
    xhr.open('POST', '/api/upload');
    xhr.send(formData);
  });
}
```

## navigator.sendBeacon

Designed for sending analytics and diagnostics data when the user leaves a page. Unlike `fetch` in an unload handler, `sendBeacon` is guaranteed to be queued by the browser even during page teardown.

### Basic Usage

```js
// Send analytics data when user navigates away
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    const data = JSON.stringify({
      page: location.pathname,
      timeSpent: performance.now(),
      scrollDepth: getMaxScrollDepth()
    });

    // sendBeacon sends a POST request
    // Returns true if queued, false if it failed
    const queued = navigator.sendBeacon('/api/analytics', data);
    console.log('Beacon queued:', queued);
  }
});
```

### Why visibilitychange, Not unload or beforeunload

| Event | Reliability | Back/forward cache |
|-------|------------|--------------------|
| `visibilitychange` | High (fires on tab switch, minimize, navigate) | Compatible |
| `pagehide` | Medium | Compatible |
| `beforeunload` | Low on mobile | Breaks bfcache |
| `unload` | Very low on mobile | Breaks bfcache |

**Using `unload` or `beforeunload` prevents the page from being stored in the browser's back/forward cache (bfcache).** This means navigating back to the page requires a full reload instead of an instant restore. Use `visibilitychange` instead.

### Sending Different Data Types

```js
// Plain string
navigator.sendBeacon('/log', 'page=home&action=leave');

// JSON with Blob
const blob = new Blob(
  [JSON.stringify({ event: 'exit', timestamp: Date.now() })],
  { type: 'application/json' }
);
navigator.sendBeacon('/analytics', blob);

// FormData
const formData = new FormData();
formData.append('event', 'click');
formData.append('target', 'buy-button');
navigator.sendBeacon('/track', formData);
```

**Gotcha: `sendBeacon` has a size limit (typically 64 KB).** The return value of `false` may indicate the payload was too large or the browser's pending beacon queue is full.

**Gotcha: You cannot set custom headers.** The `Content-Type` is determined by the data type: `text/plain` for strings, `multipart/form-data` for `FormData`, or whatever `type` you set on a `Blob`.

## Intl (Internationalization) APIs

The `Intl` namespace provides locale-aware formatting without external libraries. All formatters follow the same pattern: construct with locale and options, then call `.format()`.

```js
const parts = new Intl.DateTimeFormat('en-US', {
```

### Intl.NumberFormat

```js
// Plain numbers
new Intl.NumberFormat('en-US').format(1234567.89);     // "1,234,567.89"
new Intl.NumberFormat('de-DE').format(1234567.89);     // "1.234.567,89"

// Currency
new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD'
}).format(42.5);  // "$42.50"

new Intl.NumberFormat('ja-JP', {
  style: 'currency',
  currency: 'JPY'
}).format(1500);  // "JP\u00a51,500" (yen symbol + formatted number)

// Percent
new Intl.NumberFormat('en-US', {
  style: 'percent',
  minimumFractionDigits: 1
}).format(0.867);  // "86.7%"

// Units
new Intl.NumberFormat('en-US', {
  style: 'unit',
  unit: 'kilometer-per-hour'
}).format(120);  // "120 km/h"

// Compact notation
new Intl.NumberFormat('en-US', {
  notation: 'compact',
  compactDisplay: 'short'
}).format(1500000);  // "1.5M"

// Significant digits
new Intl.NumberFormat('en-US', {
  maximumSignificantDigits: 3
}).format(12345);  // "12,300"
```

### Intl.RelativeTimeFormat

```js
const rtf = new Intl.RelativeTimeFormat('en', { numeric: 'auto' });

rtf.format(-1, 'day');     // "yesterday"
rtf.format(-2, 'day');     // "2 days ago"
rtf.format(1, 'day');      // "tomorrow"
rtf.format(0, 'day');      // "today"
rtf.format(-1, 'week');    // "last week"
rtf.format(3, 'month');    // "in 3 months"

// numeric: 'always' forces numbers
const rtfAlways = new Intl.RelativeTimeFormat('en', { numeric: 'always' });
rtfAlways.format(-1, 'day');  // "1 day ago" (not "yesterday")
```

**Helper function to auto-select the right unit:**

```js
function timeAgo(date) {
  const rtf = new Intl.RelativeTimeFormat('en', { numeric: 'auto' });
  const now = Date.now();
  const diffMs = date.getTime() - now;
  const diffSec = Math.round(diffMs / 1000);
  const diffMin = Math.round(diffSec / 60);
  const diffHr = Math.round(diffMin / 60);
  const diffDay = Math.round(diffHr / 24);
  const diffMonth = Math.round(diffDay / 30);
  const diffYear = Math.round(diffDay / 365);

  if (Math.abs(diffSec) < 60) return rtf.format(diffSec, 'second');
  if (Math.abs(diffMin) < 60) return rtf.format(diffMin, 'minute');
  if (Math.abs(diffHr) < 24) return rtf.format(diffHr, 'hour');
  if (Math.abs(diffDay) < 30) return rtf.format(diffDay, 'day');
  if (Math.abs(diffMonth) < 12) return rtf.format(diffMonth, 'month');
  return rtf.format(diffYear, 'year');
}

console.log(timeAgo(new Date(Date.now() - 3600000))); // "1 hour ago"
```

### Intl.ListFormat

```js
const fruits = ['apple', 'banana', 'cherry'];

// Conjunction (and)
new Intl.ListFormat('en', { style: 'long', type: 'conjunction' })
  .format(fruits);  // "apple, banana, and cherry"

// Disjunction (or)
new Intl.ListFormat('en', { style: 'long', type: 'disjunction' })
  .format(fruits);  // "apple, banana, or cherry"

// Unit (no connectors)
new Intl.ListFormat('en', { style: 'narrow', type: 'unit' })
  .format(['6 ft', '2 in']);  // "6 ft 2 in"

// Other locales
new Intl.ListFormat('de', { type: 'conjunction' })
  .format(fruits);  // "apple, banana und cherry"
```

### Intl.PluralRules

```js
const pr = new Intl.PluralRules('en-US');

pr.select(0);   // "other"
pr.select(1);   // "one"
pr.select(2);   // "other"
pr.select(100); // "other"

// Practical: pluralization helper
function pluralize(count, singular, plural) {
  const rule = new Intl.PluralRules('en-US').select(count);
  return rule === 'one' ? singular : plural;
}

console.log(`3 ${pluralize(3, 'item', 'items')}`); // "3 items"
console.log(`1 ${pluralize(1, 'item', 'items')}`); // "1 item"

// Ordinal numbers (1st, 2nd, 3rd, 4th...)
const ordinalRules = new Intl.PluralRules('en-US', { type: 'ordinal' });
const suffixes = { one: 'st', two: 'nd', few: 'rd', other: 'th' };

function ordinal(n) {
  return `${n}${suffixes[ordinalRules.select(n)]}`;
}

console.log(ordinal(1));   // "1st"
console.log(ordinal(2));   // "2nd"
console.log(ordinal(3));   // "3rd"
console.log(ordinal(4));   // "4th"
console.log(ordinal(21));  // "21st"
console.log(ordinal(42));  // "42nd"
console.log(ordinal(103)); // "103rd"
```

### Intl.Segmenter

Locale-aware text segmentation by grapheme, word, or sentence.

```js
// Word segmentation
const segmenter = new Intl.Segmenter('en', { granularity: 'word' });
const segments = [...segmenter.segment('Hello, World!')];
// [{segment: "Hello", isWordLike: true, ...},
//  {segment: ",", isWordLike: false, ...},
//  {segment: " ", isWordLike: false, ...},
//  {segment: "World", isWordLike: true, ...},
//  {segment: "!", isWordLike: false, ...}]

// Count actual words
const wordCount = segments.filter((s) => s.isWordLike).length;
console.log(wordCount); // 2

// Grapheme segmentation (handles emoji, combining characters)
const graphemeSegmenter = new Intl.Segmenter('en', { granularity: 'grapheme' });
const familyEmoji = '\u{1F468}\u200D\u{1F469}\u200D\u{1F467}\u200D\u{1F466}';
const graphemes = [...graphemeSegmenter.segment(familyEmoji)];
console.log(graphemes.length); // 1 (one visible character)
console.log(familyEmoji.length); // 11 (many code units in the string)
```

**Why `Intl.Segmenter` matters:** `String.prototype.split(' ')` fails for CJK languages (Chinese, Japanese, Korean) that do not use spaces between words. `Intl.Segmenter` handles these correctly.

## Common Mistakes with Web APIs

### 1. Not Checking response.ok with Fetch

```js
// BAD: Assumes any resolved fetch is successful
const data = await fetch('/api/data').then(r => r.json());

// GOOD: Check status explicitly
const response = await fetch('/api/data');
if (!response.ok) throw new Error(`HTTP ${response.status}`);
const data = await response.json();
```

### 2. Missing Error Callback in Geolocation

```js
// BAD: No error handler; fails silently if permission denied
navigator.geolocation.getCurrentPosition((pos) => {
  console.log(pos.coords);
});

// GOOD: Always provide error callback
navigator.geolocation.getCurrentPosition(
  (pos) => console.log(pos.coords),
  (err) => console.error('Geolocation error:', err.message)
);
```

### 3. Forgetting to Revoke Object URLs

```js
// BAD: Memory leak; blob stays in memory forever
img.src = URL.createObjectURL(blob);

// GOOD: Revoke after the resource is loaded
img.src = URL.createObjectURL(blob);
img.onload = () => URL.revokeObjectURL(img.src);
```

### 4. Not Disconnecting Observers

```js
// BAD: Observers keep watching removed elements
const observer = new IntersectionObserver(callback);
observer.observe(element);
element.remove(); // observer still holds a reference

// GOOD: Clean up when the element is removed
observer.unobserve(element);
element.remove();
// Or disconnect all at once
observer.disconnect();
```

### 5. Storing Objects Directly in localStorage

```js
// BAD: Stores "[object Object]"
localStorage.setItem('user', { name: 'Alice' });

// GOOD: Serialize first
localStorage.setItem('user', JSON.stringify({ name: 'Alice' }));
```

### 6. Using unload/beforeunload for Analytics

```js
// BAD: Unreliable, especially on mobile; breaks bfcache
window.addEventListener('unload', () => {
  fetch('/analytics', { method: 'POST', body: data, keepalive: true });
});

// GOOD: Use visibilitychange + sendBeacon
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    navigator.sendBeacon('/analytics', data);
  }
});
```

### 7. Setting Content-Type for FormData Uploads

```js
// BAD: Overrides the multipart boundary
fetch('/upload', {
  method: 'POST',
  headers: { 'Content-Type': 'multipart/form-data' }, // WRONG
  body: formData
});

// GOOD: Let the browser set Content-Type with boundary
fetch('/upload', {
  method: 'POST',
  body: formData
  // No Content-Type header!
});
```

### 8. Consuming Response Body Twice

```js
// BAD: Throws "body already consumed" error
const response = await fetch('/api/data');
const text = await response.text();
const json = await response.json(); // TypeError!

// GOOD: Clone the response if you need to read it multiple ways
const response = await fetch('/api/data');
const clone = response.clone();
const text = await response.text();
const json = await clone.json();
```

### 9. Using structuredClone on Functions

```js
// BAD: Throws DataCloneError
const obj = {
  name: 'Alice',
  greet() { return `Hi, ${this.name}`; }
};
structuredClone(obj); // DataCloneError

// GOOD: Remove functions before cloning, or use a custom approach
const { greet, ...data } = obj;
const clone = structuredClone(data);
```

### 10. Infinite Loops with ResizeObserver

```js
// BAD: Callback resizes the element it observes
const observer = new ResizeObserver((entries) => {
  entries[0].target.style.width = entries[0].contentRect.width + 10 + 'px';
  // This triggers another resize, which triggers the callback again...
});

// GOOD: Use conditional logic to prevent re-triggering
const observer = new ResizeObserver((entries) => {
  const width = entries[0].contentRect.width;
  if (width < 500) {
    entries[0].target.classList.add('compact');
  }
  // Class change may or may not resize the element, but it won't loop infinitely
  // because the class is idempotent
});
```

## Best Practices

### 1. Always Wrap Storage Access in Try/Catch

Private browsing, disabled cookies, or full storage can all throw.

```js
function safeGetItem(key) {
  try {
    return localStorage.getItem(key);
  } catch {
    return null;
  }
}
```

### 2. Use AbortController Consistently

Every fetch in production should have a timeout or cancellation mechanism.

```js
// In a component or page lifecycle
const controller = new AbortController();

async function loadData() {
  const data = await apiFetch('/api/data', { signal: controller.signal });
  render(data);
}

// When the component unmounts or user navigates away
function cleanup() {
  controller.abort();
}
```

### 3. Prefer Observers Over Scroll/Resize Listeners

```js
// BAD: Scroll listener fires on every pixel
window.addEventListener('scroll', () => {
  checkElementVisibility(); // runs 60+ times per second while scrolling
});

// GOOD: IntersectionObserver fires only at thresholds
const observer = new IntersectionObserver(callback, { threshold: 0.5 });
observer.observe(element);
```

### 4. Reuse Intl Formatter Instances

Constructing `Intl.*` objects is expensive. Create them once and reuse.

```js
// BAD: Creates a new formatter on every call
function formatPrice(amount) {
  return new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' })
    .format(amount);
}

// GOOD: Create once, reuse many times
const priceFormatter = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD'
});

function formatPrice(amount) {
  return priceFormatter.format(amount);
}
```

### 5. Feature-Detect Before Using APIs

```js
// Check for API availability
if ('IntersectionObserver' in window) {
  // Use IntersectionObserver
} else {
  // Fallback: eagerly load all images
}

if ('structuredClone' in globalThis) {
  // Use structuredClone
} else {
  // Fallback: JSON round-trip
}

if ('navigation' in window) {
  // Use Navigation API
} else {
  // Fall back to History API
}
```

### 6. Clean Up After Yourself

Every observer, channel, and timer needs cleanup.

```js
// Pattern: return a cleanup function
function observeVisibility(element, callback) {
  const observer = new IntersectionObserver((entries) => {
    if (entries[0].isIntersecting) callback();
  });
  observer.observe(element);

  // Return cleanup function
  return () => observer.disconnect();
}

const cleanup = observeVisibility(element, handleVisible);
// Later...
cleanup();
```

### 7. Use sendBeacon for Analytics, Not fetch

```js
// Analytics data that must survive page navigation
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    const blob = new Blob(
      [JSON.stringify(analyticsQueue)],
      { type: 'application/json' }
    );
    navigator.sendBeacon('/api/analytics/batch', blob);
    analyticsQueue = [];
  }
});
```

### 8. Batch Performance Measurements

```js
// Measure an operation end-to-end
function measureAsync(name, fn) {
  return async (...args) => {
    performance.mark(`${name}-start`);
    try {
      return await fn(...args);
    } finally {
      performance.mark(`${name}-end`);
      performance.measure(name, `${name}-start`, `${name}-end`);
    }
  };
}

// Usage
const fetchUsers = measureAsync('fetchUsers', async () => {
  const response = await fetch('/api/users');
  return response.json();
});

await fetchUsers();

// Report all measurements
const entries = performance.getEntriesByType('measure');
entries.forEach((e) => console.log(`${e.name}: ${e.duration.toFixed(2)}ms`));
```

## Summary

The browser's built-in Web APIs handle the vast majority of what used to require third-party libraries. `fetch` replaces Axios for most use cases (with proper error handling). `structuredClone` replaces lodash's `cloneDeep`. `Intl.*` formatters replace moment.js and numeral.js. `IntersectionObserver` and `ResizeObserver` replace scroll and resize event listeners. `BroadcastChannel` replaces custom cross-tab messaging hacks. The critical skill is knowing the gotchas: fetch not rejecting on HTTP errors, localStorage being synchronous and string-only, observers needing cleanup, sendBeacon needing `visibilitychange` instead of `unload`, and structuredClone stripping prototypes and choking on functions. Master these APIs and their edge cases, and you eliminate entire categories of bugs from your production code.
