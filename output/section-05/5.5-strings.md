# 5.5 Strings

> JavaScript strings are UTF-16 encoded sequences of code units, not characters. Understanding this distinction is the foundation for correctly handling emoji, internationalization, comparison, and every string operation beyond ASCII.

---

## UTF-16 Internals

JavaScript strings are internally stored as sequences of 16-bit code units (UTF-16). Characters in the Basic Multilingual Plane (U+0000 to U+FFFF) use one code unit. Characters outside the BMP (U+10000 to U+10FFFF) use two code units called a **surrogate pair**: a high surrogate (U+D800 to U+DBFF) followed by a low surrogate (U+DC00 to U+DFFF).

### `.length` Counts Code Units, Not Characters

```javascript
"hello".length       // 5 (one code unit per character)
"cafe\u0301".length  // 5 (e + combining acute accent = 2 code units)
"caf\u00e9".length   // 4 (precomposed e-acute = 1 code unit)
"😂".length          // 2 (one surrogate pair)
"🇺🇸".length          // 4 (two regional indicators, each a surrogate pair)
"👨‍👩‍👧‍👦".length       // 11 (4 people + 3 ZWJ characters, all surrogate pairs)
"👋🏽".length          // 4 (wave + skin tone modifier, each a surrogate pair)
```

### Indexing Breaks Surrogate Pairs

```javascript
"😂"[0]           // "\uD83D" (lone high surrogate, not a valid character)
"😂"[1]           // "\uDE02" (lone low surrogate)
"😂".charAt(0)    // "\uD83D" (same problem)
"😂".charCodeAt(0) // 55357 (high surrogate code unit)

// Safe alternative: codePointAt returns the full code point
"😂".codePointAt(0) // 128514 (the actual Unicode code point)

// Creating strings from code points
String.fromCodePoint(128514)  // "😂"
String.fromCharCode(55357, 56834) // "😂" (requires both surrogates)
```

### Safe Iteration

`for...of` iterates by code point, correctly handling surrogate pairs (but not ZWJ sequences):

```javascript
// BAD: Classic for loop splits surrogates
for (let i = 0; i < "Hello 😂".length; i++) {
  // "😂" is split into two broken halves
}

// GOOD: for...of iterates by code point
for (const char of "Hello 😂") {
  console.log(char); // H, e, l, l, o, " ", 😂
}

// Spread also works by code point
[..."Hello 😂"]   // ["H", "e", "l", "l", "o", " ", "😂"]
Array.from("😂")   // ["😂"] (length 1)

// But ZWJ sequences are still split into individual code points
[..."👨‍👩‍👧‍👦"]  // ["👨", "‍", "👩", "‍", "👧", "‍", "👦"] (7 elements)
```

### `Intl.Segmenter` for Grapheme Clusters

For true user-perceived character counting, use `Intl.Segmenter`:

```javascript
const segmenter = new Intl.Segmenter("en", { granularity: "grapheme" });

function graphemeCount(str) {
  return [...segmenter.segment(str)].length;
}

graphemeCount("Hello")     // 5
graphemeCount("cafe\u0301") // 4 (e + accent = 1 grapheme)
graphemeCount("😂")         // 1
graphemeCount("👨‍👩‍👧‍👦")      // 1
graphemeCount("🇺🇸")         // 1
graphemeCount("👋🏽")         // 1
```

Word and sentence segmentation:

```javascript
// Word segmentation (works for languages without spaces)
const wordSeg = new Intl.Segmenter("ja", { granularity: "word" });
const words = [...wordSeg.segment("今日は天気がいい")]
  .filter(s => s.isWordLike)
  .map(s => s.segment);
// ["今日", "は", "天気", "が", "いい"]
```

### Well-Formed Strings (ES2024)

Lone surrogates (a high without a low, or vice versa) are invalid UTF-16. ES2024 added methods to detect and fix them:

```javascript
"hello".isWellFormed()          // true
"ab\uD800".isWellFormed()       // false (lone high surrogate)
"ab\uD800".toWellFormed()       // "ab\uFFFD" (replaced with U+FFFD)
"\uD800\uD800".toWellFormed()   // "\uFFFD\uFFFD"
```

## Template Literals

### Interpolation and Multi-Line

```javascript
const name = "Alice";
const greeting = `Hello, ${name}!`; // "Hello, Alice!"

// Multi-line (preserves newlines and indentation)
const html = `
  <div>
    <h1>${title}</h1>
    <p>${body}</p>
  </div>
`;

// Expressions in interpolation
`Total: ${(price * quantity).toFixed(2)}`;
`Status: ${isActive ? "Active" : "Inactive"}`;
```

### Tagged Templates

A tag function receives string fragments and interpolated values separately, enabling custom processing:

```javascript
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const value = i < values.length ? `<mark>${values[i]}</mark>` : "";
    return result + str + value;
  }, "");
}

const term = "JavaScript";
highlight`Learn ${term} today`; // "Learn <mark>JavaScript</mark> today"
```

### HTML Sanitization Tag

```javascript
function html(strings, ...values) {
  const escape = (str) =>
    String(str)
      .replace(/&/g, "&amp;")
      .replace(/</g, "&lt;")
      .replace(/>/g, "&gt;")
      .replace(/"/g, "&quot;")
      .replace(/'/g, "&#039;");

  return strings.reduce((result, str, i) => {
    return result + str + (i < values.length ? escape(values[i]) : "");
  }, "");
}

const userInput = '<script>alert("xss")</script>';
html`<div>${userInput}</div>`;
// "<div>&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;</div>"
```

### SQL Parameterization Tag

```javascript
function sql(strings, ...values) {
  const text = strings.reduce((query, str, i) => {
    return query + str + (i < values.length ? `$${i + 1}` : "");
  }, "");
  return { text, params: values };
}

const id = 42;
const status = "active";
sql`SELECT * FROM users WHERE id = ${id} AND status = ${status}`;
// { text: "SELECT * FROM users WHERE id = $1 AND status = $2", params: [42, "active"] }
```

### `String.raw`

The only built-in tag function. Returns the raw string without processing escape sequences. Interpolations are still evaluated.

```javascript
String.raw`\n\t\\`              // "\\n\\t\\\\" (4 visible characters: \n\t\\)
String.raw`C:\Users\Admin\file` // "C:\\Users\\Admin\\file" (literal backslashes)

// Regex patterns without double-escaping
const pattern = new RegExp(String.raw`^\d{3}\.\d{3}$`);
// Without String.raw: new RegExp("^\\d{3}\\.\\d{3}$")

// Callable as a function
String.raw({ raw: ["SELECT ", " FROM users"] }, "*"); // "SELECT * FROM users"
```

## String Search Methods

### `includes`, `startsWith`, `endsWith`

```javascript
const str = "Hello, World!";

str.includes("World")      // true
str.includes("world")      // false (case-sensitive)
str.includes("Hello", 1)   // false (starts searching from index 1)

str.startsWith("Hello")    // true
str.startsWith("World", 7) // true (check from position 7)

str.endsWith("!")          // true
str.endsWith("World", 12) // true (treat string as if only 12 chars long)
```

All three throw `TypeError` if passed a RegExp. Use `search()` or `match()` for regex patterns.

### `indexOf` and `lastIndexOf`

```javascript
const str = "banana";

str.indexOf("an")      // 1 (first occurrence)
str.indexOf("an", 2)   // 3 (search from index 2)
str.indexOf("x")       // -1 (not found)

str.lastIndexOf("an")  // 3 (last occurrence, searching backward)
```

**Classic `indexOf` bug:**

```javascript
const path = "/api/users";

// BAD: indexOf returns 0 for "/api", which is falsy
if (path.indexOf("/api")) {
  // Never runs when match is at index 0!
}

// GOOD
if (path.indexOf("/api") !== -1) { /* ... */ }

// BETTER
if (path.includes("/api")) { /* ... */ }

// Or for prefix checking
if (path.startsWith("/api")) { /* ... */ }
```

### `search`

Returns the index of the first regex match, or -1. Always starts from the beginning.

```javascript
"Hello World".search(/world/i) // 6 (case-insensitive)
"Hello World".search(/xyz/)    // -1
```

## String Extraction: `slice`, `substring`, `at`

### `slice` (Preferred)

```javascript
const str = "Hello, World!";

str.slice(7)       // "World!"
str.slice(7, 12)   // "World"
str.slice(-6)      // "orld!" (negative counts from end)
str.slice(-6, -1)  // "orld"
str.slice(7, 2)    // "" (start > end returns empty)
```

### `substring` (Less Predictable)

```javascript
const str = "Hello, World!";

str.substring(7)     // "World!"
str.substring(7, 12) // "World"
str.substring(-6)    // "Hello, World!" (negative treated as 0)
str.substring(7, 2)  // "lo, W" (swaps arguments when start > end!)
```

### `at` (ES2022)

Supports negative indexing. Returns `undefined` when out of bounds.

```javascript
const str = "Hello";

str.at(0)   // "H"
str.at(-1)  // "o" (last character)
str.at(-2)  // "l"
str.at(99)  // undefined

// Still operates on code units, not grapheme clusters
"😂".at(0)   // "\uD83D" (high surrogate, not the full emoji)
```

### Comparison Table

| Behavior | `slice(start, end)` | `substring(start, end)` |
|---|---|---|
| Negative args | Counts from end | Treats as 0 |
| start > end | Returns `""` | Swaps arguments |
| Deprecated | No | No |

**Use `slice()` as the default.** Its behavior with negative indices is intuitive and it never silently swaps arguments.

`substr(start, length)` is deprecated (Annex B) and should not be used.

## String Transformation

### Case Conversion

```javascript
"Hello".toUpperCase()  // "HELLO"
"Hello".toLowerCase()  // "hello"

// Locale-aware case conversion
"istanbul".toLocaleUpperCase("tr") // "ISTANBUL" (Turkish I with dot)
"istanbul".toUpperCase()           // "ISTANBUL" (wrong for Turkish)
```

### `replace` and `replaceAll`

```javascript
// replace: only first occurrence with string pattern
"aaa".replace("a", "b")     // "baa" (not "bbb"!)

// replaceAll: all occurrences (ES2021)
"aaa".replaceAll("a", "b")  // "bbb"

// replace with regex /g flag: all occurrences
"aaa".replace(/a/g, "b")    // "bbb"

// replaceAll with regex REQUIRES /g flag
// "aaa".replaceAll(/a/, "b") // TypeError!
"aaa".replaceAll(/a/g, "b")  // "bbb"
```

**Special replacement patterns:**

```javascript
"John Smith".replace(/(\w+) (\w+)/, "$2, $1")  // "Smith, John"

// Named groups
"2025-01-15".replace(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
  "$<day>/$<month>/$<year>"
); // "15/01/2025"

// Replacer function
"hello world".replace(/\b\w/g, (match) => match.toUpperCase());
// "Hello World"
```

| Pattern | Meaning |
|---|---|
| `$$` | Literal `$` |
| `$&` | The matched substring |
| `` $` `` | Portion before the match |
| `$'` | Portion after the match |
| `$1`, `$2` | Capturing group N |
| `$<name>` | Named capturing group |

### `match` and `matchAll`

```javascript
// match without /g: returns first match with groups
"price: $42.99".match(/\$(\d+\.\d+)/);
// ["$42.99", "42.99", index: 7, groups: undefined]

// match with /g: returns all matches, no group info
"$10 and $20".match(/\$\d+/g);
// ["$10", "$20"]

// matchAll: returns iterator with full group info for each match
const matches = "2025-01-15 and 2025-02-27".matchAll(
  /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/g
);
for (const m of matches) {
  console.log(m.groups.year, m.groups.month, m.groups.day);
}
// "2025" "01" "15"
// "2025" "02" "27"

// matchAll REQUIRES /g flag
// str.matchAll(/pattern/) // TypeError
```

### `RegExp.escape` (ES2025)

Safely escapes a string for use inside a regex:

```javascript
const userInput = "price is $10.00 (USD)";
const escaped = RegExp.escape(userInput);
// "price\\ is\\ \\$10\\.00\\ \\(USD\\)"

const regex = new RegExp(escaped);
```

## Trimming and Padding

### Trim Methods

```javascript
"  hello  ".trim()      // "hello"
"  hello  ".trimStart() // "hello  "
"  hello  ".trimEnd()   // "  hello"

// Removes ALL Unicode whitespace, not just ASCII space
"\u00A0\thello\u3000".trim() // "hello"
"\uFEFFhello".trim()         // "hello" (BOM removed)
```

### Pad Methods

```javascript
"42".padStart(5, "0")    // "00042"
"42".padEnd(5, ".")      // "42..."
"42".padStart(5)         // "   42" (default pad is space)

// padString is truncated if too long
"1".padStart(4, "abc")   // "abc1"
"12".padStart(4, "abc")  // "ab12"

// No truncation if already long enough
"hello".padStart(3, "0") // "hello" (unchanged)
```

**Practical uses:**

```javascript
// Format time
const h = String(hours).padStart(2, "0");
const m = String(minutes).padStart(2, "0");
const s = String(seconds).padStart(2, "0");
const time = `${h}:${m}:${s}`; // "09:05:03"

// Align table output
const items = [["Name", "Price"], ["Widget", "$9.99"], ["Gadget", "$149.00"]];
for (const [name, price] of items) {
  console.log(name.padEnd(10) + price.padStart(10));
}
// Name          $9.99
// Widget        $9.99
// Gadget      $149.00
```

## String Splitting

### `split`

```javascript
"a,b,c".split(",")     // ["a", "b", "c"]
"a,b,c".split(",", 2)  // ["a", "b"] (limit parameter)
"hello".split("")       // ["h", "e", "l", "l", "o"] (splits by code unit)

// Split by regex
"one1two2three".split(/\d/)  // ["one", "two", "three"]

// Preserve separators with capturing groups
"one1two2three".split(/(\d)/) // ["one", "1", "two", "2", "three"]

// Edge case: empty string
"".split(",")   // [""] (array with one empty string)
"".split("")    // [] (empty array)
```

**Splitting emoji safely:**

```javascript
// BAD: split("") breaks surrogate pairs
"Hi 😂".split("")  // ["H", "i", " ", "\uD83D", "\uDE02"] (broken emoji)

// GOOD: spread operator or Array.from
[..."Hi 😂"]       // ["H", "i", " ", "😂"]
```

## Unicode Normalization

Two strings that look identical may not be equal if they use different Unicode representations:

```javascript
const a = "caf\u00e9";      // precomposed e-acute (1 code unit)
const b = "cafe\u0301";     // e + combining acute accent (2 code units)

a === b                      // false (different code units!)
a.length                     // 4
b.length                     // 5

// Normalize to the same form before comparing
a.normalize("NFC") === b.normalize("NFC") // true
```

### Normalization Forms

| Form | Name | Effect |
|---|---|---|
| NFC | Canonical Composition | Composes to precomposed form (default, most common) |
| NFD | Canonical Decomposition | Decomposes to base + combining marks |
| NFKC | Compatibility Composition | Composes, maps compatibility characters (lossy) |
| NFKD | Compatibility Decomposition | Decomposes compatibility characters (lossy) |

```javascript
// NFKC/NFKD are lossy: they normalize visual variants
"\ufb01".normalize("NFKC") // "fi" (ligature decomposed to two chars)
"\u2126".normalize("NFKC") // "Ω" (ohm sign mapped to Greek omega)
```

## String Comparison and Sorting

### Default Comparison is by Code Unit

```javascript
"a" < "b"     // true
"Z" < "a"     // true (uppercase ASCII before lowercase)
"10" < "9"    // true (lexicographic: "1" < "9")
```

### `localeCompare`

Returns a negative, zero, or positive number indicating sort order:

```javascript
"a".localeCompare("b")      // negative
"b".localeCompare("a")      // positive
"a".localeCompare("a")      // 0

// Case-insensitive
"a".localeCompare("A", undefined, { sensitivity: "base" }) // 0

// Natural numeric sort
"file2".localeCompare("file10", undefined, { numeric: true }) // negative
```

### `Intl.Collator` (Performance)

Create the collator once and reuse its `.compare` method. 60-80% faster than `localeCompare` for sorting.

```javascript
const collator = new Intl.Collator("en", { numeric: true, sensitivity: "base" });

const files = ["file10", "file2", "file1", "file20"];
files.sort(collator.compare);
// ["file1", "file2", "file10", "file20"]
```

**Collator options:**

| Option | Values | Effect |
|---|---|---|
| `sensitivity` | `"base"`, `"accent"`, `"case"`, `"variant"` | Which differences matter |
| `numeric` | `true`/`false` | Natural number sorting |
| `caseFirst` | `"upper"`, `"lower"`, `"false"` | Case sort priority |
| `ignorePunctuation` | `true`/`false` | Skip punctuation |

```javascript
// "base": a = A = a (ignore case and accents)
// "accent": a != a, a = A (ignore case, respect accents)
// "case": a = a, a != A (ignore accents, respect case)
// "variant": a != a != A (respect everything)
```

## String Encoding and Decoding

### `TextEncoder` / `TextDecoder`

```javascript
// Encode string to UTF-8 bytes
const encoder = new TextEncoder();
const bytes = encoder.encode("Hello 😂");
// Uint8Array [72, 101, 108, 108, 111, 32, 240, 159, 152, 130]
// "😂" = 4 bytes in UTF-8

// Decode bytes back to string
const decoder = new TextDecoder("utf-8");
decoder.decode(bytes); // "Hello 😂"

// Decode with error handling
const strictDecoder = new TextDecoder("utf-8", { fatal: true });
// strictDecoder.decode(invalidBytes) // TypeError on invalid sequences
```

**`encodeInto` for performance:**

```javascript
const encoder = new TextEncoder();
const buffer = new Uint8Array(1024);

const { read, written } = encoder.encodeInto("Hello World", buffer);
// read: 11 (code units consumed), written: 11 (bytes written)
```

### Base64

```javascript
// Browser
btoa("Hello")       // "SGVsbG8=" (only ASCII)
atob("SGVsbG8=")    // "Hello"

// Unicode-safe base64
function toBase64(str) {
  return btoa(String.fromCodePoint(...new TextEncoder().encode(str)));
}
function fromBase64(b64) {
  return new TextDecoder().decode(
    Uint8Array.from(atob(b64), c => c.codePointAt(0))
  );
}

toBase64("Hello 😂")  // "SGVsbG8g8J+Ygg=="
fromBase64("SGVsbG8g8J+Ygg==") // "Hello 😂"
```

## Regex Flags for Strings

### `u` Flag (Unicode Mode)

```javascript
// Without u: dot does not match astral characters
/^.$/.test("😂")    // false (sees two code units)

// With u: dot matches full code points
/^.$/u.test("😂")   // true

// Unicode property escapes (requires u or v)
/\p{Letter}/u.test("a")    // true
/\p{Letter}/u.test("あ")   // true
/\p{Emoji}/u.test("😂")    // true
```

### `v` Flag (Unicode Sets, ES2024)

Superset of `u`. Adds set operations and multi-code-point matching:

```javascript
// Set subtraction
/[\p{Decimal_Number}--[0-9]]/v  // Non-ASCII digits

// Set intersection
/[\p{Letter}&&\p{ASCII}]/v     // ASCII letters only

// Match full emoji (including ZWJ sequences)
/\p{RGI_Emoji}/v.test("👨‍👩‍👧‍👦")  // true
```

## String Concatenation Performance

### Modern Benchmarks

| Method | Performance |
|---|---|
| `+` / `+=` | Fastest (V8 uses rope/cons-string optimization) |
| Template literals | Same as `+` |
| `Array.join()` | Slightly slower for small strings, good for large arrays |
| `String.concat()` | Consistently slowest |

```javascript
// Modern engines optimize += with rope strings
let result = "";
for (let i = 0; i < 10000; i++) {
  result += `item ${i}, `; // V8 defers actual copying
}

// Array.join is best when you already have an array
const parts = items.map(item => `<li>${item.name}</li>`);
const html = parts.join("\n"); // Single allocation for the final string
```

The old advice "always use Array.join()" dates from IE-era engines. Modern V8 optimizes `+=` with internal rope structures that defer copying until the string is read.

## Common Mistakes

### 1. Using `replace` When You Mean `replaceAll`

```javascript
// BAD: Only replaces the first occurrence
"a.b.c".replace(".", "-") // "a-b.c"

// GOOD
"a.b.c".replaceAll(".", "-") // "a-b-c"
```

### 2. Treating `.length` as Character Count

```javascript
// BAD: Length check fails with emoji
function validateUsername(name) {
  return name.length <= 20; // "😂" counts as 2
}

// GOOD: Use Intl.Segmenter for user-facing limits
function validateUsername(name) {
  const segmenter = new Intl.Segmenter("en", { granularity: "grapheme" });
  return [...segmenter.segment(name)].length <= 20;
}
```

### 3. Comparing Strings Without Normalization

```javascript
// BAD: Visually identical strings fail ===
const input = "cafe\u0301";  // user typed
const stored = "caf\u00e9";  // from database
input === stored // false!

// GOOD
input.normalize("NFC") === stored.normalize("NFC") // true
```

### 4. Using `indexOf` in Boolean Context

```javascript
// BAD: Fails when match is at index 0
if (path.indexOf("/api")) { /* ... */ }

// GOOD
if (path.includes("/api")) { /* ... */ }
```

### 5. Forgetting `String()` vs `toString()` for Null/Undefined

```javascript
// BAD: Throws on null/undefined
// null.toString()      // TypeError
// undefined.toString() // TypeError

// GOOD: String() handles all types
String(null)      // "null"
String(undefined) // "undefined"
String(42)        // "42"
String(Symbol("x")) // "Symbol(x)"
```

### 6. Splitting Emoji with `split("")`

```javascript
// BAD: Breaks surrogate pairs
"Hi 😂".split("") // ["H", "i", " ", "\uD83D", "\uDE02"]

// GOOD: Spread operator
[..."Hi 😂"]       // ["H", "i", " ", "😂"]
```

## Best Practices

1. **Use `slice()` over `substring()`.** Its negative-index behavior is predictable and it never swaps arguments.

2. **Use `includes()` instead of `indexOf() !== -1`** for readability when you only need a boolean check.

3. **Use `replaceAll()` for global string replacement** instead of `replace()` with regex `/g`.

4. **Normalize strings with `.normalize("NFC")` before comparison** when dealing with user input or data from different sources.

5. **Use `Intl.Segmenter` for character counting** in user-facing features (character limits, truncation).

6. **Use `Intl.Collator` for sorting** instead of calling `localeCompare()` repeatedly. Create the collator once.

7. **Use `String.raw` for regex patterns and file paths** to avoid double-escaping backslashes.

8. **Use `for...of` or spread (`[...str]`) instead of `split("")`** to safely iterate strings containing non-BMP characters.

## Real-World Use Cases

### Input Sanitization Pipeline

```javascript
function sanitizeInput(raw) {
  return raw
    .normalize("NFC")                // Canonical Unicode form
    .trim()                          // Remove leading/trailing whitespace
    .replace(/\s+/g, " ")           // Collapse multiple spaces
    .replace(/[<>&"']/g, (ch) => {  // Escape HTML entities
      const entities = { "<": "&lt;", ">": "&gt;", "&": "&amp;", '"': "&quot;", "'": "&#039;" };
      return entities[ch];
    });
}

sanitizeInput("  cafe\u0301  <script>  ") // "caf\u00e9 &lt;script&gt;"
```

### Locale-Aware Search and Sort

```javascript
class SearchIndex {
  #collator;
  #items;

  constructor(locale = "en") {
    this.#collator = new Intl.Collator(locale, {
      sensitivity: "base",
      usage: "search"
    });
    this.#items = [];
  }

  add(item) {
    this.#items.push(item);
  }

  search(query) {
    return this.#items.filter(item =>
      this.#contains(item.name, query)
    );
  }

  #contains(haystack, needle) {
    const len = needle.length;
    for (let i = 0; i <= haystack.length - len; i++) {
      if (this.#collator.compare(haystack.slice(i, i + len), needle) === 0) {
        return true;
      }
    }
    return false;
  }

  sort() {
    const sortCollator = new Intl.Collator(undefined, { numeric: true });
    this.#items.sort((a, b) => sortCollator.compare(a.name, b.name));
  }
}

const index = new SearchIndex("fr");
index.add({ name: "Cafe" });
index.add({ name: "Caf\u00e9" });
index.search("cafe"); // Both match (base sensitivity ignores accents)
```

### Truncation Respecting Grapheme Clusters

```javascript
function truncate(str, maxGraphemes, suffix = "...") {
  const segmenter = new Intl.Segmenter("en", { granularity: "grapheme" });
  const segments = [...segmenter.segment(str)];

  if (segments.length <= maxGraphemes) return str;

  return segments
    .slice(0, maxGraphemes)
    .map(s => s.segment)
    .join("") + suffix;
}

truncate("Hello World", 5)          // "Hello..."
truncate("Hello 😂👋🏽🇺🇸", 8)        // "Hello 😂👋🏽..."
truncate("Short", 10)               // "Short"
```

---

> **Key Takeaway:** JavaScript strings are UTF-16 code unit sequences, not character sequences. Use `Intl.Segmenter` for true character counting, `.normalize("NFC")` before comparison, `Intl.Collator` for sorting, `for...of` for safe iteration, and always prefer `slice()`, `includes()`, and `replaceAll()` over their older counterparts.
