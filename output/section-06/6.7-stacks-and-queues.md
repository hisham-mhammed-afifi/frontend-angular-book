# 6.7 Stacks and Queues

> Stacks and queues are abstract data types that control the order elements are added and removed. Stacks process the most recent item first (LIFO), queues process the oldest item first (FIFO). They underpin everything from the JavaScript call stack to task scheduling and undo systems.

---

## Stack (LIFO: Last In, First Out)

A stack has two core operations:

- **push:** Add an element to the top
- **pop:** Remove and return the element from the top

Think of a stack of plates: you add and remove from the top only.

### Array-Based Stack

JavaScript arrays have built-in `push` and `pop` methods, making them natural stacks with O(1) operations.

```js
class Stack {
  constructor() {
    this.items = [];
  }

  push(value) {
    this.items.push(value);
  }

  pop() {
    if (this.isEmpty()) return undefined;
    return this.items.pop();
  }

  peek() {
    if (this.isEmpty()) return undefined;
    return this.items[this.items.length - 1];
  }

  isEmpty() {
    return this.items.length === 0;
  }

  get size() {
    return this.items.length;
  }

  clear() {
    this.items = [];
  }
}

const stack = new Stack();
stack.push(10);
stack.push(20);
stack.push(30);
stack.peek();  // 30 (top element, not removed)
stack.pop();   // 30 (removed)
stack.pop();   // 20
stack.size;    // 1
```

For most use cases, you do not even need a class. A plain array with `push` and `pop` is sufficient:

```js
const stack = [];
stack.push("a");
stack.push("b");
stack.push("c");
stack.pop(); // "c"
stack.pop(); // "b"
```

### Stack Operations Complexity

| Operation | Time | Notes |
|-----------|------|-------|
| push | O(1) | Amortized (array may resize) |
| pop | O(1) | |
| peek | O(1) | |
| isEmpty | O(1) | |
| search | O(n) | Must scan from top to bottom |

## Classic Stack Problems

### Balanced Parentheses

The most classic stack problem. For every opening bracket, there must be a matching closing bracket in the correct order.

```js
function isValid(str) {
  const stack = [];
  const pairs = { ")": "(", "]": "[", "}": "{" };

  for (const char of str) {
    if ("([{".includes(char)) {
      stack.push(char); // Opening bracket: push onto stack
    } else if (")]}".includes(char)) {
      if (stack.length === 0 || stack.pop() !== pairs[char]) {
        return false; // No matching opener or wrong type
      }
    }
  }

  return stack.length === 0; // All openers must be matched
}

// isValid("({[]})") => true
// isValid("([)]")   => false (wrong nesting order)
// isValid("((")      => false (unclosed)
// isValid("")        => true (empty is valid)
```

### Reverse a String Using a Stack

```js
function reverseString(str) {
  const stack = [];
  for (const char of str) stack.push(char);

  let reversed = "";
  while (stack.length > 0) {
    reversed += stack.pop();
  }
  return reversed;
}

// reverseString("hello") => "olleh"
```

### Evaluate Reverse Polish Notation (Postfix)

In postfix notation, operators come after their operands. A stack naturally evaluates this.

```js
function evalRPN(tokens) {
  const stack = [];

  for (const token of tokens) {
    if ("+-*/".includes(token)) {
      const b = stack.pop(); // Second operand (popped first)
      const a = stack.pop(); // First operand
      switch (token) {
        case "+": stack.push(a + b); break;
        case "-": stack.push(a - b); break;
        case "*": stack.push(a * b); break;
        case "/": stack.push(Math.trunc(a / b)); break; // Truncate toward zero
      }
    } else {
      stack.push(Number(token));
    }
  }

  return stack.pop();
}

// evalRPN(["2", "1", "+", "3", "*"]) => 9
// Evaluation: (2 + 1) * 3 = 9
//
// evalRPN(["4", "13", "5", "/", "+"]) => 6
// Evaluation: 4 + (13 / 5) = 4 + 2 = 6
```

### Min Stack

A stack that supports retrieving the minimum element in O(1) time.

```js
class MinStack {
  constructor() {
    this.stack = [];
    this.minStack = []; // Parallel stack tracking minimums
  }

  push(val) {
    this.stack.push(val);
    // Push to minStack if it's empty or val <= current min
    if (this.minStack.length === 0 || val <= this.getMin()) {
      this.minStack.push(val);
    }
  }

  pop() {
    const val = this.stack.pop();
    // If popped value is the current min, pop from minStack too
    if (val === this.getMin()) {
      this.minStack.pop();
    }
    return val;
  }

  peek() {
    return this.stack[this.stack.length - 1];
  }

  getMin() {
    return this.minStack[this.minStack.length - 1];
  }
}

const ms = new MinStack();
ms.push(5);
ms.push(2);
ms.push(7);
ms.getMin(); // 2
ms.pop();    // 7
ms.getMin(); // 2
ms.pop();    // 2
ms.getMin(); // 5
```

## Monotonic Stack

A monotonic stack maintains elements in sorted order (either strictly increasing or decreasing from bottom to top). It is the key tool for "next greater/smaller element" problems, reducing O(n^2) brute force to O(n).

### Next Greater Element

For each element, find the first element to its right that is larger.

```js
function nextGreaterElement(arr) {
  const result = new Array(arr.length).fill(-1); // -1 means no greater element
  const stack = []; // Stores indices, elements are in decreasing order

  for (let i = 0; i < arr.length; i++) {
    // While the current element is greater than the top of the stack,
    // it is the "next greater" for those stack elements
    while (stack.length > 0 && arr[i] > arr[stack[stack.length - 1]]) {
      const idx = stack.pop();
      result[idx] = arr[i];
    }
    stack.push(i);
  }

  return result;
}

// nextGreaterElement([4, 5, 2, 25, 7])
// => [5, 25, 25, -1, -1]
//
// For 4: next greater is 5
// For 5: next greater is 25
// For 2: next greater is 25
// For 25: no greater element to the right
// For 7: no greater element to the right
```

### Daily Temperatures

For each day, how many days until a warmer temperature?

```js
function dailyTemperatures(temperatures) {
  const result = new Array(temperatures.length).fill(0);
  const stack = []; // Stack of indices with decreasing temperatures

  for (let i = 0; i < temperatures.length; i++) {
    while (
      stack.length > 0 &&
      temperatures[i] > temperatures[stack[stack.length - 1]]
    ) {
      const prevIdx = stack.pop();
      result[prevIdx] = i - prevIdx; // Days until warmer
    }
    stack.push(i);
  }

  return result;
}

// dailyTemperatures([73, 74, 75, 71, 69, 72, 76, 73])
// => [1, 1, 4, 2, 1, 1, 0, 0]
//
// Day 0 (73): next warmer is day 1 (74), wait 1 day
// Day 2 (75): next warmer is day 6 (76), wait 4 days
```

### Largest Rectangle in Histogram

Find the area of the largest rectangle that fits within a histogram.

```js
function largestRectangleArea(heights) {
  const stack = []; // Indices with increasing heights
  let maxArea = 0;

  for (let i = 0; i <= heights.length; i++) {
    const currentHeight = i < heights.length ? heights[i] : 0;

    while (stack.length > 0 && currentHeight < heights[stack[stack.length - 1]]) {
      const height = heights[stack.pop()];
      const width = stack.length === 0 ? i : i - stack[stack.length - 1] - 1;
      maxArea = Math.max(maxArea, height * width);
    }

    stack.push(i);
  }

  return maxArea;
}

// largestRectangleArea([2, 1, 5, 6, 2, 3])
// => 10 (rectangle of height 5, width 2, from index 2-3)
```

## Queue (FIFO: First In, First Out)

A queue has two core operations:

- **enqueue:** Add an element to the back
- **dequeue:** Remove and return the element from the front

Think of a line at a checkout: the first person in line is served first.

### The Array.shift() Problem

The naive queue uses `push` to enqueue and `shift` to dequeue. But `shift` is O(n) because it reindexes every remaining element.

```js
// BAD for large queues: shift is O(n)
const queue = [];
queue.push(1); // enqueue
queue.push(2);
queue.push(3);
queue.shift();  // dequeue: returns 1, but reindexes [2, 3]

// For 100,000 elements, each shift moves 99,999 elements
// Dequeueing all elements: O(n^2) total
```

For small queues (hundreds of elements), this is fine. For large queues (thousands or more), use a proper implementation.

### Linked-List Based Queue

True O(1) enqueue and dequeue using a singly linked list.

```js
class QueueNode {
  constructor(value) {
    this.value = value;
    this.next = null;
  }
}

class Queue {
  constructor() {
    this.head = null; // Front of the queue (dequeue from here)
    this.tail = null; // Back of the queue (enqueue here)
    this.length = 0;
  }

  enqueue(value) {
    const node = new QueueNode(value);
    if (this.tail) {
      this.tail.next = node;
    } else {
      this.head = node; // First element
    }
    this.tail = node;
    this.length++;
  }

  dequeue() {
    if (!this.head) return undefined;
    const value = this.head.value;
    this.head = this.head.next;
    if (!this.head) this.tail = null; // Queue is now empty
    this.length--;
    return value;
  }

  peek() {
    return this.head ? this.head.value : undefined;
  }

  isEmpty() {
    return this.length === 0;
  }

  get size() {
    return this.length;
  }
}

const q = new Queue();
q.enqueue("a");
q.enqueue("b");
q.enqueue("c");
q.dequeue(); // "a"
q.dequeue(); // "b"
q.peek();    // "c"
q.size;      // 1
```

### Index-Offset Queue

An alternative that uses an array but avoids `shift` by tracking a head index. Dequeue is O(1) but memory is not reclaimed until the queue is compacted.

```js
class ArrayQueue {
  constructor() {
    this.items = [];
    this.head = 0;
  }

  enqueue(value) {
    this.items.push(value);
  }

  dequeue() {
    if (this.isEmpty()) return undefined;
    const value = this.items[this.head];
    this.items[this.head] = undefined; // Allow GC of the value
    this.head++;

    // Compact when half the array is wasted space
    if (this.head > this.items.length / 2) {
      this.items = this.items.slice(this.head);
      this.head = 0;
    }

    return value;
  }

  peek() {
    return this.isEmpty() ? undefined : this.items[this.head];
  }

  isEmpty() {
    return this.head >= this.items.length;
  }

  get size() {
    return this.items.length - this.head;
  }
}
```

### Queue Operations Complexity

| Operation | Array (shift) | Linked List | Index-Offset |
|-----------|--------------|-------------|--------------|
| enqueue | O(1) | O(1) | O(1) |
| dequeue | O(n) | O(1) | O(1) amortized |
| peek | O(1) | O(1) | O(1) |
| space | Compact | Pointer overhead | May waste space |

## Deque (Double-Ended Queue)

A deque allows insertion and removal from both ends in O(1).

```js
class Deque {
  constructor() {
    this.items = {};
    this.head = 0;
    this.tail = 0;
  }

  pushFront(value) {
    this.head--;
    this.items[this.head] = value;
  }

  pushBack(value) {
    this.items[this.tail] = value;
    this.tail++;
  }

  popFront() {
    if (this.isEmpty()) return undefined;
    const value = this.items[this.head];
    delete this.items[this.head];
    this.head++;
    return value;
  }

  popBack() {
    if (this.isEmpty()) return undefined;
    this.tail--;
    const value = this.items[this.tail];
    delete this.items[this.tail];
    return value;
  }

  peekFront() {
    return this.items[this.head];
  }

  peekBack() {
    return this.items[this.tail - 1];
  }

  isEmpty() {
    return this.tail - this.head <= 0;
  }

  get size() {
    return this.tail - this.head;
  }
}

const dq = new Deque();
dq.pushBack(1);
dq.pushBack(2);
dq.pushFront(0);
dq.popFront();  // 0
dq.popBack();   // 2
```

### Sliding Window Maximum (Monotonic Deque)

Find the maximum value in every window of size k. A monotonic deque solves this in O(n).

```js
function maxSlidingWindow(nums, k) {
  const result = [];
  const deque = []; // Stores indices; values are in decreasing order

  for (let i = 0; i < nums.length; i++) {
    // Remove indices that are out of the current window
    while (deque.length > 0 && deque[0] <= i - k) {
      deque.shift();
    }

    // Remove indices whose values are smaller than current
    // (they can never be the max while current is in the window)
    while (deque.length > 0 && nums[deque[deque.length - 1]] <= nums[i]) {
      deque.pop();
    }

    deque.push(i);

    // Window is full, record the max (front of deque)
    if (i >= k - 1) {
      result.push(nums[deque[0]]);
    }
  }

  return result;
}

// maxSlidingWindow([1, 3, -1, -3, 5, 3, 6, 7], 3)
// => [3, 3, 5, 5, 6, 7]
//
// Window [1, 3, -1] max=3
// Window [3, -1, -3] max=3
// Window [-1, -3, 5] max=5
// Window [-3, 5, 3] max=5
// Window [5, 3, 6] max=6
// Window [3, 6, 7] max=7
```

## Priority Queue (Heap)

A priority queue dequeues the element with the highest (or lowest) priority, not the oldest. It is typically implemented with a binary heap.

### Min Heap Implementation

A min heap is a complete binary tree where every parent is smaller than or equal to its children. Stored as an array: parent at index `i`, left child at `2i + 1`, right child at `2i + 2`.

```js
class MinHeap {
  constructor() {
    this.heap = [];
  }

  get size() {
    return this.heap.length;
  }

  peek() {
    return this.heap[0];
  }

  push(val) {
    this.heap.push(val);
    this._bubbleUp(this.heap.length - 1);
  }

  pop() {
    if (this.heap.length === 0) return undefined;
    const min = this.heap[0];
    const last = this.heap.pop();

    if (this.heap.length > 0) {
      this.heap[0] = last;
      this._sinkDown(0);
    }

    return min;
  }

  _bubbleUp(idx) {
    while (idx > 0) {
      const parent = Math.floor((idx - 1) / 2);
      if (this.heap[parent] <= this.heap[idx]) break;
      [this.heap[parent], this.heap[idx]] = [this.heap[idx], this.heap[parent]];
      idx = parent;
    }
  }

  _sinkDown(idx) {
    const n = this.heap.length;
    while (true) {
      let smallest = idx;
      const left = 2 * idx + 1;
      const right = 2 * idx + 2;

      if (left < n && this.heap[left] < this.heap[smallest]) smallest = left;
      if (right < n && this.heap[right] < this.heap[smallest]) smallest = right;

      if (smallest === idx) break;
      [this.heap[smallest], this.heap[idx]] = [this.heap[idx], this.heap[smallest]];
      idx = smallest;
    }
  }
}

const heap = new MinHeap();
heap.push(5);
heap.push(3);
heap.push(8);
heap.push(1);
heap.pop(); // 1 (smallest)
heap.pop(); // 3
heap.pop(); // 5
heap.pop(); // 8
```

### Priority Queue Complexity

| Operation | Time |
|-----------|------|
| push | O(log n) |
| pop (extract min/max) | O(log n) |
| peek | O(1) |

### When to Use a Priority Queue

- Processing tasks by priority instead of arrival order
- Finding the k smallest/largest elements in a stream
- Merge k sorted lists
- Dijkstra's shortest path algorithm
- Scheduling (CPU tasks, event loops, timers)

```js
// Find the kth largest element using a min heap of size k
function findKthLargest(nums, k) {
  const heap = new MinHeap();

  for (const num of nums) {
    heap.push(num);
    if (heap.size > k) {
      heap.pop(); // Remove smallest, keeping k largest
    }
  }

  return heap.peek(); // The smallest of the k largest = kth largest
}

// findKthLargest([3, 2, 1, 5, 6, 4], 2) => 5
// Heap maintains the 2 largest: [5, 6]. peek() = 5
```

## Stacks and Queues in JavaScript's Runtime

### The Call Stack

JavaScript uses a stack to manage function execution. Each function call pushes a frame; each return pops it. This is why stack overflow occurs with deep recursion.

```js
function a() { b(); }
function b() { c(); }
function c() { throw new Error("trace"); }
a();

// Error stack trace (read bottom to top = stack order):
//   at c
//   at b
//   at a
```

### The Event Loop Task Queue

Asynchronous callbacks (`setTimeout`, `fetch.then`, event handlers) are placed into a task queue (FIFO). The event loop dequeues tasks one at a time when the call stack is empty.

```js
console.log("1");               // Runs immediately (call stack)
setTimeout(() => console.log("2"), 0); // Queued (task queue)
console.log("3");               // Runs immediately (call stack)

// Output: 1, 3, 2
// "2" was enqueued and waits for the stack to clear
```

### The Microtask Queue

Promises use a microtask queue, which has higher priority than the regular task queue. Microtasks are drained completely before the next macrotask.

```js
console.log("1");
setTimeout(() => console.log("2"), 0);       // Macrotask queue
Promise.resolve().then(() => console.log("3")); // Microtask queue
console.log("4");

// Output: 1, 4, 3, 2
// Microtask (Promise) runs before macrotask (setTimeout)
```

## Implementing a Queue with Two Stacks

A classic problem: simulate a queue using only stack operations.

```js
class QueueFromStacks {
  constructor() {
    this.inStack = [];  // For enqueue
    this.outStack = []; // For dequeue
  }

  enqueue(val) {
    this.inStack.push(val);
  }

  dequeue() {
    if (this.outStack.length === 0) {
      // Transfer all elements from inStack to outStack
      // This reverses the order, making the oldest element on top
      while (this.inStack.length > 0) {
        this.outStack.push(this.inStack.pop());
      }
    }
    return this.outStack.pop();
  }

  peek() {
    if (this.outStack.length === 0) {
      while (this.inStack.length > 0) {
        this.outStack.push(this.inStack.pop());
      }
    }
    return this.outStack[this.outStack.length - 1];
  }

  isEmpty() {
    return this.inStack.length === 0 && this.outStack.length === 0;
  }
}

// Amortized O(1) per operation
// Each element is moved from inStack to outStack exactly once
```

## Common Mistakes

- **Mistake 1: Using `Array.shift()` for large queues.** `shift` is O(n) because it reindexes all elements. For queues with thousands of elements or frequent dequeue operations, use a linked-list-based queue or index-offset approach.

```js
// BAD: O(n^2) total for dequeueing n elements
const queue = [];
for (let i = 0; i < 100000; i++) queue.push(i);
while (queue.length > 0) queue.shift(); // Very slow

// GOOD: O(n) total
const q = new Queue(); // Linked-list based
for (let i = 0; i < 100000; i++) q.enqueue(i);
while (!q.isEmpty()) q.dequeue(); // Fast
```

- **Mistake 2: Not handling empty stack/queue.** Calling `pop` or `dequeue` on an empty structure should return `undefined` or throw, not silently corrupt state. Always check `isEmpty()` first or handle the edge case in the method.

- **Mistake 3: Using a stack when you need a queue (or vice versa).** BFS requires a queue; DFS requires a stack. Using the wrong one produces incorrect traversal order.

```js
// BAD: Using a stack for BFS
const stack = [startNode];
while (stack.length) {
  const node = stack.pop(); // LIFO: this is DFS, not BFS!
}

// GOOD: Use a queue for BFS
const queue = [startNode];
while (queue.length) {
  const node = queue.shift(); // FIFO: correct BFS order
}
```

- **Mistake 4: Forgetting to handle the monotonic stack correctly.** In "next greater element" problems, the stack must contain indices, not values, if you need to record positions. Storing values loses position information.

- **Mistake 5: Overcomplicating the priority queue.** For simple cases (small datasets, infrequent operations), a sorted array with binary search insertion works fine. Only implement a full heap when operations are frequent and the dataset is large.

---

## Best Practices

- **Use plain arrays for stacks.** JavaScript arrays with `push`/`pop` are perfectly efficient stacks. No need for a custom class unless you need extra features like `getMin()`.

- **Use a linked list or index-offset for serious queues.** If your queue will hold more than a few hundred elements or processes high-throughput dequeue operations, avoid `Array.shift()`.

- **Recognize the monotonic stack pattern.** When a problem asks "for each element, find the nearest larger/smaller element," think monotonic stack. The O(n) solution is almost always a single pass with a stack.

- **Use a deque for sliding window min/max.** The monotonic deque pattern solves sliding window extremes in O(n) total, regardless of window size.

- **Prefer existing libraries for priority queues in production.** If you need a priority queue in a real project, use a well-tested library rather than rolling your own heap. For algorithm practice, implementing a heap from scratch is excellent learning.

- **Match the data structure to the access pattern.** If you process elements in the order they arrive, use a queue. If you always need the most recent element first, use a stack. If you need the highest/lowest priority, use a heap.

## Real-World Use Cases

### Use Case 1: Undo/Redo System

```js
class UndoRedoManager {
  constructor() {
    this.undoStack = [];
    this.redoStack = [];
    this.state = null;
  }

  execute(action) {
    // Save current state before applying the action
    this.undoStack.push(this.state);
    this.state = action.apply(this.state);
    this.redoStack = []; // Clear redo history after new action
  }

  undo() {
    if (this.undoStack.length === 0) return;
    this.redoStack.push(this.state);
    this.state = this.undoStack.pop();
  }

  redo() {
    if (this.redoStack.length === 0) return;
    this.undoStack.push(this.state);
    this.state = this.redoStack.pop();
  }
}

// Usage
const manager = new UndoRedoManager();
manager.state = "Hello";
manager.execute({ apply: () => "Hello World" });
manager.execute({ apply: () => "Hello World!" });

manager.state;  // "Hello World!"
manager.undo();
manager.state;  // "Hello World"
manager.undo();
manager.state;  // "Hello"
manager.redo();
manager.state;  // "Hello World"
```

### Use Case 2: BFS Level-Order Processing

```js
// Process a tree level by level using a queue
function levelOrder(root) {
  if (!root) return [];

  const result = [];
  const queue = [root];

  while (queue.length > 0) {
    const levelSize = queue.length;
    const level = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift();
      level.push(node.val);

      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(level);
  }

  return result;
}

//       1
//      / \
//     2   3
//    / \   \
//   4   5   6
//
// levelOrder(root) => [[1], [2, 3], [4, 5, 6]]
```

### Use Case 3: Task Scheduler with Priority

```js
class TaskScheduler {
  constructor() {
    this.heap = new MinHeap(); // Uses the MinHeap class from earlier
  }

  addTask(task, priority) {
    // Lower number = higher priority
    this.heap.push({ priority, task, id: Date.now() });
  }

  processNext() {
    if (this.heap.size === 0) return null;
    const { task } = this.heap.pop();
    return task;
  }

  get pendingCount() {
    return this.heap.size;
  }
}

// Adapt MinHeap to compare by .priority instead of raw values
// (modify _bubbleUp and _sinkDown to compare this.heap[x].priority)

// Usage pattern:
// scheduler.addTask(sendEmail, 3);     // Low priority
// scheduler.addTask(processPayment, 1); // High priority
// scheduler.addTask(logAnalytics, 5);   // Lowest priority
//
// scheduler.processNext(); // processPayment (priority 1)
// scheduler.processNext(); // sendEmail (priority 3)
// scheduler.processNext(); // logAnalytics (priority 5)
```

> **Key Takeaway:** Stacks (LIFO) and queues (FIFO) control processing order. Use arrays for stacks and small queues, linked lists for large queues, monotonic stacks for "next greater/smaller" problems, monotonic deques for sliding window extremes, and heaps when you need priority-based processing instead of order-based.
