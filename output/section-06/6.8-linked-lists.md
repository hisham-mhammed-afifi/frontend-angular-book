# 6.8 Linked Lists (Singly, Doubly)

> A linked list is a linear data structure where each element (node) stores a value and a pointer to the next node. Unlike arrays, linked lists do not store elements in contiguous memory, which makes insertions and deletions at known positions O(1) but random access O(n).

---

## Why Linked Lists Matter

JavaScript arrays are dynamic and handle most scenarios well. So why study linked lists?

1. **Understanding pointers.** Linked list problems train you to think in terms of references, which is fundamental to trees, graphs, and many other data structures.
2. **O(1) insertion/deletion at known positions.** Arrays require O(n) shifting for mid-list insertions. Linked lists just rewire pointers.
3. **Foundation for other structures.** Queues, stacks, hash table chaining, LRU caches, and adjacency lists all use linked lists internally.
4. **Interview staple.** Linked list problems appear frequently in technical interviews because they test pointer manipulation, edge case handling, and algorithmic thinking.

---

## Singly Linked List

Each node has a `value` and a `next` pointer. Traversal is one-directional: head to tail.

```
Head -> [10 | ->] -> [20 | ->] -> [30 | null]
```

### Node and List Structure

```js
class ListNode {
  constructor(value) {
    this.value = value;
    this.next = null;
  }
}

class SinglyLinkedList {
  constructor() {
    this.head = null;
    this.tail = null;
    this.length = 0;
  }
}
```

### Core Operations

**Push (add to tail):**

```js
push(value) {
  const node = new ListNode(value);
  if (!this.head) {
    this.head = node;
    this.tail = node;
  } else {
    this.tail.next = node;
    this.tail = node;
  }
  this.length++;
  return this;
}
```

**Pop (remove from tail):**

Removing the tail requires traversing to the second-to-last node because there is no `prev` pointer.

```js
pop() {
  if (!this.head) return undefined;

  let current = this.head;
  let prev = null;

  while (current.next) {
    prev = current;
    current = current.next;
  }

  if (prev) {
    prev.next = null;
    this.tail = prev;
  } else {
    // Only one node
    this.head = null;
    this.tail = null;
  }

  this.length--;
  return current.value;
}
// O(n) because we must traverse to find the second-to-last node
```

**Unshift (add to head):**

```js
unshift(value) {
  const node = new ListNode(value);
  node.next = this.head;
  this.head = node;
  if (!this.tail) this.tail = node;
  this.length++;
  return this;
}
// O(1)
```

**Shift (remove from head):**

```js
shift() {
  if (!this.head) return undefined;
  const value = this.head.value;
  this.head = this.head.next;
  if (!this.head) this.tail = null;
  this.length--;
  return value;
}
// O(1)
```

**Get (access by index):**

```js
get(index) {
  if (index < 0 || index >= this.length) return undefined;
  let current = this.head;
  for (let i = 0; i < index; i++) {
    current = current.next;
  }
  return current;
}
// O(n) -- no random access
```

**Insert at index:**

```js
insert(index, value) {
  if (index < 0 || index > this.length) return false;
  if (index === 0) return !!this.unshift(value);
  if (index === this.length) return !!this.push(value);

  const node = new ListNode(value);
  const prev = this.get(index - 1);
  node.next = prev.next;
  prev.next = node;
  this.length++;
  return true;
}
// O(n) to find the position, O(1) to insert once found
```

**Remove at index:**

```js
remove(index) {
  if (index < 0 || index >= this.length) return undefined;
  if (index === 0) return this.shift();
  if (index === this.length - 1) return this.pop();

  const prev = this.get(index - 1);
  const removed = prev.next;
  prev.next = removed.next;
  this.length--;
  return removed.value;
}
```

### Singly Linked List Complexity

| Operation | Time | Notes |
|-----------|------|-------|
| Push (add to tail) | O(1) | With tail pointer |
| Pop (remove from tail) | O(n) | Must find second-to-last |
| Unshift (add to head) | O(1) | |
| Shift (remove from head) | O(1) | |
| Access by index | O(n) | Must traverse |
| Insert at known node | O(1) | Pointer rewiring only |
| Search | O(n) | Must traverse |

## Doubly Linked List

Each node has `value`, `next`, and `prev` pointers. Traversal is bidirectional.

```
null <- [10 | <-> ] <-> [20 | <-> ] <-> [30 | -> null]
Head                                     Tail
```

### Node and List Structure

```js
class DoublyNode {
  constructor(value) {
    this.value = value;
    this.next = null;
    this.prev = null;
  }
}

class DoublyLinkedList {
  constructor() {
    this.head = null;
    this.tail = null;
    this.length = 0;
  }
}
```

### Core Operations

**Push:**

```js
push(value) {
  const node = new DoublyNode(value);
  if (!this.tail) {
    this.head = node;
    this.tail = node;
  } else {
    node.prev = this.tail;
    this.tail.next = node;
    this.tail = node;
  }
  this.length++;
  return this;
}
// O(1)
```

**Pop:**

Unlike the singly linked list, this is O(1) because the tail has a `prev` pointer.

```js
pop() {
  if (!this.tail) return undefined;
  const value = this.tail.value;

  if (this.head === this.tail) {
    this.head = null;
    this.tail = null;
  } else {
    this.tail = this.tail.prev;
    this.tail.next = null;
  }

  this.length--;
  return value;
}
// O(1) -- the prev pointer gives direct access to the new tail
```

**Remove a specific node (given a reference):**

```js
removeNode(node) {
  if (node.prev) {
    node.prev.next = node.next;
  } else {
    this.head = node.next; // Removing the head
  }

  if (node.next) {
    node.next.prev = node.prev;
  } else {
    this.tail = node.prev; // Removing the tail
  }

  node.prev = null;
  node.next = null;
  this.length--;
  return node.value;
}
// O(1) -- only pointer changes, no traversal
```

**Move a node to the head (used in LRU caches):**

```js
moveToHead(node) {
  if (node === this.head) return; // Already at head
  this.removeNode(node);

  node.next = this.head;
  node.prev = null;
  if (this.head) this.head.prev = node;
  this.head = node;
  if (!this.tail) this.tail = node;
  this.length++;
}
```

### Doubly Linked List Complexity

| Operation | Time | Notes |
|-----------|------|-------|
| Push (add to tail) | O(1) | |
| Pop (remove from tail) | O(1) | Has prev pointer |
| Unshift (add to head) | O(1) | |
| Shift (remove from head) | O(1) | |
| Remove known node | O(1) | Given reference to the node |
| Access by index | O(n) | Can traverse from closer end: O(n/2) |
| Search | O(n) | |
| Space per node | Higher | Extra `prev` pointer |

## Linked List vs Array

| Feature | Array | Linked List |
|---------|-------|-------------|
| Random access | O(1) | O(n) |
| Insert/delete at beginning | O(n) (shift elements) | O(1) |
| Insert/delete at end | O(1) amortized | O(1) with tail pointer |
| Insert/delete at middle | O(n) (shift elements) | O(1) if you have the node reference |
| Memory | Contiguous, compact | Scattered, pointer overhead |
| Cache performance | Excellent (locality) | Poor (fragmented) |
| Finding an element | O(n) or O(log n) if sorted | O(n) |

**In JavaScript, use arrays by default.** Use linked lists only when you need O(1) insertion/deletion at arbitrary known positions (e.g., LRU cache), when implementing queues with high throughput, or when the data structure requires pointer-based manipulation.

## Classic Linked List Algorithms

### Reverse a Singly Linked List

The most fundamental linked list problem. Reverse all pointers in place.

```js
function reverseList(head) {
  let prev = null;
  let current = head;

  while (current) {
    const next = current.next; // Save next node
    current.next = prev;       // Reverse the pointer
    prev = current;            // Move prev forward
    current = next;            // Move current forward
  }

  return prev; // New head
}

// Before: 1 -> 2 -> 3 -> 4 -> null
// After:  4 -> 3 -> 2 -> 1 -> null

// Step-by-step:
// prev=null, current=1: 1.next=null, prev=1, current=2
// prev=1,    current=2: 2.next=1,    prev=2, current=3
// prev=2,    current=3: 3.next=2,    prev=3, current=4
// prev=3,    current=4: 4.next=3,    prev=4, current=null
// Return prev=4 (new head)
```

**Recursive version:**

```js
function reverseListRecursive(head) {
  if (!head || !head.next) return head; // Base case: 0 or 1 node

  const newHead = reverseListRecursive(head.next);
  head.next.next = head; // The node after head now points back to head
  head.next = null;       // Head no longer points forward

  return newHead;
}
```

### Detect a Cycle (Floyd's Algorithm)

Use a slow pointer (1 step) and a fast pointer (2 steps). If they meet, there is a cycle.

```js
function hasCycle(head) {
  let slow = head;
  let fast = head;

  while (fast && fast.next) {
    slow = slow.next;        // 1 step
    fast = fast.next.next;   // 2 steps

    if (slow === fast) return true; // They met inside the cycle
  }

  return false; // Fast reached the end, no cycle
}

// Time: O(n), Space: O(1)
// No hash set needed, no extra memory
```

**Find where the cycle starts:**

```js
function detectCycle(head) {
  let slow = head;
  let fast = head;

  // Phase 1: Detect if cycle exists
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
    if (slow === fast) break;
  }

  if (!fast || !fast.next) return null; // No cycle

  // Phase 2: Find the cycle start
  // Move one pointer back to head, keep the other at meeting point
  // Advance both one step at a time. They meet at the cycle start.
  slow = head;
  while (slow !== fast) {
    slow = slow.next;
    fast = fast.next;
  }

  return slow; // Start of the cycle
}
```

### Find the Middle Node

Fast pointer moves twice as fast. When fast reaches the end, slow is at the middle.

```js
function findMiddle(head) {
  let slow = head;
  let fast = head;

  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }

  return slow; // Middle node (for even length, returns the second middle)
}

// 1 -> 2 -> 3 -> 4 -> 5: returns 3
// 1 -> 2 -> 3 -> 4: returns 3 (second middle)
```

### Merge Two Sorted Lists

```js
function mergeTwoLists(l1, l2) {
  const dummy = new ListNode(0); // Dummy head simplifies edge cases
  let current = dummy;

  while (l1 && l2) {
    if (l1.value <= l2.value) {
      current.next = l1;
      l1 = l1.next;
    } else {
      current.next = l2;
      l2 = l2.next;
    }
    current = current.next;
  }

  // Attach the remaining nodes
  current.next = l1 || l2;

  return dummy.next; // Skip the dummy node
}

// l1: 1 -> 3 -> 5
// l2: 2 -> 4 -> 6
// Result: 1 -> 2 -> 3 -> 4 -> 5 -> 6
```

### Check if a Linked List is a Palindrome

Combine find-middle, reverse, and compare.

```js
function isPalindrome(head) {
  if (!head || !head.next) return true;

  // Step 1: Find the middle
  let slow = head, fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }

  // Step 2: Reverse the second half
  let prev = null;
  while (slow) {
    const next = slow.next;
    slow.next = prev;
    prev = slow;
    slow = next;
  }

  // Step 3: Compare both halves
  let left = head, right = prev;
  while (right) {
    if (left.value !== right.value) return false;
    left = left.next;
    right = right.next;
  }

  return true;
}

// 1 -> 2 -> 3 -> 2 -> 1: true
// 1 -> 2 -> 3 -> 4 -> 5: false
```

### Remove the Nth Node from the End

Use two pointers separated by n nodes. When the lead pointer reaches the end, the trailing pointer is at the target.

```js
function removeNthFromEnd(head, n) {
  const dummy = new ListNode(0);
  dummy.next = head;

  let lead = dummy;
  let trail = dummy;

  // Move lead n+1 steps ahead
  for (let i = 0; i <= n; i++) {
    lead = lead.next;
  }

  // Move both until lead reaches the end
  while (lead) {
    lead = lead.next;
    trail = trail.next;
  }

  // trail.next is the node to remove
  trail.next = trail.next.next;

  return dummy.next;
}

// Remove 2nd from end of 1 -> 2 -> 3 -> 4 -> 5
// Result: 1 -> 2 -> 3 -> 5
```

### Reverse Nodes in K-Group

Reverse every group of k nodes. If the remaining nodes are fewer than k, leave them as is.

```js
function reverseKGroup(head, k) {
  // Check if there are at least k nodes remaining
  let count = 0;
  let node = head;
  while (node && count < k) {
    node = node.next;
    count++;
  }
  if (count < k) return head; // Not enough nodes, return as is

  // Reverse k nodes
  let prev = null;
  let current = head;
  for (let i = 0; i < k; i++) {
    const next = current.next;
    current.next = prev;
    prev = current;
    current = next;
  }

  // head is now the tail of the reversed group
  // Recursively reverse the rest and attach
  head.next = reverseKGroup(current, k);

  return prev; // New head of the reversed group
}

// reverseKGroup(1->2->3->4->5, 3)
// => 3->2->1->4->5
// First group [1,2,3] reversed, remaining [4,5] has < 3 nodes, left alone
```

## The Dummy Node Technique

Many linked list problems have edge cases when operating on the head node. A **dummy node** (sentinel) placed before the head eliminates these edge cases.

```js
// WITHOUT dummy: must handle head removal separately
function removeValue(head, val) {
  while (head && head.value === val) {
    head = head.next; // Special case: removing the head
  }

  let current = head;
  while (current && current.next) {
    if (current.next.value === val) {
      current.next = current.next.next;
    } else {
      current = current.next;
    }
  }
  return head;
}

// WITH dummy: uniform logic for all nodes
function removeValue(head, val) {
  const dummy = new ListNode(0);
  dummy.next = head;

  let current = dummy;
  while (current.next) {
    if (current.next.value === val) {
      current.next = current.next.next; // Same logic for head and non-head
    } else {
      current = current.next;
    }
  }

  return dummy.next;
}
```

The dummy node pattern simplifies: merging lists, removing nodes by value, inserting in sorted order, and any operation where the head might change.

## LRU Cache (Doubly Linked List + Hash Map)

The Least Recently Used (LRU) cache is the classic real-world application of a doubly linked list. It evicts the least recently used item when the cache is full.

```js
class LRUNode {
  constructor(key, value) {
    this.key = key;
    this.value = value;
    this.prev = null;
    this.next = null;
  }
}

class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.map = new Map();         // key -> node (O(1) lookup)

    // Sentinel nodes to avoid null checks
    this.head = new LRUNode(null, null); // Most recently used
    this.tail = new LRUNode(null, null); // Least recently used
    this.head.next = this.tail;
    this.tail.prev = this.head;
  }

  _addToFront(node) {
    node.next = this.head.next;
    node.prev = this.head;
    this.head.next.prev = node;
    this.head.next = node;
  }

  _removeNode(node) {
    node.prev.next = node.next;
    node.next.prev = node.prev;
  }

  get(key) {
    if (!this.map.has(key)) return -1;

    const node = this.map.get(key);
    // Move to front (most recently used)
    this._removeNode(node);
    this._addToFront(node);
    return node.value;
  }

  put(key, value) {
    if (this.map.has(key)) {
      const node = this.map.get(key);
      node.value = value;
      this._removeNode(node);
      this._addToFront(node);
    } else {
      if (this.map.size >= this.capacity) {
        // Evict least recently used (node before tail sentinel)
        const lru = this.tail.prev;
        this._removeNode(lru);
        this.map.delete(lru.key);
      }

      const node = new LRUNode(key, value);
      this._addToFront(node);
      this.map.set(key, node);
    }
  }
}

// All operations are O(1):
// - Map gives O(1) key lookup
// - Doubly linked list gives O(1) insertion, deletion, and reordering

const cache = new LRUCache(3);
cache.put("a", 1);
cache.put("b", 2);
cache.put("c", 3);
cache.get("a");     // 1 (moves "a" to most recent)
cache.put("d", 4);  // Evicts "b" (least recently used)
cache.get("b");     // -1 (evicted)
```

## Common Mistakes

- **Mistake 1: Losing the reference to the next node during reversal.** When reversing, you must save `current.next` before overwriting it. Otherwise, you lose access to the rest of the list.

```js
// BAD: Lost reference
while (current) {
  current.next = prev; // Overwrote next before saving it!
  prev = current;
  current = current.next; // current.next is now prev, not the original next
}

// GOOD: Save next first
while (current) {
  const next = current.next; // Save before overwriting
  current.next = prev;
  prev = current;
  current = next;
}
```

- **Mistake 2: Forgetting to update `head` or `tail` pointers.** When inserting or deleting at the boundaries, failing to update the head/tail reference leaves the list in an inconsistent state.

- **Mistake 3: Not handling single-node and empty-list edge cases.** Many linked list operations behave differently when the list has 0 or 1 nodes. Always test these cases.

```js
// Edge cases to test for any linked list function:
// 1. null (empty list)
// 2. Single node: [1]
// 3. Two nodes: [1, 2]
// 4. Target is the head
// 5. Target is the tail
```

- **Mistake 4: Creating cycles accidentally.** When manipulating pointers, not nullifying old references can create unintended cycles that cause infinite loops during traversal.

```js
// BAD: Moving a node without cleaning up
function moveToFront(list, node) {
  // Removed from old position but node.next still points to old neighbor
  node.next = list.head;
  list.head = node;
  // If node was the tail, the old tail.next is now a cycle back to node!
}
```

- **Mistake 5: Using linked lists when arrays are better.** In JavaScript, arrays are highly optimized with excellent cache locality. A linked list only wins when you have a reference to the target node and need O(1) remove/insert. If you are traversing to find the node first, the O(n) traversal makes the O(1) insert irrelevant.

## Best Practices

- **Use the dummy node technique.** A sentinel dummy node before the head eliminates special-case handling for operations that might change the head. Return `dummy.next` as the result.

- **Draw the pointer changes before coding.** Linked list bugs are almost always pointer mistakes. Draw boxes and arrows on paper for 3-4 nodes, then translate to code.

- **Use the fast/slow pointer for linked list properties.** Finding the middle, detecting cycles, and finding the start of a cycle all use the same fast/slow pattern. Learn it once and apply everywhere.

- **Test with edge cases systematically.** Always test: empty list, single node, two nodes, operation on head, operation on tail, and the general case.

- **Prefer doubly linked lists when you need to remove by reference.** If your algorithm requires removing a node given a direct reference (not an index), a doubly linked list enables O(1) removal without needing to find the predecessor.

- **Combine with a hash map for O(1) access + O(1) manipulation.** The LRU cache pattern (Map + doubly linked list) gives constant-time access by key and constant-time reordering by recency.

## Real-World Use Cases

### Use Case 1: Browser History Navigation

```js
class BrowserHistory {
  constructor(homepage) {
    this.current = new DoublyNode(homepage);
  }

  visit(url) {
    const page = new DoublyNode(url);
    this.current.next = page;
    page.prev = this.current;
    this.current = page;
    // Forward history is discarded when visiting a new page
  }

  back(steps) {
    while (steps > 0 && this.current.prev) {
      this.current = this.current.prev;
      steps--;
    }
    return this.current.value;
  }

  forward(steps) {
    while (steps > 0 && this.current.next) {
      this.current = this.current.next;
      steps--;
    }
    return this.current.value;
  }
}

const browser = new BrowserHistory("google.com");
browser.visit("youtube.com");
browser.visit("github.com");
browser.back(1);    // "youtube.com"
browser.back(1);    // "google.com"
browser.forward(1); // "youtube.com"
browser.visit("stackoverflow.com"); // Discards github.com from forward history
browser.forward(1); // "stackoverflow.com" (no forward, stays)
```

### Use Case 2: Text Editor with Cursor

```js
class TextEditor {
  constructor() {
    // Characters stored as a doubly linked list
    // Cursor is between two nodes
    this.head = new DoublyNode(null); // Sentinel before first char
    this.tail = new DoublyNode(null); // Sentinel after last char
    this.head.next = this.tail;
    this.tail.prev = this.head;
    this.cursor = this.head; // Cursor is after this node
  }

  insert(char) {
    const node = new DoublyNode(char);
    const after = this.cursor.next;
    this.cursor.next = node;
    node.prev = this.cursor;
    node.next = after;
    after.prev = node;
    this.cursor = node; // Cursor moves after the inserted char
  }

  delete() {
    // Delete the character before the cursor
    if (this.cursor === this.head) return; // Nothing to delete
    const toRemove = this.cursor;
    this.cursor = toRemove.prev;
    this.cursor.next = toRemove.next;
    toRemove.next.prev = this.cursor;
  }

  moveLeft() {
    if (this.cursor !== this.head) {
      this.cursor = this.cursor.prev;
    }
  }

  moveRight() {
    if (this.cursor.next !== this.tail) {
      this.cursor = this.cursor.next;
    }
  }

  toString() {
    let result = "";
    let node = this.head.next;
    while (node !== this.tail) {
      result += node.value;
      node = node.next;
    }
    return result;
  }
}

const editor = new TextEditor();
editor.insert("h");
editor.insert("e");
editor.insert("l");
editor.insert("o"); // "helo"
editor.moveLeft();   // Cursor before 'o'
editor.insert("l");  // "hello"
editor.toString();   // "hello"
```

### Use Case 3: Task Queue with Cancellation

```js
class CancellableTaskQueue {
  constructor() {
    this.head = new DoublyNode(null); // Sentinel
    this.tail = new DoublyNode(null); // Sentinel
    this.head.next = this.tail;
    this.tail.prev = this.head;
    this.taskMap = new Map(); // taskId -> node
  }

  enqueue(taskId, task) {
    const node = new DoublyNode({ taskId, task });
    // Add before tail (end of queue)
    const beforeTail = this.tail.prev;
    beforeTail.next = node;
    node.prev = beforeTail;
    node.next = this.tail;
    this.tail.prev = node;
    this.taskMap.set(taskId, node);
  }

  dequeue() {
    if (this.head.next === this.tail) return null; // Empty
    const node = this.head.next;
    this.head.next = node.next;
    node.next.prev = this.head;
    this.taskMap.delete(node.value.taskId);
    return node.value.task;
  }

  cancel(taskId) {
    if (!this.taskMap.has(taskId)) return false;
    const node = this.taskMap.get(taskId);
    node.prev.next = node.next;
    node.next.prev = node.prev;
    this.taskMap.delete(taskId);
    return true;
  }
}

// O(1) enqueue, O(1) dequeue, O(1) cancel by ID
// Without a linked list, cancelling from the middle of a queue is O(n)

const queue = new CancellableTaskQueue();
queue.enqueue("task1", () => console.log("Task 1"));
queue.enqueue("task2", () => console.log("Task 2"));
queue.enqueue("task3", () => console.log("Task 3"));
queue.cancel("task2"); // Remove task2 from the middle in O(1)
queue.dequeue()();     // "Task 1"
queue.dequeue()();     // "Task 3" (task2 was cancelled)
```

> **Key Takeaway:** Linked lists trade random access for O(1) insertion and deletion at known positions. Master the core patterns (reverse, fast/slow pointers, dummy nodes, merge) and understand when to combine a doubly linked list with a hash map for structures like LRU caches that need both O(1) lookup and O(1) reordering.
