# 6.9 Trees and Binary Search Trees

> Trees are hierarchical data structures that model relationships like file systems, the DOM, and organizational charts. Binary search trees impose an ordering property that enables O(log n) search, insert, and delete operations.

---

## Tree Fundamentals

A tree is a collection of nodes connected by edges with no cycles. Every tree has exactly one root node, and every other node has exactly one parent.

### Core Terminology

```
        A           <- root (depth 0)
       / \
      B   C         <- depth 1
     / \   \
    D   E   F       <- depth 2 (D, E, F are leaves)
   /
  G                 <- depth 3
```

| Term | Definition |
|------|-----------|
| Root | The topmost node with no parent |
| Parent | A node that has children |
| Child | A node connected below another node |
| Sibling | Nodes that share the same parent |
| Leaf | A node with no children |
| Edge | The connection between a parent and child |
| Depth | Number of edges from the root to the node |
| Height | Number of edges on the longest path from a node down to a leaf |
| Level | Depth + 1 (some definitions use depth directly) |
| Subtree | A node and all of its descendants |

```javascript
// Height of the tree = height of root = 3
// Depth of G = 3
// Height of B = 2
// D has one child (G), so D is NOT a leaf
// E, F, G are leaves
```

### Types of Trees

| Type | Rule |
|------|------|
| Binary Tree | Each node has at most 2 children |
| Binary Search Tree | Left child < parent < right child for every node |
| Balanced BST | Height difference of left/right subtrees is at most 1 |
| Complete Binary Tree | All levels filled except possibly the last, which fills left to right |
| Full Binary Tree | Every node has 0 or 2 children |
| Perfect Binary Tree | All internal nodes have 2 children, all leaves at same depth |

## Tree Traversals

Traversal means visiting every node in the tree exactly once. The order determines what kind of traversal it is.

### Depth-First Search (DFS): Recursive

DFS explores as deep as possible before backtracking. There are three orderings based on when you process the current node.

```javascript
// Inorder: Left -> Node -> Right
// For a BST, inorder traversal gives sorted output
function inorder(node, result = []) {
  if (node === null) return result;
  inorder(node.left, result);
  result.push(node.val);
  inorder(node.right, result);
  return result;
}

// Preorder: Node -> Left -> Right
// Useful for creating a copy of the tree or serialization
function preorder(node, result = []) {
  if (node === null) return result;
  result.push(node.val);
  preorder(node.left, result);
  preorder(node.right, result);
  return result;
}

// Postorder: Left -> Right -> Node
// Useful for deleting a tree or evaluating expression trees
function postorder(node, result = []) {
  if (node === null) return result;
  postorder(node.left, result);
  postorder(node.right, result);
  result.push(node.val);
  return result;
}

//       10
//      /  \
//     5    15
//    / \     \
//   3   7    18

console.log(inorder(root));    // [3, 5, 7, 10, 15, 18] <- sorted!
console.log(preorder(root));   // [10, 5, 3, 7, 15, 18]
console.log(postorder(root));  // [3, 7, 5, 18, 15, 10]
```

### When to Use Each Traversal

| Traversal | Use Case |
|-----------|----------|
| Inorder | Get sorted elements from BST |
| Preorder | Serialize/deserialize a tree, copy a tree |
| Postorder | Delete a tree (children before parent), evaluate expressions |
| Level-order | Find shortest path, level-by-level processing |

### Depth-First Search (DFS): Iterative

Recursive DFS uses the call stack implicitly. For large trees, the call stack may overflow. Iterative DFS uses an explicit stack.

```javascript
// Iterative inorder
function inorderIterative(root) {
  const result = [];
  const stack = [];
  let current = root;

  while (current !== null || stack.length > 0) {
    // Go as far left as possible
    while (current !== null) {
      stack.push(current);
      current = current.left;
    }
    // Process the node
    current = stack.pop();
    result.push(current.val);
    // Move to right subtree
    current = current.right;
  }

  return result;
}

// Iterative preorder
function preorderIterative(root) {
  if (root === null) return [];
  const result = [];
  const stack = [root];

  while (stack.length > 0) {
    const node = stack.pop();
    result.push(node.val);
    // Push right first so left is processed first (LIFO)
    if (node.right) stack.push(node.right);
    if (node.left) stack.push(node.left);
  }

  return result;
}

// Iterative postorder (using two stacks)
function postorderIterative(root) {
  if (root === null) return [];
  const stack1 = [root];
  const stack2 = [];

  while (stack1.length > 0) {
    const node = stack1.pop();
    stack2.push(node.val);
    if (node.left) stack1.push(node.left);
    if (node.right) stack1.push(node.right);
  }

  return stack2.reverse();
}
```

### Breadth-First Search (BFS): Level-Order Traversal

BFS visits nodes level by level using a queue.

```javascript
function levelOrder(root) {
  if (root === null) return [];
  const result = [];
  const queue = [root];

  while (queue.length > 0) {
    const levelSize = queue.length;
    const currentLevel = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift();
      currentLevel.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(currentLevel);
  }

  return result;
}

//       10
//      /  \
//     5    15
//    / \     \
//   3   7    18

console.log(levelOrder(root));
// [[10], [5, 15], [3, 7, 18]]
```

For better performance with large trees, use an index-based queue instead of `shift()`:

```javascript
function levelOrderOptimized(root) {
  if (root === null) return [];
  const result = [];
  const queue = [root];
  let front = 0;

  while (front < queue.length) {
    const levelSize = queue.length - front;
    const currentLevel = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue[front++];
      currentLevel.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(currentLevel);
  }

  return result;
}
```

## Binary Search Tree (BST)

A BST enforces the ordering property: for every node, all values in its left subtree are strictly less, and all values in its right subtree are strictly greater. This property enables efficient O(log n) operations on balanced trees.

### BST Class Implementation

```javascript
class BSTNode {
  constructor(val) {
    this.val = val;
    this.left = null;
    this.right = null;
  }
}

class BinarySearchTree {
  constructor() {
    this.root = null;
  }

  // --- INSERT ---
  insert(val) {
    const newNode = new BSTNode(val);
    if (this.root === null) {
      this.root = newNode;
      return this;
    }

    let current = this.root;
    while (true) {
      // Skip duplicates
      if (val === current.val) return this;

      if (val < current.val) {
        if (current.left === null) {
          current.left = newNode;
          return this;
        }
        current = current.left;
      } else {
        if (current.right === null) {
          current.right = newNode;
          return this;
        }
        current = current.right;
      }
    }
  }

  // --- SEARCH ---
  search(val) {
    let current = this.root;
    while (current !== null) {
      if (val === current.val) return current;
      if (val < current.val) {
        current = current.left;
      } else {
        current = current.right;
      }
    }
    return null; // not found
  }

  // --- CONTAINS ---
  contains(val) {
    return this.search(val) !== null;
  }

  // --- MINIMUM ---
  findMin(node = this.root) {
    if (node === null) return null;
    while (node.left !== null) {
      node = node.left;
    }
    return node;
  }

  // --- MAXIMUM ---
  findMax(node = this.root) {
    if (node === null) return null;
    while (node.right !== null) {
      node = node.right;
    }
    return node;
  }
}

const bst = new BinarySearchTree();
[10, 5, 15, 3, 7, 12, 18].forEach((v) => bst.insert(v));

//       10
//      /  \
//     5    15
//    / \   / \
//   3   7 12  18

console.log(bst.search(7));      // BSTNode { val: 7, ... }
console.log(bst.search(99));     // null
console.log(bst.findMin().val);  // 3
console.log(bst.findMax().val);  // 18
```

### BST Deletion

Deletion is the trickiest BST operation. There are three cases:

1. **Leaf node (no children):** Simply remove it.
2. **One child:** Replace the node with its child.
3. **Two children:** Replace the node with its inorder successor (smallest value in right subtree), then delete the successor.

```javascript
class BinarySearchTree {
  // ... (previous methods)

  delete(val) {
    this.root = this._deleteNode(this.root, val);
  }

  _deleteNode(node, val) {
    if (node === null) return null;

    if (val < node.val) {
      // Target is in the left subtree
      node.left = this._deleteNode(node.left, val);
    } else if (val > node.val) {
      // Target is in the right subtree
      node.right = this._deleteNode(node.right, val);
    } else {
      // Found the node to delete

      // Case 1: Leaf node
      if (node.left === null && node.right === null) {
        return null;
      }

      // Case 2: One child
      if (node.left === null) return node.right;
      if (node.right === null) return node.left;

      // Case 3: Two children
      // Find inorder successor (smallest in right subtree)
      let successor = node.right;
      while (successor.left !== null) {
        successor = successor.left;
      }

      // Replace value with successor's value
      node.val = successor.val;

      // Delete the successor from right subtree
      node.right = this._deleteNode(node.right, successor.val);
    }

    return node;
  }
}

//       10                    10
//      /  \                  /  \
//     5    15    delete(5)  7    15
//    / \   / \   -------> / \   / \
//   3   7 12  18         3   _ 12  18
//                        (successor of 5 is 7)
```

### Inorder Successor and Predecessor

The inorder successor is the next node in sorted order. The inorder predecessor is the previous node.

```javascript
// Find inorder successor of a given value
function inorderSuccessor(root, target) {
  let successor = null;
  let current = root;

  while (current !== null) {
    if (target < current.val) {
      // Current could be the successor
      successor = current;
      current = current.left;
    } else {
      // Successor must be to the right
      current = current.right;
    }
  }

  return successor;
}

// Find inorder predecessor of a given value
function inorderPredecessor(root, target) {
  let predecessor = null;
  let current = root;

  while (current !== null) {
    if (target > current.val) {
      // Current could be the predecessor
      predecessor = current;
      current = current.right;
    } else {
      // Predecessor must be to the left
      current = current.left;
    }
  }

  return predecessor;
}

//       10
//      /  \
//     5    15
//    / \   / \
//   3   7 12  18

// Successor of 7 is 10
// Predecessor of 12 is 10
```

## BST Complexity Analysis

| Operation | Average Case | Worst Case (skewed) |
|-----------|-------------|-------------------|
| Search | O(log n) | O(n) |
| Insert | O(log n) | O(n) |
| Delete | O(log n) | O(n) |
| Find Min/Max | O(log n) | O(n) |
| Traversal | O(n) | O(n) |
| Space | O(n) | O(n) |

The worst case happens when elements are inserted in sorted order, creating a degenerate tree (essentially a linked list):

```javascript
// Inserting [1, 2, 3, 4, 5] creates:
//   1
//    \
//     2
//      \
//       3
//        \
//         4
//          \
//           5
// Height = n - 1 instead of log(n)
// All operations degrade to O(n)
```

## Classic Tree Problems

### 1. Maximum Depth (Height) of a Binary Tree

```javascript
function maxDepth(root) {
  if (root === null) return 0;
  const leftDepth = maxDepth(root.left);
  const rightDepth = maxDepth(root.right);
  return 1 + Math.max(leftDepth, rightDepth);
}

// Iterative with BFS
function maxDepthBFS(root) {
  if (root === null) return 0;
  let depth = 0;
  const queue = [root];
  let front = 0;

  while (front < queue.length) {
    const levelSize = queue.length - front;
    depth++;
    for (let i = 0; i < levelSize; i++) {
      const node = queue[front++];
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
  }

  return depth;
}
```

### 2. Validate BST

A common mistake is only checking a node against its direct parent. You must ensure every node falls within the valid range defined by all of its ancestors.

```javascript
// BAD: Only checks immediate parent-child relationship
function isValidBSTBad(node) {
  if (node === null) return true;
  if (node.left && node.left.val >= node.val) return false;
  if (node.right && node.right.val <= node.val) return false;
  return isValidBSTBad(node.left) && isValidBSTBad(node.right);
}
// This fails on:
//     5
//    / \
//   1   6
//      / \
//     3   7    <- 3 is less than 5, violates BST property!

// CORRECT: Pass min/max bounds through recursion
function isValidBST(node, min = -Infinity, max = Infinity) {
  if (node === null) return true;
  if (node.val <= min || node.val >= max) return false;
  return (
    isValidBST(node.left, min, node.val) &&
    isValidBST(node.right, node.val, max)
  );
}

// Alternative: inorder traversal must produce sorted array
function isValidBSTInorder(root) {
  let prev = -Infinity;
  const stack = [];
  let current = root;

  while (current !== null || stack.length > 0) {
    while (current !== null) {
      stack.push(current);
      current = current.left;
    }
    current = stack.pop();
    if (current.val <= prev) return false;
    prev = current.val;
    current = current.right;
  }

  return true;
}
```

### 3. Lowest Common Ancestor (LCA) in BST

The BST property simplifies LCA. If both targets are smaller than the current node, LCA is in the left subtree. If both are larger, LCA is in the right subtree. Otherwise, the current node is the LCA.

```javascript
function lowestCommonAncestor(root, p, q) {
  let current = root;
  while (current !== null) {
    if (p < current.val && q < current.val) {
      current = current.left;
    } else if (p > current.val && q > current.val) {
      current = current.right;
    } else {
      return current; // split point is the LCA
    }
  }
  return null;
}

//       10
//      /  \
//     5    15
//    / \   / \
//   3   7 12  18

// LCA(3, 7)  = 5  (both < 10, go left; 3 < 5 but 7 > 5, split)
// LCA(3, 18) = 10 (3 < 10 but 18 > 10, split at root)
// LCA(12, 18) = 15
```

### 4. LCA in a General Binary Tree

Without the BST property, you must search both subtrees.

```javascript
function lcaGeneral(root, p, q) {
  if (root === null) return null;
  if (root.val === p || root.val === q) return root;

  const left = lcaGeneral(root.left, p, q);
  const right = lcaGeneral(root.right, p, q);

  // If both sides found something, current node is LCA
  if (left !== null && right !== null) return root;

  // Otherwise return whichever side found something
  return left !== null ? left : right;
}
```

### 5. Invert (Mirror) a Binary Tree

```javascript
function invertTree(root) {
  if (root === null) return null;
  // Swap children
  const temp = root.left;
  root.left = root.right;
  root.right = temp;
  // Recurse
  invertTree(root.left);
  invertTree(root.right);
  return root;
}

//       10              10
//      /  \            /  \
//     5    15   ->   15    5
//    / \   / \      / \   / \
//   3   7 12  18  18  12 7   3
```

### 6. Path Sum

Check if any root-to-leaf path sums to a target.

```javascript
function hasPathSum(root, target) {
  if (root === null) return false;

  // Leaf node check
  if (root.left === null && root.right === null) {
    return root.val === target;
  }

  const remaining = target - root.val;
  return (
    hasPathSum(root.left, remaining) ||
    hasPathSum(root.right, remaining)
  );
}

// Collect all root-to-leaf paths that sum to target
function pathSum(root, target) {
  const result = [];

  function dfs(node, remaining, path) {
    if (node === null) return;

    path.push(node.val);

    if (node.left === null && node.right === null && remaining === node.val) {
      result.push([...path]); // copy the path
    }

    dfs(node.left, remaining - node.val, path);
    dfs(node.right, remaining - node.val, path);

    path.pop(); // backtrack
  }

  dfs(root, target, []);
  return result;
}
```

### 7. Check if Tree is Balanced

A tree is balanced if the height difference of left and right subtrees is at most 1 for every node.

```javascript
// BAD: O(n^2) - recalculates height at every node
function isBalancedBad(root) {
  if (root === null) return true;
  const diff = Math.abs(maxDepth(root.left) - maxDepth(root.right));
  return diff <= 1 && isBalancedBad(root.left) && isBalancedBad(root.right);
}

// GOOD: O(n) - single pass, return -1 to signal imbalance
function isBalanced(root) {
  function checkHeight(node) {
    if (node === null) return 0;

    const left = checkHeight(node.left);
    if (left === -1) return -1; // left subtree unbalanced

    const right = checkHeight(node.right);
    if (right === -1) return -1; // right subtree unbalanced

    if (Math.abs(left - right) > 1) return -1; // current node unbalanced

    return 1 + Math.max(left, right);
  }

  return checkHeight(root) !== -1;
}
```

### 8. Serialize and Deserialize a Binary Tree

Convert a tree to a string and back. Preorder traversal with null markers works well.

```javascript
function serialize(root) {
  const result = [];

  function dfs(node) {
    if (node === null) {
      result.push("null");
      return;
    }
    result.push(String(node.val));
    dfs(node.left);
    dfs(node.right);
  }

  dfs(root);
  return result.join(",");
}

function deserialize(data) {
  const values = data.split(",");
  let index = 0;

  function dfs() {
    if (index >= values.length || values[index] === "null") {
      index++;
      return null;
    }

    const node = new TreeNode(Number(values[index]));
    index++;
    node.left = dfs();
    node.right = dfs();
    return node;
  }

  return dfs();
}

const serialized = serialize(root);
console.log(serialized);
// "10,5,3,null,null,7,null,null,15,null,18,null,null"

const deserialized = deserialize(serialized);
// Reconstructs the original tree
```

### 9. Kth Smallest Element in BST

Inorder traversal of a BST yields sorted order. Stop at the kth element.

```javascript
function kthSmallest(root, k) {
  const stack = [];
  let current = root;
  let count = 0;

  while (current !== null || stack.length > 0) {
    while (current !== null) {
      stack.push(current);
      current = current.left;
    }

    current = stack.pop();
    count++;

    if (count === k) return current.val;

    current = current.right;
  }

  return -1; // k is larger than tree size
}

// kthSmallest(root, 1) -> 3  (smallest)
// kthSmallest(root, 4) -> 10 (4th smallest)
```

## Build BST from Sorted Array

Given a sorted array, build a height-balanced BST. Pick the middle element as root, recurse on left and right halves.

```javascript
function sortedArrayToBST(nums) {
  function build(left, right) {
    if (left > right) return null;

    const mid = Math.floor((left + right) / 2);
    const node = new TreeNode(nums[mid]);
    node.left = build(left, mid - 1);
    node.right = build(mid + 1, right);
    return node;
  }

  return build(0, nums.length - 1);
}

const balanced = sortedArrayToBST([1, 2, 3, 4, 5, 6, 7]);
//       4
//      / \
//     2   6
//    / \ / \
//   1  3 5  7
// Perfectly balanced, height = log2(7) = 2
```

## Balanced BSTs: Conceptual Overview

Plain BSTs degrade to O(n) when insertions arrive in sorted order. Self-balancing BSTs fix this by performing rotations after insertions and deletions.

### AVL Trees

An AVL tree maintains a balance factor (height of left subtree minus height of right subtree) of -1, 0, or 1 for every node. When an operation violates this, rotations restore balance.

```
// Right Rotation (when left-heavy)
//
//       30              20
//      /               /  \
//     20       ->    10    30
//    /
//   10
//
// Left Rotation (when right-heavy)
//
//   10                 20
//     \               /  \
//      20     ->    10    30
//        \
//         30
```

Four rotation cases:

| Case | Condition | Fix |
|------|-----------|-----|
| Left-Left | Left child is left-heavy | Right rotation |
| Right-Right | Right child is right-heavy | Left rotation |
| Left-Right | Left child is right-heavy | Left rotation on child, then right rotation |
| Right-Left | Right child is left-heavy | Right rotation on child, then left rotation |

### Red-Black Trees

Red-black trees use node coloring (red or black) and a set of rules to maintain approximate balance. They allow slightly less balance than AVL trees but require fewer rotations on average, making them better for write-heavy workloads.

**Rules:**
1. Every node is red or black.
2. The root is black.
3. Every leaf (null) is black.
4. Red nodes cannot have red children (no two consecutive reds).
5. Every path from root to leaf has the same number of black nodes.

**When to use which:**

| Tree | Best For | Guarantees |
|------|----------|-----------|
| AVL | Read-heavy workloads | Strict balance (height <= 1.44 * log2(n)) |
| Red-Black | Write-heavy workloads | Relaxed balance (height <= 2 * log2(n)) |

In practice, you rarely implement these from scratch. JavaScript's built-in `Map` and `Set` use hash tables, not trees. But understanding balanced BSTs is important for interviews and for recognizing when they appear in libraries or databases (e.g., B-trees in database indexes).

## Trie (Prefix Tree)

A trie is a tree-like structure specialized for string operations. Each node represents a character, and paths from root to marked nodes represent complete words.

```javascript
class TrieNode {
  constructor() {
    this.children = new Map();
    this.isEnd = false;
  }
}

class Trie {
  constructor() {
    this.root = new TrieNode();
  }

  // Insert a word into the trie
  insert(word) {
    let current = this.root;
    for (const char of word) {
      if (!current.children.has(char)) {
        current.children.set(char, new TrieNode());
      }
      current = current.children.get(char);
    }
    current.isEnd = true;
  }

  // Search for an exact word
  search(word) {
    let current = this.root;
    for (const char of word) {
      if (!current.children.has(char)) return false;
      current = current.children.get(char);
    }
    return current.isEnd;
  }

  // Check if any word starts with the prefix
  startsWith(prefix) {
    let current = this.root;
    for (const char of prefix) {
      if (!current.children.has(char)) return false;
      current = current.children.get(char);
    }
    return true;
  }

  // Get all words with a given prefix (autocomplete)
  autocomplete(prefix) {
    let current = this.root;
    for (const char of prefix) {
      if (!current.children.has(char)) return [];
      current = current.children.get(char);
    }

    const results = [];
    this._collect(current, prefix, results);
    return results;
  }

  _collect(node, prefix, results) {
    if (node.isEnd) results.push(prefix);
    for (const [char, child] of node.children) {
      this._collect(child, prefix + char, results);
    }
  }
}

const trie = new Trie();
["apple", "app", "application", "apply", "bat", "ball"].forEach((w) =>
  trie.insert(w)
);

console.log(trie.search("app"));        // true
console.log(trie.search("ap"));         // false (not a complete word)
console.log(trie.startsWith("app"));    // true
console.log(trie.autocomplete("app"));  // ["app", "apple", "application", "apply"]
console.log(trie.autocomplete("ba"));   // ["bat", "ball"]
```

### Trie Complexity

| Operation | Time | Space |
|-----------|------|-------|
| Insert | O(m) where m = word length | O(m) new nodes |
| Search | O(m) | O(1) |
| startsWith | O(m) | O(1) |
| autocomplete | O(m + k) where k = total chars in results | O(k) |

## Common Mistakes

- **Only checking immediate parent-child for BST validation.** You must propagate min/max bounds through the entire subtree. A node deep in the left subtree that is greater than an ancestor violates the BST property, but a local check would miss it.

- **Not handling the two-children case in BST deletion.** When deleting a node with two children, you must find the inorder successor (or predecessor), copy its value, and then recursively delete the successor. Forgetting to delete the successor leaves a duplicate in the tree.

- **Using shift() for BFS queues.** `Array.shift()` is O(n) because it re-indexes the entire array. For large trees, this makes BFS O(n^2) instead of O(n). Use an index pointer or a proper queue implementation.

- **Forgetting the base case in recursion.** Always check `if (node === null) return ...` first. Missing this causes `TypeError: Cannot read properties of null`.

- **Confusing height and depth.** Height is measured from the node downward to the farthest leaf. Depth is measured from the root downward to the node. The height of a tree equals the depth of its deepest leaf.

- **Inserting sorted data into an unbalanced BST.** Inserting `[1, 2, 3, 4, 5]` into a plain BST creates a linked list with O(n) operations. Either shuffle the data, use a balanced BST, or build from a sorted array using the divide-and-conquer approach.

- **Mutating the path array during backtracking without copying.** When collecting paths, always push a copy (`[...path]`) into results. If you push the path reference directly, all results will reference the same (now-empty) array after backtracking completes.

---

## Best Practices

- **Use the right traversal for the job.** Inorder for sorted output. Preorder for serialization. Postorder for bottom-up computation. Level-order for level-by-level processing.

- **Prefer iterative traversal for production code.** Recursive traversal is clean but risks stack overflow on deep trees. Iterative versions with an explicit stack have predictable memory usage.

- **Build balanced BSTs from sorted data.** If you have sorted data, use the divide-and-conquer approach (pick middle as root) to build a balanced tree in O(n) time.

- **Use the "min/max bounds" technique for BST validation.** It is more efficient and less error-prone than collecting inorder traversal into an array and checking if it is sorted.

- **Return early in DFS when possible.** If you are searching for a specific node, return as soon as you find it. Do not traverse the entire tree unnecessarily.

- **Track parent pointers or use recursive returns for deletion.** The recursive `_deleteNode` pattern that returns the (potentially updated) child reference is the cleanest way to handle all three deletion cases.

### 1. Autocomplete Search Box

A trie powers the autocomplete feature in search inputs. As the user types, query the trie with the current prefix to retrieve suggestions in O(m) time where m is the prefix length.

```javascript
class SearchBox {
  constructor(dictionary) {
    this.trie = new Trie();
    dictionary.forEach((word) => this.trie.insert(word.toLowerCase()));
  }

  suggest(input, limit = 5) {
    const prefix = input.toLowerCase().trim();
    if (prefix.length === 0) return [];
    return this.trie.autocomplete(prefix).slice(0, limit);
  }
}

const search = new SearchBox([
  "JavaScript", "Java", "JSON", "jQuery",
  "TypeScript", "Tailwind", "Testing",
  "React", "Redux", "Router"
]);

console.log(search.suggest("ja"));
// ["javascript", "java", "json", "jquery"]

console.log(search.suggest("t", 3));
// ["typescript", "tailwind", "testing"]
```

### 2. File System Navigator

File systems are naturally tree structures. A tree can model directories and files for operations like finding files, calculating sizes, and displaying folder structure.

```javascript
class FSNode {
  constructor(name, isFile = false, size = 0) {
    this.name = name;
    this.isFile = isFile;
    this.size = size;
    this.children = [];
  }

  addChild(child) {
    this.children.push(child);
    return child;
  }

  // Calculate total size recursively (postorder logic)
  totalSize() {
    if (this.isFile) return this.size;
    return this.children.reduce((sum, child) => sum + child.totalSize(), 0);
  }

  // Find all files matching a pattern (DFS)
  findFiles(pattern) {
    const results = [];

    function dfs(node, path) {
      const currentPath = path + "/" + node.name;
      if (node.isFile && node.name.includes(pattern)) {
        results.push(currentPath);
      }
      for (const child of node.children) {
        dfs(child, currentPath);
      }
    }

    dfs(this, "");
    return results;
  }

  // Print tree structure (preorder)
  print(indent = "") {
    const prefix = this.isFile ? "  " : "  ";
    console.log(indent + prefix + this.name);
    for (const child of this.children) {
      child.print(indent + "  ");
    }
  }
}

const root = new FSNode("src");
const components = root.addChild(new FSNode("components"));
components.addChild(new FSNode("Header.js", true, 1200));
components.addChild(new FSNode("Footer.js", true, 800));
const utils = root.addChild(new FSNode("utils"));
utils.addChild(new FSNode("format.js", true, 400));
utils.addChild(new FSNode("validate.js", true, 600));
root.addChild(new FSNode("index.js", true, 200));

root.print();
// src
//   components
//     Header.js
//     Footer.js
//   utils
//     format.js
//     validate.js
//   index.js

console.log(root.totalSize()); // 3200
console.log(root.findFiles(".js"));
// ["/src/components/Header.js", "/src/components/Footer.js",
//  "/src/utils/format.js", "/src/utils/validate.js", "/src/index.js"]
```

### 3. Expression Evaluator

Expression trees represent mathematical expressions where leaves are operands and internal nodes are operators. Postorder traversal evaluates the expression bottom-up.

```javascript
function buildExpressionTree(postfix) {
  const stack = [];

  for (const token of postfix) {
    if ("+-*/".includes(token)) {
      const right = stack.pop();
      const left = stack.pop();
      const node = new TreeNode(token);
      node.left = left;
      node.right = right;
      stack.push(node);
    } else {
      stack.push(new TreeNode(Number(token)));
    }
  }

  return stack.pop();
}

function evaluate(node) {
  if (node.left === null && node.right === null) {
    return node.val; // leaf = operand
  }

  const left = evaluate(node.left);
  const right = evaluate(node.right);

  switch (node.val) {
    case "+": return left + right;
    case "-": return left - right;
    case "*": return left * right;
    case "/": return left / right;
  }
}

// Expression: (3 + 4) * 2
// Postfix: 3 4 + 2 *
const expr = buildExpressionTree(["3", "4", "+", "2", "*"]);

//       *
//      / \
//     +   2
//    / \
//   3   4

console.log(evaluate(expr)); // 14
```
