# 7.14 Mapped Types

> Mapped types iterate over the keys of a type and transform each property according to a rule. They are the mechanism behind `Partial`, `Required`, `Readonly`, `Pick`, and most custom utility types you will ever write.

---

## What Is a Mapped Type?

A mapped type creates a new object type by iterating over a set of keys and defining a property for each one.

```typescript
type MappedType = {
  [K in Keys]: ValueType;
};
```

The `[K in Keys]` syntax iterates over each member of `Keys` (a union of string, number, or symbol literals). For each `K`, a property is created with the specified value type.

```typescript
type Flags = {
  [K in "read" | "write" | "execute"]: boolean;
};

// Result:
// {
//   read: boolean;
//   write: boolean;
//   execute: boolean;
// }
```

---

## Mapping Over `keyof`

The most common pattern is mapping over the keys of an existing type.

```typescript
type Copy<T> = {
  [K in keyof T]: T[K];
};

interface User {
  id: string;
  name: string;
  age: number;
}

type UserCopy = Copy<User>;
// { id: string; name: string; age: number }
```

`keyof T` produces a union of all property names. `T[K]` looks up the type of property `K` on `T`. Together, they copy every property from the source type.

## Modifier Operations

Mapped types can add or remove `readonly` and `?` (optional) modifiers on each property.

### Adding Modifiers

```typescript
// Add readonly to all properties
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

// Add optional to all properties
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// Add both
type ReadonlyPartial<T> = {
  readonly [K in keyof T]?: T[K];
};
```

### Removing Modifiers with `-`

The `-` prefix removes a modifier.

```typescript
// Remove optional (make all properties required)
type MyRequired<T> = {
  [K in keyof T]-?: T[K];
};

// Remove readonly (make all properties mutable)
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

// Remove both
type MutableRequired<T> = {
  -readonly [K in keyof T]-?: T[K];
};
```

### Explicit `+` Prefix

The `+` prefix explicitly adds a modifier. It is the default and rarely written, but it exists for symmetry with `-`.

```typescript
// These are equivalent:
type A<T> = { readonly [K in keyof T]: T[K] };
type B<T> = { +readonly [K in keyof T]: T[K] };

// These are equivalent:
type C<T> = { [K in keyof T]?: T[K] };
type D<T> = { [K in keyof T]+?: T[K] };
```

## Key Remapping with `as`

TypeScript 4.1 introduced the `as` clause, which lets you transform, filter, or replace keys during mapping.

### Renaming Keys

```typescript
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface User {
  name: string;
  age: number;
  email: string;
}

type UserGetters = Getters<User>;
// {
//   getName: () => string;
//   getAge: () => number;
//   getEmail: () => string;
// }
```

The `string & K` is needed because `keyof T` might include `symbol` or `number` keys, and `Capitalize` only works on strings.

### Setters Pattern

```typescript
type Setters<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (value: T[K]) => void;
};

type UserSetters = Setters<User>;
// {
//   setName: (value: string) => void;
//   setAge: (value: number) => void;
//   setEmail: (value: string) => void;
// }
```

### Combined Getters and Setters

```typescript
type Accessors<T> = Getters<T> & Setters<T>;

type UserAccessors = Accessors<User>;
// getName, getAge, getEmail, setName, setAge, setEmail
```

### Event Handler Names

```typescript
type EventHandlers<T> = {
  [K in keyof T as `on${Capitalize<string & K>}Change`]: (
    oldValue: T[K],
    newValue: T[K],
  ) => void;
};

type UserHandlers = EventHandlers<User>;
// {
//   onNameChange: (oldValue: string, newValue: string) => void;
//   onAgeChange: (oldValue: number, newValue: number) => void;
//   onEmailChange: (oldValue: string, newValue: string) => void;
// }
```

## Filtering Keys

Returning `never` from the `as` clause removes the key from the result.

### Filter by Value Type

```typescript
type OnlyStrings<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface User {
  id: number;
  name: string;
  email: string;
  age: number;
  active: boolean;
}

type StringFields = OnlyStrings<User>;
// { name: string; email: string }
```

### Filter by Key Name

```typescript
// Keep only keys that start with "is"
type BooleanFlags<T> = {
  [K in keyof T as K extends `is${string}` ? K : never]: T[K];
};

interface Settings {
  isDarkMode: boolean;
  isCompact: boolean;
  fontSize: number;
  language: string;
}

type Flags = BooleanFlags<Settings>;
// { isDarkMode: boolean; isCompact: boolean }
```

### Remove Specific Keys (Type-Safe Omit)

```typescript
type StrictOmit<T, K extends keyof T> = {
  [P in keyof T as P extends K ? never : P]: T[P];
};

type PublicUser = StrictOmit<User, "id">;
// { name: string; email: string; age: number; active: boolean }

// Unlike built-in Omit, this errors on invalid keys:
// type Bad = StrictOmit<User, "nonexistent">; // Error
```

### Exclude Methods, Keep Data

```typescript
type DataOnly<T> = {
  [K in keyof T as T[K] extends Function ? never : K]: T[K];
};

interface UserService {
  id: string;
  name: string;
  greet(): string;
  save(): Promise<void>;
}

type UserData = DataOnly<UserService>;
// { id: string; name: string }
```

## Homomorphic Mapped Types

A mapped type is **homomorphic** when it maps over `keyof T` where `T` is a type parameter. Homomorphic mapped types preserve the structure of the original type, including optional and readonly modifiers.

### Modifier Preservation

```typescript
interface User {
  readonly id: string;
  name: string;
  email?: string;
}

// Homomorphic: preserves readonly and optional
type Transformed<T> = {
  [K in keyof T]: T[K] | null;
};

type NullableUser = Transformed<User>;
// {
//   readonly id: string | null;  // readonly preserved
//   name: string | null;
//   email?: string | null;       // optional preserved
// }
```

`Partial`, `Required`, `Readonly`, and `Pick` are all homomorphic because they use `keyof T`.

### Non-Homomorphic: `Record`

`Record` is not homomorphic because it does not derive keys from a type parameter via `keyof`.

```typescript
type Record<K extends string | number | symbol, T> = {
  [P in K]: T;
};

// It creates new properties, not transformations of existing ones
type UserMap = Record<"alice" | "bob", User>;
// { alice: User; bob: User }
```

### Why It Matters

When you create a mapped type over `keyof T`, TypeScript carries forward the original modifiers by default. You add or remove modifiers explicitly with `readonly`, `-readonly`, `?`, and `-?`. Non-homomorphic types start with a clean slate.

```typescript
interface Config {
  readonly host: string;
  port?: number;
}

// Homomorphic: keeps readonly and optional
type ValueStrings<T> = { [K in keyof T]: string };
type A = ValueStrings<Config>;
// { readonly host: string; port?: string }

// Non-homomorphic: no modifiers carried
type FromKeys = { [K in "host" | "port"]: string };
// { host: string; port: string } (no readonly, no optional)
```

## Mapping Over Unions (Not Just `keyof`)

The `in` clause can iterate over any union, not just `keyof`.

### From a String Literal Union

```typescript
type Permissions = "read" | "write" | "delete";

type PermissionMap = {
  [K in Permissions]: boolean;
};
// { read: boolean; write: boolean; delete: boolean }
```

### From Computed Unions

```typescript
type HttpMethod = "GET" | "POST" | "PUT" | "DELETE";
type Endpoint = "/users" | "/posts";

type Routes = {
  [M in HttpMethod as `${M} ${Endpoint}`]: () => Response;
};
// Produces all combinations:
// "GET /users", "GET /posts", "POST /users", "POST /posts", etc.
```

Wait: the above does not work directly because the `as` clause transforms keys one at a time. To produce all combinations, you need a different approach:

```typescript
type RouteKey = `${HttpMethod} ${Endpoint}`;

type Routes = {
  [K in RouteKey]: () => Response;
};
// "GET /users" | "GET /posts" | "POST /users" | ... etc.
```

## Mapped Types with Conditional Transformations

Combine mapped types with conditional types to transform properties based on their type.

### Make All String Properties Uppercase-Capable

```typescript
type Uppercasable<T> = {
  [K in keyof T]: T[K] extends string ? Uppercase<T[K] & string> : T[K];
};
```

### Wrap All Properties in Promise

```typescript
type Async<T> = {
  [K in keyof T]: Promise<T[K]>;
};

interface Config {
  host: string;
  port: number;
}

type AsyncConfig = Async<Config>;
// { host: Promise<string>; port: Promise<number> }
```

### Convert Methods to Async

```typescript
type AsyncMethods<T> = {
  [K in keyof T]: T[K] extends (...args: infer A) => infer R
    ? (...args: A) => Promise<Awaited<R>>
    : T[K];
};

interface UserService {
  getUser(id: string): User;
  deleteUser(id: string): boolean;
  name: string;
}

type AsyncUserService = AsyncMethods<UserService>;
// {
//   getUser(id: string): Promise<User>;
//   deleteUser(id: string): Promise<boolean>;
//   name: string;  // non-function untouched
// }
```

### Nullable Version of Each Property

```typescript
type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

type NullableUser = Nullable<User>;
// { id: string | null; name: string | null; email: string | null }
```

## Recursive Mapped Types

Mapped types can be applied recursively to handle nested structures.

### `DeepReadonly`

```typescript
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends Function
    ? T[K]
    : T[K] extends object
      ? DeepReadonly<T[K]>
      : T[K];
};

interface Config {
  database: {
    host: string;
    credentials: {
      user: string;
      password: string;
    };
  };
}

type FrozenConfig = DeepReadonly<Config>;
// All properties at every level are readonly
```

### `DeepPartial`

```typescript
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends Function
    ? T[K]
    : T[K] extends object
      ? DeepPartial<T[K]>
      : T[K];
};

// Allows partially specifying deeply nested structures
const overrides: DeepPartial<Config> = {
  database: {
    credentials: {
      password: "new-password",
    },
  },
};
```

### `DeepRequired`

```typescript
type DeepRequired<T> = {
  [K in keyof T]-?: T[K] extends object
    ? DeepRequired<T[K]>
    : T[K];
};
```

## Mapped Type Patterns Reference

### Identity (Copy)

```typescript
type Identity<T> = { [K in keyof T]: T[K] };
```

### Value Transformation

```typescript
type Stringify<T> = { [K in keyof T]: string };
type Boxed<T> = { [K in keyof T]: { value: T[K] } };
type Arrayed<T> = { [K in keyof T]: T[K][] };
```

### Key Transformation

```typescript
type Prefixed<T, P extends string> = {
  [K in keyof T as `${P}${Capitalize<string & K>}`]: T[K];
};

type Suffixed<T, S extends string> = {
  [K in keyof T as `${string & K}${S}`]: T[K];
};
```

### Selective Modification

```typescript
// Make specific keys optional, keep rest required
type PartialBy<T, K extends keyof T> =
  Omit<T, K> & Partial<Pick<T, K>>;

// Make specific keys required, keep rest as-is
type RequiredBy<T, K extends keyof T> =
  Omit<T, K> & Required<Pick<T, K>>;

// Make specific keys readonly, keep rest mutable
type ReadonlyBy<T, K extends keyof T> =
  Omit<T, K> & Readonly<Pick<T, K>>;
```

## Common Mistakes

### Mistake 1: Forgetting That `keyof` Includes `symbol` and `number`

```typescript
type PrefixKeys<T> = {
  // Error: Type 'K' is not assignable to type 'string'
  // [K in keyof T as `prefix_${K}`]: T[K];

  // Fix: intersect with string
  [K in keyof T as `prefix_${string & K}`]: T[K];
};
```

`keyof T` can include `string | number | symbol`. Template literal types only accept strings. Use `string & K` or `K extends string ? ... : never` to filter.

### Mistake 2: Accidental Modifier Loss in Non-Homomorphic Types

```typescript
interface Config {
  readonly host: string;
  port?: number;
}

// Non-homomorphic: loses readonly and optional
type Recreated = { [K in "host" | "port"]: Config[K] };
// { host: string; port: number } -- no readonly, no optional!

// Homomorphic: preserves them
type Preserved = { [K in keyof Config]: Config[K] };
// { readonly host: string; port?: number }
```

### Mistake 3: Expecting `as never` to Remove a Property from the Value

```typescript
// Bad: this does not filter keys, it changes the key name to never
type Bad<T> = {
  [K in keyof T]: T[K] extends string ? T[K] : never;
  // This keeps ALL keys but sets non-string values to never
};

// Good: filter in the as clause
type Good<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};
```

### Mistake 4: Key Overlap with Numeric and String Keys

```typescript
type HasBoth = {
  0: "zero";
  "0": "string-zero";
};

// In JavaScript, obj[0] and obj["0"] are the same property
// TypeScript may produce unexpected results when mapping over both
```

### Mistake 5: Recursive Types Without Base Cases

```typescript
// Bad: infinite recursion on all object types including Date, Array, etc.
type BadDeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? BadDeepReadonly<T[K]> : T[K];
};

// This recurses into Date (making its methods readonly, which is useless)
// and Array (producing bizarre mapped array types)

// Good: exclude known built-in objects
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends Function
    ? T[K]
    : T[K] extends Date
      ? T[K]
      : T[K] extends Map<any, any>
        ? T[K]
        : T[K] extends Set<any>
          ? T[K]
          : T[K] extends object
            ? DeepReadonly<T[K]>
            : T[K];
};
```

## Best Practices

### 1. Use Built-in Utilities Before Writing Custom Mapped Types

```typescript
// Do not reinvent the wheel
// Bad:
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Good: use the built-in
type Update = Partial<User>;
```

### 2. Use `as` for Key Filtering Instead of Separate Steps

```typescript
// Instead of:
type StringKeys<T> = {
  [K in keyof T]: T[K] extends string ? K : never;
}[keyof T];
type StringFields<T> = Pick<T, StringKeys<T>>;

// Use as clause directly:
type StringFields<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};
```

### 3. Name Your Mapped Types Clearly

```typescript
// Good: the name tells you what it does
type ReadonlyDeep<T> = { readonly [K in keyof T]: DeepReadonly<T[K]> };
type Nullable<T> = { [K in keyof T]: T[K] | null };
type WithPrefix<T, P extends string> = {
  [K in keyof T as `${P}${Capitalize<string & K>}`]: T[K];
};
```

### 4. Compose Small Mapped Types Into Complex Ones

```typescript
// Small, focused types
type Readonly<T> = { readonly [K in keyof T]: T[K] };
type Partial<T> = { [K in keyof T]?: T[K] };

// Composed
type ReadonlyPartial<T> = Readonly<Partial<T>>;
```

### 5. Handle Edge Cases in Recursive Types

Always exclude `Function`, `Date`, `Map`, `Set`, `RegExp`, and other built-in objects from recursive descent to avoid mapping their internal structure.

## Real-World Use Cases

### Type-Safe Form Builder

```typescript
type FieldConfig<T> = {
  label: string;
  placeholder?: string;
  validate?: (value: T) => string | null;
};

type FormConfig<T> = {
  [K in keyof T]: FieldConfig<T[K]>;
};

interface RegistrationData {
  username: string;
  email: string;
  age: number;
  acceptTerms: boolean;
}

const registrationForm: FormConfig<RegistrationData> = {
  username: {
    label: "Username",
    placeholder: "Enter your username",
    validate: (value) => (value.length < 3 ? "Too short" : null),
  },
  email: {
    label: "Email",
    placeholder: "you@example.com",
    validate: (value) => (value.includes("@") ? null : "Invalid email"),
  },
  age: {
    label: "Age",
    validate: (value) => (value < 18 ? "Must be 18+" : null),
  },
  acceptTerms: {
    label: "I accept the terms",
    validate: (value) => (value ? null : "Required"),
  },
};
```

### ORM-Style Column Definitions

```typescript
type ColumnType = "string" | "number" | "boolean" | "date";

type TypeScriptType<C extends ColumnType> =
  C extends "string" ? string :
  C extends "number" ? number :
  C extends "boolean" ? boolean :
  C extends "date" ? Date :
  never;

type ColumnDef = {
  type: ColumnType;
  nullable?: boolean;
  default?: unknown;
};

type SchemaToType<S extends Record<string, ColumnDef>> = {
  [K in keyof S]: S[K]["nullable"] extends true
    ? TypeScriptType<S[K]["type"]> | null
    : TypeScriptType<S[K]["type"]>;
};

const userSchema = {
  id: { type: "number" as const },
  name: { type: "string" as const },
  email: { type: "string" as const, nullable: true as const },
  createdAt: { type: "date" as const },
  active: { type: "boolean" as const },
};

type UserRow = SchemaToType<typeof userSchema>;
// {
//   id: number;
//   name: string;
//   email: string | null;
//   createdAt: Date;
//   active: boolean;
// }
```

### State Management with Derived Actions

```typescript
interface AppState {
  user: User | null;
  theme: "light" | "dark";
  notifications: Notification[];
  sidebarOpen: boolean;
}

// Generate setter action types from state
type SetterActions<T> = {
  [K in keyof T as `set${Capitalize<string & K>}`]: (value: T[K]) => void;
};

// Generate toggle actions for boolean properties
type ToggleActions<T> = {
  [K in keyof T as T[K] extends boolean ? `toggle${Capitalize<string & K>}` : never]: () => void;
};

// Generate reset actions
type ResetActions<T> = {
  [K in keyof T as `reset${Capitalize<string & K>}`]: () => void;
};

type AllActions = SetterActions<AppState> & ToggleActions<AppState> & ResetActions<AppState>;
// SetterActions produces: setUser, setTheme, setNotifications, setSidebarOpen
// ToggleActions produces: toggleSidebarOpen (only boolean props)
// ResetActions produces: resetUser, resetTheme, resetNotifications, resetSidebarOpen

function createStore(initialState: AppState): AppState & AllActions {
  const state = { ...initialState };

  const actions: Record<string, Function> = {};
  for (const key of Object.keys(state) as (keyof AppState)[]) {
    const capitalized = key.charAt(0).toUpperCase() + key.slice(1);
    actions[`set${capitalized}`] = (value: any) => { (state as any)[key] = value; };
    actions[`reset${capitalized}`] = () => { (state as any)[key] = initialState[key]; };
    if (typeof state[key] === "boolean") {
      actions[`toggle${capitalized}`] = () => { (state as any)[key] = !(state as any)[key]; };
    }
  }

  return { ...state, ...actions } as AppState & AllActions;
}
```

---

> **Key Takeaway:** Mapped types iterate over keys and transform each property, forming the backbone of TypeScript's type transformation system. Master the `[K in keyof T]` pattern, modifier operations (`readonly`, `?`, `-readonly`, `-?`), key remapping with `as`, and key filtering by returning `never`. Combine them with conditional types and template literals to build powerful, reusable type utilities.
