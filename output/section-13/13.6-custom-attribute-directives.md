# 13.6 Custom Attribute Directives

> Custom attribute directives let you encapsulate reusable DOM behavior (tooltips, auto-focus, click-outside detection, permission checks) in a class that can be applied to any element with a single attribute. They are the primary mechanism for extending HTML element behavior in Angular.

---

## Anatomy of a Directive

A directive is a class decorated with `@Directive`. Unlike components, directives have no template. They attach to a host element and modify it.

```typescript
import { Directive, inject, ElementRef } from '@angular/core';

@Directive({
  selector: '[appHighlight]',
  standalone: true,
})
export class HighlightDirective {
  private el = inject(ElementRef);

  constructor() {
    this.el.nativeElement.style.backgroundColor = 'yellow';
  }
}
```

```html
<p appHighlight>This paragraph has a yellow background.</p>
```

### Key Parts

| Part | Purpose |
|---|---|
| `@Directive` | Marks the class as a directive |
| `selector` | CSS attribute selector (conventionally prefixed) |
| `standalone: true` | Makes the directive self-contained (no NgModule needed) |
| `ElementRef` | Reference to the host DOM element |
| `inject()` | Modern dependency injection (preferred over constructor injection) |

---

## Selectors

Directive selectors determine which elements the directive attaches to. They use CSS selector syntax.

```typescript
// Attribute selector (most common)
@Directive({ selector: '[appTooltip]' })

// Attribute with value
@Directive({ selector: '[appTooltip][message]' })

// Class selector
@Directive({ selector: '.auto-focus' })

// Element selector (rare for directives, common for components)
@Directive({ selector: 'input[appTrim]' })

// Multiple selectors
@Directive({ selector: '[appDraggable], [appSortable]' })

// NOT selector (match all inputs NOT having readonly)
@Directive({ selector: 'input:not([readonly])' })
```

### Naming Convention

Use an application-specific prefix (like `app`) to avoid collisions with native HTML attributes or third-party libraries:

```typescript
// Good: prefixed with "app"
@Directive({ selector: '[appTooltip]' })
@Directive({ selector: '[appClickOutside]' })
@Directive({ selector: '[appAutoFocus]' })

// Bad: no prefix, could collide with native attributes or other libraries
@Directive({ selector: '[tooltip]' })
@Directive({ selector: '[focus]' })
```

The Angular style guide recommends using a consistent prefix. For a project called "Acme", use `[acmeTooltip]`.

## The `host` Property (Modern Approach)

The `host` metadata property is the preferred way to interact with the host element. It replaces `@HostBinding` and `@HostListener`.

### Host Event Listeners

```typescript
@Directive({
  selector: '[appHoverHighlight]',
  standalone: true,
  host: {
    '(mouseenter)': 'onMouseEnter()',
    '(mouseleave)': 'onMouseLeave()',
  }
})
export class HoverHighlightDirective {
  private el = inject(ElementRef);

  private originalBg = '';

  color = input('yellow');

  onMouseEnter() {
    this.originalBg = this.el.nativeElement.style.backgroundColor;
    this.el.nativeElement.style.backgroundColor = this.color();
  }

  onMouseLeave() {
    this.el.nativeElement.style.backgroundColor = this.originalBg;
  }
}
```

```html
<p appHoverHighlight>Yellow on hover</p>
<p appHoverHighlight color="#e3f2fd">Blue on hover</p>
```

### Host Property Bindings

Bind DOM properties and attributes on the host element:

```typescript
@Directive({
  selector: '[appBusy]',
  standalone: true,
  host: {
    '[class.is-busy]': 'isBusy()',
    '[attr.aria-busy]': 'isBusy()',
    '[style.opacity]': 'isBusy() ? 0.5 : 1',
    '[style.pointer-events]': 'isBusy() ? "none" : "auto"',
  }
})
export class BusyDirective {
  isBusy = input(false);
}
```

```html
<button appBusy [isBusy]="isLoading()">Save</button>
```

When `isLoading()` is true, the button gets the `is-busy` class, `aria-busy="true"`, reduced opacity, and disabled pointer events.

### Host Bindings with Signals

Angular 20+ type-checks all expressions in `host` bindings. Signal reads in `host` bindings are reactive and automatically re-evaluated when the signal changes:

```typescript
@Directive({
  selector: '[appSize]',
  standalone: true,
  host: {
    '[style.width.px]': 'width()',
    '[style.height.px]': 'height()',
    '[class.size-sm]': 'size() === "sm"',
    '[class.size-md]': 'size() === "md"',
    '[class.size-lg]': 'size() === "lg"',
  }
})
export class SizeDirective {
  size = input<'sm' | 'md' | 'lg'>('md');

  width = computed(() => {
    const sizes = { sm: 100, md: 200, lg: 300 };
    return sizes[this.size()];
  });

  height = computed(() => {
    const sizes = { sm: 80, md: 160, lg: 240 };
    return sizes[this.size()];
  });
}
```

## Signal Inputs on Directives

Directives use the same `input()` function as components:

```typescript
@Directive({
  selector: '[appTooltip]',
  standalone: true,
  host: {
    '(mouseenter)': 'show()',
    '(mouseleave)': 'hide()',
    '(focus)': 'show()',
    '(blur)': 'hide()',
  }
})
export class TooltipDirective {
  // The directive name itself can be an input (aliased)
  message = input.required<string>({ alias: 'appTooltip' });
  position = input<'top' | 'bottom' | 'left' | 'right'>('top');

  private el = inject(ElementRef);
  private tooltipEl: HTMLElement | null = null;

  show() {
    if (this.tooltipEl) return;
    this.tooltipEl = document.createElement('div');
    this.tooltipEl.className = `tooltip tooltip-${this.position()}`;
    this.tooltipEl.textContent = this.message();
    this.el.nativeElement.appendChild(this.tooltipEl);
  }

  hide() {
    this.tooltipEl?.remove();
    this.tooltipEl = null;
  }
}
```

```html
<button appTooltip="Save your changes" position="bottom">Save</button>
<span [appTooltip]="dynamicMessage()" position="right">Hover me</span>
```

### Aliasing the Selector as an Input

A common pattern is to alias an input to match the directive selector. This lets users pass a value directly on the selector attribute:

```typescript
@Directive({
  selector: '[appPermission]',
  standalone: true,
  host: {
    '[style.display]': 'hasPermission() ? null : "none"',
  }
})
export class PermissionDirective {
  // "appPermission" attribute doubles as the input value
  requiredPermission = input.required<string>({ alias: 'appPermission' });

  private authService = inject(AuthService);

  hasPermission = computed(() =>
    this.authService.currentUser()?.permissions.includes(this.requiredPermission()) ?? false
  );
}
```

```html
<button appPermission="admin:delete">Delete User</button>
<a appPermission="reports:view">View Reports</a>
```

## Outputs on Directives

Directives can emit custom events using `output()`:

```typescript
@Directive({
  selector: '[appClickOutside]',
  standalone: true,
  host: {
    '(document:click)': 'onDocumentClick($event)',
  }
})
export class ClickOutsideDirective {
  clickedOutside = output<void>({ alias: 'appClickOutside' });

  private el = inject(ElementRef);

  onDocumentClick(event: MouseEvent) {
    if (!this.el.nativeElement.contains(event.target)) {
      this.clickedOutside.emit();
    }
  }
}
```

```html
<div class="dropdown" (appClickOutside)="closeDropdown()">
  Dropdown content
</div>
```

The output alias matches the selector, so `(appClickOutside)` works as the event binding.

## ElementRef and Renderer2

### ElementRef (Direct DOM Access)

`ElementRef` provides direct access to the underlying DOM element:

```typescript
@Directive({
  selector: '[appAutoFocus]',
  standalone: true,
})
export class AutoFocusDirective {
  private el = inject(ElementRef);

  constructor() {
    // Focus the element after the view initializes
    afterNextRender(() => {
      this.el.nativeElement.focus();
    });
  }
}
```

```html
<input appAutoFocus placeholder="I'm focused on load" />
```

**Caution:** Direct DOM access via `ElementRef.nativeElement` is not safe for server-side rendering (SSR) because `nativeElement` is undefined on the server.

### Renderer2 (Safe DOM Manipulation)

`Renderer2` provides a platform-agnostic API for DOM manipulation that works in SSR, Web Workers, and other non-browser environments:

```typescript
@Directive({
  selector: '[appRipple]',
  standalone: true,
  host: {
    '(click)': 'createRipple($event)',
    '[style.position]': '"relative"',
    '[style.overflow]': '"hidden"',
  }
})
export class RippleDirective {
  private el = inject(ElementRef);
  private renderer = inject(Renderer2);

  createRipple(event: MouseEvent) {
    const rect = this.el.nativeElement.getBoundingClientRect();
    const ripple = this.renderer.createElement('span');

    const x = event.clientX - rect.left;
    const y = event.clientY - rect.top;

    this.renderer.setStyle(ripple, 'position', 'absolute');
    this.renderer.setStyle(ripple, 'left', `${x}px`);
    this.renderer.setStyle(ripple, 'top', `${y}px`);
    this.renderer.setStyle(ripple, 'width', '0');
    this.renderer.setStyle(ripple, 'height', '0');
    this.renderer.setStyle(ripple, 'border-radius', '50%');
    this.renderer.setStyle(ripple, 'background', 'rgba(255,255,255,0.4)');
    this.renderer.setStyle(ripple, 'transform', 'translate(-50%, -50%)');
    this.renderer.setStyle(ripple, 'animation', 'ripple-expand 0.6s ease-out');

    this.renderer.appendChild(this.el.nativeElement, ripple);

    setTimeout(() => {
      this.renderer.removeChild(this.el.nativeElement, ripple);
    }, 600);
  }
}
```

### When to Use Each

| Scenario | Use |
|---|---|
| Reading element dimensions/position | `ElementRef` (no safe alternative for `getBoundingClientRect`) |
| Setting styles, classes, attributes | `host` bindings (preferred) or `Renderer2` |
| Creating/removing child elements | `Renderer2` |
| Listening to events | `host` property |
| Focusing an element | `ElementRef.nativeElement.focus()` |
| SSR-safe code | `Renderer2` for mutations, guard `ElementRef` reads |

## The `exportAs` Property

`exportAs` makes the directive instance available as a template reference variable:

```typescript
@Directive({
  selector: '[appToggle]',
  standalone: true,
  exportAs: 'appToggle',
})
export class ToggleDirective {
  isOpen = signal(false);

  toggle() {
    this.isOpen.update(v => !v);
  }

  open() {
    this.isOpen.set(true);
  }

  close() {
    this.isOpen.set(false);
  }
}
```

```html
<div appToggle #menuToggle="appToggle">
  <button (click)="menuToggle.toggle()">
    {{ menuToggle.isOpen() ? 'Close' : 'Open' }} Menu
  </button>

  @if (menuToggle.isOpen()) {
    <nav class="menu">
      <a href="/home">Home</a>
      <a href="/about">About</a>
      <a href="/contact">Contact</a>
    </nav>
  }
</div>
```

`exportAs` is powerful because it exposes the directive's API directly in the template without needing event bindings or outputs.

## Cleanup with DestroyRef

Directives that set up subscriptions, event listeners, or timers must clean up when destroyed:

```typescript
@Directive({
  selector: '[appIntersectionObserver]',
  standalone: true,
})
export class IntersectionObserverDirective {
  private el = inject(ElementRef);
  private destroyRef = inject(DestroyRef);

  visible = output<boolean>({ alias: 'appIntersectionObserver' });

  constructor() {
    afterNextRender(() => {
      const observer = new IntersectionObserver(
        (entries) => {
          entries.forEach(entry => {
            this.visible.emit(entry.isIntersecting);
          });
        },
        { threshold: 0.1 }
      );

      observer.observe(this.el.nativeElement);

      // Clean up when the directive is destroyed
      this.destroyRef.onDestroy(() => {
        observer.disconnect();
      });
    });
  }
}
```

```html
<div (appIntersectionObserver)="onVisibilityChange($event)">
  Observed element
</div>
```

## Legacy Decorators (For Reference)

You will encounter `@HostBinding` and `@HostListener` in existing codebases. They are functionally equivalent to the `host` property but are no longer recommended for new code.

```typescript
// Legacy approach
@Directive({
  selector: '[appHighlight]',
  standalone: true,
})
export class HighlightDirective {
  @Input() color = 'yellow';

  @HostBinding('style.backgroundColor')
  bgColor = '';

  @HostListener('mouseenter')
  onMouseEnter() {
    this.bgColor = this.color;
  }

  @HostListener('mouseleave')
  onMouseLeave() {
    this.bgColor = '';
  }
}

// Modern equivalent
@Directive({
  selector: '[appHighlight]',
  standalone: true,
  host: {
    '(mouseenter)': 'onMouseEnter()',
    '(mouseleave)': 'onMouseLeave()',
    '[style.backgroundColor]': 'bgColor()',
  }
})
export class HighlightDirective {
  color = input('yellow');
  bgColor = signal('');

  onMouseEnter() {
    this.bgColor.set(this.color());
  }

  onMouseLeave() {
    this.bgColor.set('');
  }
}
```

## Common Mistakes

- **Mistake 1: Directly manipulating the DOM without considering SSR.**
  Using `ElementRef.nativeElement` to set styles, add classes, or append elements breaks in server-side rendering where the DOM does not exist. Use `Renderer2` for mutations or guard DOM access with `afterNextRender` / `afterRenderEffect`.

- **Mistake 2: Forgetting to clean up listeners and observers.**
  Directives that create `IntersectionObserver`, `MutationObserver`, `ResizeObserver`, or manual event listeners must disconnect/remove them when destroyed. Use `DestroyRef.onDestroy()` to register cleanup callbacks.

- **Mistake 3: Using generic selectors without a prefix.**
  A directive with `selector: '[tooltip]'` risks colliding with native HTML attributes or third-party libraries. Always use an application-specific prefix: `[appTooltip]`.

- **Mistake 4: Using `@HostBinding`/`@HostListener` in new code.**
  These decorators exist for backwards compatibility. The `host` metadata property is the modern, preferred approach. It is more declarative, works with signal expressions, and is type-checked in Angular 20+.

- **Mistake 5: Making directives that do too much.**
  A directive should encapsulate one specific behavior (highlight, tooltip, click-outside, auto-resize). If your directive handles multiple unrelated concerns, split it into separate directives and compose them using the Directive Composition API.

- **Mistake 6: Forgetting to import the directive in standalone components.**
  Unlike built-in bindings, custom directives must be imported in the component's `imports` array. Without the import, the directive selector is silently ignored.

---

## Best Practices

- **Use the `host` property for all host interactions.** It replaces `@HostBinding` and `@HostListener`, is more declarative, supports signal expressions, and is type-checked in Angular 20+.

- **Use `inject()` over constructor injection.** The `inject()` function is the modern Angular pattern for dependency injection and works in directives the same way as in components.

- **Use signal inputs for directive configuration.** `input()` and `input.required()` provide type-safe, reactive configuration that integrates with computed signals and host bindings.

- **Use `afterNextRender` for DOM-dependent initialization.** If your directive needs to measure or manipulate the DOM on startup, wrap that logic in `afterNextRender` to ensure it runs in the browser, not during SSR.

- **Use `Renderer2` for creating or modifying child elements.** For operations like appending tooltips, creating ripple effects, or injecting overlay elements, `Renderer2` is SSR-safe and follows Angular's abstraction layer.

- **Use `exportAs` to expose directive APIs to the template.** When the directive has methods or state that the template needs to read, `exportAs` provides clean access without extra outputs or services.

## Real-World Use Cases

### Auto-Resize Textarea Directive

A directive that automatically adjusts a textarea's height to fit its content.

```typescript
@Directive({
  selector: 'textarea[appAutoResize]',
  standalone: true,
  host: {
    '(input)': 'resize()',
    '[style.overflow]': '"hidden"',
    '[style.resize]': '"none"',
  }
})
export class AutoResizeDirective {
  private el = inject(ElementRef);

  minRows = input(2);
  maxRows = input(10);

  private lineHeight = 0;

  constructor() {
    afterNextRender(() => {
      const computed = window.getComputedStyle(this.el.nativeElement);
      this.lineHeight = parseFloat(computed.lineHeight) || 20;
      this.resize();
    });
  }

  resize() {
    const textarea = this.el.nativeElement as HTMLTextAreaElement;
    const minHeight = this.lineHeight * this.minRows();
    const maxHeight = this.lineHeight * this.maxRows();

    textarea.style.height = 'auto';
    const scrollHeight = textarea.scrollHeight;
    textarea.style.height = `${Math.max(minHeight, Math.min(scrollHeight, maxHeight))}px`;

    if (scrollHeight > maxHeight) {
      textarea.style.overflow = 'auto';
    } else {
      textarea.style.overflow = 'hidden';
    }
  }
}
```

```html
<textarea
  appAutoResize
  [minRows]="3"
  [maxRows]="15"
  [(ngModel)]="comment"
  name="comment"
  placeholder="Write a comment..."
></textarea>
```

### Long Press Directive

A directive that detects long press (touch or mouse) and emits an event after a configurable duration.

```typescript
@Directive({
  selector: '[appLongPress]',
  standalone: true,
  host: {
    '(mousedown)': 'onStart()',
    '(mouseup)': 'onEnd()',
    '(mouseleave)': 'onEnd()',
    '(touchstart)': 'onStart()',
    '(touchend)': 'onEnd()',
    '(touchcancel)': 'onEnd()',
    '[style.user-select]': '"none"',
    '[style.-webkit-user-select]': '"none"',
  }
})
export class LongPressDirective {
  duration = input(500);

  longPressed = output<void>({ alias: 'appLongPress' });

  private timeout: ReturnType<typeof setTimeout> | null = null;
  private pressed = false;

  onStart() {
    this.pressed = true;
    this.timeout = setTimeout(() => {
      if (this.pressed) {
        this.longPressed.emit();
      }
    }, this.duration());
  }

  onEnd() {
    this.pressed = false;
    if (this.timeout) {
      clearTimeout(this.timeout);
      this.timeout = null;
    }
  }
}
```

```html
<div class="item" (appLongPress)="onLongPress(item)" [duration]="800">
  Long press to delete
</div>

<button (appLongPress)="confirmAction()">Hold to confirm</button>
```

### Copy to Clipboard Directive with exportAs

A directive that copies text to the clipboard and exposes its state via `exportAs`.

```typescript
@Directive({
  selector: '[appCopyToClipboard]',
  standalone: true,
  exportAs: 'clipboard',
  host: {
    '(click)': 'copy()',
    '[style.cursor]': '"pointer"',
    '[attr.role]': '"button"',
    '[attr.tabindex]': '"0"',
    '(keydown.enter)': 'copy()',
    '(keydown.space)': 'copy()',
  }
})
export class CopyToClipboardDirective {
  text = input.required<string>({ alias: 'appCopyToClipboard' });

  copied = output<void>();

  state = signal<'idle' | 'copied' | 'error'>('idle');

  async copy() {
    try {
      await navigator.clipboard.writeText(this.text());
      this.state.set('copied');
      this.copied.emit();

      setTimeout(() => this.state.set('idle'), 2000);
    } catch {
      this.state.set('error');
      setTimeout(() => this.state.set('idle'), 2000);
    }
  }
}
```

```html
<code
  [appCopyToClipboard]="codeSnippet()"
  #cb="clipboard"
  (copied)="showNotification('Copied!')"
>
  {{ codeSnippet() }}
</code>
<span class="copy-status">
  @switch (cb.state()) {
    @case ('idle') { Click to copy }
    @case ('copied') { Copied! }
    @case ('error') { Failed to copy }
  }
</span>
```

This directive uses `exportAs` to expose its `state` signal and `copy()` method, allowing the template to display feedback without any component logic. The `text` input is aliased to the selector, so the value is passed directly on the `appCopyToClipboard` attribute.

---

> **Key Takeaway:** Custom attribute directives encapsulate reusable DOM behavior in a class applied via an attribute selector. Use the `host` property for event listeners and property bindings, `input()` for configuration, `output()` for custom events, `exportAs` to expose the directive's API to the template, and `DestroyRef` for cleanup.
