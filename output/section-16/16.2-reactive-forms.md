# 16.2 Reactive Forms

> Reactive forms give you explicit, programmatic control over form state using `FormControl`, `FormGroup`, `FormArray`, and `FormRecord` in your component class, with synchronous value access, observable streams, and powerful dynamic form capabilities.

---

## Setup

Reactive forms require `ReactiveFormsModule`:

```typescript
import { Component } from '@angular/core';
import { ReactiveFormsModule, FormControl, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-login',
  standalone: true,
  imports: [ReactiveFormsModule],
  templateUrl: './login.component.html'
})
export class LoginComponent {
  loginForm = new FormGroup({
    email: new FormControl('', [Validators.required, Validators.email]),
    password: new FormControl('', [Validators.required, Validators.minLength(8)])
  });
}
```

Unlike template-driven forms where the model is implicit (built from the template), reactive forms define the model explicitly in TypeScript. The template binds to that model.

## FormControl

`FormControl` is the atomic building block. It tracks the value, validation status, and user interaction state of a single form element.

### Creating a FormControl

```typescript
// Untyped (legacy)
const name = new FormControl('Alice');

// Typed (Angular 14+)
const name = new FormControl<string>('Alice');

// With validators
const email = new FormControl('', {
  validators: [Validators.required, Validators.email],
  asyncValidators: [uniqueEmailValidator],
  updateOn: 'blur'
});

// Non-nullable (resets to initial value instead of null)
const status = new FormControl('active', { nonNullable: true });
status.reset(); // value is 'active', not null
```

### Binding to the Template

Two approaches:

```html
<!-- Standalone FormControl (not inside a FormGroup) -->
<input [formControl]="name" />

<!-- Inside a FormGroup (using formControlName string) -->
<form [formGroup]="loginForm">
  <input formControlName="email" />
  <input formControlName="password" />
</form>
```

`[formControl]` takes a `FormControl` reference directly. `formControlName` takes a string key and looks up the control in the parent `FormGroup`.

### Reading and Writing Values

```typescript
const name = new FormControl('Alice');

// Read
console.log(name.value);        // 'Alice'
console.log(name.valid);        // true
console.log(name.status);       // 'VALID'
console.log(name.errors);       // null

// Write
name.setValue('Bob');            // sets to 'Bob'
name.patchValue('Charlie');     // same as setValue for FormControl
name.reset();                   // resets to null (or initial if nonNullable)
name.disable();                 // disables the control
name.enable();                  // re-enables
```

## FormGroup

`FormGroup` aggregates a fixed set of named controls. It tracks collective value, validity, and state.

### Creating a FormGroup

```typescript
const profileForm = new FormGroup({
  firstName: new FormControl('', Validators.required),
  lastName: new FormControl('', Validators.required),
  email: new FormControl('', [Validators.required, Validators.email]),
  address: new FormGroup({
    street: new FormControl(''),
    city: new FormControl(''),
    zip: new FormControl('', Validators.pattern(/^\d{5}$/))
  })
});
```

### Binding to the Template

```html
<form [formGroup]="profileForm" (ngSubmit)="onSubmit()">
  <input formControlName="firstName" placeholder="First name" />
  <input formControlName="lastName" placeholder="Last name" />
  <input formControlName="email" placeholder="Email" />

  <div formGroupName="address">
    <input formControlName="street" placeholder="Street" />
    <input formControlName="city" placeholder="City" />
    <input formControlName="zip" placeholder="ZIP" />
  </div>

  <button type="submit" [disabled]="profileForm.invalid">Save</button>
</form>
```

`[formGroup]` binds the root form group to the `<form>` element. `formGroupName` nests a child group. `formControlName` registers controls within the nearest group.

### The Form Value Object

```typescript
console.log(profileForm.value);
// {
//   firstName: 'Alice',
//   lastName: 'Smith',
//   email: 'alice@example.com',
//   address: {
//     street: '123 Main St',
//     city: 'Springfield',
//     zip: '62701'
//   }
// }
```

The value mirrors the group structure. Disabled controls are excluded from `value` but included in `getRawValue()`.

### getRawValue()

```typescript
profileForm.get('email')?.disable();

console.log(profileForm.value);
// { firstName: 'Alice', lastName: 'Smith', address: { ... } }
// email is MISSING because it is disabled

console.log(profileForm.getRawValue());
// { firstName: 'Alice', lastName: 'Smith', email: 'alice@example.com', address: { ... } }
// email is INCLUDED
```

Always use `getRawValue()` when submitting forms that may have disabled fields.

## setValue vs patchValue

### setValue

Requires you to provide values for ALL controls in the group. Throws an error if a key is missing or extra.

```typescript
// GOOD: all keys provided
profileForm.setValue({
  firstName: 'Bob',
  lastName: 'Jones',
  email: 'bob@example.com',
  address: {
    street: '456 Oak Ave',
    city: 'Portland',
    zip: '97201'
  }
});

// BAD: missing lastName - throws error
profileForm.setValue({
  firstName: 'Bob',
  email: 'bob@example.com',
  address: { street: '', city: '', zip: '' }
});
// Error: Must supply a value for form control with name: 'lastName'
```

### patchValue

Updates only the provided keys. Ignores missing keys silently.

```typescript
// Update only some fields
profileForm.patchValue({
  firstName: 'Bob',
  address: {
    city: 'Portland'
  }
});
// lastName, email, street, zip are unchanged
```

### When to Use Each

| Method | Use when |
|---|---|
| `setValue()` | Populating the entire form (e.g., loading data from API) |
| `patchValue()` | Updating a subset of fields (e.g., auto-filling some fields) |

### emitEvent Option

Both methods accept `{ emitEvent: false }` to suppress `valueChanges` and `statusChanges` emissions:

```typescript
// Update without triggering subscribers
profileForm.patchValue(
  { firstName: 'Silent' },
  { emitEvent: false }
);
```

This is critical for avoiding infinite loops when you subscribe to `valueChanges` and also set values programmatically inside the subscription.

## Accessing Controls

### get()

Navigate to any control by dot-separated path:

```typescript
const street = profileForm.get('address.street');
console.log(street?.value);

// Also works with array indices
const firstAlias = form.get('aliases.0');
```

### Direct Property Access (Typed Forms)

With typed forms (Angular 14+), access controls as properties:

```typescript
const email = profileForm.controls.email;
console.log(email.value);  // TypeScript knows this is string | null
```

### Checking Specific Errors

```typescript
const emailCtrl = profileForm.get('email');

if (emailCtrl?.hasError('required')) {
  console.log('Email is required');
}

if (emailCtrl?.hasError('email')) {
  console.log('Invalid email format');
}

// Or access the errors object directly
const errors = emailCtrl?.errors;
// { required: true } or { email: true } or null
```

## FormArray

`FormArray` manages an ordered list of controls. Unlike `FormGroup`, the controls are accessed by index, not by name.

### Creating a FormArray

```typescript
import { FormArray, FormControl, FormGroup, FormBuilder } from '@angular/forms';

@Component({ /* ... */ })
export class SkillsFormComponent {
  form = new FormGroup({
    name: new FormControl(''),
    skills: new FormArray([
      new FormControl('Angular'),
      new FormControl('TypeScript')
    ])
  });

  get skills(): FormArray {
    return this.form.get('skills') as FormArray;
  }
}
```

### Template Binding

```html
<form [formGroup]="form">
  <input formControlName="name" placeholder="Your name" />

  <div formArrayName="skills">
    <h3>Skills</h3>
    @for (skill of skills.controls; track $index) {
      <div>
        <input [formControlName]="$index" />
        <button type="button" (click)="removeSkill($index)">Remove</button>
      </div>
    }
  </div>

  <button type="button" (click)="addSkill()">Add Skill</button>
</form>
```

Note: `formControlName` takes the index (`$index`) when inside a `formArrayName`.

### Dynamic Operations

```typescript
addSkill(): void {
  this.skills.push(new FormControl('', Validators.required));
}

removeSkill(index: number): void {
  this.skills.removeAt(index);
}

insertSkill(index: number, value: string): void {
  this.skills.insert(index, new FormControl(value));
}

clearSkills(): void {
  this.skills.clear();
}

moveSkill(from: number, to: number): void {
  const ctrl = this.skills.at(from);
  this.skills.removeAt(from);
  this.skills.insert(to, ctrl);
}
```

### FormArray of FormGroups

For complex items (objects, not just strings):

```typescript
@Component({ /* ... */ })
export class OrderFormComponent {
  form = new FormGroup({
    customerId: new FormControl(''),
    items: new FormArray<FormGroup>([])
  });

  get items(): FormArray<FormGroup> {
    return this.form.get('items') as FormArray<FormGroup>;
  }

  addItem(): void {
    const itemGroup = new FormGroup({
      productId: new FormControl('', Validators.required),
      quantity: new FormControl(1, [Validators.required, Validators.min(1)]),
      price: new FormControl(0, Validators.required)
    });
    this.items.push(itemGroup);
  }

  removeItem(index: number): void {
    this.items.removeAt(index);
  }
}
```

```html
<form [formGroup]="form">
  <input formControlName="customerId" placeholder="Customer ID" />

  <div formArrayName="items">
    @for (item of items.controls; track $index) {
      <div [formGroupName]="$index">
        <input formControlName="productId" placeholder="Product" />
        <input formControlName="quantity" type="number" placeholder="Qty" />
        <input formControlName="price" type="number" placeholder="Price" />
        <button type="button" (click)="removeItem($index)">Remove</button>
      </div>
    }
  </div>

  <button type="button" (click)="addItem()">Add Item</button>
</form>
```

## FormRecord

`FormRecord` is a `FormGroup` variant for dynamic, open-ended key sets where all controls share the same type. Added in Angular 14.

```typescript
import { FormRecord, FormControl } from '@angular/forms';

// All values are FormControl<boolean | null>
const permissions = new FormRecord<FormControl<boolean | null>>({
  read: new FormControl(true),
  write: new FormControl(false)
});

// Dynamically add/remove
permissions.addControl('delete', new FormControl(false));
permissions.removeControl('write');
```

### FormRecord vs FormGroup vs FormArray

| Feature | `FormGroup` | `FormRecord` | `FormArray` |
|---|---|---|---|
| Keys | Fixed, known at compile time | Dynamic, can add/remove | Indexed (0, 1, 2, ...) |
| Value types | Each control can differ | All controls same type | All controls same type |
| Access | By name (`controls.email`) | By name | By index (`at(0)`) |
| Use case | Static forms | Dynamic key-value pairs | Lists of items |

```typescript
// FormRecord: dynamic permissions
const perms = new FormRecord<FormControl<boolean>>({});
userPermissions.forEach(p => {
  perms.addControl(p.name, new FormControl(p.granted));
});

// FormGroup: static login form
const login = new FormGroup({
  email: new FormControl(''),
  password: new FormControl('')
});

// FormArray: list of phone numbers
const phones = new FormArray([
  new FormControl('+1234567890')
]);
```

## FormBuilder

`FormBuilder` provides shorthand methods to create controls, groups, and arrays:

```typescript
import { Component, inject } from '@angular/core';
import { FormBuilder, Validators, ReactiveFormsModule } from '@angular/forms';

@Component({
  selector: 'app-profile',
  standalone: true,
  imports: [ReactiveFormsModule],
  templateUrl: './profile.component.html'
})
export class ProfileComponent {
  private fb = inject(FormBuilder);

  profileForm = this.fb.group({
    // Shorthand: [initialValue, validators, asyncValidators]
    firstName: ['', Validators.required],
    lastName: ['', Validators.required],
    email: ['', [Validators.required, Validators.email]],

    // Nested group
    address: this.fb.group({
      street: [''],
      city: [''],
      zip: ['', Validators.pattern(/^\d{5}$/)]
    }),

    // FormArray
    aliases: this.fb.array([
      this.fb.control('')
    ])
  });
}
```

### NonNullableFormBuilder

Creates controls that reset to their initial value instead of `null`:

```typescript
private fb = inject(NonNullableFormBuilder);

form = this.fb.group({
  name: ['Alice', Validators.required],
  // name.reset() sets value back to 'Alice', not null
  role: ['user']
  // role.reset() sets value back to 'user', not null
});
```

Alternatively, use `.nonNullable` on the standard `FormBuilder`:

```typescript
private fb = inject(FormBuilder);

form = this.fb.nonNullable.group({
  name: ['Alice'],
  role: ['user']
});
```

## Observable Streams

### valueChanges

Emits the current value whenever any control in the group changes:

```typescript
// Listen to entire form value
this.form.valueChanges
  .pipe(takeUntilDestroyed())
  .subscribe(value => {
    console.log('Form value:', value);
  });

// Listen to a single control
this.form.get('email')?.valueChanges
  .pipe(
    debounceTime(300),
    distinctUntilChanged(),
    takeUntilDestroyed()
  )
  .subscribe(email => {
    console.log('Email changed:', email);
  });
```

### statusChanges

Emits the validation status (`'VALID'`, `'INVALID'`, `'PENDING'`, `'DISABLED'`):

```typescript
this.form.statusChanges
  .pipe(takeUntilDestroyed())
  .subscribe(status => {
    this.canSubmit = status === 'VALID';
  });
```

### Converting to Signals

```typescript
import { toSignal } from '@angular/core/rxjs-interop';

formValue = toSignal(this.form.valueChanges, { initialValue: this.form.value });
formStatus = toSignal(this.form.statusChanges, { initialValue: this.form.status });
```

### events (Angular 18+)

The `events` observable provides a unified stream of all control events:

```typescript
this.form.events
  .pipe(takeUntilDestroyed())
  .subscribe(event => {
    if (event instanceof TouchedChangeEvent) {
      console.log('Touched state changed:', event.touched);
    }
    if (event instanceof PristineChangeEvent) {
      console.log('Pristine state changed:', event.pristine);
    }
    if (event instanceof StatusChangeEvent) {
      console.log('Status changed:', event.status);
    }
    if (event instanceof ValueChangeEvent) {
      console.log('Value changed:', event.value);
    }
  });
```

## updateOn Options

Control when validation and value updates trigger:

```typescript
// Per control
const email = new FormControl('', {
  validators: Validators.required,
  updateOn: 'blur'       // 'change' (default) | 'blur' | 'submit'
});

// Per group (applies to all children)
const form = new FormGroup({
  email: new FormControl(''),
  password: new FormControl('')
}, { updateOn: 'blur' });

// FormBuilder shorthand
form = this.fb.group({
  email: [''],
  password: ['']
}, { updateOn: 'blur' });
```

| Mode | When it triggers |
|---|---|
| `'change'` | On every input event (default) |
| `'blur'` | When the control loses focus |
| `'submit'` | When the form is submitted |

Child controls can override the parent's `updateOn`:

```typescript
const form = new FormGroup({
  search: new FormControl('', { updateOn: 'change' }),  // overrides parent
  email: new FormControl('')                             // inherits 'blur' from parent
}, { updateOn: 'blur' });
```

## Disabling and Enabling Controls

```typescript
// Disable a single control
this.form.get('email')?.disable();

// Disable the entire form
this.form.disable();

// Enable
this.form.get('email')?.enable();
this.form.enable();

// Check disabled state
this.form.get('email')?.disabled;  // true/false

// Disable in template (use the FormControl API, NOT the disabled attribute)
```

```html
<!-- BAD: Angular warns about mixing disabled attribute with reactive forms -->

<!-- GOOD: Disable via the FormControl -->
```

```typescript
// In component
email = new FormControl({ value: '', disabled: true });

// Or dynamically
this.form.get('email')?.disable();
```

## Form Submission

```typescript
@Component({
  selector: 'app-profile',
  template: `
    <form [formGroup]="profileForm" (ngSubmit)="onSubmit()">
      <input formControlName="firstName" />
      <input formControlName="email" />

      @if (profileForm.get('email')?.hasError('required') &&
           profileForm.get('email')?.touched) {
        <span class="error">Email is required</span>
      }

      <button type="submit" [disabled]="profileForm.invalid">Save</button>
    </form>
  `
})
export class ProfileComponent {
  private fb = inject(FormBuilder);

  profileForm = this.fb.group({
    firstName: ['', Validators.required],
    email: ['', [Validators.required, Validators.email]]
  });

  onSubmit(): void {
    if (this.profileForm.invalid) {
      this.profileForm.markAllAsTouched();
      return;
    }

    const data = this.profileForm.getRawValue();
    // Submit data to API
    console.log(data);
  }
}
```

### Reset After Submission

```typescript
onSubmit(): void {
  if (this.profileForm.valid) {
    this.saveData(this.profileForm.getRawValue());
    this.profileForm.reset();
    // Or reset to specific values:
    this.profileForm.reset({ firstName: '', email: '' });
  }
}
```

## Reacting to Value Changes

### Auto-Save

```typescript
constructor() {
  this.form.valueChanges
    .pipe(
      debounceTime(1000),
      distinctUntilChanged((a, b) => JSON.stringify(a) === JSON.stringify(b)),
      switchMap(value => this.saveService.autoSave(value)),
      takeUntilDestroyed()
    )
    .subscribe();
}
```

### Conditional Validation

```typescript
constructor() {
  this.form.get('contactMethod')?.valueChanges
    .pipe(takeUntilDestroyed())
    .subscribe(method => {
      const phoneCtrl = this.form.get('phone');
      const emailCtrl = this.form.get('email');

      if (method === 'phone') {
        phoneCtrl?.setValidators([Validators.required, Validators.pattern(/^\d{10}$/)]);
        emailCtrl?.clearValidators();
      } else {
        emailCtrl?.setValidators([Validators.required, Validators.email]);
        phoneCtrl?.clearValidators();
      }

      phoneCtrl?.updateValueAndValidity();
      emailCtrl?.updateValueAndValidity();
    });
}
```

**Important:** After calling `setValidators()` or `clearValidators()`, you must call `updateValueAndValidity()` to re-evaluate the control's status. Without it, the old validators remain in effect until the next value change.

### Avoiding Infinite Loops

```typescript
// BAD: setValue triggers valueChanges, which calls setValue again
this.form.valueChanges.subscribe(value => {
  this.form.patchValue({ computed: value.a + value.b }); // infinite loop!
});

// GOOD: Suppress events when setting programmatically
this.form.valueChanges.subscribe(value => {
  this.form.patchValue(
    { computed: value.a + value.b },
    { emitEvent: false }
  );
});
```

## Error Display Pattern

A reusable pattern for showing validation errors:

```typescript
@Component({
  selector: 'app-registration',
  template: `
    <form [formGroup]="form" (ngSubmit)="submit()">
      <div>
        <label for="username">Username</label>
        <input id="username" formControlName="username" />
        @if (showError('username')) {
          <div class="error">
            @if (form.get('username')?.hasError('required')) { Username is required. }
            @if (form.get('username')?.hasError('minlength')) {
              Minimum {{ form.get('username')?.getError('minlength').requiredLength }} characters.
            }
          </div>
        }
      </div>

      <button type="submit">Register</button>
    </form>
  `
})
export class RegistrationComponent {
  private fb = inject(FormBuilder);
  submitted = false;

  form = this.fb.group({
    username: ['', [Validators.required, Validators.minLength(3)]]
  });

  showError(controlName: string): boolean {
    const ctrl = this.form.get(controlName);
    return !!(ctrl?.invalid && (ctrl.touched || this.submitted));
  }

  submit(): void {
    this.submitted = true;
    if (this.form.invalid) {
      this.form.markAllAsTouched();
      return;
    }
    // Process form
  }
}
```

## Common Mistakes

- **Using `setValue()` with partial data.** `setValue()` requires values for every control in the group. For partial updates, use `patchValue()`. Getting them confused causes runtime errors or missed fields.

- **Forgetting `updateValueAndValidity()` after `setValidators()`.** Adding or removing validators dynamically does not re-evaluate the control. You must call `updateValueAndValidity()` afterward.

- **Creating infinite loops with `valueChanges`.** Subscribing to `valueChanges` and then calling `setValue`/`patchValue` inside the subscription triggers the subscription again. Use `{ emitEvent: false }` to break the cycle.

- **Using `form.value` when controls are disabled.** Disabled controls are excluded from `form.value`. Use `form.getRawValue()` to include all controls regardless of disabled state.

- **Using the HTML `disabled` attribute on reactive form inputs.** Angular logs a warning because it conflicts with the reactive model. Always disable/enable controls programmatically via the `FormControl` API.

- **Not unsubscribing from `valueChanges`.** `valueChanges` does not complete automatically. Use `takeUntilDestroyed()` in the constructor or `takeUntil()` with a destroy subject.

- **Accessing `FormArray` controls without a getter.** Typing `this.form.get('items')` everywhere without a typed getter leads to repetitive casting. Create a getter: `get items(): FormArray { return this.form.get('items') as FormArray; }`.

---

## Best Practices

- **Use `NonNullableFormBuilder` for forms that should never have null values.** Controls created with it reset to their initial value, not `null`, which matches most real-world form behavior.

- **Use `getRawValue()` for form submission.** It includes disabled controls, giving you the complete form value.

- **Create getters for `FormArray` access.** This avoids casting throughout your template and component code.

- **Use `emitEvent: false` for programmatic updates.** When you set values inside a `valueChanges` handler, or when bulk-loading data, suppress events to avoid cascading updates.

- **Use `markAllAsTouched()` on submit.** This reveals all validation errors at once when the user tries to submit an invalid form.

- **Prefer `fb.group()` over `new FormGroup()`.** The `FormBuilder` syntax is more concise and easier to read, especially for large forms.

- **Convert `valueChanges` to signals with `toSignal()`.** This bridges reactive forms with Angular's signal ecosystem, letting you use form values in `computed()` and `effect()` contexts.

## Real-World Use Cases

### 1. Edit Form with API Data Loading

```typescript
@Component({
  selector: 'app-user-edit',
  standalone: true,
  imports: [ReactiveFormsModule],
  template: `
    <form [formGroup]="form" (ngSubmit)="save()">
      <input formControlName="name" />
      <input formControlName="email" />

      <div formGroupName="preferences">
        <label>
          <input type="checkbox" formControlName="darkMode" />
          Dark Mode
        </label>
        <select formControlName="language">
          <option value="en">English</option>
          <option value="es">Spanish</option>
          <option value="fr">French</option>
        </select>
      </div>

      <button type="submit" [disabled]="form.invalid || form.pristine">
        Save Changes
      </button>
    </form>
  `
})
export class UserEditComponent {
  private fb = inject(NonNullableFormBuilder);
  private userService = inject(UserService);
  private route = inject(ActivatedRoute);

  userId = input.required<string>();

  form = this.fb.group({
    name: ['', Validators.required],
    email: ['', [Validators.required, Validators.email]],
    preferences: this.fb.group({
      darkMode: [false],
      language: ['en']
    })
  });

  constructor() {
    // Load user data into the form
    effect(() => {
      const id = this.userId();
      this.userService.getById(id).subscribe(user => {
        this.form.patchValue(user);
        this.form.markAsPristine();   // mark as unchanged after loading
      });
    });
  }

  save(): void {
    if (this.form.valid) {
      const data = this.form.getRawValue();
      this.userService.update(this.userId(), data).subscribe(() => {
        this.form.markAsPristine();  // reset dirty state after save
      });
    }
  }
}
```

### 2. Dynamic Invoice Form with FormArray

```typescript
@Component({
  selector: 'app-invoice',
  standalone: true,
  imports: [ReactiveFormsModule, CurrencyPipe],
  template: `
    <form [formGroup]="invoiceForm" (ngSubmit)="submit()">
      <input formControlName="clientName" placeholder="Client" />
      <input formControlName="invoiceDate" type="date" />

      <table formArrayName="lineItems">
        <thead>
          <tr>
            <th>Description</th>
            <th>Quantity</th>
            <th>Unit Price</th>
            <th>Total</th>
            <th></th>
          </tr>
        </thead>
        <tbody>
          @for (item of lineItems.controls; track $index) {
            <tr [formGroupName]="$index">
              <td><input formControlName="description" /></td>
              <td><input formControlName="quantity" type="number" /></td>
              <td><input formControlName="unitPrice" type="number" /></td>
              <td>{{ itemTotal($index) | currency }}</td>
              <td>
                <button type="button" (click)="removeItem($index)">Remove</button>
              </td>
            </tr>
          }
        </tbody>
      </table>

      <button type="button" (click)="addItem()">Add Line Item</button>

      <div class="total">
        <strong>Grand Total: {{ grandTotal() | currency }}</strong>
      </div>

      <button type="submit" [disabled]="invoiceForm.invalid">
        Create Invoice
      </button>
    </form>
  `
})
export class InvoiceComponent {
  private fb = inject(NonNullableFormBuilder);

  invoiceForm = this.fb.group({
    clientName: ['', Validators.required],
    invoiceDate: [new Date().toISOString().split('T')[0]],
    lineItems: this.fb.array<FormGroup>([])
  });

  get lineItems(): FormArray<FormGroup> {
    return this.invoiceForm.get('lineItems') as FormArray<FormGroup>;
  }

  addItem(): void {
    this.lineItems.push(this.fb.group({
      description: ['', Validators.required],
      quantity: [1, [Validators.required, Validators.min(1)]],
      unitPrice: [0, [Validators.required, Validators.min(0)]]
    }));
  }

  removeItem(index: number): void {
    this.lineItems.removeAt(index);
  }

  itemTotal(index: number): number {
    const item = this.lineItems.at(index);
    return (item.get('quantity')?.value ?? 0) * (item.get('unitPrice')?.value ?? 0);
  }

  grandTotal(): number {
    return this.lineItems.controls.reduce(
      (sum, _, i) => sum + this.itemTotal(i), 0
    );
  }

  submit(): void {
    if (this.invoiceForm.valid) {
      console.log('Invoice:', this.invoiceForm.getRawValue());
    }
  }
}
```

### 3. Multi-Step Form with Shared FormGroup

```typescript
@Component({
  selector: 'app-wizard',
  standalone: true,
  imports: [ReactiveFormsModule],
  template: `
    @switch (currentStep) {
      @case (1) {
        <div [formGroup]="form">
          <h2>Step 1: Personal Info</h2>
          <input formControlName="name" placeholder="Full name" />
          <input formControlName="email" placeholder="Email" />
          <button (click)="nextStep()" [disabled]="!isStepValid(1)">Next</button>
        </div>
      }
      @case (2) {
        <div [formGroup]="form">
          <h2>Step 2: Address</h2>
          <div formGroupName="address">
            <input formControlName="street" placeholder="Street" />
            <input formControlName="city" placeholder="City" />
            <input formControlName="zip" placeholder="ZIP" />
          </div>
          <button (click)="prevStep()">Back</button>
          <button (click)="nextStep()" [disabled]="!isStepValid(2)">Next</button>
        </div>
      }
      @case (3) {
        <div>
          <h2>Step 3: Review</h2>
          <pre>{{ form.getRawValue() | json }}</pre>
          <button (click)="prevStep()">Back</button>
          <button (click)="submit()" [disabled]="form.invalid">Submit</button>
        </div>
      }
    }
  `
})
export class WizardComponent {
  private fb = inject(NonNullableFormBuilder);
  currentStep = 1;

  form = this.fb.group({
    name: ['', Validators.required],
    email: ['', [Validators.required, Validators.email]],
    address: this.fb.group({
      street: ['', Validators.required],
      city: ['', Validators.required],
      zip: ['', [Validators.required, Validators.pattern(/^\d{5}$/)]]
    })
  });

  isStepValid(step: number): boolean {
    switch (step) {
      case 1:
        return this.form.get('name')!.valid && this.form.get('email')!.valid;
      case 2:
        return (this.form.get('address') as FormGroup).valid;
      default:
        return this.form.valid;
    }
  }

  nextStep(): void { this.currentStep++; }
  prevStep(): void { this.currentStep--; }

  submit(): void {
    if (this.form.valid) {
      console.log('Wizard complete:', this.form.getRawValue());
    }
  }
}
```

---

> **Key Takeaway:** Reactive forms give you full programmatic control over form state through `FormControl`, `FormGroup`, `FormArray`, and `FormRecord`, with observable streams (`valueChanges`, `statusChanges`) for reacting to changes; always use `getRawValue()` for submission, `patchValue()` for partial updates, and `{ emitEvent: false }` to prevent cascading updates in `valueChanges` handlers.
