# 17.1 Built-in Pipes

> Pipes transform displayed values in templates without mutating the underlying data. Angular ships with a set of built-in pipes covering formatting, casing, slicing, iteration, and async data unwrapping.

---

## How Pipes Work

A pipe takes a value, processes it, and returns a transformed result. Pipes are used in template expressions with the `|` operator.

```html
<!-- Basic syntax -->
{{ value | pipeName }}

<!-- With parameters -->
{{ value | pipeName:param1:param2 }}

<!-- Chaining pipes -->
{{ value | pipeOne | pipeTwo }}
```

All built-in pipes are standalone and can be imported directly into your component:

```typescript
import { Component } from '@angular/core';
import { DatePipe, CurrencyPipe, UpperCasePipe } from '@angular/common';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [DatePipe, CurrencyPipe, UpperCasePipe],
  template: `
    <p>{{ today | date:'fullDate' }}</p>
    <p>{{ price | currency }}</p>
    <p>{{ name | uppercase }}</p>
  `
})
export class ExampleComponent {
  today = new Date();
  price = 42.5;
  name = 'angular';
}
```

Pipes are pure by default. Angular only re-evaluates a pure pipe when the input value changes by reference (for objects) or by value (for primitives). This makes them highly performant compared to method calls in templates.

## String Transformation Pipes

### UpperCasePipe

Transforms text to all uppercase.

```typescript
import { UpperCasePipe } from '@angular/common';

// In template:
{{ 'hello world' | uppercase }}
// Output: HELLO WORLD
```

Handles `null` and `undefined` gracefully by returning an empty string.

```html
{{ nullValue | uppercase }}
<!-- Output: (empty string) -->
```

### LowerCasePipe

Transforms text to all lowercase.

```typescript
import { LowerCasePipe } from '@angular/common';

{{ 'HELLO WORLD' | lowercase }}
// Output: hello world
```

### TitleCasePipe

Capitalizes the first letter of each word and lowercases the rest.

```typescript
import { TitleCasePipe } from '@angular/common';

{{ 'hello world' | titlecase }}
// Output: Hello World

{{ 'hELLO wORLD' | titlecase }}
// Output: Hello World

{{ 'one,two,three' | titlecase }}
// Output: One,two,three
```

Word boundaries are whitespace characters only. Commas, hyphens, and other punctuation do not create word boundaries.

```html
{{ 'mary-jane watson' | titlecase }}
<!-- Output: Mary-jane Watson -->
<!-- Note: "jane" is NOT capitalized because hyphen is not a word boundary -->
```

## Number Formatting Pipes

### DecimalPipe

Formats a number as a decimal string according to locale rules.

```typescript
import { DecimalPipe } from '@angular/common';
```

**Syntax:** `{{ value | number[:digitsInfo[:locale]] }}`

**digitsInfo format:** `{minIntegerDigits}.{minFractionDigits}-{maxFractionDigits}`

```html
<!-- Default formatting -->
{{ 3.14159 | number }}
<!-- Output: 3.142 (defaults to 1.0-3) -->

<!-- Minimum 3 integer digits, exactly 2 fraction digits -->
{{ 3.14159 | number:'3.2-2' }}
<!-- Output: 003.14 -->

<!-- At least 1 integer digit, 0 to 0 fraction digits (rounds to integer) -->
{{ 3.6 | number:'1.0-0' }}
<!-- Output: 4 -->

<!-- At least 4 fraction digits, at most 6 -->
{{ 3.14 | number:'1.4-6' }}
<!-- Output: 3.1400 -->
```

The pipe uses the current locale for thousands separators and decimal points:

```html
<!-- en-US locale -->
{{ 1234567.89 | number }}
<!-- Output: 1,234,567.89 -->

<!-- de-DE locale would produce: 1.234.567,89 -->
```

**Rounding behavior:** The pipe uses "round half to even" (banker's rounding), not "round half up":

```html
{{ 2.5 | number:'1.0-0' }}
<!-- Output: 2 (rounds to even) -->

{{ 3.5 | number:'1.0-0' }}
<!-- Output: 4 (rounds to even) -->

{{ 2.55 | number:'1.1-1' }}
<!-- Output: 2.6 -->
```

### PercentPipe

Transforms a number to a percentage string. The input value is treated as a fraction (0-1), not a percentage.

```typescript
import { PercentPipe } from '@angular/common';
```

**Syntax:** `{{ value | percent[:digitsInfo[:locale]] }}`

```html
{{ 0.259 | percent }}
<!-- Output: 26% -->

{{ 0.259 | percent:'1.2-2' }}
<!-- Output: 25.90% -->

{{ 1.5 | percent }}
<!-- Output: 150% -->
```

Common mistake: passing 25 expecting "25%":

```html
<!-- BAD: This outputs 2,500% -->
{{ 25 | percent }}

<!-- GOOD: Pass the decimal fraction -->
{{ 0.25 | percent }}
<!-- Output: 25% -->
```

### CurrencyPipe

Formats a number as currency.

```typescript
import { CurrencyPipe } from '@angular/common';
```

**Syntax:** `{{ value | currency[:currencyCode[:display[:digitsInfo[:locale]]]] }}`

```html
<!-- Default (uses locale's default currency) -->
{{ 42.5 | currency }}
<!-- Output: $42.50 (en-US locale) -->

<!-- Specify currency code -->
{{ 42.5 | currency:'EUR' }}
<!-- Output: EUR42.50 -->

<!-- Use currency symbol -->
{{ 42.5 | currency:'EUR':'symbol' }}
<!-- Output: E42.50 -->

<!-- Use narrow symbol ($ instead of US$, etc.) -->
{{ 42.5 | currency:'USD':'symbol-narrow' }}
<!-- Output: $42.50 -->

<!-- Custom digit formatting -->
{{ 42.5 | currency:'USD':'symbol':'1.0-0' }}
<!-- Output: $43 -->

<!-- Empty string for display hides the symbol -->
{{ 42.5 | currency:'USD':'' }}
<!-- Output: 42.50 -->
```

**Display parameter options:**

| Value | Description | Example (USD) |
|-------|-------------|---------------|
| `'code'` | ISO 4217 currency code | `USD42.50` |
| `'symbol'` | Currency symbol | `$42.50` |
| `'symbol-narrow'` | Narrow symbol variant | `$42.50` |
| A string | Custom prefix | `'US Dollar '42.50` |
| `''` | No symbol | `42.50` |

## Date Formatting

### DatePipe

The most feature-rich built-in pipe. Formats a `Date`, a number (milliseconds since epoch), or an ISO date string.

```typescript
import { DatePipe } from '@angular/common';
```

**Syntax:** `{{ value | date[:format[:timezone[:locale]]] }}`

#### Pre-defined Format Options

```html
{{ today | date:'short' }}      <!-- 3/15/26, 12:00 AM -->
{{ today | date:'medium' }}     <!-- Mar 15, 2026, 12:00:00 AM -->
{{ today | date:'long' }}       <!-- March 15, 2026 at 12:00:00 AM GMT+3 -->
{{ today | date:'full' }}       <!-- Sunday, March 15, 2026 at 12:00:00 AM ... -->

{{ today | date:'shortDate' }}  <!-- 3/15/26 -->
{{ today | date:'mediumDate' }} <!-- Mar 15, 2026 -->
{{ today | date:'longDate' }}   <!-- March 15, 2026 -->
{{ today | date:'fullDate' }}   <!-- Sunday, March 15, 2026 -->

{{ today | date:'shortTime' }}  <!-- 12:00 AM -->
{{ today | date:'mediumTime' }} <!-- 12:00:00 AM -->
{{ today | date:'longTime' }}   <!-- 12:00:00 AM GMT+3 -->
{{ today | date:'fullTime' }}   <!-- 12:00:00 AM GMT+03:00 -->
```

#### Custom Format Strings

| Symbol | Meaning | Example |
|--------|---------|---------|
| `y` | Year (numeric) | `2026` |
| `yy` | Year (2-digit) | `26` |
| `M` | Month (numeric) | `3` |
| `MM` | Month (2-digit) | `03` |
| `MMM` | Month (abbreviated) | `Mar` |
| `MMMM` | Month (full) | `March` |
| `d` | Day of month | `5` |
| `dd` | Day of month (2-digit) | `05` |
| `E` | Day of week (abbreviated) | `Sun` |
| `EEEE` | Day of week (full) | `Sunday` |
| `h` | Hour (1-12) | `1` |
| `hh` | Hour (1-12, 2-digit) | `01` |
| `H` | Hour (0-23) | `13` |
| `HH` | Hour (0-23, 2-digit) | `13` |
| `m` | Minute | `5` |
| `mm` | Minute (2-digit) | `05` |
| `s` | Second | `9` |
| `ss` | Second (2-digit) | `09` |
| `a` | AM/PM | `AM` |
| `z` | Timezone abbreviation | `GMT+3` |
| `Z` | Timezone offset | `+0300` |

```html
{{ today | date:'yyyy-MM-dd' }}
<!-- Output: 2026-03-15 -->

{{ today | date:'dd/MM/yyyy HH:mm' }}
<!-- Output: 15/03/2026 00:00 -->

{{ today | date:'EEEE, MMMM d, y' }}
<!-- Output: Sunday, March 15, 2026 -->

{{ today | date:'h:mm a' }}
<!-- Output: 12:00 AM -->
```

#### Timezone Handling

```html
<!-- Specify timezone offset -->
{{ today | date:'short':'+0000' }}
<!-- Forces UTC display -->

{{ today | date:'short':'America/New_York' }}
<!-- IANA timezone name (supported in modern browsers) -->

{{ today | date:'HH:mm z':'+0530' }}
<!-- Output: 05:30 GMT+5:30 -->
```

#### Setting Default Options

Use `DATE_PIPE_DEFAULT_OPTIONS` to configure defaults application-wide:

```typescript
import { DATE_PIPE_DEFAULT_OPTIONS } from '@angular/common';

// In your app config or providers
providers: [
  {
    provide: DATE_PIPE_DEFAULT_OPTIONS,
    useValue: {
      dateFormat: 'mediumDate',
      timezone: '+0000'     // Force UTC
    }
  }
]
```

#### Accepted Input Types

```html
<!-- Date object -->
{{ dateObj | date }}

<!-- ISO string -->
{{ '2026-03-15T10:30:00Z' | date:'medium' }}

<!-- Milliseconds since epoch -->
{{ 1773784200000 | date:'medium' }}

<!-- null/undefined produce no output -->
{{ nullDate | date }}
```

**Gotcha:** Passing a date string without timezone info gets interpreted in the browser's local timezone:

```html
<!-- This behaves differently in different timezones! -->
{{ '2026-03-15' | date:'fullDate' }}

<!-- Safer: include timezone info -->
{{ '2026-03-15T00:00:00Z' | date:'fullDate':'+0000' }}
```

## Collection Pipes

### SlicePipe

Returns a subset of an array or string. Works like `Array.prototype.slice()`.

```typescript
import { SlicePipe } from '@angular/common';
```

**Syntax:** `{{ value | slice:start[:end] }}`

```html
<!-- Array slicing -->
@for (item of items | slice:0:3; track item) {
  <li>{{ item }}</li>
}
<!-- Shows first 3 items -->

<!-- String slicing -->
{{ 'Hello World' | slice:0:5 }}
<!-- Output: Hello -->

<!-- Negative indices (count from end) -->
{{ 'Hello World' | slice:-5 }}
<!-- Output: World -->

<!-- Negative start, negative end -->
@for (item of items | slice:-3; track item) {
  <li>{{ item }}</li>
}
<!-- Shows last 3 items -->
```

**Important:** SlicePipe creates a new array/string on every change detection cycle if used with `@for`. For large lists, consider slicing in the component class or using `computed()`:

```typescript
// Better approach for large lists
items = signal(['a', 'b', 'c', 'd', 'e']);
visibleItems = computed(() => this.items().slice(0, 3));
```

### KeyValuePipe

Transforms an `Object` or `Map` into an array of key-value pairs for iteration.

```typescript
import { KeyValuePipe } from '@angular/common';
```

```typescript
@Component({
  standalone: true,
  imports: [KeyValuePipe],
  template: `
    @for (entry of user | keyvalue; track entry.key) {
      <div>{{ entry.key }}: {{ entry.value }}</div>
    }
  `
})
export class UserComponent {
  user = {
    name: 'Alice',
    age: 30,
    role: 'Developer'
  };
}
// Output:
// age: 30
// name: Alice
// role: Developer
```

**Default sorting:** KeyValuePipe sorts entries by key alphabetically. To preserve original order or use custom sorting:

```html
<!-- Preserve insertion order (pass null to disable sorting) -->
@for (entry of user | keyvalue:null; track entry.key) {
  <div>{{ entry.key }}: {{ entry.value }}</div>
}
```

For Maps, passing `null` preserves insertion order via `Map.entries()`. For objects, it uses `Object.keys()` order.

You can also provide a custom comparator:

```typescript
// In the component class

// Or sort by value
sortByValue = (a: KeyValue<string, number>, b: KeyValue<string, number>) => {
  return a.value - b.value;
};
```

**Works with Maps:**

```typescript
config = new Map<string, boolean>([
  ['darkMode', true],
  ['notifications', false],
  ['autoSave', true]
]);
```

```html
@for (entry of config | keyvalue; track entry.key) {
  <label>
    {{ entry.key }}: {{ entry.value ? 'On' : 'Off' }}
  </label>
}
```

**Gotcha:** KeyValuePipe is impure. It runs on every change detection cycle. Avoid using it with large objects in performance-sensitive views.

## Debugging Pipe

### JsonPipe

Serializes a value to JSON. Invaluable for debugging.

```typescript
import { JsonPipe } from '@angular/common';
```

```html
<pre>{{ complexObject | json }}</pre>
```

```typescript
user = {
  name: 'Alice',
  roles: ['admin', 'editor'],
  settings: {
    theme: 'dark',
    language: 'en'
  }
};
```

```html
<pre>{{ user | json }}</pre>
<!--
{
  "name": "Alice",
  "roles": [
    "admin",
    "editor"
  ],
  "settings": {
    "theme": "dark",
    "language": "en"
  }
}
-->
```

Use it to debug signal values:

```html
<pre>{{ mySignal() | json }}</pre>
```

**Gotchas:**
- `JsonPipe` is an **impure pipe**. It re-evaluates on every change detection cycle. Use it only for debugging, never in production templates.
- It uses `JSON.stringify()` internally. It cannot serialize values with circular references, `BigInt`, `Symbol`, or `undefined` properties.

```typescript
// This will throw an error in the template
circular: any = {};
constructor() {
  this.circular.self = this.circular;
}
```

## Async Data Pipe

### AsyncPipe

Subscribes to an `Observable` or `Promise` and returns the latest emitted value. Automatically unsubscribes when the component is destroyed.

```typescript
import { AsyncPipe } from '@angular/common';
```

#### With Observables

```typescript
import { interval } from 'rxjs';

@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    <p>Timer: {{ timer$ | async }}</p>
  `
})
export class TimerComponent {
  timer$ = interval(1000);
}
```

#### With HTTP Requests

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (users$ | async; as users) {
      @for (user of users; track user.id) {
        <div>{{ user.name }}</div>
      }
    } @else {
      <p>Loading...</p>
    }
  `
})
export class UserListComponent {
  private http = inject(HttpClient);
  users$ = this.http.get<User[]>('/api/users');
}
```

#### AsyncPipe with Signals: When to Use Each

In Angular 21, signals often replace the need for `AsyncPipe`:

```typescript
// Observable approach (still valid)
@Component({
  imports: [AsyncPipe],
  template: `<p>{{ data$ | async }}</p>`
})
export class ObsComponent {
  data$ = this.http.get<string>('/api/data');
}

// Signal approach (preferred for new code)
@Component({
  template: `<p>{{ data() }}</p>`
})
export class SigComponent {
  private http = inject(HttpClient);
  data = toSignal(this.http.get<string>('/api/data'), { initialValue: '' });
}
```

Use `AsyncPipe` when:
- You are working with existing Observable-heavy code
- You need the Observable stream for operators (retry, debounce, etc.)
- The component template is the only consumer of the data

Use `toSignal()` when:
- You want to use the value in `computed()` or `effect()`
- Multiple parts of the component need the same data
- You prefer the signal-based architecture

#### Multiple Subscriptions Gotcha

Each `| async` creates a separate subscription:

```html
<!-- BAD: Two separate HTTP requests! -->
<p>Name: {{ (user$ | async)?.name }}</p>
<p>Email: {{ (user$ | async)?.email }}</p>
```

Fix with `@if ... as` or `shareReplay`:

```html
<!-- GOOD: Single subscription using @if...as -->
@if (user$ | async; as user) {
  <p>Name: {{ user.name }}</p>
  <p>Email: {{ user.email }}</p>
}
```

```typescript
// Alternative: use shareReplay in the component
user$ = this.http.get<User>('/api/user/1').pipe(
  shareReplay(1)
);
```

#### Null Handling

`AsyncPipe` returns `null` before the first emission. Account for this:

```html
<!-- The ?. is needed because async returns null initially -->
{{ (config$ | async)?.theme }}

<!-- Or use @if to guard -->
@if (config$ | async; as config) {
  <p>Theme: {{ config.theme }}</p>
}
```

## Internationalization Pipes

### I18nPluralPipe

Maps a number to a plural string based on ICU rules.

```typescript
import { I18nPluralPipe } from '@angular/common';
```

```typescript
@Component({
  standalone: true,
  imports: [I18nPluralPipe],
  template: `
    <p>{{ messageCount | i18nPlural:messageMapping }}</p>
  `
})
export class NotificationComponent {
  messageCount = 0;

  messageMapping: { [k: string]: string } = {
    '=0': 'No messages',
    '=1': 'One message',
    'other': '# messages'   // # is replaced with the actual number
  };
}
```

```html
<!-- messageCount = 0  -> "No messages" -->
<!-- messageCount = 1  -> "One message" -->
<!-- messageCount = 5  -> "5 messages" -->
<!-- messageCount = 42 -> "42 messages" -->
```

You can add more specific cases:

```typescript
itemMapping = {
  '=0': 'Your cart is empty',
  '=1': 'You have 1 item in your cart',
  '=12': 'You have a dozen items!',       // exact match
  'other': 'You have # items in your cart'
};
```

### I18nSelectPipe

Selects a string from a map based on a string key. Useful for gender, status, or role-based display.

```typescript
import { I18nSelectPipe } from '@angular/common';
```

```typescript
@Component({
  standalone: true,
  imports: [I18nSelectPipe],
  template: `
    <p>{{ role | i18nSelect:roleMapping }}</p>
  `
})
export class RoleComponent {
  role = 'admin';

  roleMapping: { [k: string]: string } = {
    'admin': 'Administrator',
    'editor': 'Content Editor',
    'viewer': 'Read-Only Viewer',
    'other': 'Unknown Role'        // fallback
  };
}
```

```html
<!-- role = 'admin'   -> "Administrator" -->
<!-- role = 'editor'  -> "Content Editor" -->
<!-- role = 'unknown' -> "Unknown Role" (falls back to 'other') -->
```

The `'other'` key acts as the default fallback. If neither the key nor `'other'` matches, the pipe returns an empty string.

## Using Pipes Programmatically

You can inject pipes into component or service code using `inject()`:

```typescript
import { DatePipe, CurrencyPipe } from '@angular/common';

@Component({
  standalone: true,
  imports: [DatePipe],
  providers: [DatePipe, CurrencyPipe], // Must be provided to inject
  template: `<p>{{ formattedDate }}</p>`
})
export class ReportComponent {
  private datePipe = inject(DatePipe);
  private currencyPipe = inject(CurrencyPipe);

  formattedDate = this.datePipe.transform(new Date(), 'fullDate');

  formatPrice(amount: number): string {
    return this.currencyPipe.transform(amount, 'USD', 'symbol') ?? '';
  }
}
```

This is useful when you need to format data outside of a template, such as in a service, a CSV export, or building notification messages.

## Locale Configuration

Most built-in pipes (DatePipe, DecimalPipe, CurrencyPipe, PercentPipe) are locale-aware. The default locale is `en-US`.

### Changing the Default Locale

```typescript
// app.config.ts
import { LOCALE_ID } from '@angular/core';
import { registerLocaleData } from '@angular/common';
import localeFr from '@angular/common/locales/fr';

registerLocaleData(localeFr);

export const appConfig = {
  providers: [
    { provide: LOCALE_ID, useValue: 'fr-FR' }
  ]
};
```

The locale affects:

| Pipe | `en-US` | `de-DE` | `fr-FR` |
|------|---------|---------|---------|
| `number` | `1,234.56` | `1.234,56` | `1 234,56` |
| `currency` | `$1,234.56` | `1.234,56 $` | `1 234,56 $US` |
| `percent` | `42%` | `42 %` | `42 %` |
| `date:'mediumDate'` | `Mar 15, 2026` | `15.03.2026` | `15 mars 2026` |

### Per-Pipe Locale Override

You can override the locale for a specific usage without changing the global setting:

```html
<!-- Force German formatting for this number -->
{{ 1234.56 | number:'1.2-2':'de-DE' }}
<!-- Output: 1.234,56 -->

<!-- Force Japanese yen formatting -->
{{ 1000 | currency:'JPY':'symbol':'1.0-0':'ja-JP' }}
<!-- Output: ¥1,000 -->
```

## Common Mistakes

- **Passing a percentage value to PercentPipe:** `{{ 25 | percent }}` outputs `2,500%`. The pipe expects a decimal fraction like `0.25`.

- **Multiple AsyncPipe subscriptions to the same Observable:** Each `| async` in the template creates a new subscription. Use `@if (obs$ | async; as value)` to subscribe once, or use `shareReplay(1)` on the Observable.

- **Ignoring timezone in DatePipe:** Date strings without timezone information are parsed in the browser's local timezone. `'2026-03-15'` will render as different dates depending on the user's timezone. Always include timezone info or pass a timezone parameter.

- **Using KeyValuePipe on large objects in tight loops:** KeyValuePipe is impure and runs on every change detection cycle. For large data sets, transform the data in the component class instead.

- **Expecting TitleCasePipe to handle hyphens:** `'mary-jane'` becomes `'Mary-jane'`, not `'Mary-Jane'`. TitleCasePipe only recognizes whitespace as word boundaries.

- **Forgetting to register locale data:** Using locale-specific formatting without calling `registerLocaleData()` causes silent fallback to `en-US` or throws at runtime.

- **Using JsonPipe in production templates:** JsonPipe is a debugging tool. Leaving `{{ data | json }}` in production code exposes internal data structures and looks unprofessional.

---

## Best Practices

- **Import only the pipes you need.** Since all built-in pipes are standalone, import them individually rather than importing `CommonModule`.

- **Prefer signals over AsyncPipe for new components.** Use `toSignal()` to convert Observables to signals. This eliminates the need for `AsyncPipe` and integrates with the signal-based architecture.

- **Use `DATE_PIPE_DEFAULT_OPTIONS` for consistent date formatting** across your application instead of repeating format strings in every template.

- **Set the locale globally once** via `LOCALE_ID` and `registerLocaleData()` rather than passing locale parameters to individual pipe usages.

- **Use computed signals instead of SlicePipe** for filtering or slicing large arrays. Signals re-compute only when their dependencies change, while pipes re-evaluate on every change detection cycle.

- **Wrap AsyncPipe results with `@if...as`** to avoid null checks scattered through the template and to prevent multiple subscriptions.

## Real-World Use Cases

### E-Commerce Product Card

```typescript
@Component({
  standalone: true,
  imports: [CurrencyPipe, PercentPipe, UpperCasePipe, DecimalPipe],
  template: `
    <div class="product-card">
      <span class="badge">{{ product.category | uppercase }}</span>
      <h3>{{ product.name }}</h3>
      <div class="pricing">
        @if (product.discount > 0) {
          <span class="original-price">
            {{ product.price | currency:'USD':'symbol' }}
          </span>
          <span class="sale-price">
            {{ discountedPrice | currency:'USD':'symbol':'1.2-2' }}
          </span>
          <span class="discount">
            -{{ product.discount | percent:'1.0-0' }}
          </span>
        } @else {
          <span class="price">
            {{ product.price | currency:'USD':'symbol' }}
          </span>
        }
      </div>
      <p class="rating">
        {{ product.rating | number:'1.1-1' }} / 5.0
        ({{ product.reviewCount | number }} reviews)
      </p>
    </div>
  `
})
export class ProductCardComponent {
  product = input.required<Product>();

  get discountedPrice(): number {
    return this.product().price * (1 - this.product().discount);
  }
}
```

### Activity Feed with Relative Dates

```typescript
@Component({
  standalone: true,
  imports: [DatePipe, TitleCasePipe, SlicePipe, AsyncPipe],
  template: `

      @for (activity of activities | slice:0:5; track activity.id) {
```

---

> **Key Takeaway:** Angular's built-in pipes handle the most common display transformations out of the box. They are pure by default (except KeyValuePipe and AsyncPipe), locale-aware, and standalone-ready. For new Angular 21 projects, prefer signals over AsyncPipe, import pipes individually instead of using CommonModule, and configure locale and date defaults at the application level.
