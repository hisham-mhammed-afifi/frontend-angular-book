# 17.5 AsyncPipe

> AsyncPipe subscribes to an Observable or Promise in the template, returns the latest emitted value, automatically unsubscribes on destroy, and triggers change detection on new emissions. It is the bridge between reactive streams and the template layer.

---

## How AsyncPipe Works Internally

`AsyncPipe` is an impure pipe (`pure: false`) that manages Observable and Promise subscriptions behind the scenes.

```typescript
// Simplified version of what Angular does internally
class AsyncPipe implements OnDestroy, PipeTransform {
  private latestValue: any = null;
  private subscription: Subscription | null = null;
  private currentObservable: Observable<any> | null = null;
  private cdr = inject(ChangeDetectorRef);

  transform(obj: Observable<any> | Promise<any> | null): any {
    if (obj === null || obj === undefined) {
      this.dispose();
      return null;
    }

    if (obj !== this.currentObservable) {
      this.dispose();
      this.currentObservable = obj;
      this.subscription = obj.subscribe(value => {
        this.latestValue = value;
        this.cdr.markForCheck(); // Triggers change detection
      });
    }

    return this.latestValue;
  }

  ngOnDestroy() {
    this.dispose();
  }

  private dispose() {
    this.subscription?.unsubscribe();
    this.subscription = null;
    this.latestValue = null;
    this.currentObservable = null;
  }
}
```

Key behaviors:

1. **First call:** Subscribes to the Observable/Promise and returns `null` (the "initial null").
2. **On emission:** Stores the value, calls `markForCheck()`, and returns the stored value on the next `transform()` call.
3. **Observable reference change:** Unsubscribes from the old Observable, subscribes to the new one.
4. **Component destroy:** Automatically unsubscribes via `ngOnDestroy`.
5. **`markForCheck()`:** This is why AsyncPipe works with `OnPush` change detection. It explicitly marks the component as dirty when new data arrives.

## Basic Usage

### With Observables

```typescript
import { Component, inject } from '@angular/core';
import { AsyncPipe } from '@angular/common';
import { HttpClient } from '@angular/common/http';

@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (users$ | async; as users) {
      @for (user of users; track user.id) {
        <div>{{ user.name }}</div>
      }
    } @else {
      <p>Loading...</p>
    }
  `
})
export class UserListComponent {
  private http = inject(HttpClient);
  users$ = this.http.get<User[]>('/api/users');
}
```

### With Promises

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    <p>{{ greeting | async }}</p>
  `
})
export class GreetingComponent {
  greeting = fetch('/api/greeting').then(r => r.text());
}
```

AsyncPipe detects whether the input is an Observable (has a `subscribe` method) or a Promise (has a `then` method) and handles each appropriately.

### With Subjects

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    <p>Count: {{ count$ | async }}</p>
    <button (click)="increment()">+1</button>
  `
})
export class CounterComponent {
  count$ = new BehaviorSubject<number>(0);

  increment() {
    this.count$.next(this.count$.value + 1);
  }
}
```

With `BehaviorSubject`, the initial null problem is avoided because `BehaviorSubject` emits its current value immediately on subscription.

## The Initial Null Problem

`AsyncPipe` returns `null` before the Observable emits its first value. This is because `transform()` must return a synchronous value, but the Observable has not emitted yet.

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    <!-- user$ hasn't emitted yet: null -->
    <!-- user$ has emitted: { name: 'Alice', age: 30 } -->
    <p>{{ (user$ | async)?.name }}</p>
  `
})
export class ProfileComponent {
  user$ = this.http.get<User>('/api/user/1');
}
```

### Problems Caused by Initial Null

**1. Null reference errors in child components:**

```html
<!-- BAD: Child component receives null before data loads -->
<app-user-card [user]="user$ | async" />
<!-- If user input is required, this throws -->
```

**2. Falsy value confusion:**

```html
<!-- BAD: If the Observable emits 0 or false, this shows "Loading..." -->
@if (count$ | async) {
  <p>Count: {{ count$ | async }}</p>
} @else {
  <p>Loading...</p>
}
```

The `@if` treats `0`, `false`, `''`, and `null` as falsy. You cannot distinguish "not loaded yet" from "loaded a falsy value."

### Solutions to Initial Null

**Solution 1: Use `@if...as` with a wrapper object**

```typescript
users$ = this.http.get<User[]>('/api/users').pipe(
  map(users => ({ data: users }))
);
```

```html
@if (users$ | async; as result) {
  <!-- result.data is always the array, even if empty -->
  @for (user of result.data; track user.id) {
    <div>{{ user.name }}</div>
  }
}
```

**Solution 2: Use `@let` to separate declaration from conditional (Angular 18+)**

```html
@let user = user$ | async;
@if (user) {
  <h1>{{ user.name }}</h1>
} @else {
  <p>Loading...</p>
}
```

`@let` declares a template variable without conditional rendering. The variable is scoped to the current view and its descendants, and Angular keeps it up-to-date automatically.

**Solution 3: Use `startWith()` to provide an initial value**

```typescript
users$ = this.http.get<User[]>('/api/users').pipe(
  startWith([] as User[])
);
```

```html
<!-- No null - starts with empty array -->
@for (user of (users$ | async) ?? []; track user.id) {
  <div>{{ user.name }}</div>
}
```

**Solution 4: Use `toSignal()` with `initialValue` (preferred in Angular 21)**

```typescript
users = toSignal(this.http.get<User[]>('/api/users'), { initialValue: [] });
```

```html
@for (user of users(); track user.id) {
  <div>{{ user.name }}</div>
}
```

## The Multiple Subscription Problem

Each `| async` in the template creates a **separate subscription**. This is the single most common AsyncPipe bug.

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    <!-- BAD: Three separate HTTP requests! -->
    <h1>{{ (user$ | async)?.name }}</h1>
    <p>{{ (user$ | async)?.email }}</p>
    <p>{{ (user$ | async)?.role }}</p>
  `
})
export class ProfileComponent {
  private http = inject(HttpClient);
  // This HTTP call executes THREE TIMES
  user$ = this.http.get<User>('/api/user/1');
}
```

### Solution 1: `@if...as` (Single Subscription)

```html
<!-- GOOD: One subscription, one HTTP request -->
@if (user$ | async; as user) {
  <h1>{{ user.name }}</h1>
  <p>{{ user.email }}</p>
  <p>{{ user.role }}</p>
}
```

The `as` keyword creates a local template variable that holds the emitted value. Only one `| async` means only one subscription.

### Solution 2: `shareReplay(1)`

When you need `| async` in multiple disconnected template locations:

```typescript
user$ = this.http.get<User>('/api/user/1').pipe(
  shareReplay(1)
);
```

```html
<!-- Now safe: shareReplay multicasts the result -->
<header>{{ (user$ | async)?.name }}</header>
<!-- ... other template sections ... -->
<footer>{{ (user$ | async)?.role }}</footer>
```

`shareReplay(1)` multicasts the Observable and replays the last emission to new subscribers. Multiple `| async` pipes share the same underlying HTTP request.

### Solution 3: `toSignal()` (Preferred in Angular 21)

```typescript
user = toSignal(this.http.get<User>('/api/user/1'));
```

```html
<!-- No subscription issues - signals are read, not subscribed to -->
<h1>{{ user()?.name }}</h1>
<p>{{ user()?.email }}</p>
<p>{{ user()?.role }}</p>
```

Signals are read synchronously and do not create subscriptions in the template.

## AsyncPipe with OnPush Change Detection

`AsyncPipe` is one of the key enablers of `OnPush` change detection. When an Observable emits, the pipe calls `ChangeDetectorRef.markForCheck()`, which marks the component and its ancestors as dirty, ensuring they are checked in the next change detection cycle.

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    @if (data$ | async; as data) {
      <div>{{ data.value }}</div>
    }
  `
})
export class OnPushComponent {
  data$ = inject(DataService).getData();
}
```

Without `AsyncPipe` (or `markForCheck()`), an `OnPush` component would not re-render when the Observable emits because `OnPush` skips change detection unless explicitly triggered.

### What Happens Step by Step

1. `OnPush` component is created, change detection runs once.
2. `AsyncPipe` subscribes to `data$`.
3. Component is "clean" -- change detection skips it.
4. `data$` emits a value.
5. `AsyncPipe` stores the value and calls `markForCheck()`.
6. Angular schedules a change detection cycle.
7. The component is now "dirty" -- change detection runs.
8. `AsyncPipe.transform()` is called, returns the stored value.
9. Template updates.

## AsyncPipe with Error Handling

AsyncPipe does not have built-in error recovery. If the Observable errors, the stream is destroyed and the pipe stops updating. As of Angular 21, unhandled errors in AsyncPipe subscriptions are reported to the application's `ErrorHandler` (previously, they could crash SSR processes in zoneless apps). You must still handle errors in the Observable chain to provide fallback behavior.

### Using `catchError`

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (data$ | async; as data) {
      @if (data.error) {
        <div class="error">{{ data.error }}</div>
      } @else {
        <div>{{ data.value }}</div>
      }
    } @else {
      <div class="loading">Loading...</div>
    }
  `
})
export class DataComponent {
  data$ = this.http.get<DataResponse>('/api/data').pipe(
    map(value => ({ value, error: null })),
    catchError(err => of({ value: null, error: err.message }))
  );
}
```

### Using `retry` with `catchError`

```typescript
data$ = this.http.get<Data>('/api/data').pipe(
  retry({ count: 3, delay: 1000 }),
  catchError(err => {
    console.error('All retries failed:', err);
    return of(null); // Emit null as fallback
  })
);
```

```html
@if (data$ | async; as data) {
  <div>{{ data.title }}</div>
} @else {
  <div>Loading or failed...</div>
}
```

### Typed Error State Pattern

```typescript
interface AsyncState<T> {
  loading: boolean;
  data: T | null;
  error: string | null;
}

@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (state$ | async; as state) {
      @if (state.loading) {
        <app-spinner />
      } @else if (state.error) {
        <app-error [message]="state.error" (retry)="reload()" />
      } @else if (state.data) {
        <app-data-view [data]="state.data" />
      }
    }
  `
})
export class SmartComponent {
  private trigger$ = new Subject<void>();

  state$ = this.trigger$.pipe(
    startWith(undefined),
    switchMap(() => this.http.get<Data>('/api/data').pipe(
      map(data => ({ loading: false, data, error: null } as AsyncState<Data>)),
      catchError(err => of({ loading: false, data: null, error: err.message } as AsyncState<Data>)),
      startWith({ loading: true, data: null, error: null } as AsyncState<Data>)
    ))
  );

  reload() {
    this.trigger$.next();
  }
}
```

## AsyncPipe with RxJS Operators

### switchMap for Dependent Data

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (userDetails$ | async; as details) {
      <h2>{{ details.user.name }}</h2>
      <p>Orders: {{ details.orders.length }}</p>
    }
  `
})
export class UserDetailsComponent {
  userId = input.required<string>();

  userDetails$ = toObservable(this.userId).pipe(
    switchMap(id => forkJoin({
      user: this.http.get<User>(`/api/users/${id}`),
      orders: this.http.get<Order[]>(`/api/users/${id}/orders`)
    }))
  );
}
```

### combineLatest for Multiple Streams

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (vm$ | async; as vm) {
      <h1>{{ vm.title }}</h1>
      @for (item of vm.filteredItems; track item.id) {
        <div [class.selected]="item.id === vm.selectedId">
          {{ item.name }}
        </div>
      }
    }
  `
})
export class DashboardComponent {
  private items$ = this.itemService.getItems();
  private filter$ = this.filterService.activeFilter$;
  private selected$ = this.selectionService.selectedId$;

  // Single "view model" observable for the template
  vm$ = combineLatest({
    items: this.items$,
    filter: this.filter$,
    selectedId: this.selected$
  }).pipe(
    map(({ items, filter, selectedId }) => ({
      title: `Items (${filter})`,
      filteredItems: items.filter(i => i.category === filter),
      selectedId
    })),
    shareReplay(1)
  );
}
```

This "view model" pattern combines multiple streams into a single Observable, requiring only one `| async` in the template.

## Automatic Unsubscription

AsyncPipe implements `OnDestroy` and unsubscribes from the Observable when the component is destroyed. This prevents memory leaks without any manual cleanup.

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `<p>{{ ticker$ | async }}</p>`
})
export class TickerComponent {
  // No need to manually unsubscribe. AsyncPipe handles it.
  ticker$ = interval(1000).pipe(
    map(n => `Tick ${n}`)
  );
}
```

### When AsyncPipe Unsubscribes

1. **Component is destroyed:** `ngOnDestroy` disposes the subscription.
2. **Observable reference changes:** When a new Observable is passed (e.g., due to input change), the pipe unsubscribes from the old one and subscribes to the new one.
3. **`null` or `undefined` is passed:** The pipe disposes the current subscription.

### Conditional Rendering and Subscriptions

```html
@if (showChart) {
  <!-- AsyncPipe subscribes when showChart is true -->
  <app-chart [data]="chartData$ | async" />
}
<!-- AsyncPipe unsubscribes when showChart becomes false -->
<!-- because the component/element is destroyed -->
```

## AsyncPipe vs toSignal()

In Angular 21, `toSignal()` is often the preferred alternative to `AsyncPipe`. Here is a detailed comparison:

| Aspect | AsyncPipe | toSignal() |
|--------|-----------|------------|
| Import | `AsyncPipe` from `@angular/common` | `toSignal` from `@angular/core/rxjs-interop` |
| Initial value | `null` | `undefined` (or custom `initialValue`) |
| Multiple reads | Each `\| async` = new subscription | Signal reads are free, no subscriptions |
| OnPush compat | Yes, calls `markForCheck()` | Yes, signals trigger change detection |
| Error handling | Must use `catchError` in stream | Must use `catchError` in stream |
| Unsubscription | Auto on destroy | Auto on destroy (injection context) |
| Use in computed | Not possible | Yes, signals compose naturally |
| Template syntax | `value$ \| async` | `value()` |
| Zoneless compat | Yes (calls `markForCheck()`) | Yes (native signal scheduling) |

### When to Use AsyncPipe

- Existing codebases with established Observable patterns
- When the Observable is only consumed in the template
- When you want the subscription lifecycle tied to the template element (conditional rendering)

### When to Use toSignal()

- New Angular 21 projects
- When the value is needed in `computed()` or `effect()`
- When the same value is read in multiple places in the template
- When you want cleaner template syntax without null handling
- When combining with other signals

### Migration Example

```typescript
// BEFORE: AsyncPipe
@Component({
  imports: [AsyncPipe],
  template: `
    @if (user$ | async; as user) {
      <h1>{{ user.name }}</h1>
      <p>{{ user.email }}</p>
    }
  `
})
export class ProfileComponent {
  user$ = this.http.get<User>('/api/user/1');
}

// AFTER: toSignal
@Component({
  template: `
    @if (user(); as user) {
      <h1>{{ user.name }}</h1>
      <p>{{ user.email }}</p>
    }
  `
})
export class ProfileComponent {
  private http = inject(HttpClient);
  user = toSignal(this.http.get<User>('/api/user/1'));
}
```

### toSignal Options

```typescript
// Default: returns Signal<T | undefined>
const data = toSignal(data$);

// With initial value: returns Signal<T>
const data = toSignal(data$, { initialValue: [] });

// For synchronous observables (BehaviorSubject):
const data = toSignal(data$, { requireSync: true });
// Returns Signal<T> (not undefined) because the Observable emits synchronously

// Manual cleanup (outside injection context):
const data = toSignal(data$, { manualCleanup: true });
// You must unsubscribe manually
```

## AsyncPipe in Zoneless Angular

In zoneless Angular (default in Angular 21), `AsyncPipe` continues to work because it calls `markForCheck()` internally. This explicitly notifies Angular's change detection scheduler, which works independently of Zone.js.

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  // No Zone.js needed - AsyncPipe's markForCheck() triggers rendering
  template: `
    @if (data$ | async; as data) {
      <p>{{ data.value }}</p>
    }
  `
})
export class ZonelessComponent {
  data$ = this.http.get<Data>('/api/data');
}
```

However, in zoneless applications, `toSignal()` is more idiomatic because signals are the primary change detection trigger.

## AsyncPipe with Structural Patterns

### Loading/Error/Data Pattern

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @if (vm$ | async; as vm) {
      @if (vm.loading) {
        <div class="skeleton" aria-busy="true">
          <div class="skeleton-line"></div>
          <div class="skeleton-line"></div>
        </div>
      } @else if (vm.error) {
        <div class="error-banner" role="alert">
          <p>{{ vm.error }}</p>
          <button (click)="retry()">Try Again</button>
        </div>
      } @else {
        <div class="content">
          @for (item of vm.data; track item.id) {
            <app-item-card [item]="item" />
          }
        </div>
      }
    }
  `
})
export class ItemListComponent {
  private reload$ = new BehaviorSubject<void>(undefined);

  vm$ = this.reload$.pipe(
    switchMap(() => this.loadItems()),
    shareReplay(1)
  );

  private loadItems() {
    return this.http.get<Item[]>('/api/items').pipe(
      map(data => ({ loading: false, data, error: null })),
      catchError(err => of({ loading: false, data: [] as Item[], error: err.message })),
      startWith({ loading: true, data: [] as Item[], error: null })
    );
  }

  retry() {
    this.reload$.next();
  }
}
```

### Polling with AsyncPipe

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, DatePipe],
  template: `
    @if (status$ | async; as status) {
      <div class="status-bar">
        <span [class]="'badge ' + status.health">{{ status.health }}</span>
        <span>Latency: {{ status.latency }}ms</span>
        <span>Updated: {{ status.timestamp | date:'mediumTime' }}</span>
      </div>
    }
  `
})
export class StatusBarComponent {
  status$ = timer(0, 30_000).pipe(
    switchMap(() => this.http.get<Status>('/api/health')),
    catchError(err => of({ health: 'error', latency: -1, timestamp: Date.now() })),
    shareReplay(1)
  );
  // Polls every 30 seconds, starts immediately
  // AsyncPipe unsubscribes when component is destroyed, stopping the polling
}
```

## Common Mistakes

- **Multiple `| async` on the same Observable causing duplicate subscriptions.** Each `| async` creates a new subscription. Use `@if...as` for a single subscription, or `shareReplay(1)` when multiple separate usages are unavoidable.

- **Not handling the initial null emission.** Before the Observable emits, `| async` returns `null`. This breaks child components expecting non-null inputs and causes template errors with strict null checks. Use `@if...as`, `startWith()`, or `toSignal()` with `initialValue`.

- **Using `@if (obs$ | async)` with Observables that emit falsy values (0, false, '').** The `@if` cannot distinguish "not loaded yet" from "loaded a falsy value." Wrap the emission in an object: `.pipe(map(v => ({ value: v })))`.

- **Forgetting `catchError` in the Observable chain.** If the Observable errors, the pipe silently stops working and the error goes to the global error handler. Always add error handling in the stream.

- **Recreating the Observable on every change detection cycle.** Defining the Observable inline in the template or via a getter creates a new Observable each time, causing the pipe to unsubscribe and resubscribe continuously:

```typescript
// BAD: Getter creates new Observable each time
get user$() {
  return this.http.get<User>('/api/user/1');
}

// GOOD: Store as a class field
user$ = this.http.get<User>('/api/user/1');
```

- **Using AsyncPipe without OnPush.** While it works with default change detection, you lose the performance benefit. `AsyncPipe` was designed to work with `OnPush`.

---

## Best Practices

- **Use `@if (obs$ | async; as value)` to subscribe once** and create a template variable. This prevents multiple subscriptions and handles the null state cleanly.

- **Prefer `toSignal()` in new Angular 21 code.** It provides better TypeScript support, cleaner template syntax, composability with other signals, and avoids the initial null problem when using `initialValue`.

- **Combine multiple Observables into a single view model** using `combineLatest` or `forkJoin`, then use one `| async` in the template. This keeps the template simple and avoids subscription management.

- **Always pair AsyncPipe with OnPush** for maximum performance. AsyncPipe's `markForCheck()` integrates seamlessly with OnPush.

- **Use `shareReplay(1)` when multiple `| async` bindings are unavoidable.** This ensures a single subscription to the source Observable.

- **Handle errors with `catchError` in the stream**, mapping errors to a state object that the template can display. Do not let errors propagate unhandled.

- **Store Observables as class fields, not getters or methods.** A getter would create a new Observable on every access, causing infinite subscribe/unsubscribe cycles.

## Real-World Use Cases

### Real-Time Notifications with AsyncPipe

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, DatePipe, SlicePipe],
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    @if (notifications$ | async; as notifications) {
      <div class="notification-panel">
        <h3>Notifications ({{ notifications.length }})</h3>
        @for (notification of notifications | slice:0:5; track notification.id) {
          <div class="notification" [class.unread]="!notification.read">
            <p>{{ notification.message }}</p>
            <time>{{ notification.createdAt | date:'shortTime' }}</time>
          </div>
        }
        @if (notifications.length > 5) {
          <a routerLink="/notifications">View all</a>
        }
      </div>
    }
  `
})
export class NotificationPanelComponent {
  private notificationService = inject(NotificationService);

  notifications$ = this.notificationService.notifications$.pipe(
    catchError(() => of([] as Notification[])),
    shareReplay(1)
  );
}
```

### Search Typeahead with AsyncPipe

```typescript
@Component({
  standalone: true,
  imports: [AsyncPipe, ReactiveFormsModule],
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    <input [formControl]="searchControl" placeholder="Search...">

    @if (results$ | async; as results) {
      @if (results.length > 0) {
        <ul class="suggestions">
          @for (result of results; track result.id) {
            <li (click)="select(result)">{{ result.name }}</li>
          }
        </ul>
      } @else if (searchControl.value) {
        <p class="no-results">No results found</p>
      }
    }
  `
})
export class SearchComponent {
  private http = inject(HttpClient);
  searchControl = new FormControl('');

  results$ = this.searchControl.valueChanges.pipe(
    debounceTime(300),
    distinctUntilChanged(),
    switchMap(term => {
      if (!term || term.length < 2) return of([]);
      return this.http.get<SearchResult[]>(`/api/search?q=${encodeURIComponent(term)}`).pipe(
        catchError(() => of([]))
      );
    }),
    shareReplay(1)
  );

  select(result: SearchResult) {
    // Handle selection
  }
}
```

```typescript
@Component({

  changeDetection: ChangeDetectionStrategy.OnPush,

        </div>
      </div>

      <div class="stats-grid">
        @for (_ of [1, 2, 3]; track $_) {
          <div class="stat-card skeleton"></div>
        }
      </div>
    }
  `
})
export class DashboardComponent {
  private http = inject(HttpClient);

  dashboard$ = forkJoin({
    revenue: this.http.get<number>('/api/stats/revenue'),
    userCount: this.http.get<number>('/api/stats/users'),
    conversionRate: this.http.get<number>('/api/stats/conversion')
  }).pipe(
    catchError(() => of({ revenue: 0, userCount: 0, conversionRate: 0 })),
    shareReplay(1)
  );
}
```

---

> **Key Takeaway:** AsyncPipe bridges Observables and templates by handling subscription, unsubscription, and change detection automatically. In Angular 21, `toSignal()` is the preferred alternative for new code because it avoids the initial null problem, prevents multiple subscriptions, and composes naturally with the signal-based architecture. Use AsyncPipe when working with existing Observable-heavy codebases or when you need the subscription lifecycle tied to template element visibility.
